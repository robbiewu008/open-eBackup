Configuring the DNS Service

After connecting the OceanProtect to the DNS server, external domain names can be resolved and accessed.

Prerequisites

  • The DNS server has been configured and is running properly.
  • TCP/UDP port 53 between the OceanProtect and DNS server is enabled.

Context

  • A DNS server is used to resolve and access external domain names.
  • If you want to configure a standby DNS server, keep the domain names of the active and standby servers consistent.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  2. In the Local Cluster Nodes area of the Backup Cluster tab page, click the node name.
  3. In the Node detail dialog box that is displayed, click Open the device management platform to go to DeviceManager.
  4. Choose Settings > Basic Information > DNS Service.
  5. Click Modify in the upper right corner of the Management DNS area to configure the DNS service for the management plane.
  6. Set Active DNS IP Address.

    1. (Optional) Set Standby DNS IP Address 1.
    2. (Optional) Set Standby DNS IP Address 2. Set Standby DNS IP Address 1 first and then Standby DNS IP Address 2.
    3. (Optional) Test the connectivity between the DNS server and the storage system.
      • You can click Test next to a DNS IP address to test its availability.
      • You can click Test All to test the connectivity between the DNS server and storage system.
    4. Click Save.
