Managing Databases

The product supports database operations such as resource authorization, reclamation, modification, and deletion.

You need to log in to the management page, choose Protection > Applications > SAP HANA, and find the target database on the Databases tab page.

Table 1 describes the related operations.

Table 1 Operations related to database protection plans

Operation

Description

Navigation Path

Modifying protection

Scenario

You can modify the protection plan to re-associate a database with an SLA. After the modification, the next backup will be performed based on the new protection plan.

Note

The modification does not affect ongoing protection jobs. The modified protection plan will apply to the next protection period.

In the row of the target database, choose More > Modify Protection.

Removing protection

Scenario

This operation enables you to remove protection for a database when it no longer needs to be protected. After removal, the system cannot execute protection jobs for the database. To protect the database, you need to associate an SLA with the database again.

Note

When a protection job of a database is running, protection cannot be removed.

In the row of the target database, choose More > Removing Protection.

Activating protection

Scenario

This operation allows you to activate the protection plan of a database again if it is disabled. After the protection plan is activated, the system executes backup, archive, or replication jobs for the database based on the plan.

In the row of the target database, choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable a protection plan when periodic protection on a database is not required. After the protection plan is disabled, the system does not automatically back up, archive, or replicate database data.

Note

Disabling a protection plan does not affect running protection jobs.

In the row of the target database, choose More > Disable Protection.

Performing manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target database, choose More > Manual Backup.

Testing connectivity

Scenario

This operation enables you to test the connectivity between the product and a database.

In the row of the target database, choose More > Test Connectivity.

Modification

Scenario

After the database username or password is changed, you can perform this operation to modify the database registration information.

In the row of the target database, choose More > Modify.

Deletion

Scenario

This operation enables you to delete a database if it no longer needs to be protected.

Note

A database that has been associated with an SLA cannot be deleted.

In the row of the target database, choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for restoring the SAP HANA database is the same as that in Restoration.
