Restoring FusionCompute VMs

This section describes how to restore a VM that has been backed up to the original location or a new location.

Context

The product allows you to restore VMs to the original location or a new location using backup copies or replication copies (data cannot be restored to the original location).

Procedure

  1. Choose Explore > Copy Data > Virtualization > FusionCompute.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    On the Resources tab page, search for the VM to be restored by name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. In the row where the copy resides, choose More > Restore.
  5. Restore a VM to the original location or a new location.

    • Restore the VM to the original location.
      1. Select Original location for restoring the VM.
      2. Target Compute Location is the original location of the VM by default.
      3. Set whether to Rename VM. After this function is enabled, you can enter a new name.
      4. Select an agent host used for restoration using copies.

        If the agent host is not selected, the system selects one or multiple agent hosts from the agent hosts configured during resource registration to perform restoration.

      5. Select whether to automatically start the VM after the restoration.
      6. Select whether to perform copy verification before restoration.

        If this option is enabled, the integrity of a copy is verified before restoration using the copy. Copy verification affects the restoration performance and is disabled by default. If no copy verification file is generated, this option cannot be enabled.

      7. Click OK.
    • Restore the VM to a new location.
      1. Select New location for restoring the VM.
      2. Set information about a new location to which the VM is restored.

        Table 1 describes the related parameters.

        Table 1 New location information

        Parameter

        Description

        Target Compute Location

        Compute resources used by the VM after being restored.

        Select a cluster or host as the target.

        Storage Location

        Datastore to which the VM configuration files and disk data are restored.

        • Same Datastore

          All disks of a VM are restored to the same datastore.

        • Different Datastores
          VM Disk Files: datastore where the VM disk data is stored.
          NOTE:

          To improve data security, you can store the VM configuration files on an independent datastore and disk data on another datastore.
      3. Set whether to Rename VM. After this function is enabled, you can enter a new name.
      4. Select an agent host used for restoration using copies.

        If the agent host is not selected, the system selects one or multiple agent hosts from the agent hosts configured during resource registration to perform restoration.

      5. Select whether to automatically start the VM after the restoration.
      6. Select whether to perform copy verification before restoration.

        If this option is enabled, the integrity of a copy is verified before restoration using the copy. Copy verification affects the restoration performance and is disabled by default. If no copy verification file is generated, this option cannot be enabled.

  6. Click OK.
