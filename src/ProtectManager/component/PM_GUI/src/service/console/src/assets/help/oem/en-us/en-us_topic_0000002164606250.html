The Number of VM Disks Increases After Data Disks Are Restored to New Locations

Symptom

After data disks are restored to new locations, the number of VM disks in the new locations are greater than that before the restoration.

Possible Cause

During disk restoration, the product checks whether the original disks exist on the target VM. If the original disks do not exist, the product creates disks for restoration. The system determines that disks need to be created in the following scenarios:
  • The original disks have been deleted.
  • Capacities of the original disks have been expanded.
  • Datastores of the original disks have been replaced.
  • IDs of the original disks have been changed.
  • The original disks are not on the target VM. The system determines that the original disks exist on the target VM in the following scenarios:
    • For versions earlier than FusionCompute 6.3:
      • If the slot IDs of a copy disk and the target disk are both 1 and the disk type is IDE, the copy disk exists on the target VM.
      • If the Uniform Resource Name (URN), for example, urn:sites:1:volumes:78 of a copy disk is the same as that of the target disk and no slot ID of the disks is 1, the copy disk exists on the target VM.
    • For FusionCompute 6.3 and later versions:
      • If a copy disk and the target disk are both system disks, the copy disk exists on the target VM.
      • If the URN, for example, urn:sites:1:volumes:78 of a copy disk is the same as that of the target disk and none of them is a system disk, the copy disk exists on the target VM.

Troubleshooting

The disks restored to new locations are suffixed with _RESTORE. You need to add partitions to the restored disks and mount the partitions again. To detach unnecessary disks, refer to the following operations. The following uses FusionCompute 8.1.1 as an example.

  1. Log in to FusionCompute using a browser.

    Login address: https://Floating IP address of the VRM node of FusionCompute:8443

  2. Click on the navigation pane on the left to go to the Resource Pools page.
  3. In the navigation tree on the left of the Resource Pools page, choose the target VM.
  4. On the VM page, click the Configuration tab.
  5. In the navigation tree on the left of the Configuration page, choose Hardware > Disk.
  6. In the row of the new disk, choose More > Detach.
  7. In the displayed dialog box, confirm the information.
