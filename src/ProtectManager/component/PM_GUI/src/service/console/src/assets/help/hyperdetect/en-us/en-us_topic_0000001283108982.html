Enabling File Interception

This section describes how to enable file interception.

Prerequisites

NFS or CIFS sharing has been enabled for the file system.

Precautions

File name extension filtering rules do not take effect on the secondary file system of a HyperMetro domain or remote replication vStore pair.

Procedure

  1. On the navigation pane, choose Explore > Ransomware Protection > Ransomware Detection.
  1. On the Detection Settings tab, locate the row that contains the vStore for which you want to enable file interception and choose More > Enable File Interception.

    • You can also select multiple vStores for which you want to enable file interception and click Enable File Interception above the vStore list.
    • If file interception is not required, choose More > Disable File Interception. Alternatively, you can select multiple vStores for which you want to disable file interception and click Disable File Interception above the vStore list.

  2. Click Number of File Name Extension Filtering Rules to view the File Name Extension Filtering Rule association status. Table 1 describes the association states. If Association Status is Applied, the File Name Extension Filtering Rule has taken effect.

    Table 1 File Name Extension Filtering Rule association status

    Association Status

    Description

    Associated

    The File Name Extension Filtering Rule has been associated with a vStore.

    Applied

    The File Name Extension Filtering Rule has been applied to the file system of the vStore. Create, open, and rename operations on files with the same extension in the file system are not allowed.

    NOTE:

    When accessing a file system through CIFS sharing, you must open the file before deleting it. Therefore, the deletion operation on files with the same extension in the file system is also intercepted.

    Applying

    The File Name Extension Filtering Rule is being applied to the file system of the vStore.

    Disassociating

    The File Name Extension Filtering Rule is being disassociated from the vStore. Application to the file system will also be canceled.

    Canceling Application

    Application of the File Name Extension Filtering Rule to the file system of the vStore is being canceled.

    Exception

    The File Name Extension Filtering Rule applied to the vStore does not take effect for some file systems.
