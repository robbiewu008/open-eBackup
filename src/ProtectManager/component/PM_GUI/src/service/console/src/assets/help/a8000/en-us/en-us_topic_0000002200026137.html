Querying Information About the Nutanix Virtualization Environment

After the Nutanix virtualization environment is registered with the OceanProtect, you can view details about Nutanix clusters, hosts, VMs, or VM groups on the management page.

Procedure

  1. Choose Protection > Virtualization > Nutanix.
  2. On the left of the page, select the Nutanix virtualization environment to be viewed and choose More > Environment Info to view the basic information of the Nutanix virtualization environment and clients.
  3. On the Clusters tab page, view cluster information.

    1. Table 1 describes the cluster details.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 Cluster information

      Parameter

      Description

      Name

      Cluster name.

      SLAs

      Path to a cluster.

      SLA Compliance

      If the entire cluster is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

      Protection Status
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status
      Whether the current resource is protected.
      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the cluster name to view the basic information about the cluster.
      • Basic cluster information

        The cluster name and SLA information are displayed.

      • Host and VM information

        The host list contains the name, location, status, SLA, protection status, and tag.

        The VM list contains the name, location, status, mark, SLA, SLA compliance, protection status, VM group, and tag.

        Click More on the right of the host or VM name to protect or manually back up the host or VM or to modify, remove, activate or disable the protection for the host or VM.

  4. On the Hosts tab page, view host information.

    1. Table 2 describes the host information.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 2 Host information

      Parameter

      Description

      Name

      Host name.

      Location

      Path to a host.

      Status

      Host status, which can be Normal or Exception.

      SLAs

      If an SLA is associated with the resource, the SLA name is displayed. Otherwise, -- is displayed.

      Protection Status
      Whether the current resource is protected.
      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the host name to view the basic information of the host and the VM list.
      • Basic host information

        The host name and SLA information are displayed.

      • VM information

        The VM list contains the name, location, status, SLA, protection status, and tag.

        Click More on the right of the VM name to protect or manually back up the VM or to modify, remove, activate, or disable the protection for the VM.

  5. On the VMs tab page, view VM information.

    1. Table 3 describes the VM information.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 3 VM information

      Parameter

      Description

      Name

      Name of a VM.

      Location

      Path to a VM.

      Status

      VM status, which can be On, Off, or Exception.

      Mark

      Mark of a VM.

      If you have set a mark for a VM in the Nutanix virtualization environment, the VM mark is displayed. Otherwise, -- is displayed.

      You can filter VMs by mark.

      SLAs

      If an SLA is associated with the resource, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status
      Whether the current resource is protected.
      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.

      VM Group

      Name of the VM group to which the VM belongs.

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the VM name to view the overall VM information, copy information, and related jobs.
      • Overview

        You can view information of a VM, such as the VM name, SLA information, job execution time, total number of copies, and disk information.

      • Copy Data

        You can search for copies by year, month, or day.

        If is displayed below the time, copies exist at that time.

      • Jobs

        All jobs related to a VM, such as backup, restoration, archive, and live mount jobs, are displayed.

  6. On the VM Groups tab page, view VM group information.
