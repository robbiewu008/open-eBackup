Managing FusionOne Compute Copies

This section describes the operations that you can perform on copy data in the OceanProtect.

You need to log in to the management page and choose Explore > Copy Data > Virtualization > FusionOne Compute to enter the FusionOne Compute copy page.

  • Click Resources to perform operations by resource. Operations on this page are performed on replication copies of the current resource. Perform the following operations in the cluster where the replication copies reside.

    Table 1 describes the related operations.

    Table 1 Operations related to replication copies

    Operation

    Description

    Navigation Path

    Creating protection for replication copies

    Scenario

    This operation enables you to periodically replicate or archive replication copies.

    Note

    If no replication copy is generated for the selected resource, you cannot associate an SLA with the resource for protection.

    In the row of the target resource, choose More > Protect and associate a replication SLA or an archive SLA to replicate or archive replication copies.

    Modifying the SLA associated with replication copies

    Scenario

    This operation enables you to modify the SLA associated with replication copies.

    In the row of the target resource, choose More > Modify Protection and associate a new SLA with replication copies.

    Removing protection for replication copies

    Scenario

    You can remove the protection plan if the replication copies no longer need to be protected. After removal, the system cannot protect these replication copies.

    In the row of the target resource, choose More > Remove Protection.

    Activating protection for replication copies

    Scenario

    This operation enables you to activate a disabled protection plan for replication copies. After the protection plan is activated, the system protects replication copies based on the protection plan.

    In the row of the target resource, choose More > Activate Protection.

    Disabling protection for replication copies

    Scenario

    You can disable the protection plan if the replication copies no longer require periodic protection. After the protection plan is disabled, the system does not execute automatic protection jobs for these replication copies.

    In the row of the target resource, choose More > Disable Protection.

    Performing manual replication on replication copies

    Scenario

    This operation enables you to perform a replication on replication copies immediately.

    Note

    Before performing this operation, ensure that an SLA based on replication copies has been created and used to create protection for the corresponding resource.

    In the row of the target resource, choose More > Manual Backup.

    Modifying the owner

    Scenario

    This operation enables you to change the owner for all generated copies of a resource and switch the quota occupied by the copies to a new owner.

    Note
    • Only the system administrator has the permission to change owners.
    • You can only change the owner to a user of the Data Protection Administrator or custom role.
    • After the owner is changed, view the ownerships of the corresponding copies on the Copies tab page.
    • After the owner is changed for a resource, no information about the resource will be displayed under the original owner.
    • After the owner is changed for a resource on the secondary cluster for replication, you need to change the value of Replication Target User of the replication SLA associated with the resource on the primary cluster for replication to the new owner. Otherwise, the replication will fail.

    In the row of the target resource, choose More > Modify Owner.
  • Click Copy to perform operations by copy.
    Table 2 describes the related operations.
    Table 2 Operations related to FusionOne Compute copies

    Operation

    Description

    Navigation Path

    Performing file-level unmounting

    Scenario

    The system automatically unmounts a successfully created file-level mount point one hour after it is created. This operation enables you to manually unmount a file-level mount point.

    In the row of the target copy, choose More > File-level Unmounting.

    Performing manual archiving

    Scenario

    This operation enables you to manually archive copies to the archive storage.

    Note
    • Only full backup copies and forever incremental (synthetic full) backup copies, as well as their replication copies and cascaded replication copies can be manually archived.
    • When the OceanProtect starts to execute a manual copy archiving job, the job will fail to be executed if an interface module undergoing the job is powered off. In this case, power on the interface module and perform manual archiving again.

    In the row of the target copy, choose More > Manual Archive.

    Performing copy verification

    Scenario

    This operation enables you to verify the integrity of a copy.

    Note

    Archive copies do not support copy verification.

    In the row of the target copy, choose More > Verify Copy.

    Modifying a copy retention policy

    Scenario

    This operation enables you to modify the copy retention period.

    Note

    The new copy retention period is calculated from the time when the modification takes effect.

    In the row of the target copy, choose More > Modify Retention Policy.

    You can set the retention period to be permanent or to xx days, weeks, months, or years.
    Performing WORM settings
    NOTE:

    Only the OceanProtect X series backup appliances and OceanProtect E1000 support this function.

    Scenario

    This operation enables you to set a copy as a WORM copy or change the WORM validity period.

    Note
    • WORM copies cannot be deleted or their validity periods cannot be shortened.
    • The custom WORM validity period cannot be longer than the copy retention period.
    • A WORM copy cannot be changed to a non-WORM copy within its validity period.
    • If you select Same as the copy retention period for WORM Validity Period and choose to retain copies permanently, the actual WORM validity period is 20 years for the copies in snapshot format and 70 years for the copies in directory format. This is because copies in snapshot format can be retained for up to 20 years and those in directory format can be retained for up to 70 years on the backup storage.

    In the row of the target copy, choose More > WORM Setting.

    Deleting a copy

    Scenario

    This operation enables you to delete an unnecessary or unavailable copy to release storage space.

    Note
    • Before deleting a copy, ensure that the copy is no longer used for restoration.
    • Copies in the Invalid state can be deleted, and indexed or non-indexed copies in the Normal state can also be deleted. Copies in other states cannot be deleted.
    • If a resource has multiple copies, the last copy cannot be deleted.
    • If a resource has multiple copies and only one full replication copy, the full replication copy cannot be deleted.
    • Copies undergoing ransomware detection cannot be deleted.
    In the row of the target copy, choose More > Delete. Alternatively, select the copies to be deleted and click Delete above the table.
    NOTE:

    To forcibly delete the copy data in the database after copy deletion fails, select Forcibly delete the copy data upon deletion failure in the displayed dialog box.

    Manually creating an index

    Scenario

    If you want to search for files or directories in a copy and Auto Indexing of Backup Copies is disabled during protection creation, you can manually create an index.

    Note
    • Reverse replication copies and cascaded replication copies do not support index creation.
    • Copies archived to tapes do not support index creation.

    In the row of the target copy, choose More > Manually Create Index.
    Performing reverse replication or cascaded replication on replication copies
    NOTE:

    Only the OceanProtect X series backup appliances and OceanProtect E1000 support this function.

    Scenario

    This operation enabled you to replicate a replication copy to the source end for data restoration or replicate a replication copy to another cluster for remote DR.

    Note

    Before performing reverse replication, delete the backup copies corresponding to the replication copies from the source end. Otherwise, the replication will fail.
    In the row of the target replication copy, choose More > Replicate. Configure copy replication parameters.
    1. Select Replication Target Cluster.
      • If Replication Target Cluster is the source cluster, reverse replication is performed.
      • If Replication Target Cluster is another cluster, cascaded replication is performed.
    2. Select Replication Target User.
    3. Enter the password of the replication target user and click Authenticate.
    4. Set Specify Target Location for replication.
    5. Set the copy retention period.
    6. Enable or disable deduplication and compression.

      You are advised to enable these functions to improve replication efficiency.

    7. Enable or disable the job failure alarm function.

    Note: Operations for Restore, File-level Restoration, and Disk Restoration are similar to those described in Restoring FusionOne Compute VMs, Restoring Files on a FusionOne Compute VM, and Restoring FusionOne Compute VM Disks.
