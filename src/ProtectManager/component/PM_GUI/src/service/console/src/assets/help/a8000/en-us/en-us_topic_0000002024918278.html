Managing Snapshots

Symptom

Failed to enable the snapshot function.

Possible Cause

The directory for which the snapshot function is to be enabled may contain subdirectories with the snapshot function enabled.

Procedure

  1. Before enabling the snapshot function, log in to the HDFS agent host as user root using PuTTY.
  2. Run the following command to execute the environment variable:

    source Installation directory/bigdata_env

  3. Run the following command to switch to the HDFS database user:

    kinit HDFS database user
    Figure 1 Login to the HDFS database user

  4. Run the following commands to manage snapshots.

    • Query all snapshots in a directory:
      hdfsdfs -ls Directory.snapshot
      Figure 2 Querying all snapshots in the root directory (/)
    • Delete snapshots from a directory:
      hdf sdfs -deleteSnapshot Directory Snapshot name
      Figure 3 Deleting snapshots in the root directory (/)
    • Query the directories for which the snapshot function is enabled and the permissions on the directories:
      hdfs lsSnapshottableDir
      Figure 4 Querying the directories for which the snapshot function is enabled and the permissions on the directories
    • Disable the snapshot function for a directory:
      hdfs dfsadmin -disallowSnapshot Directory
      Figure 5 Disabling the snapshot function of the root directory (/)

      Before disabling the snapshot function for a directory, you must delete all snapshots in the directory.
