Managing Detection Models

The ransomware protection function manages the signature database through detection models. You can view, update, activate, or delete detection models.

You need to log in to the OceanCyber, choose Data Security > Ransomware Protection > Detection model, and find the detection model to be operated. Table 1 describes the related operations.

Table 1 Operations related to detection models

Job

Description

Navigation Path

Viewing a detection model

Scenario

View the basic information and activation status of a detection model.

In the detection model list, view the version, import time, signature information, description, and activation status.

Updating a detection model

Scenario

The latest detection model is preconfigured in the system when the software version is released.

You can import the latest detection model of OceanCyber.

  • Querying the update information about a detection model: Visit the following URLs to check whether the detection model is updated:
    • Enterprise users: Click here.
    • Carrier website: Click here.
  • Automatically obtaining the update information about the detection model: The DME IQ Client needs to be deployed on the live network and integrate with the OceanCyber 300 Data Security Appliance.
    1. Add the OceanCyber 300 Data Security Appliance to DME IQ (24.0.0 or later). For details about how to add a device, see "Adding a Device" in the DME IQ Client 24.0.x User Guide.
    2. If OceanCyber releases a detection model whose version is later than the latest version in the model list of the current OceanCyber Appliance and the detection model matches the current OceanCyber Appliance, you will receive an email notification. You can obtain the detection model of the new version from the download URL in the email.
NOTE:
  • If DME IQ is not installed, obtain it from the following URLs:
    • Enterprise users: Click here.
    • Carrier website: Click here.
  • Import or reactivate a detection model of an earlier version if it works better in your scenario.

Note

  • A detection model is automatically activated after being imported. The previously activated one enters the Inactivated state.
  • The import does not affect ongoing detection jobs. Ongoing detection jobs still use the earlier detection model. The new detection model will be used in the next job.

On the Detection model page, click Add.

Activating a detection model

Scenario

To use an inactive detection model for detection, activate it first.

Note

  • Only detection models in the Inactivated status can be activated.
  • Only one detection model in the Activated status exists in the system. The activated detection model is automatically deactivated after you activate or import a new detection model.
  • If a previously activated detection model is being used for ransomware detection, the newly activated or imported detection model enters the Activated state but does not affect the ongoing detection job and will take effect in new jobs.

Choose More > Activate in the Operation column on the right of the detection model.

Deleting a detection model

Scenario

Detection models that are no longer used for ransomware detection can be deleted.

Note

Only detection models in the Inactivated status can be deleted.

Choose More > Delete in the Operation column on the right of the detection model.
