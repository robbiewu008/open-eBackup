Step 4: Registering the Database of a PostgreSQL Cluster Instance

Prerequisites

  • Before registering a PostgreSQL cluster instance, ensure that each host in the cluster has been registered.

    All hosts in the cluster must have registered with the same database.

    For details, see Step 3: Registering the Database of a Single PostgreSQL Instance.

  • Before registering an instance, check whether the owner of the bin/lib/share file in the database installation path is root. If the owner is root, set enable_root to 1 in the /opt/DataBackup/ProtectClient/Plugins/GeneralDBPlugin/bin/applications/postgresql/conf/switch.conf configuration file.
  • Before registering a PostgreSQL cluster, query the cluster type.

    Pgpool cluster

    1. Use PuTTY to log in to the PostgreSQL database host.
    2. Run the ps -ef | grep pgpool command to check whether the cluster is a Pgpool cluster. If the fields in the command output contains pgpool, the current cluster is a Pgpool cluster. Otherwise, the current cluster is not a Pgpool cluster.
      • If the following command output is displayed, the current cluster is a Pgpool cluster.

      • If only the following information is displayed, the current cluster is not a Pgpool cluster.

    Patroni cluster

    1. Use PuTTY to log in to the PostgreSQL database host.
    2. Run the ps -ef | grep patroni command to check whether the cluster is a Patroni cluster. If the fields in the command output contains patroni, the current cluster is a Patroni cluster. Otherwise, the current cluster is not a Patroni cluster.
      • If the following command output is displayed, the current cluster is a Patroni cluster.

      • If only the following command output is displayed, the current cluster is not a Patroni cluster.

    CLup cluster

    1. Use PuTTY to log in to the PostgreSQL database host.
    2. Run the ps -ef | grep clup command to check whether the cluster is a CLup cluster. If the fields in the command output contains clup, the current cluster is a CLup cluster. Otherwise, the current cluster is not a CLup cluster.
      • If the following command output is displayed, the current cluster is a CLup cluster.

      • If only the following command output is displayed, the current cluster is not a CLup cluster.

Procedure

  1. Create a cluster.

    1. On the PostgreSQL page, click the Cluster tab.
    2. Click Register.
    3. Configure the cluster and its authentication information.

      Table 1 describes related parameters.

      Table 1 PostgreSQL cluster registration information

      Parameter

      Description

      Name

      Enter a custom name.

      Cluster Type

      Select Pgpool, Patroni, or CLup.

      Type

      Select Active/Standby Replication.

      Virtual IP Address

      Enter a custom virtual IP address that is unused and in the same network segment as that of the host where the PostgreSQL instance is located.

      Nodes

      Select the host or CLup server node where the PostgreSQL instance is located.
    4. Click OK.

  2. Register the database of a cluster instance.

    1. On the PostgreSQL page, click the Instance tab.
    2. Click Register.
    3. Set Type to Cluster instance.
    4. Configure the database instance and its authentication information.

      Table 2 describes related parameters.

      Table 2 PostgreSQL cluster instance registration information

      Parameter

      Description

      Name

      Username for installing the database.

      Cluster

      Select the database host to be registered.

      Username

      OS username of the database.

      Pgpool Port

      This parameter is not displayed when the database host of the Patroni cluster is selected.

      Port number used by the Pgpool service to listen on TCP/IP connections.

      Database Username

      Name of the database administrator.

      Database Password

      Password of the database administrator.

      Database Stream Replication Username

      Username for streaming replication in the database.

      Database Stream Replication Password

      Password for streaming replication in the database.
    5. Click Add. In the displayed Add dialog box, add all nodes of the cluster instance.

      Table 3 describes related parameters.

      Table 3 Node information of a cluster instance

      Parameter

      Description

      Hosts

      Host node in the cluster.

      Database Installation Path

      Path for installing the database to be registered.

      Full Path of Patroni Configuration File

      NOTE:

      This parameter is displayed only in 1.6.0 and later versions.

      When configuring a cluster instance, set the cluster to the database host of the Patroni cluster. This parameter is displayed when you add a cluster instance node.

      Full path of the Patroni configuration file.

      Pgpool Installation Path

      Installation path of the Pgpool service.

      Service Plane IP Address

      IP address of the host for the database service to receive TCP/IP connections.

      Database Port

      Port number used by the database service to listen on TCP/IP connections.
    6. Select the registered instance name of each cluster member in sequence.
    7. Click OK.
