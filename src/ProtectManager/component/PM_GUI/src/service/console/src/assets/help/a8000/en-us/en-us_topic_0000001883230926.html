Managing Database Instances

The OceanProtect allows you to modify, remove, activate, or disable database protection.

Log in to the management page, choose Protection > Databases > MySQL/MariaDB, click Instances, and find the target instance.

Table 1 describes the related operations.

Table 1 Instance protection operations

Operation

Description

Navigation Path

Protect

Scenario

This operation enables you to protect a database instance.

In the row of the target instance, choose More > Protection.

Modify Protection

Scenario

This operation enables you to modify a protection plan to associate another SLA with a database or modify user-defined script. After the modification, the next backup will be performed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row of the target instance, choose More > Modify Protection.

Remove Protection

Scenario

This operation enables you to remove the protection plan if a database instance no longer needs to be protected. After the removal, the system cannot execute protection jobs for the database instance.

Note

  • When a protection job of the database instance is running, protection cannot be removed.
  • If the SLA associated with a resource changes, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page. You are advised not to associate the resource with another SLA after removing protection.

    After you remove protection and associate the resource with another SLA, the license capacity may be deducted repeatedly if the copy of the original SLA still exists.

In the row of the target instance, choose More > Remove Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target instance, choose More > Manual Backup.

Disable Protection

Scenario

This operation enables you to disable a protection plan when periodic protection on the database is not required. After the protection plan is disabled, the system does not execute the automatic backup job of the database.

Note

Disabling protection does not affect ongoing backup jobs.

In the row of the target instance, choose More > Disable Protection.

Activate Protection

Scenario

This operation enables you to re-activate the disabled protection plan of a database. After the protection plan is activated, the system backs up the database based on the protection plan.

Note

You can perform this operation only on a database that has been associated with an SLA and is in the Unprotected state.

In the row of the target instance, choose More > Activate Protection.

Test Connectivity

Scenario

This operation enables you to test the connectivity between ProtectAgent and the production environment.

In the row of the target instance, choose More > Test Connectivity.

Modify

Scenario

This operation enables you to change the name and node of a registered database instance.

In the row of the target instance, choose More > Modify.

Delete

Scenario

This operation enables you to delete a database instance that no longer requires protection.

In the row of the target instance, choose More > Delete.
