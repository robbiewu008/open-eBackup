Viewing Agent Host Information

After ProtectAgent is installed on a host, you can view details about the host on the management page.

Procedure

  1. Choose Protection > Clients > Clients.
  2. View the agent host information.

    Table 1 describes the related parameters.
    Table 1 Agent host information

    Parameter

    Description

    Resource ID

    ID of ProtectAgent.

    Name

    Name of the agent host.

    IP Address

    IP address of the agent host.

    Status
    Status of the agent host.
    • Online: When the agent host is powered on and ProtectAgent is running properly, Status is Online.
    • Offline: When the agent host is powered off or ProtectAgent is faulty, Status is Offline.
    • Upgrading Queuing: The agent host is preparing to update ProtectAgent.
    • Upgrading: The agent host is updating ProtectAgent.
    • Modifying Application Type: The agent host is modifying the supported protected applications.

    CPU Usage

    Percentage of agent host CPU that is occupied by ProtectAgent.

    Memory Usage

    Percentage of agent host physical memory that is occupied by ProtectAgent.

    OS

    OS type of the host.

    Agent Type

    Type of the agent host.

    • VMware backup agent: agent host of a VMware VM.
    • General backup agent: agent host except VMware agent hosts.
    • SAN Client agent: agent host configured with the SAN Client agent.

    Current Version

    Version of ProtectAgent on the agent host.

    Owner

    Data protection administrator of the agent host.

    If another data protection administrator has been authorized for the agent host, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.

    SourceDedupe Supported or Not
    Some agent host OSs support source deduplication.
    • Yes: The DataTurbo service exists when ProtectAgent is being installed or upgraded on the agent host.
    • No: The DataTurbo service does not exist when ProtectAgent is being installed or upgraded on the agent host.
    NOTE:
    • For details about how to configure the source deduplication function, see the SourceDedupe User Guide of the corresponding version.
    • After source deduplication is enabled, add the source deduplication user to sudoers before the first backup. For details, see How Can I Add a Source Deduplication User to sudoers? (Applicable to the Linux OS).

    Log Level

    Agent host log level. The options include Info, Warn, Error, and Debug.

    Sharing

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    Sharing status of the agent host.

    LAN-Free

    LAN-Free configuration status of the agent host.

    Installation Directory

    Installation directory on the agent host.

    Remarks

    Remarks of the agent host.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the agent host, the tag name is displayed. Otherwise, -- is displayed.
