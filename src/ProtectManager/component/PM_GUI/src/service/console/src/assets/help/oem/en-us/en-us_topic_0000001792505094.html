Enabling the SFTP Service

You can enable the SFTP service on the product to provide storage space for the SFTP client.

Prerequisites

  • For the newly deployed product, port P3 on IOM 0 of all controllers is connected to the SFTP client network to enable the SFTP service.
  • If the product version is upgraded from 1.3.0 or an earlier version, port P3 on IOM 1 of all controllers is connected to the SFTP client network to enable the SFTP service.

Procedure

  1. On the menu bar, choose System > Settings.
  2. In the navigation tree, click SFTP Service.
  3. On the SFTP Service tab page, click on the right to enable the SFTP service.
  4. A dialog box is displayed. If the SFTP service is enabled for the first time, you need to enter the IP address.

    • If you set the IP address type to IPv4 during initial configuration of the product, see Table 1.
      Table 1 Parameters in the IPv4 scenario

      Parameter

      Description

      IPv4 Address

      IP address of the SFTP service.

      The IP address and Subnet Mask are used by the SFTP client to connect to the SFTP server of the product. Ensure that the IP address can communicate with the SFTP client and does not conflict with IP addresses in the network segment of the SFTP client.

      Subnet Mask

      Subnet mask of the IPv4 address.

      Port

      In the new deployment scenario, select the IOM 0.P3 port. In the upgrade scenario, select the IOM 1.P3 port.

      WORM

      When the SFTP service is enabled for the first time, a file system (fixed name: SFTP_FS_e1f0b07c7ad24daaaceba47cda8b4e66) is created on the local storage. You can enable WORM and directly set the file system as a WORM file system. WORM is disabled by default. If the file system already exists on the local storage, no new file system will be created.

      NOTE:

      After WORM is enabled, if you need to modify the configuration of the WORM file system SFTP_FS_e1f0b07c7ad24daaaceba47cda8b4e66, log in to the local storage to modify it.

      Mode

      This parameter is displayed after WORM is enabled.

      • Compliance

        For files within the protection period, super administrators, common administrators, and common users cannot modify or delete the files. For files whose protection period expires, super administrators, common administrators, and common users cannot modify the files but can delete them.

      • Enterprise

        For files within the protection period, common administrators and users cannot modify or delete the files, but privileged users can delete them. For files whose protection period expires, super administrators, common administrators, and common users cannot modify the files but can delete them.

      Protection Period

      This parameter is displayed after WORM is enabled.

      Set file protection time period. The value ranges from 0 to 70 years.

      Automatic Lockout

      This parameter is displayed after WORM is enabled.

      After this function is enabled, a file automatically enters the locked state if not being modified within Lockout Wait Time (hours). The file in the locked state is protected. You can only read the file, but cannot modify, rename, or delete it.

      Lockout Wait Time

      This parameter is displayed after WORM is enabled.

      If a file is not modified within the lockout wait time, it automatically enters the locked state. A locked file can only be read, but cannot be modified or deleted. Value range: 1 minute to 10 years.

      Automatic Deletion

      This parameter is displayed after WORM is enabled.

      After this function is enabled, the system automatically deletes files whose protection period expires.

    When the SFTP service is enabled for the first time, a container network plane named sftpNetPlane is automatically created. The start and end IP addresses of the IP address segment of the network plane are the IPv4 address or IPv6 address entered when the SFTP service is enabled. You can modify the SFTP network plane. For details, see Modifying the SFTP Network.

  5. Click OK.

Follow-up Procedure

On the SFTP Service tab page, click on the right. In the dialog box that is displayed, click OK to disable the SFTP service.

Before disabling the SFTP service, ensure that no SFTP user uses the SFTP service to transfer files and that all SFTP users do not need to use the SFTP service.
