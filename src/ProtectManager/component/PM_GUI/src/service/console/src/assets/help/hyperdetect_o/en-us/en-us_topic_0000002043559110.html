Configuring Honeypot Detection

After the honeypot detection function is enabled, honeyfiles are generated in the directory of the selected file system. When an exception is detected, the system analyzes the honeyfiles to determine whether the file system is under ransomware attacks or the exception is caused by misoperations of users, so as to detect ransomware attacks in real time and reduce false detections.

Precautions

  • After the honeypot detection function is enabled, a small number of honeyfiles are placed in the shared directory for ransomware detection. Do not encrypt or delete these honeyfiles, or change their file name extensions. Otherwise, false detection may occur.
  • The honeypot detection function scans the metadata of files in the shared directory.
  • After the WORM function is enabled, there may be residual honeyfiles in the shared directory. Honeypot detection is not recommended when WORM is enabled because WORM provides a stricter data protection mechanism.
  • For file systems with HyperMetro in active-active mode configured, honeypot detection must be enabled on both the primary and secondary storage systems.
  • For file systems with HyperMetro in active-passive mode or synchronous mode configured or with remote replication configured, honeypot detection can be enabled only on the primary storage system.
  • After the ransomware protection alarm 0x5F025D0004 is cleared, the system automatically redeploys honeyfiles to ensure that there are available honeyfiles.

Prerequisites

The real-time ransomware detection function has been enabled. For details, see Enabling Real-Time Ransomware Detection.

Procedure

  1. Log in to ProtectManager.
  2. On the navigation pane, choose Data Security > Ransomware Protection > Real-Time Ransomware Detection.
  3. Click the Honeypot Detection tab.
  4. Select the file system for which you want to enable honeypot detection and choose More > Enable Honeypot Detection in the Operation column.

    • You can also select multiple file systems and click Enable Honeypot Detection above the file system list.
    • Honeypot detection can be enabled for up to 10 file systems.

  5. On the Enable Honeypot Detection page that is displayed, set the honeyfile update frequency. You are advised to update the honeyfiles periodically.

    • When Periodic update is selected, honeyfiles can be adaptively updated based on the latest file system information, improving effectiveness in ransomware detection of honeyfiles.
    • When No update is selected, honeyfiles cannot be adaptively updated, which may reduce effectiveness in ransomware detection of honeyfiles.

  6. Click OK.

    After honeypot detection is configured, you can choose More > View Honeyfile in the Operation column of the specific file system to view the path, quantity, and name of honeyfiles.
