Step 1: Creating a Common Share

You need to create a common share before performing common share resource protection.

Procedure

  1. Log in to the OceanProtect WebUI by referring to Logging In to the OceanProtect WebUI.
  2. Choose Protection > File Systems > Common Shares.
  3. Click Create to create a common share.

    Table 1 describes the related parameters.

    Table 1 Parameters for creating a common share

    Parameter

    Description

    Resource Name

    Name of a common share resource.

    File system name

    Name of a file system. The system will create the file system on DeviceManager to store backup data.

    Backup Storage Units

    Corresponding backup cluster node. After a backup storage unit is selected, copies are stored to the storage unit. If the capacity of the storage unit is insufficient, the backup job will fail.

    Rate Limiting Policies

    Average maximum amount of data that can be transmitted per second in a backup job.

    Share Protocol

    Resource protocol of a common share. The NFS and CIFS protocols are supported.

    NOTE:

    After the CIFS protocol is set, if the user or user group mounted to the file system has been deleted, manually unmount the user or user group.

    Share Path

    Path to which the file system is mounted. This parameter is displayed when the NFS protocol is selected.

    Share Client

    This parameter is displayed when the NFS protocol is selected. Enter the service IP address or service IP address segment of a share client, or enter an asterisk (*) to indicate service IP addresses of all share clients. The IP addresses can be IPv4 addresses, IPv6 addresses, or a combination of IPv4 and IPv6 addresses.

    Share Name

    User-defined CIFS share name. This parameter is displayed when the CIFS protocol is selected.

    Type

    Type of the user who accesses the CIFS share. This parameter is displayed when the CIFS protocol is selected.

    • Everyone
    • Windows local authentication user
    • Windows local authentication user group

    Users

    This parameter is displayed when the user type of CIFS shares is set to Windows local authentication user or Windows local authentication user group. Select a username or a local user group. If the required user or user group does not exist, create it on DeviceManager.

  4. Click OK.
