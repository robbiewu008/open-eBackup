Configuring the Host Where the Client Is Installed

On the host where SAP ERP has been installed, manually create a run directory and copy files to the directory, start the database, create a database user, and add host environment variables based on the obtained SAP ERP information.

Context

When installing Oracle, you will set the SID in the cmd window. When installing SAP ERP, you will set the installation path, SAP instance ID, and database instance ID.

After the installations are complete, the system generates directories based on the SID, installation path, SAP instance ID, and database instance ID. For details about the directories generated based on the input parameters, see Table 1.

  • If you forget the values of SID, SAPSID, and DBSID, you can obtain them from the directories listed in Table 1. Example:
    • C:\usr\sap\EH7 indicates that SAPSID is set to EH7 during the installation.
    • C:\oracle\EH8 indicates that DBSID is set to EH8 during the installation.
    • C:\oracle\EH9\11203 indicates that SID is set to EH9 during the installation.

Manually Creating a run Directory and Copying Files to the Directory

After SAP ERP is installed, the uc directory is generated in the <Destination Drive>:\usr\sap\<SAPSID>\SYS\exe directory. For details about Destination Drive and SAPSID, see Table 1. Create a run directory at the same level as the uc directory in the directory, and copy the following files in the uc directory to the run directory:

  • brarchive
  • brbackup
  • beconnect
  • brrecover
  • brresotre
  • brspace
  • brtools

Starting the Database and Creating a Database User

Press Win+R, enter cmd, and press Enter to open Command Prompt on a Windows host. Enter the following commands to start the database and create a database user:

sqlplus / as sysdba
startup;
create user userName identified by "passWord";
grant dba,sysoper,create session to userName;
commit;

userName and passWord indicate the name and password of the created database user. Example:
