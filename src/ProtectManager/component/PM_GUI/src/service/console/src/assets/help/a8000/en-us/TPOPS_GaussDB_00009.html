Preparing for the Backup

Before the backup, prepare information such as certificates, accounts, and IP addresses by referring to Table 1.

OLTP OPS (TPOPS) is a web-based database monitoring tool that can perform security monitoring and O&M operations on GaussDB instances.

Table 1 Preparing for the backup

Item

Mandatory or Not

How to Obtain

To Be Used In

Management IP address of the TPOPS server node

Yes

Management IP address of the node where the TPOPS server is installed. Contact the TPOPS administrator to obtain it.

NOTE:

If TPOPS is deployed in HA mode,obtain the management IP address of the active node.

Step 1: Obtaining the Address and Port of the Management Plane

Username and password

Yes

Username and password for logging in to the TPOPS server node. Contact the TPOPS administrator to obtain them.

Management plane IP address of GaussDB

Yes

Address used by OLTP OPS (TPOPS for short) or the cloud database GaussDB (supported in 1.6.0 and later versions) to provide services externally. For details, see Step 1: Obtaining the Address and Port of the Management Plane.

Step 3: Registering the GaussDB Project

Username and password

Yes

Super administrator account and password of TPOPS or the cloud database GaussDB (supported in 1.6.0 and later versions). Used to log in to TPOPS with the highest permission. The account name of the super administrator is admin, and the password is set during the TPOPS server installation.

Certificate

No

The certificate is required only when XBSA SSL certificate management needs to be enabled. For details about how to generate and configure a certificate, see "Configuring Certificate Files for Mutual SSL Authentication" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.
