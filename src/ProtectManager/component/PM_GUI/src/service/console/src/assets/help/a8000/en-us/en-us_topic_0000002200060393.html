Step 1: Creating a Replication Network Logical Port (Applicable to 1.6.0 and Later Versions)

On the OceanProtect at the primary and secondary ends, you can create logical ports for the replication network to remotely transmit replication data.

Prerequisites

The replication network IP addresses have been planned at the primary and secondary ends.

Precautions

Ensure that no job is running during network configuration.

Procedure

  1. Choose System > Settings > Network Settings.
  2. Click in the upper right corner of the Network Settings page.
  3. In the Service Network Configuration area, create a replication logical port for each controller.

    1. Click Create Port.
    2. On the displayed Configure Service Port page, set Role to Replicate.
    3. Customize a logical port name.
    4. In the Network Settings area, set the IP address type, IP address, subnet mask (IPv4)/prefix length (IPv6), and gateway of the logical port.
    5. In the Port Configuration area, set the parameters by referring to Table 1.

      The information in the Port Configuration area cannot be modified after being saved. Ensure that the information is correct during the configuration.

      Table 1 Parameter description

      Parameter

      Description

      Enable VLAN

      This option is deselected by default. Before selecting this option, ensure that the VLAN has been configured for the service. After this option is selected, the configured service ports will be used for the VLAN.

      Port Type

      Type of the port to which the logical port belongs. Available options include Ethernet port and Bond port.

      Port Selection
      Available options include Creating a port and Reusing the configured port.
      NOTE:
      • This parameter is displayed only when Enable VLAN is selected or Port Type is set to Bond port.
      • If Port Type is set to Bond port but the number of available Ethernet ports is less than 2, set Port Selection to Reusing the configured port or change the value of Port Type to Ethernet port.
      • Assume that a configured bond port is reused to create a logical port. If the bond port corresponding to the logical port is not occupied by other services, deleting the logical port will also delete the bond port on the backup storage device.

      Ethernet Port
      • If Port Type is set to Bond port, Port Selection is set to Creating a port, and the number of available Ethernet ports is greater than or equal to 2, you can select the available Ethernet ports.
      • If Port Type is set to Ethernet port, select available Ethernet ports.

      MTU (bytes)

      Maximum size of a data packet transmitted by a member port of the network plane.

      The value ranges from 1280 to 9000.

      NOTE:
      • MTU is short for maximum transmission unit.
      • If Enable VLAN is not selected, this parameter is displayed only when Bond port is selected and Port Selection is set to Creating a port.
      • If Enable VLAN is selected, this parameter is displayed only when Ethernet port is selected and Port Selection is set to Reusing the configured port or when Bond port is selected.

      VLAN ID

      VLAN ID must be configured if you select Enable VLAN.
    6. Click OK.
    7. Optional: If the local replication network and the replication network of the target cluster are not in the same network segment, you need to configure a route. Otherwise, skip this step.
      1. Click Manage Route in the Operation column of the corresponding port.
      2. On the displayed Manage Route page, click Add.
      3. Select a route type. Under Destination Address, enter a replication network segment of the target replication cluster. Set a subnet mask for an IPv4 address or a prefix for an IPv6 address, and local replication network gateway.

        If Type is set to Default route, you only need to set Gateway.

      4. Click Confirm.
      5. Click Close.

  4. Click OK.
