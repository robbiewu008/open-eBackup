Creating a User

The system administrator can create users of different roles in view of service requirements to control user operation permissions and ensure service system stability and service data security.

Context

For details about user roles and permissions, see Built-in User Roles.

Precautions

The system function permissions of custom users are the same as those of the data protection administrator.

Procedure

  1. Choose System > Security > RBAC > Users.
  2. Click Create.
  3. Set basic information such as user login and roles.

    Table 1 describes the parameters.
    Table 1 Basic user parameters

    Parameter

    Description

    User Type

    Type of the user. Available options are Local user, LDAP user, LDAP user group, and ADFS user.

    • ADFS user: logs in to the OceanProtect in ADFS authentication mode.
      NOTE:

      This user can be created in 1.6.0 and later versions.

    • Local user: logs in to the OceanProtect in local authentication mode.
    • LDAP user: logs in to the OceanProtect in LDAP server authentication mode for unified user information management.

    Name

    Name of the created user.

    • If LDAP user or LDAP user group is selected, enter the user or user group on the LDAP domain server. Otherwise, the login will fail.
    • If ADFS user is selected, ensure that the username is the same as that created on the ADFS server. Otherwise, the login will fail.

    Login Authentication Mode

    Optional values are Login password and Login password+Email One-time password. This parameter is displayed only when User Type is set to Local user.

    NOTE:
    • The Login password+Email One-time password mode provides higher security. To use this mode, ensure that you have configured the email address for receiving one-time passwords by referring to Managing the Email Service.
    • For the built-in role Remote Device Administrator, this parameter can only be set to Login password.

    Password

    Password of the created user. This parameter is displayed only when the user type is Local user.

    [Value range]

    • The password contains 8 to 64 characters.
    • The password must contain special characters. Special characters include ! " # $ % & ' ( ) * + , - . / : ; < = > ? @ [ \ ] ^ ` { _ | } ~ and spaces.
    • The password must contain uppercase letters, lowercase letters, and digits.
    • A password cannot contain more than two consecutive identical characters.
    • A password cannot be the same as the username or the username spelled backwards.
    • The password cannot contain the username.
    NOTE:
    • To modify the password policy, choose System > Security > Security Policies.
    • Users should properly manage passwords to prevent leakage.

    [Example]

    a#123456

    Confirm Password

    Confirm password of the new user.

    [Value range]

    The value of Confirm Password must be the same as that of Password.

    [Example]

    a#123456

    Recipient Email Address

    Email address for receiving one-time passwords. This parameter is displayed only when the user type is Local user and the login authentication mode is Login password+Email One-time password.

    Role

    Click Add. On the displayed page, select a built-in role provided by the system, or create a role and set user permissions. For details about how to create a role, see Creating a Role.

    NOTE:
    • The built-in roles, except Data Protection Administrator, can only be used as default roles of users, and cannot be bound with resource sets. If such a role is selected, other roles cannot be added.
    • Data Protection Administrator can be bound with resource sets and used as the default or other roles of users.

    Default Role

    Select an added role as the default role. After login, a user has the permissions of the default role (excluding the permissions on the resources for authorization).

    NOTE:

    Whether a user has permissions on a resource for authorization depends on the permissions of the role associated with the resource.

    Description

    Description of the new user.

    [Value range]

    The description can be left blank or contain up to 255 characters.

    [Example]

    User

    Max. Number of User Connections

    Whether to enable limitation on the maximum number of concurrent logins to the system using a user. If this function is disabled, the number is not limited.

    • : enabled
    • : disabled

    The default status is .

    [Value range]

    The value ranges from 1 to 8.

    The default value is 5.

    NOTE:

    When Role is set to Remote Device Administrator, you do not need to set this parameter.

    Password Always Valid

    This parameter is mandatory when the Disaster Recovery Administrator role is added. To ensure that the password is valid when you switch from eDME to the OceanProtect, enable this function.

    NOTE:

    Setting the password to never expire poses security risks.

  4. Click Next.
  5. (Optional) Associate the role with resource sets. This allows you to assign the manageable resource scope and bind different resource sets to different role permissions.

    1. Select the role to be associated with resource sets.
    2. In the resource set list, select existing resource sets. Alternatively, you can click Create Resource Set to create a resource set.
    3. Click Next.
    • Except the Data Protection Administrator, built-in roles do not support resource authorization.
    • After a resource set is authorized to a user, the user can only view or perform operations on the copies generated for the corresponding resources after the resource set is created.
    • If a copy of a resource generated before the resource is bound and authorized to a user is being used for a restoration job, the job is invisible to the user. In this case, if the user attempts to restore data using this copy, the restoration job may fail due to locking.
    • If a parent resource is authorized to a user, the user has the same permissions on the parent resource and its child resources. If a child resource is authorized to a user, the user has only the view permission on its parent resource.
    • Alarm resources do not support rights- and domain-based management. That is, all users can view all alarms after login.

  6. Preview the created user information and click Finish.
