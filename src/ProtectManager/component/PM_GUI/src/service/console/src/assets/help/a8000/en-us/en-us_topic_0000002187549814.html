What Can I Do to Create a DataTurbo Logical Port?

If you need to use the source deduplication function in the subsequent backup process, create a DataTurbo-compliant logical port by referring to this section.

Prerequisites

One DataTurbo logical port IP address has been planned for each controller.

Procedure

  1. Log in to DeviceManager by referring to Logging In to DeviceManager.
  2. Choose Services > Network > Logical Ports.
  3. Click Create to create a logical port.

    Table 1 describes the related parameters.

    Table 1 Logical port parameter description

    Parameter

    Description

    Name

    User-defined port name.

    Role

    Select Service.

    Data protocol
    Select DataTurbo.
    NOTE:

    You need to create one DataTurbo-compliant logical port for each controller. During backup, the system creates an account whose role is vStore DataTurbo administrator under the vStore to which the logical port belongs.

    IP address type

    The IP address type is fixed to IPv4.

    IP address

    IP address of the DataTurbo logical port.

    Subnet mask

    Subnet mask of the IPv4 address of the DataTurbo logical port.

    Port type

    Type of the port used to create the DataTurbo logical port. Available options include Ethernet port, Bond port, and VLAN.

    Home port

    Ethernet port, bond port, or VLAN to which the DataTurbo logical port to be created belongs.

    The system has filtered all ports used for source deduplication. Select any port in the Link up state.

  4. Click OK.
  5. Repeat the preceding steps to create a DataTurbo-compliant logical port for each controller.
