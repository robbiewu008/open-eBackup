Viewing Object Storage Information

After object storage is created, you can view its details on the management page.

Procedure

  1. Choose Protection > File Systems > Object Storage.
  2. On the Object Storage tab page, view the object storage information.

    The registered object storage is displayed on the page.

    1. Table 1 describes the details about object storage.
      Table 1 Object storage information

      Parameter

      Description

      Name

      Object storage name.

      Status

      Object storage status.

      • Online: If the connection between the agent host and object storage is normal, Status is Online.
      • Offline: If the connection between the agent host and object storage is abnormal, Status is Offline.

      Type

      Object storage type, which can be OceanStor Pacific, Huawei Cloud Stack OBS, or Alibaba Cloud OSS.

      Endpoint

      Access address of the object storage.

      Address format: domain name or service plane IP address of the object storage

      Protocol

      Protocol used for data transmission between the OceanProtect and object storage.

      AK

      Access certificate for accessing the object storage.
    2. Click an object storage name to view the basic information about the object storage and the object set to which the object storage belongs.

  3. On the Object Set tab page, view the object set information.

    The registered object sets are displayed on the page.

    1. Table 2 describes the details about object sets.
      Table 2 Object set information

      Parameter

      Description

      Name

      Name of an object set.

      Owning Object Storage

      Object storage to which an object set belongs.

      Type of Owning Object Storage

      Type of the object storage to which an object set belongs.

      SLA

      If an SLA is associated with the object set, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the object set is protected.

      • Unprotected: The object set is not associated with an SLA, or the object set is associated with an SLA but protection is disabled.
      • Protected: The object set has been associated with an SLA and protection has been activated.
      • Creating: The object set has been associated with an SLA and protection is being created.
    2. Click an object set name to view the object set overview, copy information, and related jobs.
      • Overview

        Displays the object set name, SLA information, job execution time, and total number of copies.

      • Copy Data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy at that time exists.

      • Jobs

        Displays all jobs related to the object set, including backup and restoration jobs.
