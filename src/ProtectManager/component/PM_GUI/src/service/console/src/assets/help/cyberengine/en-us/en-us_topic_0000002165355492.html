Creating an Intelligent Detection Policy

This section describes how to create an intelligent detection policy to create a ransomware detection snapshot for a file system.

Precautions

  • The OceanCyber 300 Data Security Appliance is a server deployed in single-node mode. To ensure that services can be quickly restored if the server is faulty, users need to set management data backup policies and ensure that a management data restoration system is available.
  • Ransomware detection is not performed for file systems created by the OceanCyber 300 Data Security Appliance itself.
  • Ransomware detection is not performed for the CLONE file system.
  • If Uninfected Snapshot Lock is enabled for an intelligent detection policy, the confirmed uninfected snapshots are converted to secure snapshots and cannot be manually deleted within the retention period.
  • The OceanCyber 300 Data Security Appliance uses the snapshot comparison RESTful API of the storage device to obtain the changed file information of file systems. The OceanCyber Data Security Appliance automatically enables the snapshot comparison function for detected file systems.

Procedure

  1. Choose Data Security > Intelligent Detection.
  2. Click Intelligent Detection Policies.
  3. Click Create.
  4. Customize an intelligent detection policy name.
  5. Set Detection Method.

    • Select Generate only ransomware detection snapshots. If this detection method is selected, only ransomware detection snapshots are generated and the snapshots are not detected immediately. You can select the snapshot to be detected for manual detection. To perform manual detection, choose Data Security > Snapshot Data, locate the row that contains the target snapshot, and choose More > Detect Now.
    • Select Start detection upon ransomware detection snapshot generation.
      • Set Backup Copy In-Depth Detection. This parameter applies only to the file system of the OceanProtect Backup Storage device. After this function is enabled, the OceanCyber 300 Data Security Appliance performs in-depth parsing and detection on backup copy files in the backup storage to evaluate whether the original files (files on the backup production storage device) in the backup copy files are infected. If this function is enabled, the overall detection time may be extended.
        • After Backup Copy In-Depth Detection is enabled, you can adjust the sensitivity of the backup copy detection algorithm. If Sensitivity is set to High, an alarm may be triggered when a small amount of data is encrypted or similar operations are performed, increasing the risk of misreports. Medium sensitivity is recommended for service scenarios without special requirements.
        • The sensitivity adjustment of the backup copy detection algorithm takes effect only for VM, database, or Veeam-based host file backup copies.
      • Set Uninfected Snapshot Lock. The Uninfected Snapshot Lock parameter means that snapshots for which no ransomware file is detected will be locked. After this function is enabled, uninfected snapshots will change to secure snapshots, and the snapshot retention period will be prolonged. Modification or deletion is not allowed before the snapshots expire.

  6. Configure parameters under Ransomware detection snapshot. Table 1 describes the related scenarios.

    Set the ransomware detection snapshot generation interval, ransomware snapshot generation window, and snapshot retention period based on service requirements. The suggestions are as follows:
    • The intelligent ransomware detection rate is about 1000 changed files per second. Evaluate the ransomware snapshot generation interval based on the file system file scale. It is recommended that the ransomware snapshot generation interval be greater than the time required for intelligent ransomware detection to reduce the number of stacked detection jobs.
    • It is recommended that the time when the ransomware snapshot is generated for the first time be the same as the start time of the time window.
    • The snapshot retention period must be longer than an interval between two times of ransomware detection snapshot generation.
    Table 1 Configuring an intelligent ransomware detection snapshot policy

    Scenario

    Description

    By Year

    Set a day to execute the ransomware snapshot generation job each year. Set the snapshot retention period to xx days, weeks, months, or years, or set it to Permanently. Set a time period during which ransomware snapshot generation is allowed.

    By Month

    Set the ransomware snapshot generation job to be executed once every month by specifying a day or on the last day of each month. Set the snapshot retention period to xx days, weeks, months, or years, or set it to Permanently. Set a time period during which ransomware snapshot generation is allowed.

    By Week

    Set the ransomware snapshot generation job to be executed once every Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, or Sunday (you can select multiple options). Set the snapshot retention period to xx days, weeks, months, or years, or set it to Permanently. Set a time period during which ransomware snapshot generation is allowed.

    By day

    Set an interval to execute the ransomware snapshot generation job once every xx days since the start date. Set the snapshot retention period to xx days, weeks, months, or years, or set it to Permanently. Set a time period during which ransomware snapshot generation is allowed.

    By Hour

    Set an interval to execute the ransomware snapshot generation job once every xx hours since the start date. Set the snapshot retention period to xx days, weeks, months, or years, or set it to Permanently. Set a time period during which ransomware snapshot generation is allowed.

    Once the snapshot retention period ends, the system automatically deletes the expired snapshot.

    • If the end time of a ransomware snapshot generation job is earlier than or equal to the start time, the actual end time is of the next day.
    • If the ransomware snapshot generation job is not complete within the specified time window, the system does not stop the job but generates an alarm.

  7. Set advanced parameters for configuring the intelligent detection policy. Table 2 lists the related parameters.

    Table 2 Advanced parameters for configuring an intelligent detection policy

    Parameter

    Description

    Job Failure Alarm

    This option is enabled by default. An alarm is sent if the job fails and will be automatically cleared after successful job execution.

    Automatic Retry

    This option is enabled by default. If a ransomware detection snapshot generation job fails, the system automatically retries the job.

    The number of retries ranges from 1 to 5, and the wait time ranges from 1 to 30 minutes. For example, if the number of retries is set to 3 and the wait time is set to 5 minutes, the system retries for three times at an interval of 5 minutes.
    NOTE:

    During automatic retry, the system creates a ransomware detection snapshot generation job. If the job is not in the specified time window, the job will not be executed and the generation fails.

  8. Click OK. The intelligent detection policy is created.
