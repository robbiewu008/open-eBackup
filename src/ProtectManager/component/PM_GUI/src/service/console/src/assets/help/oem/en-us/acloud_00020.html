Step 8: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA. The system protects the resources and periodically executes backup jobs based on the SLA. You can perform a backup job immediately through manual backup.

Precautions

  • A cloud server can be backed up only when it is in the running or stopped state.
  • A maximum of 16 data disks can be mounted to an agent host. Therefore, if 16 data disks have been mounted to an agent host during backup or restoration, the system performs the backup job when there are idle slots.

Context

  • During manual backup, parameters (except Automatic Retry) defined in the SLA are applied to the manual backup job.
  • Copies generated by manual backup are retained for the period defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copies are retained permanently. If full backup is not defined in the SLA but selected during manual backup, the copies generated by the manual full backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Cloud Platforms > Alibaba Cloud.
  2. Perform periodic backup of an AZ or resource set.

    1. Select the registered Alibaba Cloud region on the left of the page.
    2. Click the AZs tab on the right and select the AZ for which you want to add or modify protection.
    3. Click Protect.

      The system backs up all cloud servers in the AZ.

    4. Select a preset or custom SLA.

      You can also click Create to create an SLA.

      If a WORM policy has been configured for the resources to be protected, select an SLA without a WORM policy to avoid WORM policy conflicts.

    5. Select the agent host.

      If no agent host is selected, the system performs backup by selecting one or multiple agent hosts from the agent hosts configured during resource registration.

    6. Set the parameters as follows:
      1. Set SLA Overwrite Policy.
        By default, Retain existing SLAs of the cloud server is selected.

        If you choose Retain existing SLAs of the cloud server when modifying an SLA associated with the project, the cloud server SLA will be replaced when the SLA associated with the cloud server is the same as that associated with the project.

      2. Select SLA application policies. You can select one or more of the following options:
        • Apply the project SLA to all cloud servers that are not associated with any SLA.
        • Apply the project SLA to newly created cloud servers.
      3. Specify cloud servers to be excluded or included in the protection plan.

        You can specify cloud servers to be excluded or included by cloud server name.

        Exact match and fuzzy match of cloud server names are supported. Fuzzy match rules include prefix match (for example, abc*), suffix match (for example, *abc), and infix match (for example, *abc*).

    7. Click OK.

      If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  3. Perform periodic backup of a resource set.

    1. Click the Resource Sets tab and select the target resource set.

  4. Perform periodic backup of a cloud server.

    1. Click the Cloud Server tab and select the cloud server to be protected.
    2. Click Protect.
    3. Determine whether to enable Automatic Disk Protection.

      Automatic Disk Protection is enabled by default to back up all disks of a cloud server. You can disable Automatic Disk Protection and click Please select to manually select desired cloud disks for backup.

      Automatic disk protection is enabled by default. After this function is enabled, all disks of the cloud server, including newly detected disks, are protected. After automatic disk protection is disabled, you need to manually select disks for protection.

      If system disks are excluded, generated copies do not support file-level restoration.

    4. Select a preset or user-defined SLA.

      You can also click Create to create an SLA.

    5. Select the agent host.

      If the agent host is not selected, the system performs backup by selecting one or multiple agent hosts from the agent hosts configured during resource registration.

    6. Set Auto Indexing of Backup Copies. (This parameter is supported only in 1.6.0 and later versions.)

      If a VM or VM group is selected for protection and the disks to be backed up contain system disks, the system automatically creates an index for the backup copy after this function is enabled. When using the copy for restoration, you can select one or more files in the copy for restoration and search for files, directories, or links in the copy.

    7. Click OK.

      If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  5. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. Based on the type of data to be backed up, click the AZs, Resource Sets, or Cloud Server tab.

      When you select an AZ or resource set for backup, only the cloud servers with the same SLA as the AZ or resource set are backed up. The cloud servers with different SLAs from the AZ or resource set are not backed up.

    2. In the row of the target AZ, resource set, or cloud server, choose More > Manual Backup on the right.

      You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

    3. Set the name of the copy generated during manual backup.

      If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.

    4. Select a protection policy, which can be full backup or forever incremental (synthetic full) backup.

      For 1.6.0 and later versions, if the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    5. Click OK.
