Querying TiDB Information

Procedure

  1. Choose Protection > Databases > TiDB.
  2. On the Cluster tab page, query the TiDB cluster information.

    1. This page displays information about registered TiDB clusters.

      Table 1 describes the related parameters.

      Table 1 TiDB cluster information

      Parameter

      Description

      Name

      TiDB cluster name.

      Status

      Status of the TiDB cluster.

      • Online: If the connection between the agent host and the TiDB cluster is normal, Status is Online.
      • Offline: If the connection between the agent host and the TiDB cluster is abnormal, Status is Offline.

      Version

      Database version of the TiDB cluster.

      SLAs

      If an SLA has been associated with the backup resource, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the TiDB cluster is protected.

      • Unprotected: The cluster is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The cluster is associated with an SLA and protection is activated.
      • Creating: The cluster is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

      Owner

      Data protection administrator of the cluster.

      If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
    2. Click a cluster name and check the overview, copy information, and related jobs.
      • Overview

        Displays information such as the cluster name, cluster version, owning user, internal cluster, agent host node, TiUP node, and data nodes.

      • Copy Data

        You can search for copies by year, month, and day in sequence.

        If is displayed below the time, copies exist at the time.

      • Jobs

        Displays jobs such as backup and restoration jobs, including jobs that are being executed and historical jobs.

  3. On the Databases tab page, query the TiDB database information.

    1. This page displays the details about the backup resource.
      Table 2 describes the related parameters.
      Table 2 Backup resource parameters

      Parameter

      Description

      Name

      Databases name.

      Status

      Status of the TiDB database.

      • Online: If the connection between the agent host and the TiDB database is normal, Status is Online.
      • Offline: If the connection between the agent host and the TiDB database is abnormal, Status is Offline.

      Owning Cluster

      Name of the cluster where the protected TiDB database resides.

      SLAs

      If an SLA has been associated with the backup resource, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the TiDB database is protected.

      • Unprotected: The database is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click a database name and check the overview, copy information, and related jobs.
      • Databases

        Displays the basic database information, SLA information, job execution time, and total number of copies.

      • Copy Data

        You can search for copies by year, month, and day in sequence.

        If is displayed below the time, copies exist at the time.

      • Jobs

        Displays jobs such as backup and restoration jobs, including jobs that are being executed and historical jobs.

  4. On the Table Sets tab page, query information of table sets in TiDB databases.

    1. This page displays the details about table sets.
      Table 3 describes the related parameters.
      Table 3 Table set parameters

      Parameter

      Description

      Name

      Name of the created TiDB Table Sets.

      Status

      Status of the TiDB table set.

      • Online: If the connection between the agent host and the TiDB table set is normal, Status is Online.
      • Offline: If the connection between the agent host and the TiDB table set is abnormal, Status is Offline.

      Owning Cluster

      Name of the cluster where the TiDB table set resides.

      Owning Database

      Name of the database where the TiDB table set resides.

      SLAs

      If an SLA has been associated with the backup resource, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the TiDB table set is protected.

      • Unprotected: The table set is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The table set is associated with an SLA and protection is enabled.
      • Creating: The table set is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click a table set name and check the overview, copy information, and related jobs.
      • Table Sets

        Displays the table set name, protected objects of the table set, table set status, SLA information, job execution time, and total number of copies.

      • Copy Data

        You can search for copies by year, month, and day in sequence.

        If is displayed below the time, copies exist at the time.

      • Jobs

        Displays jobs such as backup and restoration jobs, including jobs that are being executed and historical jobs.
