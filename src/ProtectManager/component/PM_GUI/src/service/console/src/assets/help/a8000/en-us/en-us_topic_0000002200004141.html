Step 2: Manually Configuring Soft Links

Before executing a backup job, you need to register MariaDB/GreatSQL as the system service. For MySQL/MariaDB/GreatSQL, you need to manually configure the soft links of mysqlbinlog, mysqldump, mysqld, mysqladmin, and mysql on all nodes.

Procedure

  1. Log in to the host where the MySQL/MariaDB/GreatSQL instance is located.
  2. Run the systemctl status mysqld or service mysqld status command to check the running status of the MySQL/MariaDB/GreatSQL instance.

    • MariaDB/GreatSQL has been registered as a system service if the running status is displayed in the command output, for example, active(running) (the command output varies depending on the version). In this case, go to 4.
    • If no running status is displayed in the command output:
      • MySQL: Ensure that you can log in to the MySQL service.
      • MariaDB/GreatSQL: Run the cp Installation directory/mysql.server /etc/init.d/mysql command to register the database as a system service. Replace Installation directory with the MariaDB/GreatSQL installation directory.

  3. Run the following commands in sequence to set the soft links:

    ln -sf Installation directory/bin/mysqlbinlog /usr/bin/
ln -sf Installation directory/bin/mysql /usr/bin/
ln -sf Installation directory/bin/mysqldump /usr/bin/
ln -sf Installation directory/bin/mysqld /usr/bin/
ln -sf Installation directory/bin/mysqladmin /usr/bin/

  4. Run the following commands in sequence to check whether the soft links have been set. If the version number information is displayed, the soft links have been set successfully.

    mysqlbinlog --version
mysqldump --version
mysqld --version
mysql --version
mysqladmin --version
