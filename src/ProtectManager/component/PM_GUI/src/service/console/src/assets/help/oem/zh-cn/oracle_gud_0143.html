原机恢复/即时恢复任务失败，错误详情包含“ORA-01034: ORACLE not available”

现象描述

用户执行以下操作后恢复/即时恢复Oracle数据库任务失败，错误详情包含“ORA-01034: ORACLE not available”。

  1. 用户对单机（单机的卷类型为ASM）上的数据库A执行备份，产生备份副本a。
  2. 将该单机作为目标位置执行即时挂载，且挂载的副本与目标主机中存在同名数据库A。
  3. 即时挂载后再执行销毁挂载操作。
  4. 使用备份副本a执行原机恢复/即时恢复操作。

可能原因

执行原机恢复/即时恢复时，本产品会对spfile参数文件进行修改，在上述操作场景下，数据库中不存在spfile参数文件导致启动数据库失败进而恢复任务失败。

处理步骤

  1. 使用PuTTY，登录Oracle数据库主机。
  2. 执行以下命令，切换到oracle账户。

    su - oracle

  3. 执行以下命令添加数据库。

    srvctl add database -d 数据库名称 -o ${ORACLE_HOME} -p spfile参数文件路径
