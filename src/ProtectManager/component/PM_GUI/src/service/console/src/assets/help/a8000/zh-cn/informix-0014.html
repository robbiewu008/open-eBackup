步骤7：执行备份

执行备份前将需要保护的资源关联到指定SLA策略，系统将对这些资源按照SLA策略进行保护，并根据SLA策略周期性执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

注意事项

  • 手动备份产生的副本保留时长按照SLA中的定义进行保留。
  • 如果SLA中定义了复制策略或归档策略，则执行手动备份时，OceanProtect会按照SLA的定义执行一次复制或归档。
  • 如果SLA中未定义日志备份，则无法自动执行日志备份。

操作步骤

  1. 选择“保护 > 数据库 > Informix/GBase 8s”。
  2. 在“实例”页签，选择需要保护的数据库，单击“更多 > 保护”。

    您也可以勾选多个数据库进行批量保护。

  3. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

    如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  4. 配置备份策略高级参数。

    相关参数说明如表1所示。
    表1 备份策略高级参数说明

    参数

    说明

    限速策略

    备份任务平均每秒最大传输的数据量。

    源端重删
    开启后，可以减少传输的数据量，但会增加客户端的资源占用。
    说明：
    • 由于同一个代理主机只能和一个存储单元建链，如果开启源端重删，“指定目标位置”请选择“备份存储单元”。否则，由于智能均衡可能会选择到未建链的存储单元，导致源端重删启用失败。如果不同受保护对象关联了不同SLA，且SLA中指定不同的备份存储单元，请为受保护对象选择不同代理主机，否则将会导致部分备份任务源端重删启用失败。
    • 如果代理主机已经与一个存储单元建链，当该代理主机未执行源端重删备份任务时，可以在代理主机上执行dataturbo delete storage_object storage_name=?命令（命令中的storage_name=?可以通过dataturbo show storage_object命令查询），删除与存储单元的源端重删链接。再选择该代理主机执行源端重删备份任务，则可以成功与新的存储单元建链。
    • 请参考OceanProtect兼容性查询工具查看主机的操作系统类型是否支持源端重删，若不支持，配置源端重删后将不会生效。
    • 若使用内置代理（即ProtectAgent合并部署到OceanProtect）进行备份，配置源端重删后将不会生效。

    任务失败告警

    开启后，任务执行失败后自动发送告警，下次任务成功后告警自动清除。

    失败后自动重试

    开启后，备份任务失败后系统自动重试。

    支持重试次数为1~5次，等待时长为1~30分钟。如设置为重试3次，等待5分钟，即每5分钟后重试一次，总共重试3次。
    说明：

    自动重试时系统会创建一个新的备份任务，如果该备份任务不在设定的时间窗内则不会执行，此次备份失败。

  5. 单击“确定”。

    如果当前系统时间已晚于该SLA首次备份的开始时间，则关联备份SLA后可以选择在当前立即执行备份或者根据SLA中设置的备份策略等待下次调度。

  6. 可选：执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。
    1. 在需要备份的资源所在行单击“更多 > 手动备份”。

      支持选择多个资源批量进行手动备份。勾选多个已保护的资源，在资源列表左上角选择“更多 > 手动备份”。

    2. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    3. 选择保护策略：全量备份或永久增量（合成全量）备份。

      对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    4. 单击“确定”。
