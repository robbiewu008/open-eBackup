Configuring the LDAP Service

You can log in to the OceanProtect in Lightweight Directory Access Protocol (LDAP) server authentication mode to manage user information in a unified manner.

Prerequisites

  • The LDAP server has been installed and deployed.
  • If LDAPS is used, apply for a CA certificate that matches the domain authentication server's certificate from a third-party certification authority, and then import the CA certificate. For details about how to import a CA certificate, see "Importing a Certificate" in the OceanProtect Appliance 1.5.0-1.6.0 Security Configuration Guide.

Context

LDAP is a TCP/IP network protocol that enables users to access directory system agents (DSAs). LDAP functionalities are reduced from X.500 Directory Access Protocol (DAP).

The complexity of network management, especially user management, increases as the number of network applications grows. Most systems that provide a single service implement username-password authentication. However, each user has different permissions on various applications, which means that each user requires a different username and password for each application. In this condition, users must enter different usernames and passwords to access different applications. LDAP provides directory services to address this issue.

The purpose of LDAP-based authentication is to set up a directory-oriented user authentication system, specifically, an LDAP environment. When a client attempts to access applications in the LDAP environment, the LDAP server compares the username and password sent by the client with the authentication information in the directory database for identity verification.

For storage applications, client hierarchy information is stored on the LDAP server, and users are authenticated by the LDAP server when accessing the storage system.

Procedure

  1. Choose System > Security > LDAP Service Settings.
  2. Enable LDAP Service.

    If LDAP Service has been enabled, click Modify in the upper right corner.

  3. Set the LDAP binding parameters listed in Table 1.

    Table 1 LDAP binding parameters

    Parameter

    Description

    Server Type

    In 1.6.0 and later versions: Available options include Windows AD domain server and LDAP server.

    In 1.5.0 and earlier versions, only LDAP server is available.

    Client hierarchy information is stored on the domain authentication server. Users are authenticated by the server when they attempt to access shared resources.

    Protocol

    Encryption protocol used for domain authentication. The value can be LDAP or LDAPS.

    NOTE:
    • LDAP over SSL (LDAPS) is used for communication between the system and the LDAP server. If the LDAP server supports SSL, you can select LDAPS.
    • Security risks arise if LDAP is selected. You are advised to select LDAPS.

    Binding DN

    Binding directory on the server.

    Binding is a process that a client initiates a connection request to establish a session to the LDAP server. During binding, the client needs to specify accounts used to access directory information on the server. You must search the binding directory for desired content.

    [Example]

    cn=My Application,ou=applications,dc=bigcorp,dc=com

    NOTE:

    The default access account is the administrator account. If you use another account, ensure that it has the access permission on the domain service on the LDAP server.

    Binding Password

    Password for accessing the directory corresponding to the binding DN. The value contains 1 to 63 characters.

    CN Verification

    After this function is enabled, the CN field of the LDAP server certificate is verified. If the CN field is inconsistent with the value of Address Type, the LDAP server connectivity check will fail.

    NOTE:

    This parameter is displayed only when 1.6.0 or later is used and Protocol is set to LDAPS.

    Address Type

    The LDAP domain server of the management plane can be configured by IP address or Domain name.

    NOTE:
    • By default, the domain name is automatically generated by the storage system based on the value of Binding DN. You can also enter a user-defined domain name.
    • If Address Type is set to Domain Name, you need to configure the DNS service in advance. For details, see Configuring the DNS Service.

    IP Address

    IPv4 or IPv6 address to be added to the LDAP server. Enter an IP address and click .

    NOTE:
    • To remove an IP address, click on its right.
    • You can enter a maximum of four IP addresses.
    • This parameter is displayed only when Address Type is IP Address.

    Port

    Port number of the server. The value ranges from 1 to 65535.

    Default port number of the LDAP server: 389

  4. Set LDAP user parameters. Table 2 describes the parameters.

    Table 2 LDAP user parameters

    Parameter

    Description

    Directory

    Path of the created domain user.

    Name Attribute

    Name attribute of the user. This parameter defines the name of the user object and allows the query of a specific user based on the given name.

    Object Type

    Type of the user object. Each entry under the LDAP directory is associated with one or more object types, including user, group, email, and maintenance terminal.

  5. Enable LDAP user group and set LDAP parameters. Table 3 describes the parameters.

    Table 3 LDAP user group parameters

    Parameter

    Description

    Directory

    Path of the created domain user group.

    Name Attribute

    Name attribute of the group. This parameter defines the name of the group object and allows the query of a specific group based on the given name.

    Member Attribute

    Group member attribute. This parameter defines members of a group.

    Object Type

    Type of the group object. Each entry under the LDAP directory is associated with one or more object types, including user, group, email, and maintenance terminal.

  6. (Optional) Click Test in the upper right corner to test the availability of the domain authentication server.

    When Protocol is LDAPS, you must import the CA certificate before performing the test.

  7. Click Save. Confirm the information as prompted.
