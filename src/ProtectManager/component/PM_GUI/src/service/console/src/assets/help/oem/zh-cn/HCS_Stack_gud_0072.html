全局搜索副本数据

通过本节操作您可以对副本数据进行全局浏览、搜索。

背景信息

副本数据全局搜索功能支持的应用包括：FusionCompute/VMware/华为云Stack/HDFS/NAS文件系统/NAS共享/对象存储/文件集/OpenStack（仅1.6.0及后续版本）/卷/CNware/Hyper-V/阿里云/FusionOne Compute/NDMP/Nutanix。在使用本节内容，如果您只涉及其中部分应用，请忽略其他应用的相关描述。

前提条件

  • 当需要对副本进行浏览或搜索时，要求在副本生成前已开启自动索引或者在生成副本后手动创建了索引。
  • 对于VMware、FusionCompute、华为云Stack、OpenStack、CNware、Hyper-V或FusionOne Compute副本，支持全局搜索的Linux文件系统类型为ext 2/3/4、XFS，支持全局搜索的Windows文件系统类型为NTFS、FAT、FAT32。

操作步骤

  1. 单击页面上方的，进入“全局搜索”页面。

    1. 在搜索栏选择“副本数据”。
    2. 输入副本中文件对象（文件、目录或链接）的名称或路径并单击，支持模糊搜索。

      当前页面将展示满足条件的文件对象。

      当同时输入多个条件进行搜索时，界面会将至少满足其中一个条件的搜索结果展示出来，并且与搜索条件匹配度越高的搜索结果展示越靠前。

  2. 在界面左侧导航树上可以根据需要对展示结果进行再次筛选。

    参数间是“或”的关系，即筛选出来的结果是所选条件的并集。

    表1 搜索参数说明

    参数

    说明

    资源类型

    选择要搜索的资源类型。

    文件对象类型

    选择“全部”、“文件”、“目录”或“链接”进行过滤。

    资源名称

    输入资源名称。

    文件对象绝对路径

    输入文件对象的绝对路径。

  3. 完成搜索后页面右侧展示资源的搜索结果。相关参数说明如表2所示。

    备份VMware、FusionCompute、华为云Stack、OpenStack、CNware、Hyper-V或FusionOne Compute时，未写入/etc/fstab文件的挂载点信息存储在内存中，本产品不会备份该内容，也不支持文件级恢复。如新增的文件系统需支持文件级恢复，请将新增的文件系统挂载点信息写入/etc/fstab。

    表2 副本数据参数说明

    参数

    说明

    文件对象

    名称

    副本中文件、目录或链接的名称。

    绝对路径

    文件、目录或链接的绝对路径。

    类型

    文件对象类型。

    • ：目录
    • ：文件
    • ：链接

    修改时间

    生成副本时，副本中的文件、目录或链接的最后修改时间。

    资源

    名称

    副本数据对应的源资源的名称。

    类型

    资源的类型。

    索引副本

    -

    副本数据中文件、目录或链接的索引副本数量。

    • 如果备份前开启了自动索引，则每次自动备份时，都会检查备份副本中的文件或目录等是否有更新，如果有更新则为文件或目录建立新的索引，索引副本数量为1，如果没有更新，则索引副本数量+1。
    • 如果是在管理界面手动创建的索引，则系统为备份副本中的文件或目录等建立新的索引，索引副本数量为1。

    单击索引副本，可以查看资源的副本数据，界面展示的副本均是已对该对象建立索引的副本。

  4. 在搜索结果所在行，单击“更多”，可以下载文件或进行文件级恢复。

    • 对于FusionCompute、华为云Stack、HDFS、对象存储、OpenStack、卷、CNware、Hyper-V、阿里云、FusionOne Compute，不支持下载副本文件。
    • 对于NAS共享和文件集，聚合格式的副本文件不支持下载。
