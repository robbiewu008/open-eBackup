Step 2: Creating an Archive SLA for Backup Copies

If you need to archive the generated backup copies, configure the archive SLA by referring to this section.

Prerequisites

An SLA with a backup policy has been created. For details about how to create a backup SLA, refer to Step 6: Creating a Backup SLA.

Procedure

  1. Choose Protection > Protection Policies > SLAs.
  2. Locate the SLA for which you want to create an archive policy and choose More > Modify.

    The preset SLAs cannot be modified.

  3. Click the Archive Policy icon.
  4. Configure the archive policy information.

    Click Add on the right of the page to create multiple archive policies. A maximum of four archive policies can be created, and data can be archived to a maximum of four archive storage devices.

    Table 1 describes the related parameters.

    Table 1 Archive parameters

    Parameter

    Description

    Name

    The system has set a default name for the archive policy. You can also customize a name for the archive policy.

    Archive Storage Type

    The type can be Object Storage or Tape Library.

    Archive Storage
    • If Archive Storage Type is set to Object Storage, select added object storage. You can also click Add to add object storage.
    • If Archive Storage Type is set to Tape Library, select a node.

    Media Set

    If Archive Storage Type is set to Tape Library, select a media set. You can also click Add to create a media set.

    Archive Time
    Set the archive time.
    • Periodically
      • Interval: Data is archived every xx minutes, hours, days, or weeks.
      • First Run At: specifies the time when the archive job is executed for the first time.

        If the start time is earlier than the current device time, the archive job is executed immediately after a protection plan is created.

    • Immediately after successful backup

      An archive job is executed immediately after a backup job is complete.

    • Specified time after copy generation

      Time: An archive job is executed xx days after a copy is generated. The value ranges from 1 to 365 days.

    Archive & Retention Rules

    Select the backup copy that you need to archive and set the retention period of the backup copy.

    NOTE:
    • If Archive Storage Type is set to Tape Library, only archive rules can be configured. The retention rule is subject to the retention policy of the tape.
    • If multiple options are selected, copies on any date specified by these options are archived, and the obtained archive copies are retained for the longest period specified in these options.
    • Archive specified copies: An archive policy can be specified and the copy retention period can be set. Archive policies are described as follows:
      • Only the latest copy

        In each archive job, only the copy generated by the last backup job is archived.

      • All unarchived copies

        All copies that have not been archived are archived each time.

    • Archive on the specified date: Specify the date and set the copy retention period. You can select multiple of the following options:
      • Select the backup copies generated in month x of each year for archiving. The archive copies are retained for x years (the retention period can be 1 to 70 years).
      • Select the first or last backup copy of each month for archiving. The archive copies are retained for x months (the retention period can be 1 to 840 months).
      • Select the backup copies generated on a specified day of each week for archiving. The archive copies are retained for x weeks (the retention period can be 1 to 3650 weeks).
        NOTE:

        If Archive Storage Type is set to Tape Library, the retention period of archive copies cannot be specified, but is subject to the retention policy setting of the corresponding media set.
    Drivers
    NOTE:

    This parameter is available only in 1.6.0 and later versions.
    This parameter (value range: 1 to 32) is displayed only when you select Tape Library for Archive Storage Type.
    NOTE:

    The larger the number of drives, the more the tape library drives occupied by archive jobs or jobs that use archive copies for restoration, and the better the performance.

    Rate Limiting Policies

    Average maximum amount of data that can be transmitted per second in an archive job. If a tape library serves as the archive storage, no rate limiting policy can be configured.

    Select the created rate limiting policy. For details about how to create a rate limiting policy, see Step 4: Creating a Rate Limiting Policy.
    Automatic Indexing
    NOTE:

    This parameter is available only in 1.5.0.

    For filesets, NAS file systems, and NAS shares, this parameter is displayed when Archive Storage Type is set to Object Storage. This parameter is not displayed for other applications.

    This function cannot be disabled after it is enabled. The system automatically creates file indexes for copies. After indexes are created, one or more files can be restored.

    NOTE:

    Indexes occupy backup storage space. If the number of index files in the system exceeds 1 billion, perform operations as instructed in "Failed to Create a Copy Index" in the Troubleshooting.

    Network Acceleration

    After this function is enabled, Bottleneck Bandwidth and Round-trip propagation time (BBR) network acceleration can be used. You are advised to enable this function when the network latency is greater than or equal to 20 ms. If a tape library serves as the archive storage, network acceleration cannot be configured.

    Job Failure Alarm

    After this function is enabled, an alarm is automatically sent if a job fails and is automatically cleared after successful execution.

    Automatic Retry

    After this function is enabled, the system automatically retries if a periodic archive job fails.

    The number of retries ranges from 1 to 3, and the waiting time ranges from 1 to 30 minutes. For example, if the number of retries is set to 3 and the wait time is set to 5 minutes, the system retries for a total of three times at an interval of 5 minutes.

    NOTE:

    During automatic retry, the system creates an archive job.

  5. Click OK and confirm as prompted.
  6. Associate an archive SLA with resources to periodically archive copies.

    • If an SLA policy has been associated with resources, the system protects the resources based on the SLA policy.
    • If the SLA policy is not associated with any resource, associate a resource with the SLA policy for protection. For details, see Step 7: Performing Backup.
