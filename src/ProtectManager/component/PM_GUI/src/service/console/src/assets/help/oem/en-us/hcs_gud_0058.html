Viewing Huawei Cloud Stack Copy Information

This section describes how to view Huawei Cloud Stack copy information.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Huawei Cloud Stack.
  2. Select a copy viewing mode to view the copy information.

    Replication copies can be viewed only in the destination cluster.

    • Resources: View resource and copy information by cloud server. When you need to restore a faulty cloud server, you are advised to search for the target copy by clicking this tab.

      Table 1 describes the related parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Resource ID

      Resource ID.

      Name

      Cloud server name.

      Click the name of a cloud server to view its copy information.

      You can search for copies by year, month, or day.

      If is displayed below the time, copies exist at that time.

      Location

      Cloud server location. The location format is as follows: Huawei Cloud Stack platform/Tenant/Region/Project (resource set)/Cloud server.

      Status

      Whether a cloud server is registered with the current management page.

      • If yes, Status is Existing.
      • If no, Status is Not existing.

      Copy

      Quantity

      Number of copies generated based on a cloud server. The number of replication copies is displayed only in the target cluster.

      Replicated Copy SLA

      Archive SLA associated with a replication copy.

      If an archive SLA has been configured for the replication copy generated by the resource, the SLA name is displayed.

      Replicated Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is enabled.
      • Creating: The replication copy is associated with an SLA and protection is being created. The status is available only in 1.5.0 and earlier versions.
    • Copy: View information about all copies by copy. When you need to restore data to the original location or new location based on a specified copy, you are advised to click this icon and then search for the copy.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copy

      Backup Time

      Time when a backup copy is generated.

      Copy Time

      Time when a copy is generated.

      Click Copy Time to view the expiration time and other information of the copy. If -- is displayed as the expiration time, the copy is retained permanently.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being recovered.
      • Verifying: The copy is under integrity check.
      • Deletion failed: The copy fails to be deleted.

      Location

      Copy location.

      • For 1.5.0: If Generation Mode of a copy is Backup, Cascaded Replication, Reverse Replication, or Replicate, the copy is stored in the local cluster. In this case, Local is displayed.
      • For 1.6.0 and later versions: If Generation Mode of a copy is Backup, Cascaded Replication, Reverse Replication, or Replicate, the copy is stored in the corresponding backup storage unit. In this case, the name of the corresponding backup storage unit is displayed.
      • If Generation Mode of a copy is Object Storage Archive or Import, the copy is stored in an object storage bucket. In this case, the bucket name is displayed.
      • If Generation Mode of a copy is Tape Archive, the copy is stored in a tape library. In this case, the media set name is displayed.

      Generation Mode

      Mode in which a copy is generated.

      • Replicate: copies generated by a replication job.
      • Backup: copies generated by backup jobs.
      • Object Storage Archive: copies generated by object storage archive jobs.
      • Tape Archive: copies generated by tape archiving jobs.
      • Cascaded Replication: copies generated by cascaded replication jobs.
      • Reverse Replication: copies generated by reverse replication jobs.

      Copy Type

      Backup type of a generated copy. The value can be Full or Forever Incremental (Synthetic Full).

      Expired At

      Copy expiration time. If a copy is permanently valid, -- is displayed.

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.
      • : The copy has expired. This state is available only in 1.60 and later versions. It indicates that the WORM copy has expired. Copies in this state can be deleted.
      WORM Expiration Time
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.

      Copy Verification Status
      Verification status of the copy.
      • --: indicates that the current copy is not verified.
      • Available: indicates that the current copy has been verified and the data is correct.
      • Invalid: indicates that the current copy verification job fails or the copy data is inconsistent.

      Resources

      Name

      Cloud server name.

      Location

      Cloud server location.

      Status

      Whether a cloud server is registered with the current management page.

      • If yes, Status is Existing.
      • If no, Status is Not existing.
      WORM Expiration Time
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.
