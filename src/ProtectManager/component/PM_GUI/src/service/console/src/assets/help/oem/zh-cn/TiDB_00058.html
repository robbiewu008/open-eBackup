查询TiDB信息

操作步骤

  1. 选择“保护 > 数据库 > TiDB”。
  2. 在“集群”页签，查询TiDB集群信息。

    1. 该页面展示已注册的TiDB集群信息。

      相关参数说明如表1所示。

      表1 TiDB集群信息

      参数

      说明

      名称

      TiDB集群名称。

      状态

      TiDB集群的状态。

      • 在线：当代理主机与TiDB集群连接正常时，“状态”为“在线”。
      • 离线：当代理主机与TiDB集群连接异常时，“状态”为“离线”。

      版本

      TiDB集群的数据库版本。

      SLA

      当备份资源已关联SLA时，则显示SLA名称，否则显示“--”。

      SLA遵从度
      SLA遵从度指某次备份任务是否按设定的备份频率或备份窗口成功完成备份。
      • 遵从：如果某次备份任务（首次全量备份除外）的备份时间满足备份窗口或备份频率要求，且备份成功，则表示SLA遵从。
      • 不遵从：如果备份时间不满足备份窗口或备份频率要求，例如某次备份任务SLA设定的备份频率为每半小时执行一次，最终备份时间超过半小时，则表示SLA不遵从。建议调整备份频率或备份窗口。

      保护状态

      TiDB集群当前是否被保护。

      • 未保护：集群未关联SLA，或者集群已关联SLA但已禁用保护。
      • 已保护：集群已关联SLA，且已激活保护。
      • 创建中：集群已关联SLA，正在创建保护。
      标签
      说明：

      仅1.6.0及后续版本存在此参数。

      当资源已添加标签时，则此处显示标签名称。否则，显示“--”。
      所属用户
      说明：

      仅1.5.0版本存在此参数。

      该集群所属的数据保护管理员。

      当该集群已授权给数据保护管理员时，将显示数据保护管理员的名称。如果未进行授权，则显示“--”。
    2. 单击集群名称，查看总览信息、副本信息以及相关任务。
      • 总览

        显示集群名称、集群版本、所属用户、内部集群、代理主机节点、TiUP节点、数据节点信息等。

      • 副本数据

        您可以依次按照年、月、天查找相应时间的副本。

        时间下方显示，表示该时间存在副本。

      • 任务

        展示备份、恢复等任务，包括执行中的任务和历史任务。

  3. 在“数据库”页签，查询TiDB数据库信息。

    1. 该页面展示备份资源的详细信息。
      相关参数说明如表2所示。
      表2 备份资源参数说明

      参数

      说明

      名称

      数据库名称。

      状态

      TiDB数据库的状态。

      • 在线：当代理主机与TiDB数据库连接正常时，“状态”为“在线”。
      • 离线：当代理主机与TiDB数据库连接异常时，“状态”为“离线”。

      所属集群

      受保护的TiDB数据库所属集群名称。

      SLA

      当备份资源已关联SLA时，则显示SLA名称，否则显示“--”。

      保护状态

      TiDB数据库当前是否被保护。

      • 未保护：数据库未关联SLA，或者数据库已关联SLA但已禁用保护。
      • 已保护：数据库已关联SLA，且已激活保护。
      • 创建中：数据库已关联SLA，正在创建保护。
      标签
      说明：

      仅1.6.0及后续版本存在此参数。

      当资源已添加标签时，则此处显示标签名称。否则，显示“--”。
    2. 单击数据库名称，查看总览信息、副本信息以及相关任务。
      • 数据库

        显示数据库基础信息、SLA信息、任务执行时间、副本总数等。

      • 副本数据

        您可以依次按照年、月、天查找相应时间的副本。

        时间下方显示，表示该时间存在副本。

      • 任务

        展示备份、恢复等任务，包括执行中的任务和历史任务。

  4. 在“表集”页签，查询TiDB数据库中的表集信息。
