Querying Information About the FusionCompute Virtualization Environment

After the FusionCompute virtualization environment is registered with the OceanProtect, you can view details about FusionCompute clusters, hosts, VMs, or VM groups on the management page.

Context

GUIs may vary with versions. The actual GUIs prevail.

Procedure

  1. Choose Protection > Virtualization > FusionCompute.

    Figure 1 shows the overall information about the FusionCompute virtualization environment.

    Figure 1 Overall information about the FusionCompute virtualization environment

  2. On the left of the page, select the FusionCompute virtualization environment to be viewed and choose More > Environment Info to view the basic information and agent host of the FusionCompute virtualization environment.
  3. On the Cluster tab page, view the cluster information.

    1. For details about the cluster, see Table 1.
      Table 1 Cluster information

      Parameter

      Description

      Name

      Cluster name.

      SLAs

      If the entire cluster is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Indicates whether a cluster is protected.

      • Unprotected: The cluster is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The cluster is associated with an SLA and protection is enabled.
      • Creating: The cluster is associated with an SLA and protection is being created.
    2. Click the cluster name to view the cluster overview and related jobs.
      • Cluster overview

        Displays the cluster name, SLA information, host list, and VM list.

        The host list contains the host name, SLA, SLA compliance, and protection status.

        The VM list contains the VM name, status, SLA, SLA compliance, and protection status.

        Click Operation on the right of the host or VM name to protect or manually back up the host or VM or to modify, remove, activate or disable the protection for the host or VM.

      • Cluster job

        Displays all cluster-related jobs, whose types include Protect Resource, Modify Protection, and Copy Expiration. You can choose to display ongoing or historical jobs.

  4. On the Hosts tab page, view the host information.

    1. Table 2 shows the details of the host.
      Table 2 Host information

      Parameter

      Description

      Name

      Host name

      Location

      Host path.

      SLAs

      If an SLA is associated with a host, the SLA name is displayed. Otherwise, -- is displayed.

      Indicates whether a host is protected.

      • Protected: The host is associated with an SLA and protection is enabled.
      • Creating: The host is associated with an SLA and protection is being created.
    2. Click the host name to view the host information and related jobs.
      • Host overview

        Displays the host name, SLA information, and VM list.

        The VM list contains the VM name, status, SLA, SLA compliance, and protection status.

        Click Operation on the right of the VM to protect or manually back up the VM or to modify, remove, activate or disable the protection for the VM.

      • Host task

        Displays all host-related jobs, whose types include Protect Resource, Modify Protection, and Copy Expiration. You can choose to display ongoing or historical jobs.

  5. View the VM information on the VMs tab page.

    1. View information about a VM. Table 3 shows the information.
      Table 3 VM information

      Parameter

      Description

      Name

      VM name.

      Location

      VM path.

      Status

      Indicates that the VM is running, stopped, or abnormal.

      VM Group

      Name of the VM group to which the VM belongs.
    2. Click the VM name to view the overall VM information, copy information, and related jobs.
      • VM overview

        You can view the name, SLA information, job execution time, total number of copies, and disk information of a VM.

      • VM copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, a copy at that time exists.

      • VM job

        Displays all jobs related to the VM, including backup, archive, and restoration. You can choose to display jobs being executed or historical jobs.

    Only 1.6.0 and later versions support this function.
