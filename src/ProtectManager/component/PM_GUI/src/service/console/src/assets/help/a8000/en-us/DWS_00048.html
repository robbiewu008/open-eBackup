Restoring the GaussDB (DWS)

This section describes how to restore GaussDB (DWS) that has been backed up to the original or a new location.

Context

Backup copies, reverse replication copies, cascaded replication copies, archive copies, and replication copies can be used for restoration. GaussDB (DWS) can be restored to the original location or a new location. Replication copies and their archive copies, cascaded replication copies and their archive copies cannot be used for restoration of the GaussDB (DWS) to the original location

If the backup copy and archive copy coexist, use the backup copy first for restoration.

Prerequisites

  • Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.
  • Before table-level restoration to a new schema, ensure that the schema has been created in the database corresponding to GaussDB (DWS). Otherwise, the restoration fails.

Procedure

  1. Choose Explore > Copy Data > Big Data > GaussDB(DWS).
  2. You can search for copies by GaussDB (DWS) resource or copy. This section uses resources as an example.

    On the Resources tab page, locate the resource to be restored by resource name and click the name.

  3. Select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, a copy is generated in that month or day.

  4. Restore data using a specific copy.

    1. Common restoration
      1. In the row where the copy to be restored resides, choose More > Restore.
      2. Restore the GaussDB (DWS) to the original location or a new location.
        Table 1 describes the related parameters.
        Table 1 Restoring the GaussDB (DWS)

        Parameter

        Description

        Restore To

        Selects either Original location or New location.

        Location

        If you select Original location for the Restore To parameter, the original location is displayed.

        Target Cluster
        If you choose to restore the GaussDB (DWS) to New location, you must choose the target cluster to be restored. Ensure that the installation configuration file named mppdb-install-config.xml of the target cluster has been replicated to the /opt/huawei/ directory on each node in the target cluster and that the database user has the read permission. Otherwise, the restoration will fail. Note that if the installation configuration file is not named mppdb-install-config.xml, rename it.
        NOTE:

        If the database user does not have the read permission on the configuration file, log in to each node in the target cluster as user root and run the following command:

        sudo g+r /opt/huawei/mppdb-install-config.xml

        Databases

        When Restore To is set to New location for restoration using table set copies, you need to select the target database for restoration.
      3. Click OK and complete the configuration as prompted.

        Ensure that the restoration target location is correct and the remaining space of the restoration target location is greater than the size of the copy used for restoration before reduction.

    2. Table-level restoration
      1. In the row where the copy used to perform table-level restoration resides, choose More > Table-level Restoration.
      2. In Selected Copy, select the tables to be restored.
      3. Select the target location for restoration. Table 1 describes the related parameters.
      4. Select the database to be restored to the target location.
      5. Click OK and complete the configuration as prompted.
      • Ensure that the restoration target location is correct and the remaining space of the restoration target location is greater than the size of the copy used for restoration before reduction.
      • For table-level GDS-based restoration, the number of GDSs started at the target end must be greater than or equal to the number of GDSs started at the source end during backup.
      • For 1.6.0 and later versions, table-level restoration to a new location is supported.
