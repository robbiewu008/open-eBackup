Performing a Detection Periodically

After associating the file systems that require intelligent detection with a specified intelligent detection policy, the system performs intelligent detection on these file systems based on the intelligent detection policy.

Precautions

For secondary file systems in HyperMetro domain scenarios, OceanCyber does not execute the periodic intelligent detection policy. OceanCyber administrator needs to configure the periodic intelligent detection policy for the primary file system.

Procedure

  1. Choose Data Security > Intelligent Detection.
  2. Click the File Systems tab.
  3. In the row of the target file system, choose More > Protect.

    You can also select multiple file systems and choose More > Protect in the upper part.

  4. Select an existing intelligent detection policy or click Create to create an intelligent detection policy.
  5. Select Learning Mode. You are advised to enable this function so that you can better evaluate the data characteristics of backup copies.

    • If the type of the application to be backed up is file system copies, this function is valid only when the backup software is NetBackup or Commvault, the storage device is the OceanProtect, and the in-depth detection function of backup copies is enabled. If this function is disabled, false detection results may be generated during in-depth copy detection.
    • If the type of the application to be backed up is VMware, Oracle, MySQL, or host files, this function is valid only when the backup software is NetBackup, Veeam, or Commvault, the storage device is the OceanProtect, and the in-depth detection function of backup copies is enabled. If this function is disabled, false detection results may be generated during in-depth copy detection.

    After Learning Mode is enabled, you can select By duration or By times, and set the learning duration or times.

    • Learning by duration xx days indicates that the snapshot data of the xx days is selected as the dataset for learning. Learning by times xx indicates that the snapshot data of the xx times is selected as the dataset for learning.
    • The set learning duration (learning times) is the minimum learning duration (minimum number of learning times). To evaluate the data features of backup copies based on a required amount of data, the actual learning duration (learning times) may be longer than the configured learning duration (greater than the configured learning times).
    • You are advised to perform learning by duration, which ensures that a file system can obtain the required amount of data to be learned.
    • If the learning mode is switched from By duration to By times or from By times to By duration, the system performs the learning job again.

  6. Click OK.

    If the current system time is later than the start time of the ransomware detection snapshot generated for the first time based on the intelligent detection policy, you can perform the detection immediately or wait for the next scheduling based on the intelligent detection policy after the intelligent detection policy is associated.
