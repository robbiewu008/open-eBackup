HDFS执行索引任务失败

现象描述

HDFS执行索引任务失败。

可能原因

执行索引任务前，客户端所在的主机空闲内存已耗尽，需要保证执行任务前客户端所在的主机内存未耗尽。执行索引任务前，建议预留4GB以上内存，确保索引任务能正常运行。

处理步骤

  1. 使用puTTY，登录客户端所在的主机。
  2. 执行free -g命令查询空闲内存，命令回显示例如下，如果内存的“free”列显示的值为0，则表示内存剩余可用空间为0。

                  total        used        free      shared  buff/cache   available
Mem:             62           2           0           0          60          60

  3. 根据2的回显内容，确认场景：

    • 如果内存剩余空间为0，则选择以下处理方式。
      • 关停同时运行的其他占用内存的进程。
      • 修改客户端配置文件，修改“/opt/DataBackup/ProtectClient/Plugins/HadoopPlugin/”cfg目录下的plugin.cfg文件中的ios.minThreads和ios.maxThreads参数值（“/opt”为客户端默认安装路径，用户需根据实际安装路径修改），按照当前实际业务性能减小参数值，降低并发线程数量。修改文件后，请重启客户端服务。

        配置文件内容示例如下图。

      • 扩展客户端所在主机内存，或者使用内存大的独立主机安装客户端。
    • 如果内存剩余空间不为0且剩余空间较多，请收集日志并联系华为技术支持工程师协助处理。
