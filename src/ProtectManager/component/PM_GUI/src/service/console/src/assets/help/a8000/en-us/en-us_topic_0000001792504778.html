Managing TiDB Cluster Protection

You can perform related operations for registered TiDB clusters on the OceanProtect.

Related Operations

Log in to the management page, choose Protection > Databases > TiDB, click the Cluster tab, and locate the desired cluster.

Table 1 describes the related operations.

Table 1 Cluster protection operations

Operation

Description

Navigation Path

Modify Protection

Scenario

This operation enables you to modify the protection plan to associate a cluster with a new SLA or other SLAs. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target cluster, choose More > Modify Protection.

Remove Protection

Scenario

This operation enables you to remove protection for a cluster when it does not need to be protected. After the removal, the system cannot execute protection jobs for the cluster. To protect the cluster again, you need to associate an SLA with it again.

Note

When the protection job of the cluster is running, protection cannot be removed.

In the row of the target cluster, choose More > Remove Protection.

Activate Protection

Scenario

This operation enables you to activate a disabled protection plan of a cluster. After the protection plan is activated, the system executes backup, archive, or replication for the database based on the plan.

In the row of the target cluster, choose More > Activate Protection.

Disable Protection

Scenario

This operation enables you to disable the protection plan of a cluster for which periodic protection is not required. After the protection plan is disabled, the system does not automatically back up, archive, or replicate the cluster data.

Note

Disabling protection does not affect ongoing protection jobs.

In the row of the target cluster, choose More > Disable Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target cluster, choose More > Manual Backup.

Test Connectivity

Scenario

This operation enables you to test the connectivity between the OceanProtect and the cluster.

In the row of the target cluster, choose More > Test Connectivity.

Modify

Scenario

After the cluster information is modified, you can perform this operation to synchronize the cluster registration information.

In the row of the target cluster, choose More > Modify.

Delete Cluster

Scenario

This operation enables you to delete a cluster if you no longer need to protect it.

Note

A cluster associated with an SLA cannot be deleted.

In the row of the target cluster, choose More > Delete.

Note: The procedure for restoring TiDB is the same as that described in Restoration.
