Oracle 12.1集群ASM恢复到Oracle 12.1单机非ASM环境后，备份Oracle 12.1单机非ASM时会卡住

现象描述

执行Oracle备份，查看Oracle trace日志，报错如下：

Starting background process ASMB
Thu Jul 20 04:52:37 2023
ASMB started with pid=89, OS id=13500518 
WARNING: failed to start ASMB (connection failed) state=0x1 sid=''
WARNING: ASMB exiting with error

可能原因

根据Oracle官方文档，此错误与MIGRATING DATABASE FROM ASM TO FILE SYSTEM有关。

生产数据库是RAC，使用ASM磁盘组，但恢复的目标数据库是非ASM，使用文件系统。因此，此错误与Oracle Bug 19503821有关，将数据库从ASM迁移到文件系统解决方案时，RMAN CATALOG极其缓慢。

处理步骤

  1. 停止Oracle数据库备份任务。
  2. 安装Oracle的19503821补丁。
