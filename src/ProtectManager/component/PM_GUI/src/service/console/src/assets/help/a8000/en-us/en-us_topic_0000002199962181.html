Adding Operation Restrictions on Infected Copies

This section describes how to add operation restrictions for infected copies. The following operations are forbidden on an infected copy: Restoration to the original location, Restoration to a new location, File-level restoration, Live mount, Instant recovery, Download and export, and Archive.

Procedure

  1. Choose Data Security > Ransomware Protection > Ransomware Protection and WORM Configurations.
  2. Click Operation Restrictions on Infected Copies.
  3. On the Operation Restrictions on Infected Copies tab page, click Add Restriction.
  4. In the dialog box that is displayed, select one or more operations forbidden on infected copies, including Restoration to the original location, Restoration to a new location, File-level restoration, Live mount, Instant recovery, Download and export, and Archive.
  5. Select application resources.

    • Click Backup Copies and select one or more application resources.
      • Virtualization: On the Virtualization tab page, you can select the names of target resources under VMware, FusionCompute, FusionOne Compute, CNware, or Hyper-V based on the resource type.
      • Cloud Platforms: On the Cloud Platforms tab page, select the names of target resources under Huawei Cloud Stack or OpenStack based on the resource type.
      • File Systems: On the File Systems tab page, you can select the names of target resources under NAS File Systems, NAS Shares, or Filesets based on the resource type.
    • Click Replication Copies and select one or more application resources.
      • Virtualization: On the Virtualization tab page, you can select the names of target resources under VMware, FusionCompute, FusionOne Compute, CNware, or Hyper-V based on the resource type.
      • Cloud Platforms: On the Cloud Platforms tab page, select the names of target resources under Huawei Cloud Stack or OpenStack based on the resource type.
      • File Systems: On the File Systems tab page, you can select the names of target resources under NAS File Systems, NAS Shares, or Filesets based on the resource type.

  6. Click OK.
