Restoring One or More Databases in an SQL Server Instance

This section describes how to restore one or more databases of a backed-up SQL Server instance to the original or a different host using copies.

Prerequisites

  • Before restoring data to a different host, ensure that a database instance exists on the target host.
  • Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.

Procedure

  1. Choose Explore > Copy Data > Databases > SQL Server.
  2. On the Copies tab page, find the copy for restoration based on the copy information.

    Database-level restoration supports the restoration of backup copies and replication copies of an instance.

  3. In the row of the desired copy, choose More > Database-level Restoration.
  4. On the Database-level Restoration tab page, select the database objects to be restored in the left pane and the target location for restoration in the right pane.
  5. Set the parameters for database-level restoration. For details, see Table 1.

    Table 1 Parameters for database-level restoration

    Parameter

    Description

    Restore To

    Select either Original location or New location.

    Location

    If Restore To is set to Original location, the original location is displayed by default.

    Tag

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    Select tag names to filter the target host or cluster.

    Target Host/Cluster

    If Restore To is set to New location, select the instance on the target host.

    NOTE:

    Before performing restoration, ensure that you have exited Microsoft SQL Server Management Studio. During restoration, the target database is shut down.

  6. Click OK.
