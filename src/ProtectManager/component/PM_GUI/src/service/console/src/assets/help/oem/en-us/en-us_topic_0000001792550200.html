Step 3: Registering the Dameng Database

Registering a Single-Server Dameng Database

  1. Choose Protection > Databases > Dameng.
  2. Click Register to register the Dameng Single Server database.
  3. Set Type to Single Server.
  4. Configure database information.

    Table 1 describes the related parameters.

    Table 1 Registration information of a single-server Dameng

    Parameter

    Description

    Name

    User-defined resource name.

    Hosts

    Database host to be registered.

    Port

    Listening port of the registered single-server database.

    Authentication Method

    OS authentication and Database authentication can be selected.

    • OS authentication: authentication based on the OS.
    • Database authentication: authentication through the database administrator username and password.

    Log in to the database to query the database authentication mode and determine which authentication mode should be selected based on the query result. The query method is as follows:

    1. Log in to the Dameng database.
    2. Run the select name, value from v$parameter where name = 'ENABLE_LOCAL_OSAUTH'; command to check the parameter value of VALUE.
      • If the value is 1, OS authentication is supported. The authentication mode can be OS authentication or Database authentication.
      • If the value is 0, OS authentication is not supported. Set the authentication mode to Database authentication.

    Database Username

    Username for logging in to the currently registered database.

    Database Password

    Password for logging in to the currently registered database.

  5. Click OK.

Registering the Dameng Cluster Database

  1. Choose Protection > Databases > Dameng.
  2. Click Register to register the Dameng Cluster database.
  3. Set Type to Active/standby MPP cluster.

    In 1.5.0, set Type to MPP Cluster.

  4. Customize a cluster name.
  5. Click Add to add all instances in the cluster.
  6. Click OK.
