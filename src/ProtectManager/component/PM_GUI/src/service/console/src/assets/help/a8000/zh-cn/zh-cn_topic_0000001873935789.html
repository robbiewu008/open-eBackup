安装ProtectAgent（自动推送方式，适用于1.6.0及后续版本）

您需要在代理主机上安装ProtectAgent软件，ProtectAgent充当OceanProtect与受保护资源之间的代理进行数据备份和恢复。本节介绍自动推送安装包至代理主机的方式为主机安装ProtectAgent。

前提条件

  • 代理主机上未安装ProtectAgent。
  • 请确保使用的操作系统支持以自动推送方式安装ProtectAgent，具体可参考兼容性查询工具。
  • 已获取需要安装ProtectAgent的代理主机的IP地址，该IP地址必须与OceanProtect的备份网络连通。
  • 已获取登录代理主机的用户及其密码。
  • 安装ProtectAgent时，当“用户类型”配置为“普通用户”时，需要先将该普通用户添加到sudoers中，否则注册ProtectAgent将失败，具体操作步骤可参考添加普通用户至sudoers。
  • 安装ProtectAgent时，当“操作系统”选择为“Windows”时，请确保代理主机上的Windows Remote Management (WS-Management)服务状态为“正在运行”，具体操作可参考如何启动Windows Remote Management (WS-Management)服务。
  • 部署OceanProtect时，系统会预置相应版本的ProtectAgent软件包至管理界面。如果预置的ProtectAgent软件包版本不满足您的需求，那么安装ProtectAgent前请先参考上传ProtectAgent软件包上传其他版本的ProtectAgent软件包。
  • 对于代理主机为Linux OS或Unix OS时，如果用户修改了“/etc/ssh/sshd_config”配置文件中的“ListenAddress”参数的值，请确保配置值中包含代理主机的备份网络IP地址，否则检查连通性失败，无法推送安装。即：“ListenAddress”的值要么是系统默认值，要么是用户自行设置的IP地址+代理主机的备份网络IP地址。

注意事项

受保护资源为GaussDB(DWS)时，如果您不需要配置防火墙，可以执行systemctl stop SuSEfirewall2.service命令关闭防火墙。

操作步骤

  1. 受保护资源是否为HDFS、HBase、Hive、ElasticSearch、Redis或ClickHouse集群，且集群中配置了Kerberos认证。

    • 是，安装前要确保代理主机、OceanProtect以及Kerberos服务器的时间一致。如果时间不一致，请使用同一NTP服务器，否则可能导致ProtectAgent安装失败。
    • 否，安装前要确保代理主机与OceanProtect设备时间一致。
      • 参考以下步骤查看OceanProtect设备时间：
        1. 选择“系统 > 基础设施 > 集群管理”。
        2. 在“备份集群”页签的“本地集群节点”区域，单击节点名称。
        3. 在弹出的“节点详情”对话框中，查看当前设备时间。

          如果时间不一致，请修改代理主机时间，否则可能导致ProtectAgent安装失败。

  2. 选择“保护 > 客户端”。
  3. 单击“注册”，配置相关参数。

    参数说明如表1所示。

    表1 安装代理参数

    参数

    说明

    代理类型
    根据受保护资源的类型选择代理类型。
    • VMware备份代理：当受保护资源为VMware虚拟机时，选择该类型。
    • 通用备份代理：除VMware外的受保护资源，选择该类型。
    • SAN Client代理：AIX或Solaris主机使用FC组网时，选择该类型。

    操作系统

    代理主机的操作系统类型。

    说明：

    当操作系统为Windows Server 2012或其之前的版本时，不支持自动推送方式，请使用手动方式安装ProtectAgent。具体操作请参考安装ProtectAgent软件（手动方式，适用于Windows OS）。

    应用类型

    当“代理类型”选择“通用备份代理”时，请选择代理主机支持保护的应用类型。

    消息签名算法

    当“操作系统”选择“Linux”、“AIX”或“Solaris”时，请选择消息签名算法。

    消息签名算法用于校验数据保护代理与本设备之间的连通性。hmac_sha2_256为安全性更高的算法，默认选择该算法，hmac_sha1为不安全算法。

    说明：

    所有参数配置完成后，单击“测试”，当提示待注册主机不支持指定安全加密算法套件时，请重新选择其他消息签名算法。如果两类算法均不支持，请升级代理主机的OpenSSH、SSL。

    共享

    开启后将代理应用于所有数据保护管理员。

    当“代理类型”选择“通用备份代理”，“操作系统”选择“Linux”，且应用类型选择HDFS、HBase、Hive、华为云Stack、华为云Stack GaussDB、GaussDB(DWS)或对象存储时，支持配置此参数。

    IP类型

    代理主机的IP地址类型，支持IPv4和IPv6。

    源端重删

    源端重删是删除源端数据中的冗余数据后再进行传输，可以减少传输的数据量，但会增加客户端的资源占用。

    如需开启，请确保代理主机空闲内存大于2GB。支持源端重删的操作系统请查看兼容性查询工具。

    说明：
    • 开启后配置源端重删功能，具体操作请参见对应版本的《源端重删用户指南》。
    • 开启源端重删功能后，请在首次备份前将源端重删用户加到sudoers中，具体操作请参见添加源端重删用户到sudoers。
    • 如果安装ProtectAgent时未开启源端重删功能，安装完ProtectAgent后又需开启源端重删功能时，则请参考安装OceanStor DataTurbo客户端（适用于1.6.0及后续版本）章节安装OceanStor DataTurbo客户端。

    IP地址（地址段）

    请输入代理主机的IP地址或地址段。支持配置代理主机的备份网络IP。

    端口
    代理主机的SSH端口和SFTP端口。
    • 操作系统为“Windows”时，请使用Windows远程管理服务侦听端口，默认值为“5985”。
    • 其他操作系统类型，请使用SSH服务侦听端口，默认值为“22”。
      说明：

      SSH服务侦听端口号与SFTP服务侦听端口号需保持一致，否则会导致注册失败。

    是否作为业务IP

    当“操作系统”为“Windows”时，可以选择是否将代理主机的IP地址作为业务IP，默认为“否”。

    用户类型

    当“操作系统”选择“Linux”或“AIX”时，请选择代理主机的用户类型。

    • 超级管理员：具有执行sudo命令权限的用户，注册代理主机时，只需要输入登录主机的用户名和密码。
    • 普通用户：不具有执行sudo命令权限的用户，注册代理主机时，需要输入sudo密码、登录主机的用户名和密码。
      说明：

      当“用户类型”为“普通用户”时，需要先将该普通用户添加到sudoers中，否则注册ProtectAgent将失败，具体操作步骤可参考添加普通用户至sudoers。

    sudo免密

    当“用户类型”为“普通用户”时，可以选择是否使用sudo免密，即普通用户在使用sudo权限时是否需要输入当前用户的密码。

    sudo密码

    当“用户类型”为“普通用户”时，需要输入sudo密码。

    用户名

    代理主机的用户名及密码。

    说明：
    • 当“操作系统”选择“Windows”时，输入的用户名请勿包含域名。

      例如：域账户为CHINA\Administrator，仅输入Administrator即可。

    • 当“操作系统”选择“AIX”，且“用户类型”为“普通用户”时，请确保输入的用户存在对应的用户目录。查询用户目录的操作如下：
      1. 使用PuTTY，登录代理主机。
      2. 执行以下命令，查询是否存在用户目录。
        cd /home/用户名

        如果回显No such file or directory，则表示没有对应的用户目录，请执行3创建。否则表示存在对应的用户目录。

      3. 执行以下命令，创建用户目录。
        mkdir /home/用户名

    密码

    安装目录

    代理主机安装的目录。

    • 操作系统为Linux、AIX或Solaris时，默认安装在“/opt”目录下。不支持安装在/tmp、/dev、/bin、/usr目录下，不能有[|;&$><`\!]+特殊字符。
    • 操作系统为Windows时，默认安装在“C:\”目录下。
    说明：
    • 系统会自动校验填入的安装目录的有效性，选择的安装目录权限需为755，且用户和属组为root。
    • 安装目录会保存在“/etc/profile”配置文件中，环境变量为“DATA_BACKUP_AGENT_HOME”，请不要随意更改此变量。

    自动同步主机名称

    注册时主机名称使用备份代理主机上报的名称。当“代理类型”选择“VMware备份代理”时，支持配置此开关。

    开启后，OceanProtect管理界面会自动同步代理主机名称的修改。关闭后，已注册客户端的代理主机名称不自动同步但支持修改。

  4. 单击“测试”。

    系统验证填写的代理主机IP地址、用户名、密码是否正确。

    • 在“客户端信息”区域，单击某条客户端信息后的“修改”或“删除”，可以修改或删除对应的客户端信息。
    • 在“客户端信息”区域，勾选多条客户端信息，单击“批量修改用户名和密码”，可以批量修改用户名和密码。

  5. 测试成功后单击“确定”。

    当“客户端”页面显示安装了ProtectAgent的主机后，表示该主机已成功注册至管理界面。

    • 代理主机注册成功后请等待5分钟再使用该代理主机。
