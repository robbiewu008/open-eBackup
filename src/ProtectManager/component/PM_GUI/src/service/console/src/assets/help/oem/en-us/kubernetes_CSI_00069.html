Uploading the Kubernetes Installation Package to the Image Repository

Before performing backup or restoration, you need to upload the Kubernetes installation package to the image repository. During backup or restoration, you need to use images to deploy the backup pod in the production environment for data migration during the running of backup or restoration jobs. Therefore, you need to upload the Kubernetes installation package to the image repository to deploy the backup pod.

Prerequisites

Before uploading the Kubernetes installation package, query the version and patch number on the product WebUI, and download the XXX_K8s_BackupImage.tgz installation package based on the queried version and patch number. Decompress the installation package to obtain the k8s_backup_image_x86.tar or k8s_backup_image_arm64.tar compressed package of the corresponding product architecture. You can obtain the Kubernetes installation package by using the following methods:

  • For enterprise users: Click here.
  • For carrier users: Click here.
    You can query the version and patch number in the following way:
    1. Click in the upper right corner and choose About.
    2. In the dialog box that is displayed, the version and patch number are displayed in the line where Version is located.

Procedure

  1. Log in to the FusionCompute Web Client.
  2. Click Container Management and click to switch to the containerized application management page.
  3. Choose Container Images > Local Repositories > library to switch to the library page.
  4. Click Upload to upload the compressed package which has been downloaded and decompressed to the library.
