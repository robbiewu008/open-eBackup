Step 2: Registering an OceanBase Cluster

Prerequisites

  • Before registering an OceanBase cluster, ensure that each host in the cluster has been registered.
  • Due to the restrictions of the NFS4.1 protocol, the host name in the same cluster must be unique.

Procedure

  1. Choose Protection > Databases > OceanBase.
  2. Click the Cluster tab.
  3. Click Register to register the OceanBase cluster.
  4. Configure the cluster information.

    Table 1 describes the related parameters.

    Table 1 Cluster parameters

    Parameter

    Description

    Name

    Customize a cluster name.

    Username

    Enter the username of the OceanBase database administrator.

    Password

    Enter the password of the OceanBase database administrator.

    Log Archive Interval

    Controls the interval for generating checkpoint jobs in cold partitions to promote the OceanBase log archive progress.

    OBClient Host

    Select the registered OBClient host.

    OBServer Host

    Select the registered OBServer host and enter its service IP address and port number. The default port number is 2881.

    If there are multiple OBServer hosts, click Add in sequence to add all hosts.

    Before registering the cluster, log in to the OBClient host and run the following command to check whether the preceding parameters are available: If the OBServer host can be properly connected after the command is executed, the parameter settings are correct.

    obclient -hOBServer host service IP address -PPort number -uAdministrator username -pAdministrator password -A

  5. Click OK.
