Restoring Huawei Cloud Stack GaussDB Instances

This section describes how to restore Huawei Cloud Stack GaussDB that has been backed up to the original location or a new location.

Context

Backup copies, reverse replication copies, cascaded replication copies, archive copies and replication copies can be used for restoration. Huawei Cloud Stack GaussDB can be restored to the original location or a new location. Replication copies and their archive copies, cascaded replication copies and their archive copies cannot be used for restoration of the Huawei Cloud Stack GaussDB to the original location.

If the backup copy and archive copy coexist, use the backup copy first for restoration.

Prerequisites

  • Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Huawei Cloud Stack GaussDB.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the instance to be restored by instance name and click the instance name.

  3. O the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, copies exist in the month or on the day.

  4. Find the copy to be restored and choose More > Restore on the right.
  5. Restore data to the original location or a new location.

    When restoring data to a new instance, you are advised to set the parameters to the same values as those of the original instance.

    • Restoration to the original location

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Location

      Location of the instance, which cannot be modified by users.
    • Restoration to a new location
      1. Set Basic Info.
        1. Set basic parameters. Table 2 describes the parameters.
          Table 2 Basic parameters

          Parameter

          Description

          Owning Project

          Project to which the instance to be restored to a new location belongs.

          Instance Name

          Name of the instance to be restored to a new location.

          AZ ID

          AZ ID. Deployment in the same AZ or across three different AZs is supported. Separate AZs by commas (,).

          • Deployment in the same AZ: enter three identical AZ IDs. For example, enter cn-north-4a,cn-north-4a,cn-north-4a.
          • Deployment across three different AZs: enter three different AZ IDs.

          Obtain the values from the enterprise administrator.

          Specification Code

          GaussDB specification code. You can log in to the Virtual Private Cloud (VPC) console. On the displayed Basic information page of the original instance, obtain the code from the value of Instance Specifications in DB Information.

          Disk Type

          Two disk types are supported: LOCALSSD and ULTRAHIGH.

          NOTE:

          The disk type must be the same as the original instance disk type.

          Disk Capacity

          Size of the disk space to be allocated to the to-be-restored instance. The value must be greater than or equal to that of the original instance. Set the disk capacity based on the deployment scheme:

          • MCS deployment scheme: The value must be a multiple of (Number of shards x 40), ranging from (Number of shards x 40 GB) to (Number of shards x 16 TB).
          • BMS deployment scheme: This parameter is automatically calculated based on the selected specification and cannot be specified. Even if this parameter is set, it does not take effect.
        2. (Optional) Click Advanced and set advanced parameters. Table 3 describes the parameters.
          Table 3 Advanced parameters in basic information

          Parameter

          Description

          Primary AZ ID

          Select one of the three AZ IDs. This parameter is mandatory only when the original instance has an independent arbitration AZ. This parameter is not required in finance edition (data computing) and other scenarios.

          Arbitration AZ ID

          Select one of the three AZ IDs. This parameter is mandatory only when the original instance has an independent arbitration AZ.
      2. Click Private Cloud Information and set parameters. Table 4 describes the parameters.
        Table 4 Private cloud information parameters

        Parameter

        Description

        VPC ID

        VPC ID. Method 1 is recommended.

        • Method 1: Log in to the VPC console and view the VPC ID in the VPC details.
        • Method 2: Call the API used to query VPC IDs. For details, see Querying VPCs.

        Subnet ID

        Network ID of the subnet. Method 1 is recommended.

        • Method 1: Log in to VPC console and click the target subnet on the Subnets page. You can view the network ID on the displayed page.
        • Method 2: Call the API used to query subnet IDs. For details, see Querying Subnets.

        Security Group ID

        Security group to which the instance belongs. Method 1 is recommended. If you do not need to specify a security group, contact customer service to apply for a whitelist.

        • Method 1: Log in to the VPC console and view the security group ID in the security group details.
        • Method 2: Call the API used to query security group IDs. For details, see Querying Security Groups.
      3. Click Database Information and set parameters.
        1. Set basic parameters. Table 5 describes the parameters.
          Table 5 Parameters in the database information

          Parameter

          Description

          Database Password

          Database password. Enter a strong password to improve resilience, preventing resilience risks such as brute force cracking.

          Port

          Currently, the default GaussDB port is 8000 and cannot be changed.
        2. (Optional) Click Advanced and set advanced parameters. Table 6 describes the parameters.
          Table 6 Advanced parameters in database information

          Parameter

          Description

          Parameter Group ID

          Parameter group ID. For details about how to obtain the value, see Obtaining Parameter Templates. If this parameter is not specified, the default parameter template is used.

          Payment Method

          Payment method of an instance. Unspecified or pay-on-demand is supported.

          Automatic Instance Backup Policy

          Automatic backup policy for a Huawei Cloud Stack GaussDB instance. When the database backup mode is changed from XBSA backup to automatic backup, this field determines the backup frequency.

          UTC Time Zone

          The value ranges from UTC-11:00 to UTC+12:00 at the full hour. For example, the parameter can be UTC+08:00 rather than UTC+08:30.

          NOTE:

          If this parameter is not specified, the UTC time is used for GaussDB by default.

  6. Click OK.
