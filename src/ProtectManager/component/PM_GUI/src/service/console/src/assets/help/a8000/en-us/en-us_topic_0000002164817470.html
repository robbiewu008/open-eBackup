Enabling the gaussdb_feature_supportSyncBackupCrossSite_pcs-lite Whitelist

Before cross-project/TPOPS instance restoration, enable the gaussdb_feature_supportSyncBackupCrossSite_pcs-lite whitelist to ensure that the restoration function is normal.

Procedure

  1. Use an SSH tool to log in to the active TPOPS server node as user root through the management IP address of the node.
  2. On the TPOPS node, run the following command to log in to the TPOPS node as user dbadmin:

    su - dbadmin

  3. Run the following commands to connect to the TPOPS metadata database. The default password DBS_guass12#$ is used as an example.

    source gauss_env_file
gsql -p 8635 -U core -W DBS_guass12#$ -q core -h 127.0.0.1

  4. Run the following command to check whether the whitelist is enabled:

    select * from dbs_feature where id = 'gaussdb_feature_supportSyncBackupCrossSite_pcs-lite';

    If status is open in the command output, the whitelist is enabled. Otherwise, go to 5.

  5. Run the following command to enable the whitelist:

    update core.dbs_feature set status = 'open' where id = 'gaussdb_feature_supportSyncBackupCrossSite_pcs-lite';
