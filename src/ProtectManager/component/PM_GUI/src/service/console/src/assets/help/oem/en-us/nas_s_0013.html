Step 2: Adding a Storage Device

Before backing up or restoring a NAS file system, add a storage device where the NAS file system to be protected resides to the product.

Context

The product can protect a NAS file system on the following storage devices:

  • OceanStor Dorado V7
  • OceanStor V7
  • OceanStor Dorado 6.x
  • OceanStor 6.x
  • OceanProtect 1.3.0 and later versions

Precautions

  • If a storage device is earlier than OceanStor Dorado 6.1.3, a NAS file system cannot be discovered or protected by adding the storage device. Instead, a NAS share needs to be registered to protect the NAS file system. For details about how to register a NAS share, see Step 5: Registering a NAS Share (Applicable to Storage Devices Except OceanStor V5/OceanStor Pacific/NetApp ONTAP).
  • When a storage device is OceanStor Dorado 6.1.3 or later, or OceanStor 6.1.3 or later, ensure that the activated license of the storage device contains "HyperReplication".

Procedure

  1. Choose Protection > File Systems > Storage Devices.

    For 1.5.0, choose Protection > File Services > Storage Device.

  2. Click Add Device to add a storage device.
  3. Select a device type.

    • OceanStor Dorado V7
    • OceanStor V7
    • OceanStor Dorado 6.x
    • OceanStor 6.x
    • OceanProtect 1.3.0 and later versions

  4. Configure parameters as required. Table 1 describes the related parameters.

    Table 1 Parameters for adding a device

    Parameter

    Description

    Device Name

    Name of a user-defined device.

    IP Address

    Management IP address of the storage device.

    Port

    Number of the port for communication between the product and the management plane of the storage device.

    The default port number is 8088.

    Username

    Username and password of a storage device administrator. Ensure that the RESTful login mode is available for the user and the password has been initialized.

    NOTE:

    Use an administrator account. Otherwise, resources on the device may fail to be backed up.

    Password

    Certificate verification

    Enabled by default. Import the storage device certificate obtained in Step 1: Obtaining the CA Certificate of a Storage Device to ensure the security of interactions between the product and storage device. After this function is disabled, security risks may exist in the communication between the system and the storage device. Exercise caution when performing this operation.

    Specify Replication Ports

    This function is disabled by default. When it is enabled, you can select logical replication ports for NAS file system backup. If HyperMetro is configured for the storage device, this function must be enabled and logical ports that are being used by HyperMetro cannot be selected. Otherwise, NAS file system backup will fail.

    Select Replication Ports

    If Specify Replication Ports is enabled, you can select a maximum of eight replication logical ports per controller for NAS file system backup.

  5. Click OK.

    When a storage device is added successfully, information about the added storage device is displayed on the Storage Device page.

    The product automatically scans NAS file systems in the storage device and displays them on the Protection > File Systems > NAS File Systems page. For 1.5.0, the information is displayed on the Protection > File Services > NAS File Systems page. If a file system is added to the registered device, locate the row of the target device on the Storage Devices page, and choose More > Resource Scan to update the file system resources.
