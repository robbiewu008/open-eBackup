When a Database Executes a Log Backup Job, an Error Is Reported Indicating That the Database Does Not Support the Paxos Protocol

Symptom

When a database executes a log backup job, an error is reported indicating that the database does not support the Paxos protocol.

Possible Causes

If the database kernel version is earlier than 505.1.1 and the replica consistency protocol of the instance is Paxos, log backup fails.

Troubleshooting

  1. Run the following command in the database of the instance to check the database kernel version:

    gaussdb -V

    If the database kernel version is earlier than 505.1.1, perform subsequent operations.

  2. Run the following command in the database of the instance to check the consistency protocol:

    show enable_dcf
    • If off is displayed in the command output, the replica consistency protocol of the backup copy is Quorum.
    • If on is displayed in the command output, the replica consistency protocol of the backup copy is Paxos.

  3. Select an instance whose replica consistency protocol is not Paxos to back up logs.
