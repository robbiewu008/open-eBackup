Viewing the OceanBase Environment

After the OceanBase resource is registered with the OceanProtect, you can view details about OceanBase clusters on the management page.

Procedure

  1. Choose Protection > Databases > OceanBase.
  2. On the Cluster tab page, view the cluster information.

    Information of registered clusters is displayed on the page.
    1. Table 1 describes the detailed information.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 Cluster information

      Parameter

      Description

      Name

      Cluster name.

      Click the cluster name to view information about the OBClient and OBServer hosts and the tenant and copy data information of the cluster.

      Status
      Cluster status.
      • If all hosts in the cluster are online and the OceanBase environment is normal, the cluster status is Online.
      • If any host in the cluster is offline, the cluster status is Offline.

      Version

      Cluster version.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether the cluster is protected.

      • Unprotected: The cluster is not associated with an SLA, or the cluster is associated with an SLA but protection is disabled.
      • Protected: The cluster is associated with an SLA and protection is activated.
      • Creating: The cluster is associated with an SLA and protection is being created.
      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of the cluster.

      If a data protection administrator is authorized for a cluster, the name of the data protection administrator is displayed. If no authorization is performed, -- is displayed.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name and check the overview, tenant information, copy information, and jobs.
      • Overview

        Displays basic information, and OBClient host and OBServer host information.

      • Tenants

        Displays the tenant information.

      • Copy Data

        Search for copies by year, month, and day.

        If is displayed below the time, a copy at that time exists.

      • Jobs

        Displays jobs that are being executed and historical jobs.

  3. On the Tenant Sets tab page, view the tenant set information.

    The registered tenant sets are displayed on the page.
    1. Table 2 describes the tenant set information.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 2 Tenant set information

      Parameter

      Description

      Name

      Tenant set name.

      Status

      Status of the tenant set in the cluster. The value can be Online, Offline, or Partially Online.

      Owning Cluster

      Cluster to which the tenant set belongs.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether the tenant set is protected.

      • Unprotected: The tenant set is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The tenant set is associated with an SLA and protection is activated.
      • Creating: The tenant set is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name and check the overview, copy information, and jobs.
      • Overview

        Displays the SLA information, job execution time, and total number of copies.

      • Copy Data

        Search for copies by year, month, and day.

        If is displayed below the time, a copy at that time exists.

      • Jobs

        Displays jobs that are being executed and historical jobs.
