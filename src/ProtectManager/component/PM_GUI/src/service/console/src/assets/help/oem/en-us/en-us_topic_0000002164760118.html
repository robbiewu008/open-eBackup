Step 2: Registering the FusionOne Compute Virtualization Environment

Before performing FusionOne Compute backup and restoration, you must register the FusionOne Compute virtualization environment with the product.

Procedure

  1. Choose Protection > Virtualization > FusionOne Compute.
  2. Click Register, and register the FusionOne Compute virtualization platform.

    Table 1 lists the FusionOne Compute registration information.
    Table 1 FusionOne Compute registration information

    Parameter

    Description

    Name

    User-defined virtualization platform name.

    IP Address

    This IP address is used by the product to access FusionOne Compute for VM backup and restoration.

    Set this IP address to the VRM floating IP address.

    Username

    Use the username and password for interconnection in Step 1: Create a FusionOne Compute Interconnection User.

    Password

    Port

    Port number for the communication between the product and the FusionOne Compute management plane.

    The default port number is 7443.

    Clients

    Select clients used for backup and restoration.

    Certificate Verification

    This function is enabled by default. During FusionOne Compute virtualization environment registration, the system verifies the information about the accessed devices. After this function is disabled, security risks may exist in the communication between the system and FusionOne Compute. Exercise caution when performing this operation.

    Certificate

    After Certificate Verification is enabled, import the CA certificate. The certificate can be obtained by the following method:

    1. Use PuTTY to log in to the host where the client is installed.
    2. Run the following command. The values of VRM IP and Port must be the same as those in the table.
      openssl s_client -connect VRM IP:port -showcerts
    3. In the command output, copy the content of the last certificate to any new file, and rename the file as xx.pem. xx is user-defined, and xx.pem is the certificate to be imported.
    -----BEGIN CERTIFICATE-----
...(The details are omitted.)
-----END CERTIFICATE-----

    Revocation List

    After the certificate revocation list (CRL) is imported, the system checks whether the client certificate has been revoked. If the certificate has been revoked, cluster registration fails.

  3. Click OK.

    After the registration is successful, information about the registered FusionOne Compute virtualization environment is displayed on the left of the page. A maximum of eight FusionOne Compute platforms can be registered.
