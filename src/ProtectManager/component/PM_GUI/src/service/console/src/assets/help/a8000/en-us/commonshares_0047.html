Configuring Share Information

This section describes how to configure share information for common share resources that have been backed up.

Procedure

  1. Choose Explore > Copy Data > File Systems > Common Shares.
  2. You can search for copies by common share resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, find the target resource by resource name and click the resource name.

  3. Click Copy Data and choose the year, month, and day to find the copy.

    If is displayed below a month or date, copies exist in the month or on the day.

  4. Locate the row that contains the target copy, select the copy whose status is Normal, and choose More > Configure Share Information.

    Table 1 describes the related parameters.

    Table 1 Parameters for configuring share information

    Parameter

    Description

    Share Protocol

    Resource protocol of a common share. The NFS and CIFS protocols are supported.

    Share Client

    This parameter is displayed when the NFS protocol is selected. Enter the service IP address or service IP address segment of a share client, or enter an asterisk (*) to indicate service IP addresses of all share clients. The IP addresses can be IPv4 addresses, IPv6 addresses, or a combination of IPv4 and IPv6 addresses.

    Type

    Type of the user who accesses the CIFS share. This parameter is displayed when the CIFS protocol is selected.

    • Everyone
    • Windows local authentication user
    • Windows local authentication user group

    Users

    This parameter is displayed when the user type of CIFS shares is set to Windows local authentication user or Windows local authentication user group. Enter a username or select a local user group.

  5. Click OK.
