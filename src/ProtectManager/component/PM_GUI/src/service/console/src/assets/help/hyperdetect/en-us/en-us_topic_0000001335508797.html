Managing a File Name Extension Filtering Rule

You can perform association with vStores, disassociation from vStores, and File Name Extension Filtering Rule deletion on ProtectManager. When an operation is performed, the system scans the file systems of the vStore related to the operation to ensure that the operation is applied to all file systems of the vStore.

You need to log in to ProtectManager, choose Explore > Ransomware Protection > Ransomware Detection > File Name Extension Filtering Rule, and locate the target file name extension.

Table 1 describes the related operations.

Table 1 Related operations on the File Name Extension Filtering Rule

Operation

Description

Navigation Path

Viewing the File Name Extension Filtering Rule

Scenario

You can view information such as file name extensions to be intercepted and vStores associated with file name extensions by checking the File Name Extension Filtering Rule.

--

Associating with a vStore

Scenario

If you need to apply a File Name Extension Filtering Rule to a vStore, choose Associate with vStore. After a vStore is associated, the associated File Name Extension Filtering Rule is selected by default.

Locate the row that contains the target file name extension, and choose More > Associate with vStore.

Disassociating from a vStore

Scenario

If you no longer need to associate a File Name Extension Filtering Rule with a vStore, choose Disassociate from vStore.

Locate the row that contains the target file name extension, and choose More > Disassociate from vStore.

Deleting the File Name Extension Filtering Rule

Scenario

You can delete a File Name Extension Filtering Rule that is no longer needed.

Note

The preset File Name Extension Filtering Rule cannot be deleted.

Locate the row that contains the target file name extension, and choose More > Delete.
