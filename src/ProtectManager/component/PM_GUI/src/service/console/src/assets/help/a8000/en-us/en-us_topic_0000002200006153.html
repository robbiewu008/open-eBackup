Viewing Environment Information of the SQL Server Database

After the SQL Server database is registered with the OceanProtect, you can view details about the database, the host where it resides, and the cluster or availability group to which it belongs on the management page.

Procedure

  1. Choose Protection > Databases > SQL Server.
  2. View the details.

    • On the Cluster tab page, view the SQL Server cluster information.

      If a database in a cluster has been registered, the cluster information is displayed on the page.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 describes the detailed information.
      Table 1 Cluster information

      Parameter

      Description

      Name

      Name of a database cluster.

      • If you click the cluster name, host members and all authenticated databases in the cluster are displayed.

      Status
      Cluster status.
      • Online: When a host is powered on and the client installed on the host is running properly, Status of the host is Online.

        If there are online hosts in a cluster, the cluster status is Online.

      • Offline: When a host is shut down or the client installed on the host is faulty, Status of the host is Offline.

        If all hosts in a cluster are offline, the status of the cluster is Offline.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    • On the Instances tab page, view the database instance information.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 2 describes the database instance information.
      Table 2 Database instance information

      Parameter

      Description

      Name

      Name of a database instance.

      Owning Host/Cluster

      Name of the host or cluster where a database instance resides.

      Node IP Address

      IP address of the host where a database instance resides.

      Version

      Version of the database where an instance is located.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether an instance is protected.

      • Unprotected: The instance is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The instance is associated with an SLA and protection is activated.
      • Creating: The instance is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      1. Click the name and check the overview, copy information, and jobs.
        • Overview

          Displays the SLA information, job execution time, total number of copies, and information about nodes and databases.

        • Copy data

          You can search for copies by year, month, or day.

          If is displayed below the time, copies exist at that time.

        • Job

          Backup and restoration jobs are displayed.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      1. Click the name and check the overview, copy information, and jobs.
        • Overview

          Displays SLA information, job execution time, total number of copies, and protected objects.

        • Copy data

          You can search for copies by year, month, or day.

    • View the database information on the Databases tab.

      The information about the scanned database is displayed.
