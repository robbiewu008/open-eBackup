Step 2: Obtaining the Certificate

When registering an OpenStack protected environment, you can import the CA certificates of OpenStack on the product to improve access security between devices. Otherwise, the system cannot verify information about the accessed devices, causing security risks. This operation is optional. You can obtain it from the administrator or download it by yourself. This section describes how to download a certificate.

Procedure

  1. Open the Chrome browser, enter the Keystone V3 address in the address box, and press Enter.

    For details about how to obtain the Keystone V3 address, see Preparing for the Backup.

    Example: https://identity.az236.dc236.huawei.com:443/identity/v3

  2. Click Not secure on the left of the URL address box, and click Certificate is not valid.

    The displayed information varies depending on the browser. The following uses Google Chrome 120 as an example.

  3. In the dialog box that is displayed, click the General tab to view the certificate security algorithm level.

    Only algorithms with the security level of SHA-256 or higher can pass the verification of the product. Therefore, if the certificate security level does not meet the requirements, contact the administrator to obtain the certificate. If the certificate security level meets the requirements, go to 4.

  4. Click the Details tab, select the highest-level certificate name in the Certificate Hierarchy area, and click Export.
  5. Export the certificate and rename the certificate file XXX.pem as prompted. The renaming method is as follows:

    1. Upload the certificate file to the temporary directory on the agent host. The /tmp directory is used as an example in the following steps.
    2. Run the following commands to convert the certificate file to the .pem format (XXX.crt indicates the certificate file name before conversion and XXX.pem indicates the certificate file name after conversion):
      cd /tmp/
      openssl x509 -in XXX.crt -out XXX.pem
    3. Download the certificate file converted to the .pem format.

      Register OpenStack with the product. The certificate must be in .pem format.
