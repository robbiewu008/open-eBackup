Step 11: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • During manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, Automatic Indexing (available only in 1.5.0), and Application-Consistent Backup, are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy will be retained permanently. If forever incremental (synthetic full) backup is not defined in the SLA but selected for manual backup, copies generated by the manual forever incremental (synthetic full) backup will be retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Prerequisites

If the ESXi host uses a domain name for registration, ensure that the mapping between the IP address and name of the target host has been added. For details, see Backup or Restoration Fails Because the Domain Name of the ESXi Host Cannot Be Identified by the Backup Agent Host.

Procedure

  1. Choose Protection > Virtualization > VMware.
  2. Select VMs to be protected.

    1. Select a vCenter Server or ESXi host on the left of the page.
    2. On the right of the page, select VMs that you want to protect on the vCenter Server or ESXi host.
      • When you select a cluster on the Cluster tab page, the system protects all VMs in the cluster.
      • When you select a host on the Hosts tab page, the system protects all VMs on the host.
      • When you select a VM on the VMs tab page, the system protects the VM.
      • When you select a VM group on the VM Groups tab page, the system protects all VMs in the VM group.
      • VM Groups is available only in 1.6.0 and later versions.
      • A maximum of 1000 VMware VMs can be protected in a batch.

  3. Click Protect.
  4. In the Selected area, click the number of selected disks. On the displayed Selected Disks page, deselect the disks that do not need to be backed up. Skip this step if you select VM Groups.

    The Selected Disks page displays all disks to be protected. You can deselect the disks that do not need to be protected.

    • After some disks are selected for backup, the generated copy does not support live mount. After the restoration is successful, the original VMs cannot be deleted.
    • If system disks are excluded, generated copies do not support file-level restoration.

  5. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resource to be protected, select an SLA without a WORM policy to create protection to avoid WORM policy conflicts.

  6. Set advanced parameters by referring to Table 1.

    Table 1 Advanced parameters

    Parameter

    Description

    Agent Host

    NOTE:

    For 1.5.0, this parameter is available only when VMs are selected for protection.

    You can select the agent host for executing the backup job. If you leave this parameter unspecified, the system randomly selects an agent host during job execution.
    Auto Indexing of Backup Copies
    NOTE:

    Only 1.6.0 and later versions support this parameter.

    If a VM or VM group is selected for protection and the disks to be backed up contain the system disk, the system automatically creates an index for the backup copy after this function is enabled. When using the copy for restoration, you can select one or more files in the copy for restoration and search for files, directories, or links in the copy.

    Concurrent Jobs

    NOTE:

    Only 1.6.0 and later versions support this parameter.

    When you select a host, cluster, or VM group for protection, you can set the number of backup jobs that can be concurrently executed.

    The value ranges from 1 to 10.

    SLA Overwrite Policy
    When you select a cluster or host for protection, you can choose whether to overwrite the existing VM SLAs.
    NOTE:

    If you select Do not overwrite existing VM SLAs when modifying the SLA associated with the cluster or host, the VM SLAs are still overwritten when they are the same as the SLA of the cluster or host before the modification.

    SLA Application Policy
    When you select a cluster or host for protection, you can choose the application scope of the SLA policy as follows:
    • Applying the cluster or host SLA policy to all VMs that are not associated with any SLA
    • Applying the cluster or host SLA policy to newly created VMs

      After the virtualization environment is scanned, the newly discovered VMs are associated with the same SLA as the cluster or host.

    VM Name Filtering

    NOTE:

    In 1.5.0, the parameter is VM Filter.
    If you want to specify the VMs to be excluded or included in the protection plan when selecting a cluster or host for protection, enable this function and perform the following operations:
    1. Select a filtering mode.
    2. Enter a VM name or keyword in the text box to filter VMs based on the selected filtering mode.
      • You can enter multiple names or keywords. Use commas (,) to separate multiple names or keywords.
      • Exact match and fuzzy match of VM names are supported. Fuzzy match rules include prefix match (for example, abc*), suffix match (for example, *abc), and middle match (for example, *abc*).
    VM Tag Filtering
    NOTE:

    Only 1.6.0 and later versions support this parameter.
    If you want to specify the VMs with specific tags (which are set for the VMs in the VMware virtualization environment) to be excluded or included in the protection plan when selecting a cluster or host for protection, enable this function and perform the following operations:
    1. Select a filtering mode.
    2. Based on the selected filtering mode, enter tag names or keywords in the text box to filter the VMs with the corresponding tags.
      • You can enter multiple tag names or keywords. After entering a tag name or keyword, press Enter before entering the next one.
      • Exact match and fuzzy match of tag names are supported. Fuzzy match rules include prefix match (for example, abc*), suffix match (for example, *abc), and middle match (for example, *abc*).
      • To filter all VMs with tags, enter an asterisk (*) and press Enter.
      NOTE:

      To ensure the accuracy of VM tag filtering, you are advised to exclude the asterisk (*) from the tags set for VMs in the VMware virtualization environment.
    Secure archive
    NOTE:

    Only 1.6.0 and later versions support this function.
    This parameter is displayed when the selected SLA contains archive policies. If this option is selected, only backup copies that are detected to be uninfected during ransomware detection are archived.
    NOTE:

    You need to click Configure to set the ransomware protection policy for the resource to be protected if the following conditions are met: Archive Time is not set to Immediately after successful backup or Archive Policy is not set to Only the latest copy in the SLA archive policy, and the ransomware protection policy is not set. For details, see "Creating Ransomware Protection and WORM Policies" in the OceanProtect Appliance 1.5.0-1.6.0 Data Backup Feature Guide (Ransomware Protection for Copies).

    If you select an agent host that executes backup when creating protection for a host or a cluster, the agent host cannot be changed separately for VMs on the host or cluster.

  7. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  8. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.
    1. In the row of the target resource, choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple protected resources and choose More > Manual Backup in the upper left corner of the resource list.

    2. Set the name of the copy generated during manual backup.

      If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup or Forever Incremental (Synthetic Full) Backup.

      For 1.6.0 and later versions: If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    4. Click OK.
