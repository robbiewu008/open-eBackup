Step 1: Adding a Backup Storage

This section describes how to connect a backup storage to the storage system so that a logical connection is established between the backup storage and the storage system for data transmission.

Prerequisites

  • The object storage platform for storing backup data is ready.
  • You have obtained the access address and AK/SK of the object storage platform.
  • A bucket for storing backup data has been created on the object storage platform.

    To ensure data security, it is recommended that a dedicated bucket be created for backing up file systems. Otherwise, archived data may be illegally intercepted or tampered with.

  • Versioning has been disabled for the data bucket of the object storage.

Procedure

  1. Choose System > Infrastructure > Backup Storage.
  2. Click Add Backup Storage.
  3. Set backup storage parameters.

    Table 1 describes the related parameters.

    Table 1 Backup storage parameters

    Parameter

    Description

    Name

    Name of a backup storage, which can be customized.

    Type

    Type of a backup storage. Currently, only OBS complying with the S3 protocol is supported.

    Object Storage Product

    OBS product connected to the storage system.

    FusionStorage OBS, OceanStor Pacific, Huawei Cloud OBS, and AWS S3 are supported.

    Endpoint

    Access address of a backup storage

    Address format: domain name or service plane IP address of a backup storage.

    NOTE:

    The following is an example of querying the domain name on Huawei Cloud OBS:

    1. Log in to the Huawei Cloud OBS website (https://www.huaweicloud.com/intl/en-us/product/obs.html) and click Access Console.
    2. In the OBS bucket list, click the created bucket for storing file system backups. The Summary information about the bucket is displayed.
    3. In the Basic Information area, record the value of Endpoint, which is the domain name of the bucket.

    Protocol

    Protocol used for data transmission between the storage system and the backup storage.

    • When HTTPS is used, import a valid certificate so that the system can verify the backup storage information. Obtain the certificate from the backup storage administrator in advance.

      For details about how to import a certificate, see Importing a Certificate.

    • When the protocol is set to HTTP, the system cannot verify the backup storage environment information, which poses security risks. You are advised to select HTTPS for security purposes.

    AK

    Access certificate for accessing the object storage

    NOTE:

    The following is an example of creating the AK and SK on the Huawei Cloud OBS:

    1. Log in to the Huawei Cloud OBS website (https://www.huaweicloud.com/intl/en-us/product/obs.html) and click Access Console.
    2. Move the pointer on the username in the upper right corner of the page, and choose My Credential from the drop-down list. The My Credential page is displayed.
    3. Click Access Keys in the navigation tree. The Access Keys details page is displayed.
    4. Click Add Access Key, enter the key description as prompted, and click OK. A message is displayed, indicating that the key is successfully created.
    5. Click Download Now. The browser automatically generates a key file named credentials.csv.
    6. Download and open the credentials.csv file to obtain the AK and SK. The key file contains the User Name, Access Key Id, and Secret Access Key. Access Key Id corresponds to AK, and Secret Access Key corresponds to SK.

    SK

    Security certificate for accessing the OBS.

    Data Bucket

    Name of the bucket that stores backup data.

    NOTE:

    The following is an example of querying a bucket name on the Huawei Cloud OBS:

    1. Log in to the Huawei Cloud OBS website (https://www.huaweicloud.com/intl/en-us/product/obs.html) and click Access Console.
    2. In the OBS bucket list, browse and manually copy the bucket name.

    Index bucket

    Name of the bucket that stores index data. The value is the same as the data bucket name and does not need to be set.

    Use Proxy Server

    When the storage system uses a proxy server to access the backup storage, configure the address, user name, and password of the proxy server.

    NOTE:
    • Obtain the proxy information from the network administrator of the enterprise's data center.
    • If Use Proxy Server is enabled, the Address, Username, and Password parameters are displayed.

    Capacity Alarm Threshold

    Upper threshold of the used capacity of a bucket. When the used capacity reaches the threshold, an alarm is reported.

    If Capacity Alarm Threshold is enabled, the Alarm Threshold parameter is displayed.

  4. Click OK.
