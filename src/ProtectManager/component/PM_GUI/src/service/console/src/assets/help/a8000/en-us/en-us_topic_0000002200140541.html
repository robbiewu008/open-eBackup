Backup or Restoration of a VM That Contains RDM Disks Fails

Symptom

Backup or restoration of a VM that contains RDM disks fails. A message is displayed, indicating the failure of restoring VM disks or the failure of instructing client (xxx) to complete backup of disk (xxx).

Possible Causes

Jobs that involve RDM disks contain disk scanning. Due to insufficient resources of the current client, disk scanning may time out. As a result, the backup or restoration job fails.

Troubleshooting

  1. Log in to the host where the client is installed as user root.
  2. Run the following command to restart the iSCSI service:

    systemctl restart iscsi.service

  3. Run the following command to reboot the host where the client is installed:

    reboot

    If the environment is not restored:

    • Check whether the specifications of the host where the client is installed meet the requirements in the product documentation.
    • Check whether the OS of the host where the client is installed is in the compatibility list.
    • Check whether Huawei UltraPath is installed on the host where the client is installed. If the built-in multipathing software of the system is used, disk scanning will time out.

    If the fault persists, you are advised to use another host where the client is installed to perform the backup or restoration job.
