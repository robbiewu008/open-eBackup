Step 5: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA. The system protects the resources and periodically executes backup jobs based on the SLA. You can perform a backup job immediately through manual backup.

Precautions

Copies generated by manual backup are retained for the period defined in the SLA.

Procedure

  1. Choose Protection > Big Data > MongoDB.
  2. Select the instance to be protected and click Protect.

    You can also select multiple instances for batch protection.

  3. (Optional) Configure the percentage of the space required for creating a snapshot logical volume to that of the original logical volume in a backup job in the Advanced area.
  4. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resources to be protected, select an SLA without a WORM policy to avoid WORM policy conflicts.

  5. Set Percentage of Space for Created Logic Volume in the Advanced area.

    Percentage of the space required for creating a snapshot logical volume to that of the original logical volume in a backup job. Ensure that the percentage of free space in the original logical volume is greater than the parameter value.

  6. Click OK.

    If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  7. Execute manual backup.

    If you want to execute a backup job immediately, perform the following operations to execute manual backup. Otherwise, skip this step.

    1. In the row of the instance to be backed up, choose More > Manual Backup.
    2. Set the name of the copy generated by manual backup.
    3. If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.
    4. Select a protection policy, which can be Full Backup or Log Backup.
      • MongoDB does not support log backup if oplog is not enabled.
      • For 1.6.0 and later versions, if the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.
    5. Click OK.
