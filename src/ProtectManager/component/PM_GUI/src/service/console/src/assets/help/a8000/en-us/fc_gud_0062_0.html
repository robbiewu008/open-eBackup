Managing FusionCompute Registration Information

After the FusionCompute virtualization environment is registered with the OceanProtect, you can perform operations on the FusionCompute virtualization environment on the management page.

Log in to the management page and choose Protection > Virtualization > FusionCompute to go to the FusionCompute page. On the left of the FusionCompute page, select the desired virtualization environment.

Table 1 describes the related operations.

Table 1 Operations related to the FusionCompute virtualization environment

Operation

Description

Navigation Path

Viewing FusionCompute environment information

Scenario

This operation enables you to view FusionCompute environment information, such as the name, port number, IP address, and username.

Choose More > Environment Info.

Performing resource scanning

Scenario

The OceanProtect automatically scans the FusionCompute virtualization environment every hour. If VMs are added, deleted, or modified in the registered virtualization environment, the VM information is automatically updated to the OceanProtect after the scan. This operation allows you to immediately update the changed VM information to the OceanProtect.

Note

If you do not select to apply the SLA policy to the newly created VM when creating a periodic backup job, the system will not associate the newly detected VM with an SLA after scanning.

Choose More > Resource Scan.
Authorizing resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Note

  • The protection permissions on a resource that has been associated with an SLA cannot be granted. Before permission granting, remove the protection.
  • The protection permissions on a single resource can be granted only to a single data protection administrator.

Choose More > Authorize Resource and select a data protection administrator to authorize.
Reclaiming resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

The protection permissions on a resource that has been associated with an SLA cannot be reclaimed. Before reclamation, remove the protection.

Choose More > Reclaim Resource.

Modifying registration

Scenario

You can perform this operation to change the name of FusionCompute registration information. If the IP address, port number, or CA certificate of the registered FusionCompute virtualization environment is changed, you need to perform this operation to synchronize the change.

Note

Modify the FusionCompute registration information when no backup or restoration job is running. Otherwise, the job will fail.

Choose More > Modify Registration.

Canceling registration

Scenario

If you do not need to protect the FusionCompute virtualization environment or use it as a restoration target, you can perform this operation to remove the FusionCompute virtualization environment.

Note

If a backup job is being executed, the FusionCompute virtualization environment fails to be deregistered. Wait and try again until the backup job is complete.

Choose More > Unregister.
