Managing Hyper-V Registration Information

After the Hyper-V virtualization environment is registered with the OceanProtect, you can perform operations on the Hyper-V virtualization environment on the WebUI.

Log in to the OceanProtect WebUI and choose Protection > Virtualization > VMware. On the Hyper-V page that is displayed, select the Hyper-V server to be operated on the left pane.

Table 1 describes the related operations.

Table 1 Operations related to the Hyper-V virtualization environment

Operation

Description

Navigation Path

Modifying the Hyper-V registration information

Scenario

You can perform this operation to change the name of Hyper-V registration information. If the IP address of a registered Hyper-V server is changed, you need to perform this operation to synchronize the change.

Note

Modify the Hyper-V registration information when no backup or restoration job is running. Otherwise, the job will fail.

Choose More > Modify Registration.

Table 1 describes the parameters related to Hyper-V registration information.

Deregistering Hyper-V

Scenario

This operation enables you to remove a Hyper-V virtualization environment when a Hyper-V server does not need to be protected or used as a restoration target.

Note

If a backup job is being executed, the Hyper-V virtualization environment fails to be deregistered. Wait and try again until the backup job is complete.

Choose More > Unregister.

Manually scanning the Hyper-V virtualization environment

Scenario

This operation allows you to immediately update the changed VM information to the OceanProtect.

Note

If you do not select to apply the SLA policy to the newly created VM when creating a periodic backup job, the system will not associate the newly discovered VM with an SLA after scanning.

Choose More > Resource Scan.

Performing connectivity tests

Scenario

This operation tests the connectivity between the OceanProtect and the protected environment.

In the row of the target resource, choose More > Connectivity Test.
