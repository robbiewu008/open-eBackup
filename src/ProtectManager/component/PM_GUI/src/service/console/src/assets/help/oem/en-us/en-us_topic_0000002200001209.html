Restoring Files on a FusionOne Compute VM

This section describes how to restore files on a VM by using the file-level restoration function.

Context

  • The product supports file-level restoration using backup copies and replication copies. Note that data cannot be restored to the original location when a replication copy is used.
  • The types of Linux file systems that can be restored are ext2/ext3/ext4 and XFS. The types of Windows file systems that can be restored are NTFS, FAT, and FAT32.

Prerequisites

  • The IP address of the backup network plane has been configured for the VM to which files are to be restored.
  • If the target VM runs the Windows OS, ensure that ports 445 and 138 have been enabled on the target VM. If the target VM runs the Linux OS, ensure that port 22 has been enabled on the target VM.
  • If the target VM runs the Windows OS, ensure that the CIFS service has been enabled on the VM. For details, see the Microsoft official website.
  • If the target VM runs a non-Windows OS, ensure that rsync has been installed and started on the target VM and the rsync version is 3.2.5 or later.

Procedure

  1. Choose Explore > Copy Data > Virtualization > FusionOne Compute.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    Click Resources to switch to the corresponding page, search for the VM to be restored by name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or date, copies exist in the month or on the date.

  4. In the row of the target copy, choose More > File-level Restoration.

    • A copy that does not contain system disks cannot be used for file-level restoration.
    • Reverse replication copies and archive copies cannot be used for file-level restoration.
    • Linked files do not support file-level restoration.
    • During FusionOne Compute VM backup, the mount point information that is not written into the /etc/fstab file is stored in the memory. The product does not back up the information and does not support file-level restoration. If new file systems need to support file-level restoration, write the mount point information of the new file systems to the /etc/fstab file.
    • If the name of a folder or file contains garbled characters, file-level restoration is not supported. Do not select folders or files of this type. Otherwise, the restoration job fails.
    • If the target VM runs the Windows OS and only the SMBv1 service is enabled, the time of the restored file is the current VM time.

  5. Set File Obtaining Mode, which can be Select file paths from the directory tree or Enter file paths.

    • If you choose Select file paths from the directory tree for File Obtaining Mode but no file-level mounting task has been created for the current copy, click Create Mount. You can select files from the directory tree only after the mounting is successful.
    • For OceanProtect E1000, if there are more than 10,000 files in the selected directory, the system may time out, preventing you from selecting files or folders to be restored from the directory. In this case, you are advised to manually enter paths. For other models of this product, if there are more than one million files in the selected directory, the system may time out, preventing you from selecting files or folders to be restored from the directory. In this case, you are advised to manually enter paths.
    • When entering a file path, enter a complete file path, for example, /opt/abc/efg.txt or C:\abc\efg.txt. If you enter a folder path, for example, /opt/abc or C:\abc, all files in the folder are restored. The file name in the path is case sensitive.

  6. Select Original location or New location and set restoration parameters.

    • Select the original location for restoration, that is, restore data to the original directory on the original VM.
      Table 1 describes the related parameters.
      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      VM IP Address
      Select or enter the configured backup network IP address.
      NOTE:

      If the system cannot obtain the backup network IP address of the VM under Compute Location, check whether Tools is installed or manually enter the IP address. If Tools is not installed, install it by referring to the FusionOne Compute online help. After the installation is complete, choose More > Resource Scan on the Protection > Virtualization > FusionOne Compute page to scan the virtualization environment again.

      VM Username
      During file-level restoration, the target VM needs to verify the login credential of the product. Enter a username and its password for logging in to the target VM.
      • Windows OS: The default user name is Administrator.
      • Linux OS: The default user name is root.
      NOTE:
      • The login user must have the read and write permissions on the directory to which data is to be restored on the target VM.
      • After the restoration, the access permission on the file is the same as that of the login user.

      VM Password

      Overwrite Rule
      If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.
      • Replace existing files
      • Skip existing files: Files with the same names are skipped and are not replaced.
      • Only replace the files older than the restoration file
        NOTE:

        If the access to the file with the same name on the target VM is denied, the file fails to be restored during file replacement.
    • Select a new location for restoration, that is, restore data to a specified VM.

      Table 2 describes the related parameters.

      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Target Compute Location
      Select the target VM to which data is restored.
      NOTE:
      • If the specified VM runs the Windows OS, the directory to which the file is restored is the same as the source directory of the file.
        • If the corresponding drive letter does not exist on the VM, the restoration will fail.
        • If the corresponding drive letter exists but the corresponding directory does not exist on the VM, the corresponding directory will be automatically created on the VM before file restoration is performed. For example, if the path of the specified file is D:\a, the a directory will be automatically created in drive D of the specified VM before file restoration.
      • If the specified VM runs the Linux OS, the directory to which data is restored is the same as the original directory of the file. During restoration, the corresponding path is automatically created on the VM before file restoration is performed.

      VM IP Address
      Select or enter the configured backup network IP address.
      NOTE:

      If the system cannot obtain the backup network IP address of the VM under Compute Location, check whether Tools is installed or manually enter the IP address. If Tools is not installed, install it by referring to the FusionOne Compute online help. After the installation is complete, choose More > Resource Scan on the Protection > Virtualization > FusionOne Compute page to scan the virtualization environment again.

      • Windows OS: The default user name is Administrator.
      • Linux OS: The default user name is root.

      • Only replace the files older than the restoration file
        NOTE:

  7. Click Test and ensure that the target VM to be restored is properly connected to the product.
  8. Click OK.
