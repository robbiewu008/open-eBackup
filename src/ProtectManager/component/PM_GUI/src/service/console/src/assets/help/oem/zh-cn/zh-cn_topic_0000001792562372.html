步骤4：注册PostgreSQL集群实例下的数据库

前提条件

  • 注册PostgreSQL集群实例前，请确保集群中每个主机已进行注册。

    要求集群中所有主机已注册相同名称的数据库。

    具体操作请参见步骤3：注册PostgreSQL单实例下的数据库。

  • 执行实例注册前，请检查数据库安装路径下bin/lib/share文件属主是否为root，若是root请在/opt/DataBackup/ProtectClient/Plugins/GeneralDBPlugin/bin/applications/postgresql/conf/switch.conf配置文件中将enable_root配置项取值设置为1。
  • 注册PostgreSQL集群前，请先查询集群类型。

    Pgpool集群

    1. 使用PuTTY，登录PostgreSQL数据库主机。
    2. 执行which pgpool命令，查询是否是Pgpool集群。
      • 如果回显为Pgpool集群所在路径，则表示集群类型为Pgpool集群，查询结束。
      • 如果回显为no pgpool，则表示未查询到Pgpool集群，请继续执行以下操作。
    3. 执行whereis pgpool命令，查询是否是Pgpool集群。
      • 如果回显为Pgpool集群所在路径，则表示集群类型为Pgpool集群，查询结束。
      • 如果回显为pgpool:，则说明当前集群类型不是Pgpool集群。

    Patroni集群

    1. 使用PuTTY，登录PostgreSQL数据库主机。
    2. 执行which patroni命令，查询是否是Patroni集群。
      • 如果回显为Patroni集群所在路径，则表示集群类型为Patroni集群，查询结束。
      • 如果回显为no patroni，则表示未查询到Patroni集群，请继续执行以下操作。
    3. 执行whereis patroni命令，查询是否是Patroni集群。
      • 如果回显为Patroni集群所在路径，则表示集群类型为Patroni集群，查询结束。
      • 如果回显为patroni:，则说明当前集群类型不是Patroni集群。

操作步骤

  1. 创建集群。

    1. 在PostgreSQL页面选择“集群”页签。
    2. 单击“注册”。
    3. 在“注册”页面，自定义集群名称和虚拟IP地址并选择类型和集群节点。
    4. 单击“确定”。

  2. 注册集群实例下的数据库。

    1. 在PostgreSQL页面选择“实例”页签。
    2. 单击“注册”。
    3. “类型”选择“集群实例”。
    4. 配置数据库实例及其认证信息。

      相关参数说明如表1所示。

      表1 PostgreSQL集群实例注册信息

      参数

      说明

      名称

      安装数据库的用户名称。

      集群

      选择要注册的数据库主机。

      用户名

      运行数据库的操作系统用户名。

      Pgpool端口

      当集群选择为Patroni集群的数据库主机时不展示该参数。

      Pgpool服务侦听TCP/IP连接的端口号。

      数据库用户名

      数据库的管理员用户名称。

      数据库密码

      数据库管理员用户的密码。

      数据库流复制用户名

      数据库用于流复制的用户名称。

      数据库流复制用户密码

      数据库用于流复制用户的密码。
    5. 单击“添加”，在弹出的添加对话框中添加集群实例下的所有节点。

      相关参数说明如表2所示。

      表2 集群实例的节点信息

      参数

      说明

      主机

      选择集群下的主机节点。

      数据库安装路径

      要注册的数据库安装路径。

      Patroni配置文件完整路径

      说明：

      仅在1.6.0及后续版本展示该参数。

      配置集群实例时，集群选择为Patroni集群的数据库主机，添加集群实例节点可展示该参数。

      Patroni配置文件完整路径。

      Pgpool安装路径

      Pgpool服务的安装路径。

      业务IP地址

      数据库服务接收TCP/IP连接的主机IP地址。

      数据库端口

      数据库服务侦听TCP/IP连接的端口号。
    6. 依次选择每个集群成员的已注册的实例名称。
    7. 单击“确定”。
