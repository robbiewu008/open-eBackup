步骤8：创建反向复制/级联复制SLA（适用于备份存储为OceanProtect）

如果您需要配置反向复制（将复制副本复制回源端）或级联复制（将目标端的复制副本复制到其他集群），请参考本节配置复制策略的SLA。否则请忽略。

操作步骤

  1. 登录复制副本所在集群的管理界面。

    复制副本仅支持在目标集群查看、操作。

  2. 选择“保护 > 保护策略 > SLA”。
  3. 单击“创建”。
  4. 自定义SLA的名称。
  5. 选择需要创建SLA的应用类型。

    1. 单击“应用”图标。
    2. 选择“指定应用 > 副本 > 复制副本”。
    3. 单击“确定”。

  6. 单击“复制策略”图标。
  7. 配置复制策略信息，完成后单击“确定”。

    单击页面右侧的，可创建多个复制策略，最多允许创建4个。

    相关参数说明如表1。

    表1 复制策略参数说明

    参数

    说明

    名称

    用户自定义复制策略名称。

    复制目标集群

    复制到的目标集群。

    选择已添加的目标集群，或单击“添加”，添加新的集群。
    • 当“复制目标集群”为复制副本的源端集群时，即为反向复制。
    • 当“复制目标集群”为其他集群时，即为级联复制。

    复制目标端用户

    选择复制目标端的系统管理员或数据保护管理员用户。

    密码

    填写复制目标端的系统管理员或数据保护管理员用户密码。输入密码后，单击“认证”，认证密码的正确性。

    本地存储类型

    选择本地存储的类型。

    指定目标位置
    可选择复制的目标位置为备份存储单元或备份存储单元组。当选择备份存储单元组，对备份副本进行首次复制时，系统将从备份存储单元组中选择其中一个备份存储单元，作为复制的目标位置，后续复制时优先选择首次复制使用的备份存储单元。
    说明：

    当备份策略的目标位置指定备份存储单元时，复制策略的目标位置不能为同一备份存储单元。

    频率

    每xx分钟/小时/天/周执行一次复制。

    复制规则&保留规则

    选择需要复制的复制副本并设置副本的保留时长。

    一旦超过设定的保留时间，系统将自动删除过期的副本。副本保留时长必须大于复制频率。
    • 复制所有副本：指定从某个时间产生的副本进行复制，并设置副本保留时长。
      • 复制副本保留x天（支持保留1~25550天）。
      • 复制副本保留x周（支持保留1~3650周）。
      • 复制副本保留x月（支持保留1~840月）。
      • 复制副本保留x年（支持保留1~70年）。
      • 复制副本保留永久。
    • 复制指定日期的副本：指定日期并设置副本保留时长。以下选项同时可选择多项：
      • 选择每年的x月产生的备份副本进行复制，复制副本保留x年（支持保留1~70年）。
      • 选择每月的第一个或最后一个备份副本进行复制，复制副本保留x个月（支持保留1~840个月）。
      • 选择每周的第x天产生的备份副本进行复制，复制副本保留x个周（支持保留1~3650个周）。
      说明：
      • “复制指定日期的副本”仅支持复制指定日期的全量和永久增量（合成全量）备份副本。
      • 如果同时选择了多个选项，待复制副本的时间范围为多个选项的并集，生成的复制副本的保留时长为多个选项中保留时间最久的保留时长。
      • 选择每月的最后一个全量备份副本复制时，当月待复制的全量备份副本的复制任务将在下一个月执行。

    首次执行时间

    首次执行复制任务的开始时间。

    如果首次执行时间晚于当前时间，则按照首次执行时间开始复制；如果首次执行时间早于当前时间，则等待下次复制周期到达后进行复制。
    说明：
    • 请确保客户端所在主机、本地集群和目标集群的设备时间一致。
    • 如果规划了Air Gap，请确保首次执行时间在目标集群Air Gap复制时间窗内，并确保每次复制任务开始时间均在Air Gap复制时间窗内。如果复制任务在Air Gap复制时间窗内未结束，且Air Gap策略启用了“强制中止复制”，复制任务将会被强制中止且会立即失败。如果确实需要执行完成该复制任务，请暂时关闭目标集群Air Gap策略的“强制中止复制”开关。

    限速策略

    复制任务平均每秒最大传输的数据量。

    选择已创建的限速策略，创建限速策略的具体操作请参考步骤3：（可选）创建限速策略章节。

    链路重删
    开启后，远程复制过程中系统会对副本数据进行重删以提升复制效率。建议开启此开关。
    说明：
    • 开启此功能前请确保已开启目标端重删。
      • 如果SLA的备份策略中存在“目标端重删”开关，请打开“目标端重删”，否则将导致链路重删功能不生效。
      • 如果SLA的备份策略中不存在“目标端重删”开关，则默认已开启目标端重删。

    链路压缩

    开启后，远程复制过程中系统会对副本数据进行压缩以提升复制效率。建议开启此开关。

    任务失败告警

    开启后，任务执行失败后自动发送告警，下次任务成功后告警自动清除。

  8. 单击“确定”。
  9. 关联资源到创建的SLA，对复制副本进行反向复制或级联复制。

    1. 选择“数据利用 > 副本数据”，选择相应的应用类型。
    2. 查找到需要进行反向复制或级联复制的资源，在资源所在行单击“更多 > 保护”。
    3. 选择已创建的SLA。
    4. 单击“确定”。
      • 执行复制任务时需确保本地集群和目标集群的存储池已使用的容量不能超过License中的“数据备份容量”，否则复制任务将失败。参考以下步骤确认容量是否充足。
        1. 登录本地集群和目标集群备份存储设备的DeviceManager管理界面。具体操作请参考登录DeviceManager管理界面。
        2. 选择“系统 > 存储池”，查看存储池已使用的容量。
        3. 选择“设置 > License管理”，查看“数据备份容量（Data Backup Capacity）”。

          存储池已用容量不能超过“数据备份容量（Data Backup Capacity）”。

      • 复制任务产生的复制副本，仅支持在目标集群进行查看、恢复等，本地集群无法查看。
