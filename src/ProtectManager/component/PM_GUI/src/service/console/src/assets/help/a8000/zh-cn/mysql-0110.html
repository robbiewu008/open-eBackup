无法在MySQL/GreatSQL数据库主机/usr/bin目录下找到mysql、mysqlbinlog、mysqldump和mysqladmin工具

现象描述

MySQL/GreatSQL数据库备份失败，报错：“未找到可用的配置文件或无法在MySQL/GreatSQL主机/usr/bin目录下找到mysql、mysqlbinlog、mysqldump和mysqladmin工具。”

可能原因

MySQL/GreatSQL数据库可能未安装在/usr/bin目录下，并且未通过软连接的方式配置mysql、mysqlbinlog、mysqldump和mysqladmin工具。

处理步骤

  1. 登录MySQL/GreatSQL数据库主机节点。
  2. 执行which mysql命令，找到MySQL/GreatSQL数据库安装路径，此处以/mysqlpath为例。
  3. 依次执行以下命令，建立MySQL命令软连接。

    ln -s /mysqlpath/bin/mysql /usr/bin/mysql
ln -s /mysqlpath/bin/mysqlbinlog /usr/bin/mysqlbinlog
ln -s /mysqlpath/bin/mysqldump /usr/bin/mysqldump
ln -s /mysqlpath/bin/mysqladmin /usr/bin/mysqladmin

  4. 分别执行以下命令并查看回显，回显无报错，表示设置成功。

    mysql --version
mysqlbinlog --version
mysqldump --version
mysqladmin --version
