备份VMware虚拟机时VDDK打开磁盘失败

现象描述

执行VMware虚拟机备份任务时，出现VDDK打开磁盘失败，后续备份任务出现VDDK初始化失败。

可能原因

在单个ESXi主机上以NBDSSL模式打开的虚拟机磁盘数量过多，达到52个的数量限制。

处理步骤

  1. 重启客户端服务。

    1. 使用PuTTY工具以root账户登录待安装客户端的主机。
    2. 执行以下命令进入“ProtectClient”目录。

      命令中的/opt为客户端的默认安装目录，请根据实际替换。以下操作均以“/opt”为例进行说明。

      cd /opt/DataBackup/ProtectClient
    3. 执行以下命令，停止客户端服务。
      sh stop.sh
    4. 执行以下命令启动客户端服务。
      sh start.sh

  2. 重新执行备份任务，如果仍出现此问题，请重新安装客户端。
