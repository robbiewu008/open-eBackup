Step 1: Registering a Hyper-V Virtualization Environment

Before performing Hyper-V backup and restoration, you must register a Hyper-V virtualization environment with the OceanProtect.

Context

  • When registering a Hyper-V virtualization environment with the OceanProtect, you can set Type to SCVMM, Cluster, or Host. If there are multiple independent hosts, each host must be registered.
  • After registering the Hyper-V virtualization environment, you can view information about clusters, hosts, or VMs on the WebUI and protect or restore them. Hosts and VMs can be restoration targets as well.

Procedure

  1. Choose Protection > Virtualization > Hyper-V.
  2. Click Register, and register the Hyper-V virtualization platform.

    Table 1 lists the Hyper-V registration information.
    Table 1 Hyper-V registration information

    Parameter

    Description

    Type

    Management type of the registered Hyper-V server.

    • When the SCVMM is used to manage Hyper-V servers, select SCVMM.
    • When using an independent Hyper-V server, select Host.
    • When using a failover cluster, select Cluster.

    Name

    User-defined virtualization platform name.

    Clients

    Select the host where the client corresponding to the Hyper-V server is installed.

    When Type is set to Cluster, select all hosts where the client is installed in the cluster.

    Username

    Domain administrator account and password of the AD domain where the Hyper-V server being registered resides. When Type is set to SCVMM or Cluster, the username and password are displayed.

    • When Type is set to SCVMM, enter the domain administrator account and password of the SCVMM server.
    • When Type is set to Cluster, enter the domain administrator account and password of the failover cluster.

    Password

  3. Click OK.

    After the registration is successful, information about the registered Hyper-V virtualization environment is displayed on the left of the page. You can browse the list of clusters, hosts, or VMs.
