The Entire System Fails to Be Restored to a New Location and the Error Details Contain "ErrorCode: 10300005"

Symptom

The FusionCompute VM fails to be restored to a new location, and the error details contain the following information: {"ErrorCode": "10300005", "ErrorDesc":"The entered parameter VM CPU limit is invalid. Please enter a valid value."}.

Possible Cause

A CPU maximum frequency is set for the backup VM, and the CPU frequency of the host where the backup VM resides is different from that of the host where the restoration target location resides. As a result, the lower limits of the CPU maximum frequency are different.

Method 1

Set the Target Compute Location to which the FusionCompute VM is to be restored. Select the target host with the same CPU frequency as the host where the backup VM is located.

Procedure

  1. Choose Explore > Copy Data > Virtualization > FusionCompute.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    On the Resources tab page, search for the VM to be restored by its name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. In the row of the target copy, choose More > Restore.
  5. Restore the VM to the target host with the same CPU frequency.

    1. Select New location for restoring the VM.
    2. Set the location information for restoring the VM to a new location. Select a target host with the same CPU frequency as the host where the backup VM resides as the Target Compute Location.
    3. Set the remaining parameters as required.

  6. Click OK.

Method 2

On the FusionCompute management system, select the target host to create a VM, and use the disk restoration function to restore the disk.

Procedure

  1. Log in to FusionCompute using a browser.

    Login address: https://Floating IP address of the VRM node of FusionCompute:8443

  2. Create a VM on the FusionCompute management system. The VM is used to store the disk data of the FusionCompute VM to be restored.

    1. Select Resource Pools.
    2. Click Create VM. Set the computing resource information and datastore information of the created VM to the target location of the FusionCompute VM to be restored.
    3. Click OK.

  3. Use the FusionCompute VM disk restoration function to restore the disk information to the VM created in 2. For details about how to restore a disk, see Restoring FusionCompute VM Disks.
