Exporting Request Files

When the server certificate, internal communication certificate, or internal database certificate needs to be updated, you can export a request file from the certificate and submit it to the CA for issuing a server certificate. With this server certificate, you do not need to import the server private key or enter the server private key password when updating the server certificate. With this server certificate, you do not need to import the server private key when updating the internal database certificate.

Precautions

The previously exported request file becomes invalid if you click Export Request File again. The certificate file generated using the previously exported request file will fail to be imported to the OceanCyber 300 Data Security Appliance.

Procedure

  1. Choose System > Security > Certificates.
  2. In the row that contains the certificate, choose More > Export Request File.
  3. Select an encryption algorithm from the drop-down list box for exporting request files.

    Currently, the RSA 2048 and RSA 4096 encryption algorithms are supported.

  4. Click Advanced to set the identity information including Country, State or Province, City or Locality, Organization, Organization Unit, and Common Name.
  5. Optional: Click OK.
