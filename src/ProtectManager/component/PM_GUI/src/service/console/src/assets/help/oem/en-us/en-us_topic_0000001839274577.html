Restoring Files on an ECS

This section describes how to restore files on an ECS by using the file-level restoration function.

Context

  • The product supports file-level restoration using backup copies and replication copies. Note that data cannot be restored to the original location when a replication copy is used.
  • The types of Linux file systems that can be restored are ext2/ext3/ext4 and XFS. The types of Windows file systems that can be restored are NTFS, FAT, and FAT32.

Prerequisites

  • The IP address of the backup network plane has been configured for the target ECS to which files are to be restored.
  • If the target VM runs on the Windows OS, ensure that the CIFS service has been enabled on the VM. For details, visit the Microsoft official website.
  • If the target VM runs on a non-Windows OS, ensure that rsync has been installed and started on the target VM.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Huawei Cloud Stack.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    Click Resources to switch to the corresponding page, search for the VM to be restored by name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. Locate the copy to be restored and choose More > File-level Restoration.

    • File-level restoration is supported only when the index status of the copy is Indexed.
    • If the copy does not contain OS data, no index can be created for the copy. Therefore, the copy cannot be used for file-level restoration.
    • Reverse replication copies and archive copies do not support indexes.
    • During Huawei Cloud Stack backup, the mount point information that is not written into the /etc/fstab file is stored in the memory. The product does not back up the information. Therefore, no index is generated for the files at the mount point, and file-level restoration is not supported. To generate indexes for new file systems and support file-level restoration, write the mount point information of the new file systems to the /etc/fstab file.

  5. Select the files to be restored and set restoration parameters.

    • If the name of a folder or file contains garbled characters, file-level restoration is not supported. Do not select folders or files of this type. Otherwise, the restoration job fails.
    • If the target ECS runs the Windows OS and only the SMBv1 service is enabled, the time of the restored file is the current ECS time.
    • Select the original location for restoration, that is, restore data to the original directory on the original ECS.
      Table 1 describes the related parameters.
      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Cloud Server IP Address
      Select the configured backup network plane IP address.
      NOTE:

      If the ECS IP address is not in the list, exit the current page. Then, choose Protection > Cloud Platforms > Huawei Cloud Stack and rescan the virtualization environment.

      Cloud Server Username
      During file-level restoration, the target ECS needs to verify the login credential of the product. Enter a username and password for logging in to the target ECS.
      • Windows OS: The default username is Administrator.
      • Linux OS: The default username is root.
      NOTE:

      The login user must have the read and write permissions on the directory to which data is to be restored on the target ECS.

      Cloud Server Password

      Overwrite Rule
      If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.
      • Replace existing files
      • Skip existing files: Files with the same names are skipped and are not replaced.
      • Only replace the files older than the restoration file
        NOTE:

        If the access to the file with the same name on the target VM is denied, the file fails to be restored during file replacement.
    • Select a new location for restoration, that is, restore data to a specified VM.

      Table 2 describes the related parameters.

      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Target Cloud Platform

      Select the cloud platform to which the data is restored.

      Cloud Platform Tenant

      Select a cloud platform tenant.

      Region

      Select the region to which the data is restored.

      Project/Resource Set

      Select the project/resource set to which the data is restored.

      Cloud Server

      Select the ECS to which the data is restored.

      Cloud Server IP Address
      Select the configured backup network plane IP address.
      NOTE:

      If the ECS IP address is not in the list, exit the current page. Then, choose Protection > Cloud Platforms > Huawei Cloud Stack and rescan the virtualization environment.

      • Replace existing files
      • Skip existing files: Files with the same names are skipped and are not replaced.
      • Only replace the files older than the restoration file
        NOTE:

        If the access to the file with the same name on the target VM is denied, the file fails to be restored during file replacement.

  6. Click Test and ensure that the target VM to be restored is properly connected to the product.
  7. Click OK.
