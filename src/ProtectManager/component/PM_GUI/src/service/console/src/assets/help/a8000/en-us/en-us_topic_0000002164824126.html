Managing Table Sets

The OceanProtect allows you to modify, remove, activate, or disable protection for created table sets.

Related Operations

Log in to the management page, choose Protection > Databases > TiDB, click the Table Sets tab, and locate the desired table set.

Table 1 describes the related operations.

Table 1 Table set protection operations

Operation

Description

Navigation Path

Modifying protection

Scenario

This operation enables you to modify the protection plan to associate a table set with a new SLA or other SLAs. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target table set, choose More > Modify Protection.

Removing protection

Scenario

This operation enables you to remove protection for a table set when it does not need to be protected. After the removal, the system cannot execute protection jobs for the table set. To protect the resource again, you need to associate an SLA with the resource again.

Note

When the protection job of the table set is running, protection cannot be removed.

In the row of the target table set, choose More > Remove Protection.

Activating protection

Scenario

This operation enables you to activate a disabled protection plan for a table set. After the protection plan is activated, the system backs up the table set based on the protection plan.

In the row of the target table set, choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable the protection plan of a table set for which periodic protection is not required. After the protection plan is disabled, the system does not execute automatic backup jobs for the table set.

Note

Disabling protection does not affect ongoing backup, archive, or replication jobs.

In the row of the target table set, choose More > Disable Protection.

Performing a manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target table set, choose More > Manual Backup.

Testing connectivity

Scenario

This operation enables you to test the connectivity between the OceanProtect and the table set.

In the row of the target table set, choose More > Test Connectivity.

Modifying the name of a table set

Scenario

This operation enables you to change the name of a table set.

In the row of the target table set, choose More > Modify.

Deleting a table set

Scenario

This operation enables you to delete a table set if you no longer need to protect it.

Note

A table set associated with an SLA cannot be deleted.

In the row of the target table set, choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for restoring a TiDB table set is the same as that described in Restoring TiDB Backup Resources.
