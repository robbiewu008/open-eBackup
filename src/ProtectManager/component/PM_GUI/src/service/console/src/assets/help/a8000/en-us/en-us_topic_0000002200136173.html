Managing Databases or Mailboxes

The OceanProtect allows you to modify, remove, activate, or disable protection for databases/mailboxes.

  1. Choose Protection > Applications > Exchange, click the Databases or Mailboxes tab, and find the database or mailbox to be operated.

    Table 1 describes the related operations.

    Table 1 Operations related to database/mailbox protection

    Operation

    Description

    Navigation Path

    Modify Protection

    Scenario

    You can modify a protection plan to associate another SLA with a database or modify a user-defined script. After the modification, the next backup will be executed based on the new protection plan.

    Note

    • The modification does not affect the protection job that is being executed. The new protection plan will take effect in the next protection period.
    • If the SLA associated with a resource changes, you are advised to modify the current SLA on the Protection > SLAs page. You are advised not to associate the resource with another SLA by modifying protection.

      When you associate the resource with another SLA by modifying protection, the license capacity may be deducted repeatedly if the copy of the original SLA still exists.

    In the row of the target database or mailbox, choose More > Modify Protection.

    Remove Protection

    Scenario

    You can remove the protection plan if the database no longer needs to be protected. After the removal, the system cannot execute protection jobs for the database.

    Note

    • Protection cannot be removed when a protection job of a database or mailbox is running.
    • If the SLA associated with a resource changes, you are advised to modify the current SLA on the Protection > SLAs page. You are advised not to associate the resource with another SLA after removing protection.

      When you associate the resource with another SLA after removing protection, the license capacity may be deducted repeatedly if the copy of the original SLA still exists.

    In the row of the target database or mailbox, choose More > Remove Protection.

    Activate Protection

    Scenario

    You can activate a disabled protection plan for a database or mailbox. After the protection plan is activated, the system backs up the database or mailbox based on the protection plan.

    In the row of the target database or mailbox, choose More > Activate Protection.

    Disable Protection

    Scenario

    You can disable the protection plan for a database or mailbox if periodic protection is no longer required. After the protection plan is disabled, the system does not execute the automatic backup job of the database or mailbox.

    Note

    Disabling protection does not affect ongoing backup jobs.

    In the row of the target database or mailbox, choose More > Disable Protection.

    Manual Backup

    Scenario

    This operation enables you to perform a backup immediately.

    Note

    Copies generated by manual backup are retained for the duration defined in the SLA.

    In the row of the target database or mailbox, choose More > Manual Backup.

    Adding a tag

    Scenario

    This operation enables you to quickly filter and manage resources.

    Note
    • The system administrator can add a maximum of 10,000 tags to a resource.
    • A non-system administrator must have the permission of Production resource management to add tags to resources.
    • A non-system administrator can add a maximum of five tags to a resource.
    Choose More > Add Tag.
    NOTE:
    • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
    • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.

    Removing a tag

    Scenario

    This operation enables you to remove the tag of a resource when it is no longer needed.

    Choose More > Remove Tag.

    Note: The procedure for restoring Exchange resources is the same as that described in Recovery.
