Disks Fail to Be Opened Using VDDK During VMware VM Backup

Symptom

During a VMware VM backup job, disks fail to be opened using VDDK. In subsequent backup jobs, VDDK fails to be initialized.

Possible Causes

The number of VM disks opened in NBDSSL mode on a single ESXi host reaches the upper limit of 52.

Troubleshooting

  1. Restart the client service.

    1. Use PuTTY to log in to the host where the client is to be installed as user root.
    2. Run the following command to go to the ProtectClient directory:

      In the command, /opt is the default installation directory of the client. Replace it with the actual installation directory. The following operations use /opt as an example.

      cd /opt/DataBackup/ProtectClient
    3. Run the following command to stop the client service:
      sh stop.sh
    4. Run the following command to start the client service:
      sh start.sh

  2. Perform the backup job again. If the problem persists, reinstall the client.
