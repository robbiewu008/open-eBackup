Creating a File Name Extension Filtering Rule

The system is preset with rules for filtering known ransomware file name extensions. These rules are associated with all file systems in the system by default. During file interception, the system checks whether the file name extensions in the file system are specified in the file name extension filtering rule. If yes, the system blocks the creation, opening, and renaming of the file system's files whose file name extensions are specified in file name extension filtering rule. You can create a file name extension filtering rule based on service requirements.

Procedure

  1. In the navigation pane, choose Data Security > File Interception.
  2. Go to the File Name Extension Filtering Rule tab page and click Create.

    The Create dialog box is displayed on the right.

  3. Set parameters of the file name extension filtering rule.

    Table 1 describes related parameters.

    Table 1 Parameters of the file name extension filtering rule

    Parameter

    Description

    File Name Extensions to Be Intercepted

    File name extensions to be intercepted. You can enter one or more file name extensions.

    [Value range]

    • The value contains 1 to 127 characters.
    • A single file name extension can contain only digits, letters, spaces, and special characters (!"#$ %& amp ;'()*+-./:;& lt;=&gt ;?@[\]^_`{|}~). The wildcard * must be the last character and cannot exist separately.
    • Multiple file name extensions are separated by commas (,).

  4. Click OK.
