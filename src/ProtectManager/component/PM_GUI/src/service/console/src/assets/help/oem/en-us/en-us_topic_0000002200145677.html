Management VMs/Hosts/Clusters

The product allows you to modify, remove, activate, or disable protection for VMs, hosts, or clusters. This section uses a VM as an example to describe the related operations. The operations on a host or cluster are the same.

Log in to the management page, choose Protection > Virtualization > CNware, and find the target VM.

Table 1 describes the related operations.

Table 1 Operations related to VM protection plans

Operation

Description

Navigation Path

Modifying protection

Scenario

This operation enables you to modify a protection plan of a VM to associate the SLA with the VM again, add or remove protected VM disks, or modify information such as the host where the client is installed. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target VM, choose More > Modify Protection.

Removing protection

Scenario

This operation enables you to remove the protection for a VM when it does not need to be protected. After the removal, the system cannot execute protection jobs for the VM. To protect the VM again, you need to associate an SLA with the VM again.

Note

When the protection job of the VM is running, the protection cannot be removed.

In the row of the target VM, choose More > Remove Protection.

Performing manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target VM, choose More > Manual Backup.

Activating protection

Scenario

This operation enables you to re-activate a disabled protection plan of a VM. After the protection plan is activated, the system backs up, archives, or replicates the VM data based on the protection plan.

Note

You can perform this operation only on a VM that has been associated with an SLA and is in the Unprotected state.

In the row of the target VM, choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable the protection plan of a VM when periodic protection on the VM is not required. After the protection plan is disabled, the system does not automatically back up, archive, or replicate the VM.

Note

Disabling a protection plan does not affect protection jobs that are running.

In the row of the target VM, choose More > Disable Protection.

Adding a tag

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.

Removing a tag

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The operations for restoring CNware VMs are the same as those described in Restoration.
