What Should I Do If SAP HANA Backup or Restoration Jobs Fail Because the Number of I/O Channels Is Too Small

Symptom

SAP HANA backup or restoration jobs of the general database fail because the command execution is abnormal.

Possible Cause

The initial number of I/O channels is too small. As a result, after a large number of SAP HANA backup or restoration jobs are executed, the channels are used up and the jobs fail.

Procedure

  1. Use PuTTY to log in to the host where the client used for running the current job is installed.

    The IP address of the host where the client is installed is the IP address displayed in the sub-job information. (You can view the IP address on the Jobs page of the OceanProtect WebUI.)

  2. Run the following command to open the configuration file:

    vim /etc/sysctl.conf

  3. Check the value of fs.aio-max-nr. If the value is far less than 18446744073709551615, change the value to 18446744073709551615.
  4. After modifying and saving the configuration file, run the sysctl -p /ect/sysctl.conf command for the configuration to take effect.
  5. Initiate backup or restoration jobs again.
