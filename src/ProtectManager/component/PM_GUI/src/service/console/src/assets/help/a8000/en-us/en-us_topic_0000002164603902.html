Checking the PostgreSQL Environment

Procedure

  1. Choose Protection > Databases > PostgreSQL.
  2. View the related information on the Instance tab.

    1. This page displays details about PostgreSQL instances.

      Table 1 describes related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 PostgreSQL instance information

      Parameter

      Description

      Name

      Database instance name.

      Status

      Status of the PostgreSQL cluster.

      • Online: If the connections between the host where the client is installed and all nodes in the PostgreSQL cluster are normal, Status is Online.
      • Offline: If the connections between the host where the client is installed and all nodes in the PostgreSQL cluster are abnormal, Status is Offline.

      IP Address

      IP address or domain name of a database instance.

      Owning Host/Cluster

      Name of the host or cluster where a database resides.

      Version

      Database version.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether a database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    1. Click the name and check the overview, copy data, and jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy Data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Jobs

        Displays running jobs and historical jobs.

  3. View the related information on the Cluster tab.

    1. This page displays the details about the cluster where the PostgreSQL database resides.

      If the PostgreSQL cluster has been discovered in the OceanProtect, information about the cluster and all hosts in the cluster is displayed on the page.

      Table 2 describes related parameters.

      Table 2 Information about the cluster where the PostgreSQL database resides

      Parameter

      Description

      Name

      Name of the cluster where a database resides.

      Status
      Cluster status.
      • Online: If the host is powered on and the client installed on the host is running properly, Status is Online for the host.

        If there are online hosts in the cluster, the cluster status is Online.

      • Offline: If the host is shut down or the client installed on the host is faulty, Status is Offline for the host.

        If all hosts in the cluster are offline, the cluster status is Offline.

      Cluster Type

      Type of the cluster.

      Type

      Cluster type. Active/Standby Replication: Information of the standby node in the cluster is synchronized from the active node. The information is consistent between the active and standby nodes.

      Virtual IP Address

      Virtual IP address of a database cluster.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click a cluster name to view details about hosts and nodes in the cluster.
