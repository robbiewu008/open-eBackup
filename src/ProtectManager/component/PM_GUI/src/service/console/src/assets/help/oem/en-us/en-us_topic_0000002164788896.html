Managing Event Dump (Applicable to Some Models)

Through event dump, the system periodically exports events to local disks of the storage system and deletes event records from the database beyond the retention period.

Context

You can use the product to download dumped event files and delete unnecessary event files from the directory.

Precautions

If an alarm indicating that the event storage space is about to reach the upper limit is generated, go to DeviceManager and enable alarm dump.

  1. Log in to DeviceManager by referring to Logging In to DeviceManager.
  2. Choose Settings > Alarm Settings > Alarm Dump to display the Alarm Dump page.
  3. Click next to Alarm Dump to enable alarm dump.
  4. Set alarm dump parameters. Table 1 describes the related parameters.
    Table 1 Alarm dump parameters

    Parameter

    Description

    Transport Protocol

    Transport protocol of alarm dump, which can be SFTP or FTP.

    NOTE:

    The storage systems support FTP for compatibility concerns. You are advised to use SFTP to ensure data transmission security.

    Username

    Username used for logging in to the SFTP or FTP server.

    [Value range]

    • The name must be unique.
    • The value contains 1 to 63 characters.
    • Special characters !':;|`$<>&-()#?"\* cannot be contained.

    Password

    Password used for logging in to the server.

    [Value range]

    • The value contains 1 to 63 characters.
    • A valid password cannot contain extended ASCII characters or Unicode characters. It is recommended that a password should contain the following characters: digits (0 to 9), letters (a to z and A to Z), spaces, and special characters ([\]^_{|}~`@!"#$%&'()*+-./:;<=>?).

    Server IP Address

    IP address of the server.

    [Value range]

    • When an IPv4 address is used:
      • The 32-bit IPv4 address is divided into four 8-bit fields that are expressed in dotted decimal notation.
      • Each field of the IPv4 address must be an integer.
      • The value of the first field ranges from 1 to 223 (excluding 127).
      • The values of the other fields range from 0 to 255.
      • An IP address cannot be set to a special address such as a broadcast address.
    • When an IPv6 address is used:
      • The 128-bit IPv6 address is divided into eight 16-bit fields. Each 16-bit field consists of four colon-separated hexadecimal numbers.
      • In each 16-bit field, leading zeros can be omitted for simplicity. However, at least one digit must be reserved in each field.
      • To achieve further simplicity, you can use a double colon (::) in place of a series of zeros. A double colon (::) can be used only once in an address. It can also represent the neighboring consecutive zeros at the beginning or end of an address.
      • The IP address cannot be set to a special address, such as a network address, loopback address, or multicast address.

    File Save Path

    File path for storing the dumped storage system information. A save path is set on the FTP software. You must create a folder in the save path and enter the folder name in File Save Path on DeviceManager.

    [Value range]

    • The value contains 1 to 255 characters.
    • Special characters !':;|`$<>&-()#?"\* are not allowed. The first character cannot be a period (.). The first and last characters cannot be spaces. The first character after a slash (/) in the path cannot be a period (.).
  5. (Optional) Click Test to verify the parameter values.
    • If an error dialog box is displayed, at least one parameter value is incorrect. Modify the parameter and try again.
    • If a success dialog box is displayed, the alarm dump parameters have been configured correctly.
  6. Click Save.

Procedure

  1. Choose System > Settings > Event Dump.
  2. Modify dump parameters.

    1. In the Dump Parameters area, click Modify.
    2. Set dump parameters. Table 2 describes the related parameters.
      Table 2 Event dump parameters

      Parameter

      Description

      Dumped At

      Time when events are dumped.

      The dump time is usually set to a point in time during off-peak hours.

      Dump Cycle

      Interval for the system to automatically dump event information.

      [Value range]

      7 to 120 days.

      For example, if the dump interval is set to 7 days, the system automatically dumps events every 7 days.

      Keep Database Records Within the Latest

      Days within which generated event records in the database are reserved.

      [Value range]

      7 to 120 days.

      For example, if this parameter is set to 7, the event records of the last seven days are retained in the database of the storage system.

      File Type

      Format of the event dump file.

      The value can be Excel or CSV.

      Language

      Event language.

      Simplified Chinese and English are supported.
    3. Click Save.

  3. Configure and enable the SFTP server.

    1. In the SFTP Server area, click Modify.
    2. Click to enable the SFTP server, and set SFTP server parameters. Table 3 describes the related parameters.
      Table 3 SFTP server parameters

      Parameter

      Description

      Username

      Username for the SFTP server used to store events.

      [Value range]

      • The value contains 1 to 255 characters.
      • The value cannot contain the following special characters: ! ' : ; | ` $ < > & - ( ) # ? " \ *.

      [Example]

      admin

      Password

      Password for logging in to the SFTP server.

      [Value range]

      The value contains 1 to 255 characters.

      [Example]

      user123456

      Service IP Address

      IP address of the SFTP server.

      [Value range]

      • When an IPv4 address is used:
        • The 32-bit IPv4 address is divided into four 8-bit fields that are expressed in dotted decimal notation.
        • Each field of the IPv4 address must be an integer.
        • The value of the first field ranges from 1 to 223 (excluding 127).
        • The values of the other fields range from 0 to 255.
        • An IP address cannot be set to a special address such as a broadcast address.
      • When an IPv6 address is used:
        • The 128-bit IPv6 address is divided into eight 16-bit fields. Each 16-bit field consists of four colon-separated hexadecimal numbers.
        • In each 16-bit field, leading zeros can be omitted for simplicity. However, at least one digit must be reserved in each field.
        • To achieve further simplicity, you can use a double colon (::) in place of a series of zeros. A double colon (::) can be used only once in an address. It can also represent the neighboring consecutive zeros at the beginning or end of an address.
        • The IP address cannot be set to a special address, such as a network address, loopback address, or multicast address.

      [Example]

      192.168.1.100

      fc00::1234

      Port

      Port number of the SFTP server.

      [Value range]

      1 to 65535

      [Example]

      22

      File Save Path

      Path for event dump.

      [Value range]

      • The value contains 1 to 1024 characters.
      • Special characters !':;|`$<>&-()#?"\* are not allowed. The first character cannot be a period (.). The first and last characters cannot be spaces. The first character after a slash (/) in the path cannot be a period (.).

      [Example]

      /test
    3. (Optional) Click Test to verify the parameter values.
      • If an error dialog box is displayed, at least one parameter value is incorrect. Modify the parameter and try again.
      • If a success dialog box is displayed, SFTP server parameters are correctly set.
    4. Click Save.

Follow-up Procedure

  • View and download dump files:

    In the Dump History area, you can view the event dump history. In the Dump File area where the dumped event records are located, you can click the file name to download the file to the local PC for backup.

  • Delete dump files:

    In the Dump History area, click More next to a dumped file and select Delete to delete the dumped file stored in the system.

  • Upload dump events:

    In the Dump History area, click More next to a dumped file and select Upload to upload the dump event saved in the system to the enabled SFTP server.
