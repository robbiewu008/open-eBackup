Querying DB2 Information

Procedure

  1. Choose Protection > Databases > DB2.
  2. On the Cluster tab page, query the DB2 cluster information.

    1. This page displays information about DB2 clusters registered with the product.

      Table 1 describes the related parameters.

      Table 1 DB2 cluster information

      Parameter

      Description

      Name

      DB2 cluster name.

      Status

      DB2 cluster status.

      • Online: If the connections between the agent host and all nodes in the DB2 cluster are normal, Status is Online.
      • Offline: If the connections between the agent host and all nodes in the DB2 cluster are abnormal, Status is Offline.
      • Exception: If the connections between the agent host and some nodes in the DB2 cluster are abnormal, Status is Exception.

      Type

      Cluster type.

      Owner

      User to whom resources of a tenant are allocated for management.
    2. Click the cluster name to view the cluster basic information and the information about the databases related to the cluster.

  3. On the Instance tab page, query the DB2 instance information.

    1. This page displays details about DB2 instances.

      Table 2 describes the related parameters.

      Table 2 DB2 instance parameters

      Parameter

      Description

      Name

      Instance name.

      Status

      Instance status.

      Owning Host/Cluster

      Host or cluster to which the instance belongs.

      Version

      Instance version.
    1. Click the name and check the overview information and databases.
      • Overview

        Displays the instance name, IP address, and status.

      • Databases

        Displays the database name, SLAs, SLA compliance, and protection status.

  4. On the Databases tab page, query the DB2 database information.

    1. This page displays details about DB2 databases.

      Table 3 describes the related parameters.

      Table 3 DB2 database parameters

      Parameter

      Description

      Name

      Database name.

      Owning Host/Cluster

      Host or cluster to which the database belongs.

      Owning Instance

      Instance to which the database belongs.

      SLAs

      If an SLA has been associated with the DB2 database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the DB2 database has been protected.

      • Unprotected: The DB2 database is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The DB2 database is associated with an SLA and protection is activated.
      • Creating: The DB2 database is associated with an SLA and protection is being created.
    1. Click the name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Related jobs

        Displays backup and restoration jobs.

  5. On the Tablespace Set tab page, view the information about the DB2 tablespace set.

    1. This page displays details about DB2 tablespace sets.

      Table 4 describes the related parameters.

      Table 4 DB2 tablespace set parameters

      Parameter

      Description

      Name

      Name of a DB2 tablespace set.

      Owning Cluster

      Cluster to which the DB2 tablespace set belongs.

      Owning Database

      Database to which the DB2 tablespace set belongs.

      SLAs

      If an SLA has been associated with the DB2 tablespace set, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the DB2 tablespace set has been protected.

      • Unprotected: The DB2 tablespace set is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The DB2 tablespace set is associated with an SLA and protection is activated.
      • Creating: The DB2 tablespace set is associated with an SLA and protection is being created.
    2. Click the name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Related jobs

        Displays backup and restoration jobs.
