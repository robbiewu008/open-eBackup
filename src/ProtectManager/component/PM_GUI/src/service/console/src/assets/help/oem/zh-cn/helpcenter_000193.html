在本产品管理界面配置SNMP Trap通知

背景信息

Trap是用来标志某事件发生的一种SNMP（Simple Network Management Protocol）消息类型。如果通过SNMP的方式通知告警信息，需要设置Trap服务器地址。通过设置Trap服务器地址，本产品的告警信息将被发送到Trap指定的第三方网管工具。

前提条件

系统管理员已登录管理界面。

操作步骤

  1. 在导航栏上，选择“系统 > 设置 > SNMP Trap”。
  2. 在“Trap参数配置”区域，配置SNMPv2c协议或SNMPv3协议相关参数。

    使用SNMPv2c协议将存在安全风险，建议您使用安全的SNMPv3协议。

    • 配置SNMPv3协议参数
      1. 单击“修改”。
      2. 设置“协议版本”为“SNMPv3”。

        当第三方网管的SNMP需要配置引擎ID时，您可以单击“协议版本”右侧的“查看SNMP V3 Engine ID”，根据页面显示的控制器SNMP V3 Engine ID进行配置。

      3. 设置SNMPv3协议参数，相关参数说明如表1所示。
        表1 参数说明

        参数

        说明

        用户名

        用户的名称。

        长度范围为1~64。

        授权认证协议

        用户的授权认证算法，包括“无”、“HMAC_MD5”、“HMAC_SHA1”和“HMAC_SHA2”。

        默认值为：HMAC_SHA2。

        说明：

        HMAC_SHA算法的安全性高于HMAC_MD5，建议您采用更安全的HMAC_SHA2算法。

        授权认证密码

        用户的授权认证密码。

        确认授权认证密码

        确认用户的授权认证密码。

        数据加密协议

        用户的加密协议，包括“无”、“DES”和“AES”。

        默认值为：AES。

        说明：

        加密算法的安全性能排序：AES>DES，建议您采用更安全的AES加密算法。

        数据加密密码

        用户使用数据加密的密码。

        说明：

        数据加密密码和授权认证密码不允许相同。

        确认数据加密密码

        确认用户使用数据加密的密码。

        环境名称

        环境引擎名称。

        环境引擎ID

        代表一个SNMP引擎的管理性唯一标识符，和环境名称一起使用，确定一个唯一地标识一个SNMP实体的环境。

        只有发送端的环境和接收端的环境完全匹配才对SNMP消息包进行处理，否则SNMP消息包被丢弃。
      4. 单击“保存”。
    • 配置SNMPv2c协议参数
      1. 单击“修改”。
      2. 设置“协议版本”为“SNMPv2c”。
      3. 设置团体字，相关参数说明参见表2。
        表2 团体字修改参数说明

        参数

        说明

        团体字

        有读权限和写权限的团体名称。

        说明：

        在SNMP中，团体字用于对SNMP代理访问用户进行认证和授权。配置SNMP团体字可以确保只有经过授权的用户可以访问和管理服务器。
      4. 单击“保存”。

  3. 在“Trap服务器”区域，单击“添加”，配置Trap服务器，相关参数说明参见表3。

    表3 配置Trap服务器参数说明

    参数

    说明

    IP地址

    目前只支持IPv4类型的IP地址。

    端口

    Trap服务器侦听Trap消息的端口。

    描述

    Trap服务器描述信息。

后续处理

在“Trap服务器”区域，您可以查看Trap服务器信息，同时可以删除或修改某个Trap IP地址。

本产品支持对接DME，对接操作参见对接DME的操作步骤。具体的版本配套关系参见《备份一体机 1.5.0-1.6.0 版本配套表》。
