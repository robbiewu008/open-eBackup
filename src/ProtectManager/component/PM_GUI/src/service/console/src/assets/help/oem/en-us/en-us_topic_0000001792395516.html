Performing Periodic Backup

The project/resource set or ECS to be protected needs to be associated with a specified SLA. The system periodically performs backup based on the SLA.

Precautions

  • Ensure that the production storage has reserved snapshot space for backup. Reserved snapshot space = Data volume of the source LUN x Percentage of data changes in the backup period. By default, you are advised to reserve 20% of the source LUN capacity for creating snapshots.
  • An ECS can be backed up only when it is in the following states: Running, Starting, Stopping, Stopped, Reinstalling OS, Reinstalling OS failed, Changing OS, and OS Change failed.
  • The default number of concurrent backup jobs on each ProtectAgent is 20. You are advised not to change the number. Otherwise, disk scanning may fail, causing backup failures.
  • If ECSs or EVS disks on Huawei scale-out block storage are migrated to SAN storage, you must register SAN storage resources with the product before backing up ECSs or EVS disks.
  • Before the first backup, complete resource scanning so that the latest ECS information can be displayed. For details, see "Managing an Agent Host" in the ProtectAgent Installation Guide.
  • If a snapshot fails to be deleted due to a backup job exception, the system automatically deletes unnecessary snapshots when the next full backup job is performed.

Periodically Backing Up Projects or Resource Sets

  1. Choose Protection > Cloud Platforms > Huawei Cloud Stack.
  2. Click the Project/Resource Set tab and select the project or resource set to be protected.

    If the project/resource set to be protected is not displayed, choose More > Resource Scan in the left pane to update resource information.

  3. Click Protect above the list.

    • The system backs up all ECSs in the project/resource set.
    • You can also locate the row that contains the project/resource set to be protected and choose More > Protect .

  4. Select a preset or user-defined SLA.

    You can also click Create to create an SLA.

  5. Select an agent host. If you do not select an agent host, the system randomly selects an agent host when executing the job.
  6. Set the parameters as follows:

    1. Choose whether to overwrite the existing SLA of the ECS.
      The default value is Retain existing SLAs of the cloud server.

      If you choose Retain existing SLAs of the cloud server when modifying the SLA associated with the project or resource set, the ECS SLA will be overwritten if the SLA of the ECS is the same as that of the project or resource set.

    2. You can select one or more of the following options for SLA application at a time:
      • Apply the project/resource set SLA to all unassociated cloud servers
      • Apply the project/resource set SLA to newly created cloud servers

        After Huawei Cloud Stack is scanned, the newly discovered ECS is associated with the same SLA as the project or resource set.

    3. Specify ECSs to be excluded or included in the protection plan.

      You can specify ECSs to be excluded or included based on ECS names.

      Exact match and fuzzy match of ECS names are supported. Fuzzy match rules include prefix match (for example, abc*), suffix match (for example, *abc), and middle match (for example, *abc*).

  7. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

Periodically Backing Up ECSs

  1. Choose Protection > Cloud Platforms > Huawei Cloud Stack.
  2. Click the Cloud Server tab. Select the ECS to be protected.

    If the ECS to be protected is not displayed, choose More > Resource Scan in the left pane to update resource information.

  3. Click Protect above the list.

    You can also locate the row that contains the ECS to be protected and choose More > Protect .

  4. Select the EVS disk to be backed up.

    Automatic EVS Disk Protection is enabled by default to back up all EVS disks of an ECS. You can disable Automatic EVS Disk Protection and click Please select to manually select some EVS disks for backup.

    After some EVS disks are selected for backup, the generated copies support only EVS disk-level restoration but not ECS-level restoration.

  5. Select a preset or user-defined SLA.

    You can also click Create to create an SLA.

  6. Select an agent host. If you do not select an agent host, the system randomly selects an agent host when executing the job.
  7. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
