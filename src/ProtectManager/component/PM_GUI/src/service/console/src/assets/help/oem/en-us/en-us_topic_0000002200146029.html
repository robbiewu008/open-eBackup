Abnormal Status of the Dameng Database Cluster After Restoration

Symptom

After a cluster is restored, the following exceptions exist:

  • Some standby nodes in the cluster are in the Offline state (excluding the scenario where both the active and standby nodes are offline). To view the Dameng cluster details, choose Protection > Databases > Dameng on the product WebUI and click a cluster name.
  • The database is not started properly.

Possible Causes

After the cluster is restored, not all Dameng database instances are started properly.

Troubleshooting

  1. Use PuTTY to log in to the host where the Dameng database monitor is located.

    Obtain the IP address of the host where the monitor is located and the account for logging in to the host from the database administrator.

  2. Run the following command to query the path of the monitor:

    find / -name dmmonitor

    Information similar to the following is displayed. /dm7/bin/dmmonitor is the path of the monitor.

    [root@dm7_monitor_16183 ~]# find / -name dmmonitor
/dm7/bin/dmmonitor

  3. Run the following command to query the path where the monitor configuration file is stored:

    find / -name dmmonitor.ini

    Information similar to the following is displayed. /dm7/data/dmmonitor.ini is the path of the monitor configuration file.

    [root@dm7_monitor_16183 ~]# find / -name dmmonitor.ini
/dm7/data/dmmonitor.ini

  4. Run the following command to switch to the Dameng database administrator account. The following uses username dmdba as an example.

    su - dmdba

  5. Run the following command to stop running processes:

    pkill dmmonitor

  6. Run the following command to start the monitor. In the command, /dm7/bin/dmmonitor indicates the path of the monitor, and /dm7/data/dmmonitor.ini indicates the path of the monitor configuration file.

     /dm7/bin/dmmonitor /dm7/data/dmmonitor.ini

  7. Log in to the monitor.

    1. Run the login command to access the monitor login page.
    2. Enter username SYSDBA and password to log in to the monitor.

  8. Run the following command to stop the database:

    stop instance all

    The database is stopped if the command output indicating that exiting the activity instances of all groups is successful is displayed.

  9. Run the following command to start the database:

    startup instance all

    The database is restarted successfully if the command output indicating that dmwatcher has been configured with automatic restart is displayed.

Suggestions

None

Reference

Parameters in the command for restarting the daemon process:

[dmdba@DM2 ~]$ $DM_HOME/bin/DmWatcherServicedmrw restart
