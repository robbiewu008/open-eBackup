How Should I Add a Source Deduplication User to sudoers?

If ProtectAgent is installed in automatic push mode, User Type is Super administrator, and source deduplication is required, perform operations in this section to add the source deduplication user to sudoers before initial backup.

Procedure

  1. Log in to the agent host as user root.
  2. Run the following command to search for the directory of the sudoers configuration file:

    whereis sudoers

  3. Run the following command to change the permission on the sudoers configuration file.

    chmod u+w /etc/sudoers

  4. Back up the source file.

    mkdir /etc/bak_sudoers
cp /etc/sudoers /etc/bak_sudoers

  5. Run the following command to open configuration file /etc/sudoers.

    vim /etc/sudoers

  6. Check whether the Defaults requiretty field exists in the configuration file. If yes, perform the following steps:

    • Method 1: Delete the Defaults requiretty field.
    • Method 2: Add the source deduplication user to sudoers.
      • To perform source deduplication for VMware, add VMware DataTurbo user VMDataTurbo to the configuration file.
        Defaults:VMDataTurbo !requiretty
      • To perform source deduplication for another application, add DataTurbo user data_turbo_account corresponding to the application to the configuration file.
        Defaults:data_turbo_account !requiretty

  7. View the configuration file to check whether the modification is successful.

    cat /etc/sudoers
    • If the modification fails, roll back the configuration file. After confirming that the rollback is successful, repeat 5 and 6.
      cp /etc/backup_sudoers/sudoers /etc/sudoers
cat /etc/sudoers
    • If the modification is successful, delete the backup file.
      rm -fr /etc/backup_sudoers
