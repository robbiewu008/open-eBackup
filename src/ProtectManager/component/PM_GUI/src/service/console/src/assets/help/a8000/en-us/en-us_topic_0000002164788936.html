Importing a Management Data Backup

This section describes how to import an original management data backup stored in a local directory into the system for overwriting system management data.

Precautions

  • The size of the imported backup data cannot exceed 4 GB.
  • When restoring the management data backup, do not enable the SFTP service. Enable the service only after the restoration is complete.
  • Do not perform operations related to data restoration when the SFTP service is being enabled. Perform these operations only after the service is enabled.
  • Management data generated before the SFTP service is enabled cannot be imported.
  • After the management data backup is imported, the system automatically performs a restoration job. For details about the impact, see Precautions.
  • Management data can be imported when the OceanProtect is running properly.
  • In 1.6.0 and later versions, management data can be imported after the OceanProtect is redeployed.
  • To import the management data downloaded from the SFTP server to the OceanProtect, download the data by following the instructions provided in How Do I Download Management Data from the SFTP Server (Applicable Only to 1.6.0 and Later Versions) (Applicable to OceanProtect X Series Backup Appliances). Otherwise, the management data will fail to be imported.

Procedure

  1. In the navigation pane, choose System > Settings > Management Data Backup.
  2. Click Import Backup.

    Set the parameters for importing a backup. Table 1 describes the related parameters.

    Table 1 Parameters for importing a backup

    Parameter

    Description

    File

    Select a file.

    NOTE:

    Only .zip files can be imported, and the file size cannot exceed 4 GB.

    Data Encryption Password

    Enter the data encryption password set in the backup policy.

    Signature Verification

    This function is enabled by default.

    • After this function is enabled, the integrity and source of the file will be verified.
    • After the function is disabled, you need to enter the password of super administrator admin for logging in to DeviceManager.

    Password of User admin

    Enter the password of user admin for logging in to DeviceManager.

  3. Click OK. In the displayed dialog box as shown in the following figure, confirm the information and click OK.
