Planning the Replication Network

Before creating a replication job, ensure that the replication network has been established and meets the related network requirements. This section describes the requirements for the replication network and networking support.

Requirements for the Replication Network

For replication, the replication and backup networks of the OceanProtect must use the same interface modules. Table 1 describes the requirements for the replication network.

Table 1 Requirements for the replication network

Item

Network Requirement

Replication network

For details about network connections for the replication networks of the primary and secondary storage systems, see "Networking Planning" in the OceanProtect Appliance 1.5.0-1.6.0 Installation Guide.

Replication network protocol

TCP/IP protocol

IP address

For details about the configuration requirements, see "Network Planning" in the OceanProtect Appliance 1.5.0-1.6.0 Installation Guide.

IP address type

IPv4 or IPv6

Network quality requirements
  • Round-trip time (RTT) ≤ 100 ms
  • Bandwidth ≥ 10 MB
  • Packet loss rate < 0.1%

Management network

If you plan to add the replication target cluster based on a management IP address, ensure that the management network of the primary end is connected to that of the secondary end.

Replication Network

The OceanProtect supports the following replication network types:

  • 1:1 replication
  • 1:N replication (the maximum value of N is 4)
  • N:1 replication (the maximum value of N is 16)
  • Reverse replication
  • Cascaded replication (A > B > C)
