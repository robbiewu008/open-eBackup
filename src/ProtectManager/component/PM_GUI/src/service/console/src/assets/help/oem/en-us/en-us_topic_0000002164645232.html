Managing NAS Shares

You can modify, remove, enable, or disable NAS share protection as well as perform other management operations.

Choose Protection > File Services > NAS Shares and find the desired NAS share.

For 1.5.0, choose Protection > File Services > NAS Shares.

Table 1 describes the operations.

Table 1 Operations related to NAS share protection

Operation

Description

Navigation Path

Modifying protection

Scenario

You can modify the protection plan of a NAS share. After the modification, the next backup will be executed based on the new protection plan.

Note

  • The modification does not affect the protection job that is in progress. The new protection plan is used from the next protection period.
  • If the SLA associated with a resource changes, you are advised to modify the current SLA by choosing Protection > Protection Policies > SLAs. You are advised not to associate the resource with another SLA by modifying the protection.
  • For some models of this product, if the client is co-deployed with the product to protect NAS shares, the default value and maximum value of Channels are 10 and 15, respectively. If this parameter is set to a value greater than the maximum value, the available memory of the process may be insufficient.

In the row of the target resource, choose More > Modify Protection.

Removing protection

Scenario

You can remove the protection plan if the SLA share no longer needs to be protected. After the protection plan is removed, the system cannot execute protection jobs for the NAS share.

Note

  • When a protection job of the NAS share is in progress, protection cannot be removed.
  • If the SLA associated with a resource changes, you are advised to modify the current SLA by choosing Protection > Protection Policies > SLAs. You are advised not to associate the resource with another SLA after removing the protection.

Choose More > Remove Protection.

Performing restoration

Scenario

If a file of the NAS share is lost or damaged, you can restore the file using its backup.

In the row of the target resource, choose More > Restore.

Disabling protection

Scenario

If you do not need to perform periodic protection for the NAS share, you can disable the protection plan of the NAS share. After the protection plan is disabled, the system does not perform automatic backup jobs for the NAS share.

Note

Disabling protection does not affect ongoing backup jobs.

Choose More > Disable Protection.

Activating protection

Scenario

After the protection plan of the NAS share is disabled, you can activate it again. After the protection plan is activated, the system backs up the NAS share based on the protection plan.

Note

This operation can be performed only for a NAS share that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection.

Performing modification

Scenario

If the domain name, IP address, or authentication mode of a registered NAS share is changed, you need to perform this operation to synchronize the change.

If a NAS share registered using the storage device has authentication information, you need to configure the authentication information.

Choose More > Modify.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
