Restoring MongoDB

This section describes how to use copies to restore a MongoDB instance that has been backed up to the original or a different host.

Prerequisites

  • Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.
  • Before the restoration, ensure that the instance in the target location has been stopped. Otherwise, the restoration will fail.

Procedure

  1. Choose Explore > Copy Data > Big Data > MongoDB.
  2. Search for copies by MongoDB resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the resource to be restored by resource name and click the resource name.

    On the Copy page, find the target copy based on the copy information, and choose More > Restore in the row where the copy is located.

  3. Select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, copies exist in the month or on the day.

  4. Choose More > Restore in the row where the copy is located.

    Table 1 describes the related parameters.

    Table 1 Parameters for restoring a MongoDB instance

    Parameter

    Description

    Restore To

    Select either Original location or New location.

    Location

    If you select Original location, the target location is displayed by default.

    Target Location

    If you select New location, select the target location for the restoration.

    User for Instance Start

    If this parameter is not specified, the owning user of the dbpath directory in the MongoDB directory is used by default.

  5. Click OK.
