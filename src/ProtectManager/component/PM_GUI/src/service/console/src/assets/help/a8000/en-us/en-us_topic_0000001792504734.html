Enabling or Disabling the Log Backup Function in the Production Environment

Procedure

  1. Log in to the TiUP node.
  2. Run the following command to enable or disable the log backup function (172.0.0.0:80 indicates the IP address and port number of the PD node, and local:///test/bkpbinlog0000/ indicates the absolute path of the directory for storing log backup files. Enter the values based on the site requirements).

    • Ensure that the TiDB cluster has the read and write permissions on the directories for storing log backup files on the TiUP and TiKV nodes (TiFlash nodes will be included as well, if they exist in the TiDB cluster).
    • Before the log backup function is enabled, ensure that the absolute paths of the directories for storing log backup files on the TiUP and TiKV nodes are the same and the directories are empty (If TiFlash nodes exist in the TiDB cluster, the absolute paths of the directories for storing log backup files on the TiFlash nodes must be the same as well).
    • Enable the log backup function.
      tiup br log start --task-name=pitr --pd 172.0.0.0:80 --storage "local:///test/bkpbinlog0000/"
    • Disable the log backup function.
      tiup br log stop --task-name=pitr --pd 172.0.0.0:80

  3. Run the following command to query the status of the log backup function (172.0.0.0:80 indicates the IP address and port number of the PD node):

    tiup br log status --task-name=pitr --pd 172.0.0.0:80
