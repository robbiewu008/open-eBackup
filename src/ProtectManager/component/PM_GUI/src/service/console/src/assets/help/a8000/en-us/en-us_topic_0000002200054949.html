Backup of a VM Fails When a VMKernel NIC with the NFC Backup Attribute Is Configured on the ESXi Host Where the VM Is Located

Symptom

A VM or disk backup job fails. According to the job details, instructing the client to complete the disk backup job failed, and "Failed to connect to xxx.xxx.xxx.xxx:902" is displayed in the log of the host where the client is installed. xxx.xxx.xxx.xxx indicates the IP address of the VMKernel NIC with the NFC backup attribute on the ESXi host where the VM is located.

Possible Causes

In Hot-Add, NBD, or NBDSSL transmission mode, if a VMKernel NIC with the NFC backup attribute is configured on the ESXi host where the VM is located, the NFC traffic will be forwarded to the NIC. However, the host where the client is installed cannot communicate with the IP address of the NIC. As a result, the VDDK fails to be invoked and the backup fails.

Troubleshooting

During client network planning, if a VMKernel NIC with the NFC backup attribute is configured on the ESXi host where the target VM is located, ensure proper connectivity between the host where the client is installed and the VMKernel NIC.
