Restoring a Redis Cluster

This section describes how to restore a Redis cluster that has been backed up to the original or a new location.

Context

Backup copies, replication copies, or imported copies can be used for restoration. Restoration to the original or a new location is supported.

Prerequisites

  • Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.
  • Before Redis restoration, all instances in the Redis cluster must be stopped on the MRS console.

Procedure

  1. Choose Explore > Copy Data > Big Data > Redis.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the cluster to be restored by resource name and click the name.

  3. Select the year, month, and day in sequence to find copies.

    If is displayed below a month or day, a copy is generated in that month or day.

  4. Restore data using a specified copy.

    1. In the row that contains the target copy, choose More > Restore.
    2. Restore the Redis cluster to the original or a new location.
      Table 1 describes the related parameters.
      Table 1 Restoring a Redis cluster

      Parameter

      Description

      Restore To

      Select Original location or New location.

      Location

      If you select restoration to Original location, the original location is displayed.

      Target Cluster

      If you select restoration to New location, you must specify the target cluster.
    3. Click OK.
    4. In the displayed dialog box, confirm the information and click OK.
