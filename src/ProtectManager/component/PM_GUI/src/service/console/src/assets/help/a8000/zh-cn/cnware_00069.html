恢复CNware虚拟机

当需要对已备份的虚拟机进行整机恢复时，可以参考本节恢复虚拟机至原位置或新位置。

背景信息

OceanProtect支持使用备份副本、复制副本（不支持恢复至原位置）、反向复制副本（不支持恢复至原位置）、级联复制副本（不支持恢复至原位置）、归档副本（复制副本再归档后不支持恢复至原位置）、即时挂载产生的副本、导入的副本进行虚拟机恢复，支持恢复至原位置或新位置。
  • 恢复至原位置

    恢复至原位置时，系统将在原虚拟机所在路径下（即原虚拟机所属的集群或主机）创建一个新的虚拟机，将备份数据全量恢复到新虚拟机。

  • 恢复至新位置

    恢复至新位置时，系统进行全量恢复，将备份数据恢复到指定的新路径下，即由用户指定新虚拟机所在的主机/集群以及对应的数据存储。

操作步骤

  1. 选择“数据利用 > 副本数据 > 虚拟化 > CNware”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在的行，单击“更多 > 恢复”。
  5. 恢复虚拟机至原位置或新位置。

    • 恢复虚拟机至原位置。

      相关参数说明如表1所示。

      表1 虚拟机恢复至原位置参数信息

      参数

      说明

      新虚拟机名称

      设置恢复到的虚拟机名称。

      网络位置

      选择虚拟机网卡要恢复到的端口组名称。

      启用网络适配器

      该选项用于设置恢复后的虚拟机网卡的连接状态。默认不启用，所有虚拟机网卡都处于未启用状态，虚拟机恢复后，需要登录CNware管理界面设置虚拟机网卡连接；启用后，可能导致新虚拟机网络与原虚拟机网络冲突，请恢复虚拟机后，检查新虚拟机网络。

      客户端

      指定执行恢复任务的客户端所在的主机，不指定则使用系统自动分配的客户端所在的主机。

      恢复后自动开机

      开启后，恢复完成后虚拟机将自动开机。

      恢复前执行副本校验

      执行恢复前校验副本完整性。当副本校验状态为“无效”或“--”时，不支持开启此功能。
    • 恢复虚拟机至新位置。

      相关参数说明如表2所示。

      表2 虚拟机恢复至新位置参数信息

      参数

      说明

      新虚拟机名称

      设置恢复到的虚拟机名称。

      目标位置

      选择目标虚拟机所在主机。

      虚拟机磁盘文件

      选择存放虚拟机磁盘文件的目标数据存储。

      网络位置

      选择虚拟机网卡要恢复到的端口组名称。

      启用网络适配器

      该选项用于设置恢复后的虚拟机网卡的连接状态。默认不启用，所有虚拟机网卡都处于未启用状态，虚拟机恢复后，需要登录CNware管理界面设置虚拟机网卡连接；启用后，可能导致新虚拟机网络与原虚拟机网络冲突，请恢复虚拟机后，检查新虚拟机网络。

      客户端

      指定执行恢复任务的客户端所在的主机，不指定则使用系统自动分配的客户端所在的主机。

      恢复后自动开机

      开启后，恢复完成后虚拟机将自动开机。

      恢复前执行副本校验

      执行恢复前校验副本完整性。当副本校验状态为“无效”或“--”时，不支持开启此功能。

  6. 单击“确定”。
