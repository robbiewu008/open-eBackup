How Do I Add Environment Variables on a Production Host When the Client Is Independently Deployed on the Production Host? (Applicable to 1.5.0SPC13 and Later Versions)

If the client is independently deployed on production hosts, you need to add environment variables on each host where the client is deployed. Otherwise, the Roach command will fail to be executed.

Procedure

  1. Use PuTTY to log in to all CNs in the GaussDB (DWS) cluster where the client has been deployed.
  2. Run the following command to open the env file (user omm is used as an example):

    vi /home/omm/profile/env
    Ensure that the following content has been added to the env file. The client installation directory /opt is used as an example. 
    export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/opt/DataBackup/ProtectClient/interfaces/xbsa/lib

  3. Run the following command to open the .bashrc file (user omm is used as an example):

    vi /home/omm/.bashrc
    Ensure that the following content has been added to the .bashrc file. The client installation directory /opt is used as an example. 
    export DATA_BACKUP_AGENT_HOME=/opt
