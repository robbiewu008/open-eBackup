Creating a User

The system administrator can create users of different roles in view of service requirements to control user operation permissions and ensure service system stability and service data security.

Context

For details about user roles and permissions, see Overview of User Roles.

Procedure

  1. Choose System > Security > RBAC > Users.

    For 1.5.0, choose System > Security > Users and Roles > Users.

  2. Click Create.
  3. Set user information.

    Table 1 lists the parameters.
    Table 1 User parameters

    Parameter

    Description

    User Type

    Type of the user. Available options are Local user, LDAP user, LDAP user group, and ADFS user.

    • ADFS user: logs in to the OceanProtect in ADFS authentication mode.
      NOTE:

      This user can be created in 1.6.0 and later versions.

    • Local user: logs in to the OceanProtect in local authentication mode.
    • LDAP user: logs in to the OceanProtect in LDAP server authentication mode for unified user information management.

    Name

    Name of the created user.

    • If LDAP user or LDAP user group is selected, enter the user or user group on the LDAP domain server. Otherwise, the login will fail.
    • If ADFS user is selected, ensure that the username is the same as that created on the ADFS server. Otherwise, the login will fail.

    [Value range]

    • The name contains 5 to 64 characters.
    • The name contains only digits, letters, underscores (_), and hyphens (-). It must start with a letter or underscore (_).
    • The username must be unique.

    [Example]

    user12345

    Login Authentication Mode

    Optional values are Login password and Login password+Email One-time password. This parameter is displayed only when User Type is set to Local user.

    NOTE:
    • The Login password+Email One-time password mode provides higher security. To use this mode, ensure that you have configured the email address for receiving one-time passwords by referring to Managing the Email Service.
    • Built-in accounts support only the Login password mode.

    Password

    Password of the created user. This parameter is displayed only when the user type is Local user.

    [Value range]

    • The password contains 8 to 64 characters.
    • The password must contain special characters. Special characters include ! " # $ % & ' ( ) * + , - . / : ; < = > ? @ [ \ ] ^ ` { _ | } ~ and spaces.
    • The password must contain uppercase letters, lowercase letters, and digits.
    • A password cannot contain more than two consecutive identical characters.
    • A password cannot be the same as the username or the username spelled backwards.
    • The password cannot contain the username.
    NOTE:
    • You can choose System > Security > Security Policies to modify the password policy.
    • Users should properly manage passwords to prevent leakage.

    [Example]

    a#123456

    Confirm Password

    Confirm password of the new user.

    [Value range]

    The value of Confirm Password must be the same as that of Password.

    [Example]

    a#123456

    Recipient Email Address

    Email address for receiving one-time passwords. This parameter is displayed only when the user type is Local user and the login authentication mode is Login password+Email One-time password.

    Role

    Select a built-in role provided by the system to set the permission range of the user.

    [Value range]

    User Type is set to Local user: System Administrator, Data Protection Administrator, Auditor, Remote Device Administrator, or Disaster Recovery Administrator

    User Type is set to LDAP user, LDAP user group, or ADFS user: System Administrator, Data Protection Administrator, or Auditor

    [Example]

    System administrator

    NOTE:

    Only one disaster recovery administrator can be created.

    Description

    Description of the new user.

    [Value range]

    The description can be left blank or contain up to 255 characters.

    [Example]

    User

    Max. Number of User Connections

    Maximum number of concurrent logins to the system using the new user. If this parameter is not set, the number is not limited.

    • : enabled
    • : disabled

    The default status is .

    [Value range]

    The value ranges from 1 to 8.

    Default value: 5

    NOTE:

    When Role is set to Remote Device Administrator, you do not need to set this parameter.

    Password Always Valid

    This parameter is mandatory when the role is set to Disaster Recovery Administrator. To ensure that the password is valid when you switch from eDME to the OceanProtect, enable this function.

    NOTE:

    Setting the password to never expire poses security risks.

  4. Click OK.
