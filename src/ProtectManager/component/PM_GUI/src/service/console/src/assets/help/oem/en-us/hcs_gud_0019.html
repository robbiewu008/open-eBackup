Step 7: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • Ensure that the production storage has reserved snapshot space for backup. Reserved snapshot space = Data volume of the source LUN x Percentage of data changes in the backup period. By default, you are advised to reserve 20% of the source LUN capacity for creating snapshots.
  • A cloud server can be backed up only when it is in any of the following states: Running, Starting, Stopping, Stopped, Reinstalling OS, Reinstalling OS failed, Changing OS, and OS Change failed.
  • The default number of concurrent backup jobs on each ProtectAgent is 20. You are advised not to change the number. Otherwise, disk scanning may fail, causing backup failures.
  • If cloud servers or EVS disks on Huawei scale-out block storage are migrated to SAN storage, you must register SAN storage resources with the product before backing up cloud servers or EVS disks.
  • Before the first backup, complete resource scanning so that the latest cloud server information can be displayed. For details, see "Managing an Agent Host" in the ProtectAgent Installation Guide.
  • For 1.6.0 and later versions: If a snapshot fails to be deleted due to a backup job exception, the system automatically deletes unnecessary snapshots when the next full backup job is performed.
  • During manual backup, parameters (except Automatic Retry) defined in the SLA are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy will be retained permanently. If full backup is not defined in the SLA but selected during manual backup, the copies generated by the manual full backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Cloud Platforms > Huawei Cloud Stack.
  2. Select the ECSs to be protected after clicking the Project/Resource Set or Cloud Server tab, and click Protect.

    • When you select a project/resource set on the Project/Resource Set tab page, the system protects all ECSs in the project/resource set.
    • If the project/resource set/ECS to be protected is not displayed, choose More > Resource Scan in the left pane to update resource information.

  3. Determine whether to enable automatic EVS disk protection if you select a cloud server on the Cloud Server tab page for protection. Otherwise, skip this step.

    Automatic EVS Disk Protection is enabled by default. The system protects all EVS disks (including newly detected ones) whose Storage Type is FusionStorage, OceanStor Dorado, or OceanStor on the cloud server.

    You can disable Automatic EVS Disk Protection and click Please select to manually select some EVS disks for backup.

    • After some EVS disks are selected for backup, the generated copies support only EVS disk-level restoration but not cloud server-level restoration.
    • If system disks are excluded, generated copies do not support file-level restoration.

  4. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resource to be protected, select an SLA without a WORM policy to create protection to avoid WORM policy conflicts.

  5. Set parameters in Advanced by referring to Table 1.

    Table 1 Advanced parameters

    Parameter

    Description

    Agent Host

    Select the agent host for executing backup. If you do not select an agent host, the system randomly selects one to execute backup jobs.

    NOTE:
    • For a version earlier than 1.5.0.SPC9: If an external agent is selected and the protection permission of the tenant's resources has been authorized to the specified data protection administrator, choose More > Authorize Resource in the row where the external agent resides on the Protection > Clients page to authorize the external agent to the data protection administrator to whom the tenant's resources are authorized. Otherwise, the backup will fail.
    • For 1.6.0 and later versions: Choose System > Security > RBAC and add an external agent to the resource set associated with the authorized user. For details, see "Modifying a Resource Set" in the OceanProtect Appliance 1.5.0-1.6.0 Administrator Guide.
    • For other versions, if an external agent is selected and the protection permission of the tenant's resources has been authorized to a specified data protection administrator, check whether sharing is enabled in the row where the external agent resides on the Protection > Clients page.
      • If sharing is enabled, no further action is required.
      • If sharing is disabled, choose More > Authorize Resource to authorize the external agent to the data protection administrator to whom the tenant's resources have been authorized. Otherwise, the backup will fail.
    Auto Indexing of Backup Copies
    NOTE:

    Only 1.6.0 and later versions support this parameter.

    If Cloud Server is selected for protection and the disks to be backed up contain the system disk, the system automatically creates an index for the backup copy after this function is enabled. When using the copy for restoration, you can select one or more files in the copy for restoration and search for files, directories, or links in the copy.

    SLA Overwrite Policy
    When you select a project/resource set for protection, you can choose whether to overwrite existing SLAs of ECSs with the SLA of the project/resource set.
    NOTE:

    If you choose Retain existing SLAs of the cloud server when modifying an SLA associated with the project/resource set, the ECS SLAs will be replaced when the ECS SLA is the same as that of the project/resource set.

    SLA Application Policy
    When you select a project/resource set for protection, you can choose the application scope of the SLA policy as follows:
    • Apply the project/resource set SLA policy to all ECSs that are not associated with any SLA.
    • Apply the project/resource set SLA policy to newly created ECSs.

      After the project/resource set is scanned, the newly detected ECS is associated with the same SLA as the project/resource set.

    Cloud Server Filtering
    If you want to specify the ECSs to be excluded or included in the protection plan when selecting a project/resource set for protection, enable this function and perform the following operations:
    1. Select a filtering mode.
    2. Enter an ECS name or keyword in the text box to filter ECSs based on the selected filtering mode.
      • Use commas (,) to separate multiple names or keywords.
      • Exact match and fuzzy match of ECS names are supported. Fuzzy match rules include prefix match (for example, abc*), suffix match (for example, *abc), and middle match (for example, *abc*).
    Secure archive
    NOTE:

    Only 1.6.0 and later versions support this function.
    This parameter is displayed when the selected SLA contains archive policies. If this option is selected, only backup copies that are detected to be uninfected during ransomware detection are archived.
    NOTE:

    You need to click Configure to set the ransomware protection policy for the resource to be protected if the following conditions are met: Archive Time is not set to Immediately after successful backup or Archive Policy is not set to Only the latest copy in the SLA archive policy, and the ransomware protection policy is not set. For details, see "Creating Ransomware Protection and WORM Policies" in the OceanProtect Appliance 1.5.0-1.6.0 Data Backup Feature Guide (Ransomware Protection for Copies).

  6. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  7. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.
    1. In the row of the target resource, choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple protected resources and choose More > Manual Backup in the upper left corner of the resource list.

    2. Set the name of the copy generated during manual backup.

      If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup or Forever Incremental (Synthetic Full) Backup.

      For 1.6.0 and later versions: If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    4. Click OK.
