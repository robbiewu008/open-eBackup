Restoring Hive Backup Sets

This section describes how to restore a Hive backup set that has been backed up to the original location or a new location.

Context

Backup copies, replication copies, and imported copies can be used for restoration. Restoration to the original location or a new location is supported. Restoration to the original location using a replication copy is not supported.

Prerequisites

During the backup set restoration, ensure that the registered user of the target cluster has been authorized to operate Hive database tables and the Location directory where the table data files reside.

Procedure

  1. Choose Explore > Copy Data > Big Data > Hive.
  2. You can search for copies by Hive backup set resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the backup set to be restored based on the backup set name and click the backup set name.

  3. Click the Copy Data tab page and select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in that month or day.

  4. In the row where the copy resides, choose More > Restore.

    Table 1 describes related parameters.

    Table 1 Backup set restoration parameters

    Parameter

    Description

    Restore To

    The location to which the backup set is restored.

    • Original location: Restore the backup set to the original Hive cluster.
    • New location: Restore the backup set to a new location. You need to specify the target Hive cluster.

    Target Cluster

    When restoring the backup set to a new location, you need to specify the target Hive cluster.

    NOTE:

    Encrypted Hive tables cannot be restored to a new location.

    Metadata Storage Path

    Select a path for storing metadata.

    This path is used to temporarily store table metadata. After the restoration is complete, the stored metadata is automatically deleted.

    Table Overwriting Rule

    If a table with the same name exists in the restoration path, you can select either of the following:

    • Ignore the existing table: Tables with the same names are skipped and are not replaced.
    • Replace the existing table

    Table Data File Overwriting Rule

    You can choose to delete or retain all data files in Location during the restoration.

    • Delete existing table data files
    • Retain existing table data files

    The option Delete existing table data files is selected by default. If the locations of multiple tables are the same, you are advised to select Retain existing table data files and select a data storage path for the table to be restored. Otherwise, all data files in the Location will be deleted during the restoration.

    Table Data Storage Path

    When Table Data File Overwriting Rule is set to Retain existing table data files, select a data storage path for the table to be restored.

  5. Click OK.
