Managing an Agent Host

You can modify, delete, or authorize an agent host on the management page.

You need to log in to the management page, choose Protection > Clients, and find the desired host.

Table 1 describes the related operations.

Table 1 Operations related to an agent host

Operation

Description

Navigation Path

Update Agent

Scenario

This operation enables you to install ProtectAgent of another version on an agent host.

Follow-up operations

For FusionCompute and Huawei Cloud Stack, if the production storage is OceanStor Pacific and the OceanStor Pacific certificates have been replaced on the agent host, you must replace the certificates again after updating ProtectAgent.

  • If the protected environment is FusionCompute, perform 5.h to 5.j.
  • If the protected environment is Huawei Cloud Stack, perform 13 and 14.

Note

  • Before updating ProtectAgent, collect agent host logs for future system maintenance.
    In the following description, assume logs are saved in the default installation directory of ProtectAgent. Replace the directory based on the actual conditions.
    • Window OS

      The log file is stored in the C:\DataBackup\ProtectClient\ProtectClient-E\log\Plugins\GeneralDBPlugin directory. Copy the log file to a directory other than C:\DataBackup\ProtectClient.

      NOTE:

      Before updating the agent, ensure that the log files related to the old package are closed.

    • Non-Windows OS

      Run the sh /opt/DataBackup/ProtectClientcollectlog.sh command to collect agent host logs.

      By default, the log file sysinfoxxx is stored in /opt/DataBackup/ProtectClientProtectClient-E/stmp, which is in the ProtectAgent installation directory. Move the file to a directory other than /opt.

  • When the agent is updated, the installation directory on the agent host cannot be changed. To change the installation directory, uninstall ProtectAgent and register it again.
  • During the update, ProtectAgent cannot execute protection jobs.
  • If the update fails, the system automatically performs rollback.
  • If the update and rollback both fail, you can download the ProtectAgent software package from the management page and manually install it by referring to Installing ProtectAgent (Manual Mode, Applicable to the Linux OS).

    If the host status remains Offline 5 minutes after the update is complete, the update and rollback both fail.

  • If the firewall is enabled for the protected resource and the Nginx listening port is enabled, and the agent host is offline after ProtectAgent is updated, the port may be occupied by another process during the update and the system allocates another Nginx listening port.

    In this case, check and enable the Nginx listening port by referring to Enabling the Input and Output Functions of the Nginx Listening Port.

  • Before updating ProtectAgent, if the protected resource is OpenStack, ensure that the domain name resolution configuration is correct. For details, see Configuring Domain Name Resolution (for OpenStack).

In the row of the target host, choose More > Update Agent.

Sync Trap Info

Scenario

If you have configured SNMP trap notification on the management page and want the alarm information of an Oracle host to be reported to the trap server through SNMP, you can synchronize the SNMP trap configuration to the Oracle host. If the SNMP authentication information or trap server information is modified on the management page, you need to synchronize the information.

Note

  • Only Oracle hosts support trap information synchronization.
  • For details about this operation, see Synchronizing Trap Configuration to an Oracle Host in the Data Backup Feature Guide (for Oracle).

In the row of the target host, choose More > Sync Trap Info.

Modify Application Type

Scenario

This operation enables you to modify the application type that can be protected by the current agent host.

Note
  • The application type can be changed only when the agent host is in the Online state and the agent type is General backup agent.
  • The ProtectAgent version must be the same as the version of the product.

In the row of the target host, choose More > Modify Application Type.

NOTE:

Click the host name. On the Overview page that is displayed, you can view the types of applications that can be protected by the agent host.

Resource Scan

Scenario

The product automatically triggers resource scan every hour. If you need to immediately update the information about a plug-in that is running properly on the agent host so that the corresponding applications can select the host for protection, you can manually scan resources.

Note

The agent host can be scanned only when it is in the Online state.

In the row of the target host, choose More > Resource Scan.

Modify Host

Scenario

This operation allows you to change the status of Auto-Sync Host Name and the host name.

Note

  • You can modify host information only when Agent Type of the host is VMware Backup Agent.
  • The host name cannot be changed when Auto-Sync Host Name is enabled.

In the row of the target host, choose More > Modify Host.

Delete Host

Scenario

If you do not need to use the agent host to perform protection tasks, you can delete it.

Note

  • A host that has been associated with an SLA cannot be deleted.
  • A host that is not in the Offline status cannot be deleted.

In the row of the target host, choose More > Delete Host.

Set Remarks

Scenario

This operation enables you to set remarks for an agent host.

In the row of the target host, choose More > Set Remarks.

Export Logs

Scenario

This operation enables you to export backup agent logs.

Note

  • The system and data protection administrators cannot export logs of offline agents.
  • Logs of a maximum of 10 agents can be exported at a time. A maximum of 1.5 GB log files of a single agent can be exported.
  • The new log export record overwrites the old one. Each user in the system has only one export record of the Agent Log type.
  • An auditor cannot export logs or download collected logs.

Select the target host and choose More > Export Logs in the upper area of the page.

Configure Log Level

Scenario

This operation enables you to configure backup agent log level.

Select the target host and choose More > Configure Log Level in the upper area of the page.
