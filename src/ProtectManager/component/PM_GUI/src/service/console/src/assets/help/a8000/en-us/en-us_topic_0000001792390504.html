Managing SLAs

This section describes the operations that you can perform on SLAs on the management page.

You need to log in to the management page and choose Protection > Protection Policies > SLAs to go to the SLAs page.

Table 1 describes the related operations.

Table 1 SLA-related operations

Operation

Description

Navigation Path

Cloning an SLA

Scenario

This operation helps you quickly create an SLA based on an existing SLA.

In the row of the target SLA, choose More > Clone.

Modifying an SLA

Scenario

This operation enables you to modify the SLA name as well as the backup, archive, or replication policy configured in the SLA.

Note

  • The application type cannot be changed.
  • The preset SLAs cannot be modified.
  • After the copy retention period of an SLA is changed, the retention policy of a copy generated based on the SLA before the change is not automatically changed. To change the retention period for this copy, you need to manually change its retention policy.

In the row of the target SLA, choose More > Modify.

Deleting an SLA

Scenario

This operation enables you to delete an SLA that is no longer used.

Note

  • An SLA that has been associated with resources cannot be deleted. Before deleting an SLA, ensure that protection for the resources associated with the SLA has been removed.
  • Preset SLAs in the system cannot be deleted.

In the row of the target SLA, choose More > Delete.

NOTE:

You can also select multiple SLAs and click Delete in the upper left corner to delete them in batches.
