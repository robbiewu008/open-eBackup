Starting the Windows OS in WinPE Mode and Then Configuring an IP Address

If the target host for bare metal restoration runs Windows, start the target host in WinPE mode and configure an IP address for the target host to communicate with the OceanProtect Appliance.

Prerequisites

The target host has been shut down.

Procedure

  1. Upload image file OP_Win10XPE_v3.iso to the VMware ESXi virtual platform.

    1. Log in to the VMware ESXi virtual platform.
    2. Choose Storage > Datastores > Datastore browser.
    3. Click the name of the datastore with sufficient capacity.
    4. Select a directory and click Upload to upload the image file to the directory.

  2. Mount image file OP_Win10XPE_v3.iso to the target host.

    1. Click Virtual Machines and select the target host.
    2. On the target host tab page, choose Hardware Configuration > CD/DVD drive 1 > Select disc image.
    3. Select image file OP_Win10XPE_v3.iso uploaded in 1.

  3. Set the boot options of the target host.

    1. Click Virtual Machines and select the target host.
    2. On the top of the target host, choose Actions > Edit settings > VM Options > Boot Options.
    3. Set boot options:
      • EFI: Select Force EFI setup.
      • BIOS: Select Force BIOS setup.

  4. Restart the VM.

    1. Click Virtual Machines and select the target host.
    2. Click Power On in the upper part of the target host tab page.
    3. Choose Console > Open browser console.
    4. Set the boot mode on the boot screen:
      • EFI: Select EFI UMware Virtual SATA CDROM Drive (0.0) and press Enter.
      • BIOS: Choose Boot > CD-ROM Drive, move CD-ROM Drive to the top, press F10 to save the settings and exit, and click OK.

        After the bare metal restoration for Windows is complete, you need to restore the BIOS boot sequence of the target host.

  5. After the target host is started, configure an IP address for the target host to communicate with the OceanProtect Appliance.
