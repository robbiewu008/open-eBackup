When an Oracle Database with a Large Amount of Data Is Being Restored, the Restoration Job Stops Running for a Long Time

During database restoration, the system scans database files and performs restoration. If there are a large number of files in the database, it takes a long time to scan the database files. Before the scanning is complete, the job progress does not increase on the management page.

If the restoration takes a long time, perform the following steps to check whether the system is running properly:

  1. Log in to the Oracle database host.

    • If the Linux OS is used, use PuTTY to log in to the Oracle database host and run the su - oracle command to switch to user oracle.
    • If the Windows OS is used, log in to the Oracle database host as the system administrator.

  2. Go to the /ProtectClient-E directory and view the Oracle restoration file.

    The restoration file name starts with recover_oracle.
    • If the Linux OS is used, perform the following operations:
      1. Run the following commands to view the Oracle restoration file:

        cd /opt/DataBackup/ProtectClient/ProtectClient-E

        ll tmp/recover_oracle*

        Replace /opt in the command with the actual client installation directory.

      2. Run the following command to view the last part in the file. recover_oracle1234567.rst is used as an example here.

        tail -f tmp/recover_oracle1234567.rst | grep cataloging

        If the command output keeps updating, the restoration job is running properly. Otherwise, contact technical support engineers.

    • If the Windows OS is used, check whether a file whose name starts with recover_oracle exists in the following path: If yes, open the file to check whether the content keeps updating. If yes, the restoration job is running properly. Otherwise, contact technical support engineers.

      C:\DataBackup\ProtectClient\ProtectClient-E\stmp

      Replace C:\ with the actual client installation directory.
