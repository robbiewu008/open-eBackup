Preparing for Backup

Before backing up the ClickHouse, prepare related information as follows:

  • Table 1 lists the required information.
  • Table 2 lists the information required when the authentication mode for communication between the ClickHouse cluster and OceanProtect is Database authentication.
  • Table 3 lists the information required when the authentication mode for communication between the ClickHouse cluster and OceanProtect is Kerberos authentication.
Table 1 Preparations for backup

Item

Mandatory

How to Obtain

Section

Installation path of the ClickHouse application in the production environment

Yes

Obtain the item from the MRS cluster administrator.

Step 1: Registering a ClickHouse Cluster

ClickHouse service IP address

Yes

Obtain the item from the MRS cluster administrator.

Port for the client to connect to the ClickHouse server through TCP SSL

Yes

Obtain the item from the MRS cluster administrator.

Set the port for the client to connect to the ClickHouse server through TCP SSL. The default value is 21427. The value ranges from 21420 to 21469.

Checking whether Kerberos authentication is enabled for the ClickHouse cluster

Yes

Check whether the value of hadoop.security.authentication in the core-site.xml file of the Hadoop cluster is kerberos. If information similar to the following is displayed, Kerberos authentication is enabled.

Table 3 Information required for Kerberos authentication

Item

Mandatory

How to Obtain

Section

Principal name of Kerberos authentication

Yes

Obtain the principal name from the Kerberos server administrator.

NOTE:

Set this parameter to the human-machine username of the big data platform. The user must have the permission to manage the resources to be accessed.

Step 1: Registering a ClickHouse Cluster

.conf configuration file (for example, krb5.conf) used during Kerberos authentication

Yes

Contact the Kerberos server administrator to obtain the file from the Kerberos server.

Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.

Password of the principal name

No

If Kerberos authentication is performed using a password, you need to obtain the password of the principal name.

.keytab file

No

If Kerberos authentication is performed using a keytab file, you need to obtain the .keytab file.

Contact the Kerberos server administrator to obtain the file from the Kerberos server.

Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.
