Viewing SQL Server Copy Information

This section describes how to view information about SQL Server database copies.

Procedure

  1. Choose Explore > Copy Data > Databases > SQL Server.
  2. Select a copy viewing mode to view copy information.

    Replication copies can be viewed only in the target cluster.

    • Click Resources to view database copy information by database. When you need to restore a faulty database, you are advised to search for the copy of the corresponding time point through this entry.

      Table 1 describes the related parameters.

      Table 1 Database parameters

      Parameter

      Description

      Resources

      Name

      Database name.

      Type

      SQL Server copy type.

      • Database
      • Instance
      • Availability group
      • Cluster instance

      Location

      The displayed content varies according to the SQL Server copy type.

      If the type is instance, the instance name/cluster name is displayed.

      If the type is availability group, the availability group name/cluster name is displayed.

      If the type is database, the database name/instance name/cluster name is displayed.

      Status
      Indicates whether the database is registered with the current cluster.
      • If yes, the Status is Existing.
      • If no, the Status is Not existing.

      Copy

      Quantity

      Number of backup copies or replication copies of the database with the current cluster.

      Replicated Copy SLA

      Replication copy SLA.

      Archive SLA associated with a replication copy.

      If an archive SLA has been configured for the replication copy generated by the resource, the SLA name is displayed.

      Replicated Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is enabled.
      • Creating: The replication copy is associated with an SLA and protection is being created. The status is available only in 1.5.0 and earlier versions.
    • Click Copy to view all copy information by database copy. When you need to restore data by using a specified copy, you are advised to search for the copy through this entry.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copy

      Copy Time

      Time when a copy is generated.

      Click Copy Time to view information of the copy such as expiration time. If -- is displayed for the expiration time, the copy is retained permanently.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being restored.
      • Deletion failed: The copy deletion fails.

      Location

      Location of the copy.

      • If Generation Mode of a copy is Backup, Cascaded Replication, Reverse Replication, or Replicate, the copy is stored in the corresponding backup storage unit and the name of the backup storage unit is displayed.
      • If Generation Mode of a copy is Object Storage Archive or Import, the copy is stored in an object storage bucket. In this case, the bucket name is displayed.
      • If Generation Mode of a copy is Tape Archive, the copy is stored in a tape library. In this case, the media set name is displayed.

      Generation Mode
      Generation mode of a copy.
      • Replicate: copies generated by replication jobs.
      • Backup: copies generated by backup jobs.
      • Object Storage Archive: copies generated by archiving jobs (object storage as the archive storage).
      • Tape Archive: copies generated by archiving jobs (tape library as the archive storage).
      • Cascaded Replication: copies generated by cascaded replication jobs.
      • Reverse Replication: copies generated by reverse replication jobs.

      Copy Type

      Type of backup used for generating copies.

      The type can be Full backup, Differential backup, or Log backup.

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.

      Resources

      Name

      Database name.

      Type

      Copy data type.

      Location

      Host where the database is located and instance.

      Status
      Indicates whether the database is registered with the current cluster.
      • If yes, the Status is Existing.
      • If no, the Status is Not existing.
