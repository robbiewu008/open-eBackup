Adding Object Storage (When the Archive Storage Is Object Storage)

This section describes how to add the object storage to the OceanProtect when the archive storage is object storage. If the archive storage is not the object storage, skip this section.

Prerequisites

  • An object storage platform for storing archive copies is ready.
  • The archive network of the OceanProtect has been configured and is properly connected to the object storage network.

    If the archive network is not configured, configure it by referring to "Configuring the Archive Network" in the OceanProtect Appliance 1.5.0-1.6.0 Administrator Guide.

  • You have obtained the access address and AK/SK of the object storage platform.
  • You have created a data bucket for storing archived data and an index bucket for storing index data on the object storage platform.

    To ensure data security, create a private bucket that can be used only by the OceanProtect. Otherwise, archived data may be illegally intercepted or tampered with.

Precautions

When OceanStor Pacific 8.1.0, 8.1.1, or 8.1.2, is used as the archive storage or when OceanStor Pacific 8.1.3 or later is used as the archive storage and copies are archived to a protocol-interworking bucket of OceanStor Pacific, the following restrictions exist:

  • If the /index directory exists in a copy, the copy cannot be archived.
  • If a copy contains directories and files with the same name, the copy cannot be archived.

The preceding restrictions do not apply when the archive storage is OceanStor Pacific 8.1.3 or later and data is archived to native buckets of OceanStor Pacific.

Procedure

  1. Choose System > Infrastructure > Archive Storage Library.
  2. Click Add Archive Storage.
  3. Set archive storage parameters.

    Table 1 describes the related parameters.

    Table 1 Archive storage parameters

    Parameter

    Description

    Name

    Name of an archive storage, which is customized.

    Type

    Type of the archive storage.

    NOTE:

    For 1.5.0 and earlier versions, only S3-compliant object storage is supported.

    Object Storage Product

    Object storage product connected to the OceanProtect.

    OceanStor Pacific, OceanStor 100D (FusionStorage OBS), Huawei Cloud OBS, and AWS S3 are supported.

    NOTE:

    If OceanStor Pacific is selected, you need to log in to DeviceManager of OceanStor Pacific by referring to Logging In to DeviceManager and perform the following operations to enable V2 authentication: choose Settings > Object Service Settings > Security Settings, and on the displayed Security Access Policy tab page, click to the right of Enable V2 Authentication.

    Endpoint

    Access address of an archive storage.

    Address format: domain name or service plane IP address of an archive storage.

    NOTE:

    If the access address of archive storage is in domain name format, configure the DNS service by referring to "(Optional) Configuring DNS" in the OceanProtect Appliance 1.5.0-1.6.0 Installation Guide to support domain name resolution.

    Protocol

    Protocol used for data transmission between the OceanProtect and archive storage.

    • When HTTPS is used, ensure that the imported CA certificate is a complete certificate chain file. For details, see "Creating a Certificate Chain File" in the OceanProtect Appliance 1.5.0-1.6.0 Security Configuration Guide.
    • When HTTPS is used, import a valid certificate so that the OceanProtect can verify the archive storage information. Obtain the certificate from the archive storage administrator in advance.

      For details about how to import a certificate, see Importing a Certificate in the OceanProtect Appliance 1.5.0-1.6.0 Security Configuration Guide.

    • When HTTP is used, the OceanProtect cannot verify the archive storage environment information, which poses security risks. You are advised to select HTTPS for security purposes.
    NOTE:

    If HTTP is selected, you need to log in to DeviceManager of OceanStor Pacific by referring to Logging In to DeviceManager and perform the following operations to enable HTTP: choose Settings > Object Service Settings > Security Settings, and on the displayed Security Access Policy tab page, click to the right of Enable HTTP.

    Certificate

    If HTTPS is used, select a certificate whose Type is Object Storage for verifying archive storage environment information.

    NOTE:

    You can view the certificate Type in System > Security > Certificate.

    AK

    Access certificate for accessing the object storage.

    SK

    Security certificate for accessing the object storage.

    Data bucket

    Name of the bucket that stores archived data.

    Index bucket

    Name of the bucket that stores index data.

    Use Proxy Server

    When the OceanProtect uses an agent server to access the archive storage, configure the address, username, and password of the agent server.

    NOTE:
    • If an agent server is required, ensure that the maximum size of the request body allowed by the agent server is greater than 100 MB.
    • The agent server supports only the HTTP protocol. The address format is http://Agent server IP address:Port number.

    Capacity Alarm Threshold

    Upper threshold of the used capacity of a bucket. When the used capacity reaches the threshold, an alarm is reported.

  4. Click OK.
