Restoring Informix/GBase 8s

This section describes how to restore an Informix/GBase 8s instance that has been backed up to the original or a new location.

Context

Backup, archive, and replication copies can be used for restoration. Restoration to the original or a new location is supported. If you use replication copies or archive copies generated by replication copies, the data cannot be restored to the original location.

Prerequisites

Before restoration, shut down the Informix/GBase 8s database. If the Informix/GBase 8s cluster is an HDR cluster, shut down the Informix/GBase 8s database on both the active and standby nodes.

  1. Use PuTTY to log in to the Informix/GBase 8s database host as user root.
    • On the Informix database host, run su - informix to switch to the database user.
    • On the GBase 8s database host, run su - gbasedbt to switch to the database user.
  2. Run the source ~/.bash_profile command to configure environment variables.
  3. Run the onstat -l command to check the status of the Informix/GBase 8s database.

    If On-Line (Prim), Read-Only (sec), or On-Line is displayed in the command output, the database is not shut down. Proceed with subsequent operations to shut down the database.

    • If the Informix/GBase 8s single-node system is not shut down, On-Line is displayed.
    • If the database on the active node of the Informix/GBase 8s cluster is not shut down, On-Line (Prim) is displayed.
    • If the database on the standby node of the Informix/GBase 8s cluster is not shut down, Read-Only (sec) is displayed.
  4. Run the onmode -ky command to shut down the database.

Procedure

  1. Choose Explore > Copy Data > Databases > Informix/GBase 8s.
  2. Search for copies by Informix/GBase 8s instance resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the resource to be restored by resource name and click the name.

  3. Select the resource for copy-based restoration.
  4. Specify a copy or a point in time for restoration.

    You can specify a copy or any point in time between two copies for restoration.

    For the OceanProtect E6000 Appliance, when a replication copy is used for restoration, other copies of the resource cannot be replicated at the remote end.
    • Restoration using a specified copy
      1. Select the year, month, and day in sequence to find the copy.

        If is displayed below a month or day, a copy is generated in the month or on the day.

      2. On the copy data page, click and choose More > Restore in the row where the desired copy resides.
    • Restoration to a specified point in time
      1. Select the year, month, and day in sequence to find the copy.

        If is displayed below a month or day, a copy is generated in the month or on the day.

      2. On the copy data page, click . In the blue part on the timeline, select a copy and click More > Restore.
        • For 1.5.0, a maximum of 100 copies can be displayed on the timeline. You can click to view all copies.
        • During point-in-time restoration, the information about the copy used for restoration cannot be obtained on the management page. Therefore, you cannot view the copy information of the restoration job on pages of the restoration job and related events.
      3. In the displayed dialog box, confirm the information and click Close.

      4. Restore the Informix/GBase 8s database to its original or a new location.
        Table 1 describes the related parameters.
        Table 1 Restoring the Informix/GBase 8s database

        Parameter

        Description

        Restore To

        Select either Original location or New location.

        Location

        If you select restoration to Original location, the original location is displayed.

        Tag

        NOTE:

        This parameter is available only in 1.6.0 and later versions.

        If you set Restore To to New location, you can filter the resource to be restored by tag.

        Target Instance

        Target instance for restoration, which is mandatory if you select New location for the Restore To parameter.
      5. Click OK.

  5. Click OK.
