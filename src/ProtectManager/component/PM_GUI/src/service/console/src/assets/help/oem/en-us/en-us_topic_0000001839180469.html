Step 4: Registering a Dataset

A dataset is a resource set that contains specified labels in the Kubernetes namespace. Datasets can be registered to protect data for resources with specified labels.

Precautions

During the dataset registration, ensure that the entered label contains all information about the application. Otherwise, the restoration may fail.

Procedure

  1. Choose Protection > Containers > Kubernetes CSI.
  2. Click the Dataset tab and click Register to register the Kubernetes CSI dataset information.

    Table 1 describes the Kubernetes CSI dataset registration information.

    Table 1 Dataset registration information

    Parameter

    Description

    Name

    Customize the dataset name.

    Namespace

    Name of the namespace where the dataset to be registered is located.

    NOTE:

    Before a namespace is selected, ensure that the namespace already exists in the cluster registered with the product.

    Include Selectors

    Multiple labels can be added to specify resources that contain labels in the namespace. You can query resource labels on the Kubernetes management platform.

    The following is an input example of the Key and Value of a label:

    Label:app.kubernetes.io/name=mysql
Key:app.kubernetes.io/name
Value:mysql

    Exclude Selectors

    Multiple labels can be added to exclude resources that contain labels in the namespace. You can query resource labels on the Kubernetes management platform.

    The following is an input example of the Key and Value of a label:

    Label:app.kubernetes.io/name=mysql
Key:app.kubernetes.io/name
Value:mysql

    A maximum of 16 datasets can be created in a namespace.

  3. Click OK.
