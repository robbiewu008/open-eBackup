Performing Manual Backup

If you want to perform a backup immediately, you can manually perform the backup.

Context

  • During manual backup, parameters (except Automatic Retry) defined in the SLA are applied to the manual backup job.
  • Copies generated by manual backup are retained for the period defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy is retained permanently. If full backup is not defined in the SLA but selected during manual backup, the copies generated by the manual full backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Cloud Platforms > OpenStack.
  2. Click the Project or Cloud Server tab based on the type of the data to be backed up.

    On the Project tab page, when a project is selected to perform backup, only cloud servers associated with the same SLA as the project are backed up.

  3. Click More > Manual Backup on the right of the row where the desired project or cloud server resides.

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

  4. Set the name of the copy generated during manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  5. Select a protection policy, which can be full backup or forever incremental backup (synthetic full backup).
  6. Click OK.
