Preparations for Backup

Before backing up SAP HANA databases, prepare related information by following instructions in Table 1.

Table 1 Preparations for backup

Item

Mandatory or Not (Y/N)

How to Obtain

To Be Used In

Database name and instance ID

Y
Name of the database to be protected. You can query the database name using the following method:
  1. Use PuTTY to log in to the host where the SAP HANA database resides.
  2. Run the following command to log in to the SAP HANA database terminal system. Username s00adm is used as an example.
    su - s00adm
hdbsql -i 00 -n localhost:30013 -d SYSTEMDB -u SYSTEM -p Password customized by the user during database installation;
  3. Run the following command to query the database name and instance ID:
    SELECT * FROM M_DATABASES;

    Information similar to the following is displayed, where SYSTEMDB is the database name and S00 is the instance ID.

    DATABASE_NAME,DESCRIPTION,ACTIVE_STATUS,ACTIVE_STATUS_DETAILS,OS_USER,OS_GROUP,RESTART_MODE
"SYSTEMDB","SystemDB-S00-00","YES","","","","DEFAULT"
"S00","S00-00","YES","","","","DEFAULT"

Database host

Y

Host where the database resides.

If the database is deployed in a cluster, select all hosts where the database is distributed.

Name and password of a database user

Y

Username and password used for accessing the database.

systemId and SQL Port of the database

Y
  • You can query systemId on the SAP HANA Studio, as shown in the following figure.

  • SQL Port indicates the SQL port on the master node of the system database. You can query the SQL port on the SAP HANA Studio.

    In single-tenant mode, the default SQL Port is 30015. In multi-tenant mode, the default SQL Port is 30013.

Username and password of the system database

N

This parameter is required only when the database to be protected is a tenant database.
