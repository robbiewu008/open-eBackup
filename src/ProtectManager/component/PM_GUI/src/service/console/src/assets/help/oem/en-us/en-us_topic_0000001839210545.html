About SLA

Service level agreement (SLA) is a set of configurable data protection policies that define rules for data protection and copy retention. These policies are associated with an environment to implement protection.

To meet enterprises' different data protection requirements for various environments, product allows users to customize SLAs.

Custom SLAs

Users can customize SLAs for backup and replication jobs. In an SLA, user can customize basic parameters such as the job execution frequency, time window, and copy retention duration, and other advanced parameters for various protected resources.

Table 1 describes different types of custom SLAs.

Table 1 Custom SLAs

SLA

Description

Backup SLA

Users can define the execution frequency and retention period of backup copies of a backup job.

Replication SLA

Users can specify the target cluster to which the backup copy is replicated, and customize the replication job execution frequency and copy retention period.
