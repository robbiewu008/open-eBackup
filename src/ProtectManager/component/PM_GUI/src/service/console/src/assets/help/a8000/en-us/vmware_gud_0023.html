Step 4: Obtaining the VMware Certificate

When registering a VMware environment to be protected, you are advised to import the CA certificate of the environment to be protected. Otherwise, the system cannot verify the information about the devices to be accessed, resulting in security risks. This section describes how to obtain the CA certificate of a VMware environment to be protected.

Procedure

  • Method 1: Obtain it from the VMware administrator.
  • Method 2: Download it by yourself.
    1. If the environment to be protected is an independent ESXi host, perform this step to check whether the ESXi host has been managed by a vCenter Server. Otherwise, skip this step.
      Assume that ESXi 8.0 is used.
      1. Log in to ESXi using a browser.

        Login address: https://ESXi management IP address

      2. Choose Host > Manage > Security & users > Certificates, and view the Issuer parameter.
        OU=VMware Engineering,O=vcenter21.hw.com,ST=California,C=US,DC=local,DC=vsphere,CN=CA
        • If the value of Issuer contains a vCenter Server domain name (for example, vcenter21.hw.com in the preceding example), the ESXi host has been managed by the vCenter Server and the ESXi host certificate is issued by this vCenter Server. In this case, go to 2.
        • If no vCenter Server domain name is contained in the value of Issuer, the ESXi host has never been managed by a vCenter Server. In this case, obtain the CA certificate from the VMware administrator and skip the subsequent steps.
    2. Access the address for logging in to the vCenter Server using a browser.

      Login address: https://vCenter Server domain name

    3. Click Download trusted root CA certificates on the right of the page to download the certificate package to a local directory.

      If garbled characters are displayed in the browser during the download, you are advised to:

      • Method 1: Use another browser (for example, Internet Explorer) for download.
      • Method 2: Right-click the blank area on the page with garbled characters and choose Save as from the shortcut menu to save the compressed certificate package to the local end.
    4. Decompress the certificate package and change the file name extension of the certificate as follows:
      • For VMware vSphere 6.0, find the certificate file in *.0 format and change the certificate format to .0.pem.
      • For the VMware vSphere versions 6.5, 6.7, 7.x, and 8.0, find the certificate file in *.crt format and change the certificate format to .pem.

        If there are multiple *.crt files, double-click a *.crt file to open it. If the values of Issued to and Issued by in the certificate information are CA, the *.crt file is the desired one.
