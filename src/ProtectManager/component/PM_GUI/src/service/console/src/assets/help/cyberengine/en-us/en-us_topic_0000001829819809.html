Viewing File System Information

Prerequisites

File system resources on the storage system have been available on the OceanCyber 300 Data Security Appliance.

Context

File systems created by users on the storage system can be automatically discovered on the OceanCyber 300 Data Security Appliance and displayed in Data Security > Ransomware Protection > Intelligent Detection > File Systems.

When the OceanCyber 300 Data Security Appliance scans for a new file system that is created on the storage system, it may take some time for the OceanCyber 300 Data Security Appliance to discover the file system.

Precautions

During intelligent ransomware detection, if the file system snapshot directory of the ransomware detection snapshot being detected is invisible, the system will create a clone file system whose name starts with DEE-ANTI-CLONE based on the ransomware detection snapshot. The clone file system is used to generate metadata features of the snapshot. During the detection, do not delete or rename the clone file system whose name starts with this prefix. If the file is deleted or renamed, the detection may fail. After the intelligent detection, the system automatically deletes the clone file system whose name starts with this prefix.

Procedure

  1. Choose Data Security > Ransomware Protection > Intelligent Detection > File Systems.
  2. In the file system list, view the file system information. Table 1 describes the details about a file system.

    Table 1 File system details

    Parameter

    Description

    Name

    File system name.

    Storage Device

    Storage device to which the file system belongs.

    Tenant

    vStore to which the file system belongs.

    Intelligent Detection Policies

    If the file system has been associated with an intelligent detection policy, the intelligent detection policy name is displayed. Otherwise, -- is displayed.

    Compliance

    Whether a ransomware detection snapshot is successfully generated in compliance with the preset ransomware detection snapshot generation interval or ransomware detection snapshot generation window. For example, if the ransomware detection snapshot generation time in a ransomware detection snapshot generation job (except the first generation) meets the ransomware detection snapshot generation window or interval requirements and the generation is successful, Met is displayed. If the ransomware detection snapshot generation interval set in intelligent detection for a ransomware detection snapshot generation job is once every hour but the ransomware detection snapshot generation time exceeds an hour, Missed is displayed.

    Protection Status

    Whether the file system is protected.

    • Unprotected: The file system is not associated with an intelligent detection policy, or the file system is associated with an intelligent detection policy but protection is disabled.
    • Protected: The file system has been associated with an intelligent detection policy and protection has been activated.
    • Creating: The file system has been associated with an intelligent detection policy and protection creation is in progress.

    learning Status

    Learning status of the file system.
