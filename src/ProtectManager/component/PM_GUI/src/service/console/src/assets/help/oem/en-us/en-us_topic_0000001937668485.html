Step 1: Adding a Storage Device

Before backing up or restoring an NDMP NAS file system, add a storage device where the NDMP NAS file system to be protected resides to the product.

Context

The product can protect NDMP-based NAS file systems.

Procedure

  1. Choose Protection > File Systems > Storage Device.
  2. Click Add Device to add a storage device.
  3. Select the NDMP Server device type.
  4. Configure parameters as required. Table 1 describes the related parameters.

    Table 1 Parameters for adding a device

    Parameter

    Description

    Device Name

    Name of a user-defined device.

    IP Address

    Management IP address of the storage device.

    Port

    Number of the port for communication between the product and the management plane of a storage device.

    The default port number is 10000.

    Username

    Username and password of a storage device NDMP administrator. Ensure that the RESTful login mode is available for the user and the password has been initialized.

    NOTE:

    Use an administrator account. Otherwise, resources on the device may fail to be backed up.

    Password

    Agent Host

    By default, the built-in agent is selected. If an external agent exists, you can also select the external agent.

  5. Click OK.

    When a storage device is added successfully, information about the added storage device is displayed on the Storage Device page.

    The product automatically scans NAS file systems in the storage device and displays them on the Protection > File Systems > NAS File Systems page. If a file system is added to the registered device, you can choose More > Resource Scan in the row that contains the device to update the file system resources.
