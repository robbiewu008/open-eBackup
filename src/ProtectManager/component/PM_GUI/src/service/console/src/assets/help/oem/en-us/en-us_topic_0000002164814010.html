After the System Disk of a VMware VM Is Restored to a New VM, the Target VM Fails to Be Started

Symptom

After the system disk of a VMware VM is restored to a new VM, the target VM, upon startup, enters the Boot Manager page and cannot be started using any boot option.

Possible Causes

When the system disk of a VM is restored to a new VM that runs a different OS or OS version, the boot option of the source VMware VM is BIOS, but that of the target VM can only be EFI.

Troubleshooting

  1. Use a browser to log in to the vCenter Server and choose VM Options > Boot Options > Firmware to view the boot option of the corresponding VM.
  2. If the boot options of the source VM and target VM are different, perform VM restoration.
