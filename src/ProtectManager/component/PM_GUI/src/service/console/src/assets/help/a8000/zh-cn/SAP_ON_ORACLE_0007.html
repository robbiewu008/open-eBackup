配置客户端所在主机

在已安装SAP ERP的主机上，请手动创建run文件夹和拷贝文件、启动数据库并创建数据库用户、根据获取的SAP ERP信息添加主机环境变量。

背景信息

在安装Oracle时，用户会在cmd界面设置SID；在安装SAP ERP时，用户会设置安装路径、SAP实例ID、数据库实例ID。

完成安装后，系统会根据用户设置的SID、安装路径、SAP实例ID、数据库实例ID生成相关目录。根据所输入参数生成的相关目录说明，请参见表1。

  • 若用户忘记了设置的“SID”、“SAPSID”和“DBSID”值，可查看表1中列举的相关目录获取对应的值。例如：

    • “C:\oracle\EH9\11203”，即安装时设置的“SID”为“EH9”。

手动创建run文件夹和拷贝文件

完成SAP ERP的安装后，系统在“<Destination Drive>:\usr\sap\<SAPSID>\SYS\exe”（“Destination Drive”和“SAPSID”相关说明，请参见表1）路径下，会生成uc文件夹。请在该路径新建一个与uc文件夹同级的run文件夹，并将uc文件夹内的如下7个文件拷贝至新建的run文件夹中：

  • brarchive
  • brbackup
  • beconnect
  • brrecover
  • brresotre
  • brspace
  • brtools

启动数据库并创建数据库用户

在windows主机的命令提示符（cmd）窗口（您可以使用快捷键Win + R、输入cmd后按Enter进入），执行以下步骤，启动数据库并创建数据库用户：

sqlplus / as sysdba
startup;
create user userName identified by "passWord";
grant dba,sysoper,create session to userName;
commit;

其中，“userName”和“passWord”为用户自定义创建的数据库用户名和密码。例如：
