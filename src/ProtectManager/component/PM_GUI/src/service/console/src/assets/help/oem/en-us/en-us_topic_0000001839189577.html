Manually Executing Backup

If you want to perform a backup job immediately, you can manually perform the backup.

Context

  • During manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Channels, Rate Limiting Policies, and Transmission and Data Encryption are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication policy has been defined in the SLA, the product will perform replication once based on the SLA when you perform manual backup.
  • If no log backup is defined in the SLA, log backup cannot be manually executed.

Procedure

  1. Choose Protection > Databases > Dameng.
  2. In the row that contains the database to be backed up, choose More > Manual Backup.

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

  3. Set the name of the copy generated during manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  4. Select a protection policy, which can be Full backup, Incremental backup, Differential Backup, or Log Backup.
  5. Click OK.
