Viewing Exchange Copy Information

This section describes how to view details about Exchange copies.

Procedure

  1. Choose Explore > Copy Data > Applications > Exchange.
  2. Choose a resource viewing mode in the upper right corner of the page to view the copy information.

    Replication copies can be viewed only in the target cluster.

    • Click the Resources tab to view the copy information of a resource by single-node system, availability group, database, or mailbox. If a resource is faulty and data needs to be restored, you are advised to search for the copies at the time point through this entry.

      Table 1 describes the related parameters.

      Table 1 Resource parameters

      Parameter

      Description

      Resources

      Name

      Database name.

      Location

      IP address of the host where the database resides.

      Status

      Whether the database is registered with the current OceanProtect WebUI.

      • Yes
      • No

      Copy

      Quantity

      Number of backup copies or replication copies of the database instance on the current OceanProtect.

      Replicated Copy SLA

      Allows you to specify the target to which the backup copy is replicated, and customize the replication job execution interval and copy retention period.

      Replicated Copy Protection Status
      Indicates whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is activated.
    • View all copy data by copy. When you need to recover data by using a specified copy, you are advised to search for the copy through this entry.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copy

      Backup Time

      Time when a backup copy is generated.

      Copy Time

      Time when a copy is generated.

      You can click the time displayed in the Copy Time column to view information such as the copy expiration time.

      If the expiration time is --, the copy is retained permanently.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being used for recovery.
      • Deletion failed: The copy deletion fails.

      Location
      Location of a copy.
      • If Generation Mode of a copy is Backup, Cascaded Replication, Reverse Replication, or Replicate, the copy is stored in the corresponding backup storage unit and the name of the backup storage unit is displayed.
      • If Generation Mode of a copy is Object Storage Archive or Import, the copy is stored in an object storage bucket. In this case, the bucket name is displayed.
      • If Generation Mode of a copy is Tape Archive, the copy is stored in a tape library. In this case, the media set name is displayed.

      Generation Mode

      Mode in which a copy is generated.

      • Replicate: copies generated by replication jobs.
      • Backup: copies generated by backup jobs.
      • Object Storage Archive: copies generated by object storage archive jobs.
      • Tape Archive: copies generated by tape archiving jobs.
      • Live Mount: copies generated by live mount jobs.
      • Cascaded Replication: copies generated by cascaded replication jobs.
      • Reverse Replication: copies generated by reverse replication jobs.

      Copy Type

      Backup type of a generated copy.

      Options are full, incremental, log, and forever incremental (synthetic full) backup.

      Expired At

      Copy expiration time. If a copy is permanently valid, -- is displayed.

      WORM

      Indicates whether a copy has been set as a WORM copy.

      • : Disabled. The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : Enabled. The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : Setting. The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : Failed. The copy fails to be set as a WORM copy. A copy which fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.

      WORM Expiration Time

      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.

      Resources

      Name

      Name of the Exchange resource for which a copy is generated.

      Type

      Exchange copy type.

      • Exchange single-node system
      • Exchange availability group
      • Exchange database
      • Exchange mailbox

      Location

      Location of the host where the Exchange resource with a copy generated resides.

      Status

      Whether the Exchange resource with a copy generated is registered with the current OceanProtect WebUI.

      • Yes
      • No

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
