Restoring Files in a NAS Share

This section describes how to restore a NAS share file to its original location, a new location, or a local location.

Context

  • The product supports file-level restoration using backup copies.

Precautions

If a folder or file name contains garbled characters, file-level restoration is not supported. Do not select folders or files of this type. Otherwise, file-level restoration fails.

Prerequisites

  • File-level restoration is supported only when the index status of the copy is Indexed.
  • To restore a NAS share to a new location, register the NAS share at the new location or the storage device where the NAS share resides with the product. The following lists the supported storage device types.
    • OceanStor Dorado 6.x
    • OceanStor 6.x
    • OceanStor V5
    • OceanStor Pacific
    • NetApp ONTAP
    • OceanProtect 1.3.0 and later versions
  • If you want to restore a NAS share to a new location, you need to configure the access permission for the product on the new storage device. For details, see Step 5: Configuring Access Permissions. During the operation, configure a client or add a user for the target NAS share.

Procedure

  1. Choose Explore > Copy Data > File Systems > NAS Shares.

    For 1.5.0, choose Explore > Copy Data > File Services > NAS Shares.

  2. A copy can be searched by NAS share resource or copy. This section uses the NAS share resource as an example.

    On the Resources tab page, locate the NAS share to be recovered by NAS share name and click the name.

  3. On the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or day.

  4. Locate the copy for restoration and click More > File-level Restoration on the right of the row where the copy is located.
  5. Select the object to be restored and the restoration target location.

    The restoration target location can be Original location, New location, or Local Host Location.

    • Original location: Restores data to the original file system.
    • New location: Restores data to the file system of another device.
    • Local Host Location: Restores data to the product.

    For details about the parameters of the original location, see Table 1. For details about the parameters of a new location, see Table 2 and Table 3. Table 1 describes the local location parameters.

    Table 1 Parameters for restoring data to the local location

    Parameter

    Description

    File Systems

    Select the file system to be restored to the local device. When restoring data to the local location for the first time, you can click Create to create a file system. For details about how to create a file system, see Creating a File System.

    Overwrite Rule

    If a file with the same name exists in the restoration path, you can select replacing or skipping the existing file.

    • Replace existing files
    • Skip existing files
    • Only replace the files older than the restoration file

  6. Click OK.
