Step 7: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA. The system protects the resources and periodically executes backup jobs based on the SLA. You can perform a backup job immediately through manual backup.

Precautions

  • If a full backup fails and the next backup is a log backup, the log backup will fail because the log backup depends on the copy generated in the last successful full backup. In this case, manually perform a full backup.
  • During manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

Prerequisites

The prerequisites for backing up a single-node GaussDB T database are as follows:

  • The GaussDB T database is running properly.
  • If full backup or log backup is required, ensure that log archiving has been enabled before backup.

The prerequisites for backing up a GaussDB T cluster database are as follows:

The database needs to run in ARCHIVELOG mode and archive redo logs to multiple locations. For details about how to query and change the log archiving mode, see Step 2: Setting the Redo Log Mode.

Procedure

  1. Choose Protection > Databases > GaussDB T.
  2. Select the database instance to be protected and click Protect.

    You can also select multiple database instances for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resources to be protected, select an SLA without a WORM policy to avoid WORM policy conflicts.

  1. Click OK.

    If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  2. Optional: Execute manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. Choose More > Manual Backup in the row containing the database to be backed up. Multiple resources can be manually backed up at the same time.
    2. Set the name of the copy generated during manual backup.

      If this parameter is left empty, the system sets the name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup, Incremental backup, Differential Backup, or Log Backup.
      • Incremental backup and Differential Backup apply only to GaussDB T cluster backup scenarios.
      • For 1.6.0 and later versions, if the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.
    4. Click OK.
