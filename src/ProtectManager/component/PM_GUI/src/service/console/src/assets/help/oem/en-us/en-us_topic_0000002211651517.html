Registering a Database (Applicable to the Cluster Deployment Mode)

This section describes how to register a DB2 database cluster when the database to be protected is deployed in a cluster.

Procedure

  1. On the DB2 page, click the Cluster tab.
  2. Click Register.
  3. On the Register page, customize a cluster name and select the type and cluster node.
  4. Table 1 describes the related parameters.

    Table 1 Parameters for registering a cluster

    Parameter

    Description

    Name

    User-defined cluster name.

    Type
    The value can be Database Partitioning Feature, PowerHA, HADR, or RHEL HA.
    NOTE:

    RHEL HA clusters are supported in 1.6.0 and later versions.

    To determine the cluster type, perform the following steps:
    • HADR
      1. Use PuTTY to log in to the host where the DB2 database resides.
      2. Switch to the DB2 database user, for example, db2inst1.
        su - db2inst1
      3. Check whether the cluster is of the HADR type. Database HADRDB is used as an example.
        db2 get db cfg for HADRDB | grep -i 'HADR'

        In the command output, HADR is displayed in the right column, indicating that the cluster is of the HADR type.

      4. Check the status of the HADR cluster. For example, run the following command to check the HADR cluster status for database HADRDB:
        db2pd -db HADRDB -hadr
        • The command output shown in the following figure indicates the active node in the HADR cluster.

        • The command output shown in the following figure indicates the standby node in the HADR cluster.

    • Database Partitioning Feature
      1. Use PuTTY to log in to the host where the DB2 database resides.
      2. Switch to the DB2 database user, for example, db2inst1.
        su - db2inst1
      3. Check whether the cluster is of the Database Partitioning Feature type.
        $ db2 get db cfg for Database name | grep -i 'partition'
        In the command output, Database Partitioning Feature indicates that the cluster is of the Database Partitioning Feature type. 
        Catalog database partition number                (CATALOGPARTITION) = 0
 Partitioning strategy                               (DATABASEPARTITIONINGSTRATEGY) = Database Partitioning Feature
    • PowerHA
      1. Use PuTTY to log in to the host where the DB2 database resides.
      2. Check whether the OS is AIX.
        uname -s

        In the command output, if AIX is displayed, go to 3. If AIX is not displayed, the cluster type is not PowerHA.

      3. Check whether the cluster is of the PowerHA type.
        lssrc -g cluster

        In the command output, clstrmgrES and clconfd are subsystems of a PowerHA cluster, indicating that the cluster is of the PowerHA type.

    • RHEL HA
      1. Use PuTTY to log in to the host where the DB2 database resides.
      2. Check whether the cluster is of the RHEL HA type.
        pcs cluster status

        The command output shown in the following figure indicates that the cluster is of the RHEL HA type.

    Nodes

    Host where the DB2 instance resides.

  5. Click OK.
