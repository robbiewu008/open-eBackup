Viewing Copy Information About an NDMP NAS File System

This section describes details about NDMP NAS file system copies.

Procedure

  1. Choose Explore > Copy Data > File Systems > NDMP.
  2. Select a copy viewing mode to view the copy information.

    Replication copies can be viewed only in the target cluster.

    • Resources: View the copy information of file systems by file system. If data in a file system is lost and you want to restore the file system, you are advised to search on this portal for the copy at the corresponding time point.

      Table 1 describes the related parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Name

      Name of a NAS file system.

      Click the name of a NAS file system to view its copy data.

      You can search for copies by year, month, or day.

      If is displayed below the time, a copy at that time exists.

      Click Clear Index to delete all indexes of a resource.

      NOTE:
      • If a copy is being indexed, it cannot be deleted until the indexing is complete.
      • If Automatic Indexing is enabled for the SLA associated with a resource and Automatic Indexing is enabled for the backup policy, you need to disable Automatic Indexing or disassociate the resource from the SLA before deleting the index. If Automatic Indexing is enabled for the archiving policy, you need to disassociate the resource from the SLA before deleting the index.

      Location

      Location of a NAS file system.

      Status

      Whether the resource where the NAS file system resides is registered with the current management platform.

      • If yes, Status is Existing.
      • If no, Status is Not existing.

      Copies

      Quantity

      Number of copies generated based on the NAS file system. The number of replication copies is displayed only in the target cluster.

      Replicated Copy SLA

      Archive SLA associated with a replication copy.

      If an archive SLA has been configured for the replication copy generated by the resource, the SLA name is displayed.

      Replicated Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is enabled.
      • Creating: The replication copy is associated with an SLA and protection is being created. The status is available only in 1.5.0 and earlier versions.
    • Copies: View information about all copies by copy. When you need to restore data to the original host or a different host based on a specified copy, you are advised to search for the copy through this portal.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copies

      Backup Time

      Time when a backup copy is generated.

      Copy Time

      Time when a copy is generated.

      Click Copy Time to view the expiration time and other information of the copy. If -- is displayed as the expiration time, the copy is retained permanently.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being restored or instantly recovered.
      • Mounting: The copy is being mounted.
      • Unmounting: The copy is being destroyed.
      • Mounted: The copy has been mounted.
      • Deletion failed: The copy fails to be deleted.

      Location

      Location of a copy.

      • When Generation Mode of a copy is Backup, Cascaded Replication, Reverse Replication, Replicate, or Live Mount, the copy is stored on the local OceanProtect. In this case, Local is displayed.
      • If Generation Mode of a copy is Object Storage Archive or Import, the copy is stored in a bucket of the object storage. In this case, the bucket name is displayed.
      • When Generation Mode of a copy is Tape Archive, the copy is stored in the tape library. The media set name is displayed.

      Generation Mode
      Generation mode of a copy.
      • Replication: copies generated by a replication job.
      • Backup: copies generated by a backup job.
      • Object Storage Archive: copies generated by an archiving job (object storage as the archive storage).
      • Tape Archive: copies generated by an archiving job (tape library as the archive storage).
      • Live Mount: Copies are generated by live mount jobs.
      • Import: archive copies imported from archive storage to the OceanProtect.
      • Cascaded Replication: Copies are generated by cascaded replication jobs.
      • Reverse Replication: Copies are generated by reverse replication jobs.

      Copy Type

      Backup type of a generated copy, which includes Forever Incremental (Synthetic Full).
      Expired At
      NOTE:

      This parameter is available only in 1.5.0SPC26 and later versions.

      Copy expiration time. If a copy is permanently valid, -- is displayed.

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.
      • : The copy has expired. This state is available only in 1.6.0 and later versions. It indicates that the WORM copy has expired. Copies in this state can be deleted.
      WORM Expiration Time
      NOTE:

      This parameter is available only in 1.6.0 and later versions.
      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.
      NOTE:

      The WORM expiration time is based on the WORM file system compliance clock. If the WORM file system compliance clock time is inconsistent with the system time, check the WORM status after the WORM file system compliance clock reaches the set expiration time.

      Copies

      Index
      If the Automatic Indexing function is enabled, the system creates indexes for all files in a NAS file system, helping you quickly search for the required files for restoration. Possible states are described as follows:
      • Indexed: The system has created an index for a copy.
      • Indexing: The system is creating an index for a copy after it is generated.
      • Not indexed: The Automatic Indexing function is disabled in an SLA.
      • Index deleting: The index is being deleted.
      • Index failed: The system fails to create an index for a copy.

      Resources

      Name

      Name of a NAS file system.

      Location

      Location of a NAS file system.

      Status

      Whether the NAS file system is registered with the current management platform.

      • If yes, Status is Existing.
      • If no, Status is Not existing.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
