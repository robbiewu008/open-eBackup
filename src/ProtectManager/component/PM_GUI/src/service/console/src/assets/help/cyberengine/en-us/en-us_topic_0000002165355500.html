Performing a Detection Manually

To execute a detection job immediately, you can perform manual detection.

Precautions

In HyperMetro domain scenarios, the secondary file system does not support intelligent detection.

Procedure

  1. Choose Data Security > Intelligent Detection.
  2. Click the File Systems tab.
  3. In the row of the target file system, choose More > Manually Detect.

    You can also select multiple file systems and choose More > Manually Detect in the upper part.

  4. Enter the snapshot name. If you do not set this parameter, the default snapshot name snapshot_Timestamp is used.
  5. Configure the snapshot retention policy. Once the snapshot retention period ends, the system automatically deletes expired snapshots.
  6. Set Backup Copy In-Depth Detection.

    This parameter applies only to the file system of the OceanProtect Backup Storage device. After this function is enabled, the OceanCyber 300 Data Security Appliance performs in-depth parsing and detection on backup copy files in the backup storage to evaluate whether the original files (files on the backup production storage device) in the backup copy files are infected. If this function is enabled, the overall detection time may be extended.
    • After Backup Copy In-Depth Detection is enabled, you can adjust the sensitivity of the backup copy detection algorithm. If Sensitivity is set to High, an alarm may be triggered when a small amount of data is encrypted or similar operations are performed, increasing the risk of misreports. Medium sensitivity is recommended for service scenarios without special requirements.
    • The sensitivity adjustment of the backup copy detection algorithm takes effect only for VMs, databases, or Veeam-based host file backup copies.
    • You can use the backup software to check whether a protected object corresponding to the backup copy is infected by performing the following steps:
      1. On the OceanCyber management page, obtain the information about the corresponding backup job in the backup software based on the infected backup copy.
        1. On the Snapshot Management and Restoration page, locate the row that contains the target resource and choose More > Latest Detection Result.
        2. In the Protected Object List area, click the name of a protected object to view Owning Backup Job.
          • If Commvault is used as the backup software, Job ID is displayed in this column, corresponding to Job ID in the backup software.
          • If NetBackup is used as the backup software, the policy of the backup job is displayed. You can locate the backup job in the backup software based on the policy, client name, and copy generation time.
          • If Veeam is used as the backup software, the backup job name is displayed in this column. You can locate the backup job in the backup software based on the backup job name, client name, and copy generation time.
      2. On the target backup software page, identify the protected object and copy generation time based on the backup job information.
      3. Confirm the production storage and host information based on the protected object and copy generation time.
      4. Check whether the original data is infected on the production storage.

  7. Set Uninfected Snapshot Lock. The Uninfected Snapshot Lock parameter means that snapshots for which no ransomware file is detected will be locked. After this function is enabled, uninfected snapshots will change to secure snapshots, and the snapshot retention period will be prolonged. Modification or deletion is not allowed before the snapshots expire.
  8. Click OK.
