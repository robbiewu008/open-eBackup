Managing Instances

The system allows you to modify, remove, activate, or disable protection plans for instances.

Choose Protection > Cloud Platforms > GaussDB. On the Instance tab page, locate the target instance.

For 1.5.0, choose Protection > Cloud Platforms > GaussDB.

Table 1 describes the related operations.

Table 1 Instance protection plan operations

Operation

Description

Navigation Path

Modify Protection

Scenario

You can modify a protection plan to modify the associated SLA information of an instance. After the modification, the next backup will be performed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target instance, choose More > Modify Protection.

Remove Protection

Scenario

You can remove the protection if the instance no longer needs to be protected. After the removal, the system cannot execute protection jobs for the instance. To protect the instance again, you need to associate an SLA with the instance again.

Note

When a protection job of the instance is running, protection cannot be removed.

In the row of the target instance, choose More > Remove Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target instance, choose More > Manual Backup.

Activate Protection

Scenario

This operation enables you to activate the protection plan for an instance. After the protection plan is activated, the system backs up, archives, or replicates the instance data based on the protection plan.

Note

This operation can be performed only on instances that have been associated with an SLA and are in the Unprotected state.

In the row of the target instance, choose More > Activate Protection.

Disable Protection

Scenario

This operation enables you to disable the protection plan of an instance for which periodic protection is not required. After the protection plan is disabled, the system does not automatically back up, archive, or replicate the instance data.

Note

Disabling a protection plan does not affect protection jobs that are running.

In the row of the target instance, choose More > Disable Protection.

Allow Restoration

Scenario

If you want to use the selected instance as the target instance for restoration, you can use Allow Restoration.

Note

If restoration is disabled for the project to which the instance belongs, do not perform this operation.

In the row of the target instance, choose More > Allow Restoration or More > Disallow Restoration.

Disallow Restoration

Scenario

If you want to disallow the selected instance to be used as the target instance for restoration, you can use Disallow Restoration.

Note

If restoration is enabled for the project to which the instance belongs, do not perform this operation.

In the row of the target instance, choose More > Disallow Restoration.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note:

  • For details about how to restore data, see Restoration.
  • For details about protection operations, see Step 7: Performing Backup.
