步骤7：创建复制SLA

如果您需要对本地备份副本进行复制，请参考本节配置复制策略的SLA。

前提条件

已创建备份策略的SLA。具体操作请参考步骤5：创建备份SLA。

操作步骤

  1. 选择“保护 > 保护策略 > SLA”。
  2. 查找到需要创建复制策略的SLA，单击“更多 > 修改”。

    系统预置的SLA不支持修改。

  3. 单击“复制策略”图标。
  4. 配置复制策略信息，完成后单击“确定”。

    单击页面右侧的“添加”，可创建多个复制策略，最多允许创建4个。

    相关参数说明如表1。

    表1 复制策略参数说明

    参数

    说明

    名称

    用户自定义复制策略名称。

    复制模式
    支持的复制模式。
    • 跨域：集群之间进行复制。
    • 域内：同一集群多个节点之间进行复制。
    说明：

    仅集群高可用场景下集群内存在多个节点时，才支持选择“域内”。

    复制目标集群

    复制到的目标集群。

    选择已添加的目标集群，或单击“添加”，添加新的集群。

    说明：

    复制模式为“跨域”时，才支持设置该参数。

    复制目标端用户

    说明：

    仅1.6.0及后续版本支持设置该参数。

    选择复制目标端的系统管理员或数据保护管理员用户。

    密码

    说明：

    仅1.6.0及后续版本支持设置该参数。

    填写复制目标端的系统管理员或数据保护管理员用户密码。输入密码后，单击“认证”，认证密码的正确性。
    本地存储类型
    说明：

    仅1.6.0及后续版本支持设置该参数。

    选择本地存储的类型。固定为“OceanProtect”。

    指定目标位置
    可选择复制的目标位置为备份存储单元或备份存储单元组。当选择备份存储单元组，对备份副本进行首次复制时，系统将从备份存储单元组中选择其中一个备份存储单元，作为复制的目标位置，后续复制时优先选择首次复制使用的备份存储单元。
    说明：
    • 对于1.5.0版本，复制模式为“域内”时，才支持设置该参数。
    • 当备份策略的目标位置指定备份存储单元时，复制策略的目标位置不能为同一备份存储单元。

    频率
    多长时间执行一次复制。
    • 备份完成后立即进行复制。
    • 每xx分钟/小时/天/周执行一次复制。

    复制规则&保留规则

    选择需要复制的备份副本并设置复制副本的保留时长。

    一旦超过设定的保留时间，系统将自动删除过期的副本。副本保留时长必须大于复制频率。
    • 复制所有副本：指定副本开始复制的时间并设置副本保留时长。
    • 复制指定日期的副本：指定日期并设置副本保留时长，仅支持复制指定日期的全量和永久增量（合成全量）备份副本。以下选项同时可选择多项：
      • 选择每年的x月产生的全量备份副本进行复制，复制副本保留x年（支持保留1~70年）。
      • 选择每月的第一个或最后一个全量备份副本进行复制，复制副本保留x个月（支持保留1~840个月）。
      • 选择每周的第x天产生的全量备份副本进行复制，复制副本保留x个周（支持保留1~3650个周）。
      说明：
      • 对于不支持永久增量（合成全量）备份的应用，仅支持复制指定日期的全量副本。
      • 如果同时选择了多个选项，待复制副本的时间范围为多个选项的并集，生成的复制副本的保留时长为多个选项中保留时间最久的保留时长。
      • 对于1.6.0及后续版本，选择每月的最后一个全量备份副本复制时，当月待复制的全量备份副本的复制任务将在下一个月执行。

    首次执行时间

    首次执行复制任务的开始时间。

    如果首次执行时间晚于当前时间，则按照首次执行时间开始复制；如果首次执行时间早于当前时间，则等待下次复制周期到达后进行复制。
    说明：
    • 请确保代理主机、本地集群和目标集群的设备时间一致。
    • 如果规划了Air Gap，请确保首次执行时间在目标集群Air Gap复制时间窗内，并确保每次复制任务开始时间均在Air Gap复制时间窗内。如果复制任务在Air Gap复制时间窗内未结束，且Air Gap策略启用了“强制中止复制”，复制任务将会被强制中止。如果确实需要执行完成该复制任务，请暂时关闭目标集群Air Gap策略的“强制中止复制”开关。

    限速策略

    复制任务平均每秒最大传输的数据量。

    选择已创建的限速策略，创建限速策略的具体操作请参考步骤3：创建限速策略章节。
    链路重删
    说明：

    对于1.5.0版本，该参数名为“数据重删”。
    开启后，远程复制过程中系统会对副本数据进行重删以提升复制效率。建议开启数据重删功能。
    说明：
    • 请确保备份策略已打开目标端重删开关，否则将导致数据重删功能不生效。
    • 对于1.6.0及后续版本，当“本地存储类型”为“OceanProtect”或“OceanStor Pacific”时，才支持配置此参数。
    链路压缩
    说明：

    对于1.5.0版本，该参数名为“数据压缩”。

    开启后，远程复制过程中系统会对副本数据进行压缩以提升复制效率。建议开启数据压缩功能。
    WORM设置
    说明：

    仅1.5.0版本存在此参数。

    开启后，生成的副本为WORM副本。WORM副本不支持删除和缩短保留时间。

    任务失败告警

    开启后，任务执行失败后自动发送告警，下次任务成功后告警自动清除。

  5. 单击“确定”，根据界面提示完成确认。
  6. 关联资源到复制SLA，以对副本进行周期性复制。

    • 如果已有资源关联至该SLA策略，则系统将按照SLA策略的定义对资源进行保护。
    • 如果该SLA策略未关联资源，则请关联资源至该SLA策略以对其进行保护。具体操作请参考步骤6：执行备份。
      • 执行复制任务时需确保本地集群和目标集群的存储池已使用的容量不能超过License中的“数据备份容量”，否则复制任务将失败。参考以下步骤确认容量是否充足。
        1. 选择“系统 > 基础设施 > 集群管理”。
        2. 在“备份集群”页签的“本地集群节点”区域，单击节点名称。
        3. 在弹出的“节点详情”对话框中，单击“打开设备管理”，进入DeviceManager。
        4. 选择“系统 > 存储池”，查看存储池已使用的容量。
        5. 选择“设置 > License管理”，查看“数据备份容量（Data Backup Capacity）”。

          存储池已用容量不能超过“数据备份容量（Data Backup Capacity）”。

      • 复制任务产生的复制副本，仅支持在目标集群进行查看、恢复等，本地集群无法查看。
