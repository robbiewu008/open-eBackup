Step 1: Enabling TDSQL Database Permission

Before executing a TDSQL backup job, you need to enable the permission of the corresponding TDSQL version. Otherwise, the backup or restoration will fail.

Procedure

  1. Log in to the TDSQL CHITU management console, select an instance in Instance management, choose Database management > User Management, and click Create User to create a user for backup.
  2. Log in to the TDSQL database as an administrator (for example, tdsqlsys_agent).
  3. Run the following command to locate the primary node. If the command output contains Empty, the node is the primary node.

    show slave status

  4. Log in to the primary node and run the following commands to enable the permission of the corresponding version for the user.

    GRANT BACKUP_ADMIN, PROCESS, RELOAD, SELECT, LOCK TABLES, SUPER ON *.* TO 'Username created in 1'@%';
