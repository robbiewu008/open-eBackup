Configuring Management Data Backup

Before performing an upgrade or adjusting key data, you can back up management data on the OceanCyber WebUI. The system then can use the backup data to restore configuration data (such as user, SLA, resource, and alarm notification data) in a timely manner if the data is deleted due to misoperations.

Precautions

  • If a job is being executed on the OceanCyber WebUI, you are advised not to initiate management data backup.
  • If a management data backup or restoration job is being executed, you are not allowed to execute the job again. If a manual task is being executed, the scheduled management data backup task will be canceled.
  • Backup can be performed only after a backup policy is configured.

Procedure

  1. Choose System > Settings > Management Data Backup.
  2. Configure the management data backup policy.

    In the Management Data Backup area, click Backup Policy. On the Backup Policy page, set the backup policy information. Table 1 describes the parameters. After the configuration is complete, click OK.
    Table 1 Backup policy parameters

    Parameter

    Description

    Backup Time

    Time for backing up data by day.

    If a manual backup job is being executed at the specified time, the automatic backup job will be automatically canceled.

    Retained Backups

    Number of latest available management data backup.

    The value must range from 3 to 20.

    Default value: 7.

    SFTP IP Address

    Information about the SFTP server for storing backup files.

    Ensure that the SFTP user has the permission to read and write the /var directory.

    After the SFTP IP address is changed, the backup records of the original SFTP server become invalid.

    Default SFTP port: 22.

    SFTP User Name

    SFTP Password

    SFTP Port

    Backup Path

    Path for management data backup. After a backup path is set, the path is automatically generated on the SFTP server. If this parameter is not set, the default value is /var/sysbackup.

    NOTE:

    Ensure that the SFTP user has the execute permission on the backup path.

    Data Encryption Password

    Enter a password. The password is used to encrypt the system management data and ensure backup data security. This password is also used for encryption in manual backup. Keep this password secure. When the system configurations need to be restored after a system fault, the password needs to be used to decrypt the backup system management data.

    Confirm Data Encryption Password

    Enter the data encryption password again.

  3. Click Manual Backup to manually back up the management data immediately.

    During manual backup, the system also uses the Data Encryption Password set in the backup policy to encrypt and store management data. When the system recovers from a fault, the data encryption password is also required for decryption.

  4. On the Manual Backup page, enter the description information and click OK.
