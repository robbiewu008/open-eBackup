Viewing a File System Copy

This section describes how to view details about a file system copy.

Procedure

  1. Choose Explore > Copy Data > Local File System.
  2. Choose a resource viewing mode in the upper right corner of the page to view the copy information.

    • Resources: View the copy information of file systems by resource. If data in a file system is damaged or lost and the file system needs to be restored, you are advised to search for the copy of the corresponding point in time from this entry. Table 1 describes the parameters.
      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Name

      Name of a file system.

      Location

      Location of a file system on the storage.

      Status

      Indicates whether a file system is discovered by ProtectManager.

      • If yes, the status is Existing.
      • If no, the status is Not existing.

      Copy

      Quantity

      Number of copies generated for a file system.
    • Copy: View information about all copies by copy. To restore a file system to the original location or a new location by using a specified copy, you are advised to search for the copy from this entry. Table 2 describes the parameters.
      Table 2 Copy parameters

      Parameter

      Description

      Copy

      Copy Time

      Time when a copy is generated.

      Click Copy Time to view the expiration time and other information of the copy. If -- is displayed for the expiration time, the copy is retained permanently.

      Status

      Copy status.

      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being restored.

      Location

      Name of the bucket of the backup storage for storing copies.

      Generation Mode

      Mode in which a copy is generated. In this section, the value is Backup.

      Copy Type

      Backup type of a generated copy.

      The value can be Full Backup or Incremental Backup.

      Index

      Indexing status.

      • Not indexed: The copy is not indexed.
      • Indexed: The copy has been indexed.
      • Indexing: The copy is being indexed.
      • Deleting: The copy is being deleted.
      • Deletion failed: The copy fails to be deleted.

      Resources

      Name

      Name of a file system.

      Location

      Location of a file system on the storage.

      Status

      Indicates whether a file system is discovered by ProtectManager.

      • If yes, the status is Existing.
      • If no, the status is Not existing.
