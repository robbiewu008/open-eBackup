Managing the GaussDB (DWS) Cluster

On the product, you can modify or delete registered GaussDB (DWS) clusters and schema sets or table sets in the clusters.

Related Operations

Log in to the management page, choose Protection > Big Data > GaussDB (DWS), click the Cluster, Schema Sets, or Table Sets tab, and find the cluster on which you want to perform related operations.

Table 1 describes the related operations.

Table 1 Operations related to the GaussDB (DWS) cluster

Operation

Description

Navigation Path

Modifying cluster information

Scenario

This operation allows you to modify the username, environment variable file path, node, and agent host of a cluster.

Note

  • The cluster name cannot be modified.
  • If the cluster is associated with a schema or table object, only the node and agent host can be modified.
  • If the cluster is not running or is abnormal during the modification, the cluster can be modified, but the SLA policy cannot be added.

In the row of the target cluster, choose More > Modify.

For details about GaussDB (DWS) cluster parameters, see Step 1: Registering a GaussDB (DWS) Cluster.

Removing protection

Scenario

This operation enables you to remove protection for databases or table sets in a GaussDB (DWS) cluster when the protection is no longer required. After the removal, the system cannot execute protection jobs for the GaussDB (DWS) cluster. To protect the backup set again, associate an SLA with it again.

Note

When the protection job of the database is running, protection cannot be removed.

In the row of the target cluster, choose More > Remove Protection.

Deleting cluster information

Scenario

This operation enables you to delete a GaussDB (DWS) cluster if you no longer need to protect databases or table sets in it or restore data to it.

Note

  • A cluster cannot be deleted if a backup or restoration job is being executed.
  • A cluster cannot be deleted if the cluster or a database, schema set, or table set in the cluster has been associated with an SLA.
  • Deletion of a cluster will delete all its databases, schemas, and tables.

In the row of the target cluster, choose More > Delete.

Scanning resources

Scenario

The OceanProtect Appliance automatically scans the GaussDB (DWS) environment every 5 minutes. If the registered GaussDB (DWS) changes, the change is automatically updated to the OceanProtect Appliance after scanning. You can perform this operation to immediately update the changed information to the OceanProtect Appliance.

In the row of the target cluster, choose More > Resource Scan.

Testing connectivity

Scenario

Manual connectivity test of the registered GaussDB (DWS) environment is used to check the connectivity between the OceanProtect Appliance and the agent host and between the OceanProtect Appliance and the production environment.

In the row of the target cluster, choose More > Test Connectivity.

Modifying a schema set or table set

Scenario

This operation enables you to modify a schema set or table set.

In the row where the schema set or table set resides, choose More > Modify.

Deleting a schema set or table set

Scenario

This operation enables you to delete a schema set or table set if you no longer need to protect it.

Note

A schema set or table set associated with an SLA cannot be deleted.

In the row where the schema set or table set resides, choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
