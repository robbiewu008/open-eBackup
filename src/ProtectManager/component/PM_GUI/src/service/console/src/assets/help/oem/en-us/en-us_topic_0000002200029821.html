Managing Security Policies

System security policies include Password Policy, Weak Password Dictionary, Session Timeout Duration, Login Policy, Key Update Policy, Access Control, and Internal Component Password Update. Proper settings of system security policies improve system security.

Procedure

  1. Choose System > Security > Security Policies.
  2. View and modify security policies.

    • Password Policy
      1. In the Password Policy area of the Security Policies tab page, click Modify on the right.
      2. Modify the password policy.
        Table 1 lists the parameters.
        Table 1 Password policy parameters

        Parameter

        Description

        Minimum Length

        Minimum length of a password. This parameter aims to ensure that excessively long or short passwords are not set.

        [Value range]

        The value contains 8, 10, or 14 characters.

        Complexity

        Complexity of a password. This parameter aims to ensure that simple passwords are not set.

        [Value range]

        • A password must contain special characters, uppercase letters, lowercase letters, and digits.
        • A password must contain special characters and at least two of the following types: uppercase letters, lowercase letters, and digits.

        Special characters include !"#$%&'()*+,-./:;<=>?@[\]^`{_|}~ and spaces.

        Max. Historical Passwords Retained

        Maximum number of historical passwords that a user can keep. The value 0 indicates that the number of historical passwords that a user can keep is not limited. The new password cannot be the same as any of the historical passwords.

        [Value range]

        The value is an integer from 0 to 12.

        Max. Retention Duration of Historical Passwords

        Maximum number of days that a user's historical passwords can be retained. The value 0 indicates that there is no restriction on the number of days that a user's historical passwords can be retained. The new password cannot be the same as any of the historical passwords.

        [Value range]

        0 to 365 days.

        Validity Period&Change Interval

        You can enable or disable Validity Period&Change Interval.

        • : Enabled
        • : Disabled

        The default status is .

        NOTE:
        • If this parameter is disabled, a password never expires. To ensure system security, you are advised to enable Validity Period&Change Interval.
        • If Validity Period & Change Interval is enabled, you need to set Validity Period and Change Interval.
        • For 1.6.0 and later versions: The parameter setting takes effect only for human-machine accounts, and it is permanently effective for machine-machine accounts by default.

        Validity Period

        After the validity period of a password expires, the system prompts you to change the password. Once prompted, change the password in a timely manner.

        [Value range]

        60 to 360 days.

        Change Interval

        Minimum period of time before a password must be changed.

        [Value range]

        1 to 9999 minutes.
      3. Click Save.
    • Weak Password Dictionary

      Weak password verification applies to three scenarios: creating a user, changing a password, and resetting a user password.

      The default weak password dictionary is Admin@123.
      1. In the Weak Password Dictionary area on the Security Policies tab page, click Add.
      2. Enter a weak password in the text box.

        A weak password can be easily cracked. The value contains 8 to 64 characters.

      3. Click Save.
    • Session Timeout Duration
      1. In the Session Timeout Duration area of the Security Policies tab page, click Modify on the right.
      2. Change the session timeout duration.

        Session Timeout Duration: If a login user does not perform operations within the specified period of time and then tries to perform any operation, the system displays a timeout message. In this case, click OK to go to the login page. The value ranges from 1 to 100 in minutes.

      3. Click Save.
    • Login Policy
      1. In the Login Policy area of the Security Policies tab page, click Modify on the right.
      2. Modify the login policy.
        Table 2 lists the parameters.
        Table 2 Login policy parameters

        Parameter

        Description

        Consecutive Incorrect Inputs

        Maximum number of consecutive login attempts allowed

        NOTE:
        • After the maximum number of consecutive login attempts allowed, the user is automatically locked out.
        • In this case, you can manually unlock the user as the default user sysadmin or another user who has the permission to unlock a user. Alternatively, you can wait till the user is automatically unlocked after the lockout duration.
        • When you reset the password or unlock the user, if the number of incorrect password attempts exceeds the preset value, the reset or unlock operation fails.

        [Value range]

        The value is an integer from 1 to 9.

        Lockout Duration

        Duration for which a user is automatically locked after the number of consecutive incorrect login attempts reaches the value of Consecutive Incorrect Inputs.

        NOTE:

        The user will be automatically unlocked after the lockout duration. The lockout duration applies only when a user is automatically locked out. If a user is manually locked out, the user can only be unlocked manually.

        [Value range]

        3 to 2000 minutes
      3. Click Save.
    • Key Update Policy
      1. In the Key Update Policy area of the Security Policies tab page, click Modify on the right.
      2. Modify the key update interval.

        Key update interval: interval at which the key is automatically updated. The value ranges from 30 to 180 days.

      3. Click Save.
      4. (Optional) To update the key immediately, click Update Now on the right. In the dialog box that is displayed, click OK.
    • Access control: After this function is enabled, only IP addresses in the IP address whitelist can access the system.
      1. In the Access Control area of the Security Policies tab page, click Modify on the right.
      2. Click Add and enter an accessible IP address or IP address segment in the text box.

        IP address or IP address segment: IP address whitelist, that is, IP addresses that can be used to access the system. Example: 192.168.1.100, fc00::1234, or 192.168.1.10-192.168.1.11.

      3. Click Save, and then click Save on the right of Access Control.
      4. Click on the right. In the dialog box that is displayed, read the warning message carefully, select I have read and understand the consequences associated with performing this operation., and click OK to enable access control.

        If you do not need to verify the IP address whitelist, click on the right. In the dialog box that is displayed, select I have read and understand the consequences associated with performing this operation., and click OK to disable access control.

        Access control can be enabled only when at least one IP address or IP address segment exists.

    • Internal Component Password Update
      You can update the passwords of database super administrators, database common users, in-memory database component users, internal message queue component users, CIFS local authentication users, and DataTurbo users.

      After the password is updated, the system restarts and ongoing data protection jobs fail.

      1. In the Internal Component Password Update area on the Security Policies tab page, click Update on the right.
      2. The Danger dialog box is displayed. Confirm the content in the dialog box, select I have read and understand the consequences associated with performing this operation., and click OK.
