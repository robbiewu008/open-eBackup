Viewing the OceanBase Environment

After the OceanBase resource is registered with the OceanProtect, you can view details about OceanBase clusters on the management page.

Procedure

  1. Choose Protection > Databases > OceanBase.
  2. On the Cluster tab page, view the cluster information.

    Information of registered clusters is displayed on the page.

    Table 1 describes the detailed information.
    Table 1 Cluster information

    Parameter

    Description

    Name

    Cluster name.

    Click the cluster name to view information about the OBClient and OBServer hosts and the tenant and copy data information of the cluster.

    Status
    Cluster status.
    • If all hosts in the cluster are online and the OceanBase environment is normal, the cluster status is Online.
    • If any host in the cluster is offline, the cluster status is Offline.

    Version

    Cluster version.

    SLAs

    If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether the instance is protected.

    • Unprotected: The instance is not associated with an SLA, or the instance is associated with an SLA but protection is disabled.
    • Protected: The instance is associated with an SLA and protection is activated.
    • Creating: The instance is associated with an SLA and protection is being created.

    Owner

    Data protection administrator of the cluster.

    If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.

  3. On the Tenant Sets tab page, view the tenant set information.

    The registered tenant sets are displayed on the page.
    1. Table 2 describes the tenant set information.
      Table 2 Tenant set information

      Parameter

      Description

      Name

      Tenant set name.

      Status

      Status of the tenant set in the cluster. The value can be Online, Offline, or Partially Online.

      Owning Cluster

      Cluster to which the tenant set belongs.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the instance is protected.

      • Unprotected: The instance is not associated with an SLA, or the instance is associated with an SLA but protection is disabled.
      • Protected: The instance is associated with an SLA and protection is activated.
      • Creating: The instance is associated with an SLA and protection is being created.
    2. Click the name and check the overview, copy information, and jobs.
      • Overview

        Displays the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, a copy at that time exists.

      • Job

        Ongoing and historical jobs are displayed.
