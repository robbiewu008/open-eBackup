Fileset Copies

This section describes how to perform ransomware detection on fileset copies.

Prerequisites

An anti-ransomware policy has been created for the resource.

Precautions

If the storage device is upgraded from 1.2.0 through a fast upgrade, upgrade the kernel version of the storage device to the current version of the OceanProtect Backup Storage by following instructions in the OceanProtect Backup Storage X.X.X Upgrade Guide. X.X.X indicates the product version.

Performing Detection

In addition to detecting copies based on scheduled ransomware protection policies, you can also manually detect all copies of individual resources. To perform manual detection, perform the following steps:

  1. Choose Explore > Ransomware Protection.
  2. Click Filesets.
  3. Locate the row of the target resource and choose More > Manually Detect.

    In the displayed dialog box, select the copies to be detected and click OK.

    Alternatively, you can locate the row of the target resource and click the number of copies. In the displayed window on the right, choose More > Manually Detect on the right of the target copy.

Viewing the Detection Report

  1. Choose Explore > Ransomware Protection.
  2. Click Filesets.
  3. Locate the row that contains the target resource and click Copy.
  4. In the displayed window, choose More > Detection Report on the right of a copy. The detection report displays details.
  5. (Optional) Handle false detection.

    If an uninfected copy is detected as Infected, you can correct the copy status through false detection handling.

    1. Locate the row that contains the target resource and choose More > Handle False Detection in the Operation column on the right.
    2. In the displayed dialog box, confirm the information and click OK. The system changes the copy state from Infected (incorrect detection) to Uninfected.

    False detection handling is supported only for copies in the Infected state.
