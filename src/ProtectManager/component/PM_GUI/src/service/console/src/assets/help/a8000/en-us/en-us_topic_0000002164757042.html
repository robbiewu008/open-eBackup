Oracle Database Backup Fails with an Error (RMAN-06062) Reported

Symptom

The Oracle database backup job fails, with the following error message reported: "RMAN-06062: can not backup SPFILE because the instance was not started with SPFILE".

Possible Causes

Starting the Oracle database with a pfile causes the backup job failure. Currently, OceanProtect supports backup and restoration jobs only for databases that are started with an spfile.

Troubleshooting

  1. Log in to the Oracle database host.

    • If the Linux OS is used, use PuTTY to log in to the Oracle database host and run the su - oracle command to switch to user oracle.
    • If the Windows OS is used, log in to the Oracle database host as the system administrator.

  2. Run the following commands to log in to the Oracle database instance.

    • OS authentication:
      • If the Linux OS is used, run the following commands:

        export ORACLE_SID=Database instance name

        sqlplus / as sysdba

      • If the Windows OS is used, perform the following operations:
        1. Press Win+R to open the Run window.
        2. Enter cmd and run the following commands:

          set ORACLE_SID=Database instance name

          sqlplus / as sysdba

    • Database authentication:
      • If the Linux OS is used, run the following commands:

        export ORACLE_SID=Database instance name

        sqlplus

        Enter the name of a user with the sysdba permissions as prompted:

        username as sysdba

        Enter the password as prompted.

      • If the Windows OS is used, perform the following operations:
        1. Press Win+R to open the Run window.
        2. Enter cmd and run the following commands:

          set ORACLE_SID=Database instance name

          sqlplus

          Enter the name of a user with the sysdba permissions as prompted:

          username as sysdba

          Enter the password as prompted.

  3. Run the show parameter spfile command.

    • If the command output is empty, the database is started with a pfile.
    • If the command output is not empty, the database is started with an spfile.

  4. Create an spfile from the pfile.

    • For Linux OSs:
      create spfile='$ORACLE_HOME/dbs/spfile.ora' from pfile
    • For Windows OSs:
      create spfile='$ORACLE_HOME\dbs\spfile.ora' from pfile

  5. Enable the spfile.

    • For Linux OSs:
      alter system set spfile='$ORACLE_HOME/dbs/spfile.ora' scope=memory;
    • For Windows OSs:
      alter system set spfile='$ORACLE_HOME\dbs\spfile.ora' scope=memory;

  6. Perform 3 to check whether the modification takes effect.

    After the preceding steps are performed, the modification takes effect. Afterwards, the spfile is first used to restart the Oracle database.
