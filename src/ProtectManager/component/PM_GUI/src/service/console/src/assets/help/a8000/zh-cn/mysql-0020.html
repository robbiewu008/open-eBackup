步骤9：执行备份

执行备份前将需要保护的资源关联到指定SLA策略，系统将对这些资源按照SLA策略进行保护，并根据SLA策略周期性执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

背景信息

手动备份产生的副本保留时长按照SLA中的定义进行保留。

前提条件

受保护的资源已关联SLA。

操作步骤

  1. 选择“保护 > 数据库 > MySQL/MariaDB/GreatSQL”。
  2. 在“数据库”页签，选择需要保护的数据库，单击“保护”。

    MySQL/MariaDB/GreatSQL的默认数据库名称在“数据库”页面不显示。

    您也可以勾选多个数据库进行批量保护。

  3. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

    如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  4. 参考表1，在“高级配置”中配置参数。

    表1 备份高级参数说明

    参数

    说明

    备份前执行脚本

    您可以根据实际需求，配置“备份前执行脚本”、“备份成功执行脚本”和“备份失败执行脚本”，当备份任务事件中出现“开始执行前置任务”提示后，配置的脚本会在对应场景下执行。

    请输入脚本名称，脚本名称以.sh结尾。确保脚本已存放在ProtectAgent安装目录下的“DataBackup/ProtectClient/ProtectClient-E/sbin/thirdparty”目录，且登录数据库主机的用户（默认为root）对该脚本有可执行权限。

    说明：

    当配置了“备份成功执行脚本”时，即使该脚本执行失败，管理界面上也会显示备份任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。

    备份成功执行脚本

    备份失败执行脚本

  5. 单击“确定”。

    如果当前系统时间已晚于该SLA首次备份的开始时间，则关联备份SLA后可以选择在当前立即执行备份或者根据SLA中设置的备份策略等待下次调度。

  6. 可选：执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。

    1. 在需要备份的数据库所在行单击“更多 > 手动备份”。
    2. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    3. 选择保护策略：全量备份、增量备份、差异备份或日志备份。

      对于MySQL/MariaDB而言，集群类型为“主主复制”，则仅支持全量备份、增量备份和差异备份。

      对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    4. 单击“确定”。
