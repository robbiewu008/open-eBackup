Users Cannot Be Created on the Primary Domain Controller After System State Restoration

Symptom

The message "Cannot create the object because directory service was unable to allocate a relative identifier" is displayed when a user is created.

Possible Causes

  • When system state restoration is performed for the Relative Identifier (RID) master, it attempts to synchronize with other domain controllers. If there is no available domain controller for synchronization or the synchronization does not work, the synchronization process will fail.
  • The RID pool is used up or the RID allocation object is faulty.

Procedure

For details, see Event ID 16650: The account-identifier allocator failed to initialize in Windows Server.
