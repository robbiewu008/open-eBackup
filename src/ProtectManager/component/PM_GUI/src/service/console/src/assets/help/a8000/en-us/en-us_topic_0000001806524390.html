Performing Periodic Backup

Associate the mailbox to be protected with a specified SLA. The system backs up these resources periodically based on the SLA.

Prerequisites

Before the backup, ensure that the Exchange Server service and related dependent services are running properly. For details about the corresponding services, see Overview of Exchange services on Exchange servers.

Context

For mailbox backup, a registered user must have the ApplicationImpersonation role permissions. When the mailbox is backed up for the first time, the agent host automatically creates a user group named Databackup, which has the ApplicationImpersonation role permissions, and adds the registered user to the user group.

Procedure

  1. Choose Protection > Applications > Exchange.
  1. On the Email tab page, select the mailbox that you want to protect and choose More > Protect.

    You can choose multiple mailboxes for batch protection.

  2. Select a created SLA.

    You can also click Create to create an SLA.

  3. Select whether to continue to protect emails or folders after they fail to be backed up.
  4. Click OK.

    If the current system time is later than the start time of the first backup set in the SLA, you can choose to perform the backup immediately or wait for the next scheduling based on the backup policy set in the SLA after the backup SLA is associated with the resource.
