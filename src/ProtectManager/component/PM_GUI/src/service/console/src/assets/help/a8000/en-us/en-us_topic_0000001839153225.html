Manual Backup

You can manually run a backup job to back up data immediately.

Context

  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected for a manual backup job is not defined in the SLA, the generated backup copy is retained permanently. For example, if incremental backup is not defined in the SLA but is selected for the manual backup job, the copies generated by the manual incremental backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Big Data > Elasticsearch.
  2. Click the Backup Sets tab page.
  3. Choose More > Manual Backup in the row where the backup set you want to back up is located.

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

  4. Set the name of the copy generated through manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  5. Select a protection policy, which can be Full backup or Forever Incremental (Synthetic Full) Backup.
  6. Click OK.
