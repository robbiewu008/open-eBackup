执行实例级备份

操作步骤

  1. 选择“保护 > 数据库 > SQL Server”。
  2. 在“实例”页签，选择需要保护的实例，单击“保护”。

    您也可以勾选多个实例进行批量保护。

  3. 选择用户自行创建的SLA。

    您也可以单击“创建”，创建新的SLA。

    • SQL Server不支持使用系统预置的SLA（Gold、Silver、Bronze）。
    • 如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  4. 参考表1，在“高级配置”中配置参数。

    表1 备份高级参数说明

    参数

    说明

    备份前执行脚本

    您可以根据实际需求，配置“备份前执行脚本”、“备份成功执行脚本”和“备份失败执行脚本”，当备份任务事件中出现“开始执行前置任务”提示后，配置的脚本会在对应场景下执行。

    请输入脚本名称，脚本名称以.bat结尾。请确保脚本已存放在安装目录\DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty，且登录数据库主机的用户对该脚本有可执行权限。

    说明：

    当配置了“备份成功执行脚本”时，即使该脚本执行失败，管理界面上也会显示备份任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。

  5. 单击“确定”。

    如果当前系统时间已晚于SLA首次备份的开始时间，可以在弹出的对话框中选择立即执行手动备份或者根据SLA中设置的备份策略周期性自动执行备份。

  6. 执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。

    1. 在需要备份的数据库所在行单击“更多 > 手动备份”。

      支持选择多个资源批量进行手动备份。勾选多个资源，在页面左上角选择“更多 > 手动备份”。

    2. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    3. 选择保护策略：全量备份、差异备份或日志备份。

      对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    4. 单击“确定”。
