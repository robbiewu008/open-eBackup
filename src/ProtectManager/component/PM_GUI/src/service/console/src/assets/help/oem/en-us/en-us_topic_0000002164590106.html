Viewing Information

After a Kubernetes cluster is registered with the product, you can view information about the clusters, namespaces, and datasets.

Procedure

  1. Choose Protection > Containers > Kubernetes CSI.
  2. On the Cluster tab page, view the cluster information.

    1. Table 1 describes details about clusters.
      Table 1 Cluster information

      Parameter

      Description

      Name

      Cluster name.

      Version

      Kubernetes cluster version number.

      Status
      Cluster status, which can be Online or Offline.
      • Online: When the product is properly connected to the Kubernetes cluster, Status is Online.
      • Offline: When the connection between the product and Kubernetes cluster is abnormal, Status is Offline.

      IP Address

      Management IP address of the cluster.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of a cluster.

      If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If not authorized, -- is displayed.
    1. Click the name of a cluster to view its basic information and namespace information.
      • Basic cluster information

        The cluster name and IP address are displayed.

      • Namespace information

        The names, SLAs, SLA compliance, and protection status of namespaces are displayed.

  3. On the Namespace tab page, view the namespace information.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    1. View details about the namespaces, as described in Table 2.
      Table 2 Namespace information

      Parameter

      Description

      Name

      Namespace name.

      Owning Cluster

      Cluster to which the namespace belongs.

      Cluster IP Address

      Management IP address of the cluster to which the namespace belongs.

      SLAs

      If an SLA is associated with the namespace, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status
      Whether a namespace is protected.
      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name of a namespace to view the namespace overview, copy data, and job information.
      • Overview

        The namespace name, SLA information, job execution time, total number of copies, and dataset list are displayed.

        The dataset list contains Name, Owning Namespace, SLAs, SLA Compliance, and Protection Status.

      • Copy Data

        You can search for copies by year, month, or day.

        If is displayed below the time, copies exist at the time. Click a date to go to the copy data page.

      • Jobs

        All jobs are displayed, including backup and restoration jobs. You can choose to display ongoing or historical jobs.

  4. On the Dataset tab page, view table information.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    1. For details about the dataset, see Table 3.
      Table 3 Dataset information

      Parameter

      Description

      Name

      Dataset name.

      Owning Namespace

      Name of the namespace where a dataset resides.

      Owning Cluster

      Name of the cluster where a dataset resides.

      SLAs

      If an SLA is associated with the resource, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status
      Whether the current resource is protected.
      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click a name to view the dataset overview, copy data, and job information.
      • Overview

        The name, SLA information, job execution time, total number of copies, and volume information are displayed.

      • Copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, copies exist at the time. Click a date to go to the copy data page.

      • Job

        All jobs, including backup and restoration jobs, are displayed.
