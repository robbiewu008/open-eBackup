Querying Information About the CNware Virtualization Environment

After the CNware virtualization environment is registered with the OceanProtect, you can view details about VMs, hosts, or clusters on the management page.

Procedure

  1. Choose Protection > Virtualization > CNware.
  2. On the Clusters tab page, view cluster information.

    Table 1 describes the cluster information.
    Table 1 Cluster information

    Parameter

    Description

    Name

    Cluster name.

    Location

    Path to a cluster.

    SLAs

    If the entire cluster is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    Protection Status

    Indicates whether a cluster is protected.

    • Unprotected: The cluster is not associated with an SLA, or the cluster is associated with an SLA but protection is disabled.
    • Protected: The cluster is associated with an SLA and protection is enabled.
    • Creating: The cluster is associated with an SLA and protection is being created.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

  3. On the Hosts tab page, view host information.

    Table 2 describes the host information.
    Table 2 Host information

    Parameter

    Description

    Name

    Host name.

    Click a host name. The VM list and virtual device nodes are displayed.

    If a host belongs to a cluster, the VM list is not displayed.

    The virtual device node is the number of the slot where the virtual disk is located. It is represented in the format of Disk interface type (Bus number:Slot number), for example, IDE(0:0).

    Location

    Path to a host.

    Status

    Host status, which can be Online or Offline.

    SLAs

    If an SLA is associated with a host, the SLA name is displayed. Otherwise, -- is displayed.

    Protection Status

    Whether a host is protected.

    • Unprotected: The host is not associated with an SLA, or the host is associated with an SLA but protection is disabled.
    • Protected: The host is associated with an SLA and protection is enabled.
    • Creating: The host is associated with an SLA and protection is being created.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

  4. On the VMs tab page, view VM information.

    1. Table 3 describes the VM information.
      Table 3 VM information

      Parameter

      Description

      Name

      Name of a VM.

      Location

      Path to a VM.

      Status

      VM status, which can be power-on or power-off.

      Mark

      Mark of a VM.

      If you have set a mark for a VM in the CNware virtualization environment, the VM mark is displayed. Otherwise, -- is displayed.

      You can filter VMs by mark.

      SLAs

      If an SLA is associated with a VM, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Indicates whether a VM is protected.

      • Unprotected: The VM is not associated with an SLA, or the VM is associated with an SLA but protection is disabled.
      • Protected: The VM is associated with an SLA and protection is enabled.
      • Creating: The VM is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the VM name to view the overall VM information, copy information, and related jobs.
      • VM overview

        You can view information of a VM, such as the VM name, SLA information, job execution time, total number of copies, and disk information.

      • VM copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, copies exist at that time.

      • Jobs related to a VM

        All jobs related to a VM, such as backup, restoration, archive, and live mount jobs, are displayed.

  5. On the VM Groups tab page, view VM group information.

    Table 4 describes the VM group information.
    Table 4 VM group information

    Parameter

    Description

    Name

    VM group name.

    VM Count

    Number of VMs in a VM group.

    SLAs

    If the entire cluster is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
    • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a VM group is protected.

    • Unprotected: The VM group is not associated with an SLA, or the VM group is associated with an SLA but protection is disabled.
    • Protected: The VM group is associated with an SLA and protection is activated.
    • Creating: The VM group is associated with an SLA and protection is being created.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
