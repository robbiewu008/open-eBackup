Global Search for Copies

This section describes how to perform a global search for file system copies.

Prerequisites

  • File system copies have been generated in the environment.
  • If you need to browse or search for file system copies, ensure that the automatic indexing function must be enabled for the SLA that is used for generating file system copies.

Procedure

  1. Click on the right of the navigation bar. The Global Search page is displayed.

    • Search for copies by file name.
      1. Select Copy Data in the search box.
      2. Enter the name or path of the file object (file or directory) in the copy and click . Fuzzy search is supported.

        The file objects that meet the search criteria are displayed.

    • Perform advanced search.
      1. Click Advanced Search.
      2. Set search criteria on the Advanced Search page.
        Table 1 Advanced search for copies

        Parameter

        Description

        Keyword

        Select Copy Data and enter the name or path of the file object in the copy. Fuzzy search is supported.

        File Object Type

        Select All, File, or Directory for filtering.

        Resource Name

        Enter a resource name.

        Absolute path to file object

        Enter the absolute path to the file object.

        Note: Keyword is mandatory and other parameters are optional. The logic relationship between parameters is AND. That is, the search result is filtered based on all the selected conditions.

  2. The search result is displayed on the page.

    Table 2 describes the related parameters.

    Table 2 Copy data parameters

    Parameter

    Description

    File Object

    Name

    Name of a file or directory in a copy.

    Absolute path

    Absolute path of a file or directory.

    Type

    Type of a file object.

    • : Directory
    • : File

    Modified At

    Last modification time of a file or directory in a copy when the copy is generated.

    Resources

    Name

    Name of the source resource corresponding to the copy data.

    Type

    Resource type.

    Index Copies

    -

    Number of index copies of files or directories in the copy data.

    If fine-grained restoration has been enabled for the SLA that is used to generate a copy, the system checks whether there are updates for the files or directories in the backup copy during each automatic backup. If yes, a new index is created for the files or directories. The number of index copies is 1. If no, the number of index copies is increased by 1.

    Click an index copy to view the copy data of the resource. Indexes have been created for the copies displayed on the page.

  3. In the line where the search result is located, perform file-level restoration for the file or directory.
