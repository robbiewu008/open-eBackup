Restoring Management Data

This section describes how to restore management data to the system.

Precautions

  • If you use a backup copy at a certain point in time to restore the management data, the system management data will be overwritten. Therefore, exercise caution when performing this operation.
  • Management data cannot be restored using a backup copy on one system to another system.
  • After the backup copy generated at a point in time is used to restore management data, generated after the point in time are not displayed on the GUI. Consequently, these copies cannot be for restoration.
  • After the management data is restored, historical jobs are cleared.
  • In the cluster HA scenario, if you need to restore the management data of the active node, remove the HA member by referring to "Removing an HA Member" in the Cluster HA Feature Guide and then restore the management data. After the restoration is complete, add the HA member again.
  • Before restoring management data, ensure that you can log in to the product WebUI.

Procedure

  1. In the navigation pane, choose System > Settings > Management Data Backup.
  2. In the row of the target management data backup, choose More > Restore in the Operation column.
  3. Enter the data encryption password set during backup policy configuration and click OK.
  4. Read the warning message carefully, select I have read and understand the consequences associated with performing this operation., and click OK.
