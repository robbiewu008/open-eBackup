Log Backup Failed Due to DN/CN Restart or Active/Standby Switchover

Symptom

During log backup, some operations are performed, such as DN/CN restart or active/standby switchover. As a result, log backup fails.

Possible Causes

The xlog becomes incomplete due to DN/CN restart or active/standby switchover. As a result, log backup fails.

Troubleshooting

  1. Log in to the host where the client is installed.
  2. Run the cd /opt/DataBackup/ProtectClient/ProtectClient-E/slog/Plugins/GeneralDBPlugin command to go to the log directory of the host where the client is installed. (In this step, assume that the client is installed in the /opt directory. Change the directory based on the actual installation directory).
  3. Run the following command to search for key error information:

    grep -nr "GS-53429 : The recovery time is invalidxlog has been archived"

  4. If the error information can be found in 3, the xlog is incomplete. In this case, perform a full backup again.
