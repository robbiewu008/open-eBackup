Managing Clusters/Hosts/VMs/VM Groups

The OceanProtect allows you to perform operations (such as modifying, removing, activating, or disabling protection) for clusters, hosts, VMs, or VM groups. This section uses a VM as an example to describe the related operations. The operations for a cluster, host, or VM group are the same.

Log in to the management page, choose Protection > Virtualization > Nutanix, and find the target VM.

Table 1 describes the related operations.

Table 1 Operations related to VM protection plans

Operation

Description

Navigation Path

Modifying protection

Scenario

You can modify a protection plan of a VM to associate an SLA with the VM again, add or remove a protected VM disk, or modify client information. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target VM, choose More > Modify Protection.

Removing protection

Scenario

This operation enables you to remove the protection plan for a resource when it does not need to be protected. After the removal, the system cannot execute protection jobs for the resource. To protect the resource again, you need to re-associate it with an SLA.

Note

You cannot remove protection for a VM with the ongoing protection job.

Choose More > Remove Protection.

Performing manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup.

Activating protection

Scenario

This operation enables you to activate a disabled protection plan for a resource. After the protection plan is activated, the system backs up, archives, or replicates the resource based on the protection plan.

Note

You can perform this operation only on a VM that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable the protection plan for a resource for which periodic protection is not required. After the protection plan is disabled, the system does not perform automatic backup, archiving, or replication for the resource.

Note

Disabling protection does not affect ongoing backup, archive, or replication jobs.

Choose More > Disable Protection.

Adding a tag

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.

Removing a tag

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note:

  • For details about protection operations, see Step 8: Performing Backup.
  • The operations for restoring Nutanix VMs are the same as those described in Restoration.
