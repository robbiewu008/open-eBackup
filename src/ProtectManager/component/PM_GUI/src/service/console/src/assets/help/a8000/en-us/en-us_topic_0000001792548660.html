Step 4: Downloading and Importing a Certificate

Before using the remote replication function, import the CA certificate of the target cluster to the local cluster to which remote replication is applied. Otherwise, the system cannot verify the information about the accessed device, causing security risks.

Downloading a Certificate from the Target Cluster

  1. Log in to the management page of the replication target cluster.
  2. Choose System > Security > Certificates.
  3. In the row where Type is set to Server Certificate, choose More > Download CA certificate.

    Download a CA certificate and keep it properly.

Importing the Certificate to the Local Cluster

  1. Log in to the management page of the local cluster.
  2. Choose System > Security > Certificates.
  3. Click Add External Certificate to configure external certificate information.

    Table 1 describes the related parameters.

    Table 1 External certificate parameters

    Parameter

    Description

    Name

    Name of the certificate, which can be customized.

    Type

    Choose Backup/Multi-domain/Replication Cluster.

    CA Certificate

    Click , and import the certificate downloaded in the target cluster.

  4. Click OK.

    During the import, if the system displays an error message similar to "The component (OpenAPI) has used this certificate", the local cluster already exists and has the same CA certificate as the target cluster. In this case, you do not need to import the certificate and can directly perform subsequent steps.
