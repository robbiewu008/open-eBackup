Installing OceanStor DataTurbo

If the source deduplication function needs to be enabled after the client is installed, you can refer to this section to install OceanStor DataTurbo and enable the source deduplication function.

Prerequisites

  • Ensure that the free memory of the host where the client is installed is greater than 2 GB.
  • Ensure that the OS of the host where the client is installed supports source deduplication. For details about the OSs that support source deduplication, refer to Compatibility Query Tool.

Procedure

  1. Download the client software package. For details, see Downloading a Client Software Package.
  2. Open the downloaded client software package and obtain the OceanStor DataTurbo installation package dataturbo.zip from the ..\DataProtect_XXX_client_general_XXX.zip\DataProtect_XXX_client_general_XXX\third_party_software directory.

    The name of the client software package varies depending on the version, client type, and OS.

  3. Upload the OceanStor DataTurbo installation package to the host where the client is installed.
  4. Install OceanStor DataTurbo and start the service. For details, see "Installation" and "Starting the Service" in the SourceDedupe User Guide of the corresponding version.

    • For the Linux OS, after source deduplication is enabled, add the source deduplication user to sudoers before the first backup. For details, see How Can I Add a Source Deduplication User to sudoers? (Applicable to the Linux OS).
    • For the Windows OS, after source deduplication is enabled, add the SID (S-1-5-18) of user system to the ...\oceanstor\dataturbo\conf\whitelist whitelist and run the sc stop dataturbo and sc start dataturbo commands in sequence to restart the DataTurbo service.
