扫描磁带库

系统每隔10分钟自动扫描磁带库，发现连接到OceanProtect的磁带库。如果您发现新接入的磁带库还未扫描至OceanProtect，可以通过本节操作，手动扫描磁带库，将其添加至OceanProtect。

约束与限制

  • 不支持不带机械臂的独立驱动器。
  • 支持LTO-3以上物理磁带库及虚拟磁带库。
  • 仅支持有条形码的磁带库。
  • 物理磁带库与OceanProtect之间通过FC光纤通道连接，虚拟磁带库使用iSCSI协议。
  • 磁带库的不同驱动器需要支持相同的磁带标准。
  • 不支持磁带部分空间过期。
  • 仅支持RW类型磁带，不支持WORM类型的磁带。

前提条件

  • 已有磁带库成功连接OceanProtect。
  • 对于物理磁带库，需要确保有导入导出槽位、且磁带库允许进行磁带挂载和解挂载操作。以昆腾磁带库为例，在磁带库的配置管理界面，进行如下配置。其他磁带库厂商的相应操作，请咨询厂商。
    • 选择“Library”，在右侧“Operations”区域，单击“IE Area”，根据需要选择Slots（导入导出槽位数量）。
    • 选择“Library”，在右侧“Operations”区域，单击“Settings”，确保勾选“Automatic Inventory”、“Library-Assisted Drive Unload”、“Logical Drive Serial Number Addressing”、“Logical System Addressing”参数。

操作步骤

  1. 选择“系统 > 基础设施 > 归档存储库”。
  2. 在“磁带”页签，选择某个节点，单击“磁带库”。
  3. 单击“扫描磁带库”，系统将自动识别已成功连接OceanProtect的磁带库。扫描完成后，在磁带库列表可以查看新扫描的磁带库的信息。相关参数说明如表表1所示。

    表1 磁带库参数说明

    参数

    说明

    名称

    磁带库的名称，由系统生成。命名方式为：TapeLibrary- + 磁带库所属的控制器名称 + 编号。编号按照磁带库扫描的顺序排序。

    例如，第一个被扫描到的磁带库，其所属的控制器名称为0A，该磁带库的名称为TapeLibrary-0A0。

    状态

    磁带库当前状态。

    • 在线：磁带库与OceanProtect正常连接。
    • 离线：磁带库与OceanProtect断开连接。

    序列号

    磁带库的序列号。

    厂商

    磁带库的厂商。支持虚拟磁带库的所有厂商，支持的物理磁带库厂商包含IBM和Quantum。

    产品型号

    磁带库的具体型号。

    槽位数量

    磁带库的槽位数量。

    驱动数量

    磁带库的驱动数量。

    所属控制器

    磁带库所属的控制器。
