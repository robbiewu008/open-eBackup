Creating a Periodic Backup

After resources to be protected are associated with an SLA, the system protects the resources based on the SLA.

Procedure

  1. Choose Protection > Databases > GoldenDB.
  2. On the Instance tab, select the instance to be protected and click Protect.

    You can select multiple databases.

  3. Select an SLA.

    You can also click Create to create an SLA.

  4. Set advanced backup parameters.

    Table 1 describes the related parameters.
    Table 1 Advanced backup parameters

    Parameter

    Description

    Script to Run Before Backup

    You can configure Script to Run Before Backup, Script to Run upon Backup Success, and Script to Run upon Backup Failure based on site requirements. When the message "ProtectAgent started executing the prerequisite task." is displayed during a backup job, the configured script is executed in the corresponding scenario.

    Enter the absolute path and name of the script and ensure that the user who logs in to the database host has the permission to run the script.
    • Windows OSs support .bat scripts, for example, C:\script.bat.
    • Non-Windows OSs support .sh scripts, for example, /opt/script.sh.
      NOTE:

      If Script to Run upon Backup Success is configured, the backup job status is displayed as Successful on the OceanProtect even if the script fails to be run. Check whether the job details contain information indicating that a post-processing script fails to be run. If yes, correct the script in a timely manner.

  5. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
