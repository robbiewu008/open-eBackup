Creating a File System

Before restoring a NAS share to a local location for the first time, you need to create a file system.

Procedure

  1. Set the file system name.

    By default, the name is prefixed with Restore_.

  2. Configure the sharing mode.

    Table 1 describes the related parameters.

    You can configure NFS and CIFS shares at the same time.

    Table 1 Sharing mode parameters

    Parameter

    Description

    NFS

    Share Path

    Share path for accessing the file system. The path name is the same as the file system name and cannot be changed.

    Client
    Information about the client that accesses a file system.
    • You can enter share client IP addresses, share client IP address segments, or an asterisk (*) to represent IP addresses of all share clients.
    • The IP addresses can be IPv4 addresses, IPv6 addresses, or a combination of IPv4 and IPv6 addresses.
    • You can enter multiple IP addresses or IP address segments and separate them by semicolons (;), spaces, or Enter. Example: 192.168.0.10;192.168.0.0/24;*

    CIFS

    Share Name

    Share name for accessing a file system.

    The default share name is Restore_Timestamp. You can change it as needed.

    Type

    Type of the user who accesses a CIFS share.

    • Everyone
    • Windows Local Authentication User
    • Windows Local Authentication User Group

    Users

    Username for accessing a CIFS share.

    This parameter is mandatory when Type is set to Windows Local Authentication User or Windows Local Authentication User Group.

    If the required user or user group does not exist, create one on DeviceManager.

    • Perform the following operations for X series backup appliances:
      1. Log in to the management page on another browser page.
      2. Choose System > Infrastructure > Cluster Management.
      3. On the Backup Clusters tab page, click a node name under the Local Cluster Nodes area.
      4. In the Node Details dialog box that is displayed, click Open the device management platform to go to DeviceManager.
      5. Choose Services > File Service > Authentication Users > Windows Users from the navigation pane. On the Local Authentication Users or Local Authentication User Groups tab page, create a user or user group.
    • For OceanProtect E1000, perform the following operations:
      1. Log in to DeviceManager by referring to Logging In to DeviceManager.
      2. Choose Resources > Access > Authentication User > Windows Users from the navigation pane. On the Local Authentication User or Local Authentication User Group tab page, create a user or user group.

  3. Click OK.
