Step 1: Configure the XBSA Library Path

Before backing up the Informix database, you need to specify the path and file name of the XBSA shared library.

Procedure

  1. Log in to the host where the Informix database is located.
  2. Run the su -informix command to switch to the Informix database user.
  3. Run cat /opt/informix/etc/onconfig |grep ^BAR_BSALIB_PATH to check whether an instance is configured.

    If the following information is displayed, the instance has been configured and no further action is required. Otherwise, go to the next step to configure the instance.

    Backup appliance 1.5.0:

    BAR_BSALIB_PATH /usr/openv/lib/libxbsa64iif.so

    Backup appliance 1.6.0 and later versions:

    BAR_BSALIB_PATH /Parent directory where the agent is installed/DataBackup/ProtectClient/interfaces/xbsa/lib/libxbsa64iif.so

    Replace onconfig with the actual name of the instance configuration file.

  4. Run the vi /opt/informix/etc/onconfig command to access the instance configuration file and configure the database instance.
  5. Change the value of BAR_BSALIB_PATH.

    Backup appliance 1.5.0:

    Change the value of BAR_BSALIB_PATH to /usr/openv/lib/libxbsa64iif.so.

    Example: BAR_BSALIB_PATH /usr/openv/lib/libxbsa64iif.so.

    Backup appliance 1.6.0 and later versions:

    Change the value of BAR_BSALIB_PATH to /Parent directory where the agent is installed/DataBackup/ProtectClient/interfaces/xbsa/lib/libxbsa64iif.so.

    Example: BAR_BSALIB_PATH /Parent directory where the agent is installed/DataBackup/ProtectClient/interfaces/xbsa/lib/libxbsa64iif.so
