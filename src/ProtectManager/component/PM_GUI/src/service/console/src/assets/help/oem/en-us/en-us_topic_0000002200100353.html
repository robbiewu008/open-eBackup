Failed to Execute a Hyper-V Backup Job and the Error Details Contain "Read File Failed, Error: 22"

Symptom

A Hyper-V backup job fails to be executed, and the VM to be backed up cannot be started. A message is displayed, indicating that the disk file is occupied by another process. Assume you log in to the OceanProtect WebUI, choose Protection > Clients, locate the row that contains the client, and export logs. The VirtualPlugin.log file contains "Read file failed, error: 22".

Possible Causes

The Hyper-V VM disk file to be backed up is occupied by another process of the Hyper-V host. As a result, an error is reported when the client reads the disk file during the backup process. The error message shows "22: ERROR_BAD_COMMAND The device does not recognize the command."

Troubleshooting

  1. Log in to the Hyper-V host machine to be backed up.
  2. Open the Windows task manager. On the Performance tab page, click Open Resource Monitor.
  3. Switch to the CPU tab. On the Associated Handles tab page, search for the name of the occupied disk file. Obtain the disk file name from the error information about the VM startup job.
  4. Manually stop the process that occupies the disk file.
  5. Start the VM to be backed up and check whether the VM can start up properly.

    • If yes, the fault is rectified.
    • If no, contact Huawei technical support engineers.
