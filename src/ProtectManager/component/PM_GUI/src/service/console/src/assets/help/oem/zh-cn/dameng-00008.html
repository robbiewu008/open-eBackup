数据使用章节

数据库的安装用户名及密码

数据库的安装用户名可通过以下方式查询，安装用户的密码请联系数据库管理员获取。

  1. 使用PuTTY登录Dameng数据库所在主机。
  2. 执行以下命令，查询数据库的安装用户名。
    ps -ef | grep dm.ini

    回显类似如下信息，其中dm.ini文件路径所在行的dmdba即为数据库安装用户。

    [root@dameng16543 ~]# ps -ef | grep dm.ini
root       508  8886  0 09:45 pts/2    00:00:00 grep --color=auto dm.ini
dmdba    21665     1  0 Jul16 ?        00:16:37 /dm8/bin/dmserver path=/dm8/data/DMServer/DAMENG/dm.ini -noconsole

步骤3：注册Dameng数据库

数据库侦听的端口号
  1. 使用PuTTY登录Dameng数据库所在主机。
  2. 执行以下命令，查询dm.ini配置文件所在路径。
    ps -ef | grep dm.ini
  3. 执行以下命令，查询端口号。
    cat dm.ini配置文件所在路径/dm.ini | grep PORT_NUM
    说明：

    命令中的dm.ini配置文件所在路径请根据2中查询结果替换。

    回显类似如下，其中“PORT_NUM”对应的值即为数据库侦听的端口。 
                    PORT_NUM                        = 5236                  #Port number on which the database server will listen

数据库认证方式
  1. 使用PuTTY登录Dameng数据库所在主机，并执行以下命令，切换至数据库安装用户，以dmdba为例。
    su - dmdba
  2. 进入数据库安装的bin目录，以“/dm8/bin”为例。
    cd /dm8/bin
  3. 执行以下命令，登录Dameng数据库。
    ./disql 数据库的安装用户名/数据库的安装用户的密码@LOCALHOST:数据库侦听的端口号

    示例：

    ./disql SYSDBA/SYSDBA@LOCALHOST:5236
  4. 执行以下命令，查看VALUE参数项取值。
    select name, value from v$parameter where name = 'ENABLE_LOCAL_OSAUTH';
    • 如果取值为“1”，则表示为OS认证。
    • 如果取值为“0”，则表示非OS认证。
