Restoring a Fileset

This section describes how to restore a fileset that has been backed up to the original or a new location.

Prerequisites

Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.

Procedure

  1. Choose Explore > Copy Data > File Systems > Fileset.

    For version 1.5.0, choose Explore > Copy Data > Bare Metal > Fileset.

  2. You can search for copies by fileset resource or copy. This section uses resources as an example.

    On the Resources tab page, find the resource to be restored by resource name and click the resource name.

  3. Click Copy Data and choose the year, month, and day to find the copy.

    If is displayed below a month or date, copies exist in the month or on the day.

  4. Choose More > Restore in the row where the copy is located.

    Table 1 describes the related parameters.

    Table 1 Parameters for restoring a fileset

    Parameter

    Description

    Restore To

    Fileset restoration location.

    • Original location: Restores data to the original location on the original host.
    • New location: Restores data to a new location. You need to specify the target host and path for restoration.
      NOTE:

      If restored files are not displayed in the restoration target location after restoration to a new location in Windows OS, open the folder in Windows OS and choose View > Options to open the Folder Options dialog box. Click the View tab and deselect Hide protected operating system files (Recommended).

    Location

    This parameter is displayed only when data is restored to the original location.

    Target Host

    This parameter is displayed only when data is restored to a new location. You need to select a target host.

    You can search for a host name or IP address to select the host.

    Target Path

    This parameter is displayed only when data is restored to a new location. You need to select a target location.

    After expanding the data source, you can search for a file name to select the corresponding path.

    Incremental restoration

    After this function is enabled, files added and modified during incremental backup can be restored, but files deleted during incremental backup will not be deleted.

    NOTE:

    This parameter is displayed only in 1.6.0 and later versions. Only copies of 1.6.0 and later versions can be restored.

    Overwrite Rule

    If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.

    • Replace existing files
    • Skip existing files: Files with the same names are skipped and are not replaced.
    • Only replace the files older than the restoration file: The latest file with the same name in the target path is retained.

    Script to Run Before Restoration

    You can execute a user-defined script before backup, upon backup success, or upon backup failure based on your need.

    In a non–Windows OS, place the execution script in the DataBackup/ProtectClient/ProtectClient-E/sbin/thirdparty directory, for example, the relative path of /opt/DataBackup/ProtectClient/ProtectClient-E/sbin/thirdparty/.

    The script type supported in a non–Windows OS is .sh.

    You can also place the script in any directory and enter the absolute path of executing the script.

    In Windows, place the execution script in the DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty directory, for example, the relative path of C:\DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty\.

    The script type supported in Windows is .bat.

    You can also place the script in any directory and enter the absolute path of executing the script.
    NOTE:

    If Script to Run upon Restoration Success is configured, the status of the restoration job is displayed as Succeeded on the OceanProtect even if the script fails to be executed. Check whether the job details contain a message indicating that the post-processing script fails to be executed. If yes, modify the script in a timely manner.
