Managing File Systems

ProtectManager allows you to modify, remove, activate, or disable protection plans for local file systems.

You need to log in to ProtectManager, choose Protection > Storage > Local File System, and locate the desired local file system.

Table 1 describes the operations on protection plans related to a local file system.

Table 1 Operations on protection plans related to a local file system

Operation

Description

Navigation Path

Resource Scan

Scenario

If a file system is added to or deleted from the storage system, it may take about one minute for the ProtectManager to discover the file system change. If you want to manually scan for file systems in the current storage system, perform the rescan operation.

In the local file system area, click Resource Scan.

Modify Protection

Scenario

To associate a file system with an SLA again, you can modify the protection plan. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.

In the row where the desired file system resides, choose More > Modify Protection.

Remove Protection

Scenario

If a file system does not need to be protected, you can remove its protection. After the protection is removed, the storage system cannot execute protection jobs for the file system. To protect the file system again, you need to re-associate it with an SLA.

Note

When the protection job of the file system is running, the protection cannot be removed.

In the row where the desired file system resides, choose More > Remove Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row where the desired file system resides, choose More > Manual Backup.

Disable Protection

Scenario

If you do not need to perform periodic protection for a file system temporarily, you can disable the protection plan of the file system. After the protection plan is disabled, the system does not automatically back up the file system.

Note

Disabling protection does not affect ongoing backup jobs.

In the row where the desired file system resides, choose More > Disable Protection.

Activate Protection

Scenario

After the protection plan of a file system is disabled, you can activate the protection plan to make it take effect again. After the protection plan is activated, the system backs up the file system based on the protection plan.

Note

This operation can be performed only for a file system that has been associated with an SLA and is in the Unprotected state.

In the row where the desired file system resides, choose More > Activate Protection.
