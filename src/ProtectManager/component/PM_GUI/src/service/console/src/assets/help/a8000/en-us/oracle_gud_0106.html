Checking Whether the Target Recovery Host Is a Host in the HACS Cluster

Before recovering data to a host in the HACS cluster, freeze the cluster. You can perform the following steps to check whether the target host is in the HACS cluster.

  1. Log in to the Oracle database host.

    Use PuTTY to log in to the Oracle database host and run the su - root command to switch to user root.

  2. Run the following command to check whether the HACS is running on the host:

    /etc/init.d/corosync status

    If information similar to the following is displayed, the HACS is running on the current host:

    linux-sv0z:~# /etc/init.d/corosync status
corosync (pid 9854) is running...
