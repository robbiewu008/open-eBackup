Restoring Cloud Servers

This section describes how to restore a cloud server resource to the original or a new location after the cloud server has been backed up, when the cloud server is damaged.

Context

The product allows you to restore cloud disks to the original or a new location using backup copies or replication copies (data cannot be restored to the original cloud server using replication copies).

  • Restoration to the original location

    Restore data to the cloud server in the current project.

  • Restoration to a new location

    Restore data to another cloud server or a new cloud server on another platform or in another project.

Precautions

A maximum of 16 disks can be mounted to an agent host. Therefore, if 16 disks have been mounted to an agent host during backup and restoration, the system performs the restoration job when there are idle slots.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Alibaba Cloud.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the cloud server to be restored by cloud server name and click the name.

  3. On the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. Find the copy to be restored and choose More > Restoration on the right.
  5. Select the cloud disks to be restored and configure the restoration.
  6. Restore data to the original location or a new location.

    • Restoration to the original location

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Password

      Password of the VM to be restored. This parameter is available when you select a system disk.

      Agent Host

      Select the agent host for executing the restoration job.

      Power on Automatically After Restoration

      If this function is enabled, the cloud server automatically powers on after the restoration is complete. This option is disabled by default.

      Copy Verification Before Restoration

      If this option is enabled, the integrity of a copy is verified before the copy is used for restoration. This operation affects the restoration performance. This option is disabled by default. If no copy verification file is generated, this option cannot be enabled.
    • Restoration to a new location

      Table 2 describes the related parameters.

      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Target Organization

      Select the cloud platform to which the backup copy is restored.

      Target AZ

      Select the name of the target AZ to which the backup copy is restored.

      Cloud Server

      Select the name of the cloud server to which the backup copy is restored.

      Target Resource Set

      Select the target resource set to which the backup copy is restored.

      Password

      Password of the VM to be restored. This parameter is available when you select a system disk.

      Agent Host

      Select the agent host for executing the restoration job.

      Power on Automatically After Restoration

      If this function is enabled, the cloud server automatically powers on after the restoration is complete. This option is disabled by default.

      Copy Verification Before Restoration

      If this option is enabled, the integrity of a copy is verified before the copy is used for restoration. This operation affects the restoration performance. This option is disabled by default. If no copy verification file is generated, this option cannot be enabled.

  7. Click OK.
