Uploading a Client Software Package

If the current version of the built-in client software package on the OceanProtect does not meet your requirements, you can upload a client software package to the OceanProtect as a system administrator by referring to this section. If the current version meets your requirements, skip this section.

Context

  • During deployment of the OceanProtect, a client software package is pre-installed on the system. You can choose Protection > Clients > Clients, and click Software Package Management to view the client software version in the current system in Upload mode. If the version of the pre-installed client software package does not meet your requirements, upload a client software package of another version by referring to this section.
  • Only one built-in client software package exists on the OceanProtect. The uploaded new client software package will replace the old one.

Precautions

The GUI varies depending on the version.

Prerequisites

You have obtained the client software package DataProtect_xxx_client.zip xxx indicates the version number of the client.

  • For enterprise users: Click here.
  • For carrier users: Click here.

Procedure

  1. Choose Protection > Clients > Clients.
  2. Click Software Package Management and set related parameters.

    Table 1 describes the parameters.

    Table 1 Uploading a client software package

    Parameter

    Description

    Mode

    Select Upload.
    Client File
    NOTE:

    For 1.5.0, the parameter name is Proxy File.

    Click and select the obtained client software package.

    NOTE:
    • For 1.5.0, click .
    • Only software packages in .zip format can be uploaded.
    • The size of a file to be imported cannot exceed 2 GB for 1.5.0.
    • The size of a file to be imported cannot exceed 3 GB for 1.6.0 and later versions.

  3. Click OK to upload the software package.
