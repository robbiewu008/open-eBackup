恢复单机/可用性组

当需要对已备份的单机或可用性组进行恢复时，可以参考本节使用副本恢复单机/可用性组的数据库至原位置或新位置。

前提条件

  • 执行恢复前，请确保Exchange Server服务和相关的依赖服务正常运行。对应服务可参考Exchange 服务器上的 Exchange 服务概述。

注意事项

  • 恢复至原位置或新位置时，请确保目标位置空间足够，否则会恢复失败。
  • 恢复前需要对待恢复的数据库进行卸载（支持人工或系统自动卸载）。

操作步骤

  1. 选择“数据利用 > 副本数据 > 应用 > Exchange”。
  2. 支持以单机资源维度或以副本维度查找副本，本节以单机资源维度为例进行说明。

    在“资源”页签，根据单机名称查找到需要恢复的单机，并单击对应的单机名称。

  3. 依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在行，单击“更多 > 恢复”。

    • 恢复单机/可用性组至原位置。

      相关参数说明如表1所示。

      表1 恢复Exchange单机/可用性组至原位置

      参数

      说明

      单机/可用性组

      选择恢复的目标主机或可用性组。原位置恢复该参数不可修改。

      恢复前自动卸除数据库
      开启该开关，恢复前会自动卸除目标数据库，此时目标数据库将无法使用。
      说明：

      关闭该开关，当目标数据库处于装入状态时，将会导致恢复失败。

      恢复后自动装入数据库

      恢复完成后，目标数据库将处于卸除状态。开启该开关后，会自动装入目标数据库。

      数据库名称前缀

      恢复后数据库名称的前缀。数据库名称前缀和数据库名称后缀参数设置后，恢复后数据库名称格式为：前缀_待恢复数据库名称_后缀。

      数据库名称后缀

      恢复后数据库名称的后缀。数据库名称前缀和数据库名称后缀参数设置后，恢复后数据库名称格式为：前缀_待恢复数据库名称_后缀。
    • 恢复单机/可用性组至新位置。

      相关参数说明如表2所示。

      表2 恢复Exchange单机/可用性组至新位置

      参数

      说明

      单机/可用性组

      选择恢复的目标主机或可用性组。

      数据库文件路径

      指定恢复目标数据库“.edb”文件所在目录的绝对路径，路径中的盘符必须存在。

      日志文件路径

      指定恢复目标数据库“.log”文件所在目录的绝对路径，路径中的盘符必须存在。

      恢复前自动卸除数据库
      开启该开关，恢复前会自动卸除目标数据库，此时目标数据库将无法使用。
      说明：

      关闭该开关，当目标数据库处于装入状态时，将会导致恢复失败。

      恢复后自动装入数据库

      恢复完成后，目标数据库将处于卸除状态。开启该开关后，会自动装入目标数据库。

      数据库名称前缀

      恢复后数据库名称的前缀。数据库名称前缀和数据库名称后缀参数设置后，恢复后数据库名称格式为：前缀_待恢复数据库名称_后缀。

      数据库名称后缀

      恢复后数据库名称的后缀。数据库名称前缀和数据库名称后缀参数设置后，恢复后数据库名称格式为：前缀_待恢复数据库名称_后缀。

  5. 选择需要恢复的数据库，在“已选数据库”区域可预览“恢复后数据库名称”。
  6. 对于Exchange单机，可选择恢复的高级配置。

    配置参数如表3所示。

    表3 恢复Exchange单机高级配置参数

    参数

    说明

    恢复前执行脚本

    仅对于单机恢复，可以设置“恢复前执行脚本”、“恢复成功执行脚本”、“恢复失败执行脚本”。

    您可根据实际情况，在恢复任务执行前、或恢复任务执行成功后、或恢复任务执行失败后，执行用户自定义的脚本，脚本类型为bat。若将脚本放置在DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty目录下，则只需填写脚本名称，若将脚本放置在其他目录下，则需要填写脚本绝对路径。
    说明：

    当配置了“恢复成功执行脚本”时，即使该脚本执行失败，OceanProtect上也会显示恢复任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。

    恢复成功执行脚本

    恢复失败执行脚本

  7. 单击“确定”。

后续操作

恢复后，需要在生产端更新用户邮箱状态，并创建新的Active Directory域用户与恢复后的邮箱建立连接，以验证恢复后的用户邮箱数据。具体操作请参考验证恢复后的用户邮箱数据（适用于Microsoft Exchange Server 2010）或验证恢复后的用户邮箱数据（适用于Microsoft Exchange Server 2013及后续版本）。
