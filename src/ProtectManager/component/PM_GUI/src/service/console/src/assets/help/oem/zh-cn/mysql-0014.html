步骤2：手动配置软连接

在执行备份任务前，MariaDB/GreatSQL需要注册为系统服务，MySQL/MariaDB/GreatSQL需要为所有节点手动配置mysqlbinlog、mysqldump、mysqld、mysqladmin和mysql软连接。

操作步骤

  1. 登录MySQL/MariaDB/GreatSQL实例所在的主机。
  2. 执行systemctl status mysqld或service mysqld status命令，查看MySQL/MariaDB/GreatSQL实例运行状态。

    • 如果有运行状态回显（不同的版本回显可能有差异，例如回显为active(running)），则表示MariaDB/GreatSQL已注册为系统服务，请继续执行4。
    • 如果无运行状态回显，
      • MySQL：请确保MySQL服务能够正常登录。
      • MariaDB/GreatSQL：请执行cp 安装目录/mysql.server /etc/init.d/mysql命令，注册为系统服务，安装目录为MariaDB/GreatSQL的安装目录。

  3. 依次执行以下命令设置软连接。

    ln -sf 安装目录/bin/mysqlbinlog /usr/bin/
ln -sf 安装目录/bin/mysql /usr/bin/
ln -sf 安装目录/bin/mysqldump /usr/bin/
ln -sf 安装目录/bin/mysqld /usr/bin/
ln -sf 安装目录/bin/mysqladmin /usr/bin/

  4. 分别执行以下命令，查看是否已设置软连接，如果有版本号回显，则表示软连接设置成功。

    mysqlbinlog --version
mysqldump --version
mysqld --version
mysql --version
mysqladmin --version
