Replacing the Management Plane Domain Authentication Certificate

When a storage device communicates with an LDAP domain server, the storage device functions as a client and the LDAP domain server functions as a server. In this scenario, a CA certificate must be imported, but a client certificate is optional.

Context

No default certificate exists in the domain authentication scenario.

Procedure

  1. Obtain the certificate signing request (CSR) file and private key.

    The following two methods are supported:
    • On DeviceManager, export the CSR file in the domain authentication scenario. The private key is generated in the storage device and saved to a database.
      1. Log in to DeviceManager.
        1. Log in to OceanProtect.
        2. Choose System > Infrastructure > Local Storage.
        3. Click Open the device management platform in the Local Storage area. The DeviceManager page is displayed.
      2. Choose Settings > Certificate > Certificate Management.
      3. Select Management plane domain authentication certificate.
      4. Click Export Request File.
      5. Set Certificate Key Algorithm.
        Table 1 describes related parameters.
        Table 1 Exporting CSR file parameters

        Parameter

        Description

        Certificate Key Algorithm

        Select an algorithm to generate a CSR file. The default value is RSA_2048.

        Country

        Value of the country parameter in the subject field when a CSR file is generated. The default value is CN.

        State or Province

        Value of the state or province name parameter in the subject field when a CSR file is generated. No default value.

        City or Locality

        Value of the locality parameter in the subject field when a CSR file is generated. No default value.

        Organization

        Value of the organization parameter in the subject field when a CSR file is generated. The default value is Huawei.

        Organization Unit

        Value of the organizational unit parameter in the subject field when a CSR file is generated. The default value is Storage.

        Common Name

        Value of the common name parameter in the subject field when a CSR file is generated. The default value is ESN.

        Subject Alternative Name

        An extended field in the X509 standard for exporting the CSR file. The SSL certificate that uses the Subject Alternative Name field can extend the domain names supported by the certificate so that one certificate can support the resolution of multiple domain names.

        Examples are as follows:

        DNS:*.huawei.com,IP:10.10.10.10,IP:13::17,email:me@huawei.com,RID:1.2.3.5,URI://my.url.here/,otherName:1.0.0.1;UTF8:some\sother\sidentifier,dirName:dir_name\n[dir_name]\nC=UK\nST=London\nCN=My\sname country=CN state_province=Beijing locality=Chengdu organization=Huawei organizational_unit=Huawei\sStorage common_name=Huawei\sIT@storage

        Parameters such as Country, State or Province, and City or Locality are hidden. You can click Advanced to display them.

      6. click OK.
    • Use the OpenSSL tool to generate a private key in plaintext and a CSR file.
      Run the following command on the CLI of OpenSSL to generate a private key in plaintext and a CSR file:
      openssl genrsa -out test.key 2048 
      openssl req -new -sha256 -days 3650 -subj 
      /C=XXX/ST=XXXX/L=XXX/O=XXXX/OU=XXXX/CN=XXXX -key test.key -out test.csr

      Parameter description:

      • C: Country
      • ST: State/Province
      • L: City/Town
      • O: Organization
      • OU: Organization unit
      • CN: indicates the name. The ESN of the current storage device is used as the CN. You can view the ESN on the home page of DeviceManager.

  2. Send the exported CSR file to a third-party CA center. After the CA center signs it or you use your enterprise's root certificate to sign it, the certificate file is generated.

    Table 2 describes common third-party CA centers.

    Table 2 Frequently used third-party CA centers

    Name of Third-Party CA Center

    Introduction

    Link

    VeriSign

    Digital certificates of VeriSign are the most complete and support most applications and devices. The supported SSL certificates include Secure Site Pro, Secure Site, Secure Site Pro with EV, and Secure Site with EV.

    https://www.verisign.com/

    GeoTrust

    Digital certificates of GeoTrust are mainly SSL certificates, including the following three types:

    • The ultrafast SSL series (QuickSSL Premium, RapidSSL, and Power Server ID) that verifies domain name ownership and does not verify the business license
    • Certificate that requires verification of the business license (True Business ID)
    • Newly launched EV SSL certificate (True Business ID with EV)

    https://www.geotrust.com/ssl/

  3. Import the certificate file.

    • If the CSR file is generated on DeviceManager, import the signed certificate and the CA certificate downloaded from the trusted third-party CA center that signed the LDAP domain server certificate to the storage device on DeviceManager.
    • If the CSR file is generated using the OpenSSL tool, import the plaintext private key, signed certificate, and CA certificate downloaded from the trusted third-party CA center that signed LDAP domain server certificate to the storage device on DeviceManager.
    To import the certificates on DeviceManager, perform the following steps:
    1. Log in to DeviceManager.
      1. Log in to OceanProtect.
      2. Choose System > Infrastructure > Local Storage.
      3. Click Open the device management platform in the Local Storage area. The DeviceManager page is displayed.
    2. Choose Settings > Certificate > Certificate Management.
    3. Select Management plane domain authentication certificate.
    4. Click Import Certificate.
    5. Select Certificate File, CA Certificate File, and Private Key File.
    6. click OK.

    In the scenario where the server verifies the client, to replace the security certificate on the domain server, the CA certificate of the storage device must be imported to the domain server after the certificate and private key are imported to the storage device.
