Managing File System Copies

This section describes the operations that you can perform on copy data on ProtectManager.

Before performing these operations, you need to log in to ProtectManager and choose Explore > Copy Data > Local File System. The local file system copy page is displayed. Click the Copy tab and perform related operations by copy.

Table 1 describes the relevant operations.

Table 1 Operations related to file system copies

Operation

Description

Navigation Path

Modifying a copy retention policy

Scenario

This operation enables you to modify the retention period of a copy.

Note

  • The modified copy retention period is calculated once the modification takes effect.
  • If the expiration time T of any copy in a copy chain except the latest one arrives, the copy and the associated copies whose expiration time is earlier than T are deleted in sequence.
    NOTE:
    • A copy chain refers to a full copy and all its associated incremental copies. If a full copy does not have any associated incremental copy, the full copy independently forms a copy chain.
    • The expiration of the latest copy chain cannot be triggered.

In the row where the copy resides, choose More > Modify Retention Policy.

You can set the retention period to be permanent or to xx days, xx weeks, xx months, or xx years.

Deleting a copy

Scenario

If a copy is not required or becomes invalid, you can delete it to free up storage space.

Note

  • Before deleting a copy, ensure that the copy is no longer used for restoration.
  • Copies in the Invalid state can be deleted, and unindexed copies in the Normal state can also be deleted. Copies in other states cannot be deleted.
  • After a copy is deleted, all the copies that depend on the copy are also deleted.
  • Copies in the latest copy chain cannot be deleted (except that the latest copy chain comprises only one copy).
    NOTE:

    A copy chain refers to a full copy and all its associated incremental copies. If a full copy does not have any associated incremental copy, the full copy independently forms a copy chain.

In the row where the copy resides, choose More > Delete.

Clear Index

Scenario

If the index of a copy is not required, you can delete the index.

In the row where the copy resides, choose More > Clear Index.

Note: The configurations for restore and file level restore are similar to those in Restoring a File System and Restoring Files in a File System, which are not repeated in this section.
