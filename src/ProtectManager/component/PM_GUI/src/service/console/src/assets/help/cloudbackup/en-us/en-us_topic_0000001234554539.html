Restoring Files in a File System

You can use file level restore to restore some files in a file system that has been backed up.

Prerequisites

Automatic indexing has been enabled for the SLA associated with the file system.

Precautions

  • Restoration to a file system in WORM, CLONE, or non-UNIX security mode is not supported.
  • Restoration to a file system whose name starts with pvc_ is not supported.
  • Restoration to the secondary file system of asynchronous remote replication is not supported.
  • Restoration to the snapshot directory is not supported.
  • After HyperMetro brain-split occurs, the end that fails arbitration does not support restoration.
  • Ensure that no other user is reading or writing the file system during the restoration. Otherwise, data may be inconsistent or even damaged.
  • The restoration operation is irreversible. If the restoration fails, data is not rolled back and some data may be damaged.

Procedure

  1. Choose Explore > Copy Data > Local File System.
  2. You can search for copies by resource or copy. This section uses the resource dimension as an example.

    On the Resources tab page, locate the file system to be restored by file system name and click the name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, copies exist in the month or on the day.

  4. Locate the copy to be restored and choose More > File-level Restoration.
  5. Determine whether to select Only the selected files or directories will be restored to the target path. The upper-level directory will not be restored. according to the actual requirements.
  6. In the left pane, select the files to be restored. In the right pane, select the target location for restoration.

    • File level restore is supported only when the index status of the copy is Indexed.
    • If the backup copy does not contain OS data, no index can be created for the copy. Therefore, the copy cannot be used for file level restore.
    • If a folder or file name contains garbled characters, file level restore is not supported. Do not select folders or files of this type. Otherwise, file level restore fails.
    • Select the original location for restoration, that is, the original directory of the original file system.

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring a file system

      Parameter

      Description

      Skip existing files

      The files that already exist in the file system are not processed during the restoration.

      Replace existing files

      The files that already exist in the file system are replaced during the restoration.

      Only replace the files older than the restoration file

      Among the files that already exist in the file system, only the outdated files are replaced.
    • Select a new location for restoration, that is, a specified file system.

      Table 1 describes the related parameters.

  7. Click OK.
  8. Read the warning message carefully, select I have read and understand the consequences associated with performing this operation., and then click OK.
