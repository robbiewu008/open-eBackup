恢复Informix

当需要对已备份的Informix实例进行恢复时，可以参考本节恢复Informix实例至原位置或新位置。

背景信息

支持使用备份副本、复制副本、归档副本进行恢复。支持恢复至原位置（复制副本不支持恢复至原位置，复制副本的归档副本不支持恢复至原位置）或新位置。

前提条件

恢复前需关闭Informix数据库，若Informix集群类型为HDR集群，主备节点均需关闭数据库。

  1. 使用PuTTY，登录Informix数据库主机，切换为数据库安装用户。
  2. 执行source ~/.bash_profile命令，配置环境变量。
  3. 执行onstat -l命令，查看Informix数据库状态。

    若回显中出现On-Line (Prim)、Read-Only (sec)或On-Line，则表示数据库未关闭，请继续执行后续操作关闭数据库。

    Informix单机未关闭数据库显示On-Line，Informix集群主节点未关闭数据显示On-Line (Prim)、Informix集群从节点未关闭数据显示Read-Only (sec)。

  4. 执行onmode -ky命令，关闭数据库。

操作步骤

  1. 选择“数据利用 > 副本数据 > 数据库 > Informix”。
  2. 您可以以Informix实例资源维度或以副本维度查找副本，本节以资源维度为例进行说明。

    在“资源”页签，根据资源名称查找到需要恢复的资源，并单击资源名称。

  3. 选择副本恢复的资源。
  4. 指定副本或指定时间点进行恢复。

    • 指定副本恢复
      1. 依次选择年、月、天找到副本。

        当时间上显示，即表示该月、该天存在副本。

      2. 在副本数据页面单击，单击需要恢复的副本所在行的“更多 > 恢复”。
    • 指定时间点恢复
      1. 依次选择年、月、天找到副本。

        当时间上显示，即表示该月、该天存在副本。

      2. 在副本数据页面单击，在时间轴上选择蓝色覆盖区域的副本，单击“更多 > 恢复”。
        • 对于1.5.0版本，时间轴上最多仅展示100个副本，单击可以查看所有副本。
        • 执行时间点恢复时，由于管理界面无法获取恢复时所使用的副本信息，因此，在恢复任务和相关的事件页面，用户无法看到该恢复任务的副本信息。
      3. 恢复Informix数据库至原位置或新位置。
        相关参数说明如表1。
        表1 恢复Informix数据库

        参数

        说明

        恢复至

        选择恢复至“原位置”或“新位置”。

        位置

        当选择恢复至“原位置”时，显示原位置。

        目标实例

        当选择恢复至“新位置”时，选择需要恢复的目标实例。
        1. 单击“确定”。
        2. 系统弹出如下提示，确认提示内容后，单击“关闭”。

  5. 单击“确定”。
