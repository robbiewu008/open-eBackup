Viewing ClickHouse Environment Information

Procedure

  1. Choose Protection > Big Data > ClickHouse.
  2. On the Cluster tab page, query the ClickHouse cluster information.

    1. This page displays information about ClickHouse clusters registered with the product.

      Table 1 describes related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 ClickHouse cluster information

      Parameter

      Description

      Name

      ClickHouse cluster name.

      Status

      ClickHouse cluster status.

      • Online: If the connections between the host where the client is installed and all nodes in the ClickHouse cluster are normal, Status is Online.
      • Offline: If the connections between the host where the client is installed and all nodes in the ClickHouse cluster are abnormal, Status is Offline.
      • Exception: If the connections between the host where the client is installed and some nodes in the ClickHouse cluster are abnormal, Status is Exception.

      Version

      ClickHouse version number.
      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of a host or cluster.

      If a host or cluster has been authorized to a data protection administrator, the name of the data protection administrator is displayed. If no authorization is performed, -- is displayed.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a label has been added for the resource, the label name is displayed. Otherwise, -- is displayed.
    2. Click the cluster name to view the cluster basic information and the information about the database or table set related to the cluster.

  3. On the Databases tab page, query the ClickHouse database information.

    1. This page displays the details about the ClickHouse database.

      Table 2 describes related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 2 ClickHouse database parameters

      Parameter

      Description

      Name

      Database name.

      Owning Cluster

      Cluster to which a database belongs.

      SLAs

      If an SLA has been associated with the ClickHouse database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether the ClickHouse database has been protected.

      • Unprotected: The ClickHouse database is not associated with an SLA, or the ClickHouse database is associated with an SLA but protection is disabled.
      • Protected: The ClickHouse database is associated with an SLA and protection has been activated.
      • Creating: The ClickHouse database is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    1. Click the name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Related jobs

        Displays backup and restoration jobs.

  4. On the Table Sets tab page, view the information about the ClickHouse table set.

    1. This page displays the details about the ClickHouse table set.

      Table 3 describes related parameters.

      Table 3 ClickHouse table set parameters

      Parameter

      Description

      Name

      Name of a ClickHouse table set.

      Owning Cluster

      Cluster to which the ClickHouse table set belongs.

      Owning Database

      Database to which the ClickHouse table set belongs.

      SLAs

      If an SLA has been associated with the ClickHouse table set, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether the ClickHouse table set has been protected.

      • Unprotected: The ClickHouse table set is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The ClickHouse table set is associated with an SLA and protection is enabled.
      • Creating: The ClickHouse table set is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Related jobs

        Displays backup and restoration jobs.
