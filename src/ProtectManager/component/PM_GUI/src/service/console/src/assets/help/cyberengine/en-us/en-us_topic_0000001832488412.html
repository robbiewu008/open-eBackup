Viewing Certificate Information

This section describes how to view basic information of certificates.

Procedure

  1. Choose System > Security > Certificates.
  2. Click the name of a certificate. In the dialog box that is displayed on the right, view the information.

    Table 1 lists the parameters.

    Table 1 Certificate parameters

    Parameter

    Description

    Name

    Certificate name.

    Type

    Certificate type.

    [Value range]

    Email, External Storage, LDAP, or Syslog.

    Certificate Status

    Current certificate status.

    [Value range]

    Valid or Invalid

    Expired At

    Expiration time of the current certificate.

    NOTE:

    The OceanCyber, innerComponent, and innerDatabase certificates are valid for a long time.

    Expiration Warning Days

    When the set warning days value before the expiration time of the current certificate is reached, the system generates an alarm. An appropriate warning days value helps you monitor the expiration time of the certificate.

    [Value range]

    The value is an integer from 7 to 180.

    The default value is 30 days.

    User

    User of the current certificate.

    Issuer

    Issuer of the current certificate.

    Certificate Status

    Status and expiration time of the current certificate.

    [Value range]

    Valid or Invalid

    Certificate Encryption Algorithm

    Encryption algorithm of the current certificate content.

    Only the RSA encryption algorithm is supported.

    Algorithm Signature

    Algorithm signature of the current certificate.

    Encryption Length

    Encryption length for the current certificate content. For the server certificate, internal communication certificate, and internal database certificate, the default value of this parameter is 3072.

    Fingerprint

    Hash value of the certificate content. Each certificate has a unique hash value. When a device tries to obtain a certificate, it verifies the certificate fingerprint to ensure that the certificate content is not illegally tampered with. If the device obtains the certificate fingerprint which is different from that configured in the PKI domain, the device rejects the certificate.
