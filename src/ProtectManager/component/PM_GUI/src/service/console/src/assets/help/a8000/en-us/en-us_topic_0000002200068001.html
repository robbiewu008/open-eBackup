Step 3: Registering an AntDB Cluster Instance

Full path of the configuration file (this file is unavailable in a single-node database)

Contact the database administrator to obtain the information.

You can use PuTTY to log in to the AntDB database host and run the ps -ef |grep adbhamgr |grep -v grep command to query the configuration file path. In the command output, /etc/adbhamgr/adbhamgr_antdbcluster.yaml is the configuration file path, as shown in the following figure:

[root@antdb-165-71 ~]# ps -ef |grep adbhamgr |grep -v grep
antdb     179351       1  0 16:18 ?        00:00:01 /home/antdb/antdb/tool/adbpy3/bin/adbpy3 /home/antdb/antdb/tool/adbhamgr/bin/adbhamgr /etc/adbhamgr/adbhamgr_antdbcluster.yaml

Database installation path
  • Single-node system

    Use PuTTY to log in to the AntDB database host and run the ps -ef |grep postgres command. Information similar to the following is displayed. /home/antdb/antdb preceding -D is the database installation path.

    [root@antdb-165-71 ~]# ps -ef |grep postgres
antdb     179351       1  0 16:18 ?        00:00:01 /home/antdb/antdb/bin/postgres -D /home/antdb/antdb/data
  • Cluster

    Find the path in the bin_dir line in the configuration file (for example, adbhamgr_antdbcluster.yaml). If information similar to the following is displayed, the path is /home/antdb/app.

    bin_dir: /home/antdb/app/bin

Database port
  1. Use PuTTY to log in to the AntDB database host (in the case of a database cluster, query the port on any host). Run the ps -ef |grep postgres first. Information similar to the following is displayed. /home/antdb/antdb/data following -D is the data directory in the database.
    [root@antdb-165-71 ~]# ps -ef |grep postgres
antdb     179351       1  0 16:18 ?        00:00:01 /home/antdb/antdb/bin/postgres -D /home/antdb/antdb/data
  2. Run the grep -w port Full path of the data directory in the database/postgresql.conf command based on the database directory in 1. The following information is displayed, and the port number is 6655.
    [root@antdb-165-71 ~]# grep -w port /home/antdb/antdb/data/postgresql.conf
port = 6655                     #(change requires restart)

Username and password of the database
  • Single-node system

    Contact the database administrator to obtain the information.

  • Cluster

    In the configuration file (for example, adbhamgr_antdbcluster.yaml), check the values of username and password under the superuser line.

Username and password for streaming replication of the database (this user is not required for the database of a single server)

In the configuration file (for example, adbhamgr_antdbcluster.yaml), check the values of username and password under the replication line.
