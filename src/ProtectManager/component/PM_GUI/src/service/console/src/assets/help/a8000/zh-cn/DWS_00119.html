对于GaussDB(DWS) 8.0.0版本，集群副本恢复到新集群，因下发恢复命令的节点拷贝cluster_static_config文件出错，恢复任务失败

故障现象

GaussDB(DWS) 8.0.0版本集群副本恢复到新集群，恢复任务失败，任务详情中显示“Roach命令执行失败”。

可能原因

下发恢复命令的GaussDB(DWS)节点元数据恢复已成功，但将cluster_static_config文件拷贝到其他节点时出错，导致数据恢复失败。

处理建议

  1. 使用PuTTY，登录已部署客户端的GaussDB(DWS)集群的CN节点。
  2. 依次执行以下命令切换到集群用户并加载环境变量。

    su - 集群用户名
    source 环境变量文件路径

  3. 执行cd $GAUSSLOG/roach/controller/命令进入日志文件目录，查看以“roach_controller”开头的日志文件中是否包含以下错误信息：

    cluster_static_config: No such file or directory

    示例如下：

    • 如果包含错误信息，请重新执行一次恢复任务。
    • 如果不包含错误信息，请执行4。

  4. 登录已部署客户端的GaussDB(DWS)集群的其他CN节点，执行2~3。如果所有节点都不包含该错误信息，请收集日志并联系华为技术支持工程师协助处理。
