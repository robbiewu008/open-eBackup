Creating a Ransomware Protection and WORM Policy

This section describes how to set a ransomware protection and WORM policy for backup and replication copies.

Procedure

  1. Choose Data Security > Ransomware Protection > Ransomware Protection and WORM Configurations.

    For 1.5.0, choose Explore > Data Utilization Policies.

  2. Click Ransomware Protection and WORM Policies.
  3. Click Create.
  4. Enter basic information in the displayed dialog box. Table 1 describes the basic parameters.

    Table 1 Basic parameters of a ransomware protection and WORM policy

    Parameter

    Description

    Name

    Ransomware protection and WORM policy name, which can be customized.

    NOTE:

    The name of a ransomware protection & WORM policy cannot be changed after creation.

    Description

    Ransomware protection and WORM policy description. You are advised to enter description related to ransomware protection and WORM to facilitate subsequent management.

  5. Configure the ransomware protection and WORM policy. Table 2 describes the parameters.

    Table 2 Ransomware protection and WORM policy parameters

    Parameter

    Description

    Ransomware Protection Policy

    This option is disabled by default. After this option is enabled, you can set a scheduling plan for the ransomware protection policy to perform ransomware detection on backup and replication copies.

    Schedule

    This parameter is displayed after Ransomware Protection Policy is enabled.

    Scheduling plan of the ransomware protection policy.

    • Detect immediately after successful backup or replication
    • Periodic Detection
      Periodic detection parameters:
      • Detection Frequency: every xx minutes, hours, days, or weeks
      • Time of the First Detection: start time of the first detection
        NOTE:

        If the first detection time is set to an earlier time, detection starts in the next ransomware protection policy period after the setting is complete.

      • Detection Range: You can perform detection for the latest copy, all copies, and copies generated after a specified time point.

    WORM Policy

    This option is disabled by default. After this option is enabled, you can set WORM for backup and replication copies.

    WORM Setting

    This parameter is displayed after WORM Policy is enabled.

    When Ransomware Protection Policy is enabled, this parameter is used to set uninfected copies as WORM copies after ransomware detection is complete.

    When Ransomware Protection Policy is disabled, this parameter is used to set copies as WORM copies after backup or replication is successful.

    NOTE:
    • The expiration time of WORM copies will be delayed and the copies cannot be modified or deleted before they expire.
    • WORM copies cannot be archived.

  6. Set resource object parameters. Table 3 describes the parameters.

    Table 3 Resource object parameters

    Parameter

    Description

    Type

    Type of a resource object.

    • Backup Copy: Backup copies of the local cluster
    • Replicated Copy: Replication copies replicated from other clusters to the local cluster
      NOTE:

      Ensure that the local cluster functions as the secondary end of the remote replication pair and a replication copy has been generated.

    Resource Type

    Resource type. Ransomware detection supports only VMware VM, NAS File Systems, NAS Shares, Filesets, CNware VM, Huawei Cloud Stack, FusionCompute VM, OpenStack Cloud Server, Hyper-V VM, and FusionOne Compute VM. WORM is supported for all resources except GaussDB (DWS).

    NOTE:

    For 1.5.0, ransomware detection supports only VMware, NAS File Systems, Filesets, and NAS Shares.

  7. Select the resource on which detection is to be performed and click OK. The system performs detection on the copies of the selected resource according to the scheduling plan configured in the ransomware protection and WORM policy.

    Dynamic detection is performed by default after a ransomware protection policy is configured. For details about how to switch to static detection, see Managing the Detection Mode.
