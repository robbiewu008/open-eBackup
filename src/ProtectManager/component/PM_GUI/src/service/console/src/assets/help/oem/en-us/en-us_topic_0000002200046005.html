Restoring a NAS File System

This section describes how to restore a NAS file system that has been backed up to the original location or a new location.

Context

The product supports restoration to the original location or a new location using backup copies or replication copies (data cannot be restored to the original location).

Prerequisites

  • To restore a NAS file system to a new location, a storage device residing at the new location needs to be added to the product. For details about how to add a storage device, see Step 2: Adding a Storage Device. The following lists the supported storage device types.
    • OceanStor Dorado V700R001
    • OceanStor V700R001
    • OceanStor Dorado 6.x
    • OceanStor 6.x
    • OceanProtect 1.3.0 and later versions
    • OceanStor V5
    • OceanStor Pacific
    • NetApp ONTAP
  • If a NAS file system is restored to a new location, create a target NAS file system on the storage device where the new location resides and configure a NAS share that uses the same share protocol as the copy to be restored. This is because the NAS file system is restored to a new location through NAS share. Therefore, you need to configure NAS share.
  • If you want to restore a NAS file system to a new location, you need to configure the access permission for the product on the new storage device. For details, see Step 4: Configuring Access Permissions (Applicable to OceanStor V5/OceanStor Pacific/NetApp ONTAP). During the operation, configure a client or add a user for the target NAS share.

Procedure

  1. Choose Explore > Copy Data > File Systems > NAS File Systems.

    For 1.5.0, choose Explore > Copy Data > File Services > NAS File Systems.

  2. Copies can be searched by NAS file system resource or copy. This section uses the NAS file system resource as an example.

    On the Resources tab page, locate the NAS file system to be recovered by file system name and click the name.

  3. On the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed under a month or date, copies exist in the month or on the date.

  4. In the row of the target copy, choose More > Restore.
  5. For 1.6.0, select Direct Recovery if the copy is generated when the Automatic Indexing of Copies Archived to Tapes function is enabled and is archived to tapes. For other copies, skip this step. The Direct Recovery function is enabled by default. In this case, the copy archived to tapes is directly restored without generating intermediate files, saving space. After this function is disabled, intermediate files will be generated during restoration, occupying space; however, the restoration speed is accelerated.
  6. Restore a NAS file system to its original location or a new location.

    • Restore data to the original location.

      If an OceanStor Dorado storage system with a HyperMetro relationship is used at the production end and the file system needs to be restored to the original location, use the following solutions:

      • Solution 1: Manually split the file system HyperMetro domain before restoring the file system to the original location. After restoration is complete, create a file system HyperMetro domain with the secondary end. For details about how to manually split the file system HyperMetro domain, see "Splitting a File System HyperMetro Domain" in the OceanStor Dorado 6.1.x HyperMetro Feature Guide for File.
      • Solution 2: Register the NAS share of the file system to be restored with the product (for details, see (Optional) Step 5: Registering a NAS Share). Then restore the NAS file system to a new location. The name of the restored share is the name of the NAS share that has been registered with the file system to be restored (for details, see this section).

      NOTE:
      • For an NFS share, ensure that the current device has the read and write permissions on the NFS share directory and the no_root_squash option is enabled. For details about how to configure NFS share access permissions, see Step 4: Configuring Access Permissions (Applicable to OceanStor V5/OceanStor Pacific/NetApp ONTAP).
      • For a CIFS share, ensure that the current device has the read and write permissions on the CIFS share directory. You are advised to use a user in the Administrator group for backup. Otherwise, only files and directories whose read and write permissions are granted to the device can be backed up. For details about how to configure CIFS share access permissions, see Step 4: Configuring Access Permissions (Applicable to OceanStor V5/OceanStor Pacific/NetApp ONTAP).

      Domain Name/IP Address

      Domain name or service IP address for accessing a NAS share.

      Clients

      Select clients for performing backup jobs.

      Authentication Method

      Authentication mode of a file system. This parameter is available when the share protocol of a backup copy is CIFS. Authentication mode of the CIFS protocol, which can be NTLM Authentication or Kerberos Authentication.

      In NTLM Authentication mode, you need to enter Domain Name, as well as Username and Password for accessing the NAS share.

      In Kerberos Authentication mode, choose the Kerberos authentication information. If a NAS share is registered for the first time, click Create and configure Kerberos authentication information. Table 2 describes the parameters.

      Overwrite Rule

      If files with the same names exist in the restoration path, you can choose to replace or skip existing files.

      • Replace existing files
      • Skip existing files
      • Only replace the files older than the restoration file

      Channels

      Number of subjobs that can be concurrently executed on the clients when restoration is executed. The default number of channels for the NAS file system is 10, and the maximum number is 40.

      More channels indicate better backup performance and a higher NAS share access rate. However, more CPUs and memory of the client will be occupied. Each channel occupies about 200 MB memory.

      For 1.5.0, you are advised to reduce the number of channels due to limited container resources if the client is deployed on the product.
