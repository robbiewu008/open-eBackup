After a VM Disk Is Overwritten for Restoration, Service Data Is Abnormal After the Restoration Is Complete

Symptom

After a VM disk is overwritten for restoration, service data is abnormal after the restoration is complete.

Possible Cause

When a disk is overwritten for restoration, the bus type of the original disk is different from that of the target disk. As a result, the disk device name changes after the restoration. For example, after data of a disk using the SCSI bus type is restored to a disk using the VIRTIO bus type, disk device /dev/sdb changes to /dev/vdb. As a result, the service data that references the original disk device name is abnormal.

Troubleshooting

  • Method 1: Restore the entire VM.
  • Method 2: Update the name of the disk device used by the service.

    For example, in the /etc/fstab configuration file, the name of the originally mounted disk is /dev/sdb. After the restoration, change the name to /dev/vdb.
