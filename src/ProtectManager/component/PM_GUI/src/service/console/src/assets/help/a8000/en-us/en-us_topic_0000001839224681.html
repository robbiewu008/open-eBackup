Modifying the SFTP Network

This section describes how to modify SFTP network information.

Prerequisites

The SFTP service has been enabled.

Procedure

  1. Choose System > Container Service > Network Planes.
  2. Click sftpNetPlane to view the basic information about the SFTP network.

    Table 1 describes the key parameters.

    Table 1 SFTP network parameters

    Parameter

    Description

    IP Address Type

    The value can be IPv4 address or IPv6 address.

    Subnet Mask

    Subnet mask of the IPv4 address of the SFTP network.

    NOTE:

    This parameter is available only when IP Address Type is IPv4 address.

    Prefix

    Prefix of the IPv6 address of the SFTP network.

    NOTE:

    This parameter is available only when IP Address Type is IPv6 address.

    MTU (Bytes)

    Minimum MTU value of the switch port on the network link from the switch connected to the production host to the SFTP network switch connected to the SFTP port of the OceanProtect. You are advised to use the default MTU value 1500 for all switch ports on the network link.

    IP Address Failover

    Shared virtual IP addresses are used. If node switchover is performed locally, the sending and receiving ends of the other end do not sense the node address change. In this way, communication continuity is ensured.

    VLAN

    SFTP network VLAN ID.

    Maximum Pods per Node

    Maximum number of pods on a single node. The default value is 7. A value less than or equal to 7 is supported.

  3. Click More > Modify on the right of sftpNetPlane to modify SFTP network information. Table 1 describes the parameters.
  4. Log in to the CLI and run the change container_application general name=dataprotect namespace=dpa dynamic_config="atomic=true,wait=true,timeout=1800" command to restart the data backup feature container service for the configuration to take effect. It takes about 10 minutes to restart the data backup feature container service.

    During container restart, you are not allowed to modify the network plane on DeviceManager.
