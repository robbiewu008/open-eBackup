NAS Share Copies

This section describes how to perform ransomware detection on NAS share copies.

Prerequisites

An anti-ransomware policy has been created for the resource.

Performing the Detection

In addition to detecting copies based on scheduled ransomware protection policies, you can also manually detect all copies of individual resources. To perform manual detection, perform the following steps:

  1. Choose Explore > Ransomware Protection.
  2. Click NAS Shares.
  3. Locate the row that contains the target resource and choose More > Manually Detect.

    In the displayed dialog box, select the copy to be checked and click OK.

    Alternatively, you can locate the row that contains the target resource and click Copy. In the displayed window on the right, choose More > Manually Detect on the right of the target copy.

Viewing the Report

  1. Choose Explore > Ransomware Protection.
  2. Click NAS Shares.
  3. Locate the row that contains the target resource and click Copy.
  4. In the displayed window, choose More > Detection Report on the right of a copy. The detection report displays details.
  5. (Optional) Handle false detection.

    If an uninfected copy is detected as Infected, you can correct the copy status through false alarm handling.

    1. Locate the row that contains the target resource and choose More > Handle False Detection in the Operation column.
    2. In the displayed dialog box, confirm the information and click OK. The system changes the Infected state of the incorrectly detected copy to Uninfected.

    False alarm handling is supported only for copies in the Infected state.
