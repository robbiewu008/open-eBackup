Managing Domains

The OceanProtect allows you to modify and delete domains. You need to log in to the management page and choose Protection > Cloud Platforms > OpenStack. On the Domain tab page, locate the target domain.

Table 1 describes the related operations.

Table 1 Relevant operations of managing domains

Operation

Description

Navigation Path

Modifying a domain

Scenario

If the username or password of a domain administrator is changed, perform this operation to change the username or password accordingly.

In the row of the target domain, choose More > Modify.

Change the username and password of the domain administrator.

Deleting a domain

Scenario

If a cloud server in a domain no longer needs to be protected or the domain will not be used as the restoration target, perform this operation to delete the domain.

Note

The domain cannot be deleted if it contains protected cloud servers. Remove the protection first.

In the row of the target domain, choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
