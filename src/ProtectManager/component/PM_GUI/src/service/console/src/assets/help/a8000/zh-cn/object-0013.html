步骤7：执行备份

执行备份前将需要保护的资源关联到指定SLA，系统将对这些资源按照SLA进行保护，并根据SLA周期性执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

注意事项

  • 手动备份时，SLA中定义的参数（“失败后自动重试”除外）会应用到手动备份任务中。
  • 手动备份产生的副本保留时长按照SLA中的定义进行保留。

操作步骤

  1. 选择“保护 > 文件系统 > 对象存储”。
  2. 单击“对象集合”页签，勾选需要保护的对象集合，单击“保护”。

    您也可以勾选多个对象集合进行批量保护。

  3. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

    如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  4. 配置对象集合保护高级参数。

    相关参数说明如表1所示。
    表1 对象集合保护高级参数说明

    参数

    说明

    ACL备份

    默认关闭，开启后，将备份桶和对象的ACL属性，同时也会降低对象扫描速度。

    断点续备

    默认关闭，开启后，如果上次备份失败后再次执行备份，备份任务则会基于上次备份中断的地方继续备份，而不需要重新开始备份。默认重试次数为3次。

    说明：

    如果在进入备份阶段前备份任务就已失败，则下次备份无法触发断点续备功能。

    小文件聚合

    开启后，将聚合同层目录下的小文件为大文件，节省存储空间。

    备份副本自动索引

    开启后，系统自动为副本创建文件索引，创建索引后，支持搜索副本中的文件、目录或链接。您也可以为副本手动创建索引。

    说明：

    索引需要占用备份存储空间，当系统中的索引文件总数超过10亿时，请按照《OceanProtect DataBackup 1.5.0-1.6.0 故障处理》中的“创建副本索引失败”章节进行处理。

    磁带归档副本自动索引

    默认关闭，当归档存储类型选择为“磁带库”时，显示该参数。当需要查看归档副本内的详细数据，或者需要节省恢复时占用的存储空间，此时可以开启开关。开启后，系统自动为副本创建文件索引，创建索引后，支持恢复单个或多个文件。

    说明：

    关闭磁带归档副本自动索引，磁带副本将无法直接恢复到生产存储，会额外在OceanProtect创建临时文件系统，占用备份存储空间，当恢复任务结束后，该临时文件系统会被自动清理回收。

  5. 单击“确定”。

    如果当前系统时间已晚于SLA首次备份的开始时间，可以在弹出的对话框中选择立即执行手动备份或者根据SLA中设置的备份策略周期性自动执行备份。

  6. 执行手动备份。

    当您需要立即执行一次备份任务时，可以通过手动备份实现。

    1. 单击“对象集合”页签，在需要备份的对象集合所在行单击“更多 > 手动备份”。

      支持选择多个资源批量进行手动备份。勾选多个资源，在页面左上角选择“更多 > 手动备份”。

    2. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    3. 选择保护策略：全量备份、增量备份或永久增量（合成全量）备份。
      • 永久增量（合成全量）备份仅在创建增量备份策略时，勾选了“设置为永久增量（合成全量）备份”时展示。
      • 对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。
    4. 单击“确定”。
