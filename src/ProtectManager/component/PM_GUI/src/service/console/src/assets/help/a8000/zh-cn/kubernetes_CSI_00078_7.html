上传Kubernetes安装包至Kubernetes集群

本节以Kubernetes 1.23版本为例说明如何上传Kubernetes安装包至Kubernetes集群，不同版本操作可能略有差异。如果您有镜像仓库，也可上传镜像至镜像仓库。

前提条件

上传Kubernetes安装包前，请用户在了解OceanProtect管理界面上查询版本及补丁号，再根据查询到的版本及补丁号下载“OceanProtect_DataProtect_X.X.X_K8s_BackupImage.tgz”安装包，解压安装包后，获得对应产品架构的“k8s_backup_image_x86.tar”或“k8s_backup_image_arm64.tar”压缩包，Kubernetes安装包通过以下方式获取。

  • 企业用户：点此前往
  • 运营商用户：点此前往

可通过以下方式查询版本及补丁号。

  1. 登录OceanProtect管理界面。
  2. 单击右上角图标，选择“关于”。
  3. 在弹框中“版本”所在行显示出版本及补丁号。

操作步骤

  1. 上传获取到的Kubernetes安装包到Kubernetes集群的所有节点。
  2. 进入安装包路径后，执行以下命令解压安装包。

    tar -xvzf OceanProtect_DataProtect_X.X.X_K8s_BackupImage.tgz

  3. 进入镜像包路径后，执行以下命令导入镜像包至Kubernetes节点。

    此处以x86架构为例，ARM架构请导入相应的镜像包。
    ctr -n k8s.io i import k8s_backup_image_x86.tar

  4. 在其他Kubernetes集群节点依次执行2~3的操作。
