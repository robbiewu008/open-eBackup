Backup or Restoration Fails Because the Domain Name of the ESXi Host Cannot Be Identified by the Backup Agent Host

Symptom

The backup or restoration fails. The following error information can be found in the log of the backup agent:

Failed to establish connection with the NetBackup master server. Error: HTTPSConnectionPool(host='xxx.server', port=443): Max retries exceeded with url: /netbackup/security/ping (Caused by NewConnectionError('<urllib3.connection.HTTPSConnection object at 0x7f9e466b0ef0>: Failed to establish a new connection: [Errno -2] Name or service not known',))

Possible Causes

During the backup or restoration, the backup agent host needs to connect to the ESXi host that can access the disk. However, the domain name of the target ESXi host cannot be identified by the backup agent.

Troubleshooting

  1. Log in to the agent host.
  2. Add the corresponding domain name server to /etc/resolv.conf, or add the mapping between the IP address and name of the target host to /etc/hosts.
