Managing Real-Time Detection Policies

The OceanCyber 300 Data Security Appliance allows users to view, clone, modify, or delete real-time detection policies.

Log in to the OceanCyber 300 Data Security Appliance, choose Data Security > Real-Time Detection > Real-Time Detection Policies, and locate the real-time detection policy to be operated. Table 1 describes the related operations.

Table 1 Operations related to real-time detection policies

Job

Description

Navigation Path

Viewing a real-time detection policy

Scenario

This operation enables you to view real-time detection policies to learn about the number and details of real-time detection policies.

In the Real-Time Detection Policies list, view the policy name, snapshot lock time, false alarm analysis status, honeypot detection status, honeyfile update frequency, number of associated file systems, and number of associated whitelists.

Cloning a real-time detection policy

Scenario

This operation enables you to clone a real-time detection policy.

In the row that contains the real-time detection policy, choose More > Clone.

Modifying a real-time detection policy

Scenario

This operation enables you to modify a real-time detection policy. Modifying a real-time detection policy does not affect ongoing real-time detection jobs. The new real-time detection policy will take effect in the next period.

In the row that contains the real-time detection policy, choose More > Modify.

Deleting a real-time detection policy

Scenario

This operation enables you to delete a real-time detection policy that is not required temporarily.

Note

A real-time detection policy that has been associated with a file system cannot be deleted.

In the row that contains the real-time detection policy, choose More > Delete.
