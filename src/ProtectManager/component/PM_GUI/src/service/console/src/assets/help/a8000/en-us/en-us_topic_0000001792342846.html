Adding HA Members

If you want to use the HA feature after adding a member node, select a member node as an HA member. When the active node is faulty, services are automatically switched to the HA member node (standby node). For 1.6.0 and later versions, Internal Communication Network Plane is renamed as Internal Communication Network. This section uses 1.5.0 as an example.

Prerequisites

  • A member node has been added to the system.
  • The deployment mode of the member node to be added as an HA member must be the same as that of the active node.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  2. Click Backup Cluster.
  3. Choose HA Management > Add HA Member.
  4. In the Available Member Node area, select the member node to be added as an HA member.
  5. In the Network Settings area, enter the HA member information. Table 1 describes the related parameters.

    Table 1 HA member network parameters

    Parameter

    Description

    Floating IP Address

    Enter the floating IP address of the internal communication network plane. The floating IP address must be able to communicate with the internal communication network plane of the active node. If the internal communication network plane of the active node reuses the backup network, do not use the backup network IP address as the floating IP address.

    Quorum Gateway

    Quorum gateways are used to check the link status of the standby node during active/standby switchover. The number of quorum gateways ranges from 1 to 3. Quorum gateways must be able to communicate with the internal communication network plane and the IP addresses of the quorum gateways must be unique. The IP address cannot start with 127.

  6. Click OK.

    Due to the active node breakdown and the heartbeat interruption, the standby node cannot detect the floating IP address of the active node. But if the standby node can detect quorum gateways, the standby node will become the active node. The active/standby switchover takes about 5 minutes.
