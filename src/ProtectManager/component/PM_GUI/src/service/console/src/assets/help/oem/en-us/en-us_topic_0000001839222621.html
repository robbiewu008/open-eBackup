Viewing MySQL/MariaDB Database Environment Information

After a MySQL/MariaDB database is registered with the product, you can view the database and details about the host or cluster where the database resides on the management page.

Procedure

  1. Choose Protection > Databases > MySQL/MariaDB.
  2. On the Databases tab page, view the database information.

    The registered databases are displayed on the page.
    1. Table 1 describes the database information.
      Table 1 Database information

      Parameter

      Description

      Name

      Database name.

      Owning host/cluster

      Name of the host or cluster where the database resides.

      Instance

      Name of a database instance.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Indicates whether the database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
    2. Click the name and check the overview, copy information, and jobs.
      • Overview

        Displays the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, a copy at that time exists.

      • Jobs

        Displays jobs of backup, restoration, and so on.

  3. On the Cluster tab, view the database cluster information.

    If a database in a cluster has been registered, the cluster information is displayed on the page.

    Table 2 describes the detailed information.
    Table 2 Host and cluster information

    Parameter

    Description

    Name

    Name of a database cluster.

    If you click the cluster name, all hosts in the current cluster and all authenticated databases in the cluster are displayed.

    Type

    Type of the cluster.

    Status
    Status of a host or cluster.
    • Online: When the host is powered on and ProtectAgent on the host is running properly, Status of the host is Online.

      If there are online hosts in the cluster, the cluster status is Online.

    • Offline: If the host is shut down or ProtectAgent on the host is faulty, Status of the host is Offline.

      If all hosts in the cluster are offline, the cluster status is Offline.

    Owner

    Data protection administrator of a host or cluster.

    If a host or cluster has been authorized to a data protection administrator, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
