Restoring One or More Tables

This section describes how to restore one or more tables in a GaussDB (DWS) cluster, schema set, or table set that has been backed up to the original location or a new location.

Context

Backup copies, reverse replication copies, cascaded replication copies, archive copies, and replication copies can be used for restoration. GaussDB (DWS) can be restored to the original location or a new location. Replication copies and their archive copies, cascaded replication copies and their archive copies cannot be used for restoration of the GaussDB (DWS) to the original location

If the backup copy and archive copy coexist, use the backup copy first for restoration.

Prerequisites

  • Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.
  • Before table-level restoration to a new schema, ensure that the schema has been created in the database corresponding to GaussDB (DWS). Otherwise, the restoration fails.

Procedure

  1. Choose Explore > Copy Data > Big Data > GaussDB(DWS).
  2. Search for copies by GaussDB (DWS) resource or copy. This section uses resources as an example.

    On the Resources tab page, locate the resource to be restored by resource name and click the name.

  3. Select the year, month, and day in sequence to find the copy.

    If is displayed under a month or date, copies exist in the month or on the date.

  4. Restore data using a specific copy.

    • Restore one or more tables in the GaussDB (DWS) cluster copy by using a GaussDB (DWS) cluster copy.
      1. In the row of the target GaussDB (DWS) cluster copy, choose More > Table-Level Restoration.
      2. Restore a table in the GaussDB (DWS) cluster copy to the original location or a new location.

        For 1.5.0, click Select and select the target location for restoration.

        Table 1 describes the related parameters.
        Table 1 Target location description

        Parameter

        Description

        Restore To

        Select either Original location or New location.

        Location

        If you select Original location for Restore To, the original location is displayed.

        Target Cluster
        If you select New location for Restore To, you must select the target cluster to be restored. Ensure that the installation configuration file named mppdb-install-config.xml of the target cluster has been replicated to the /opt/huawei/ directory on each node in the target cluster and that the database user has the read permission. Otherwise, the restoration will fail. Note that if the installation configuration file is not named mppdb-install-config.xml, rename it.
        NOTE:

        If the database user does not have the read permission on the configuration file, log in to each node in the target cluster as user root and run the following command:

        sudo g+r /opt/huawei/mppdb-install-config.xml
      3. Select the database where the table to be restored resides.
      4. Select one or more tables to be restored.

        For 1.6.0 and later versions: You can enter a keyword in the search box to quickly search for one or more tables to be restored.

      5. If you select New location for Restore To, enter the names of the schema and table to which the table belongs after restoration.

        For 1.6.0 and later versions: You can select multiple tables and click Batch Set Restoration Location to set the names of the schemas to which the tables belong after restoration.

      6. Click OK and confirm the information as prompted.
        • Ensure that the restoration target location is correct and the remaining space of the restoration target location is greater than the size of the copy used for restoration before reduction.
        • For 1.6.0 and later versions, table-level restoration to a new location is supported.
        • Table-level restoration can be used to restore only the table structure and data of a table set. If you want to restore information such as view and function of a table set, manually create such information by referring to Product Documentation of the corresponding GaussDB (DWS) version.
    • Restore one or more tables in the schema set or table set copy by using a schema set/table set copy.
      • Common restoration (restoring all tables in a table set copy)

        Common restoration cannot be used for schema set copies.

        1. In the row of the target table set copy, choose More > Restore.
        2. Restore a table in the copy to the original location or a new location.

          For 1.5.0, click Select and select the target location for restoration.

          Table 2 describes the related parameters.
          Table 2 Target location description

          Parameter

          Description

          Restore To

          Select either Original location or New location.

          Location

          If you select Original location for Restore To, the original location is displayed.

          Target Cluster

          If you select New location for Restore To, you must select the target cluster to be restored.

          Databases

          If Restore To is set to New location for restoration using table set copies, you need to select the target database for restoration.
        3. Click OK and confirm the information as prompted.
          • Ensure that the restoration target location is correct and the remaining space of the restoration target location is greater than the size of the copy used for restoration before reduction.
          • For table-level GDS-based restoration, the number of GDSs started at the target end must be greater than or equal to the number of GDSs started at the source end during backup.
          • For 1.6.0 and later versions, table-level restoration to a new location is supported.
          • Table-level restoration can be used to restore only the table structure and data of a table set. If you want to restore information such as view and function of a table set, manually create such information by referring to Product Documentation of the corresponding GaussDB (DWS) version.
      • Table-level restoration (restoring one or more tables in a schema set or table set copy)
        1. In the row of the target schema set or table set copy, choose More > Table-Level Restoration.
        2. Restore a table in the copy to the original location or a new location.

          For 1.5.0, click Select and select the target location for restoration.

          Table 3 describes the related parameters.
          Table 3 Target location description

          Parameter

          Description

          Restore To

          Select either Original location or New location.

          Location

          If you select Original location for Restore To, the original location is displayed.

          Target Cluster

          If you select New location for Restore To, you must select the target cluster to be restored.

          Databases

          If Restore To is set to New location for restoration using table set copies, you need to select the target database for restoration.
        3. Select the database where the table to be restored resides.
        4. Select one or more tables to be restored.

          For 1.6.0 and later versions: You can enter a keyword in the search box to quickly search for one or more tables to be restored.

        5. If you select New location for Restore To, enter the names of the schema and table to which the table belongs after restoration.

          For 1.6.0 and later versions: You can select multiple tables and click Batch Set Restoration Location to set the names of the schemas to which the tables belong after restoration.

        6. Click OK and confirm the information as prompted.
          • Ensure that the restoration target location is correct and the remaining space of the restoration target location is greater than the size of the copy used for restoration before reduction.
          • If table-level GDS restoration is performed using table set copies, the number of GDSs started at the target end must be greater than or equal to the number of GDSs started at the source end during backup.
          • For 1.6.0 and later versions, table-level restoration to a new location is supported.
          • Table-level restoration can be used to restore only the table structure and data of a table set. If you want to restore information such as view and function of a table set, manually create such information by referring to Product Documentation of the corresponding GaussDB (DWS) version.
