Viewing the Hive Environment Information

Procedure

  1. Choose Protection > Big Data > Hive.
  2. On the Clusters tab page, query the Hive cluster information.

    1. View the information about Hive clusters registered with the management page.

      Table 1 describes related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 Hive cluster information

      Parameter

      Description

      Name

      Hive cluster name

      Status
      Status of the Hive cluster
      • Online: If the host where the client is installed is properly connected to the Hive cluster, Status is Online.
      • Offline: If the host where the client is installed is improperly connected to the Hive cluster, Status is Offline.

      HiveServer2 Link

      HiveServer2 link address of the Hive cluster

      Authentication Mode

      Authentication mode for communication between the Hive cluster and the product.
      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of a cluster

      If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the cluster name to view the information of the backup set related to the cluster.

  3. On the Backup Sets tab page, query Hive backup set information.

    1. View the details about the backup sets.

      Table 2 describes related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 2 Backup set parameters

      Parameter

      Description

      Name

      Name of a backup set

      Owning Cluster

      Name of the protected Hive cluster

      SLAs

      If an SLA has been associated with the backup set, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether the backup set is protected.

      • Unprotected: The backup set is not associated with an SLA, or the backup set is associated with an SLA but protection is disabled.
      • Protected: The backup set is associated with an SLA and protection is activated.
      • Creating: The backup set is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    1. Click the name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Related jobs

        Displays backup and restoration jobs.
