步骤5：创建备份SLA

您可以根据自身需求自定义SLA以对资源进行保护。

操作步骤

  1. 选择“保护 > 保护策略 > SLA”。
  2. 单击“创建”。
  3. 自定义SLA的名称。
  4. 选择需要创建SLA的应用类型。

    1. 单击“应用”图标，选择要创建SLA的应用。
      • 通用SLA

        通用SLA可以应用到除文件集、SQL Server、PostgreSQL、Kingbase、Redis、ClickHouse、MongoDB、NAS共享、NAS文件系统、华为云Stack GaussDB、TDSQL、Kubernetes CSI、TiDB、卷、通用共享、Active Directory、GaussDB T 单机、对象存储、GaussDB（1.6.0及后续版本）、应用入口的SAP HANA（1.6.0及后续版本）以外的资源。

      • 指定应用

        选择需要创建SLA的一种应用类型，本节以创建指定应用为例进行说明。

    2. 单击“确定”。

  5. 配置备份策略。

    1. 单击“备份策略”图标。
    2. 配置备份策略基础参数。

      相关参数说明如表1所示。

      请根据业务情况合理设置备份频率、副本保留时长和允许备份的时间段，建议如下：
      • 备份频率至少大于备份时长。
      • 全量副本的保留时长大于增量备份（或永久增量（合成全量）备份）副本的保留时长。
      • 副本保留时长必须大于备份频率。
      • 至少需要创建一个备份策略，如需添加其他备份策略，请单击界面右侧的。
      • 每种备份类型最多支持添加4个备份策略。
      表1 备份策略基础参数说明

      参数

      说明

      全量备份

      名称

      设置备份策略名称。

      -
      配置全量备份的备份周期、副本保留时长以及允许备份的时间段。
      • 配置全量备份的备份周期以及副本的保留时长。
        • 按年

          设置每年xx月xx日执行一次，副本保留xx天/周/月/年或永久保留。

          如果当年不存在该日期，则不产生副本。

        • 按月

          设置每月xx日（可选择多日）或最后一天执行一次，副本保留xx天/周/月/年或永久保留。

          如果设置为每月xx日执行一次备份，当月不存在该日期时，则不产生副本。

        • 按周

          设置每周一/二/三/四/五/六/日执行一次（可多选），副本保留xx天/周/月/年或永久保留。

        • 按天

          设置从xx年xx月xx日开始，每xx天执行一次备份，副本保留xx天/周/月/年或永久保留。

        • 按小时

          设置从xx年xx月xx日开始，每xx小时执行一次备份，副本保留xx天/周/月/年或永久保留。

      • 设置允许执行全量备份的时间段，该时间范围以外的时间不会进行全量备份任务调度。
        说明：
        • 如果结束时间小于或等于开始时间，则结束时间实际为跨日的结束时间。
        • 如果备份任务在指定的时间窗口内未执行完成，系统不会中止本次备份任务，但会上报事件。
        • 一旦副本超过设定的保留时长，系统将自动删除过期的副本。

      增量备份

      名称

      设置备份策略名称。

      备份

      勾选“设置为永久增量（合成全量）备份”，可将增量备份策略设置为永久增量（合成全量）备份策略。

      -
      配置增量备份的备份周期、副本保留时长以及允许备份的时间段。增量备份和差异备份不能同时存在。
      • 配置增量备份的备份周期以及副本的保留时长。
        • 按年

          设置每年xx月xx日执行一次，副本保留xx天/周/月/年或永久保留。

          如果当年不存在该日期，则不产生副本。

        • 按月

          设置每月xx日（可选择多日）或最后一天执行一次，副本保留xx天/周/月/年或永久保留。

          如果设置为每月xx日执行一次备份，当月不存在该日期时，则不产生副本。

        • 按周

          设置每周一/二/三/四/五/六/日执行一次（可多选），副本保留xx天/周/月/年或永久保留。

        • 按天

          设置从xx年xx月xx日开始，每xx天执行一次备份，副本保留xx天/周/月/年或永久保留。

        • 按小时

          设置从xx年xx月xx日开始，每xx小时执行一次备份，副本保留xx天/周/月/年或永久保留。

        • 按分钟

          设置从xx年xx月xx日开始，每xx分钟执行一次备份，副本保留xx天/周/月/年或永久保留。

      • 设置允许执行增量备份的时间段，该时间范围以外的时间不会进行增量备份任务调度。
        说明：
        • 如果结束时间小于或等于开始时间，则结束时间实际为跨日的结束时间。
        • 如果备份任务在指定的时间窗口内未执行完成，系统不会中止本次备份任务，但会上报事件。
        • 一旦副本超过设定的保留时长，系统将自动删除过期的副本。
        • 1.6.0及后续版本支持增量备份上一次备份失败的文件。

      备份任务失败产生的副本存放在代理主机上的路径如下：

      • Windows OS系统：安装目录\ProtectClient\ProtectClient-E\log\Plugins\FilePlugin
      • 非Windows OS系统：安装目录/ProtectClient/ProtectClient-E/slog/Plugins/FilePlugin
    3. 对于1.6.0及后续版本，如果需生成WORM副本，请打开WORM开关，并设置WORM有效期。否则，请跳过本步骤。
      WORM有效期支持设置为“与副本留存期一致”或“自定义有效期”。
      • WORM副本不支持删除或缩短有效期。
      • 自定义WORM有效期不能大于副本保留时长。
      • 在WORM有效期内不支持修改为非WORM副本。
    4. 配置备份策略高级参数。
      相关参数说明如表2所示。
      表2 备份策略高级参数说明

      参数

      说明
      指定目标位置
      说明：

      仅X系列备份一体机支持此参数。
      请指定备份副本的目标存储位置。
      • 备份存储单元：对应于备份集群节点。选择备份存储单元后，副本将存储至该存储单元。存储单元容量不足时，备份任务将会失败。
      • 备份存储单元组：是备份存储单元的集合。选择备份存储单元组后，系统根据存储单元组的存储策略自动选择备份目标存储单元。
      说明：
      • 对于1.5.0版本，如果您部署了集群高可用特性，该参数才显示。您也可以不指定目标位置，如不指定，则系统通过智能均衡自动选择备份目标存储单元。
      • 对于1.6.0及后续版本，则必须指定目标位置。

      限速策略

      备份任务平均每秒最大传输的数据量。

      选择已创建的限速策略。

      自动索引

      说明：

      仅1.5.0版本存在此参数。

      开启后，系统自动为副本创建文件索引，创建索引后，支持搜索副本中的文件、目录或链接。您也可以为副本手动创建索引。

      说明：

      索引需要占用备份存储空间，当系统中的索引文件总数超过10亿时，请按照《OceanProtect 备份一体机 1.5.0-1.6.0 故障处理》中的“创建副本索引失败”章节进行处理。

      目标端重删

      当平均文件大小>=32KB 时，建议开启此开关，节省备份存储空间。
      源端重删
      说明：

      仅X系列备份一体机支持此参数。
      开启后，可以减少传输的数据量，但会增加客户端的资源占用。
      说明：
      • 由于同一个代理主机只能和一个存储单元建链，如果开启源端重删，“指定目标位置”请选择“备份存储单元”。否则，由于智能均衡可能会选择到未建链的存储单元，导致源端重删启用失败。如果不同受保护对象关联了不同SLA，且SLA中指定不同的备份存储单元，请为受保护对象选择不同代理主机，否则将会导致部分备份任务源端重删启用失败。
      • 如果代理主机已经与一个存储单元建链，当该代理主机未执行源端重删备份任务时，可以在代理主机上执行dataturbo delete storage_object storage_name=?命令（命令中的storage_name=?可以通过dataturbo show storage_object命令查询），删除与存储单元的源端重删链接。再选择该代理主机执行源端重删备份任务，则可以成功与新的存储单元建链。
      • 请参考OceanProtect兼容性查询工具查看主机的操作系统类型是否支持源端重删，若不支持，配置源端重删后将不会生效。
      • 若使用内置代理（即ProtectAgent合并部署到OceanProtect）进行备份，配置源端重删后将不会生效。
      任务超过时间窗告警
      说明：

      仅1.6.0及后续版本支持设置该参数。

      开启后任务执行超过时间窗将发送告警，告警需手动清除。

      任务失败告警

      开启后，在任务失败后发送告警，下一次任务成功后告警自动清除。

      失败后自动重试

      开启后，备份任务失败后系统自动重试。

      支持重试次数为1~5次，等待时长为1~30分钟。如设置为重试3次，等待5分钟，即每5分钟后重试一次，总共重试3次。
      说明：
      • 自动重试时系统会创建一个新的备份任务，如果该备份任务不在设定的时间窗内则不会执行，此次备份失败。
      • 该功能仅在周期性备份时生效。手动发起备份任务，任务失败无法自动重试。
    5. 单击“确定”。

  6. 可选：配置归档策略。

    OceanProtect自带存储空间有限，如果您需要长期保留副本数据，可以将副本数据归档到外部存储。

    您可以单击“归档策略”图标配置归档策略，或随后通过修改SLA的方式配置归档策略。

    配置归档策略的具体操作请参考归档。

  7. 可选：配置复制策略。

    OceanProtect支持本地数据复制到异地，当灾难发生后在异地数据中心通过副本数据恢复生产，进而承接业务，实现异地数据保护。

    您可以单击“复制策略”图标配置复制策略，或随后通过修改SLA的方式配置复制策略。

    配置复制策略的具体操作请参考步骤7：创建复制SLA。

    对于1.6.0及后续版本：

    • SLA中备份策略的WORM有效期需小于复制策略中副本保留时长。
    • 如果本端未设置WORM，目标端设置WORM，则WORM有效期与复制副本保留时长一致。
    • 如果本端设置了WORM，目标端未设置WORM，则复制副本的WORM有效期与本端WORM副本有效期一致，复制副本保留时长>=本端的备份副本WORM有效期。
    • 如果本端和目标端都设置WORM，且目标端在“数据安全”WORM配置中按资源粒度设置WORM，则所有复制副本均会有WORM属性，WORM有效期和复制副本保留时长一致。
    • 本端备份副本为永久保留，且“WORM有效期”设置为“与副本留存期一致”，则WORM有效期为永久有效。当生成了备份副本后，修改SLA策略（添加复制策略并设置“复制规则&保留规则”为“复制所有副本”），目标端WORM有效期与复制副本保留时长一致。

  8. 单击“确定”。
