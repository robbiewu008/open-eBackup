Step 2: (Optional) Generating a Token with the Minimum Permissions

You need to perform the operations in this section to generate a token with the minimum permissions only when token authentication is selected during subsequent Kubernetes cluster registration. If there is no special requirement, the token with the super administrator permissions can be used.

Procedure

  1. Query the EIPs bound to worker nodes in the CCE cluster.

    1. Log in to ManageOne Operation Portal.
    2. Choose Service List > Cloud Container Engine > CCE cluster. Select the target Kubernetes cluster name to go to the cluster information page.
    3. Choose Cluster > Nodes, and click the Nodes tab to view the EIPs bound to all worker nodes in the CCE cluster.

  2. Log in to any node in the CCE cluster using the EIP obtained in 1.
  3. Create cluster role resources. In the Kubernetes production cluster, create the pm-k8s-sa-role.yaml file and write the following content to the file.

    apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: pm-k8s-sa-role
rules:
  - apiGroups: [ "*" ]
    resources: [ "*" ]
    verbs: [ "list", "create", "get" ]
  - apiGroups: [ "*" ]
    resources: [ "pods", "persistentvolumeclaims", "volumesnapshots" ]
    verbs: [ "delete" ]
  - apiGroups: [ "*" ]
    resources: [ "statefulsets", "replicasets", "deployments", "daemonsets", "jobs", "cronjobs" ]
    verbs: [ "patch" ]

  4. On the background page of the Kubernetes cluster, run the kubectl apply -f pm-k8s-sa-role.yaml command to create ClusterRole.
  5. Create a service account. In the Kubernetes production cluster, create the pm-k8s-sa.yaml file and write the following content to the file.

    apiVersion: v1
kind: ServiceAccount
metadata:
  name: pm-k8s-sa
  namespace: kube-system

  6. On the background page of the Kubernetes cluster, run the kubectl apply -f pm-k8s-sa.yaml command to create ServiceAccount.
  7. Create ClusterRoleBinding. In the Kubernetes production cluster, create the pm-k8s-cluster-role-binding.yaml file and write the following content to the file.

    apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: pm-k8s-cluster-role-binding
subjects:
- kind: ServiceAccount
  name: pm-k8s-sa      
  namespace: kube-system
roleRef:
  kind: ClusterRole
  name: pm-k8s-sa-role
  apiGroup: rbac.authorization.k8s.io

  8. On the background page of the Kubernetes cluster, run the kubectl apply -f pm-k8s-cluster-role-binding.yaml command to create ClusterRoleBinding.
  9. Create Secret to save the token information. In the Kubernetes production cluster, create the pm-k8s-secret.yaml file and write the following content to the file.

    apiVersion: v1
kind: Secret
type: kubernetes.io/service-account-token
metadata:
  namespace: kube-system
  name: pm-k8s-secret
  annotations:
    kubernetes.io/service-account.name: pm-k8s-sa

  10. On the background page of the Kubernetes cluster, run the kubectl apply -f pm-k8s-secret.yaml command to create Secret.
  11. Obtain the token with the minimum permissions. In the production cluster, run the kubectl describe secret/pm-k8s-secret -n kube-system command. The value of the token field in the command output is the required token with the minimum permissions.
