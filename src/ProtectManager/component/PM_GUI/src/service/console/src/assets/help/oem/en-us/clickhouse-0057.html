Managing ClickHouse Database/Table Set Protection

This section describes how to modify, remove, activate, or disable ClickHouse database/table set protection on the product.

Related Operations

Choose Protection > Big Data > ClickHouse, click the Databases or Table Sets tab, and locate the database or table set on which you want to perform related operations.

Table 1 describes related operations.

Table 1 Database/Table set protection operations

Operation

Description

Navigation Path

Modify Protection

Scenario

If you need to reselect an SLA associated with a database or table set, you can modify the protection plan. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row where the database or table set resides, choose More > Modify Protection.

Remove Protection

Scenario

This operation enables you to remove the protection of a database or table set when it does not need to be protected. After the removal, the system cannot execute protection jobs for the database or table set. To protect the database or table set again, associate an SLA with it again.

Note

Protection cannot be removed when a protection job of the database or table set is running.

In the row where the database or table set resides, choose More > Remove Protection.

Activate Protection

Scenario

This operation enables you to re-activate a disabled protection plan of a database or table set. After the protection plan is activated, the system backs up the database or table set based on the protection plan.

Note

You can perform this operation only when the database or table set has been associated with an SLA and is in Inactivated status.

In the row where the database or table set resides, choose More > Activate Protection.

Disable Protection

Scenario

This operation enables you to disable the protection plan of a database or table set when its periodic protection is not needed. After the protection plan is disabled, the system does not execute the automatic backup job of the database or table set.

Note

Disabling protection does not affect ongoing backup jobs.

In the row where the database or table set resides, choose More > Disable Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row where the database or table set resides, choose More > Manual Backup.

Note: The procedure for restoring ClickHouse databases or table sets is similar to that in Restoring a ClickHouse Database or Table Set and is not described here.
