Parameter "Standby delay time consuming" Recording a High Value After a Non-distributed Instance of TDSQL MySQL V5.7.17 Is Restored

Symptom

After a non-distributed instance of TDSQL MySQL V5.7.17 is restored, the parameter value of Standby delay time consuming of the standby node is high.

Possible Cause

Due to the restrictions of TDSQL MySQL V5.7.17, after a non-distributed instance is restored, there is a possibility that the active/standby synchronization status of the standby node is abnormal.

Procedure

  1. Log in to the TDSQL CHITU management console.
  2. On the Instance management page, select an instance and go to the Instance details page.
  3. Click Redo the backup machine.
  4. In the displayed dialog box, select the target backup machine.
  5. Click Redo the backup machine.
