Step 3: Registering a Hive Cluster

Before Hive backup and restoration, register the Hive cluster with the product.

Prerequisites

Before registering a Hive cluster, ensure that you have obtained the following files. This section uses FusionInsight 6.5.1 as an example. Operations may vary on different platforms. For details, see the corresponding version of product documentation of the big data platform.

  1. Log in to FusionInsight Manager using a browser.
  2. Click on the left of the home page to download the client.
  3. Decompress the downloaded client to obtain the following files:
    • The ../Hive/config/hiveclient.properties file
    • The ../Hive/config/hive-site.xml file
    • The ../Hive/config/core-site.xml file
    • The ../Hdfs/config/hdfs-site.xml file

The following describes how to obtain the jaas.conf and hive-site.xml configuration files when the big data platform is Transwarp Data Hub (TDH). This part uses TDH 8.0.2 as an example. Operations may vary with versions. For details, see the Transwarp Data Hub product documentation.

  1. Use a browser to log in to TDH.
  2. Choose Dashboard > Cluster. On the Service List tab page, click the Hive cluster name configured in TDH.
  3. In the upper right corner of the Hive cluster page, click the icon and click Download Service Configurations to obtain jaas.conf and hive-site.xml from the downloaded package.
  4. Choose Dashboard > Cluster. On the Service List page, click the HDFS cluster name configured in TDH.
  5. In the upper right corner of the HDFS cluster page, click the icon and click Download Service Configurations to obtain core-site.xml and hdfs-site.xml from the downloaded package.
  6. Delete the following information from the core-site.xml file:
    <property>
    <name>hadoop.security.group.mapping.ldap.bind.password.file</name>
    <value>/etc/hdfs1/conf/ldap-conn-pass.txt</value>
</property>

Procedure

  1. Choose Protection > Big Data > Hive.
  2. On the Cluster page, click Register to register a Hive cluster.

    Table 1 describes the Hive cluster registration information.

    Table 1 Hive cluster registration information

    Parameter

    Description

    Name

    Customize a Hive cluster name.

    HiveServer2 Link

    Open the hiveclient.properties file obtained in Prerequisites. The value of zk.quorum is the configured value.

    NOTE:
    • If the big data platform is TDH, obtain the attribute value transwarp.docker.inceptor from the hive-site.xml file and replace the domain name in the value with the IP address of the corresponding production node. Input example: 127.0.0.0:8000
    • If the parameter ZooKeeper Namespace is left blank, you can enter only one HiveServer2 link.

    ZooKeeper Namespace

    Open the hiveclient.properties file obtained in Prerequisites. The value of zooKeeperNamespace is the configured value.

    hive-site.xml

    Upload the hive-site.xml, hdfs-site.xml, core-site.xml, and hiveclient.properties files obtained in Prerequisites to the management page.

    This file is used to configure Hive cluster information.

    NOTE:
    • The hiveclient.properties file is optional.
    • The big data cluster supports only IPv4 addresses. IPv6 addresses are not supported. Check whether configurations in the configuration file are correct.
    • The configuration file uploaded to the management page must be the same as that in the big data cluster. Do not modify the configuration file locally.

    core-site.xml

    hiveclient.properties

    Hive Version

    Hive version number.

    The following uses FusionInsight 6.5.1 as an example to describe how to check the Hive version:

    1. Log in to FusionInsight Manager using a browser.
    2. Choose Cluster > Hive to view the Hive version.
    NOTE:

    If the big data platform is TDH and the big data platform version is 8.X or 6.X, the Hive version is 1.2.0.

    Authentication Mode

    Configure the authentication mode for communication between the Hive cluster and the product. Set this parameter based on the authentication mode configured for the Hive cluster.

    • Simple Authentication: Select this authentication mode when Kerberos authentication is disabled for the Hive cluster. In this authentication mode, usernames are used for authentication, which is less secure.
    • Kerberos Authentication: Select this authentication mode when Kerberos authentication is enabled for the Hive cluster. The Kerberos protocol is used for identity authentication between the Hive cluster and the product.

    Username

    This parameter is mandatory only when Authentication Mode is set to Simple Authentication.

    NOTE:
    • The user must have the SHOW_DATABASE permission on the database to be backed up, the SELECT permission on the table to be backed up, the CREATE and SHOW_DATABASE permissions on the database to be restored, and the CREATE, DROP, ALTER, and SELECT permissions on the table to be restored.
    • The user must have the read, write, and execution permissions on the temporary metadata storage path specified when the backup set is created, the read and execution permissions on the directory where the table data of the database to be backed up is located as well as the lower-layer files and directories, the read, write, and execution permissions on the temporary storage path of metadata specified during restoration, the read, write, and execution permissions on the directory where the data restored to the target database table is located, and the read, write, and execution permissions on the directory where the table metadata and table data are located.

    Kerberos

    This parameter is mandatory only when Authentication Mode is set to Kerberos Authentication.

    Select the created Kerberos authentication. When you register a Hive cluster for the first time, click Create and configure Kerberos authentication parameters. For details about the parameters, see Table 2.

    HiveServer2 Principal

    This parameter is mandatory only when Authentication Mode is set to Kerberos Authentication.

    Open the hiveclient.properties file obtained in Prerequisites. The value of principal is the configured value.

    NOTE:

    If the big data platform is TDH, the value of this parameter is the value of principal in the Client module in the jaas.conf file.

    Agent Host

    Select the agent host for data protection.

    NOTE:

    Do not allocate the same agent host to multiple big data clusters with different Kerberos authentication configurations or authentication modes. Otherwise, the backup or restoration job fails.

    Certificate Verification

    This parameter is mandatory only when Authentication Mode is set to Kerberos Authentication. The Certificate Verification function is enabled by default.

    If you want to encrypt data during communication between the product and the big data platform for secure communication, import the certificate and enter the certificate password.

    Import the JKS certificate generated in (Optional) Step 2: Generating and Obtaining a Certificate. The certificate password is the keystore password set in (Optional) Step 2: Generating and Obtaining a Certificate.

    NOTE:
    • This configuration item can be enabled only when the big data platform is Cloudera CDH.
    • After the Certificate Verification function is disabled, security risks may exist in the communication between the system and the big data platform. Exercise caution when performing this operation.

    Security Protocol

    This parameter is mandatory only when Authentication Mode is set to Kerberos Authentication. The Security Protocol function is enabled by default.

    Whether to enable the TLS security protocol.

    • If the TLS security protocol is disabled, TLS 1.2 is used by default.
    • If the TLS security protocol is enabled, TLS 1.2 or later is supported.
      NOTE:
      • This configuration item can be enabled only when the big data platform is Cloudera CDH.
      • After the Security Protocol function is disabled, security risks may exist in the communication between the system and the big data platform. Exercise caution when performing this operation.

    Set Kerberos authentication parameters. For details about related parameters, see Table 2.

    Table 2 Kerberos authentication parameters

    Parameter

    Description

    Name

    User-defined Kerberos authentication name.

    Principal Name

    Principal name of Kerberos authentication.

    NOTE:
    • Set this parameter to the principal name that has been configured on the Kerberos server.
    • The principal name must have the SHOW_DATABASE permission on the database to be backed up, the SELECT permission on the table to be backed up, the CREATE and SHOW_DATABASE permissions on the database to be restored, and the CREATE, DROP, ALTER, and SELECT permissions on the table to be restored.
    • The principal name must have the read, write, and execution permissions on the temporary metadata storage path specified when the backup set is created, the read and execution permissions on the directory where the table data of the database to be backed up is located as well as the lower-layer files and directories, the read, write, and execution permissions on the temporary storage path of metadata specified during restoration, the read, write, and execution permissions on the directory where the data restored to the target database table is located, and the read, write, and execution permissions on the directory where the table metadata and table data are located.

    Configuration Mode
    Kerberos authentication mode, which must be the same as that configured on the Kerberos server. Two authentication modes are available:
    • Password: Use the principal name and password for identity authentication.
    • Keytab File: Use the principal name and keytab file for identity authentication.
      NOTE:

      If the big data platform is MRS, Hive cluster registration may fail when password authentication is used due to problems of some MRS versions. Therefore, the keytab file authentication mode is recommended.

    Config File

    .conf configuration file used during Kerberos authentication (for example, krb5.conf). Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the management page.

    If renew_lifetime exists in the configuration file, comment it out. Otherwise, the registration may fail. For example: 
    #renew_lifetime = 7d

    Password

    This parameter is mandatory only when Configuration Mode is set to Password.

    Set this parameter to the password of Principal Name.

    Confirm Password

    Confirm password of Principal Name.

    Keytab File

    .keytab file used during Kerberos authentication (for example, krb5.keytab). This parameter is mandatory only when Configuration Mode is set to Keytab File.

    Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the management page.

  3. Click OK.
