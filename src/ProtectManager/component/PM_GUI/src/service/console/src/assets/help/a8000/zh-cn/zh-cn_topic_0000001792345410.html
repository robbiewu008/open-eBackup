查看归档存储信息

通过浏览归档存储信息，可以了解归档存储的基本信息。

前提条件

已添加归档存储。

操作步骤

  1. 在“对象存储”页签，单击某个归档存储的“名称”，在右侧弹出的对话框中，查看该归档存储信息。

    参数说明如表1。

    表1 归档存储参数说明

    参数

    说明

    名称

    用户自定义归档存储的名称。

    类型

    归档存储的类型，当前仅支持S3协议的对象存储。

    对象存储产品

    接入OceanProtect的对象存储产品。

    支持OceanStor 100D(FusionStorage OBS)、OceanStor Pacific、华为云 OBS以及AWS S3。

    Endpoint

    归档存储的访问地址。

    地址格式：归档存储的域名或业务面IP地址。

    协议

    OceanProtect与归档存储之间的数据传输的协议。

    • 当使用“HTTPS”协议时，请导入“对象存储”类型的证书以便OceanProtect验证归档存储的信息。

      导入证书的具体操作请参见《OceanProtect 备份一体机 1.5.0-1.6.0 安全配置指南》中的“导入证书”章节。

      说明：
      • 归档存储为OceanStor Pacific时，获取证书的方法请参见《OceanStor Pacific系列 8.1.5 安全配置指南》的“导出内置CA证书”章节，导出“Service”证书。导出证书时，需要将证书后缀名修改为“.pem”。如果添加归档存储时，系统提示“CA证书已过期或无效”，其他参数配置已确认符合要求，请参考添加归档存储时，系统提示CA证书已过期或无效章节处理。
      • 归档存储为华为云OBS或FusionStorage OBS时，请联系归档存储的管理员获取证书。
      • 归档存储为AWS S3时，请联系厂商获取证书。
    • 当协议设置为HTTP时，OceanProtect将无法验证归档存储环境信息，将存在安全风险。建议您选择安全性高的HTTPS协议。

    AK
    对象存储服务支持通过AK/SK认证方式进行认证鉴权，即使用Access Key ID（AK）/Secret Access Key（SK）加密的方法来验证某个请求发送者身份。
    • Access Key Id（AK）：访问密钥ID。与私有访问密钥关联的唯一标识符；访问密钥ID和私有访问密钥一起使用，对请求进行加密签名。
    • Secret Access Key（SK）：与访问密钥ID结合使用的私有访问密钥，对请求进行加密签名，可标识发送方，并防止请求被修改。

    请联系归档存储的管理员获取AK/SK。

    SK

    数据桶

    存放归档数据的桶名。

    索引桶

    存放索引数据的桶名。

    使用代理服务器

    当OceanProtect使用代理服务器访问归档存储时，请配置代理服务器的地址及用户名、密码。

    说明：
    • 如需使用代理服务器，请确保代理服务器最大允许请求体大小大于100MB。
    • 代理服务器仅支持http协议，地址格式如下：http://代理服务器IP地址:端口号。

    容量告警阈值

    设置桶使用容量的最大阈值，当使用容量达到设定的阈值时上报告警。
