Step : Executing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication or archive policy has been defined in the SLA, the product will perform replication or archiving once based on the SLA when you perform manual backup.
  • If no log backup is defined in the SLA, log backup cannot be automatically executed.

Procedure

  1. Choose Protection > Databases > Kingbase.
  2. On the Instance tab, select the database to be protected and click Protect.

    You can also select multiple databases for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resource to be protected, select an SLA without a WORM policy to create protection to avoid WORM policy conflicts.

  4. Click OK.

    If the current system time is later than the start time of the first backup set in the SLA, you can choose to perform the backup immediately or wait for the next scheduling based on the backup policy set in the SLA after the backup SLA is associated with the resource.

  5. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.
    1. Click the Instance tab.
    2. In the row that contains the database to be backed up, choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

    3. Set the name of the copy generated by manual backup.

      If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.

    4. Select a protection policy, which can be Full backup, Log Backup, or Incremental backup.

      For 1.6.0 and later versions: If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    5. Click OK.
