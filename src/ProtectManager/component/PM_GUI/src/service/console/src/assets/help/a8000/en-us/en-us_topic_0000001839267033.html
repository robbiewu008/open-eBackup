Step 3: Configuring the Application-Consistent Backup Script

If the VMware VM runs on a non-Windows OS, you need to configure the pre- and post-processing scripts for application consistency backup on the protected VM before performing application consistency backup. In non-application-consistent backup scenarios, skip this section.

This section describes how to configure the application consistency backup scripts in the assumption that some database applications run on a VMware VM.

Table 1 Application-consistent backup scripts

Script

Database Type

Operation

Description
  • pre-freeze-script
  • post-thaw-script

DB2 database
  1. Log in to the protected VMware VM.
  2. Check whether the corresponding script exists in the /usr/sbin/ directory.
    • If no, create the script and configure the script by referring to the following sections.
    • If yes, check whether the script content is correct by referring to the following sections.

Before performing an application consistency backup and creating a consistency snapshot, the system invokes the pre-freeze-script script to quiesce database I/Os to ensure data consistency. After a consistency snapshot is created, the system invokes the post-thaw-script script to unquiesce database I/Os.
