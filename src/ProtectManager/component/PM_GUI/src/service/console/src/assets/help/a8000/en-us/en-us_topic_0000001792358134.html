Managing Informix Database Clusters

The OceanProtect allows you to perform operations, such as deletion and resource authorization or reclaiming, on the cluster where the database resides.

Related Operations

You need to log in to the management page, choose Protection > Databases > Informix, click the Cluster tab, and find the cluster where you want to perform related operations.

Table 1 describes the related operations.

Table 1 Operations related to a host or cluster

Operation

Description

Navigation Path

Modifying a cluster

Scenario

This operation enables you to refer to this section to determine whether to enable or disable log backup.

Choose More > Modify in the row where the cluster resides.

Deleting a cluster

Scenario

You can delete a host or cluster if its databases do not require protection. After the host or cluster is deleted, its information disappears from the management page.

Note

  • If a backup or restoration job is in progress, the cluster cannot be deleted. Wait until the job is complete.
  • A cluster in the Online status cannot be deleted.

Choose More > Delete in the row where the cluster resides.

Authorizing resources

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Note

  • Users cannot be authorized to protect resources that have been associated with SLAs. You need to remove the protection of resources before performing authorization.
  • A single resource can be authorized only to a single data protection administrator.

Choose More > Authorize Resource in the row where the target cluster resides, and select a data protection administrator to authorize resources.

Reclaiming resources

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

Users are not allowed to reclaim resources that have been associated with SLAs. You need to remove the protection of resources before reclaiming them.

Choose More > Reclaim Resource in the row where the cluster resides.

Test Connectivity

Scenario

This operation enables you to test the connectivity between the OceanProtect and the database.

Choose More > Test Connectivity in the row where the database resides.
