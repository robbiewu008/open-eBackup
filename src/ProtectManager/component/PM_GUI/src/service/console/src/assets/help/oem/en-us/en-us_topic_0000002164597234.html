A Backup Job Stops Running for a Long Time

Symptom

During RMAN-based backup, the backup job stops running for a long time, and the OS and kernel version of the host where the client is installed is Red Hat 7.6 3.10.0-957.el7.x86_64.

Possible Causes

The kernel issue of Red Hat 7.6 3.10.0-957.el7.x86_64 causes the process on the product to restart. As a result, the file system on the backup storage cannot be written, and the backup job is suspended.

Troubleshooting

  1. Use PuTTY to log in to the host where the client is installed.
  2. Run the following commands to check whether the host OS is Red Hat 7.6 and whether the kernel version is 3.10.0-957.el7.x86_64:

    cat /etc/redhat-release

    uname -r

    • If yes, go to 3.
    • If no, contact technical support.

  3. Download the Red Hat's kernel package.
  4. Upload the downloaded kernel package to the database host.
  5. Run the following command to install the new kernel package:

    rpm -ivh Kernel package

    The following uses kernel-3.10.0-957.10.1.el7.x86_64.rpm as an example. 
    rpm -ivh kernel-3.10.0-957.10.1.el7.x86_64.rpm

  6. After restarting the OS, run the following command to check whether the current kernel version is the new one:

    uname -r
