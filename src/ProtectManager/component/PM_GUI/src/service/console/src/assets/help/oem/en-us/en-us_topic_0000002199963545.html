A Snapshot-based Full Backup Job at the Storage Layer Fails, Indicating that LUN Snapshots Fail to Be Created

Symptom

An Oracle snapshot-based full backup job at the storage layer fails, with an error message displayed indicating that LUN snapshots fail to be created, and the error details contain the LUN ID.

Possible Causes

According to the error details, the number of protection groups to which the to-be-backed-up LUN has been added reaches the upper limit (a LUN can be added to a maximum of eight protection groups).

Troubleshooting

  1. Log in to DeviceManager of the OceanStor Dorado storage system in the production environment.
  2. In the navigation bar, choose Services > Block Service > LUN Groups. Switch to the LUNs tab page and locate the LUN corresponding to the ID in the error details.

    If the error details contain multiple LUN IDs, perform the following steps for the LUN corresponding to each ID.

  3. Click the LUN name to go to the LUN details page.
  4. Switch to the Protection tab page and select Snapshot. On the snapshot details page that is displayed, obtain the value of Owning Snapshot CG.
  5. In the navigation bar, choose Data Protection > Protection Entities > Protection Groups. Switch to the Snapshot CGs tab page and locate the value of PG corresponding to Owning Snapshot CG obtained in 4.
  6. Switch to the Protection Groups tab page, locate the PG obtained in 5, and click its name. Switch to the Members tab page and select the LUN name obtained in 2.

    Ensure that the selected LUN can be removed from the protection group. If the prefix of a protection group name is protect_group, the protection group may be automatically created by a backup job. Contact technical support engineers to check whether the protection group can be removed.

  7. Click Remove.
