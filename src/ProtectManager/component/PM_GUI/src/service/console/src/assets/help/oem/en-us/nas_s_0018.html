Periodically Executing Backup

Associate the NAS file system to be protected with a specified SLA. The system periodically backs up these resources based on the SLA definition.

Procedure

  1. Choose Protection > File Systems > NAS File Systems.

    For 1.5.0, choose Protection > File Services > NAS File Systems.

  2. In the row where a NAS file system to be protected resides, choose More > Protect .
  3. Select the created SLA.

    You can also click Create to create an SLA.

  4. Set the share protocol.

    Specify the share protocol used for backing up a file system. Ensure that the share protocol selected during restoration is the same as that specified during backup. Otherwise, the metadata and permission information of the file system will be lost during restoration.

    If a share protocol is set as the CIFS protocol, ensure that a NAS file system to be protected on the production storage is in the NTFS security mode. Otherwise, backup fails.

  5. Select the agent host for executing the backup job.
  6. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
