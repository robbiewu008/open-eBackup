Querying Information About the VMware Virtualization Environment

After the VMware virtualization environment is registered with the product, you can view details about clusters, vCenter Servers or ESXi hosts, VMware VMs, or VM groups on the management page.

Context

GUIs may vary with versions. The actual GUIs prevail.

Procedure

  1. Choose Protection > Virtualization > VMware.

    Figure 1 shows the overall information about the VMware virtualization environment.

    Figure 1 Overall information about the VMware virtualization environment

  2. On the left of the page, select the VMware virtualization environment to be viewed and choose More > Environment Info to view the basic information of the VMware virtualization environment.
  3. On the Clusters tab page, view the cluster information.

    Table 1 lists the details about the cluster.
    Table 1 Cluster information

    Parameter

    Description

    Name

    Cluster name.

    SLAs

    If the entire cluster is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    Protection Status

    Indicates whether a cluster is protected.

    • Unprotected: The cluster is not associated with an SLA, or the cluster is associated with an SLA but protection is disabled.
    • Protected: The cluster is associated with an SLA and protection is activated.
    • Creating: The cluster is associated with an SLA and protection is being created.

    VM Count

    Number of VMs in the current cluster.

    Click the number of VMs. The host list, VM list, and virtual device nodes in the cluster are displayed on the page.

  4. On the Hosts tab page, view the ESXi host information.

    Table 2 lists the details about ESXi hosts.
    Table 2 Host information

    Parameter

    Description

    Name

    ESXi host name.

    Click the host name. The VM list and virtual device nodes are displayed.

    If the ESXi host belongs to a cluster, the VM list is not displayed.

    The virtual device node is the number of the slot where the virtual disk is located. It is represented in the format of "disk interface type (bus number: slot number)", for example, IDE(0:0).

    Location

    Path to the ESXi host.

    Status

    ESXi host status, which can be Online or Offline.

    SLAs

    If an SLA is associated with a host, the SLA name is displayed. Otherwise, -- is displayed.

    Protection Status

    Whether a host is protected.

    • Unprotected: The host is not associated with an SLA, or the host is associated with an SLA but protection is disabled.
    • Protected: The host is associated with an SLA and protection is activated.
    • Creating: The host is associated with an SLA and protection is being created.

    VM Count

    Number of VMs on an ESXi host.

    If the host is a cluster member, the number of VMs on the host is displayed as 0. You need to view the number of VMs on the Cluster tab page.

  5. On the VMs tab page, view the VM information.

    1. View information about a VM. Table 3 describes the information.
      Table 3 VM information

      Parameter

      Description

      Name

      Name of a VM.

      Location

      Path to a VM.

      Status

      VM status, which can be power-on or power-off.

      OS

      Operating system type of the host where the VM is located.

      Mark

      Mark of a VM.

      If you have set a mark for a VM in the VMware virtualization environment, the VM mark is displayed. Otherwise, -- is displayed.

      You can filter VMs by mark.

      SLAs

      If an SLA is associated with a VM, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether a VM is protected.

      • Unprotected: The VM is not associated with an SLA, or the VM is associated with an SLA but protection is disabled.
      • Protected: The VM is associated with an SLA and protection is activated.
      • Creating: The VM is associated with an SLA and protection is being created.

      VM Group

      Name of the VM group to which the VM belongs.
    2. Click the VM name to view the overall VM information, copy information, and related jobs.
      • VM overview

        You can view information of a VM, such as the VM name, SLA information, job execution time, total number of copies, and disk information.

      • VM copy data

        You can search for copies by year, month, or day.

        On the Year or Month tab page, indicates that copies exist at the corresponding time.

      • Jobs related to a VM

        All jobs related to a VM, such as backup, restoration, archive, and live mount jobs, are displayed.

  6. On the VM Groups tab page, view VM group information.

    Only 1.6.0 and later versions support this function.

    Table 4 describes the information of VM groups.
    Table 4 VM group information

    Parameter

    Description

    Name

    VM group name.

    VM Count

    Number of VMs in the current VM group.

    SLAs

    If the entire VM group is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a VM group is protected.

    • Unprotected: The VM group is not associated with an SLA, or the VM group is associated with an SLA but protection is disabled.
    • Protected: The VM group is associated with an SLA and protection is activated.
    • Creating: The VM group is associated with an SLA and protection is being created.
