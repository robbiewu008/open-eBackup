Step 5: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • During manual backup, the parameters (except Automatic Retry) defined in the SLA are applied to the manual backup job.
  • Copies generated by manual backup are retained for the period defined in the SLA.

Procedure

  1. Choose Protection > File Systems > Common Shares.
  2. Select the target common share resource and click Protect.

    You can also select multiple common share resources for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

  4. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  5. Execute manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. Choose More > Manual Backup in the row that contains the target common share resource.
      1. You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup.
    2. Set the name of the copy generated through manual backup.
    3. If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.
    4. Select a protection policy, which can be full backup or incremental backup.
    5. Click OK.
