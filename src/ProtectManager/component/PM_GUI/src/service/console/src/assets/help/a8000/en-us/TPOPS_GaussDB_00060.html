Viewing GaussDB Information

After GaussDB is registered with the OceanProtect, you can view the project and instance information of GaussDB.

Procedure

  1. Choose Protection > Databases > GaussDB.

    For 1.5.0, choose Protection > Cloud Platforms > GaussDB.

  2. On the Project tab page, view project information.

    Table 1 describes the detailed project information.
    Table 1 Project information

    Parameter

    Description

    Name

    Project name.

    Project Address

    IP address used by TPOPS to provide services for external systems.

    Status

    Project status.

    • Online: GaussDB is properly connected to the OceanProtect.
    • Offline: GaussDB is disconnected from the OceanProtect.

    Allow Restoration to the Resource or Not

    NOTE:

    Only 1.5.0.SPC7 and later versions support this parameter.

    Whether the project and instances in the project can be used as the target location for restoration. The default value is No.

    NOTE:

    In the Operation column, choose More > Allow Restoration/Disallow Restoration.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

    Owner

    Data protection administrator to which resources are allocated for management. If -- is displayed, the resource has not been allocated to the data protection administrator.

  3. On the Instance tab page, view the instance information.

    Table 2 describes the detailed instance information.
    Table 2 Instance information

    Parameter

    Description

    Name

    Instance name.

    Click an instance name to view the overview, copy data, and jobs of the instance.

    • Overview

      The name, SLA information, job execution time, and total number of copies are displayed.

    • Copy Data

      Copy data of an instance.

    • Jobs

      All jobs, including backup and restoration jobs, are displayed.

    Project Address

    IP address used by TPOPS to provide services for external systems.

    Status

    Project status.

    • Online: GaussDB is properly connected to the OceanProtect Appliance.
    • Offline: GaussDB is disconnected from the OceanProtect Appliance.

    Owning Project

    Project to which an instance belongs.

    SLAs

    If an instance is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
    • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether the instance is protected.

    • Unprotected: The instance is not associated with an SLA, or the instance is associated with an SLA but protection is disabled.
    • Protected: The instance is associated with an SLA and protection is activated.
    • Creating: The instance is associated with an SLA and protection is being created.

    Allow Restoration to the Resource or Not

    NOTE:

    Only 1.5.0.SPC7 and later versions support this parameter.

    Whether the instance can be used as the target instance for restoration.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
