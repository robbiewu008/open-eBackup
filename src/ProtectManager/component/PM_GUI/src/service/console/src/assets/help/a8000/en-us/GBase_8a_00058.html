Managing Databases

The OceanProtect allows you to modify, remove, activate, or disable protection for databases.

You need to log in to the management page, choose Protection > Databases > General Databases, and find the target database.

Table 1 describes the related operations.

Table 1 Operations related to database protection plans

Operation

Description

Navigation Path

Modifying protection

Scenario

You can modify the protection plan to re-associate a database with an SLA. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target database, choose More > Modify Protection.

Removing protection

Scenario

You can remove the protection if the database no longer needs to be protected. After the removal, the system cannot execute protection jobs for the database. To protect the database again, you need to associate an SLA with the database again.

Note

When the protection job of the database is running, the protection cannot be removed.

In the row of the target database, choose More > Remove Protection.

Activating protection

Scenario

If the protection plan of a database is disabled, you can perform this operation to activate it again. After the protection plan is activated, the system executes backup, archive, or replication jobs for the database based on the plan.

In the row of the target database, choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable a protection plan when periodic protection on the database is not required. After the protection plan is disabled, the system does not automatically back up, archive, or replicate the database data.

Note

Disabling a protection plan does not affect protection jobs that are running.

In the row of the target database, choose More > Disable Protection.

Performing a manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target database, choose More > Manual Backup.
Authorizing resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

In the row of the target database, choose More > Authorize Resource, and select a data protection administrator to authorize resources.
Reclaiming resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

In the row of the target database, choose More > Reclaim Resource.

Testing connectivity

Scenario

This operation enables you to test the connectivity between the OceanProtect and the database.

In the row of the target database, choose More > Test Connectivity.

Modifying database registration information

Scenario

After the database username or password is changed, you can perform this operation to modify the database registration information.

In the row of the target database, choose More > Modify.

Deleting a database

Scenario

This operation enables you to delete a database if you no longer need to protect it.

Note

A database that has been associated with an SLA cannot be deleted.

In the row of the target database, choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for restoring GBase 8a databases is the same as that described in Restoration.
