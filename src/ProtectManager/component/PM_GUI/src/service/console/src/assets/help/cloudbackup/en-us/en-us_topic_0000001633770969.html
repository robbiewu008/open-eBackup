Replacing the DeviceManager Certificate

It applies to the scenario where an external client communicates with the storage device. In this scenario, the storage device serves as a server while the external network management tool serves as a client. The certificate must be imported, but the CA certificate is optional.

Context

  • In the device management certificate scenario, the server has a default certificate and does not support the certificate revocation list (CRL).
  • To enhance security of links in device management certificate scenarios, you are advised to replace the default security certificates and private keys of the network management tool client and storage device with your security certificates and private keys.

Procedure

  1. Obtain the CSR file and private key.

    The following two methods are supported:
    • On DeviceManager, export the CSR file in the device management certificate scenario. In this case, the corresponding private key is generated on the storage device and saved to the database.

      For details, see how to export the CSR fi....

    • Use the OpenSSL tool to generate a private key in plaintext and a CSR file.

      For details, see Use the OpenSSL tool to ....

  2. Send the exported CSR file to the third-party CA center. After the CA center signs it or you use your enterprise's root certificate to sign it, the corresponding certificate and CA certificate are generated.

    For details about common third-party CA centers, see Table 2.

  3. Import the certificate file.

    • If the CSR file is generated on DeviceManager, import the signed certificate to the storage device.
    • If the CSR file is generated using the OpenSSL tool, import the plaintext private key file and signed certificate to the storage device on DeviceManager.

    For details about how to import a certificate, see 3. Change the operation of selecting the management plane domain authentication certificate to selecting the DeviceManager certificate.
