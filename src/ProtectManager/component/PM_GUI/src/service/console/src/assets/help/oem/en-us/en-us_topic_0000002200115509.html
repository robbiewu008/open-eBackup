Configuring SNMP Trap Notifications on the Product WebUI

Context

Trap is a type of SNMP message that indicates the occurrence of an event. To send alarm notifications by SNMP, you must configure a trap server address. After the trap server address is set, alarm information of the product will be sent to the third-party network management tool specified by the trap.

Prerequisites

The system administrator has logged in to the management interface.

Procedure

  1. In the navigation pane, choose System > Settings > SNMP Trap.
  2. In the Trap Parameter Settings area, configure SNMPv2c or SNMPv3 parameters.

    There will be security risks in enabling SNMPv2c. You are advised to use the secure SNMPv3 protocol.

    • Set SNMPv3 parameters.
      1. Click Modify.
      2. Set Protocol Version to SNMPv3.

        If you need to configure an engine ID for SNMP of a third-party NMS, click View SNMPv3 Engine ID next to Protocol Version and configure the engine ID based on the controller SNMPv3 engine ID displayed on the page.

      3. Set SNMPv3 parameters. Table 1 describes the related parameters.
        Table 1 Parameter description

        Parameter

        Description

        Username

        Username.

        The value consists of 1 to 64 characters.

        Authentication Protocol

        Authentication algorithms of the user, including None, HMAC_MD5, HMAC_SHA1, and HMAC_SHA2.

        Default value: HMAC_SHA2.

        NOTE:

        HMAC_SHA is more secure than HMAC_MD5. You are advised to use HMAC_SHA2.

        Authentication Password

        Authentication password of the user.

        Confirm Authentication Password

        Authentication password of the user.

        Data Encryption Protocol

        Encryption protocols of the user, including None, DES, and AES.

        Default value: AES

        NOTE:

        AES is more secure than DES. For security purposes, you are advised to use AES.

        Data Encryption Password

        Data encryption password used by the user.

        NOTE:

        Data Encryption Password and Authentication Password must be different.

        Confirm Data Encryption Password

        Data encryption password used by the user.

        Environment Name

        Name of the environment engine.

        Environment Engine ID

        Unique identifier of an SNMP engine. This ID is used together with Environment Name to identify the environment of an SNMP entity.

        An SNMP message packet is processed only when the environment of the transmitter fully matches that of the receiver. Otherwise, the SNMP message packet is discarded.
      4. Click Save.
    • Set SNMPv2c parameters.
      1. Click Modify.
      2. Set Protocol Version to SNMPv2c.
      3. Set the community name. Table 2 describes the related parameters.
        Table 2 Parameters for modifying a community name

        Parameter

        Description

        Community Name

        Name of the community that has the read and write permissions.

        NOTE:

        In SNMP, community names are used to authenticate and authorize SNMP agent access users. You can configure the SNMP community name to ensure that only authorized users can access and manage the server.
      4. Click Save.

  3. In the Trap Server area, click Add and configure the trap server. Table 3 describes the related parameters.

    Table 3 Trap server configuration parameters

    Parameter

    Description

    IP Address

    Currently, only IPv4 addresses are supported.

    Port

    Port used by the trap server to listen to trap messages.

    Description

    Trap server description.

Follow-up Procedure

In the Trap Server area, you can view Trap Server information and delete or change a trap IP address.

The product can be connected to DME. For details, see Operations for Connecting to DME. For details about the version mapping, see the OceanProtect Appliance 1.5.0-1.6.0 Version Mapping Table.
