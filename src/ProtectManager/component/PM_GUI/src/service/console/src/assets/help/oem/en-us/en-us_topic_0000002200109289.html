Managing Anonymization Policies

You can view, modify, delete, and clone anonymization policies.

Precautions

The preset anonymization policies cannot be modified or deleted.

If a customized anonymization policy has been associated with a database, the system automatically disassociates the policy from the database after data anonymization is complete.

Management Operations

Table 1 describes the operations for managing anonymization policies.

Table 1 Operations for managing anonymization policies

Task

Description

Navigation Path

Viewing anonymization policies

You can view the quantity and details of anonymization policies.
In the navigation pane, choose Data Security > Data Anonymization > Anonymization Policies to view the name, description, rule source, and number of associated objects of each anonymization policy. Anonymization policies can be displayed in thumbnails or a list.
NOTE:

For 1.5.0, choose Explore > Data Utilization Policies in the navigation pane and click Anonymization Policies.

Click an anonymization policy to view its details, such as the name, description, associated identification rules, and associated anonymization rules.

You can click the number of associated objects to view their information.

Modifying anonymization policies

You can modify the name, description, and identification rule of a custom anonymization policy.

In the Operation column on the right of the row containing the anonymization policy, choose More > Modify.

Deleting anonymization policies

You can delete a custom anonymization policy if it is no longer required.

In the Operation column on the right of the row containing the anonymization policy, choose More > Delete. You can delete multiple custom anonymization policies in batches.

Cloning anonymization policies

If an anonymization policy to be created is similar to an existing policy, you can clone and alter the existing policy to create one.

In the Operation column on the right of the row containing the anonymization policy, choose More > Clone.

If the anonymization policy to be cloned is a preset policy, click Clone.
