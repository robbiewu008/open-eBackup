Anonymizing Data in an Oracle Database

This section describes how to use the data anonymization feature to anonymize data in a restored or mounted Oracle database.

Prerequisites

  • You have obtained the management IP address and name of the database to be anonymized.
  • You have obtained the user information (user name and password) of the database tablespace to be anonymized, and the user has the remote access permission.
  • You have obtained the service IP address and port number of the database to be anonymized.
  • The database listener has been configured. For details about how to configure database listening, see "Configuring Oracle Database Listening Parameters" in the OceanProtect Appliance 1.5.0-1.6.0 Data Backup Feature Guide (for Oracle).

Precautions

  • Do not perform any data anonymization operations on the production database.
  • Do not operate the database during sensitive data identification and anonymization.

Identifying Sensitive Data

  1. On the top navigation bar, choose Explore > Data Anonymization. On the left navigation pane, click Oracle.
  2. On the right Oracle page, search for the database to be anonymized based on its management IP address and name.
  3. In the Operation column of the database to be anonymized, choose More > Identify Sensitive Data.
  4. In the Identify Sensitive Data dialog box, select a data anonymization policy and click OK.

    In case a large number of anonymization policies exist, you can enter a policy name to search. Fuzzy search is supported. Click on the right to display anonymization policies in thumbnails or a list.

    If the existing anonymization policies do not meet your requirements, you can click Create to add a new policy. For details about the parameters related to anonymization policies, see Table 1.

  5. In the Authenticate window, enter the Oracle database authentication information and click OK.

    Enter the authentication information based on the user information (user name and password) of the database tablespace, service IP address, and port number obtained in the prerequisites. The default port number of the Oracle database is 1521.

  6. Read the information and click OK to start identifying sensitive data.

    It may take a long time to complete sensitive data identification. When the database status is Identifying, do not perform any operation on the database.

    If Identification Status of the database is displayed as Identified, the identification is successful.

    If Identification Status of the database is displayed as Identification failed, click the database name. On the displayed page, click the Jobs tab, and then click the Historical Jobs tab. View the failure cause, and rectify the fault based on the handling suggestion.

Anonymizing Sensitive Data

  1. In the Operation column of the database to be anonymized, choose More > Anonymize.
  2. In the Anonymize window, click the database tablespace and database table to view the identification result.

    If (xxx) is displayed on the right of a column name in a database table, the corresponding column contains sensitive data. xxx indicates the name of an identification rule.

    You can verify or modify the identification result.

    • Verification

      Verify whether the identification result is correct.

      Select one to three columns of identification results for a single database table and click Verify. In the dialog box that is displayed, enter the username, password, service IP address, port number, start location, and maximum number of rows of the database, and click OK. In the displayed Verification Result window, you can check whether the identification results are correct. If the identification results are incorrect, you can modify them.

    • Modification

      If the identification result is regarded as incorrect or does not meet your requirements, you can modify the identification result or the anonymization rule.

      Click on the right of a column name. In the Modify dialog box, enable or disable Sensitive. After enabling the switch, you also need to set Sensitive Data Name and select a new anonymization rule. Click OK to complete the modification.

      If the anonymization rule Partial-Mask is selected, you also need to enter the start and end positions of sensitive data in this column.

  3. Click OK.
  4. In the Authenticate window, enter the Oracle database authentication information and click OK.

    Enter the authentication information based on the user information (user name and password) of the database tablespace, service IP address, and port number obtained in the prerequisites. The default port number of the Oracle database is 1521.

  5. Read the message in Danger, select I have read and understand the consequences associated with performing this operation., and click OK to start data anonymization.

    The anonymized data will be written into the database. Ensure that the database to be anonymized is a non-production database.

    Do not operate the database during data anonymization.

    If Anonymization Status of the database is displayed as Anonymized, data anonymization is successful.

    If Anonymization Status of the database is displayed as Anonymization failed, click the database name. On the displayed page, click the Jobs tab, and then click the Historical Jobs tab. View the failure cause, and rectify the fault based on the handling suggestion.

Viewing the Anonymization Report

  1. In the Operation column of the database whose data has been anonymized, choose More > View Anonymization Report.

    In the Anonymization Report window, view the anonymization result. If (xxx) is displayed on the right of a column name in a database table, the corresponding column contains sensitive data. xxx indicates a sensitive data name.

  2. You can also click Download Report to save the anonymization report.

    The anonymization report, which is in CSV format, contains the ID, database ID, database name, database tablespace, database table name, column name, identification rule name, and anonymization rule type.
