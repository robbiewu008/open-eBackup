When a Backup or Restoration Job Is Being Executed, "The job is waiting to be scheduled by DMC." Is Displayed in the Job Details

Symptom

When a backup or restoration job is being executed, "The job is waiting to be scheduled by DMC." is displayed in the job details and exists for more than 10 minutes.

Possible Causes

  • Cause 1: The VDDK is not installed on the host where the client is installed or the version of the installed VDDK is incorrect.
  • Cause 2: The number of running jobs reaches the upper limit.

Troubleshooting

  1. Log in to the host where the client is installed as user root using PuTTY.

    • If the client for job execution is specified, log in to the host where the client is installed and check whether the VDDK has been installed.
    • If no client is specified for job execution, log in to all hosts where the client is installed to check whether the VDDK has been installed.

  2. Run the following command to check whether the VDDK of the required version has been installed:

    cd /opt/DataBackup/ProtectClient/ProtectClient-E/lib

    In the preceding command, /opt indicates the client installation directory. Replace it with the actual directory.

    • If the vddk directory does not exist or the VDDK of the required version is not installed, install it as instructed in "Adding a VDDK (Applicable to VMware)" of the OceanProtect DataBackup 1.5.0-1.6.0 Client Installation Guide.
    • If the vddk directory exists and the VDDK of the required version has been installed, the fault may be caused by cause 2. In this case, wait for job scheduling or contact technical support engineers.
