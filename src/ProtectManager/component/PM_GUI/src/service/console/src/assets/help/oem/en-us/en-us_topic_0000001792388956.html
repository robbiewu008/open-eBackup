Manually Performing Backup

If you want to perform a backup immediately, you can manually perform the backup.

Prerequisites

The prerequisites for the GaussDB T database backup in a single-node system are as follows:

  • You need to ensure that the GaussDB T database is running properly.
  • If full backup or log backup is required, you need to ensure that log archiving has been enabled before backup.

The prerequisites for backing up the GaussDB T cluster database are as follows:

The database needs to run in ARCHIVELOG mode and archive redo logs to multiple locations. For details about how to query and change the log archive mode, see Step 2: Setting the Redo Log Mode.

Precaution

After a full backup job fails, if the next backup is a log backup, the log backup job will fail because log backup depends on the copy generated in the last successful full backup. In this case, manually perform a full backup once.

Context

  • During the execution of a manual backup job, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, will take effect.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

Procedure

  1. Choose Protection > Databases > GaussDB T.
  2. Click More > Manual Backup in the row containing the database to be backed up. Multiple resources can be manually backed up at the same time.
  3. Set the name of the copy generated during manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  4. Select a protection policy, which can be Full backup, Incremental backup, Differential Backup, or Log Backup.

    Incremental backup and Differential Backup apply only to GaussDB T cluster backup scenarios.

  5. Click OK.
