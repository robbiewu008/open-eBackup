Creating a Report

The product provides the report function, through which you can view the key resource status (such as storage capacity usage) and running status of key jobs (such as backup and restoration jobs). This section describes how to create a report.

Procedure

  1. Select Reports.
  2. Click Create. Table 1 describes the parameters.

    Table 1 Parameters for creating a report

    Parameter

    Description

    Report Name

    User-defined name of the report.

    Report Type

    Type of the report to be created.

    • Storage space report: Collects the total capacity, used capacity, and unused capacity of the product.
    • Backup job report: Collects backup job information.
    • Restoration job report: Collects restoration job information.
    • Restoration drill job report: Collects restoration drill job information.
    • Host resource usage report: Collects host resource usage information.
    • Resource usage report: Collects resource usage information.
    • Tape usage report: Collects tape usage information.
    NOTE:
    • 1.6.0 and later versions support the creation of Host resource usage report, Resource usage report, and Tape usage report.
    • For 1.6.0 and later versions, Backup job report is displayed as Protection job report.
    • For 1.6.0 and later versions, Protection job report contains information about backup, archive, and replication jobs. For versions earlier than 1.6.0, Backup job report contains only backup job information.
    • For the product, if you want to create a storage space report, ensure that the performance monitoring function has been enabled for the corresponding node on the Performance page.

    Data Source

    Select local storage or an external cluster that has been connected to the product.

    NOTE:

    For 1.6.0 and later versions, during creation of a protection job report, restoration job report, restoration drill job report, or tape usage report: if you leave Data Source unspecified, the generated report will contain all job information of all nodes; if you specify Data Source, the generated report will contain only information of jobs (except those failing to be distributed) distributed to the selected nodes.

    Statistical Range

    Time range within which data is collected. This parameter is displayed only when Report Type is Backup job report, Restoration job report, or Restoration drill job report.

    • Last week: Data in the recent week is collected.
    • Last month: Data in the recent month is collected.
    • Last three months: Data in the recent three months is collected.
    • Custom: The start time and end time for data collection are specified.
      NOTE:
      • The time range cannot exceed 365 days.
      • For 1.6.0 and later versions, Backup job report is displayed as Protection job report.

    Statistical Interval

    Data collection interval of the report. This parameter is displayed only when Report Type is Backup job report, Restoration job report, or Restoration drill job report. The system sets a default interval based on the statistical range you select.

    • If the statistical range is shorter than or equal to 10 days, the statistical interval is set to 1 day.
    • If the statistical range is longer than 10 days but shorter than or equal to 30 days, the statistical interval is set to 5 days.
    • If the statistical range is longer than 30 days but shorter than or equal to 90 days, the statistical interval is set to 10 or 30 days.
    • If the statistical range is longer than 90 days but shorter than or equal to 365 days, the statistical interval is set to 30 days.
    NOTE:

    For 1.6.0 and later versions, Backup job report is displayed as Protection job report.

    Report Format

    Format of the report. In the current version, optional values are xls and pdf.
