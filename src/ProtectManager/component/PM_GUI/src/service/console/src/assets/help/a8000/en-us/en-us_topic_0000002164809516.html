Viewing Exchange Environment Information

You can view details about the Exchange resources on the WebUI after Exchange single-node systems or availability groups are registered with the OceanProtect.

Procedure

  1. Choose Protection > Applications > Exchange.
  2. Click the corresponding tab to view the information about the related resources.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    • Click the Single-Node Systems/Availability Groups tab. The registered Exchange single-node systems or availability groups are displayed.
      1. Table 1 lists the details about an Exchange single-node system or availability group.
        Table 1 Information about Exchange single-node systems/availability groups

        Parameter

        Description

        Name

        Database name.

        Status

        Exchange Server status.

        • Online
        • Offline

        Type

        Exchange deployment type.

        • Single-node system
        • Availability group

        SLAs

        If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

        SLA Compliance
        Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
        • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
        • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

        Protection Status

        Whether a database is protected.

        • Unprotected: The database is not associated with an SLA, or it is associated with an SLA but protection is disabled.
        • Protected: The database is associated with an SLA and protection is activated.
        • Creating: The database is associated with an SLA and protection is being created.

        Tag

        If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      2. Click the name of a resource to view the overview, copy, and job information.
        • Overview

          Displays the single-node system or availability group name, SLA information, job execution time, total number of copies, node information, and database information.

        • Copy Data

          You can search for copies by year, month, or day.

          If is displayed below the time, copies exist at that time.

        • Jobs

          Displays all jobs, including backup, archive, and restoration jobs. You can choose to display ongoing or historical jobs.

    • On the Databases tab page, view the database information.
      The information about scanned databases is displayed.

        Table 2 Database parameters

        Description

        Tag

        If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      2. Click the name of a database resource to view the overview, copy, and job information.
        • Overview

          Displays the database resource name, SLA information, job execution time, and total number of copies.

        • Copy Data

          You can search for copies by year, month, or day.

          If is displayed below the time, copies exist at that time.

        • Jobs

          Displays all jobs, including backup, archive, and restoration jobs. You can choose to display ongoing or historical jobs.

    • On the Mailboxes tab page, view information about mailboxes.
      Information about scanned mailboxes is displayed.
