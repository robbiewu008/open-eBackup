Step 7: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication policy has been defined in the SLA, the product will perform replication once based on the SLA when you perform manual backup.
  • If no log backup is defined in the SLA, log backup cannot be automatically executed.

Procedure

  1. Choose Protection > Databases > Informix/GBase 8s.
  2. On the Instance tab, select the database to be protected and choose More > Protect.

    You can also select multiple databases for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

  4. Set advanced parameters for the backup policy.

    Table 1 describes the related parameters.
    Table 1 Advanced parameters of a backup policy

    Parameter

    Description

    Rate Limiting Policies

    Average maximum amount of data that can be transmitted per second in a backup job.

    Source Deduplication
    After this function is enabled, the amount of data to be transmitted can be reduced, but more client resources will be occupied.
    NOTE:
    • An agent host can only be linked with one storage unit. To enable source deduplication, select Backup Storage Unit for Specify Target Location. Otherwise, a storage unit that is not linked with the agent host may be selected through intelligent balancing. As a result, source deduplication fails to be enabled. If different protected objects are associated with different SLAs, where different backup storage units are specified, select different agent hosts for the protected objects. Otherwise, source deduplication of some backup jobs will fail to be enabled.
    • If an agent host has been linked with a storage unit but is not performing a backup job with source deduplication, you can run the dataturbo delete storage_object storage_name=? command on the agent host to delete the source deduplication link with the storage unit. The value of storage_name in the command can be queried by running the dataturbo show storage_object command. This way, when the agent host performs a backup job with source deduplication, the link between the agent host and a new storage unit can be established.
    • Check whether the host OS supports source deduplication by referring to OceanProtect Compatibility Query. If the host OS does not support source deduplication, source deduplication does not take effect after being configured.
    • If a built-in agent (that is, ProtectAgent is co-deployed on the product) is used for backup, source deduplication does not take effect after being configured.

    Job Failure Alarm

    After this function is enabled, an alarm is automatically sent if a job fails and is automatically cleared after successful execution.

    Automatic Retry

    After this function is enabled, the system automatically retries a backup job if it fails.

    The number of retries ranges from 1 to 5, and the wait time ranges from 1 to 30 minutes. For example, if the number of retries is set to 3 and the wait time to 5 minutes, the system makes 3 retries and each retry every 5 minutes.
    NOTE:

    During automatic retry, the system creates a backup job. If the backup job is not within the backup time window, the job is not executed and fails.

  5. Click OK.

    If the current system time is later than the start time of the first backup set in the SLA, you can choose to perform the backup immediately or wait for the next scheduling based on the backup policy set in the SLA after the backup SLA is associated with the resource.

  6. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.
    1. In the row of the target resource, choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple protected resources and choose More > Manual Backup in the upper left corner of the resource list.

    2. Set the name of the copy generated during manual backup.

      If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup or Forever Incremental (Synthetic Full) Backup.

      For 1.6.0 and later versions: If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    4. Click OK.
