Restoring GoldenDB

This section describes how to use copies to restore a GoldenDB instance that has been backed up to the original or a different host.

Prerequisites

Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.

Procedure

  1. Choose Explore > Copy Data > Databases > GoldenDB.
  2. Search for copies by GoldenDB resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the resource to be restored by resource name and click the resource name.

  3. You can search for copies by resource or copy.

    • On the Resources page, find the instance to be restored based on the name, and then click the instance name.
    • On the Copy page, find the target copy based on the copy information, and choose More > Restore in the row where the copy is located.

    This section uses the database resource dimension as an example to describe the related operations.

  4. Select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, copies exist in the month or on the day.

  5. Choose More > Restore in the row where the copy is located.

    Table 1 describes the related parameters.

    Table 1 Parameters for restoring a GoldenDB instance

    Parameter

    Description

    Restore To

    Select either Original location or New location.

    Location

    If you select Original location, the target location is displayed by default.

    Target Cluster

    New location: Data will be restored to the original cluster by default.

    Target Instance

    New location: Select the target instance for the restoration.

  6. Click OK.
