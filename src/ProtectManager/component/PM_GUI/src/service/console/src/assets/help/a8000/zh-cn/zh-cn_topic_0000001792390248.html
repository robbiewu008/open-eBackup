开启nginx端口的输入输出功能（适用于部分应用）

当代理主机开启了防火墙功能，请参考本节打开nginx侦听端口的输入输出功能，nginx侦听端口是OceanProtect与代理主机的通信端口。本节以Red Hat Enterprise Linux Server release 7.2版本为例进行说明。如果未开启防火墙或者受保护资源不是FusionCompute、Oracle、HDFS、HBase、Hive、Elasticsearch、ClickHouse、Redis、PostgreSQL、MySQL/MariaDB/GreatSQL、GaussDB T、GaussDB(DWS)、华为云Stack GaussDB、openGauss、Dameng、对象存储、NAS文件、SAP HANA、Kingbase、MongoDB、TDSQL、GBase 8a、TiDB、Informix/GBase 8s、NDMP NAS文件、GaussDB、GoldenDB、OceanBase、华为云Stack、Kubernetes FlexVolume、OpenStack、DB2、FusionOne Compute或文件集，请忽略本节。

操作步骤

  1. 使用PuTTY工具以root账户登录代理主机。
  2. 执行以下命令查看是否已开启防火墙。

    systemctl status firewalld

    当回显显示“Active: active (running)”时，表示已开启防火墙，继续执行后续步骤。否则结束处理。

  3. 执行以下命令查看nginx侦听端口。

    netstat -anp | grep nginx
    示例如下，“59521”为nginx的侦听端口。
    [root@localhost ~]# netstat -anp | grep nginx
tcp        0      0 192.168.97.167:59521       0.0.0.0:*               LISTEN      2483/nginx: master
unix  3      [ ]         STREAM     CONNECTED     20149    2483/nginx: master
unix  3      [ ]         STREAM     CONNECTED     20148    2483/nginx: master

  4. 执行以下命令查询已开启输入输出功能的端口列表中是否包含nginx侦听端口。

    firewall-cmd --permanent --zone=public --list-ports

    回显中包含nginx侦听端口，则处理结束。否则继续执行后续步骤。

  5. 执行以下命令开启nginx侦听端口的输入输出功能。

    以端口号“59521”为例说明。

    firewall-cmd --zone=public --add-port=59521/tcp --permanent

  6. 执行以下命令重启防火墙。

    firewall-cmd --reload
