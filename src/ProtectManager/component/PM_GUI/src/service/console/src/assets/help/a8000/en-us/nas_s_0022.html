Step 2: Adding a Storage Device (Applicable to OceanStor V5/OceanStor Pacific/NetApp ONTAP)

This section describes how to add a storage device if you want the OceanProtect to automatically scan file systems on the storage device before performing NAS share backup and restoration. Otherwise, skip this section.

Context

The OceanProtect can protect a NAS share on the following storage devices:

  • OceanStor V5
  • OceanStor Pacific
  • NetApp ONTAP

Precautions

If the storage device is NetApp ONTAP, ensure that the configuration of management interfaces is completed on the NetApp ONTAP management page. Otherwise, the storage device fails to be added.

Procedure

  1. Choose Protection > File Systems > Storage Devices.

    For 1.5.0, choose Protection > File Services > Storage Device.

  2. Click Add Device to add a storage device.
  3. Select a device type.

    • OceanStor V5
    • OceanStor Pacific
    • NetApp ONTAP

  4. Configure parameters as required. Table 1 describes the related parameters.

    Table 1 Parameters for registering a device

    Parameter

    Description

    Device Name

    Name of a user-defined device.

    IP Address

    Management IP address of the storage device.

    Port

    Number of the port for communication between the OceanProtect and the management plane of the storage device.

    For NetApp ONTAP storage devices, the default port number is 443. For other types of storage devices, the default port number is 8088.

    Username

    Username and password of a storage device administrator. Ensure that the RESTful login mode is available for the user and the password has been initialized.

    NOTE:

    Use an administrator account. Otherwise, resources on the device may fail to be backed up.

    Password

    Certificate verification

    Enabled by default. Import the storage device certificate obtained in Step 1: Obtaining the CA Certificate of a Storage Device to ensure the security of interactions between the OceanProtect and storage device. After this function is disabled, security risks may exist in the communication between the system and the storage device. Exercise caution when performing this operation.

  5. Click OK.

    When a device is registered successfully, information about the registered storage device is displayed on the Storage Device page.

    The OceanProtect automatically scans NAS shares in the storage device and displays them on the Protection > File Systems > NAS Shares page. For 1.5.0, they are displayed on the Protection > File Services > NAS File Systems page. If authentication information is required for accessing a NAS share, configure authentication information for the NAS share on the OceanProtect. For details, see Step 3: Configuring NAS Share Information (Applicable to OceanStor V5/OceanStor Pacific/NetApp ONTAP).
