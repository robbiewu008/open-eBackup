Querying GaussDB (DWS) Information

Procedure

  1. Choose Protection > Big Data > GaussDB (DWS).
  2. On the Cluster tab page, query the GaussDB (DWS) cluster information.

    1. This page displays information about GaussDB (DWS) clusters registered with the product.

      Table 1 describes the related parameters.

      Table 1 GaussDB (DWS) cluster information

      Parameter

      Description

      Name

      Name of the GaussDB (DWS) cluster

      Status

      Status of the GaussDB (DWS) cluster

      • Offline: If the connection between the agent host and GaussDB (DWS) cluster is abnormal, Status is Offline.
      • Normal: The GaussDB (DWS) cluster is available and the data has been backed up. All processes are running and the primary-standby relationship is normal.
      • Unavailable: The GaussDB (DWS) cluster is unavailable.
      • Degraded: The GaussDB (DWS) cluster is available but the data has not been backed up.
      • Unstarted: The GaussDB (DWS) cluster is not started.

      Version

      Specifies the GaussDB (DWS) version.

      SLA

      If an SLA is associated with a resource, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the GaussDB (DWS) cluster is protected.

      • Unprotected: The GaussDB (DWS) cluster is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The GaussDB (DWS) cluster has been associated with an SLA and protection has been activated.
      • Creating: The GaussDB (DWS) cluster has been associated with an SLA and protection is being created.

      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of the cluster.

      If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no authorization is performed, -- is displayed.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the cluster name and check the overview, copy information, and related jobs.

  3. On Schema Sets and Table Sets tab pages, view information about GaussDB (DWS) schema sets and table sets.

    1. This page displays the details about GaussDB (DWS) schema sets and table sets.

      Table 2 describes the related parameters.

      Table 2 GaussDB (DWS) schema set/table set parameters

      Parameter

      Description

      Name

      Name of the GaussDB (DWS) schema set or table set

      Owning Cluster

      Cluster to which the GaussDB (DWS) schema set or table set belongs

      Owning Database

      Database to which the GaussDB (DWS) schema set or table set belongs

      SLA

      If an SLA has been associated with the GaussDB (DWS) schema set or table set, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the GaussDB (DWS) schema set or table set is protected.

      • Unprotected: The GaussDB (DWS) schema set or table set is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The GaussDB (DWS) schema set or table set is associated with an SLA and protection is enabled.
      • Creating: The GaussDB (DWS) schema set or table set is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Related jobs

        Displays backup and restoration jobs.
