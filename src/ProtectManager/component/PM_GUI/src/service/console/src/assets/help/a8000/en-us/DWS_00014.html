Step 1: Registering a GaussDB (DWS) Cluster

Before GaussDB (DWS) backup and restoration, register the GaussDB (DWS) cluster with the OceanProtect.

Prerequisites

  • ProtectAgent has been installed. If ProtectAgent is installed on the production host and an independent physical server, you also need to install RoachClient. For details, see "Installing roach_client (Applicable to GaussDB (DWS))" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.
  • Before registering the resource for the first time, you have modified configuration files of GaussDB (DWS) nodes where ProtectAgent has been installed as required. For details, see Modifying the GaussDB (DWS) Configuration File.
  • If ProtectAgent is deployed on the production host, you need to add environment variables before the first backup. For details, see Precautions for Deploying ProtectAgent on Production Hosts (Applicable to 1.6.0 and Later Versions).
  • Ports required for backup and restoration jobs have been enabled. For details, see the Communication Matrix.

Precautions

After the GaussDB (DWS) cluster is registered successfully, the cluster name cannot be changed because the name uniquely identifies the cluster. Exercise caution when setting the cluster name.

Procedure

  1. Choose Protection > Big Data > GaussDB (DWS).
  2. On the Cluster page, click Register to register a GaussDB (DWS) cluster.

    Table 1 describes the related parameters.
    Table 1 Registering a GaussDB (DWS) Cluster

    Parameter

    Description

    Name

    Customize the GaussDB (DWS) cluster name.

    Username

    Name of the system user for running the GaussDB (DWS) cluster.

    Environment Variable File Path

    Path of the environment variable configuration file of the cluster user.

    Nodes

    Select at least one GaussDB (DWS) node that has CN instances and has ProtectAgent installed.

    Agent Host

    Select agent hosts where RoachClient is installed. For a GaussDB (DWS) cluster whose kernel version is GaussDB A 8.0.0, you do not need to select agent hosts.

    Agent hosts that have been added to other GaussDB (DWS) clusters cannot be added.

  3. Click OK.
