Managing Projects/Cloud Servers or Cloud Server Groups (Applicable to 1.6.0 and Later Versions)

The OceanProtect allows you to modify, remove, activate, or disable protection for databases. This section uses a project as an example to describe related operations. For a cloud server, operations are the same.

You need to log in to the management page and choose Protection > Cloud Platforms > OpenStack. On the Project tab page, locate the target project.

Table 1 describes the related operations.

Table 1 Project protection plan operations

Operation

Description

Navigation Path

Modifying protection

Scenario

This operation enables you to modify a protection plan to re-associate a project with an SLA or modify information such as the host where the client is installed. After the modification, the next backup will be performed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • Assume that a backup job has been executed and the protection is modified by replacing the SLA with copy verification disabled with an SLA with copy verification enabled. After protection modification, the first non-full backup job will be converted to a full backup job.

  • Assume that a backup job has been executed and the SLA parameter Verify Copy associated with the resource is modified from being disabled to enabled. After the SLA parameter modification, the first non-full backup job will be converted to a full backup job during resource protection.

  • If the protection policy in the SLA associated with a resource needs to be changed, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row of the target project, choose More > Modify Protection.

Removing protection

Scenario

This operation enables you to remove protection for a project when it does not need to be protected. After the removal, the system cannot execute protection jobs for the project. To protect the project/resource set again, you need to associate an SLA with the project/resource set again.

Note

If a protection job of the project is in progress, protection cannot be removed.

In the row of the target project, choose More > Remove Protection.

Performing a manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target project, choose More > Manual Backup.

Activating protection

Scenario

This operation enables you to activate the protection plan for a project. After the protection plan is activated, the system executes backup, archive, or replication for the project based on the plan.

Note

This operation can be performed only for a project that has been associated with an SLA and is in Unprotected status.

In the row of the target project, choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable the protection plan of a project for which periodic protection is no longer required. After the protection plan is disabled, the system does not execute automatic backup, archive, or replication for the project.

Note

Disabling a protection plan does not affect protection jobs that are running.

In the row of the target project, choose More > Disable Protection.

Scanning an OpenStack project

Scenario

The OceanProtect automatically scans projects every hour. If any project or cloud server under the registered OpenStack changes, the changes will be automatically updated to the OceanProtect after the scan. This operation enables you to immediately update the changed information to the OceanProtect.

Note

If you do not select to apply the SLA policy to newly created cloud servers upon periodic backup creation, the system will not associate newly discovered cloud servers with any SLA after the scan.

Choose More > Resource Scan.

Authorizing OpenStack resources

NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Choose More > Authorize Resource and select a data protection administrator for authorization.

Reclaiming OpenStack resources

NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Choose More > Reclaim Resource.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
