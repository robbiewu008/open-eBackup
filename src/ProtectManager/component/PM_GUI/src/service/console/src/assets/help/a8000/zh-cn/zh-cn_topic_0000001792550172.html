恢复openGauss

当需要对已备份的openGauss备份集进行恢复时，可以参考本节恢复openGauss备份集至原位置或新位置。

背景信息

支持使用备份副本、复制副本、归档副本、导入的副本进行恢复。支持恢复至原位置（复制副本不支持恢复至原位置，复制副本的归档副本不支持恢复至原位置）或新位置。

注意事项

基于openGauss3.0数据库应用，在进行实例恢复时，恢复任务显示失败，且集群处于降级状态，请尝试手动重建备节点（降级状态：主节点已经启动成功，备节点启动失败）。

数据库系统用户登录备用节点，执行gs_ctl build 数据目录命令重建备节点。

数据目录为实例运行时指定的数据目录，若运行时未指定，默认为安装部署时数据库主节点配置的数据目录。

前提条件

  • 恢复前请确保恢复目标位置的数据目录剩余空间大于恢复副本缩减前大小，否则恢复将失败。
  • 执行新位置恢复前，请确保数据库安装用户具有恢复目标路径的读写权限。
  • 当恢复的资源为MPP主备集群时，关闭数据库前，请确保主备数据库信息正确，否则需要对恢复目标资源执行资源扫描操作。

操作步骤

  1. 选择“数据利用 > 副本数据 > 数据库 > openGauss”。
  2. 您可以以openGauss数据库资源维度或以副本维度查找副本，本节以资源维度为例进行说明。

    在“资源”页签，根据资源名称查找到需要恢复的资源，并单击资源名称。

  3. 依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 您可以指定副本进行恢复。

    恢复备份集至原位置或新位置。

    相关参数说明如表1。
    表1 恢复openGauss备份集

    参数

    说明

    恢复至

    选择恢复至“原位置”或“新位置”。

    位置

    当选择恢复至“原位置”时，默认显示恢复的位置。

    目标集群

    当选择恢复至“新位置”时，可选择恢复的目标集群。

    目标实例

    当选择恢复至“新位置”时，可选择恢复的目标实例。

    数据库重命名

    仅支持数据库恢复时，对恢复完成后的数据库进行重命名。

  5. 单击“确定”。
