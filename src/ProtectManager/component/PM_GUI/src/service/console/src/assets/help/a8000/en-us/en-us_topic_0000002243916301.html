After the System Disk of a FusionCompute VM Is Restored to a New VM, the Target VM Fails to Be Started

Symptom

After the system disk of a FusionCompute VM is restored to a new VM, the target VM, upon startup, enters the Boot Manager page and cannot be started using any boot option.

Possible Cause

When the system disk of a VM is restored to a new VM that runs a different OS or OS version, the boot option of the source FusionCompute VM is BIOS, but that of the target VM can only be UEFI.

Troubleshooting

The following operations use FusionCompute 8.6.0 as an example. The GUI may vary with versions.

  1. Log in to FusionCompute using a browser.

    Login address: https://Floating IP address of the VRM node of FusionCompute:8443

  2. In the left navigation tree, click .
  3. In the navigation tree under Resource Pools, select the corresponding VM.
  4. On the Configuration tab page, choose Management > Options > Boot Firmware to view the Boot Firmware configuration.

    If the Boot Firmware options of the source VM and target VM are different, perform VM restoration.
