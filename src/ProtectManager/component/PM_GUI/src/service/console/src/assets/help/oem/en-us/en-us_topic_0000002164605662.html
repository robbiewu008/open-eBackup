Exiting the Cluster Safe Mode

When hdfs-related commands are executed on a big data cluster node, the output "Access denied for user root. Superuser privilege is required" may be displayed. This section describes how to check whether a node is in the safe mode and exits the safe mode.

Procedure

  1. Log in to any NameNode in the big data cluster.
  2. Run the ps -ef|grep -i hdfs command to check whether there is a running hdfs process. For example, if the following command output is displayed, the process exists:

    hdfs 16493 11954 2 15:19 ? 00:00:29 /usr/java/jdk1.7.0_67//bin/java -Dproc_namenode -Xmx1000m -Dhdfs.audit.logg
hdfs 16523 11954 1 15:19 ? 00:00:14 /usr/java/jdk1.7.0_67//bin/java -Dproc_secondarynamenode -Xmx1000m -Dhdfs.a

  3. If output similar to 2 is displayed, run the following commands to switch to the required user and exit the safe mode:

    su - hdfs
    hdfs dfsadmin -safemode leave
