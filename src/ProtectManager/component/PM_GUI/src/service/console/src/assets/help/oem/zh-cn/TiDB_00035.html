添加对象存储（归档存储是对象存储）

当将对象存储作为归档存储时，请参考本节将对象存储添加至本产品。如果归档存储不是对象存储请忽略本节。

注意事项

  • 当归档存储为OceanStor Pacific 8.1.0、8.1.1、8.1.2，或者归档存储为OceanStor Pacific 8.1.3及后续版本且归档至OceanStor Pacific的互通桶时，存在以下限制：
    • 副本中存在/index目录时，不支持归档该副本。
    • 副本中存在同名的目录和文件时，不支持归档该副本。

      当归档存储为OceanStor Pacific 8.1.3及后续版本且归档至OceanStor Pacific的原生桶时不存在上述限制。

前提条件

  • 已准备用于存放归档数据的对象存储平台。
  • 已配置本产品的归档网络，且归档网络与对象存储网络连接正常。

    如果未配置归档网络，可参考《管理员指南》中的“配置归档网络”章节进行配置。

  • 已获取对象存储平台的访问地址。
  • 当对象存储产品为OceanStor Pacific、OceanStor 100D(FusionStorage OBS)、华为云 OBS或AWS S3时，已获取AK（Access Key）和SK（Secret Access Key）。
  • 当对象存储产品为OceanStor Pacific、OceanStor 100D(FusionStorage OBS)、华为云 OBS或AWS S3时，已在对象存储平台创建数据桶（存放归档数据的桶）和索引桶（存放索引数据的桶）。

    为确保数据的安全性，请创建私有桶，且桶仅为本产品使用。否则，将存在归档数据被非法获取或被篡改的风险。

操作步骤

  1. 选择“系统 > 基础设施 > 归档存储库”。
  2. 单击“添加归档存储”。
  3. 配置归档存储参数。

    相关参数说明如表1。

    表1 归档存储参数说明

    参数

    说明

    名称

    用户自定义归档存储的名称。

    类型

    归档存储的类型。

    说明：

    对于1.5.0及之前版本，仅支持S3协议的对象存储。

    对象存储产品
    接入本产品的对象存储产品。支持的产品如下：
    • OceanStor Pacific
    • OceanStor 100D (FusionStorage OBS)
    • 华为云 OBS
    • AWS S3
    • Microsoft Azure Blob Storage（仅1.6.0及后续版本支持）
    说明：

    当选择的对象存储产品是OceanStor Pacific时，需要登录OceanStor Pacific的DeviceManager管理界面。并通过如下操作开启V2鉴权：选择“设置 > 对象服务设置 > 安全设置”，进入“安全设置”页面的“安全访问策略”页签，在“开启V2鉴权”右侧单击，开启V2鉴权。

    Endpoint

    归档存储的访问地址。

    地址格式：归档存储的域名或业务面IP地址。

    Microsoft Azure Blob Storage的Endpoint获取方式如下：
    1. 登录Microsoft Azure官网。
    2. 在主页选择“存储帐户”，选择对应的存储账户名称。
    3. 在“设置 > 终结点”界面，获取主终结点或辅助终结点的“Blob 服务”的值。
    说明：
    • 如果归档存储的访问地址为域名格式，请参考产品对应型号的《安装指南》中的“（可选）配置DNS”章节，配置DNS服务以支持域名解析。
    • 当通过业务面IP地址连接对象存储时，如果使用协议默认端口（HTTP默认端口为80，HTTPS默认端口为443），则无需输入端口号。否则，请以IP地址:端口的格式输入。例如：10.10.1.1:60。

    协议

    本产品与归档存储之间的数据传输的协议。

    • 当使用“HTTPS”协议时，请导入有效的证书以便本产品验证归档存储的信息。请提前向归档存储的管理员获取证书。导入证书的具体操作请参见《安全配置指南》中的“添加外部证书”。

      如果CA证书不是根CA证书，请先制作完整的证书链文件再导入证书。具体操作请参见《安全配置指南》中的“制作证书链文件”。

    • 当协议设置为HTTP时，本产品将无法验证归档存储环境信息，将存在安全风险。建议您选择安全性高的HTTPS协议。
      说明：

      当选择的对象存储产品是OceanStor Pacific且选择的协议是HTTP时，需要登录OceanStor Pacific的DeviceManager管理界面。并通过如下操作开启HTTP协议：选择“设置 > 对象服务设置 > 安全设置”，进入“安全设置”页面的“安全访问策略”页签，在“开启HTTP协议”右侧单击，开启HTTP协议。

    证书

    使用“HTTPS”协议时，该参数才显示。请选择证书“类型”为“对象存储”的证书，用于验证归档存储环境信息。

    说明：

    证书“类型”可以在“系统 > 安全 > 证书”界面查看。

    AK

    访问对象存储的接入证书。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，不存在此参数。

    SK

    访问对象存储的安全证书。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，不存在此参数。

    数据桶

    存放归档数据的桶名。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，不存在此参数。

    索引桶

    存放索引数据的桶名。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，不存在此参数。

    使用代理服务器

    当本产品使用代理服务器访问归档存储时，请配置代理服务器的地址及用户名、密码。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，不存在此参数。

    说明：
    • 如需使用代理服务器，请确保代理服务器最大允许请求体大小大于100MB。
    • 代理服务器仅支持http协议，地址格式如下：http://代理服务器IP地址:端口号。

    容量告警阈值

    设置桶使用容量的最大阈值，当使用容量达到设定的阈值时上报告警。当“对象存储产品”选择“AWS S3”或“Microsoft Azure Blob Storage”时，不存在此参数。
    Account Name
    说明：

    仅1.6.0及后续版本支持设置该参数。

    Microsoft Azure Blob Storage的存储账户名。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，才显示该参数。

    获取方式如下：
    1. 登录Microsoft Azure官网。
    2. 在主页选择“存储帐户”，选择对应的存储账户名称。
    3. 在“安全性和网络 > 访问密钥”界面，获取“存储帐户名称”字段的值。
    Account Key
    说明：

    仅1.6.0及后续版本支持设置该参数。

    Microsoft Azure Blob Storage的访问密钥。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，才显示该参数。

    获取方式如下：
    1. 登录Microsoft Azure官网。
    2. 在主页选择“存储帐户”，选择对应的存储账户名称。
    3. 在“安全性和网络 > 访问密钥”界面，获取“密钥”的值。
    Blob端口
    说明：

    仅1.6.0及后续版本支持设置该参数。
    本产品与Microsoft Azure Blob Storage之间的数据传输的协议的端口。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，才显示该参数。
    • 协议为HTTP时，端口号默认为80。
    • 协议为HTTPS时，端口号默认为443。

    Blob容器

    说明：

    仅1.6.0及后续版本支持设置该参数。

    存放归档数据的Blob容器。当“对象存储产品”选择“Microsoft Azure Blob Storage”时，才显示该参数。

    获取方式如下：
    1. 登录Microsoft Azure官网。
    2. 在主页选择“存储帐户”，选择对应的存储账户名称。
    3. 在“数据存储 > 容器”界面，获取容器“名称”的值。

  4. 单击“确定”，并根据界面提示完成确认。
