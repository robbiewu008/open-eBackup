Restoring an Elasticsearch Backup Set

This section describes how to restore an Elasticsearch backup set that has been backed up to the original location or a new location.

Context

Backup copies, replication copies, archive copies, or imported copies can be used for restoration. Restoration to the original location or a new location is supported. Restoration to the original location using a replication copy is not supported. Restoration to the original location is not supported after a replication copy is archived.

Procedure

  1. Choose Explore > Copy Data > Big Data > Elasticsearch.
  2. You can search for copies by Elasticsearch backup set resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the backup set to be restored based on the backup set name and click the backup set name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed under a month or date, a copy exists in the month or on the date.

  4. Choose More > Restore in the row where the copy is located.

    Table 1 describes related parameters.

    Table 1 Backup set restoration parameters

    Parameter

    Description

    Restore To

    The location to which the backup set is restored.

    • Original location: Restoration to the original Elasticsearch cluster.
    • New location: Restoration to a new Elasticsearch cluster. You must specify the target Elasticsearch cluster.

    Target Cluster

    The target Elasticsearch cluster specified for the restoration to a new location.

    Overwrite Rule

    If an index with the same name exists in the restoration path, you can select either of the following:

    • Ignore the existing index, that is, the indexes with the same names are skipped and are not replaced.
    • Replace the existing index.

  5. Click OK.
