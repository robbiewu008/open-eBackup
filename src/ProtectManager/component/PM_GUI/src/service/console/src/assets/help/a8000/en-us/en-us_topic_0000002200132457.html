Restoring Files

This section describes how to restore files on a VM by using the file-level restoration function.

Context

  • The OceanProtect supports file-level restoration using backup copies and replication copies. Note that data cannot be restored to the original location when a replication copy is used.
  • The types of Linux file systems that can be restored are ext2/ext3/ext4 and XFS. The types of Windows file systems that can be restored are NTFS, FAT, and FAT32.

Prerequisites

  • The IP address of the backup network plane has been configured for the VM to which files are to be restored.
  • If the target VM runs the Windows OS, ensure that ports 445 and 138 have been enabled on the target VM. If the target VM runs the Linux OS, ensure that port 22 has been enabled on the target VM.
  • If the target VM runs the Windows OS, ensure that the CIFS service has been enabled on the VM. For details, see the Microsoft official website.
  • If the target VM runs a non-Windows OS, ensure that rsync has been installed and started on the target VM and the rsync version is 3.2.5 or later.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Alibaba Cloud.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    Click the Resources tab, search for the VM to be restored by name, and click the VM name.

  3. On the Copy Data tab page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. Find the copy used for restoration and click Copy Time.
  5. On the copy information page, choose Operation > File-level Restoration. Alternatively, on the Resources tab page, locate the row that contains the target copy and choose More > File-level Restoration.

    • A copy that does not contain system disks cannot be used for file-level restoration.
    • Reverse replication copies and archive copies cannot be used for file-level restoration.
    • Linked files do not support file-level restoration.
    • During Alibaba Cloud VM backup, the mount point information that is not written into the /etc/fstab file is stored in the memory. The OceanProtect does not back up the information and does not support file-level restoration for Alibaba Cloud VMs. To support file-level restoration for new file systems, write the mount point information of the new file systems to the /etc/fstab file.
    • If the name of a folder or file contains garbled characters, file-level restoration is not supported. Do not select folders or files of this type. Otherwise, the restoration job fails.
    • If the target VM runs the Windows OS and only the SMBv1 service is enabled, the time of the restored file is the current VM time.

  6. Set File Obtaining Mode, which can be Select file paths from the directory tree or Enter file paths.

    • If you choose Select file paths from the directory tree for File Obtaining Mode but no file-level mounting task has been created for the current copy, click Create Mount. You can select files from the directory tree only after the mounting is successful.
    • For OceanProtect E1000, if there are more than 10,000 files in the selected directory, the system may time out, preventing you from selecting files or folders to be restored from the directory. In this case, you are advised to manually enter paths. For other models of this product, if there are more than one million files in the selected directory, the system may time out, preventing you from selecting files or folders to be restored from the directory. In this case, you are advised to manually enter paths.
    • When entering a file path, enter a complete file path, for example, /opt/abc/efg.txt or C:\abc\efg.txt. If you enter a folder path, for example, /opt/abc or C:\abc, all files in the folder are restored. The file name in the path is case sensitive.

  7. On the displayed Restore File page, select Original location or New location for Restore To and set restoration parameters.

    Table 1 describes the related parameters.

    Table 1 File-level restoration parameters

    Parameter

    Description

    Target Location

    Location of the VM to which the file is restored.

    NOTE:

    Ensure that the OS of the target location is the same as that of the VM to be restored.

    Target Path
    • If no target path is not specified, data will be restored to the target VM based on the original path where the selected files are stored.
    • If the target path is specified, the files will be restored to the specified path on the target VM. Enter an absolute path.
      NOTE:
      • Assume that the target VM runs the Windows OS and the specified path for file restoration is D:\a. If drive D does not exist on the target VM, the restoration will fail. If drive D exists on the target VM but does not have the a directory, the a directory will be automatically created on drive D of the VM before file restoration is performed.
      • Assume that the target VM runs the Linux OS and the specified path for file restoration is /opt/b. If the target path does not exist on the VM, the corresponding path will be automatically created on the VM before file restoration is performed.

    Cloud Server IP Address

    Select or manually enter the IP address of the target VM.

    Cloud Server Username
    During file-level restoration, the target VM needs to verify the login credential of the OceanProtect. Enter a username and its password for logging in to the target VM.
    • Windows OS: The default username is Administrator.
    • Linux OS: The default username is root.
    NOTE:
    • The login user must have the read and write permissions on the directory to which data is to be restored on the target VM.
    • After the restoration, the access permission on the file is the same as that of the login user.

    Cloud Server Password

    Overwrite Rule

    If files with the same names exist in the restoration path, you can choose to replace or skip existing files.

    • Replace existing files
    • Skip existing files
    • Only replace the files older than the restoration file
    NOTE:

    If the access to the file with the same name on the target VM is denied, the file will fail to be restored during file replacement.

  8. Click Test and ensure that the network connection between the target VM to be restored and the OceanProtect is normal.
  9. Click OK.
