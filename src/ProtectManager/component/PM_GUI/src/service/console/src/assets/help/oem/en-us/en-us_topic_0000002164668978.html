Viewing the GoldenDB Environment

After the GoldenDB instance is registered with the product, you can view details about the instance and the host or cluster where the instance resides on the management page.

Procedure

  1. Choose Protection > Databases > GoldenDB.
  2. On the Instance tab page, view instance information.

    The registered instances are displayed on the page.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    1. View detailed information about the database described in Table 1.
      Table 1 Database information

      Parameter

      Description

      Name

      Database name.

      Status

      Status of the instance in the cluster.

      Node IP Address

      IP address of the host or cluster to which the instance belongs.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether the instance is protected.

      • Unprotected: The instance is not associated with an SLA, or the instance is associated with an SLA but protection is disabled.
      • Protected: The instance is associated with an SLA and protection is activated.
      • Creating: The instance is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name and check the overview, copy information, and jobs.
      • Overview

        Displays the SLA information, job execution time, and total number of copies.

      • Copy Data

        You can search for copies by year, month, or day.

        under the time indicates that a copy exists at the time.

      • Jobs

        Ongoing and historical jobs are displayed.

  3. On the Cluster tab page, view cluster information.

    Information of registered clusters is displayed on the page.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    Table 2 describes the detailed information.
    Table 2 Cluster information

    Parameter

    Description

    Name

    Cluster name.

    Click the cluster name to view management node information and instance information of the cluster.

    Status
    Cluster status.
    • Online: If the host is powered on and the client on the host is running properly, Status is Online for the host.

      If there are online hosts in the cluster, the cluster status is Online.

    • Offline: If the host is shut down or the host where the client is installed is faulty, Status is Offline for the host.

      If all hosts in the cluster are offline, the cluster status is Offline.

    Node IP Address

    IP address of the node where the cluster resides.

    Version

    Cluster version.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
