(Optional) Creating Backup Storage Unit Groups

A backup storage unit is used to manually set the storage location of backup copies during backup. If you want to automatically switch the storage location of backup copies or automatically set the storage location of replication copies during replication, perform operations in this section to create a backup storage unit group.

Prerequisites

  • When creating a backup storage unit group, ensure that at least two normal backup storage units exist and select at least one backup storage unit of the local cluster.
  • If you want to use the backup storage unit of an external device to create a backup storage unit group, you must enable Parallel Data Storage when creating the backup storage unit group.

Precautions

  • When creating a backup storage unit group, you cannot select storage units of the same backup storage device.
  • When a backup storage unit group is created or modified, only GaussDB (DWS) supports parallel data storage. During SLA creation, you cannot select this backup storage unit group as the copy storage location for applications that do not support parallel data storage.
  • In the LAN-free scenario, the storage policy does not take effect.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  2. Click Backup Storage Unit Groups.
  3. Click Create to create a backup storage unit group. Table 1 describes the related parameters.

    Table 1 Parameters for creating a backup storage unit group

    Parameter

    Description

    Name

    Name of the backup storage unit group, which can be customized.

    Description

    Enter the description of the backup storage unit group.

    Parallel Data Storage

    This function is disabled by default. If this function is enabled, copy data is stored on all backup storage units. Otherwise, copy data is stored on the backup storage unit corresponding to the job execution node.

    NOTE:
    • If you need to back up GaussDB (DWS) and use an external backup storage unit, enable Parallel Data Storage. Only GaussDB (DWS) supports parallel data storage. During SLA creation, you cannot select this backup storage unit group as the copy storage location for unsupported applications.
    • After parallel data storage is enabled, you need to select at least one backup storage unit that belongs to the local cluster.
    • After the Parallel Data Storage function is disabled, only nodes in the backup cluster can be selected.

    Fault Timeout Period

    Time required for the system to wait for a faulty standby node or member node to restore if a job is distributed to the node. If the node is not restored within the fault timeout period, the system switches to another node to execute the job. The default fault timeout period is 15 minutes. The value ranges from 1 to 120 minutes.

  4. Select all backup storage units in the Available Backup Storage Unit list and set the alarm threshold (80% by default) in the Selected Backup Storage Unit list.
  5. Click Next.
  6. Confirm the storage policy. Two storage policies are supported: Reduction Ratio First and Intelligent Balancing. Select an option based on the site requirements.

    • Reduction Ratio First: This mode preferentially ensures the data reduction ratio and preferentially uses the storage unit used during the first backup. If the node or ProtectAgent network is faulty, the storage unit is switched based on the policy algorithm.
    • Intelligent Balancing: During each full backup backup, the system automatically selects the optimal storage unit using the algorithm based on the capacity and job load of the storage cluster.

  7. Click OK.
