Performing Periodic Backup

After resources to be protected are associated with an SLA, the system protects the resources based on the SLA.

Procedure

  1. Choose Protection > File Systems > Volume.
  2. Select the volume to be protected and click Protect.

    You can also select multiple volumes for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

  4. Configure advanced parameters for volume protection.

    Table 1 describes the related parameters.
    Table 1 Advanced parameters for volume protection

    Parameter

    Description

    OS Backup

    This function is disabled by default. After this function is enabled, both the selected volumes and system volumes are backed up, and the following information is collected: quantity of disk drives, volume information, file system information, and network information.

    NOTE:

    After OS backup is enabled, the /boot directory is backed up regardless of whether the volume where the /boot directory is located is selected.

    Execution Script
    You can execute custom scripts of the sh type based on your needs before a backup job is executed, or after the backup job is successfully or fails to be executed. If the script is stored in the DataBackup/ProtectClient/ProtectClient-E/sbin/thirdparty directory, you only need to enter the script name. If the script is stored in another directory, you need to enter the absolute path of the script.
    NOTE:

    If Script to Run upon Restoration Success is configured, the status of the restoration job is displayed as Succeeded on the product even if the script fails to be executed. Check whether the job details contain a message indicating that the post-processing script fails to be executed. If yes, modify the script in a timely manner.

  5. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
