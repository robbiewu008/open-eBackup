步骤1：注册SAP HANA数据库（Backint备份方式）

Backint备份方式是通过第三方备份工具写入备份数据，每个SAP HANA服务启动Backint for SAP HANA agent，将备份数据发送给第三方备份工具。

操作步骤

  1. 选择“保护 > 应用 > SAP HANA”。
  2. 注册SAP HANA实例。

    在“实例”页签，单击“注册”。SAP HANA实例注册信息相关参数说明，如表2所示。
    表1 SAP HANA实例注册信息

    参数

    说明

    名称

    用户自行设置的实例名称。

    实例ID

    安装数据库时设置的ID。实例ID查询方式请参考备份前准备。

    主机

    数据库所在的主机。

    如果数据库是集群部署，请选择数据库所在的所有主机。

    认证模式

    选择“数据库认证”或“Hdbuserstore模式”。

    说明：
    • 当选择数据库认证模式时，用户需要在注册时输入数据库用户名与密码。

    Hdbuserstore KEY
    如果数据库是集群部署，当选择Hdbuserstore模式时，用户需要在实例所在的所有主机设置Hdbuserstore KEY参数，可通过以下方式设置。
    1. 使用PuTTY工具，以root用户登录SAP HANA数据库所在主机。
    2. 执行以下命令切换SAP HANA数据库用户，以用户名s00adm为例。
      su - s00adm
    3. 执行以下命令设置Hdbuserstore KEY参数。
      hdbuserstore set S00SAPDB "hana1:30013" SYSTEM 1234

      其中S00SAPDB是用户设置的KEY参数，hana1是主机名，30013是默认端口号，1234是数据库密码，请根据实际替换。

    4. 执行以下命令查看Hdbuserstore KEY参数。
      hdbuserstore list

      回显如下所示，KEY所在行是Hdbuserstore KEY参数。

    系统数据库用户名

    认证模式选择“数据库认证”时，请输入系统数据库用户名与密码。

    系统数据库密码

    系统数据库端口

    安装数据库时所设置的端口号，默认端口号是30013。

    日志备份

    功能开启后才能进行日志备份。

    定期日志备份时间间隔

    设置日志备份的时间间隔。

    说明：

    建议该时间间隔的取值在表1中设置的日志备份时间间隔的50%~100%之间。例如当备份策略中设置的日志备份时间间隔为60分钟，则此参数建议配置的时间间隔取值范围为30分钟~60分钟。

    日志备份路径

    设置数据日志备份生成的路径。

    说明：
    • 请确保日志备份目录可用空间大于所有日志备份副本将占用的空间，并且确保SAP HANA日志备份路径权限至少为755。
    • 如果生产环境已经设置了日志备份路径，则这里设置之后会改变原有日志备份路径。
    • 在执行备份任务后，设置的日志备份路径才会生效。

  3. 单击“确定”。
  4. 注册SAP HANA数据库。

    在“数据库”页签，单击“注册”。SAP HANA数据库注册信息相关参数说明，如表2所示。
    表2 SAP HANA数据库注册信息

    参数

    说明

    类型

    数据库类型。

    • “系统数据库”。
    • “租户数据库”。
    说明：

    在注册系统数据库、租户数据库或集群时，用户需要具有备份管理员的权限，建议系统数据库用户名使用SYSTEM。

    名称
    待保护的数据库名称，该名称与生产环境的数据库名称保持一致，查询方式请参考备份前准备。
    说明：
    • 如果类型选择“系统数据库”，则选择注册安装数据库时所设置的数据库名称。
    • 如果类型选择“租户数据库”，则选择注册租户数据库时所设置的租户数据库名称。

    实例

    选择对应已注册的实例。

    主机

    数据库所在的主机。

    如果数据库是集群部署，请选择数据库分布的所有主机。

    认证模式

    选择“数据库认证”或“Hdbuserstore模式”。

    说明：
    • 当选择数据库认证模式时，用户需要在注册时输入数据库用户名与密码。

    Hdbuserstore KEY
    如果数据库是集群部署，当选择Hdbuserstore模式时，用户需要在数据库所在的所有主机设置Hdbuserstore KEY参数，可通过以下方式设置。
    1. 使用PuTTY工具，以root用户登录SAP HANA数据库所在主机。
    2. 执行以下命令切换SAP HANA数据库用户，以用户名s00adm为例。
      su - s00adm
    3. 执行以下命令设置Hdbuserstore KEY参数。
      hdbuserstore set S00SAPDB "hana1:30013" SYSTEM 1234

      其中S00SAPDB是用户设置的KEY参数，hana1是主机名，30013是默认端口号，1234是数据库密码，请根据实际替换。

    4. 执行以下命令查看Hdbuserstore KEY参数。
      hdbuserstore list

      回显如下所示，KEY所在行是Hdbuserstore KEY参数。

    系统数据库用户名

    认证模式选择“数据库认证”时，请输入系统数据库用户名与密码。

    系统数据库密码

    系统数据库端口

    安装数据库时所设置的端口号，默认端口号是30013。

  5. 单击“确定”。
