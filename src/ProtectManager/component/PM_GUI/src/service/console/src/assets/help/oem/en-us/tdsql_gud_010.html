Step 1: Enabling the TDSQL Database Permission (Applicable to Non-distributed Instances)

Before backing up a non-distributed TDSQL instance, you need to enable the permission of the corresponding version. Otherwise, the backup or restoration will fail. In other scenarios, skip this section.

Context

Unless otherwise specified, the operations in this section use TDSQL 10.3.22.1 as an example. The operations may vary depending on the version.

Procedure

  1. Log in to the TDSQL CHITU management console, select an instance in Instance management, choose Database management > User Management, and click Create User to create a user for backup.

    You do not need to select any permission when creating a user.

  2. Log in to the active database instance node as user root.

    To obtain the IP address of the active node, select an instance on the Instance management page of the TDSQL CHITU management console, and go to the DB monitoring page.

  3. Run the following command to log in to the TDSQL database as an administrator (for example, tdsqlsys_agent):

    /data/tdsql_run/Active node port number/mysql-server-Database version/bin/mysql --user='Administrator username' --password='Administrator password' -S socket file path
    Replace the Active node port number, Database version, Administrator username, Administrator password, and socket file path in the command with the actual ones.
    • To obtain the Active node port number, select an instance on the Instance management page of the TDSQL CHITU management console, and go to the DB monitoring page.
    • To obtain the Database version, choose System Management > Version management on the TDSQL CHITU management console and obtain the value before the first hyphen in the current version of DB-Mysql.

      For example, if the current version is 8.0.24-v24-txsql-2.0.1-V2.0R751D003-20221125, replace the Database version in the command with 8.0.24.

    • Contact the database administrator to obtain the Administrator username and Administrator password.
    • To obtain the socket file path, run the cat /data/tdsql_run/Active node port number/mysql-server-Database version/etc/my_Active node port number.cnf | grep sock command.

    Example:

    /data/tdsql_run/4004/mysql-server-8.0.24/bin/mysql --user='tdsqlsys_agent' --password='123456' -S /data1/tdengine/data/4004/prod/mysql.sock

  4. Log in to the primary node and run the following commands to enable the permission of the corresponding version for the user.

    • If MySQL 8.0.24 is used, run the following command:
      GRANT BACKUP_ADMIN, PROCESS, RELOAD, SELECT, LOCK TABLES, SUPER ON *.* TO 'Username created in 1'@'%';
    • If MariaDB 10.1.9 is used, run the following command:
      GRANT SELECT, RELOAD, PROCESS, SUPER, LOCK TABLES ON *.* TO 'Username created in 1'@'%';
