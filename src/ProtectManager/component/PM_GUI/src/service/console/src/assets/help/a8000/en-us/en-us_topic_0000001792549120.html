Viewing the MongoDB Environment

After a MongoDB instance is registered with the OceanProtect, you can view details about the instance and the host or cluster where the instance resides on the WebUI.

Procedure

  1. Choose Protection > Big Data > MongoDB.
  2. Query the MongoDB database information.

    This page displays information about the MongoDB databases that have been registered with the OceanProtect.
    1. View detailed information about the database described in Table 1.
      Table 1 Database information

      Parameter

      Description

      Name

      Database name.

      Status

      Status of the instance in the cluster.

      Version

      MongoDB version number.

      Type

      Database type, which can be Single Instance, Cluster-replica set, or Cluster-sharded.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether the instance is protected.

      • Unprotected: The instance is not associated with an SLA, or the instance is associated with an SLA but protection is disabled.
      • Protected: The instance is associated with an SLA and protection is activated.
      • Creating: The instance is associated with an SLA and protection is being created.
    2. Click the name and check the overview, copy information, and jobs.
      • Overview

        Displays the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, a copy at that time exists.

      • Job

        Ongoing and historical jobs are displayed.
