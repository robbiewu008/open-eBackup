A VMware Restoration Job Fails, and a Message Indicating that the VM Fails to Be Created Is Displayed in the Job Details

Symptom

A VMware restoration job fails to be executed when the vSAN storage is selected as the target datastore for restoration. In the job details, a message is displayed indicating that the VM fails to be created.

Possible Causes

The default storage policy of the selected vSAN storage is incompatible with that of the VM.

Troubleshooting

  1. Log in to the vCenter Server on a browser.

    Login address: https://vCenter Server IP address

  2. On the vCenter Server, find the vSAN storage selected during VMware restoration.
  3. Select the corresponding vSAN storage in the directory tree and view the value of Default Storage Policy on the Configure tab page. If the value is vSAN Default Storage Policy, change it to Management Storage Policy - Single Node.
