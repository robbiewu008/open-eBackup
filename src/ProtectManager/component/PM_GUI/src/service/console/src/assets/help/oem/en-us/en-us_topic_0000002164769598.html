Step 5: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA. The system protects the resources and periodically executes backup jobs based on the SLA. You can perform a backup job immediately through manual backup.

Precautions

If the host where the client is installed runs the Windows OS, the product mounts file systems of the backup storage to the C:\mnt\databackup\ path by default. To modify the mount path, open the agent_cfg.xml configuration file in the Client installation directory\DataBackup\ProtectClient\ProtectClient-E\conf directory and change the value of win_mount_public_path value.

Replace Client installation directory in the directory with the actual one.

Context

  • During manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication policy has been defined in the SLA, the product will perform replication once based on the SLA when you perform manual backup.

Prerequisites

Before the backup, ensure that the Active Directory service is running properly.

Procedure

  1. Choose Protection > Applications > Active Directory.
  2. Select the Active Directory domain controller to be protected and click Protect.

    You can also select multiple Active Directory domain controllers for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

  4. Configure advanced parameters.

    Table 1 describes the related parameters.
    Table 1 Advanced parameters of resource protection

    Parameter

    Description

    Object-Level Backup

    This function is enabled by default. After this function is enabled, the generated backup copies can be used for object-level restoration.

  5. Click OK.

    If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  6. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row that contains the Active Directory domain controller you want to back up, choose More > Manual Backup.

      You can select multiple Active Directory domain controllers for manual backup in batches. Specifically, select multiple Active Directory domain controllers and choose More > Manual Backup in the upper left corner of the page.

    2. Set the name of the copy generated during manual backup.

      If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.

    3. Select Full Backup for Protection Policy.

      If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    4. Click OK.
