How Do I Change the Number of NFS Volumes Mounted to an ESXi Host?

Live mount is to mount the linked clone of a copy generated on the OceanProtect to an ESXi host, create an NFS datastore, and start the VM. If the number of mount jobs created on a single ESXi host exceeds the maximum number of NFS volumes that can be mounted, the mount jobs will fail. To change the number of NFS volumes that can be mounted to an ESXi host, perform the following steps (vCenter 6.7.0 is used as an example):

Procedure

  1. Log in to the vCenter Server on a browser.

    Login address: https://vCenter Server IP address

  2. Select the ESXi host whose configuration needs to be modified to display the host page.
  3. Choose Configure > System > Advanced System Settings.
  4. Click EDIT, find NFS.MaxVolumes, and change the value of NFS.MaxVolumes.

    Before performing this operation, record the number of volumes that can be mounted before the modification for rollback if necessary.
