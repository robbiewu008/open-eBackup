About SLA

Service level agreement (SLA) is a set of configurable data protection policies that define rules for data protection and copy retention. These policies are associated with an environment to implement protection. To meet enterprises' different data protection requirements for various environments, the OceanProtect Appliance allows users to customize SLAs.

Custom SLAs

Users can customize SLAs for backup, archiving, and replication jobs. In an SLA, user can customize basic parameters such as the job execution interval, time window, and copy retention period, and other advanced parameters for various protected resources.

Table 1 describes different types of SLAs.

Table 1 Custom SLAs

SLA

Description

Backup SLA

Define parameters such as the backup job execution interval and backup copy retention period.

Replication SLA

Specify the target cluster to which a backup copy is replicated, and define parameters such as the replication job execution interval and replication copy retention period.

Archive SLA

Specify the archive storage (object storage or tape libraries) to which backup copies or replication copies are to be archived, and define parameters such as the archive job execution interval and archive copy retention period.
