步骤2：（可选）创建IPsec策略（适用于备份存储为OceanProtect）

在开启复制链路加密开关前，需在复制链路两端存储设备为复制网络逻辑端口创建IPsec策略。创建IPsec策略后，远程复制过程中会对传输数据进行加密，确保数据安全性。当为复制网络逻辑端口创建IPsec策略时，复制链路两端存储设备均需执行以下操作。

前提条件

  • 复制网络为IP网络时，才支持创建IPsec策略。
  • 复制网络IP地址为IPv4时，才支持创建IPsec策略。
  • 复制网络逻辑端口创建在跨卡绑定端口时，不支持创建IPsec策略。
  • 仅SmartIO接口模块（10Gbit/s）接口模块才支持配置IPsec策略。

操作步骤

  1. 查看复制链路两端所有待创建IPsec策略的复制网络逻辑端口的最大传输单元（MTU，Maximum Transmission Unit）是否小于交换机端口的最大传输单元。

    交换机端口MTU请联系网络管理员获取，复制网络逻辑端口MTU的查询和修改方式如下：

    为确保创建IPsec策略后复制链路的网络连通性，复制链路两端的复制网络逻辑端口MTU必须小于交换机端口MTU，且差值大于等于100字节。

    1. 分别登录主端和从端存储系统的DeviceManager管理界面。具体操作请参考登录DeviceManager管理界面。
    2. 选择“服务 > 网络 > 逻辑端口”。
    3. 筛选“角色”为“复制”的逻辑端口，获取“当前端口”的值。
    4. 在左侧菜单栏，选择“以太网络”。
    5. 根据1.c获取的值，在“位置”列筛选对应的以太网端口，并查看该端口“最大传输单元(字节)”的值。
      • 如果所有复制网络逻辑端口MTU小于交换机端口MTU，且差值大于等于100字节，则执行2。
      • 否则，请修改复制网络逻辑端口MTU。
        1. 单击以太网端口的名称。
        2. 在弹出的详情页面右上角，查看单击“操作 > 修改”，在“最大传输单元(字节)”中输入修改后的值。

          复制网络逻辑端口MTU的取值范围为[1280, 交换机端口MTU-100]。

          例如，交换机端口MTU为1500字节，则复制网络逻辑端口MTU的取值范围为[1280, 1400]，推荐设置为1300字节。

        3. 单击“确定”。

  2. 切换接口模块的安全类型为IPsec。

    1. 选择“系统 > 硬件 > 设备”。
    2. 单击需要切换安全类型的接口模块所在的控制框。
    3. 单击，切换到存储设备的后视图。
    4. 单击需要切换安全类型的接口模块。
    5. 在弹出的接口模块页面选择“操作 > 切换安全类型”。
    6. 选择安全类型“IPsec”。

      安全类型为“IPsec”时，该接口模块上所有端口的TOE功能关闭且无法单独进行开启。

    7. 按照界面提示完成确认操作。

  3. 为复制网络逻辑端口创建IPsec策略。

    1. 选择“服务 > 网络 > 逻辑端口”。
    2. 选择需要创建IPsec策略的复制网络逻辑端口，单击“管理IPsec策略”。
    3. 单击“创建”，创建IPsec策略。

      相关参数说明如表1所示。

      表1 IPsec策略参数说明

      参数名称

      参数说明

      名称

      IPsec策略的名称。

      远端IP地址

      复制链路上从端存储设备的复制网络IP地址。

      仅支持IPv4，最多支持输入32个IP地址，多个IP地址以英文分号、空格或按回车键分隔。

      说明：

      IPsec策略创建成功后，后续您也可以通过修改IPsec操作添加新的IP地址或删除已有IP地址。

      加密算法

      数据加密传输所使用的加密算法。复制链路两端的加密算法需要保持一致。数据加密算法包括：AES和SM4。

      说明：

      某些产品型号不支持设置加密算法，请以实际界面展示为准。对于不支持设置加密算法的型号，默认使用AES算法进行加密。

      预共享密钥

      自定义预共享密钥，复制链路两端的预共享密钥需要保持一致。

      [取值范围]

      • 长度范围是16~127位。
      • 至少包含特殊字符、大写字母、小写字母以及数字中的任意两种。特殊字符包括：!"#$%&'()*+,-./:;<=>?@[\]^`{_|}~和空格。
    4. 单击“确定”。

      如果后续您需要删除不再使用的IPsec策略，请分别在两端存储设备删除。先在一端存储设备删除后，复制业务会断开，另外一端删除后，复制业务将自动恢复。因此，建议您在没有复制业务时执行删除IPsec策略的操作，并且删除一端的IPsec策略后立即删除另外一端的IPsec策略。
