执行Hyper-V备份任务失败，错误详情包含“Read file failed, error: 22”

现象描述

执行Hyper-V备份任务失败，同时待备份虚拟机无法开机，提示磁盘文件被其他进程占用。登录OceanProtect管理界面，选择“保护 > 客户端”，在对应的客户端所在行，导出日志。查询VirtualPlugin.log日志文件中包含“Read file failed, error: 22”。

可能原因

待备份的Hyper-V虚拟机磁盘文件的进程被Hyper-V主机的其他进程占用，导致备份流程中客户端读取磁盘文件报错，报错内容包含“22：ERROR_BAD_COMMAND，设备无法识别命令”。

处理步骤

  1. 登录待备份的Hyper-V宿主机。
  2. 打开windows任务管理器，在“性能”页面，打开资源监视器。
  3. 在“CPU”页签，打开“关联的句柄”页签，搜索被占用的磁盘文件名。磁盘文件名在执行虚拟机开机任务的报错信息中获取。
  4. 手动关闭占用磁盘文件的进程。
  5. 启动待备份虚拟机，观察是否能正常开机。

    • 是：即代表故障排除。
    • 否：请联系华为技术支持工程师。
