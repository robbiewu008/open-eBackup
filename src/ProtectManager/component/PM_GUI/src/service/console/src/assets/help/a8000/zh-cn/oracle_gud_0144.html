恢复任务提示数据库处于运行状态

执行恢复任务时界面提示：不允许目标主机中存在处于运行状态的同名数据库，请关闭数据库后重试。

按照如下步骤关闭数据库后重新执行任务。

  1. 登录Oracle数据库主机。

    • 如果是Linux操作系统，请使用PuTTY登录Oracle数据库主机，并执行su - oracle命令，切换到oracle账户。
    • 如果是Windows操作系统，请使用系统管理员登录Oracle数据库主机。

  2. 执行以下命令，登录Oracle数据库实例。

    • OS认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus / as sysdba

      • 如果是Windows操作系统，请执行以下操作：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus / as sysdba

    • 数据库认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus

        根据提示输入具备sysdba权限的用户：

        username as sysdba

        根据提示输入密码。

      • Windows OS执行以下命令：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus

          根据提示输入具备sysdba权限的用户：

          username as sysdba

          根据提示输入密码。

  3. 执行以下命令，关闭数据库实例。

    shutdown immediate;

    回显如下表示数据库关闭完成。

    ORACLE instance shut down.

  4. 重试任务。
