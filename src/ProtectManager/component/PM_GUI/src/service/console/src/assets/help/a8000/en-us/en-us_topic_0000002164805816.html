Creating a Media Set (When the Archive Storage Is a Tape Library) (Applicable to OceanProtect X Series Backup Appliances)

This section describes how to create a media set when a tape library is used as the archive storage. If the archive storage is not a tape library, skip this section.

Context

  • A media set is created based on tapes and is a tape storage pool for storing archive data. Therefore, ensure that a tape library has been scanned and updated to the OceanProtect before creating a media set.
  • The system automatically scans tape libraries every 24 hours. When detecting a tape library connected to the OceanProtect, the system automatically updates the tape library to the OceanProtect.

    If a tape library is not updated to the OceanProtect, you can manually scan the tape library. For details, see "Scanning Tape Libraries" in the OceanProtect DataBackup 1.5.0-1.6.0 Administrator Guide.

Prerequisites

Ensure that Access Status is Accessible for all tapes in the tape library. You can view the value of Access Status on the tape management page.

Procedure

  1. Choose System > Infrastructure > Archive Storage Library.
  2. On the Tape tab page, select a node and click Media Set.
  3. Click Create.

    Table 1 describes the parameters for creating a media set.

    Table 1 Parameters for creating a media set

    Parameter

    Description

    Name

    Media set name, which is user-defined.

    Driver Block Size

    The value can be 64 KB, 128 KB, or 256 KB. Data is written to tapes block by block. Data is written to tapes in a media set based on the set block size.

    NOTE:

    You are advised to select the driver block size based on the file size.

    • For large files, select a large driver block size to ensure read and write efficiency.
    • For small files, select a small drive block size to make full use of tape space.

    Data Compression
    Decreases data volume to reduce storage space and improve write speed.
    • Enable: indicates to enable the hardware compression function of the tape library to reduce the required tape capacity and improve the write speed.
    • Disable: indicates to disable the hardware compression function of the tape library.

    Owning Controller

    Select the controller to which the tapes belong.

    Tape

    In the tape library list on the left, select a tape library and select tapes in the tape library on the right.

    NOTE:
    • Tapes in multiple tape libraries can be added. Tapes on different controllers cannot be added.
    • Tapes that have been added to other media sets cannot be added to the media set.
    • If Location of a tape to be selected is displayed as Driver for a long time, the tape is not identified, or the value of Number of Drives is different from the actual number, the tape may be locked on the driver. In this case, manually unload the tape by referring to the tape device operation guide. After the tape is unlocked from the driver, perform Scan Tape Library again.

    Insufficient Available Tapes
    • After this function is enabled, you need to set Alarm Threshold. If the total number of empty tapes and expired tapes that can be overwritten is less than the threshold, an alarm is generated. To clear the alarm, you can insert empty tapes or mark tapes with data stored as empty.
      NOTE:
      • If data compression is enabled, the maximum amount of data that can be stored on a tape is unknown. For example, if the nominal capacity of a tape is 50 GB, the tape may actually be able to store 60 GB, 80 GB, or 100 GB data. Therefore, alarms cannot be generated based on the tape capacity, but can be generated only based on the total number of empty tapes and expired tapes that can be overwritten.
      • The alarm threshold cannot exceed the total number of selected tapes.
    • After this function is disabled, the number of available tapes is unknown. You are advised to enable this function.

    Alarm Threshold

    Set the number of tapes. When the number of available tapes is less than the threshold, an alarm is generated.

    Retention Range
    • No Data Protection (Overwrite Tape as Needed): Data in tapes is not protected. When new data is written to a tape, the original data is overwritten. To select this retention policy, ensure that there are at least two tapes in the current media set.
    • Permanent Data Retention: Data in tapes is permanently retained.
    • Temporary Data Protection: Data in tapes is protected periodically for xx days, months, or years. For example, if the period for temporary data protection is set to 10 days, the tape is marked as empty after 10 days since the last data is written into the tape. Then, the tape can be overwritten.

  4. Click OK. After a media set is created, the system updates the status and capacity information of the media set every hour. Before the system updates data, the status and capacity of a media set may be inconsistent with the actual situation.
