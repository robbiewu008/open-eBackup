Failed to Delete a Volume in a Backup Job After the Host Machine Is Restarted

Symptom

After the OpenStack host machine is restarted, the backup job is successful, and the Failed to delete volume tag is printed in the job details.

Possible Causes

The host where the client is installed and host machine use the same volume group name. When the host machine is restarted, a logical volume in the volume group fails to be deactivated during the restart. As a result, the volume fails to be unmounted and, consequently, fails to be deleted.

Procedure

  1. Log in to FusionSphere OpenStack OM.
  2. Create a VM as the new host where the client is installed and delete the original host.
