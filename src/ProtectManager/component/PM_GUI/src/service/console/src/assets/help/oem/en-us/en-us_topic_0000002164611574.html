Managing Projects, Resource Sets, and ECSs

The product allows you to modify, remove, activate, or disable protection for projects, resource sets, and ECSs. This section uses a project/resource set as an example to describe related operations. Operations on ECSs are the same.

You need to choose Protection > Cloud Platforms > Huawei Cloud Stack, and find the target project/resource set on the Project/Resource Set tab page.

Table 1 describes the related operations.

Table 1 Project/Resource set management operations

Operation

Description

Navigation Path

Modifying protection

Scenario

You can modify a protection plan to re-associate a project/resource set with an SLA or modify client information. After the modification, the next backup will be executed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • Assume that a backup job has been executed and the protection is modified by replacing the SLA with copy verification disabled with an SLA with copy verification enabled. After protection modification, the first non-full backup job will be converted to a full backup job.

  • Assume that a backup job has been executed and the SLA parameter Verify Copy associated with the resource is modified from being disabled to enabled. After the SLA parameter modification, the first non-full backup job will be converted to a full backup job during resource protection.

  • If the protection policy in the SLA associated with a resource needs to be changed, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row that contains the target project/resource set, choose More > Modify Protection.

Removing protection

Scenario

You can remove the protection if the project/resource set no longer needs to be protected. After the removal, the system cannot execute protection jobs for the project/resource set. To protect the project/resource set again, you need to associate an SLA with the project/resource set again.

Note

Protection cannot be removed when a protection job of the project/resource set is running.

Choose More > Remove Protection.

Activating protection

Scenario

This operation enables you to re-activate a disabled protection plan of a project/resource set. After the protection plan is activated, the system backs up or replicates the project/resource set data based on the protection plan.

Note

You can perform this operation only on a project or resource set that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable the protection plan of a project/resource set when its periodic protection is not needed. After the protection plan is disabled, the system does not perform automatic backup and replication jobs for the project/resource set.

Note

Disabling a protection plan does not affect protection jobs that are running.

Choose More > Disable Protection.

Performing manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup.

Scanning projects/resource sets

Scenario

The product automatically scans for the projects/resource sets every 24 hours. When changes of projects/resource sets or ECSs in registered Huawei Cloud Stack are detected, the information is automatically updated to the product after the scan. This operation enables you to immediately update the changed information to the product.

Note

If you do not select to apply the SLA policy to the newly created ECS when creating a periodic backup job, the system will not associate the newly discovered ECS with an SLA after the scan.

Choose More > Resource Scan.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: Restoration is supported only for ECSs. For details, see Restoration.
