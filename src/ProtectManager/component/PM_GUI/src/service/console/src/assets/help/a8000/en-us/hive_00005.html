Function Overview

Backup

  • Full backup: backs up all data of resources at the current time point.
  • Forever incremental (synthetic full) backup: If no full copy exists, full backup is performed for the first forever incremental (synthetic full) backup, and incremental backup is performed for subsequent backup jobs. Copies generated by forever incremental (synthetic full) backup are forever incremental (synthetic full) copies, which improve restoration performance and do not occupy extra space.
  • Source deduplication: reduces the amount of data to be transmitted, but occupies more client resources. Only the X series backup appliances support source deduplication.
  • Backup link encryption: encrypts transmitted data during data backup to ensure data security.
  • Transmission and storage encryption: adopts the AES256 encryption algorithm to encrypt backup data.

Replication

  • Backup copy replication: replicates backup copies from the source end to the target end. If a disaster occurs in the local data center, the replication copies can be used to restore service data.
  • Replication link encryption: After an IPsec policy is created for the replication network logical ports on the storage devices at both ends of the replication link, data is encrypted during remote replication to ensure security.
  • Reverse replication: replicates replication copies from the target end to the source end.
  • Cascaded replication: replicates replication copies from the target end to the other end.

Archiving

  • Manual archiving (supported only in 1.6.0 and later versions) or periodic archiving by creating an SLA is supported.
  • Archiving backup/replication copies that need to be retained for a long time to object storage
  • Archiving backup/replication copies that need to be retained for a long time to tape libraries

Restoration

  • Copy-based restoration to the original or a new location: supports data restoration using backup copies, reverse replication copies, cascaded replication copies (data cannot be restored to the original location), archive copies (data cannot be restored to the original location using the archive copies generated by replication copes and cascaded replication copies), and replication copies (data cannot be restored to the original location).
