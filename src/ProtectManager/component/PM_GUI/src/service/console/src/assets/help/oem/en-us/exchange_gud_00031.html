Step 4: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Prerequisites

Before the backup, ensure that the Exchange Server service and related dependent services are running properly. For details about the corresponding services, see Overview of Exchange services on Exchange servers.

  • Before performing the backup, ensure that the product time is the same as the client time. Otherwise, data may be lost after log restoration.

Context

  • During manual backup, the parameters defined in the SLA (except Automatic Retry), such as Rate Limiting Policies, are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication or archive policy has been defined in the SLA, the will perform replication or archive once based on the SLA when you perform manual backup.
  • For mail backup, a registered user must have the ApplicationImpersonation role permissions. When the mailbox is backed up for the first time, the agent host automatically creates a user group named Databackup, which has the ApplicationImpersonation role permissions, and adds the registered user to the user group.

Procedure

  1. Choose Protection > Applications > Exchange.
  1. On the Email tab page, select the mailbox that you want to protect and choose More > Protect.

    You can choose multiple mailboxes for batch protection.

  2. Select a created SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resource to be protected, select an SLA without a WORM policy to create protection to avoid WORM policy conflicts.

  3. Select whether to continue to protect emails or folders after they fail to be backed up.
  4. Click OK.

    If the current system time is later than the start time of the first backup set in the SLA, you can choose to perform the backup immediately or wait for the next scheduling based on the backup policy set in the SLA after the backup SLA is associated with the resources.

  5. Optional: Perform manual backup.

    If you want to perform a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. On the Email tab page, locate the row that contains the mailbox that you want to protect, and choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

    2. Select a protection policy, which can be Full backup or Incremental backup.

      For 1.6.0 and later versions: If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    3. Set the name of the copy generated during manual backup.

      If this parameter is left empty, the system sets the name to Resource name_Timestamp by default.

    4. Click OK.
