Viewing openGauss Information

After an openGauss database is registered with the product, you can view details about the database and the host or cluster where the database resides on the WebUI.

Procedure

  1. Choose Protection > Databases > openGauss.
  2. View related information on the Databases tab page.

    1. View database information. Table 1 describes related parameters.
      Table 1 Database information

      Parameter

      Description

      Name

      Database name.

      Owning Cluster

      Name of the host or cluster where the database resides.

      Owning Instance

      Database instance name.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether a database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
    2. Click the name of a resource and check the overview, copy information, and related jobs.
      • Resource overview

        The unique cluster ID, SLA information, job execution time, and total number of copies are displayed.

      • Copy data

        You can search for copies by year, month, or day.

        above the time indicates that a copy exists at the time.

      • Resource-related jobs

        Displays all jobs related to resources, including restoration and archive jobs.

  3. On the Cluster tab page, query the openGauss cluster information.

    1. This page displays information about openGauss clusters registered with the product.

      Table 2 describes related parameters.

      Table 2 openGauss cluster information

      Parameter

      Description

      Name

      openGauss cluster name.

      Status

      openGauss cluster status.

      • Normal: The openGauss cluster is available and the data has been backed up. All processes are running and the primary-standby relationship is normal.
      • Unavailable: The openGauss cluster is unavailable.
      • Degraded: The openGauss cluster is available but the data has not been backed up.

      Version

      openGauss version.

      Type

      Cluster type.

      • Single-node
      • Active-standby replication

      Owner

      Data protection administrator of a host or cluster.

      If a host or cluster has been authorized to a data protection administrator, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
    2. Click the cluster name and check the overview, copy information, and related jobs.

  4. View related information on the Instance tab.

    1. This page displays details about openGauss instances.
      Table 3 describes related parameters.
      Table 3 openGauss instance information

      Parameter

      Description

      Name

      Database instance name.

      Status

      Database instance status.

      • Online
      • Offline

      Owning Cluster

      Name of the cluster where the instance resides.

      Synchronization Mode

      Data synchronization mode among nodes in a cluster.

      • Synchronous
      • Asynchronous

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether a database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
    1. Click the name and check the overview, copy information, and jobs.
      • Overview

        The SLA information, job execution time, and total number of copies are displayed.

      • Copy data

        You can search for copies by year, month, or day.

        above the time indicates that a copy exists at the time.

      • Job

        Backup and restoration jobs are displayed.
