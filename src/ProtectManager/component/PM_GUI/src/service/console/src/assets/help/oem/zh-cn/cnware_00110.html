恢复任务失败，界面提示新增磁盘失败

故障现象

对CNware虚拟机执行恢复，本产品任务界面提示新增磁盘失败，检查CNware任务，失败任务提示虚拟机处于开机状态或目标挂载磁盘被依赖，导致恢复任务失败。

可能原因

  • 当虚拟机状态异常或CNware发生异常时，执行CNware虚拟机关机操作，任务提示成功。实际虚拟机仍然处于开机状态，导致修改目标虚拟机引导顺序任务失败。
  • 当虚拟机状态异常或CNware发生异常时，执行快照删除，CNware提示任务成功。在磁盘恢复任务中，实际快照仍存在，由于该磁盘被快照依赖，导致磁盘无法挂载到其他虚拟机。

处理步骤

  • 修改目标虚拟机引导顺序任务失败
    1. 登录CNware管理平台。
    2. 选择“计算 > 云资源池”，找到目标虚拟机，单击“关机”。
    3. 检查关机任务。关机任务成功，若虚拟机仍为开机状态，则单击“关闭电源”，手动执行关闭电源操作。
    4. 重新执行恢复任务。
  • 磁盘无法挂载到其他虚拟机。
    1. 登录CNware管理平台。
    2. 选择“计算 > 云资源池”，找到目标虚拟机。
    3. 单击“任务”，筛选失败任务，找到最近的挂载失败任务，在任务详情中查找依赖的磁盘并删除。
    4. 重新执行恢复任务。
