Performing Manual Backup

If you want to perform a backup immediately, you can manually perform the backup.

Context

  • During the execution of a manual backup job, parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, Automatic Indexing, and Application-Consistent Backup, take effect.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy is retained permanently. If forever incremental (synthetic full) backup is not defined in the SLA but selected for manual backup, copies generated by the manual forever incremental (synthetic full) backup will be retained forever.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.
  • When a manual backup is initiated for a cluster or host, if the cluster or host has a VM whose SLA protection policy is inconsistent with that of the cluster or host, the VM will not initiate a backup job.

Procedure

  1. Choose Protection > Virtualization > CNware.
  1. Click the VMs, Hosts, or Cluster tab based on the type of the data to be backed up.

    When you select Hosts or Cluster for backup, only VMs with the same SLA as the host or cluster are backed up. VMs with different SLAs from the host or cluster are not backed up.

  2. Choose More > Manual Backup in the row where the VM, host, or cluster to be backed up is located.

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

  3. Set the name of the copy generated through manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  4. Select a protection policy, which can be Full backup or Forever Incremental (Synthetic Full) Backup.
  5. Click OK.
