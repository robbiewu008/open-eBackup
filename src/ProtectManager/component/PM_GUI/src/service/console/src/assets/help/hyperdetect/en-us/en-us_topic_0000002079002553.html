Managing SLAs

You can clone, modify, and delete SLAs on ProtectManager.

You need to log in to ProtectManager and choose Protection > Protection Policies > SLAs to go to the SLA page.

Managing SLAs describes the operations related to SLAs.

Table 1 SLA-related operations

Operation

Description

Navigation Path

Cloning an SLA

Context

This operation enables you to quickly create an SLA based on an existing SLA.
  1. Locate the row that contains the SLA to be cloned and choose More > Clone. The Clone SLA page is displayed.
  2. Set SLA parameters.

Modifying an SLA

Context

This operation enables you to modify a backup policy configured in an SLA.
  1. Locate the row that contains the SLA to be modified and choose More > Modify. The Modify SLA page is displayed.
  2. Set SLA parameters.

Deleting an SLA

Context

This operation enables you to delete an SLA that is no longer used.

Precautions

An SLA that has been associated with resources cannot be deleted. To delete such an SLA, you must remove protection for the resources associated with the SLA first.
  1. Locate the row that contains the SLA to be deleted and choose More > Delete. The Danger dialog box is displayed.
    NOTE:

    You can also select multiple SLAs and click Delete in the upper left corner to delete them in a batch.

  2. Read the message carefully and perform operations as required.
