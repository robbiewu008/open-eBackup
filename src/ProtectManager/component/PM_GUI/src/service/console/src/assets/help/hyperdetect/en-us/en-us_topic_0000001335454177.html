Viewing File System Information

Prerequisites

File system resources on the storage system have been available on ProtectManager.

Context

File systems created on the storage system can be automatically displayed on the Local File System page of ProtectManager.

  • When ProtectManager scans for a new file system that is created on the storage system, it may take some time for ProtectManager to discover the file system.
  • File systems discovered on ProtectManager cannot be deleted on ProtectManager.
  • Clone file systems are not displayed on ProtectManager.
  • File systems whose names start with pvc_ created through HyperDetect are not displayed on ProtectManager.

Procedure

  1. Choose Protection > Storage > Local File System.
  2. In the local file system list, view the file system information.

    1. Table 1 describes the parameters related to a file system.
      Table 1 Parameters related to a file system

      Parameter

      Description

      Resource ID

      ID of the local file system.

      Name

      Name of a local file system.

      Tenant

      vStore to which the file system belongs. The default value is System_vStore.

      SLA

      If the local file system has been associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance

      SLA compliance indicates whether a backup job is successfully executed based on the specified backup interval or backup time window. For example, if the backup time of a backup job (except initial backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met. If the backup interval set in the SLA for a backup job is once every half an hour, but the backup time exceeds half an hour, SLA compliance is missed.

      Protection Status

      Whether the local file system is protected.

      • Unprotected: The local file system is not associated with an SLA, or the local file system is associated with an SLA but protection is disabled.
      • Protected: The local file system is associated with an SLA and protection is enabled.
      • Creating: The local file system is associated with an SLA and protection is being created.
    2. Click the name of a local file system to view its summary information, copies, and related jobs.
      • Summary information about a local file system

        The summary information includes the name, SLA information, task execution time, and total number of copies.

      • Snapshot data of a local file system

        You can search for copies by year, month, or day.

        If is displayed below the time, a copy at that time exists.

      • Jobs related to the local file system

        All jobs related to the local file system are displayed, including ransomware detection snapshot backup and restoration jobs.
