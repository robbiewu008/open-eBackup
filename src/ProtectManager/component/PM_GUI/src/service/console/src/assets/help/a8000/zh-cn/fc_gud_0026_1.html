步骤1：创建复制网络逻辑端口（适用于1.6.0及后续版本）

分别在主从端的OceanProtect中创建复制网络逻辑端口，用于远程复制数据传输。

前提条件

已规划主从端复制网络IP地址。

注意事项

请确保网络配置时无正在运行的任务。

操作步骤

  1. 选择“系统 > 设置 > 网络配置”。
  2. 单击“网络配置”界面右上角的。
  3. 在“业务网络配置”区域，为每个控制器创建复制逻辑端口。

    1. 单击“创建端口”。
    2. 在弹出的“配置业务端口”界面，选择“角色”为“复制”。
    3. 自定义逻辑端口名称。
    4. 在“网络配置”区域，设置逻辑端口的IP地址类型、IP地址、子网掩码（IPv4）/前缀长度（IPv6）、网关。
    5. 在“端口配置”区域，参考表1配置。

      “端口配置”区域的信息保存后不支持修改，配置时请务必确保信息正确。

      表1 参数说明

      参数

      说明

      是否使用VLAN

      默认不勾选。勾选前请确认业务已配置VLAN，勾选后，配置的业务端口会应用于VLAN。

      端口类型

      逻辑端口所属的端口的类型，包括“以太网端口”和“绑定端口”。

      端口选择
      支持“创建新端口”或“复用当前已配置端口”。
      说明：
      • 当勾选了“是否使用VLAN”或“端口类型”选择“绑定端口”时，才显示此参数。
      • 当“端口类型”选择“绑定端口”，但当前可用以太网端口数量小于2个时，“端口选择”请设置为“复用当前已配置端口”或修改“端口类型”为“以太网端口”。
      • 复用当前已配置的绑定端口创建逻辑端口，如果该逻辑端口对应的绑定端口未被其他业务占用，则删除逻辑端口的同时也会删除备份存储设备上的绑定端口。

      以太网端口
      • 当“端口类型”选择“绑定端口”且“端口选择”为“创建新端口”时，如果当前可用的以太网端口数量不小于2个，则可以选择可用的以太网端口。
      • 当“端口类型”选择“以太网端口”时，请选择可用的以太网端口。

      最大传输单元(字节)

      网络平面的成员端口传输的最大数据包的大小。

      取值范围为1280~9000。

      说明：
      • 最大传输单元即MTU（Maximum Transmission Unit）。
      • 未勾选“是否使用VLAN”时，选择“绑定端口”且“端口选择”为“创建新端口”时，该参数才能显示。
      • 勾选“是否使用VLAN”时，选择“以太网端口”且“端口选择”为“创建新端口”时，或选择“绑定端口”时，该参数才能显示。

      VLAN ID

      如果勾选了“是否使用VLAN”，则必须配置VLAN ID。
    6. 单击“确定”。
    7. 可选：当本端复制网络与目标集群复制网络不在同一网段时，需要配置路由。否则，请跳过本步骤。
      1. 单击对应端口信息操作列的“路由管理”。
      2. 在“路由管理”界面，单击“新增”。
      3. 选择路由类型，在“目的地址”中输入目标复制集群的复制网络网段，填写子网掩码（IPv4）/前缀（IPv6）、本端复制网络的网关。

        当路由类型为“默认路由”时，仅需设置“网关”。

      4. 单击“确认”。
      5. 单击“关闭”。

  4. 单击“确定”。
