Managing Instances

The system allows you to modify, remove, activate, or disable protection for databases.

Choose Protection > Cloud Platforms > Huawei Cloud Stack GaussDB. On the Instance tab page, locate the target instance.

Table 1 describes the related operations.

Table 1 Instance protection plan operations

Operation

Description

Navigation Path

Modifying protection

Scenario

You can modify a protection plan to modify the associated SLA information of an instance. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target instance, choose More > Modify Protection.

Removing protection

Scenario

You can remove the protection if the instance no longer needs to be protected. After the removal, the system cannot execute protection jobs for the instance. To protect the project/resource set again, you need to associate an SLA with the project/resource set again.

Note

When a protection job of the instance is running, protection cannot be removed.

In the row of the target instance, choose More > Remove Protection.

Manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target instance, choose More > Manual Backup.

Activating protection

Scenario

This operation enables you to activate a disabled protection plan for an instance. After a protection plan is activated, the system backs up, archives, or replicates instance data based on the protection plan.

Note

This operation can be performed only on instances that have been associated with an SLA and are in the Unprotected state.

In the row of the target instance, choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable the protection plan of an instance for which periodic protection is not required. After a protection plan is disabled, the system does not automatically back up, archive, or replicate instance data.

Note

Disabling a protection plan does not affect protection jobs that are running.

In the row of the target instance, choose More > Disable Protection.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: For details about how to restore data, see Restoration.
