Restoring Hyper-V VMs

This section describes how to restore an entire VM that has been backed up to the original location or a new location.

Context

The product allows you to restore VMs to the original or a new location using backup copies or replication copies (restoration to the original location is not supported).
  • Restoring to the original location

    If you restore a VM to the original location, the system creates a VM in the path of the original VM (that is, the Hyper-V server to which the original VM belongs) and fully restores the backup data to the new VM.

  • Restoring to a new location

    If you restore a VM to a new location, the system performs full restoration and restores the backup data to a specified new path. That is, the user specifies the Hyper-V server where the new VM is located and the corresponding path.

Procedure

  1. Choose Explore > Copy Data > Virtualization > Hyper-V.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    Click Resources to switch to the corresponding page, search for the VM to be restored by name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. In the row of the target copy, choose More > Restore.
  5. Restore a VM to the original location or a new location.

    • Restore the VM to the original location.
      1. Select Original location.
      2. Set a name for the new VM.
      3. Select whether to enable the network adapter of the VM after the restoration.
        • Disabled: All network adapters of the VM are disconnected after the restoration. To remotely log in to the VM, you need to set the connection status of the network adapters on the Hyper-V management page after the restoration.
        • Enabled: After this option is enabled, if the original VM of the copy still exists, the new VM may have a network conflict with the original VM after the restoration. Check the network of the new VM after the restoration.
      4. Select whether to automatically start the VM after the restoration.
      5. Select whether to delete the original VM after the restoration is successful.

        If this option is enabled, the original VM will be deleted after the VM is successfully restored. Otherwise, the original VM will be retained.

      6. Click OK.
    • Restore the VM to a new location.
      1. Select New location.
      2. Set a name for the new VM.
      3. Set information about a new location to which the VM is restored.
        Table 1 New location information

        Parameter

        Description

        Compute Location

        Compute resources used by the VM after being restored.

        Select a Hyper-V server as the target host.

        NOTE:

        The Hyper-V version of the target host cannot be earlier than that of the original Hyper-V.

        Storage Location

        File path on a Hyper-V server to which VM configuration files and disk data are restored.

        • Different Datastores
          • VM Profile: path of the VM configuration files on the Hyper-V server.
          • VM Disk Files: path of the virtual disk on the Hyper-V server.
            NOTE:

            To improve data security, you can store the VM configuration files on an independent datastore and disk data on another datastore.

        • Same Datastore

          The VM configuration files and all virtual disk data are stored in the same path on the Hyper-V server.
      4. Select whether to enable the network adapter of the VM after the restoration.
        • Disabled: All network adapters of the VM are disconnected after the restoration. To remotely log in to the VM, you need to set the connection status of the network adapters on the Hyper-V management page after the restoration.
        • Enabled: After this option is enabled, if the original VM of the copy still exists, the new VM may have a network conflict with the original VM after the restoration. Check the network of the new VM after the restoration.
      5. Select whether to automatically start the VM after the restoration.
      6. Click OK.

  6. Click Close.
