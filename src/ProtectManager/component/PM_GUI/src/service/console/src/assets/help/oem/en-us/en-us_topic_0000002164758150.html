Tag-based Global Search

This section describes how to browse or search for production resources by tag.

Context

Global search for resources supports all applications connected to the product.

Procedure

  1. Click in the upper part of the page to display the Global Search page.

    1. Select Tag in the search drop-down list.
    2. Enter a tag name and click . Details about the resources to which the tag is added are displayed.

  2. In the navigation tree on the left, you can filter the displayed results again as required.

    The logic relationship between parameters is OR. That is, the search results filtered based on either one or more selected conditions will be displayed.

    Table 1 Search parameters

    Parameter

    Description

    Resource Type

    Select the resource type to be searched for.

    Protection Status
    Whether a resource is currently protected.
    • Unprotected: The resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
    • Protected: The resource has been associated with an SLA and protection has been enabled.
    • Creating: The resource has been associated with an SLA and protection is being created.

    SLA Compliance

    Filter data based on whether SLA compliance is met.

    Resource Location

    Enter the location of the resource.
    After the search is complete, the search result is displayed on the right of the page. Table 2 describes the related parameters.
    Table 2 Parameter description

    Parameter

    Description

    Name

    Resource name.

    Type

    Resource type.

    Location

    Location of a resource.

    SLAs

    If an SLA is associated with the resource, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
    • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

    Protection Status
    Whether the current resource is protected.
    • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
    • Protected: The current resource is associated with an SLA and protection is activated.
    • Creating: The current resource is associated with an SLA and protection is being created.

    Tag

    Name of the tag associated with the current resource.

  3. In the row of the target resource, click More to perform operations on the resource, such as adding and modifying protection.
