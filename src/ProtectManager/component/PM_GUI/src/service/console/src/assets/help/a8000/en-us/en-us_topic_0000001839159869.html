Periodically Executing Backup

Associate the NAS share to be protected with a specified SLA. The system backs up these resources periodically based on the SLA.

Precautions

  • For a NAS share discovered through device registration, if the NAS share has authentication information, complete the configuration by referring to Step 3: Configuring NAS Share Information. Otherwise, the backup fails.
  • For an NFS share, ensure that the current device has the read and write permissions on the NFS share directory and the no_root_squash option is enabled. For details about how to configure NFS share access permissions, see Step 5: Configuring Access Permissions.
  • For a CIFS share, ensure that the current device has the read and write permissions on the CIFS share directory. You are advised to use a username in the Administrator group for backup. Otherwise, only files and directories that have granted the read and write permissions to the device can be backed up. For details about how to configure CIFS share access permissions, see Step 5: Configuring Access Permissions.

Procedure

  1. Choose Protection > File Systems > NAS Shares.

    For 1.5.0, choose Protection > File Services > NAS Shares.

  2. In the row where a NAS share to be protected resides, click More > Protect .
  3. Select the created SLA.

    You can also click Create to create an SLA.

  4. Set advanced parameters.

    • Channels

      The more the channels, the better the backup performance. However, more memory of the production host will be occupied. Each channel occupies about 200 MB memory.

    • Small File Aggregation

      After this function is enabled, the system aggregates multiple small files in the same directory into a large file to improve backup efficiency. If incremental backup is contained but forever incremental (synthetic full) backup is not configured in the SLA, small file aggregation cannot be disabled. If forever incremental (synthetic full) backup is configured in the SLA, small file aggregation cannot be enabled.

      You need to set the following two parameters:

      • File Size After Aggregation: size of the aggregated file. The value can be 128 KB, 256 KB, 1024 KB, or 4096 KB.
      • Maximum Aggregated File Size: maximum size of the file to be aggregated. The value can be 128 KB, 256 KB, 1024 KB, or 4096 KB.

      For small file aggregation, the file size after aggregation must be larger than or equal to the size of the largest file aggregated.

    • SMB Hard Link Protection

      This option is enabled by default. If too many files exist in the same directory of the NAS share dataset to be protected, the backup job may fail. It is recommended that this option be disabled in this scenario.

    • SMB ACL Protection

      This option is enabled by default. If too many files exist in the same directory of the NAS share dataset to be protected, the backup job may fail. It is recommended that this option be disabled in this scenario.

    • Hot Data Backup

      After this function is enabled, the system backs up only the files whose modification time is within the specified time range before backup execution. For example, if the function is enabled and set to 60 days, the system backs up the files whose modification time is within 60 days before backup execution.

  5. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
