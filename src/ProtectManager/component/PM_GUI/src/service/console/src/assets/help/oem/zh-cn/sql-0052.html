恢复SQL Server数据库

当需要对已备份的数据库进行恢复时，可以参考本节使用副本恢复数据库至原机或异机。

前提条件

  • 在进行异机恢复时，请确保目标主机上存在数据库实例。
  • 恢复前请确保恢复目标位置的数据目录剩余空间大于恢复副本缩减前大小，否则恢复将失败。

操作步骤

  1. 选择“数据利用 > 副本数据 > 数据库 > SQL Server”。
  2. 您可以以数据库资源维度或以副本维度查找副本，本节以资源维度为例进行说明。

    在“资源”页签，根据资源名称查找到需要恢复的资源，并单击资源名称。

  3. 选择副本恢复的资源。
  4. 指定副本或指定时间点进行恢复。

    • 指定副本恢复
      1. 依次选择年、月、天找到副本。

        当时间上显示，即表示该月、该天存在副本。

      2. 在副本数据页面单击，单击需要恢复的副本所在行的“更多 > 恢复”。
    • 指定时间点恢复
      1. 依次选择年、月、天找到副本。

        当时间上显示，即表示该月、该天存在副本。

      2. 在副本数据页面单击，在时间轴上选择蓝色覆盖区域的副本，单击“更多 > 恢复”。

  5. 恢复SQL Server数据库相关参数说明如表1所示

    表1 恢复SQL Server数据库参数说明

    参数

    说明

    恢复至

    选择恢复至“原位置”或“新位置”。

    位置

    当选择恢复至“原位置”时，默认显示原位置。

    目标主机/集群

    当选择恢复至“新位置”时，选择原主机或异主机恢复。

    目标实例

    当选择恢复至“新位置”时，选择目标主机上的实例。

    说明：
    • 当恢复的资源类型为“数据库”时展示该参数。
    • 请确保目标实例不能存在同名数据库或同名可用性组。

    目标路径

    说明：

    仅1.5.0版本本支持设置该参数。

    选择恢复至“新位置”时展示该参数，将数据库文件恢复到目标主机的路径。

    说明：
    • 当恢复的资源类型为“数据库”时展示该参数。
    • 请确保该目标路径盘符已存在且不是盘符根目录，数据库用户对该目标路径具有写权限。
    • 请确保目标路径下不能存在同名数据库文件。

    数据库重命名

    开启后，可对恢复后的数据库重命名。

    说明：
    • 当恢复的资源类型为“数据库”时展示该参数。
    • 当开启了数据库重命名且数据库恢复至原位置，恢复完成后，需选择数据库所属实例进行资源扫描（在“保护 > 数据库 > SQL Server > 实例”页面，在实例所在行单击“更多 > 资源扫描”。），并对该执行一次全量备份。

    恢复前执行脚本

    您可以根据实际需求，在恢复任务执行前、执行成功后、执行失败后，执行自定义脚本。

    请输入脚本名称，脚本名称以.bat结尾。确保脚本已存放在数据库主机的“DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty”目录，且登录数据库主机的用户（默认为root）对该脚本有可执行权限。

    说明：
    • 当恢复的资源类型为“数据库”时展示该参数。
    • 当配置了“恢复成功执行脚本”时，即使该脚本执行失败，本产品上也会显示恢复任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。
