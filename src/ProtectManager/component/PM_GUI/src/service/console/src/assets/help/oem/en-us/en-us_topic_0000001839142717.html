Step 4: Enabling Log Mode for the MySQL/MariaDB Database

Before executing a MySQL/MariaDB backup job, you need to enable the log mode for the MySQL/MariaDB database. Otherwise, resources fail to be connected.

Procedure

  1. Log in to the MySQL/MariaDB database as user root.
  2. Run the following command to check whether the archive mode is enabled for the MySQL/MariaDB database.

    show variables like 'log_bin';

    If Value is ON in the command output, the log mode is enabled.

    +-------------+-----+
|Variable_name|Value|
+-------------+-----+
|   log_bin   |  ON |
+-------------+-----+
1 row in set(0.00 sec)

  3. Run the following command to check whether the log mode of the MySQL/MariaDB database is ROW.

    show variables like 'binlog_format';

    If the Value is ROW in the command output, the log mode is ROW.

    +-------------+-----+
|Variable_name|Value|
+-------------+-----+
|binlog_format| ROW |
+-------------+-----+
1 row in set(0.00 sec)

  4. Perform this step if the log mode is disabled or not set to ROW. Otherwise, skip this step.

    1. Log in to the Linux host.
    2. Run the vi /etc/my.cnf command to open the my.cnf configuration file and add the following content to the file.
      log_bin = mysql_bin
binlog_format = ROW
server-id=1

      In the cluster scenario, ensure that the value of each server-id on each node is unique.

    3. Run the restart command to restart the MySQL service.

      For example, run the service mysqld restart command to restart the MySQL service.
