Step 2: Registering an openGauss Cluster

Before backing up and restoring an openGauss database or instance, the system automatically scans for the openGauss database and instance. You do not need to manually register the openGauss cluster with the product.

Prerequisites

Before registering an openGauss cluster, install ProtectAgent on all nodes in the openGauss cluster. For details, see the ProtectAgent Installation Guide.

Procedure

  1. Choose Protection > Databases > openGauss.
  2. Click the Clusters tab.
  3. Click Register to register an openGauss cluster.
  4. Configure the cluster authentication information.

    Table 1 describes the related parameters.

    Table 1 openGauss cluster registration information

    Parameter

    Description

    Name

    Custom instance name.

    Username

    Name of the OS user who runs the database.

    Type

    The cluster type can be Single Server or Active/Standby Replication.

    Environment Variable File Path

    During database installation, this path must be specified if separated deployment is used.

    NOTE:

    This path is the environment configuration file path set by the user during separated deployment.

    Nodes

    Host where the openGauss instance resides.

    NOTE:

    The selected nodes must have the openGauss database installed and belong to the same cluster.

  5. Click OK.
  • Once the cluster is registered, the system automatically scans it every 5 minutes to gather instance and database information.
  • You can also manually scan the cluster. On the Clusters tab page, choose More > Resource Scan in the row of the target cluster to obtain instance and database information.
