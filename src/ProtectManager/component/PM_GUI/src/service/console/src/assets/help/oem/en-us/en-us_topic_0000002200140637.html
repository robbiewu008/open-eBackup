A Backup Job Keeps Retrying During VM Backup in Hot-Add Transmission Mode

Symptom

During backup of a VM in Hot-Add transmission mode, a message, indicating that node XX is preparing for the backup job, is repeatedly displayed in the backup job details, stack logs containing "==================stack list==================" are repeatedly displayed in the client logs, and it is found on the vCenter Server that residual disks exist on the VM where the client is installed.

Possible Causes

The previous backup or restoration job of the VM is not completed properly, leading to residual disks on the VM where the client is installed. When backup is performed again, the VDDK cannot automatically clear residual disks. As a result, the backup job keeps retrying.

Troubleshooting

  1. Stop the backup job that is being retried and ensure that no other job is being executed on the VM where the client is installed.
  2. Log in to the vCenter Server on a browser.

    Login address: https://vCenter Server IP address

  3. In the left navigation tree, right-click the VM where the client is installed and choose Edit Settings from the shortcut menu.
  4. Select all residual disks except system disks and placeholder disks and click on the right to remove them. Do not select Delete files from datastore.
  5. Click OK.
