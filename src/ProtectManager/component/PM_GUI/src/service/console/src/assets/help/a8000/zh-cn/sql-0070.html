查看SQL Server数据库环境信息

当SQL Server数据库已注册至OceanProtect后，您可以在管理界面查看数据库及其所在主机或所属集群/可用性组的详细信息。

操作步骤

  1. 选择“保护 > 数据库 > SQL Server”。
  2. 查看详细信息。

    • 在“集群”页签浏览SQL Server集群信息。

      当集群有数据库已通过注册，集群信息会显示在该页面。

      详细信息如表1所示。
      表1 集群信息

      参数

      说明

      名称

      数据库集群名称。

      • 单击集群名称，显示当前集群的主机成员，及集群下已通过认证的所有数据库。

      状态
      集群的状态。
      • 在线：主机处于开机状态且主机上的ProtectAgent正常运行时，主机“状态”为“在线”。

        集群中存在在线状态的主机时，集群状态为“在线”。

      • 离线：主机关机或者主机上的ProtectAgent故障时，主机“状态”为“离线”。

        集群中所有主机均离线时，集群状态为“离线”。
      标签
      说明：

      仅1.6.0及后续版本存在此参数。

      当资源已添加标签时，则此处显示标签名称。否则，显示“--”。
    • 在“实例”页签浏览数据库实例信息。
      数据库实例的详细信息如表2所示。
      表2 数据库实例信息

      参数

      说明

      名称

      数据库实例名称。

      所属主机/集群

      数据库实例所在主机或集群的名称。

      节点IP地址

      数据库实例所在的主机IP地址。

      版本

      实例所在数据库的版本。

      SLA

      当数据库已关联SLA时，则显示SLA名称，否则显示“--”。

      SLA遵从度
      SLA遵从度指某次备份任务是否按设定的备份频率或备份窗口成功完成备份。
      • 遵从：如果某次备份任务（首次全量备份除外）的备份时间满足备份窗口或备份频率要求，且备份成功，则表示SLA遵从。
      • 不遵从：如果备份时间不满足备份窗口或备份频率要求，例如某次备份任务SLA设定的备份频率为每半小时执行一次，最终备份时间超过半小时，则表示SLA不遵从。建议调整备份频率或备份窗口。

      保护状态

      实例当前是否被保护。

      • 未保护：实例未关联SLA，或者数据库已关联SLA但已禁用保护。
      • 已保护：实例已关联SLA，且已激活保护。
      • 创建中：实例已关联SLA，正在创建保护。
      标签
      说明：

      仅1.6.0及后续版本存在此参数。

      当资源已添加标签时，则此处显示标签名称。否则，显示“--”。
      1. 单击名称，查看总览信息、副本信息以及任务。
        • 总览

          显示SLA信息、任务执行时间、副本总数、节点和数据库等信息。

        • 副本数据

          您可以依次按照年、月、天查找相应时间的副本。

          时间下方显示，表示该时间存在副本。

        • 任务

          展示备份、恢复等任务。

    • 在“可用性组”页签浏览数据库可用性组信息。
