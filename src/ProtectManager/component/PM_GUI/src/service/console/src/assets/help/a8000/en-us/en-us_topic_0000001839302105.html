Managing Tenant Sets

The OceanProtect allows you to modify, remove, activate, or disable tenant set protection.

Log in to the management page, choose Protection > Databases > OceanBase, click Tenant Sets, and find the desired tenant set.

Table 1 describes the related operations.

Table 1 Operations related to tenant set protection

Operation

Description

Navigation Path

Modify Protection

Scenario

This operation enables you to modify a protection plan to perform SLA association again for a tenant set or modify user-defined script information. After the modification, the next backup will be performed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

Choose More > Modify Protection in the row where the tenant set is located.

Remove Protection

Scenario

This operation enables you to remove the protection plan for a tenant set when the tenant set no longer needs to be protected. After removal, the system cannot execute protection jobs for the tenant set.

Note

  • When a protection job of the database is running, the protection cannot be removed.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the associated SLA on the Protection > Protection Policies > SLAs page, rather than perform SLA re-association after protection removal.

    After you remove protection and perform SLA re-association, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

Choose More > Remove Protection in the row where the tenant set is located.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup in the row where the tenant set is located.

Disable Protection

Scenario

This operation enables you to disable the protection plan of a tenant set for which periodic protection is not required. After the protection plan is disabled, the system does not perform automatic backup jobs for the tenant set.

Note

Disabling protection does not affect ongoing backup jobs.

Choose More > Disable Protection in the row where the tenant set is located.

Activate Protection

Scenario

This operation enables you to activate the protection plan for a tenant set. After the protection plan is activated, the system backs up the tenant set based on the protection plan.

Note

This operation can be performed only on instances that have been associated with an SLA and are in the Unprotected state.

Choose More > Activate Protection in the row where the tenant set is located.

Modify Information About a Tenant Set

Scenario

This operation enables you to modify the name of a tenant set and select tenants again

Choose More > Modify in the row where the tenant set is located.

Delete a Tenant Set

Scenario

This operation enables you to delete a tenant set that is no longer used.

Note

Before deleting a tenant set, remove protection from the tenant set.

Choose More > Delete in the row where the tenant set is located.

Note: The procedure for restoring OceanBase clusters is the same as that described in Restoring OceanBase.
