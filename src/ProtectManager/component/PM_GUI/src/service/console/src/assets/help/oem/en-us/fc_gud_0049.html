Restoring FusionOne Compute VM Disks

This section describes how to restore disks of a VM that has been backed up to the original VM or a new VM.

Context

The product allows you to restore disks to the original location or a new location using backup copies or replication copies (data cannot be restored to the original location).

Procedure

  1. Choose Explore > Copy Data > Virtualization > FusionOne Compute.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    On the Resources tab page, search for the VM to be restored by its name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or date, copies exist in the month or on the day.

  4. In the row of the target copy, choose More > Disk Restoration.
  5. Select the disk to be restored and click Select.
  6. Restore the VM disk to the original location or a new location.

    • Restore the VM disk to the original location.
      1. Select Original location for restoring the VM.
      2. Target Compute Location is the original location of the VM by default.
      3. Select an agent host used for restoration using copies.

        If the agent host is not selected, the system selects one or multiple agent hosts from the agent hosts configured during resource registration to perform restoration.

      4. Select whether to automatically start the VM after the restoration.
      5. Select whether to perform copy verification before restoration.

        If this option is enabled, the integrity of a copy is verified before restoration using the copy. Copy verification affects the restoration performance and is disabled by default. If no copy verification file is generated, this option cannot be enabled.

      6. Click OK.
    • Restore the VM disk to a new location.
      1. Select New location for restoring the VM.
      2. Set information about a new location to which the VM is restored.
        Table 1 describes the related parameters.
        Table 1 New VM information

        Parameter

        Description

        Target Compute Location

        Select the virtualization environment and VM to which the backup disk is restored.

        NOTE:

        The OS type of the target VM must be the same as that of the source VM. For example, both of them use the Windows or Linux OS.

        Storage Location

        Target datastore to which the backup disk is restored.

        • Same Datastore: All disks are restored to the same datastore.
        • Different Datastores: A datastore is selected for each disk to implement data restoration.
      3. Select an agent host used for restoration using copies.

        If the agent host is not selected, the system selects one or multiple agent hosts from the agent hosts configured during resource registration to perform restoration.

      4. Select whether to automatically start the VM after the restoration.
      5. Select whether to perform copy verification before restoration.

        If this option is enabled, the integrity of a copy is verified before restoration using the copy. Copy verification affects the restoration performance and is disabled by default. If no copy verification file is generated, this option cannot be enabled.

  7. Click OK.
