Viewing a Backup Storage Unit

If there is only one storage pool in the backup storage device of the local cluster, a backup storage unit is generated by default. If there are multiple storage pools, a backup storage unit is generated by default for the storage pool where the data backup feature containerized application resides. You can manually create other backup storage units.

Procedure

  1. Choose System > Infrastructure.
  2. Click Backup Storage Unit Groups.
  3. Click Backup Storage Units to view information about the backup storage unit. Table 1 describes the related parameters.

    Table 1 Backup storage unit parameters

    Parameter

    Description

    Name

    Name of the backup storage unit. For the automatically added backup storage unit, its name is the same as that of the backup storage device.

    Health Status

    Health status of the backup storage unit.

    • Normal: The functions and running parameters of the backup storage unit (storage pool) are normal.
    • Degraded: The backup storage unit (storage pool) can work properly but cannot achieve the optimal performance.
    • Faulty: The functions of the backup storage unit (storage pool) are abnormal and the backup storage unit cannot work properly.

    Running Status

    Running status of the backup storage unit. The value can be Online, Initializing, Reconstructing, Precopy, Deleting, Balancing, or Offline.

    Device Type

    Device type of the backup storage unit.

    Storage Device

    Name of the storage device to which the backup storage unit belongs.

    Storage Pool

    Name of the storage pool corresponding to the backup storage unit.

    Capacity

    Capacity of the backup storage unit. The used capacity, total capacity, and used capacity percentage are displayed.

    Alarm Threshold

    Alarm threshold of the backup storage unit.
