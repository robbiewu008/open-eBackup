Step 4: (Optional) Modifying the agent_cfg.xml Backup File (Applicable to the OceanProtect E6000 Appliance)

In Linux, you can select the asynchronous (async) or synchronous (sync) mode when mounting a file system. In the asynchronous mode, the performance is high because writing data to the memory is faster than to a disk. In the synchronous mode, the data security is high and data is not easily lost when the system breaks down or is powered off. The mode of mounting a file system can be configured based on site requirements.

Procedure

  1. Use PuTTY to log in to the host of a production node as user root.
  2. Run the vim Root directory where the client is installed/DataBackup/ProtectClient/ProtectClient-E/conf/agent_cfg.xml command to open the agent_cfg.xml file.

    Example:

    vim /opt/DataBackup/ProtectClient/ProtectClient-E/conf/agent_cfg.xml

  3. Modify the mode of mounting a file system. The default mode is async (asynchronous).

    Change the mode of mounting a file system in the general_mount_option parameter to sync.

  4. Press Esc. Run the :wq! command to save the modifications and exit.
