Viewing the Oracle Database Environment

After the Oracle database or cluster is registered with the OceanProtect, you can view the database or cluster information on the management page.

Procedure

  1. Choose Protection > Databases > Oracle.
  2. View the related information on the Clusters tab page.

    1. This page displays information about registered Oracle clusters.

      Table 1 describes the related parameters.

      Table 1 Oracle database cluster information

      Parameter

      Description

      Name

      Cluster name.

      Status
      Cluster status.
      • Online: If there are online hosts in the cluster, the cluster status is Online.
      • Offline: If all hosts in the cluster are offline, the cluster status is Offline.

      Type

      Cluster type.

      Owner

      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of the cluster.

      If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click a cluster name to view details about hosts and databases in the cluster.

  3. View the related information on the Databases tab page.

    1. This page displays the details about the Oracle database.
      Viewing the Oracle Database Environment describes the related parameters.
      Table 2 Oracle database parameters

      Parameter

      Description

      Name

      Database name.

      Status

      Database status.

      If a database can be accessed by the OceanProtect, the status is Online. Otherwise, the status is Offline.

      IP Address

      IP address of the cluster where the database resides.

      Type

      Database type, which can be Single Server or Clusters.

      OS

      OS type of the host where the database is located.

      Version

      Database version.

      Authentication Status

      Authentication status of the database.

      If the authentication mode has been correctly configured for the database, the authentication status is Authenticated. Otherwise, the authentication status is Unauthenticated.

      SLAs

      If an SLA is associated with the resource, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Indicates whether a database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is enabled.
      • Creating: The database is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    1. Click the database name and view the database information, copy information, and related jobs.
      • Database overview

        Display the database name, SLA information, job execution time, and total number of copies.

      • Database copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, copies exist at that time.

      • Jobs related to a database

        Display all jobs related to a database, including backup, restoration, archive, and instant recovery.

        If the progress of an ongoing job is not updated for a long time, resolve this issue by referring to What Should I Do If the Progress of an Oracle Data Protection Job Remains Unchanged for More Than 10 Minutes?.
