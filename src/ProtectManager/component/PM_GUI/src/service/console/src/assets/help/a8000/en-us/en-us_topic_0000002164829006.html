Configuring the DNS Service (Applicable to the OceanProtect E6000 Appliance)

This section describes how to configure the DNS service for resolution of and access to user-defined domain names.

Prerequisites

  • The DNS server has been configured and is running properly.
  • Port 53 of the TCP/UDP protocol has been enabled for communication between the storage system and DNS server.

Context

  • A DNS server is used to resolve and access external domain names.
  • If you want to configure a standby DNS server, keep the domain names of the active and standby servers consistent.

Procedure

  1. Disable the security sandbox.

    1. Log in to DeviceManager.
    2. Choose Cluster > Cluster > Hardware > Nodes and click Disable Security Sandbox.
    3. Select all nodes, specify Password of root, and click OK.
    4. In the displayed Disable Security Sandbox dialog box, enter the password for logging in to DeviceManager, select I have read and understand the consequences associated with performing this operation, and click OK.

  2. Use PuTTY to log in to the nodes in the OceanProtect cluster as user root.

    Perform subsequent operations on all nodes in the OceanProtect cluster.

  3. Run the vim /etc/resolv.conf command to open the resolv.conf configuration file and press i to enter the insert mode. Add the following content to the configuration file:

    nameserver IP address of the DNS server

  4. Press Esc and enter :wq! to save the settings and exit.
  5. Enable the security sandbox.

    1. Log in to DeviceManager.
    2. Choose Cluster > Cluster > Hardware > Nodes and click Enable Security Sandbox.
    3. Select all nodes and click OK.
