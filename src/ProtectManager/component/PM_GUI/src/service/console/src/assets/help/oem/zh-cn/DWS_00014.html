步骤1：注册GaussDB(DWS)集群

执行GaussDB(DWS)备份恢复前，需要将GaussDB(DWS)集群注册至本产品。

前提条件

  • 已安装ProtectAgent。如果ProtectAgent部署到生产主机和独立物理服务器，还需要安装RoachClient。安装操作请参见《ProtectAgent安装指南》中的“安装roach_client（适用于GaussDB(DWS)）”章节。
  • 首次注册资源前，需要按照实际需求修改已安装ProtectAgent的GaussDB(DWS)节点的配置文件，详情请参见修改GaussDB(DWS)配置文件。
  • ProtectAgent部署到生产主机时，首次执行备份前，需添加环境变量，详情请参见ProtectAgent部署到生产主机注意事项（适用于1.6.0及后续版本）。
  • 已放开备份和恢复任务所需端口，具体请参见通信矩阵。

注意事项

GaussDB(DWS)集群注册成功后不支持修改集群名称，因名称是集群的唯一标识，需要谨慎填写。

操作步骤

  1. 选择“保护 > 大数据 > GaussDB(DWS)”。
  2. 在“集群”页面，单击“注册”，注册GaussDB(DWS)集群。

    相关参数说明如表1所示。
    表1 注册GaussDB(DWS)集群

    参数

    说明

    名称

    自定义GaussDB(DWS)集群名称。

    用户名

    运行GaussDB(DWS)集群的系统用户名称。

    环境变量文件路径

    集群用户的环境变量配置文件路径。

    节点

    请至少选择一个有CN实例，且安装了ProtectAgent的GaussDB(DWS)节点。

    代理主机

    选择安装了RoachClient的代理主机。对于内核版本为GaussDB A 8.0.0的GaussDB(DWS)集群，不需要选择代理主机。

    不支持添加已经加入到其他GaussDB(DWS)集群的代理主机。

  3. 单击“确定”。
