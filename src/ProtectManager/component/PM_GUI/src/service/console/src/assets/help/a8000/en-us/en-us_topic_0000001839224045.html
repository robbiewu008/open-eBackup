About SLA

Service level agreement (SLA) is a set of configurable data protection policies that define rules for data protection and copy retention. These policies are associated with an environment to implement protection.

To simplify data protection configuration, the OceanProtect provides users with three types of preset SLAs: Gold, Silver, and Bronze. For example, users can select Gold SLA for protecting data of mission-critical services and Bronze SLA for protecting data of web servers.

In addition, to meet enterprises' different data protection requirements for various environments, the OceanProtect allows you to customize SLAs.

Preset SLAs

The preset SLAs in the system can be applied to all application types. Each preset SLA defines the backup interval, backup time window, and first execution time of a backup job.

Table 1 describes the preset SLAs.

  • Backup Interval: First backup time and backup interval
  • Copy Retention: Copy retention period
  • Backup Time Window: Time range during which a backup job can be executed. The backup job cannot be scheduled beyond this time range.

Custom SLAs

Users can customize SLAs for backup, archive, and replication jobs. In an SLA, user can customize basic parameters such as the job execution frequency, time window, and copy retention duration, and other advanced parameters for various protected resources.

Table 2 describes different types of custom SLAs.

Table 2 Custom SLAs

SLA

Description

Backup SLA

Users can define the execution frequency and retention period of backup copies of a backup job.

Replication SLA

Users can specify the target cluster to which the backup copy is replicated, and customize the replication job execution frequency and copy retention period.

Archive SLA

Users can specify the archive storage (object storage or tape libraries) to which backup copies or replication copies are archived, and customize the archive job execution frequency and copy retention period.
