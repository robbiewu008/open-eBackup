Step 2: Checking the HDFS ACL Switch Status

If you need to back up ACL information of HDFS files, ensure that the HDFS ACL configuration switch has been turned on on the big data platform before the backup. Otherwise, skip this section.

Procedure

FusionInsight Manager or CDH as the access scenario

  1. Log in to FusionInsight Manager using a browser.

    This section uses FusionInsight 6.5.1 as an example to describe how to check the HDFS ACL switch status. The operations vary depending on the big data platform.

  2. Choose Cluster > HDFS.
  3. Click Configure.
  4. Check whether the value of dfs.namenode.acls.enabled is true.

    You can search for keyword dfs.namenode.acls.enabled or acl to quickly locate the parameter.

    • If the value of dfs.namenode.acls.enabled is true, ACL information of HDFS files is backed up by default.
    • If the value of dfs.namenode.acls.enabled is false but you need to back up ACL information of HDFS files, change the value of dfs.namenode.acls.enabled to true and then restart the HDFS service as prompted.
