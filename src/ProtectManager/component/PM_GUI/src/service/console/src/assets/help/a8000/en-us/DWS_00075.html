Configuring the Full Mapping Between ProtectAgent and the OceanProtect Appliance

Procedure

  1. Use PuTTY to log in to the CN in the GaussDB (DWS) cluster where ProtectAgent has been deployed.
  2. Run the vi /opt/DataBackup/ProtectClient/ProtectClient-E/conf/agent_cfg.xml command to edit the configuration file.
  3. Modify the configuration item agent_storage_relation in <Backup>. ProtectAgent host IP address 1 to ProtectAgent host IP address n are the IP addresses of all agent hosts that are registered with the OceanProtect and used for GaussDB (DWS) protection services. ESN 1 to ESN n are the ESNs of the OceanProtect used by agent hosts to perform protection services.

    agent_storage_relation = "Agent host IP address 1@ESN 1;Agent host IP address 2@ESN 2;...Agent host IP address n@ESN n"

    Assume that the agent hosts whose IP addresses are 192.168.10.186 and 192.168.10.13 have been registered with the OceanProtect whose ESN is 2102353GTJ10M1000001. For example, the backup service uses a backup storage unit group that contains a backup storage unit whose ESN is 2102353GTJ10M1000003. The following shows how to modify the configuration items on each agent host:

    agent_storage_relation = "192.168.10.186@2102353GTJ10M1000001;192.168.10.13@2102353GTJ10M1000003"

    After the configuration is complete, the agent host whose IP address is 192.168.10.186 backs up data to the OceanProtect whose ESN is 2102353GTJ10M1000001, and the agent host whose IP address is 192.168.10.13 backs up data to the OceanProtect whose ESN is 2102353GTJ10M1000003.
