Restoring a Single File or Multiple Files in an Oracle Database (Applicable to 1.6.0 and Later Versions)

This section describes how to restore damaged or lost files in an Oracle database to the Oracle host.

Context

The GUI may vary with versions.

Prerequisites

The client and other related software have been installed on the target host for restoration.

For details, see the OceanProtect DataBackup 1.5.0-1.6.0 Client Installation Guide.

Procedure

  1. Choose Explore > Copy Data > Databases > Oracle.
  2. Search for copies by database resource or copy. This section uses database resources as an example.

    On the Resources tab page, locate the database to be restored by database name and then click the database name.

  3. Specify a copy for restoration.

    For the OceanProtect E6000 Appliance, when a replication copy is used for restoration, other copies of the resource cannot be replicated at the remote end.

    1. On the By Date tab page, select a year, month, and day in sequence to search for copies.

      If is displayed under a month or date, copies exist in the month or on the date.

    2. Specify a copy for restoration. Click on the right, locate the row that contains the target copy, and choose More > Database File Restoration.

      Database files do not support point-in-time restoration.

    3. Restore the Oracle database to the original location.

      Select one or more files to be restored under Data Files, Log Files, Control Files, or Parameter Files.

      • If you set Target Restoration Location, ensure that the path exists on the target host and the database installation user has the read and write permissions on the path. Otherwise, the restoration job will fail. For log copies, you must set Target Restoration Location for the file to be restored.
      • If you enable Modify Database Settings when selecting parameter files, you can modify the database settings based on the actual situation of the target host. Incorrect parameter settings will cause a database restoration failure. Database parameters cannot be added or deleted.

  4. Click OK.
  5. After the file restoration job is complete, the database administrator needs to restore the database based on the restored files. For details, see the Oracle official document.
