How Do I Check the Network When an Instance Fails to Be Backed Up or Restored? (Applicable to OLTP OPS)

Symptom

An instance fails to be backed up or restored, and the error code is 0x5E254100.

Possible Causes

The backup or restoration operation fails because the network between the client and the database instance is disconnected. You can perform the following steps to check the network connectivity:

Troubleshooting

  1. Check the network connectivity between the client backup plane and the TPOPS service plane. Run the following command on the host where the client is installed. If the address can be pinged, the network connection is normal.

    ping Address used by TPOPS to provide services for external systems

  2. Check the network connectivity between the client backup plane and the GaussDB instance service plane. Run the following command on the GaussDB instance node. If the IP address can be pinged, the network connection is normal.

    ping Address used by backupAgent on the host where the client is installed to provide services for external systems
