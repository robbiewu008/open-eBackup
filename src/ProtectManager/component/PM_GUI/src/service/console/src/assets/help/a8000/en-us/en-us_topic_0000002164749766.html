Managing Clusters

The OceanProtect allows you to modify, remove, activate, or disable protection for Kubernetes clusters.

Related Operations

You need to log in to the management page and choose Protection > Containers > Kubernetes CSI to go to the Cluster tab page.

Table 1 describes the related operations.

Table 1 Cluster-related operations

Operation

Description

Navigation Path

Scanning for cluster resources

Scenario

The OceanProtect automatically scans Kubernetes clusters every hour. If a namespace or dataset in a registered Kubernetes cluster has changed, the OceanProtect automatically updates the namespace or dataset after scanning. This operation enables you to immediately update the changed information to the OceanProtect.

Note

If you do not choose to apply SLA policies to the newly created dataset when creating a periodic backup job, the system will not associate the newly discovered table with an SLA after scanning.

Choose More > Resource Scan.

Testing cluster connectivity

Scenario

Manual connectivity test of the registered Kubernetes cluster is used to check the connectivity between the cluster and the production environment.

Choose More > Test Connectivity.
Authorizing resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Note

  • The protection permissions on a resource that has been associated with an SLA cannot be granted. Before permission granting, remove the protection.
  • The protection permissions on a single resource can be granted only to a single data protection administrator.

Choose More > Authorize Resource and select the data protection administrator for authorization.
Reclaiming resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

The protection permissions on a resource that has been associated with an SLA cannot be reclaimed. Before reclamation, remove the protection.

Choose More > Reclaim Resource.

Modifying the cluster registration information

Scenario

This operation enables you to change the registered Kubernetes cluster name or to synchronize the changes of the registered Kubernetes CSI configuration file, management IP address of storage resources, username, and password.

Note

Modify the registration information when no backup or restoration job is running. Otherwise, the job will fail.

Choose More > Modify.

Table 1 describes the parameters of Kubernetes cluster registration information.

Deleting a cluster

Scenario

If you no longer need to protect a Kubernetes cluster or use it as a restore target, you can perform this operation to delete the Kubernetes cluster.

Note

If a namespace or dataset in a cluster has been associated with an SLA, the cluster cannot be deleted. Disassociate the namespace or dataset from the SLA before deletion.

Choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
