Obtaining the kubeconfig Configuration File

This section describes how to obtain the kubeconfig configuration file when you need to register a cluster in kubeconfig authentication mode.

Procedure

  1. Log in to the Kubernetes cluster in the background.
  2. Run the following command to create a service user with the specified name. The following steps use the created username test as an example.

    oc create serviceaccount Username

  3. Run the following command to grant the cluster-admin role to user test:

    oc adm policy add-cluster-role-to-user cluster-admin -z test

  4. Run the following command to set the parameter of the token to be exported. In the following command, the value of duration indicates the period when the token takes effect, and 10 minutes is used as an example.

    export TOKEN=$(oc create token test --duration=600s)

  5. Run the following command to export the kubeconfig configuration file. 192.168.0.2:6443 indicates the IP address and port number of the Kubernetes cluster, and /tmp/serviceaccount-kubeconfig indicates the export path of the kubeconfig configuration file. You can customize the export path.

    oc login --server=192.168.0.2:6443 --token=$TOKEN --kubeconfig=/tmp/serviceaccount-kubeconfig
