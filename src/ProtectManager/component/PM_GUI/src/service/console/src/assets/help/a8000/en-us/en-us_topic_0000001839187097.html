Managing Protection for Clusters/Hosts/VMs/VM Groups

The OceanProtect allows you to modify, remove, activate, or disable protection plans for clusters, hosts, VMs, or VM groups. (Only 1.6.0 and later versions support management of protection for VM groups.) This section uses a VM as an example to describe related operations. The operations on a cluster, host, or VM group are similar.

You need to log in to the management page, choose Protection > Virtualization > VMware, and find the target VM.

Table 1 describes the related operations.

Table 1 Operations related to VM protection plans

Operation

Description

Navigation Path

Modify Protection

Scenario

You can modify a protection plan of a VM to associate the SLA with the VM again, add or remove a protected VM disk, or modify the agent host. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target VM, choose More > Modify Protection.

Remove Protection

Scenario

You can remove the protection if the VM no longer needs to be protected. After the removal, the system cannot execute protection jobs for the VM. To protect the VM again, you need to associate an SLA with the VM again.

Note

  • When the protection job of the VM is running, the protection cannot be removed.
  • When removing protection for VMs in batches, ensure that the selected VMs have been associated with SLAs.

Choose More > Remove Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup.

Activate Protection

Scenario

If the protection plan of a VM is disabled, you can perform this operation to activate it again. After the protection plan is activated, the system backs up, archives, or replicates the VM data based on the protection plan.

Note

You can perform this operation only on a VM that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection.

Disable Protection

Scenario

This operation allows you to disable the protection plan of a VM when periodic protection on the VM is not required. After the protection plan is disabled, the system does not automatically back up, archive, or replicate the VM.

Note

  • Disabling a protection plan does not affect ongoing protection jobs.
  • You can perform this operation only on a VM that has been associated with an SLA and is in the Protected state.

Choose More > Disable Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target VM, choose More > Manual Backup.

Note: The procedure for restoring VMware VMs is the same as that described in Restoration.
