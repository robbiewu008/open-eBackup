Restoration Failed Because the Replica Consistency Protocol of the Database Instance Is Inconsistent with That of the Restoration Target Instance

Symptom

When a database restoration job is being executed, an error is reported indicating that the replica consistency protocol of the instance is inconsistent with that of the restoration target instance.

Possible Causes

When a GaussDB restoration job is being executed, the replica consistency protocol of the instance must be consistent with that of the restoration target instance.

Troubleshooting

  1. Run the following command in the database of the instance to check the consistency protocol:

    show enable_dcf;
    • If off is displayed in the command output, the replica consistency protocol of the backup copy is Quorum.
    • If on is displayed in the command output, the replica consistency protocol of the backup copy is Paxos.

  2. Select an instance whose replica consistency protocol is the same as that of the backup copy for a restoration job.
