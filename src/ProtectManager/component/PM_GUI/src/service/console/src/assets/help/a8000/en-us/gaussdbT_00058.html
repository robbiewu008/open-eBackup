Managing Databases

The OceanProtect allows you to perform operations on databases, such as modifying, removing, activating, and disabling protection.

Related Operations

You need to log in to the OceanProtect WebUI, choose Protection > Databases > GaussDB T, and find the target database.

Table 1 describes the related operations.
Table 1 Database protection operations

Operation

Description

Navigation Path

Modifying protection

Scenario

This operation enables you to modify the protection plan to reselect an SLA associated with a database. After the modification, the next backup will be executed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row of the target database, choose More > Modify Protection.

Removing protection

Scenario

This operation enables you to remove the protection plan if a database no longer needs to be protected. After the removal, the system cannot execute protection jobs for the database. To protect the database again, you need to associate an SLA with it again.

Note

  • When the protection job of the database is running, protection cannot be removed.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the associated SLA on the Protection > Protection Policies > SLAs page, rather than perform SLA re-association after protection removal.

    After you remove protection and perform SLA re-association, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row of the target database, choose More > Remove Protection.

Manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target database, choose More > Manual Backup.

Disabling protection

Scenario

This operation enables you to disable the protection plan of a database for which periodic protection is not required. After the protection plan is disabled, the system does not execute automatic backup jobs for the database.

Note

Disabling protection does not affect ongoing backup jobs.

In the row of the target database, choose More > Disable Protection.

Activating protection

Scenario

This operation enables you to activate a disabled protection plan of a database. After the protection plan is activated, the system backs up the database based on the protection plan.

Note

This operation can be performed only for a database that has been associated with an SLA and is in Unprotected state.

In the row of the target database, choose More > Activate Protection.

Deleting a database

Scenario

This operation enables you to delete a database when it no longer needs to be protected or is used as the restoration target.

Note

If a database has been associated with an SLA, the database cannot be deleted. You need to remove the database protection before deleting the database.

In the row of the target database, choose More > Delete.

Testing connectivity

Scenario

This operation enables you to perform manual connectivity test of the registered database to check the connectivity between the database and the agent host and between the database and the production environment.

In the row of the target database, choose More > Test Connectivity.

Modifying database information

Scenario

This operation enables you to modify the name, username, deployment mode, and node of a database.

In the row of the target database, choose More > Modify.
Authorizing resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Note

  • The protection permissions on a resource that has been associated with an SLA cannot be granted. Before permission granting, remove the protection.
  • The protection permissions on a single resource can be granted only to a single data protection administrator.

In the row of the target resource, choose More > Authorize Resource. Then, select the desired data protection administrator.
Reclaiming resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

The protection permissions on a resource that has been associated with an SLA cannot be reclaimed. Before reclamation, remove the protection.

In the row of the target resource, choose More > Reclaim Resource.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The configuration of database restoration is similar to Restoring the GaussDB T Database and is not described in this section.
