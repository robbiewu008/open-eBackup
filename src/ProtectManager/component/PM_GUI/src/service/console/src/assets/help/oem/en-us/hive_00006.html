Constraints

  • Backup and restoration jobs cannot be executed for a resource at the same time.
  • For 1.5.0, replication and restoration jobs cannot be executed for a resource at the same time.
  • If an expiration or deletion job is triggered for a copy, the copy cannot be used for replication and restoration jobs.
  • Multiple backup jobs cannot be executed for a resource at the same time.
  • After resource restoration is executed, the next backup is a full backup by default.
  • Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.
