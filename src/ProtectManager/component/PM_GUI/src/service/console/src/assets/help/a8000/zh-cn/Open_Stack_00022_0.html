步骤12：执行备份

执行备份前将需要保护的资源关联到指定SLA策略，系统将对这些资源按照SLA策略进行保护，并根据SLA策略周期性执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

注意事项

  • 云服务器状态处于以下状态时支持备份：运行中、关机、挂起。
  • 由于代理主机最多允许挂载24块磁盘，因此在做备份恢复时，如果代理主机已挂载了24块磁盘，系统将等待有空闲槽位时再执行备份任务。

背景信息

  • 手动备份时，SLA中定义的参数（“失败后自动重试”除外）会应用到手动备份任务中。
  • 手动备份产生的副本保留时长按照SLA中定义的保留时长进行保留。

    如果手动备份时选择的保护策略未在SLA中定义，则产生的备份副本保留时长为永久保留。如SLA中未定义全量备份，而手动备份时选择的是全量备份，则本次手动全量备份产生的副本保留时长为永久保留。

  • 如果SLA定义了复制策略或归档策略，则执行手动备份时，系统会按照SLA的定义执行一次复制或归档。

操作步骤

  1. 选择“保护 > 云平台 > OpenStack”。
  2. 周期性备份项目。

    1. 选择“项目”页签，选择需要保护的项目。
    2. 单击“保护”。

      系统将备份该项目下的所有云服务器。

    3. 选择系统预置或用户自行创建的SLA。

      您也可以单击“创建”，创建新的SLA。

      如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

    4. 选择代理主机。

      当用户未选择时，系统会从注册资源时配置的代理主机中选择1到多个代理主机执行备份。

    5. 设置以下参数。
      1. 选择是否覆盖云服务器已有的SLA。
        默认为“不覆盖云服务器已有的SLA”。

        后续修改项目关联的SLA时，修改时选择“不覆盖云服务器已有的SLA”，如果云服务器的SLA与项目本身的SLA一致，则云服务器的SLA会被覆盖。

      2. 选择SLA应用策略，支持同时选择以下一个或多个选项：
        • 将项目/资源集SLA应用到所有未关联SLA的云服务器上。
        • 将项目/资源集SLA应用到新创建的云服务器上。
      3. 指定保护计划中要过滤的云服务器的过滤方式。可选择“先包含再排除”、“仅包含”或“仅排除”进行对应的方式过滤添加保护的云服务器。

        您可以通过云服务器名称或名称的关键词指定排除或包含部分云服务器，多个云服务器可用英文逗号隔开。

        支持精确匹配和模糊匹配云服务器名称，模糊匹配规则包括：前缀匹配（如abc*）、后缀匹配（如*abc）、中间匹配（如*abc*）。

    6. 单击“确定”。

      如果当前系统时间已晚于该SLA首次备份的开始时间，则关联备份SLA后可以选择在当前立即执行备份或者根据SLA中设置的备份策略等待下次调度。

  3. 周期性备份云服务器/云服务器组。

    1. 选择“云服务器”/“云服务器组”页签，选择需要保护的云服务器/云服务器组。
    2. 单击“保护”。
    3. 选择是否自动保护磁盘。

      仅周期性备份云服务器需要操作此步骤。

      系统默认开启“自动保护磁盘”，备份云服务器下的所有磁盘。您可以关闭“自动保护磁盘”，单击“请选择”手动选择部分云磁盘备份。

      系统默认开启自动保护磁盘，开启后将保护云服务器下的所有磁盘，包含新发现的磁盘。关闭自动保护后，需手动选择磁盘进行保护。

      如果系统盘被排除，则生成的副本不支持文件级恢复。

    4. 选择系统预置或用户自行创建的SLA。

      您也可以单击“创建”，创建新的SLA。

    5. 选择代理主机。

      当用户未选择时，系统会从注册资源时配置的代理主机中选择1到多个代理主机执行备份。

    6. 设置备份副本自动索引（仅1.6.0及后续版本支持此参数）。
      当选择虚拟机或虚拟机组进行保护，且备份的磁盘包含了系统盘时，开启后，系统会对备份副本自动创建索引。使用该副本进行恢复时，支持选择副本中的单个或多个文件进行恢复，支持搜索副本中的文件、目录或链接。

      索引需要占用备份存储空间，当系统中的索引文件总数超过10亿时，请按照《OceanProtect 备份一体机 1.5.0-1.6.0 故障处理》中的“创建副本索引失败”章节进行处理。

    7. 单击“确定”。

      如果当前系统时间已晚于该SLA首次备份的开始时间，则关联备份SLA后可以选择在当前立即执行备份或者根据SLA中设置的备份策略等待下次调度。

  4. 可选：执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。

    1. 根据需要备份的类型，选择“项目”、“云服务器”页签。

      选择“项目”进行备份时，仅会备份与项目SLA相同的云服务器，不会备份与项目SLA不同的云服务器。

    2. 在需要备份的项目或云服务器所在行右侧，单击“更多 > 手动备份”。

      支持选择多个资源批量进行手动备份。勾选多个资源，在页面左上角选择“更多 > 手动备份”。

    3. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    4. 选择保护策略：全量备份或永久增量（合成全量）备份。

      对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    5. 单击“确定”。
