Configuring Domain Name Resolution (Applicable to OpenStack)

You must add domain name resolution of each OpenStack service to the agent host for normal communication between the host and OpenStack. If the OpenStack community edition is used, skip this section.

Procedure

  1. Obtain domain name resolution information of each OpenStack service.

    1. Use a browser to log in to FusionSphere OpenStack OM.
    2. Choose Resources > Configuration from the navigation pane.
    3. Click the Services tab to obtain the domain names of the nova, keystonev3, cinder, and neutron services. Table 1 shows an example.
      Table 1 Example of obtaining domain names

      Service Name

      Access Address

      nova

      compute.az236.dc236.test.com

      keystonev3

      identity.az236.dc236.test.com

      cinderv3

      volume.az236.dc236.test.com

      neutron

      network.az236.dc236.test.com

  2. Use PuTTY to log in to the agent host.
  3. Open the hosts file of the agent host and add the obtained domain name resolution to the file. The format is as follows:

    IP address of the reverse proxy of the external_api plane   Domain name

    Example:

    192.168.10.10    compute.az236.dc236.test.com
192.168.10.10    identity.az236.dc236.test.com
192.168.10.10    volume.az236.dc236.test.com
192.168.10.10    network.az236.dc236.test.com
