Managing ClickHouse Clusters/Table Sets

This section describes how to modify or delete ClickHouse clusters registered with the product.

Related Operations

Choose Protection > Big Data > ClickHouse, click the Cluster or Table Sets tab, and locate the cluster on which you want to perform related operations.

Table 1 describes related operations.

Table 1 ClickHouse-related operations

Operation

Description

Navigation Path

Modifying cluster information

Scenario

This operation enables you to change the authentication mode and other information about a ClickHouse cluster.

In the row where the cluster resides, choose More > Modify.

For details about ClickHouse cluster parameters, see Step 1: Registering a ClickHouse Cluster.

Deleting a cluster

Scenario

This operation enables you to delete a ClickHouse cluster if you no longer need to protect databases or table sets in this cluster or restore data to it.

Note

  • A cluster cannot be deleted if a backup or restoration job is being executed.
  • A cluster cannot be deleted if a database or table set in it has been associated with an SLA.
  • Deletion of a cluster will delete all its databases and tables.

In the row where the cluster resides, choose More > Delete.
Authorizing resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize this administrator as the system administrator to protect resources.

Note

  • The protection permissions on a resource that has been associated with an SLA cannot be granted. Before permission granting, remove the protection.
  • The protection permissions on a single resource can be granted only to a single data protection administrator.

Choose More > Authorize Resource and select the data protection administrator for authorization.
Reclaiming resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

The protection permissions on a resource that has been associated with an SLA cannot be reclaimed. Before reclamation, remove the protection.

Choose More > Reclaim Resource.

Modifying a table set

Scenario

Only table sets can be modified. This operation enables you to modify the name of and selected tables in a table set.

In the row where the table set resides, choose More > Modify.

Deleting a table set

Scenario

Only table sets can be deleted. This operation enables you to delete a table set if you no longer need to protect it.

Note

A table set associated with an SLA cannot be deleted.

In the row where the table set resides, choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
