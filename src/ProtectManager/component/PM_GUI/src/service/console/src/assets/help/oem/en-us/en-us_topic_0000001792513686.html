Managing Redis Clusters

This section describes how to modify or delete Redis clusters registered with the product.

Related Operations

You need to log in to the management page, choose Protection > Big Data > Redis, and find the target cluster.

Table 1 describes the related operations.

Table 1 Operations related to a Redis cluster

Operation

Description

Navigation Path

Modifying cluster information

Scenario

This operation enables you to change the authentication mode and other information of a Redis cluster.

In the row that contains the target cluster, choose More > Modify.

For details about Redis cluster parameters, see Step 1: Registering a Redis Cluster.

Deleting a cluster

Scenario

This operation enables you to delete a Redis cluster if you no longer need to protect it or restore data to it.

Note

  • A cluster cannot be deleted if a backup or restoration job is being executed.
  • A cluster that has been associated with an SLA cannot be deleted.

In the row that contains the target cluster, choose More > Delete.

Authorizing cluster resources

Scenario

By default, only the system administrator has the permission to protect resources. You can grant this permission to a data protection administrator through the system administrator.

Note

  • Users cannot be authorized to protect resources that have been associated with SLAs. You need to remove the protection of resources before performing authorization.
  • A single resource can be authorized only to a single data protection administrator.

In the row that contains the target cluster, choose More > Authorize Resource.

Reclaiming cluster resources

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

Users are not allowed to reclaim resources that have been associated with SLAs. You need to remove the protection of resources before reclaiming them.

In the row that contains the target cluster, choose More > Reclaim Resource.
