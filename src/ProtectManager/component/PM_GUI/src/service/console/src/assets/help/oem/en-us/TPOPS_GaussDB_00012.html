Step 4: Enabling Instance Monitoring on the TPOPS Management Page

TPOPS depends on the instance monitoring module to periodically obtain instance status information, and other modules will display or analyze the obtained information based on the real-time status information. If the monitoring stops, TPOPS cannot obtain the latest instance status data, which affects alarm reporting. As a result, modules depending on real-time data collection cannot work properly, and backup or restoration cannot be performed.

You can determine whether to perform operations in this section based on the management platform in use.
  • For the OLTP OPS management platform, perform operations in this section.
  • For the cloud database GaussDB management platform, skip this section.

Procedure

  1. Log in to the TPOPS management page.

    The TPOPS login URL varies according to the TPOPS version.

    • https://TPOPS management plane IP address:8443/gaussdb, for example, https://192.168.10.2:8443/gaussdb
    • https://TPOPS management plane IP address:7070/APW, for example, https://192.168.10.3:7070/APW

  2. Check whether monitoring has been enabled for the instance to be backed up. If not, choose More > Start Monitoring on the right of the row that contains the instance and wait until monitoring is enabled.
