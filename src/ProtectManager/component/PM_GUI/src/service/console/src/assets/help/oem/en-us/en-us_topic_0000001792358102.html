Checking the Informix Environment

Procedure

  1. Choose Protection > Databases > Informix.
  2. View the related information on the Clusters tab page.

    1. This page displays the details about the cluster where the Informix database resides.

      If the product has discovered the Informix cluster, information about the cluster and all hosts in the cluster is displayed on the page.

      Table 1 describes the related parameters.

      Table 1 Information about the cluster where the Informix database resides

      Parameter

      Description

      Name

      Name of the cluster where a database resides.

      Status
      Cluster status.
      • Online: When the host is powered on and ProtectAgent on the host is running properly, Status of the host is Online.

        If there are online hosts in the cluster, the cluster status is Online.

      • Offline: If the host is shut down or ProtectAgent on the host is faulty, Status of the host is Offline.

        If all hosts in the cluster are offline, the cluster status is Offline.

      Log Backup

      Whether log backup is enabled.

      Owner

      Data protection administrator of a cluster.

      If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
    2. Click a cluster name to view details about hosts and databases in the cluster.

  3. View the related information on the Instance tab page.

    1. This page displays details about Informix instances.
      Table 2 describes the related parameters.
      Table 2 Informix instance information

      Parameter

      Description

      Name

      Database instance name.

      Status

      Database status.

      Owning Cluster

      Name of the cluster where a database resides.

      Version

      Database version.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Indicates whether a database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is enabled.
      • Creating: The database is associated with an SLA and protection is being created.
    1. Click the name and check the overview, copy information, and jobs.
      • Overview

        Displays SLA information, job execution time, and total number of copies.

      • Copy Data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Jobs

        Displays backup and restoration jobs.
