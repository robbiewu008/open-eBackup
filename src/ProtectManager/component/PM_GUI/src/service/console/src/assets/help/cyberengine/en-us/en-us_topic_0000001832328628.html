Restoring Management Data After System Reinstallation

This section describes how to restore management data that is backed up before reinstallation to the current system.

Precautions

  • Restoration using the management data backed up on the SFTP server will overwrite the system management data. Exercise caution when performing this operation.
  • When you use the management data backed up on the SFTP server to restore the data, the ransomware detection snapshots generated after the system is reinstalled are not displayed on the page. Consequently, these ransomware detection snapshots cannot be used for restoration.
  • After the system is reinstalled, it takes about 10 minutes to restore the management data. After the restoration, services can be used properly.

Procedure

  1. Obtain the management data backup package that is to be used for restoration from the SFTP server.
  2. Choose System > Settings.
  3. In the navigation tree, click Management Data Backup.
  4. On the Management Data Backup page, click Import Backup. In the window displayed on the right, select the management data backup package obtained in 1 and click OK.
  5. In the row that contains the imported management data backup, choose More > Restore.
  6. Enter the data encryption password set during backup policy configuration and click OK.
  7. Read the warning message carefully, select I have read and understand the consequences associated with performing this operation., and click OK.
