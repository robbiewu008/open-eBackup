Function Overview

Backup

  • Full backup: backs up all data at a specific point in time, regardless of whether the data has been modified since the last backup. In the case of large data volumes, this backup type may take a long time and occupy a large amount of storage space.
  • Forever incremental (synthetic full) backup: backs up only the data that is added or modified since the first full backup. Compared with full backup, forever incremental backup significantly improves the backup efficiency in scenarios with large data volumes. If no full copy exists, the first forever incremental (synthetic full) backup will be a full backup, and incremental backups are performed subsequently. Copies generated by forever incremental (synthetic full) backups are forever incremental (synthetic full) copies, which improve the restoration performance and do not occupy extra space.
  • Rate limiting policy: limits the bandwidth for backup, replication, and archive.
  • Source deduplication: deduplicates backup data at the source end before the data is sent to the target end. This technology saves network transmission bandwidth and storage space.
  • Transmission and storage encryption: adopts the AES256 encryption algorithm to encrypt backup data.
  • Automatic retry upon failure: allows the system to automatically retry if a periodic backup job fails. The number of retries ranges from 1 to 5, and the wait time ranges from 1 to 30 minutes. For example, if the number of retries is set to 3 and the wait time is set to 5 minutes, the system retries for a total of three times at an interval of 5 minutes.

Replication

  • Backup copy replication: replicates backup copies from the source end to the target end. If a disaster occurs in the local data center, the replication copies can be used to restore service data.

Restoration

  • Copy-based restoration to the original or a new location: can be implemented using backup copies and replication copies (data cannot be restored to the original location).
