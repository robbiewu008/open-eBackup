Enabling File Interception

This section describes how to enable file interception.

Prerequisites

NFS or CIFS sharing has been enabled for the file system.

Precautions

  • When file interception is enabled for the secondary file system of a HyperMetro domain or remote replication vStore pair, file name extension filtering rules do not take effect on the corresponding storage device due to the restrictions of the HyperMetro or remote replication function of the storage device. In this case, the OceanCyber generates an alarm. After file interception is enabled for the corresponding primary file system, the NAS HyperMetro configuration is automatically synchronized to the secondary file system. In this case, you can manually scan the file system on the OceanCyber. The alarm is cleared after the file name extension association status becomes normal.
  • If DataTurbo sharing is enabled for file systems of the storage device, file interception cannot be enabled.
  • Real-time detection applies to OceanProtect 1.3.0, OceanStor Dorado 6.1.6, and their later versions, but it does not apply to file systems with DataTurbo enabled.

Procedure

  1. In the navigation pane, choose Data Security > File Interception.
  1. On the File Systems tab page, select the file system for which you want to enable file interception and choose More > Enable File Interception.

    • You can also select multiple file systems for which you want to enable file interception and choose Enable File Interception in the upper part.
    • If file interception is not required, choose More > Disable File Interception. Alternatively, select multiple file systems for which you want to disable file interception and choose Disable File Interception in the upper part.

  2. In the dialog box that is displayed, read the content carefully and click OK.
  3. Click Number of File Name Extension Filtering Rules of the file to view the association status of file name extension filtering rules. Table 1 describes the association status. If the association status is Applied, file name extension filtering rules have taken effect.

    Table 1 Association status of file name extension filtering rules

    Association Status

    Description

    Associated

    A file name extension filtering rule has been associated with a file system.

    Applied

    A file name extension filtering rule has been applied to the file system. You are not allowed to create, open, and rename operations on files with the same extension in the file system.

    NOTE:

    When accessing a file system through the CIFS share, you must open the file before deleting it. Therefore, the deletion operation on files with the same extension in the file system is also intercepted.

    Applying

    A file name extension filtering rule is being applied to the file system.

    Disassociating

    A file name extension filtering rule is being disassociated from the file system. Application to the file system will also be canceled.

    Canceling Application

    The application of a file name extension filtering rule to the file system is being recalled.

    Exception

    The applied file name extension filtering rule does not take effect for some file system shares.
