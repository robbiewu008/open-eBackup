Step 3: Manually Configuring Environment Variables

Before executing a backup job for MySQL or MariaDB, you need to register MySQL or MariaDB as the system service and manually configure the mysqlbinlog, mysqldump, mysqld, and mysql environment variables on all nodes.

Procedure

  1. Log in to the host where the MySQL or MariaDB instance is located.
  2. Run the systemctl status mysqld or service mysqld status command to check the running status of the MySQL or MariaDB instance.

    • If any running status is displayed in the command output, MySQL or MariaDB has been registered as a system service. In this case, go to 4.
    • If no running status is displayed in the command output, MySQL or MariaDB needs to be registered as a system service.
      1. Find mysql.server in the MySQL or MariaDB installation directory.
      2. Run the cp Installation directory/mysql.server /etc/init.d/mysql command to register the system service.

  3. Run the following commands in sequence to set environment variables:

    ln -sf /Installation directory/bin/mysqlbinlog /usr/bin/

    ln -sf /Installation directory/bin/mysql /usr/bin/

    ln -sf /Installation directory/bin/mysqldump /usr/bin/

    ln -sf /Installation directory/bin/mysqld /usr/bin/

  4. Run the mysqlbinlog --version, mysqldump --version, mysqld --version, and mysql --version commands separately to check whether the environment variables have been set. If a version number is displayed in each command output, the environment variables have been set successfully.
