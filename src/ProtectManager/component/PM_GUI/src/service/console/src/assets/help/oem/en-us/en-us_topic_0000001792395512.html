Restoring an ECS or an EVS Disk

This section describes how to restore an ECS or EVS disk that has been backed up to the original or a new location.

Context

The product allows you to restore ECSs or EVS disks to the original or a new location using backup copies or replication copies (data cannot be restored to the original ECS).

  • Restoration to the original location

    Restores data to an ECS in the current resource set.

  • Restoration to a new location

    Restoration to the following new locations is supported:

    • Another resource set of the Huawei Cloud Stack tenant to which the copy belongs
    • Resource set of another Huawei Cloud Stack tenant to which the copy belongs
    • Another Huawei Cloud Stack platform

Precautions

  • An ECS can be restored only when it is in the following states: Running, Starting, Stopping, Stopped, Reinstalling OS, Reinstalling OS failed, Changing OS, and OS Change failed.
  • If only some EVS disks that have undergone volume management (such as logical volume management on Linux or dynamic volume management on Windows) on ECSs are restored, data on these EVS disks may not be properly accessed.
  • During the restoration, the ECS will be locked and you are not allowed to perform any configuration operations on the ECS.
  • Copies of Huawei Cloud Stack 6.5.1 cannot be used for restoration to Huawei Cloud Stack 8.X.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Huawei Cloud Stack.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the ECS to be restored by ECS name and click the name.

  3. O the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. Locate the copy to be restored and choose More > Restoring EVS Disks.
  5. Select the EVS disk to be restored and click Select on the right of Restoration Target Location.

    You can restore an ECS by selecting all EVS disks in the ECS copy.

  6. Restore data to the original location or a new location.

    • Restoration to the original location
      Table 1 describes the related parameters.
      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Agent Host

      Select an agent host used for copy restoration.

      Power on Automatically After Restoration

      If this option is enabled, the ECS automatically starts after the restoration is successful. This option is disabled by default.

      Copy Verification Before Restoration

      If this option is enabled, the integrity of a copy is verified before the copy is restored. This operation affects the restoration performance. This option is disabled by default. If no copy verification file is generated, this option cannot be enabled.
    • Restoration to a new location
      Table 2 describes the related parameters.
      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Cloud Server

      Select the ECS to which the backup copy is restored.

      NOTE:

      The OS type of the target ECS must be the same as that of the original ECS. For example, both of them use the Windows or Linux OS.

      Agent Host

      Select an agent host used for copy restoration.

      Power on Automatically After Restoration

      If this option is enabled, the ECS automatically starts after the restoration is successful. This option is disabled by default.

      Copy Verification Before Restoration

      If this option is enabled, the integrity of a copy is verified before the copy is restored. This operation affects the restoration performance. This option is disabled by default. If no copy verification file is generated, this option cannot be enabled.

  7. Click OK.
  8. Select the target EVS disk for each EVS disk to be restored.

    • Data on data disks cannot be restored to system disks, but data on system disks can be restored to data disks.
    • The restoration can be successful only when the target EVS disk is in the Running status and its capacity is greater than or equal to that of the EVS disk to be restored.

  9. Click OK.
