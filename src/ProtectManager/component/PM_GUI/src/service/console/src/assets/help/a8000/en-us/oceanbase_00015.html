Step 7: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • Copies generated by manual backup are retained for the period defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy is retained permanently. For example, if incremental backup is not defined in the SLA but is selected for the manual backup job, the copies generated by the manual incremental backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform a manual backup.
  • During manual backup, the parameters defined in the SLA (except Automatic Retry), such as Rate Limiting Policies, are applied to the manual backup job.

Procedure

  1. Choose Protection > Databases > OceanBase.
  2. On the Clusters or Tenant Sets tab page, select the cluster or tenant set to be protected and click Protect.

    You can select multiple clusters or tenant sets for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resource to be protected, select an SLA without a WORM policy to create protection to avoid WORM policy conflicts.

  4. Click OK.

    If the current system time is later than the start time of the first backup set in the SLA, you can choose to perform the backup immediately or wait for the next scheduling based on the backup policy set in the SLA after the backup SLA is associated with the resource.

  5. Optional: Execute manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.
    1. In the row where the cluster or tenant set to be backed up resides, choose More > Manual Backup.
    2. Set the name of the copy generated during manual backup.

      If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup, Incremental backup, or Log Backup.
      • Tenant sets only support full backup.
      • For 1.6.0 and later versions: If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.
    4. Click OK.
