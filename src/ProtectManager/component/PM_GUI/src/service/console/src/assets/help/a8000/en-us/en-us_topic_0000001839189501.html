Viewing openGauss Copy Information

This section describes how to view openGauss copy details.

Procedure

  1. Choose Explore > Copy Data > Databases > openGauss.
  2. Select a copy viewing mode to view copy information.

    Replication copies can be viewed only in the target cluster.

    • Click Resources to view the imported copy information by openGauss resource. If an openGauss cluster is faulty and you need to recover the cluster, you are advised to search for the copy of the corresponding time point through this entry.

      Table 1 describes related parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Name

      Unique identifier of an openGauss cluster.

      Click the name to view openGauss protection details.

      Type

      openGauss copy type.

      • Database
      • Instance

      Location

      Protected openGauss cluster resource.

      Status

      Whether the database exists in the current cluster.

      • Existing
      • Not existing

      Copy

      Quantity

      Total number of copies generated by openGauss. The number of replication copies is displayed only in the cluster of the target OceanProtect.

      Replicated Copy SLA

      Archive SLA associated with a replication copy.

      If an archive SLA has been configured for the replication copy generated by the resource, the SLA name is displayed.

      Replicated Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is enabled.
      • Creating: The replication copy is associated with an SLA and protection is being created. The status is available only in 1.5.0 and earlier versions.
    • Click Copy to view all copy information by copy. When you need to recover data by using a specified copy, you are advised to search for the copy through this entry.

      Table 2 describes related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copy

      Copy Time

      Time when a copy is generated.

      Click Copy Time to view the expiration time and other information of the copy. If -- is displayed for the expiration time, the copy is retained permanently.

      Status

      Copy status.

      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is in progress of recovery or instant recovery.
      • Deletion failed: The copy fails to be deleted.

      Location

      Location of a copy.

      • If the copy generation mode is Replicate, the copy is stored in the local cluster. In this case, Local is displayed.
      • If the copy generation mode is Archive:
        • If the copy is stored in the object storage, the bucket name is displayed here.
        • If the copy is stored on a tape, the media set name is displayed.

      Generation Mode
      Mode in which a copy is generated.
      • Replicate: Copies generated by replication jobs.
      • Backup: Copies generated by backup jobs.
      • Object Storage Archive: Copies generated by archive jobs (object storage as the archive storage).
      • Tape Archive: Copies generated by archive job (tape library as the archive storage).
      • Cascaded Replication: Copies generated by cascaded replication jobs.
      • Reverse Replication: Copies generated by reverse replication jobs.

      Copy Type

      Backup type of a generated copy.

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.

      Resources

      Name

      Unique identifier of an openGauss cluster.

      Type

      openGauss copy type.

      • Database
      • Instance

      Location

      Protected openGauss cluster resource.

      Status

      Whether a resource is registered with the current cluster.

      • If yes, the status is Existing.
      • If no, the status is Not existing.
