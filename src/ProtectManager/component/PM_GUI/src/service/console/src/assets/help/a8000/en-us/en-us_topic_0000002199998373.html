Failure to Restore a Cluster Copy to a New Cluster Because the Temporary Directory /tmp/omm_mppdb Does Not Exist.

Fault Symptom

After a cluster copy is restored to a new cluster, the configuration file has been uploaded to all nodes in the new cluster, and the read permission has been assigned to the cluster user, the error message "The database user cannot access the installation configuration file." is still displayed in the task details.

Possible Cause

The temporary directory /tmp/omm_mppdb of the GaussDB (DWS) node fails to be created or is incorrectly deleted in an abnormal scenario.

Suggestion

  1. Log in to any node of the GaussDB (DWS) cluster.
  2. Run the following commands in sequence to switch to the cluster user and load environment variables:

    su - Cluster username
    source Environment variable file path

  3. Run the gs_ssh -c 'hostname' command to check whether the command output of each node contains the "scp: /tmp/omm_mppdb/: Not a directory" error information.

    • If the error information is displayed, check whether the /tmp/omm_mppdb/ directory exists on the corresponding node. If the directory does not exist, log in to the node, manually create the /tmp/omm_mppdb/ directory, set the read and write permissions for the cluster user, and try again.
    • If no error information is displayed, collect logs and contact technical support for assistance.
