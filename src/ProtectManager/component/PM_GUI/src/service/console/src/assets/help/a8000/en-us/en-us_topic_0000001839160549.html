Step 1: Enable the snapshot function of the directory where the database table resides.

Before performing protection, enable the snapshot function for the directory where the Hive database table to be protected resides.

Prerequisites

Before enabling this function, ensure that nodes in the big data cluster are not in the safe mode. For details about how to query whether a node is in the safe mode and how to exit the safe mode, see Exiting the Cluster Safe Mode.

Procedure

  1. Use PuTTY to log in to any NameNode in the big data cluster.
  2. Run the following command to enable the snapshot function for the directory:

    hdfs dfsadmin -allowSnapshot directory_name

    directory_name indicates the parent directory of the database table to be protected.
