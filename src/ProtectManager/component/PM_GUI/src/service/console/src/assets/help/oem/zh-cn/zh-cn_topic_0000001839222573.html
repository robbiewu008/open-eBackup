步骤5：注册MySQL/MariaDB数据库

当MySQL/MariaDB数据库所在主机安装ProtectAgent后，本产品会自动发现MySQL/MariaDB数据库主机，您需要在管理界面上注册主机上的数据库，以对其进行保护或将其作为恢复目标。

背景信息

本产品支持备份MySQL/MariaDB数据库以及其他自定义数据库，不支持备份系统数据库information_schema、performance_schema、sys和mysql。因此不支持注册系统数据库。

前提条件

注册MySQL/MariaDB集群实例前，请确保集群中每个节点已进行注册，否则未注册的节点无法备份数据，在恢复场景下，会造成恢复时间点的用户数据不一致。

要求集群中所有节点的数据同步。

注册MySQL/MariaDB单实例

  1. 选择“保护 > 数据库 > MySQL/MariaDB”。
  2. 选择“实例”页签。
  3. 单击“注册”，注册MySQL/MariaDB。

    MySQL集群环境下的所有实例不能用于单实例注册。

  4. “类型”选择“单实例”。
  5. 配置数据库实例及其认证信息。

    相关参数说明如表1所示。

    表1 MySQL/MariaDB单实例注册信息

    参数

    说明

    名称

    自定义实例名称。

    主机

    选择要注册的数据库主机。

    添加访问MySQL地址

    访问MySQL实例的IP地址。

    用户名

    请填写步骤1：开启MySQL/MariaDB数据库权限创建的用户名。

    密码

    用户名密码。

    端口

    Linux环境下，界面显示为端口号，默认为3306。

    字符集

    说明：

    仅在1.6.0及后续版本展示该参数。

    数据库字符集。

    MySQL配置文件路径

    说明：

    仅在1.6.0及后续版本展示该参数。

    如果配置文件非默认路径，如/etc/my.cnf等，请填入MySQL配置文件路径。

  6. 单击“确定”。

注册MySQL/MariaDB集群实例

  1. 创建集群。

    1. 在MySQL/MariaDB页面选择“集群”页签。
    2. 单击“注册”。
    3. 在“注册”页面，自定义集群名称并选择类型和集群节点。
      相关参数说明如表2。
      表2 注册集群参数说明

      参数

      说明

      名称

      自定义集群名称。

      类型

      集群类型可选择主备复制和PXC(Percona XtraDB Cluster)类型。

      • 若选择“主备复制”类型，则需要包含MySQL/MariaDB主实例所在节点。
      • 若选择“PXC(Percona XtraDB Cluster)”类型，则需要包含MySQL/MariaDB集群实例所在节点。

      集群节点

      MySQL/MariaDB实例所在的主机。
      • 所选择的主机必须来源于同一集群。
      • 当数据库集群类型为主备集群时，在 /etc/my.cnf配置文件中，如果有relay-log配置项，请设置relay_log_recovery取值为on与之配合使用（避免恢复失败）。
    4. 单击“确定”。

  2. 注册集群实例下的数据库。

    1. 在MySQL/MariaDB页面选择“实例”页签。
    2. 单击“注册”。
    3. “类型”选择“集群实例”。
    4. 输入实例名，选择集群。
    5. 单击“添加”，在弹出的添加对话框中添加集群实例下的所有节点。

      相关参数说明如表3所示。

      表3 集群实例的节点信息

      参数

      说明

      主机

      选择集群下的主机节点。

      eAppMySQL地址

      访问eAppMySQL服务IP地址。

      端口

      Linux环境下，界面显示为端口号，默认为3306。

      用户名

      数据库用户名。

      密码

      数据库密码。

      字符集

      说明：

      仅在1.6.0及后续版本展示该参数。

      数据库字符集。

      MySQL配置文件路径

      说明：

      仅在1.6.0及后续版本展示该参数。

      如果配置文件非默认路径，如/etc/my.cnf等，请填入MySQL配置文件路径。
    6. 单击“确定”。
