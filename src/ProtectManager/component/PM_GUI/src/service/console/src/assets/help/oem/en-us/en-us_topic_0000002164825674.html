When ClickHouse and Redis Are Deployed Together, the ClickHouse Node Is Offline or Backup or Restoration Fails Occasionally

Symptom

When ClickHouse and Redis are deployed on the same MRS production node, a node is offline, or backup or restoration fails.

Possible Cause

When ClickHouse and Redis are deployed together, the Kerberos authentication principal configured for Redis does not have permissions required for ClickHouse backup and restoration.

Suggestion

  1. Log in to the ClickHouse node that is offline or where backup or restoration fails.
  2. Run the cd /opt/DataBackup/ProtectClient/ProtectClient-E/slog/Plugins/GeneralDBPlugin/ command. (Replace /opt in the command with the actual installation directory of the client.) Go to the log file directory and check whether the clickhouse.log log file contains the following error information:

    DB::Exception XXX: Not enough privileges.

    XXX indicates the Kerbors authentication principal name of Redis.

    • If such error information is contained, grant permissions required for ClickHouse backup and restoration to the Kerbors authentication principal of Redis.
    • If no error information is displayed, collect logs and contact technical support engineers.
