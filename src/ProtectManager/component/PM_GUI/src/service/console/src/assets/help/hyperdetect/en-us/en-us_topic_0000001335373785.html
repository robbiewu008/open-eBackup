Managing SLAs

You can clone, modify, and delete SLAs on the ProtectManager.

Before performing these operations, you need to log in to ProtectManager and choose Protect > Protection Policies > SLAs to go to the SLA page.

Managing SLAs describes the operations related to SLAs.

Table 1 SLA-related operations

Operation

Description

Navigation Path

Cloning an SLA

Scenario

This operation enables you to quickly create an SLA based on an existing SLA.

In the row where the SLA resides, choose More > Clone.

Modifying an SLA

Scenario

This operation enables you to modify a backup policy configured in an SLA.

In the row where the SLA resides, choose More > Modify.

Deleting an SLA

Scenario

This operation enables you to delete an SLA that is no longer used.

Note

An SLA that has been associated with resources cannot be deleted. Before deleting such an SLA, you must remove protection for the resources associated with the SLA.

In the row where the SLA resides, choose More > Delete.

NOTE:

You can also select multiple SLAs and click Delete in the upper left corner to delete them in batches.
