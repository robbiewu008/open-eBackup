Precautions for Deploying ProtectAgent on Production Hosts (Applicable to 1.6.0 and Later Versions)

You need to add environment variables when deploying ProtectAgent on production hosts. Otherwise, Roach commands will fail to be executed.

Procedure

  1. Use PuTTY to log in to all CNs in the GaussDB (DWS) cluster where ProtectAgent has been deployed.
  2. Run the following command to open the env file (user omm is used as an example):

    vi /home/omm/profile/env
    Ensure that the following content has been added to the env file. The ProtectAgent installation directory /opt is used as an example.
    export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/opt/DataBackup/ProtectClient/interfaces/xbsa/lib

  3. Run the following command to open the .bashrc file (user omm is used as an example):

    vi /home/omm/.bashrc
    Ensure that the following content has been added to the .bashrc file. The ProtectAgent installation directory /opt is used as an example.
    export DATA_BACKUP_AGENT_HOME=/opt
