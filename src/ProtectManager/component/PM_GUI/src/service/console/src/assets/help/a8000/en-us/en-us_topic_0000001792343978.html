Managing Protection for Projects/Cloud Servers or Cloud Server Groups (Applicable to 1.6.0 and Later Versions)

The OceanProtect allows you to modify, remove, activate, or disable protection plans for projects, cloud servers, or cloud server groups (applicable to 1.6.0 and later versions). This section uses a project as an example to describe related operations. For a cloud server, operations are the same.

You need to log in to the management page and choose Protection > Cloud Platforms > OpenStack. On the Project tab page, locate the target project.

Table 1 describes the related operations.

Table 1 Project protection plan operations

Operation

Description

Navigation Path

Modify Protection

Scenario

You can modify a protection plan to re-associate a project with an SLA or modify agent host information. After the modification, the next backup will be performed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.
  • If a backup job has been performed, reconfigure the SLA policy (change to the SLA with copy verification enabled) when modifying the protection plan. After the modification, the initial backup job will be converted to a full backup job if it is a non-full backup job.
  • If the SLA parameter associated with the resource changes (from copy verification disabled to copy verification enabled) after a backup job has been performed, when the protection plan for the resource is implemented after the change, the initial backup job will be converted to a full backup job if it is a non-full backup job.

Choose More > Modify Protection in the row that contains the target project.

Remove Protection

Scenario

You can remove protection for a project that no longer needs to be protected. After the removal, the system cannot execute protection jobs for the project. To protect the project/resource set again, you need to associate an SLA with the project/resource set again.

Note

If a protection job of the project is in progress, protection cannot be removed.

Choose More > Remove Protection in the row that contains the target project.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup in the row that contains the target project.

Activate Protection

Scenario

This operation enables you to activate the protection plan for a project. After the protection plan is activated, the system executes backup, archive, or replication for the project based on the plan.

Note

This operation can be performed only for a project that has been associated with an SLA and is in Unprotected status.

Choose More > Activate Protection in the row that contains the target project.

Disable Protection

Scenario

This operation enables you to disable the protection plan of a project for which periodic protection is no longer required. After the protection plan is disabled, the system does not execute automatic backup, archive, or replication for the project.

Note

Disabling a protection plan does not affect protection jobs that are running.

Choose More > Disable Protection in the row that contains the target project.

Scanning an OpenStack project

Scenario

The OceanProtect automatically scans projects every hour. If any project or cloud server under the registered OpenStack changes, the changes will be automatically updated to the OceanProtect after the scan. This operation enables you to immediately update the changed information to the OceanProtect.

Note

If you do not select to apply the SLA policy to newly created cloud servers upon periodic backup creation, the system will not associate newly discovered cloud servers with any SLA after the scan.

Choose More > Resource Scan.

OpenStack Resource Authorization

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Choose More > Authorize Resource and select a data protection administrator for authorization.

Reclaiming OpenStack resources

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Choose More > Reclaim Resource.
