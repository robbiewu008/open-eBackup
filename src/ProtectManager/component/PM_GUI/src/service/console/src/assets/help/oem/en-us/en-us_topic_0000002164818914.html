Managing Database Clusters

The product allows you to modify and delete clusters.

Log in to the management page, choose Protection > Databases > TDSQL, click the Cluster tab, and find the desired cluster.

Table 1 describes the related operations.

Table 1 Operations related to a cluster

Operation

Description

Navigation Path
Authorizing resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Note

  • Users cannot be authorized to protect resources that have been associated with SLAs. You need to remove the protection of resources before performing authorization.
  • A single resource can be authorized only to a single data protection administrator.

In the row of the target cluster, choose More > Authorize Resource and select a data protection administrator for resource authorization.
Reclaiming resources
NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

Users are not allowed to reclaim resources that have been associated with SLAs. You need to remove the protection of resources before reclaiming them.

In the row of the target cluster, choose More > Reclaim Resource.

Testing connectivity

Scenario

Manual connectivity test of a registered TDSQL cluster is used to check the connectivity between the cluster and the host where the client is installed as well as the production environment.

Choose More > Test Connectivity.

Modification

Scenario

You can modify the cluster name and node information following this operation.

In the row of the target cluster, choose More > Modify.

Deletion

Scenario

You can delete a cluster if its databases do not require protection. After the cluster is deleted, its information is no longer displayed on the management page.

Note

If a database has been registered with a cluster, the cluster cannot be deleted.

In the row of the target cluster, choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
