查看TDSQL数据库环境信息

当TDSQL数据库已注册至本产品后，您可以在管理界面查看数据库集群及其实例的详细信息。

操作步骤

  1. 选择“保护 > 数据库 > TDSQL”。
  2. 在“集群”页签浏览数据库集群信息。

    当集群有数据库已通过注册，集群信息会显示在该页面。

    部分参数默认不展示，您可以单击“操作”列后的，勾选对应参数，即可在当前界面展示。

    详细信息如表1所示。
    表1 主机和集群信息

    参数

    说明

    名称

    数据库集群名称。

    单击集群名称，显示当前集群的主机成员，及集群下已通过认证的所有数据库实例。

    状态
    主机或集群的状态。
    • 在线：主机处于开机状态且主机上的客户端正常运行时，主机“状态”为“在线”。

      集群中存在在线状态的主机时，集群状态为“在线”。

    • 离线：主机关机或者客户端所在的主机故障时，主机“状态”为“离线”。

      集群中所有主机均离线时，集群状态为“离线”。
    所属用户
    说明：

    仅1.5.0版本存在此参数。

    该主机或集群所属的数据保护管理员。

    当该主机或集群已授权给数据保护管理员时，将显示数据保护管理员的名称。如果未进行授权，则显示“--”。
    标签
    说明：

    仅1.6.0及后续版本存在此参数。

    当资源已添加标签时，则此处显示标签名称。否则，显示“--”。

  3. 在“非分布式实例”或“分布式实例”页签浏览数据库实例信息。

    该界面显示的是已注册的数据库实例。

    部分参数默认不展示，您可以单击“操作”列后的，勾选对应参数，即可在当前界面展示。

    1. 实例的详细信息如表2所示。
      表2 实例信息

      参数

      说明

      名称

      数据库实例名称。

      状态
      数据库实例的状态。
      • 在线：当客户端所在主机与TDSQL集群所有节点连接正常时，“状态”为“在线”。
      • 离线：当客户端所在主机与TDSQL集群所有节点连接异常时，“状态”为“离线”。

      所属集群

      实例所在集群的名称。

      SLA

      当数据库已关联SLA时，则显示SLA名称，否则显示“--”。

      SLA遵从度
      SLA遵从度指某次备份任务是否按设定的备份频率或备份窗口成功完成备份。
      • 遵从：如果某次备份任务（首次全量备份除外）的备份时间满足备份窗口或备份频率要求，且备份成功，则表示SLA遵从。
      • 不遵从：如果备份时间不满足备份窗口或备份频率要求，例如某次备份任务SLA设定的备份频率为每半小时执行一次，最终备份时间超过半小时，则表示SLA不遵从。建议调整备份频率或备份窗口。

      保护状态
      当前资源是否被保护。
      • 未保护：当前资源未关联SLA，或者已关联SLA但已禁用保护。
      • 已保护：当前资源已关联SLA，且已激活保护。
      • 创建中：当前资源已关联SLA，正在创建保护。
      标签
      说明：

      仅1.6.0及后续版本存在此参数。

      当资源已添加标签时，则此处显示标签名称。否则，显示“--”。
    2. 单击名称，查看总览信息、副本信息以及相关任务。
      • 总览

        显示SLA信息、任务执行时间、副本总数等。

      • 副本数据

        您可以依次按照年、月、天查找相应时间的副本。

        时间下方显示，表示该时间存在副本。

      • 相关任务

        展示备份、恢复等任务。
