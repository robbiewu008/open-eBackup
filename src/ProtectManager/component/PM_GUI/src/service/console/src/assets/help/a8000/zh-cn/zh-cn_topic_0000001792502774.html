步骤1：开放GaussDB(DWS)备份恢复所需端口

操作步骤

  1. 登录GaussDB(DWS)节点。
  2. 拷贝configure_firewall_port.sh脚本到GaussDB(DWS)节点的root目录。
  3. 执行以下命令备份iptables到临时文件，以iptables.test为例（以下备份方式适用于打开iptables的场景）。

    iptables-save > iptables.test

  4. 执行以下命令开放GaussDB(DWS)备份必需的端口（必需端口为：55000:55050、9010:9050、 15000:15050、17000、59520:59539）。

    sh configure_firewall_port.sh add 192.168.0.0/16
    • “192.168.0.0/16”为GaussDB(DWS)业务网络平面和OceanProtect备份网络平面的网段，如果两个平面的网段不一致，需要对两个平面的网段都执行此脚本。
    • 如果需要开放其他端口，需要执行以下命令添加（示例中“30600”、“59600:59610”为需要添加的端口）。
    sh configure_firewall_port.sh add 192.168.0.0/16 30600 59600:59610

  5. 如果4失败，或未达到预期，执行以下命令恢复iptables规则并保存配置（以下恢复方式适用于打开iptables的场景）。

    iptables-restore < iptables.test
service iptables save
