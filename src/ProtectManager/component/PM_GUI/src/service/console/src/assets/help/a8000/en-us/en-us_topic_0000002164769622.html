The DataTurbo Service Fails to Be Started After Copy-based Active Directory Restoration Is Performed

Symptom

The DataTurbo service fails to be started after copy-based Active Directory restoration is performed. On the host where the client is installed, Not allowed is displayed for rdnginx.exe in Task Manager.

Possible Causes

After the backup, the client is uninstalled and reinstalled, and the DataTurbo password is regenerated. After the full copy is used for system state restoration, the DataTurbo password is restored to that before the reinstallation. As a result, the service fails to be started.

Procedure

  1. Double-click the uninstall.bat script in the C:\DataBackup\ProtectClient directory to uninstall the client.
  2. Install the client.
