Creating the SAML SSO Configuration

This section describes how to create the SAML SSO configuration.

Context

SAML SSO consists of an identity provider (IdP) and a service provider (SP).

  • Identity provider: A system that collects and stores user identity information, such as usernames and passwords, and authenticates users during login.
  • Service provider: A system that establishes a trust relationship with the identity provider and uses the user information provided by the identity provider to provide services for users.

When the OceanProtect is connected to eDME as the backup server, eDME is the identity provider (IdP) and the OceanProtect is the service provider (SP).

Prerequisites

Before connecting to eDME, obtain the metadata of the eDME O&M plane and operation plane to ensure that the OceanProtect can be accessed from eDME and the quotas and functions of the OceanProtect can be used. For details about how to import metadata, see "Configuring SAML SSO" in O&M Portal and "Configuring SAML SSO" in Operation Portal of the eDME Product Documentation of the version supported by the OceanProtect. For details about the mapping between the OceanProtect and eDME, see the OceanProtect DataBackup Version Mapping Table of the corresponding version.

Procedure

  1. Choose System > Security > SAML SSO Configuration.
  2. Click Create.
  3. Enter the SAML SSO Configuration information. Creating the SAML SSO Configuration describes the parameters.

    Table 1 SAML SSO Configuration parameter description

    Parameter

    Description

    Name

    Name of the SAML SSO configuration to be created.

    Protocol

    Protocol configured for SAML SSO. The default value is SAML2.0.

    Description

    Description of the SAML SSO configuration.

    Metadata File

    Metadata file exported from the eDME (IdP).

  4. Click OK.
