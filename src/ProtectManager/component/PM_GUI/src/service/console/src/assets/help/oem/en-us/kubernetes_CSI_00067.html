Pod Configuration in the Production Environment for Application-Consistent Backup (MySQL as the Containerized Application)

If there are multiple pre-hook or post-hook commands for creating a snapshot for a PVC, you can write multiple pre-executed commands to the same script and place the script in the persistent directory of a container. Running the script to execute multiple commands simplifies the command input process.

The following shows examples of the script content of the pre-hook or post-hook command. Set password to the actual user password. (For subsequent operations, the pre-script location /opt/backup/dataprotect_mysql_consistency_pre_script.sh and the post-script location /opt/backup/dataprotect_mysql_consistency_post_script.sh are used as examples. Change the content based on the actual script locations).

#Pre-hook command script
mysql -u root -p password -e "flush tables with read lock;system sleep 300;"
#Post-hook command script
pid=$(ps -ef | grep "/bin/sh -c /opt/backup/dataprotect_mysql_consistency_pre_script.sh" | grep -v grep | awk '{print $2}')  
[ ! $pid ] && echo "pid is null" && exit
subtree=$(pstree -p $pid) subtree=$(pstree -p $pid)  
right_str=${subtree##*\(}  
sleep_pid=${right_str%*)}  
kill -9 $sleep_pid

Procedure

  1. Upload the script to any persistent directory of the container. Run the following commands on the container page to add permissions to the script file:

    chmod +x /opt/backup/dataprotect_mysql_consistency_pre_script.sh
chmod +x /opt/backup/dataprotect_mysql_consistency_post_script.sh

    You can also run the echo command in the container directory to overwrite or append the script content. The following is an example.

    echo "mysql -uroot -p{password} -e "flush tables with read lock;system sleep 300;"" > /opt/backup/dataprotect_mysql_consistency_pre_script.sh

  2. Add pod labels and annotations by referring to Pod Configuration in the Production Environment for Application-Consistent Backup (General). The following shows examples of the pre-hook or post-hook command values entered when you add annotations:

    #Pre-hook command
kubectl annotate pod -l label=value -n namespace pre.hook.dataprotect.backup.io/command='["/bin/sh", "-c","/opt/backup/dataprotect_mysql_consistency_pre_script.sh"]'
#Post-hook command
kubectl annotate pod -l label=value -n namespace post.hook.dataprotect.backup.io/command='["/bin/sh", "-c","/opt/backup/dataprotect_mysql_consistency_post_script.sh"]'

    To ensure that the pre-hook or post-hook command can be executed properly, the script location in the command must be the same as the actual upload location of the script.
