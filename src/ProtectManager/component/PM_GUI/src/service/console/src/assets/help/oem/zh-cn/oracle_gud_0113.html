查看Oracle数据库环境信息

当Oracle数据库或集群已注册至本产品后，您可以在管理界面查看数据库或集群信息。

操作步骤

  1. 选择“保护 > 数据库 > Oracle”。
  2. 在“集群”页签查看相关信息。

    部分参数默认不展示，您可以单击“操作”列后的，勾选对应参数，即可在当前界面展示。

    1. 该页面展示已注册的Oracle集群信息。

      相关参数说明如表1所示。

      表1 Oracle数据库集群信息

      参数

      说明

      名称

      集群的名称。

      状态
      集群的状态。
      • 在线：集群中存在在线状态的主机时，集群状态为“在线”。
      • 离线：集群中所有主机均离线时，集群状态为“离线”。

      类型

      集群类型。

      所属用户

      说明：

      仅1.5.0版本存在此参数。

      该集群所属的数据保护管理员。

      当该集群已授权给数据保护管理员时，将显示数据保护管理员的名称。如果未进行授权，则显示“--”。
      标签
      说明：

      仅1.6.0及后续版本存在此参数。

      当资源已添加标签时，则此处显示标签名称。否则，显示“--”。
    2. 单击集群名称，查看集群中主机及数据库的详细信息。

  3. 在“数据库”页签查看相关信息。

    1. 该页面展示Oracle数据库的详细信息。
      相关参数说明如查看Oracle数据库环境信息所示。
      表2 Oracle数据库信息

      参数

      说明

      名称

      数据库名称。

      状态

      数据库状态。

      当本产品可正常访问数据库时，该状态为“在线”，否则为“离线”。

      IP地址

      数据库所属集群的IP地址。

      类型

      数据库类型。包括“单机”和“集群”。

      操作系统

      数据库所在主机的操作系统类型。

      版本

      数据库的版本。

      认证状态

      数据库的认证状态。

      当数据库已配置认证方式，且配置信息正确时，认证状态为“已认证”，否则为“未认证”。

      SLA

      当资源已关联SLA时，则显示SLA名称，否则显示“--”。

      SLA遵从度
      SLA遵从度指某次备份任务是否按设定的备份频率或备份窗口成功完成备份。
      • 遵从：如果某次备份任务（首次全量备份除外）的备份时间满足备份窗口或备份频率要求，且备份成功，则表示SLA遵从。
      • 不遵从：如果备份时间不满足备份窗口或备份频率要求，例如某次备份任务SLA设定的备份频率为每半小时执行一次，最终备份时间超过半小时，则表示SLA不遵从。建议调整备份频率或备份窗口。

      保护状态

      数据库当前是否被保护。

      • 未保护：数据库未关联SLA，或者数据库已关联SLA但已禁用保护。
      • 已保护：数据库已关联SLA，且已激活保护。
      • 创建中：数据库已关联SLA，正在创建保护。
      标签
      说明：

      仅1.6.0及后续版本存在此参数。

      当资源已添加标签时，则此处显示标签名称。否则，显示“--”。
    1. 单击数据库名称，查看数据库总览信息、副本信息以及相关任务。
      • 数据库总览

        显示数据库名称、SLA信息、任务执行时间、副本总数等。

      • 数据库副本数据

        您可以依次按照年、月、天查找相应时间的副本。

        时间下方显示，表示该时间存在副本。

      • 数据库相关任务

        数据库相关的所有任务展示，包括备份、恢复、即时恢复、归档等任务。

        如果正在运行的任务长时间未更新进度，可参考Oracle数据保护相关任务超过10分钟未更新任务进度如何处理处理。
