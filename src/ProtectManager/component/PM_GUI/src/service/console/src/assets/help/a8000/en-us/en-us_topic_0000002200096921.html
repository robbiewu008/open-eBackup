Failed to Back Up or Restore Data Because the Value of a Configuration Item in the plugin_attribute_1.1.0.json Configuration File Is Incorrect

Symptom

The backup or restoration job fails, and an error message is displayed indicating a configuration error.

Possible Cause

Data failed to be backed up or restored because the value of the configuration item in the plugin_attribute_1.1.0.json configuration file is incorrect.

Procedure

  1. Log in to any hos where the client is installed.
  2. Run the vi command to open the plugin_attribute_1.1.0.json configuration file. /opt is the client installation path in the example. Replace it with the actual client deployment path.

    vi /opt/DataBackup/ProtectClient/Plugins/ElasticSearchPlugin/plugin_attribute_1.1.0.json

  3. In the insert mode, search for parameters memory_min_limit and memory_max_limit and change them to the values used in the current scenario.

    • memory_min_limit indicates the minimum memory (MB) that the system can provide for plug-ins. The value must be greater than 70 MB.
    • memory_max_limit indicates the maximum memory (MB) that the system can provide for plug-ins. The value must be less than the total memory of the host where the client is installed.

  4. Save the configuration file and restart the host where the client is installed.
