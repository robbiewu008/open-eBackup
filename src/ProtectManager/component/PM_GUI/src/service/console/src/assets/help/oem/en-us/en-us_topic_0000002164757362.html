Preparing for the Restoration

  • Ensure that the restoration target location is correct and the remaining space of the restoration target location is greater than the size of the copy used for restoration before reduction.
  • For table-level restoration to a new schema, the schema has been created in the corresponding GaussDB (DWS) database before the restoration.
  • Table-level restoration can be used to restore only the table structure and data of a table set. However, it cannot be used to restore the permissions, indexes, constraints, sequences, composite types, views, and functions associated with tables. If the preceding parameters of the table set to be restored have been modified, manually create the parameters based on the parameter information prepared in Preparing for Backup. For details, see the Product Documentation of the corresponding GaussDB (DWS) version.
