Performing Periodic Backup

The instance to be protected needs to be associated with a specified SLA. The system periodically performs backup based on the SLA.

Prerequisites

  • Backup operations can be performed only when the database is in the normal or degraded state. Only full backup or incremental backup can be performed, if the database is in the degraded state.
  • After the agent host is replaced or the project information is modified, you need to scan resources again.
  • For OLTP OPS, before a backup is initiated, ensure that the default backup root path has been configured for the instance on the OLTP OPS management page.
  • For OLTP OPS, before a backup is initiated, ensure that monitoring has been enabled for instances managed by OLTP OPS.
  • Before a backup is performed, you need to start backUpAgent on the agent host. For details about how to start the backUpAgent, see "Starting backUpAgent" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.
  • Before a backup is initiated, ensure that the XBSA backup whitelist has been enabled on TPOPS. For details about how to enable the whitelist, see Step 1: Enabling the XBSA Backup Whitelist on the TPOPS Node.

Precautions

For version 1.5.0, there is a hole area where data files are not retained between the log backup and the next data backup.

Procedure

  1. Choose Protection > Databases > GaussDB.

    For 1.5.0, choose Protection > Cloud Platforms > GaussDB.

  2. Click the Instance tab.
  3. In the row that contains the target instance, choose More > Protect.

    You can select multiple databases.

  4. Select an SLA.

    You can also click Create to create an SLA.

  5. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
