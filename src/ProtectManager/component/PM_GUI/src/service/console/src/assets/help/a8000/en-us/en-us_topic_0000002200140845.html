Step 3: Registering an HDFS Cluster

Before the HDFS fileset backup and restoration, register the HDFS cluster with the OceanProtect.

Prerequisites

You have obtained the hdfs-site.xml and core-site.xml files in the HDFS cluster by performing the following steps:

  • If the big data platform is Cloudera's Distribution including Apache Hadoop (CDH):

    This part uses Cloudera Express 6.3.1 as an example. Operations may vary with versions. For details, see the CDH product documentation.

    1. Use a browser to log in to Cloudera Manager.
    2. Choose Clusters > HDFS.
    3. Choose Actions > Download Client Configuration on the right of the page.
    4. Obtain the hdfs-site.xml and core-site.xml files from the downloaded package.
  • If the big data platform is MapReduce Service (MRS):

    This section uses FusionInsight Manager 8.2.0 as an example. Operations may vary with versions. For details, see the product documentation of the corresponding big data platform.

    1. Log in to FusionInsight Manager using a browser.
    2. Choose Cluster > Name of the target cluster > Dashboard > More > Download Client. On the page that is displayed, select Configuration Files Only to download the HDFS client configuration file package.
    3. Decompress the downloaded package to obtain the following files:
      • ../HDFS/config/hdfs-site.xml
      • ../HDFS/config/core-site.xml
  • If the big data platform is Transwarp Data Hub (TDH):

    This part uses TDH 8.0.2 as an example. Operations may vary with versions. For details, see the Transwarp Data Hub product documentation.

    1. Use a browser to log in to Transwarp Data Hub.
    2. Choose Dashboard > Cluster and click an HDFS cluster to go to the cluster page.
    3. In the upper right corner of the HDFS cluster page, select the icon, and click Download Service Configurations.
    4. Obtain the hdfs-site.xml and core-site.xml files from the downloaded package.
    5. Delete the following information in the core-site.xml file:
    <property>
    <name>hadoop.security.group.mapping.ldap.bind.password.file</name>
    <value>/etc/hdfs1/conf/ldap-conn-pass.txt</value>
</property>

Procedure

  1. Choose Protection > Big Data > HDFS.
  2. On the Cluster page, click Register to register an HDFS cluster.

    Table 1 describes the HDFS cluster registration information.

    Table 1 HDFS cluster registration information

    Parameter

    Description

    Name

    Customize an HDFS cluster name.

    hdfs-site.xml

    Upload the hdfs-site.xml and core-site.xml files obtained in Prerequisites to the management page.

    The files are used to configure HDFS cluster information, such as the cluster address and port number.

    NOTE:
    • The big data cluster supports only IPv4 addresses. IPv6 addresses are not supported. Check whether configurations in the configuration file are correct.
    • The configuration file uploaded to the management page must be the same as that in the big data cluster. Do not modify the configuration file locally.
    • Do not change an existing HDFS cluster to another HDFS cluster. Otherwise, the created filesets will become invalid.

    core-site.xml

    Authentication Mode

    Configure the authentication mode for communication between the HDFS cluster and the OceanProtect. Set this parameter based on the authentication mode configured for the HDFS cluster.

    • Simple Authentication: Select this authentication mode when Kerberos authentication is disabled for the HDFS cluster. In this authentication mode, usernames are used for authentication, which is less secure.
    • Kerberos Authentication: Select this authentication mode when Kerberos authentication is enabled for the HDFS cluster. The Kerberos protocol is used for identity authentication between the HDFS cluster and the OceanProtect.

    Username

    This parameter is required only when Authentication Mode is set to Simple Authentication.

    NOTE:
    • Set this parameter to the human-machine username of the big data platform.
    • The entered user must have the read, write, and execute permissions on the resources to be accessed.
    • If you need to restore HDFS ACL information during HDFS file restoration, the registered principal name must have HDFS super user permissions. For example, you can add the principal name to the supergroup on the big data platform so that the user has HDFS super user permissions.

    Kerberos

    This parameter is required only when Authentication Mode is set to Kerberos Authentication.

    Select the created Kerberos authentication. When you register an HDFS cluster for the first time, click Create and configure Kerberos authentication parameters. Table 2 describes the related parameters.

    Clients

    Select the host where the client is installed for data protection.

    NOTE:

    Do not allocate the same host where the client is installed to multiple big data clusters with different Kerberos authentication configurations or authentication modes. Otherwise, the backup or restoration job fails.

    ACL Backup

    NOTE:

    For 1.5.0, the parameter name is Back Up ACL.

    Whether to back up the ACL.

    The ACL is backed up only when both the HDFS ACL configuration (dfs.namenode.acls.enabled) on the big data platform and the current function are enabled.

    Set Kerberos authentication parameters. For details about related parameters, see Table 2.

    Table 2 Kerberos authentication parameters

    Parameter

    Description

    Name

    User-defined Kerberos authentication name.

    Principal Name

    Principal name of Kerberos authentication.

    NOTE:
    • Set this parameter to the human-machine username of the big data platform. The user must have the read, write, and execute permissions on the resources to be accessed.
    • If you need to restore HDFS ACL information during HDFS file restoration, the registered principal name must have HDFS super user permissions. Here are some examples:
      • If the big data platform is FusionInsight or CDH, you can add the principal name to the supergroup on the big data platform so that the principal name has HDFS super user permissions.
      • If the big data platform is TDH, you can add the admin role to the principal name on the big data platform so that the principal name has HDFS super user permissions.

    Configuration Mode
    Kerberos authentication mode, which must be the same as that configured on the Kerberos server. Two authentication modes are available:
    • Password: Use the principal name and password for identity authentication.
    • Keytab File: Use the principal and keytab file for identity authentication.

    Configuration File

    The .conf configuration file (for example, krb5.conf) used during Kerberos authentication. Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the management page.

    Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.

    Due to problems in Kerberos, if renew_lifetime exists in the configuration file, comment it out. Otherwise, the registration may fail. For example:
    #renew_lifetime = 7d

    Password

    This parameter is required only when Configuration Mode is set to Password.

    Set this parameter to the password of Principal Name.

    Confirm Password

    Confirm the password of Principal Name.

    Keytab File

    The .keytab file used during Kerberos authentication (for example, krb5.keytab). This parameter is required only when Configuration Mode is set to Keytab File.

    Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the management page.

    Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.

  3. Click OK.
