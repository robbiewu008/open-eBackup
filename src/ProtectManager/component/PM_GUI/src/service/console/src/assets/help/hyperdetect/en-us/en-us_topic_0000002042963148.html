Viewing File System Information

Prerequisites

File system resources on the storage system have been discovered on ProtectManager.

Context

File systems created on the storage system can be automatically discovered by ProtectManager and displayed on the Local File Systems page.

  • When ProtectManager scans for a file system that is newly created on the storage system, it may take some time for ProtectManager to discover it.
  • File systems discovered by ProtectManager cannot be deleted on ProtectManager.
  • Clone file systems are not displayed on ProtectManager.
  • File systems whose names start with pvc_ created through HyperDetect are not displayed on ProtectManager.

Procedure

  1. Choose Protection > Storage > Local Resources.
  2. Click the Local File Systems tab. In the local file system list, view file system information.

    1. Table 1 describes the parameters related to a file system.
      Table 1 Parameters related to a file system

      Parameter

      Description

      Resource ID

      ID of a local file system.

      Name

      Name of a local file system.

      vStore

      vStore to which a file system belongs. The default value is System_vStore.

      SLAs

      If a local file system has been associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance

      Indicates whether a backup job is successfully executed in compliance with the specified backup interval or backup time window. For example, if the backup time of a backup job (except the first backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met. If the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed.

      Protection Status

      Indicates whether a local file system is protected.

      • Unprotected: The local file system is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The local file system is associated with an SLA and protection is activated.
      • Creating: The local file system is associated with an SLA and protection is being created.
    2. Click the name of a local file system to view its overview, copy data, and related jobs.
      • Overview of the local file system

        Displays the name, SLA information, job execution time, and total number of copies of the local file system.

      • Snapshot data of a local file system

        Allows you to search for copies by year, month, and day.

        If is displayed below the time, a copy at that time exists.

      • Jobs related to the local file system

        Displays all jobs related to the local file system, including ransomware detection snapshot backup and restoration jobs.
