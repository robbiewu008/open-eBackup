Modifying the GaussDB (DWS) Configuration File

If the client is installed on a GaussDB (DWS) node, you need to modify the GaussDB (DWS) configuration file. Otherwise, the backup will fail.

Procedure

  1. Use PuTTY to log in to the GaussDB (DWS) cluster node.
  2. Run the following command to access the configuration file directory.

    cd /opt/DataBackup/ProtectClient/Plugins/GeneralDBPlugin/bin/
applications/dws/

  3. Run the vi dws.conf command to edit the configuration file. Table 1 describes the parameters.

    Table 1 GaussDB (DWS) configuration file parameters

    Parameter

    Description

    deployType

    GaussDB (DWS) deployment mode.

    • 0: The GaussDB (DWS) cluster is deployed on the host machine of the server.
    • 1: The GaussDB (DWS) cluster is deployed in the sandbox.

    loginSandboxIp

    IP address used for GaussDB (DWS) deployment in the sandbox (applicable only to scenarios where GaussDB (DWS) is deployed in the sandbox).

    NOTE:

    Run the hostname -i command on the CN where the client is deployed to query the IP address.

    sandboxPath

    Path for storing the sandbox on the host (applicable only to scenarios where GaussDB (DWS) is deployed in the sandbox).

    authmethod

    Client authentication policy.

    • trust: Password is not verified.
    • reject: Access is denied.
    • md5/sha256/cert: Client certificate authentication is performed.
    • gss: Kerberos authentication is performed.
