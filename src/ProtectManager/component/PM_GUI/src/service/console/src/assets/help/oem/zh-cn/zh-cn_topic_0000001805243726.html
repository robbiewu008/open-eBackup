恢复CNware虚拟机磁盘

当需要对已备份的虚拟机指定磁盘恢复时，可以参考本节恢复虚拟机的磁盘至原虚拟机或新虚拟机。

背景信息

本产品支持使用备份副本、复制副本（不支持恢复至原虚拟机）、即时挂载产生的副本（副本需要已从虚拟机卸载）、导入的副本进行磁盘恢复，支持恢复至原虚拟机或新虚拟机。

  • 恢复磁盘至原虚拟机

    恢复磁盘至原虚拟机时，如果原虚拟机对应槽位存在磁盘，该磁盘会被替换，如果对应槽位原磁盘还存在，则会尝试增量恢复（只有基于最新的备份副本，执行第一次恢复时才是增量恢复，其他恢复均为全量恢复。）；如果原虚拟机对应槽位没有磁盘，则会进行全量恢复，即创建磁盘进行恢复，并挂载在对应槽位上。

  • 恢复磁盘至新虚拟机

    恢复磁盘到新虚拟机时，如果指定的新虚拟机对应槽位存在磁盘，该磁盘会被替换；如果新虚拟机对应槽位没有磁盘，则会进行全量恢复，即创建磁盘进行恢复，并挂载在对应槽位上。

操作步骤

  1. 选择“数据利用 > 副本数据 > 虚拟化 > CNware”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 找到要恢复的副本，选择“更多 > 磁盘恢复”。
  5. 勾选要恢复的磁盘。
  6. 恢复虚拟机磁盘到原虚拟机或新虚拟机。

    • 恢复虚拟机磁盘到原虚拟机。

      相关参数说明如表1所示。

      表1 磁盘恢复至原位置参数信息

      参数

      说明

      代理主机

      指定执行恢复任务的代理主机，不指定则使用系统自动分配的代理主机。

      恢复后自动开机

      开启后，恢复完成后虚拟机将自动开机。

      恢复前执行副本校验

      执行恢复前校验副本完整性。当副本校验状态为“无效”或“--”时，不支持开启此功能。
    • 恢复虚拟机磁盘到新虚拟机。
      相关参数说明如表2所示。
      表2 磁盘恢复至新位置参数信息

      参数

      说明

      目标位置

      选择备份磁盘恢复到的虚拟机。

      说明：

      目标虚拟机与源虚拟机的操作系统类型必须相同，如：同是Windows OS或同是Linux OS。

      虚拟机磁盘文件

      为备份磁盘选择恢复到的目标磁盘。

      代理主机

      指定执行恢复任务的代理主机，不指定则使用系统自动分配的代理主机。

      恢复后自动开机

      开启后，恢复完成后虚拟机将自动开机。

      恢复前执行副本校验

      执行恢复前校验副本完整性。当副本校验状态为“无效”或“--”时，不支持开启此功能。

  7. 单击“确定”。
