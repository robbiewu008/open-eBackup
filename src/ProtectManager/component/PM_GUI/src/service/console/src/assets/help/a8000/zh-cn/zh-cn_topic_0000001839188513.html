步骤1：注册MongoDB集群

前提条件

注册MongoDB集群前，请确保集群中每个主机已进行注册。

操作步骤

  1. 选择“保护 > 大数据 > MongoDB”。
  2. 单击“注册”，注册MongoDB集群。
  3. 注册对话框中选择集群类型。

    • 单实例
      1. 配置数据库实例及其认证信息。

        相关参数说明如表1所示。

        表1 MongoDB单实例注册信息

        参数

        说明

        名称

        实例名称。

        主机

        选择要注册的数据库主机。

        端口

        MongoDB实例的端口。

        认证方式

        实例认证方式支持OS认证和数据库认证。

        • OS认证：基于操作系统的认证。
        • 数据库认证：通过数据库管理员用户和数据库管理员密码进行认证。

        数据库用户名

        仅在认证方式为数据库认证时展示，数据库的管理员用户的名称。

        数据库密码

        仅在认证方式为数据库认证时展示，数据库管理员用户的密码。

        数据库安装目录

        请填写mongo/mongod/mongos文件所在的目录地址，如果未填写，则默认使用全局变量中配置的目录地址。

        数据库工具安装目录

        请填写mongodump/mongorestore工具所在的目录地址，如果未填写，则默认使用全局变量中配置的工具目录地址。
      2. 单击“确定”。
    • 集群实例
      1. 自定义实例名，并选择集群类型，包含：副本和分片。
      2. （可选）批量配置数据安装目录或批量配置数据库工具安装目录。配置完成后，可以为添加的所有节点配置数据库安装目录或数据库工具安装目录。
      3. 单击“添加”，在添加对话框中添加集群实例下的所有节点。

        相关参数说明如表2所示。

        表2 MongoDB集群实例的节点信息

        参数

        说明

        主机

        选择要注册的数据库主机。

        端口

        MongoDB的端口。

        认证方式

        实例认证方式支持OS认证和数据库认证。

        数据库用户名

        仅在认证方式为数据库认证时展示，数据库的管理员用户的名称。

        数据库密码

        仅在认证方式为数据库认证时展示，数据库管理员用户的密码。

        数据库安装目录

        请填写mongo/mongod/mongos文件所在的目录地址，如果未填写，则默认使用全局变量中配置的目录地址。

        数据库工具安装目录

        请填写mongodump/mongorestore工具所在的目录地址，如果未填写，则默认使用全局变量中配置的工具目录地址。
      4. 单击“确定”。
