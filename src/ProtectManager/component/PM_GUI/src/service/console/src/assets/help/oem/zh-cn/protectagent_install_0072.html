备份恢复传输模式（适用于FusionCompute）

本产品对FusionCompute虚拟化环境进行保护时，当FusionCompute的生产存储为不同类型的存储时，备份恢复时支持不同的数据传输模式。

LAN-Free模式

当FusionCompute的生产存储为华为分布式块存储或eVol存储时，为了获得更高的备份恢复效率，在备份恢复过程中，本产品通过查询目标磁盘在华为分布式存储或eVol存储上的映射信息，将目标卷挂载至客户端所在主机，因此可以避免数据传输速率受到FusionCompute存储局域网络带宽的限制。

图1 LAN-Free模式

LAN-Base模式

当FusionCompute的生产存储为虚拟化存储（虚拟化本地硬盘、虚拟化SAN存储（IP-SAN和FC-SAN）、虚拟化NAS存储）时，在备份恢复数据过程中，通过客户端所在主机向VRM获取磁盘位置等信息，并与CNA节点建立连接，进行备份恢复数据传输。
