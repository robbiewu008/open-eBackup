Step 5: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy will be retained permanently. If incremental backup is not defined in the SLA but is selected during manual backup, the copies generated by the manual incremental backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.
  • During manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, are applied to the manual backup job.

Procedure

  1. Choose Protection > Big Data > Redis.
  1. In the row of the Redis cluster to be protected, choose More > Protect.

    You can select multiple Redis clusters for batch protection.

  2. Select an SLA.

    You can also click Create to create an SLA.

  3. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  4. Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row of the Redis cluster to be protected, choose More > Protect.
    2. You can select multiple Redis clusters for batch protection.
    3. Select an SLA.
    4. You can also click Create to create an SLA.
    5. Click OK.
    6. If the current system time is later than the start time of the first backup set in the SLA, you can choose to perform the backup immediately or wait for the next scheduling based on the backup policy set in the SLA after the backup SLA is associated with the resources.
