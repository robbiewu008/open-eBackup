Associating an Air Gap Policy

After creating an Air Gap policy, you need to associate the storage device with the Air Gap policy for the policy to take effect.

Procedure

  1. Choose Data Security > Air Gap.

    For 1.5.0, choose Explore > Air Gap.

  2. Click the Storage Devices tab.
  3. Locate the row that contains the desired storage device, and choose More > Associate Policy.
  4. In the window that is displayed on the right, set related information. Table 1 describes the parameters.

    Table 1 Parameters for associating a policy

    Parameter

    Description

    Air Gap Policies

    Select a created Air Gap policy.

    Logical Port

    This parameter indicates the ports that support replication within the time window set in the Air Gap policy.

    Forcibly Suspend Replication

    After this function is enabled, if the replication duration exceeds the replication time window set in the Air Gap policy, the replication job will be suspended forcibly. This option is enabled by default.

    Linked Detection

    NOTE:

    Only 1.6.0 and later versions support this parameter.

    Whether to enable association intelligent detection with Air Gap policies. After this function is enabled, the system forcibly disconnects the replication link when detecting that the file system of the current device is infected. In this case, the original Air Gap policy becomes invalid. After confirming that the file system is secure, you can choose More > Enable Policy on the Storage Devices tab page to enable the policy again.

  5. Click OK.
