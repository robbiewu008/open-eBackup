Oracle数据库备份失败，错误详情包含“RMAN-06059: expected archived log not found, loss of archived log compromises recoverability”

现象描述

Oracle数据库备份任务失败，错误详情包含“RMAN-06059: expected archived log not found, loss of archived log compromises recoverability”。

可能原因

执行备份任务前，备份了日志的文件系统（DeviceManager上以L开头的文件系统为存放日志备份的文件系统）被删除，导致Oracle主机中存在失效的日志文件，进而备份失败。

处理步骤

  1. 登录Oracle数据库主机。

    • 如果是Linux操作系统，请使用PuTTY登录Oracle数据库主机，并执行su - oracle命令，切换到oracle账户。
    • 如果是Windows操作系统，请使用系统管理员登录Oracle数据库主机。

  2. 执行以下命令进入RMAN。

    • 如果是Linux操作系统，请执行以下命令：

      export ORACLE_SID=实例名称

      rman target /

    • 如果是Windows操作系统，请执行以下命令：
      1. 在键盘上按“Win+R”，打开运行窗口。
      2. 输入“cmd”，执行以下命令。

        set ORACLE_SID=实例名称

        rman target /

  3. 执行以下命令检验并删除失效的日志文件。

    crosscheck archivelog all;

    delete expired archivelog all;
