Step 3: Configuring the Scripts for Application-Consistent Backup

If VMware VMs run a non-Windows OS, you need to configure the pre- and post-processing scripts for application-consistent backup on the VMs to be backed up before performing application-consistent backup. If VMware VMs run a Windows OS or in a non-application-consistent backup scenario, skip this section.

The following part describes how to configure the application-consistent backup scripts under the assumption that some database applications run on a VMware VM.

DB2 database
  1. Log in to the VMware VM for which you want to perform application-consistent backup.
  2. Check whether the corresponding script exists in the /usr/sbin/ directory.
    Assume that the DB2 database is used.
    • If /usr/sbin/pre-freeze-script and /usr/sbin/post-thaw-script exist, check whether the script content is correct by referring to DB2 Database.
    • If not exist, create the corresponding scripts and configure the scripts by referring to DB2 Database.

Before performing an application-consistent backup and creating a consistency snapshot, the system invokes the pre-freeze-script script to quiesce database I/Os to ensure data consistency. After a consistency snapshot is created, the system invokes the post-thaw-script script to unquiesce database I/Os.
