Step 2: Enabling the Automatic Backup Function of zkmeta

Before performing a TDSQL backup job, you need to enable the automatic backup function of zkmeta. Otherwise, cross-cluster database restoration will fail.

Procedure

  1. Log in to a schedule node as user root.
  2. Run the following command to modify configuration file confscheduler.xml:

    cd /data/application/scheduler/conf/

    vi scheduler.xml

    Add the following information to the configuration file:

    <back_zkmeta value="on"/>

  3. Log in to a ZooKeeper node as user root.

    cd /data/application/zookeeper/bin/

    ./zkCli.sh -server 127.0.0.1:Port number

    Example:

    cd /data/application/zookeeper/bin/
./zkCli.sh -server 127.0.0.1:2118

    Replace the port number in the command with the actual ZooKeeper port number queried on the TDSQL CHITU management console.

  4. Run the following commands to modify the configuration:

    1. Run the following command to query the configuration item content:

      get /tdsqlzk/configs/agent@global

      The following is an example of the query result:

      {"generate_es_conf":"0","oc_dir":"/data/oc_agent","ocagent_port":"8966"}
    2. Add the following content to the returned JSON result:
      "local_backup":"1"

      The following is an example of the added result:

      {"generate_es_conf":"0","oc_dir":"/data/oc_agent","ocagent_port":"8966","local_backup":"1"}
    3. Run the following command to set the result added in 4.b to the configuration item:

      set /tdsqlzk/configs/agent@global {"generate_es_conf":"0","oc_dir":"/data/oc_agent","ocagent_port":"8966","local_backup":"1"}

      Replace {"generate_es_conf":"0","oc_dir":"/data/oc_agent","ocagent_port":"8966","local_backup":"1"} in the preceding command with the actual one.

  5. Log in to all nodes where the cluster is located as user root and run the reboot command to restart the production host.
