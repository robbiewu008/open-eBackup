Manually Executing Backup

If you want to perform a backup immediately, you can manually perform the backup.

Context

  • During the execution of a manual backup job, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies and Automatic Indexing, will take effect.
  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication or archive policy has been defined in the SLA, the OceanProtect will perform replication or archiving once based on the SLA when you perform manual backup.

Precautions

  • For a NAS share discovered through device registration, if the NAS share has authentication information, complete the configuration by referring to Step 3: Configuring NAS Share Information. Otherwise, the backup fails.
  • For an NFS share, ensure that the current device has the read and write permissions on the NFS share directory and the no_root_squash option is enabled. For details about how to configure NFS share access permissions, see Step 5: Configuring Access Permissions.
  • For a CIFS share, ensure that the current device has the read and write permissions on the CIFS share directory. You are advised to use a username in the Administrator group for backup. Otherwise, only files and directories that have granted the read and write permissions to the device can be backed up. For details about how to configure CIFS share access permissions, see Step 5: Configuring Access Permissions.

Procedure

  1. Choose Protection > File Systems > NAS Shares.

    For 1.5.0, choose Protection > File Services > NAS Shares.

  2. In the row where a NAS share to be protected resides, click More > Manual Backup .

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

  3. Set the name of the copy generated by the manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  4. Select a protection policy.

    • If Small File Aggregation is not configured for NAS share protection, the protection policy can be set to Full backup or Forever Incremental (Synthetic Full) Backup.
    • If Small File Aggregation is configured for NAS share protection, the protection policy can be set to Full backup or Incremental backup.

  5. Click OK.
