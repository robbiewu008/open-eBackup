Scanning Tape Libraries

The system automatically scans for tape libraries every 10 minutes to detect tape libraries that are connected to the product. If a newly connected tape library has not been added to the product, you can perform operations in this section to manually scan for the tape library and add it to the product.

Constraints

  • Independent drives without robotic arms are not supported.
  • Physical tape libraries and virtual tape libraries of LTO-3 or higher are supported.
  • Only tape libraries with bar codes are supported.
  • The product is connected to physical tape libraries through Fibre Channels and to virtual tape libraries using the iSCSI protocol.
  • Different drives of tape libraries need to support the same tape standard.
  • Expiration of some tape space is not supported.
  • Only RW tapes are supported. WORM tapes are not supported.

Prerequisites

  • A tape library has been connected to the product.
  • For a physical tape library, ensure that there are import/export slots and tape mounting and unmounting operations are allowed. Assume that a Quantum tape library is used. The following operations are performed on the configuration management page of the tape library. For details about the operations related to tape libraries from other vendors, contact the vendors.
    • Select Library. In the Operations area on the right pane, click IE Area. In the displayed dialog box, set Slots (indicating the number of import/export slots) as required.
    • Select Library. In the Operations area on the right pane, click Settings. In the displayed dialog box, select Automatic Inventory, Library-Assisted Drive Unload, Logical Drive Serial Number Addressing, and Logical System Addressing.

Procedure

  1. Choose System > Infrastructure > Archive Storage Library.
  2. On the Tape tab page, select a node and click Tape Library.
  3. Click Scan Tape Library. The system automatically identifies the tape libraries that have been successfully connected to the product. After the scanning is complete, you can view information about the newly scanned tape libraries in the tape library list. Table 1 describes the related parameters.

    Table 1 Parameters related to the tape library

    Parameter

    Description

    Name

    Name of a tape library, which is generated by the system. The naming convention is TapeLibrary- + Name of the controller to which the tape library belongs + ID. IDs are sorted according to the scanning sequence of tape libraries.

    For example, if the name of the controller to which the first scanned tape library belongs is 0A, the name of the first scanned tape library is TapeLibrary-0A0.

    Status

    Current status of a tape library.

    • Online: The tape library is properly connected to the product.
    • Offline: The tape library is disconnected from the product.

    Serial No.

    Serial number of a tape library.

    Vendor

    Manufacturer of a tape library. All vendors of virtual tape libraries are supported. The supported vendors of physical tape libraries include IBM and Quantum.

    Product Model

    Model of a tape library.

    Number of Slots

    Number of tape library slots.

    Number of Drivers

    Number of tape library drivers.

    Owning Controller

    Controller to which a tape library belongs.
