Restoration Failed After Metadata Tables of the GraphBase Service in the HBase Application on the MRS Platform Is Backed Up

Symptom

After the GraphBase service is installed on the MRS platform, metadata tables meta_graph and graphbaseORM_systemNotifications are created in the default namespace of the HBase application. If the GraphBase service is not installed in the target cluster, the restoration will fail.

Possible Cause

When restoring metadata tables meta_graph and graphbaseORM_systemNotifications of the GraphBase service, use the required GraphBase service to verify the table information. Because the GraphBase service is not installed in the target cluster to be restored, the table information fails to be verified and the restoration job fails.

Procedure

  1. Install the GraphBase service in the target cluster to be restored.
  2. Deliver the restoration job again after performing 1.
  3. If the fault persists after performing 1 to 2 for multiple times, contact Huawei technical support.
