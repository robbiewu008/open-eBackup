Modifying Target Deduplication Settings

Once the target deduplication function is enabled, subsequent modifications to the function do not take effect. A file system is created for each protected object to store backup copies. After a backup copy is generated, any modification to the target deduplication function does not take effect. To modify the setting of the target deduplication function, you need to delete all copies of protected objects and create a new backup policy for these objects. Then the modification to the target deduplication function takes effect. The following describes how to modify the target deduplication function for NAS shares.

Procedure

  1. Choose Explore > Copy Data > File Systems > NAS Shares.

    For 1.5.0, choose Explore > Copy Data > File Services > NAS Shares.

  2. On the Copies tab page, locate the NAS share to be deleted based on the NAS share name. In the row of the target copy, choose More > Delete.
  3. Repeat 2 until all copies of the protected objects are deleted.
  4. Choose More > Modify Protection to create a backup policy or modify the corresponding backup policy, and then perform the backup again.
