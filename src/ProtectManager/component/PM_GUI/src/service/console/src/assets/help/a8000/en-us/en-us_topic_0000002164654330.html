Message "Virtual machine Consolidation Needed status" Is Displayed on the vCenter Server

Symptom

After a backup job is executed, the message "Virtual machine Consolidation Needed status" is displayed on the vCenter Server.

Possible Causes

The protected disk fails to be detached from the VM where the client is installed when the backup job is complete.

Troubleshooting

  1. Log in to the vCenter Server on a browser.

    Login address: https://vCenter Server IP address

  2. In the left navigation pane, select the corresponding VM.
  3. In the right pane, click the Monitor tab.
  4. Choose Tasks and Events > Tasks.
  5. In the task list, search for the tasks named Consolidate virtual machine disk files.

    • If Status is Unable to access file since it is locked, record the start time of the latest failed consolidation task.
    • Otherwise, contact technical support engineers.

  6. Obtain the VMDK suffix path of the protected disk file and the IP address of the host where the client is installed, and log in to the vCenter Server to detach the disk.

    • For 1.5.0: Perform the following operations:
      1. Choose Protection > Virtualization > VMware.
      2. In the navigation tree on the left, select the vCenter Server where the VM is located.
      3. Switch to the VMs tab page, and click in the Name column.
      4. Enter the VM name in the search box to search for the VM.
      5. Click the name of the VM.
      6. On the page that is displayed in the right pane, choose Jobs > Historical Jobs.
      7. On the Historical Jobs page that is displayed, find the backup job in the corresponding period based on the time recorded in 5.
      8. Click the backup job. Under Events on the displayed page, locate the event that indicates the failure of detaching the protected disk from the VM where the client is installed when the job is complete, and obtain the VMDK suffix path of the protected disk file and the IP address of the host where the client is installed.
      9. Log in to the vCenter Server to detach the disk based on the suggestions in the event description.
    • For 1.6.0 and later versions: Perform the following operations:
      1. Choose Insight > Alarms and Events > Alarms.
      2. On the Alarms tab page, search for the alarm that indicates the failure of detaching the disk from the VM where the client is installed based on the time recorded in 5.
      3. Click the alarm description.
      4. On the page that is displayed, obtain the VMDK suffix path of the protected disk file and the IP address of the host where the client is installed.
      5. Log in to the vCenter Server to detach the disk based on the alarm handling suggestions.
