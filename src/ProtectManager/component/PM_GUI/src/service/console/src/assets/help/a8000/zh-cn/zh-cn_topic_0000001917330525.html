获取kubeconfig配置文件

当后续需要通过kubeconfig认证的方式注册集群时，可参考本节操作获取kubeconfig配置文件。

操作步骤

获取kubeconfig配置文件

  1. 查询kubeconfig配置文件信息前，需要确认当前集群控制平面是否通过用户使用场景的地址连接。使用PuTTY，登录待获取配置文件的Kubernetes集群管理节点。
  2. 执行kubectl cluster-info命令查询当前集群的控制平面。

  3. 登录ManageOne运营面。
  4. 依次选择“服务列表 > 云容器引擎CCE > CCE集群”，选择需要获取配置文件的Kubernetes集群名进入集群信息页面。
  5. 查询连接信息，确认当前集群控制平面是否通过用户使用场景的地址连接。

    • 否（如集群控制平面使用内网地址连接，而用户场景使用公网地址连接），需要根据使用场景切换kubectl的访问方式。操作步骤如下：
      1. 登录Kubernetes集群管理节点，根据使用场景，切换kubectl的访问模式。
        • VPC网络内接入访问请执行以下命令。
          kubectl config use-context internal
        • 互联网接入访问请执行以下命令。
          kubectl config use-context external
          互联网接入访问，如需开启双向认证请继续执行以下命令。
          kubectl config use-context externalTLSVerify
      2. 设置完成后再次执行kubectl cluster-info查询集群信息，确认切换访问模式成功。
    • 是，执行下一步。

  6. 在Kubernetes集群管理节点，执行以下命令查询配置文件信息。

    kubectl config view --flatten > kubeconfig.yaml

  7. 使用传输工具（如SFTP）下载当前命令执行目录的“kubeconfig.yaml”文件。
