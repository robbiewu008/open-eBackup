Viewing Information

After a Kubernetes cluster is registered with the OceanProtect, you can view information about the clusters, namespaces, and datasets.

Procedure

  1. Choose Protection > Containers > Kubernetes CSI.
  2. On the Cluster tab page, view the cluster information.

    Table 1 describes details about clusters.
    Table 1 Cluster information

    Parameter

    Description

    Name

    Cluster name.

    Click the name of a cluster to view its basic information and namespace information.

    Version

    Kubernetes cluster version number.

    Status

    Cluster status. The value can be Online and Offline.

    • Online: When the OceanProtect is properly connected to the Kubernetes cluster, Status is Online.
    • Offline: When the connection between the OceanProtect and Kubernetes cluster is abnormal, Status is Offline.

    IP Address

    Management IP address of the cluster.

    Owner

    The data protection administrator to which cluster resources are allocated for management. If -- is displayed, the resource has not been allocated to the data protection administrator.

  3. On the Namespace tab page, view the namespace information.

    For details about the namespace, see Table 2.
    Table 2 Namespace information

    Parameter

    Description

    Name

    Namespace name.

    Click the name of a namespace to view the namespace overview, copy data, and job information.

    • Overview

      The name, SLA information, job execution time, total number of copies, and dataset information are displayed.

    • Job

      All jobs, including backup and restoration jobs, are displayed.

    Owning Cluster

    Name of the cluster to which the namespace belongs.

    Cluster IP Address

    Management IP address of the cluster to which the namespace belongs.

    SLAs

    If an SLA is associated with the namespace, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status
    Whether a namespace is protected.
    • Unprotected: The namespace is not associated with an SLA, or it is associated with an SLA but protection is disabled.
    • Protected: The namespace is associated with an SLA and protection is enabled.
    • Creating: The namespace is associated with an SLA and protection is being created.

  4. On the Dataset tab page, view table information.

    For details about the dataset, see Table 3.
    Table 3 Dataset information

    Parameter

    Description

    Name

    Dataset name.

    Click the name of a dataset to view the dataset overview, copy data, and job information.

    • Overview

      The name, SLA information, job execution time, total number of copies, and volume information are displayed.

    • Copy data

      You can search for copies by year, month, or day.

      If is displayed below the time, a copy at that time exists. Click a date to go to the copy data page.

    • Job

      All jobs, including backup and restoration jobs, are displayed.

    Owning Namespace

    Name of the namespace where a dataset resides.

    Owning Cluster

    Name of the cluster where a dataset resides.

    SLAs

    SLA associated with the dataset.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status
    Whether a dataset is protected.
    • Unprotected: The dataset is not associated with an SLA, or the dataset is associated with an SLA but protection is disabled.
    • Protected: The dataset is associated with an SLA and protection is enabled.
    • Creating: The dataset is associated with an SLA and protection is being created.
