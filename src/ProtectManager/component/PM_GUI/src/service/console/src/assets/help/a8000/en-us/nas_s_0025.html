Step 4: Configuring Access Permissions (Applicable to OceanStor V5/OceanStor Pacific/NetApp ONTAP)

Before NAS share backup, you need to configure access permissions for the OceanProtect on the production storage device so that the OceanProtect can read data from the production storage in share mode.

In this section, OceanStor Dorado of versions earlier than 6.1.3 is used as the production storage to show how to configure NFS and CIFS share access permissions for the OceanProtect.

NFS Share

If the NAS share to be protected is an NFS share, perform the following steps:

  1. Log in to DeviceManager of the production storage device.
  2. Choose Services > File Service > Shares.
  3. On the NFS Shares tab page, choose More > Add Client on the right of the NFS share to be protected.
  4. Set client parameters.

    Table 1 describes the parameters.
    Table 1 Parameters for adding a client

    Parameter

    Description

    Type

    Select Host.

    Clients

    Client IP addresses accessing the share. Enter the backup IP address segment configured on the OceanProtect.

    You can obtain the backup IP address segment from the corresponding backup network plane (choose System > Container Service > Network Planes on DeviceManager of the OceanProtect).

    Permission

    Select Read-write.

  5. Select Advanced in the upper right corner to set advanced parameters.

    Set root Permission Constraint to no_root_squash and retain default values for other parameters.

  6. Click OK.

CIFS Share

If the NAS share to be protected is a CIFS share, perform the following steps:

  1. Log in to DeviceManager of the production storage device.
  2. Choose Services > File Service > Authentication Users.

    If there has been a local authentication user in the Administrators group, perform 5 to 8.

  3. Click the Windows Users tab and click Create to create a local Windows authentication user.
  4. Set parameters for the local authentication user.

    Table 2 describes the parameters.

    Table 2 Parameters for creating a local authentication user

    Parameter

    Description

    Name

    Set a new user name.

    Password

    Set a password for the new user.

    Confirm Password

    Enter the password again.

    Status

    Select Enable.

    Available Groups

    Select Administrators.

  5. In the navigation tree on the left, click Shares. Then, click the CIFS Shares tab.
  6. Choose More > Add User or User Group on the right of the CIFS share to be protected.
  7. Set user or user group parameters.

    Table 3 describes the parameters.
    Table 3 Parameters for adding a user or user group

    Parameter

    Description

    Type

    Select Local Windows authentication user.

    Available Users or User Groups

    Select the user created in 4.

    Permission

    Select Read-write.

  8. Click OK.
