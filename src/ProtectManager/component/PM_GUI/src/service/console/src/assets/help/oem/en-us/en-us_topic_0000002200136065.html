Step 1: Registering an Exchange Single-Node System or Availability Group

After the client is installed on the host where the Exchange database resides, the product automatically scans the Exchange database and user mailboxes. The product uses the scanned databases and mailboxes as objects to be protected.

Registering an Exchange Single-Node System

  1. Choose Protection > Applications > Exchange.
  2. On the Single-Node Systems/Availability Groups tab page, click Register.
  3. Select Single Server in the Type area.
  4. Configure information about the Exchange single-node system.

    Table 1 describes the related parameters.

    Table 1 Exchange single-node system registration information

    Parameter

    Description

    Type

    Select Single-node system.

    Name

    Customize the name of the Exchange single-node system to be registered.

    Hosts

    Select the host to be registered. In the single-node scenario, select one host node.

    Username
    Username of the Exchange host user who performs backup and restoration. The input format is as follows (assume that the username is Administrator and the domain name is EX):
    • EX\Administrator
    • Administrator@ex.com

    Password

    Password of the Exchange host user who performs backup and restoration.

    Concurrency

    Number of concurrent database or mailbox backup or restoration jobs in a single-node system. The number of concurrent jobs ranges from 1 to 10.

  5. Click OK.

Registering an Exchange Availability Group

  1. Choose Protection > Applications > Exchange.
  2. On the Single-Node Systems/Availability Groups tab page, click Register.
  3. Select Availability Group for Type.
  4. Configure Exchange availability group information.

    Table 2 describes the related parameters.

    Table 2 Exchange availability group information

    Parameter

    Description

    Type

    Choose Availability Group.

    Name

    Customize the name of the Exchange availability group to be registered.

    Hosts

    Select the host to be registered. You can select multiple host nodes.

    Username
    Username of the Exchange host user who performs backup and restoration. The input format is as follows (assume that the username is Administrator and the domain name is EX):
    • EX\Administrator
    • Administrator@ex.com

    Password

    Password of the Exchange host user who performs backup and restoration.

    Concurrency

    Number of concurrent database or mailbox backup/restoration jobs in an availability group. The number of concurrent jobs ranges from 1 to 10.

  5. Click OK.
