创建介质集（归档存储是磁带库）（适用于部分型号）

当将磁带库作为归档存储时，请参考本节创建介质集。如果归档存储不是磁带库请忽略本节。

背景信息

  • 介质集是基于磁带创建的，是一个磁带存储池，用于存放归档数据。因此创建介质集前需确保已有磁带库被扫描并更新至本产品。
  • 系统定期自动扫描磁带库（对于1.5.0版本系统每隔10分钟自动扫描，对于1.6.0及后续版本系统每隔24小时自动扫描），发现连接到本产品的磁带库后会将其自动更新至本产品。

    如果您发现磁带库还未更新至本产品，可手动扫描磁带库，具体操作可参考《管理员指南》中的“扫描磁带库”。

前提条件

请确保磁带库内的所有磁带的“Access Status”为“Accessible”，“Access Status”可在磁带管理界面查看。

操作步骤

  1. 选择“系统 > 基础设施 > 归档存储库”。
  2. 在“磁带”页签，选择某个节点，单击“介质集”。
  3. 单击“创建”。

    创建介质集参数说明如表1所示。

    表1 创建介质集参数

    参数

    说明

    名称

    自定义介质集的名称。

    驱动块大小

    支持64KB、128KB、256KB。向磁带写入数据时，是一块一块写入的。介质集所属磁带数据会以设置的块大小进行写入。

    说明：

    建议按照文件大小选择驱动块大小。

    • 对于大文件，选择较大的驱动块大小，以保证读写效率。
    • 对于小文件，选择较小的驱动块大小，以充分利用磁带空间。

    数据压缩
    缩减数据量以减少存储空间，提升写入速度。
    • 启用：将开启磁带库的硬件压缩功能，启用压缩可以节省磁带容量并提升写入速度。
    • 禁用：将禁用磁带库的硬件压缩功能。

    所属控制器

    选择磁带所属控制器。

    磁带

    在左侧的磁带库列表中，选择某个磁带库，勾选右侧磁带库中的磁带。

    说明：
    • 支持添加多个磁带库中的磁带。不支持跨控制器添加磁带。
    • 向介质集添加磁带时，不支持添加已经加入其它介质集的磁带。
    • 如果待选择的磁带“位置”长时间显示为“Driver”、磁带未识别、或“驱动数量”与实际不一致，可能为磁带被锁定在驱动上导致，请先参考磁带设备操作指南手动卸载磁带，解锁驱动后，并重新“扫描磁带库”。

    可用磁带不足告警
    • 功能开启后需要设置“告警阈值”。当空白磁带和过期可覆盖磁带的总数小于该值时，将产生告警。此时，您可以插入空白磁带或者将存在数据的磁带标记为空。
      说明：
      • 如果启用了数据压缩，一个磁带能实际存储的最大数据量是未知的。比如磁带标称容量为50GB，实际可能存储60G、80G、100G的数据，所以无法做到按容量进行告警，只能按空白磁带和过期可覆盖磁带总数进行告警。
      • 告警阈值设置不能超过已选择磁带数量。
    • 功能关闭后，您将无法知道可用磁带数量。建议开启。

    告警阈值

    请设置磁带数量，当可用磁带数量小于该值时，将产生告警。

    保留范围
    • 不保护数据（根据需要循环覆盖磁带）：不保护磁带中的数据，有新的数据写入时，直接覆盖原有数据。选择此保留策略时，请确保当前介质集中至少有两个磁带。
    • 永久保留数据：永久保留磁带中的数据。
    • 定期保护数据：定期保护磁带中的数据。保护xx天/月/年。例如，设置定期保护数据的时间为10天，那么从对磁带最后一次写数据完成，经过10天后，将磁带标记为空，可以对磁带进行覆盖写。

  4. 单击“确定”。介质集创建完成后，系统每隔1小时刷新介质集的状态和容量。可能存在介质集的状态与容量与实际不一致的情况，请等待系统刷新。
