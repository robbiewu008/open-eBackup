Managing Event Dump

Through event dump, the system periodically exports events to local disks of the storage system and deletes event records from the database beyond the retention period.

Context

You can use the OceanCyber WebUI to download dumped event files and delete unnecessary event files from the directory.

Procedure

  1. Choose System > Settings > Event Dump.
  2. Modify dump parameters.

    1. In the Dump Parameters area, click Modify.
    2. Set dump parameters. Table 1 describes related parameters.
      Table 1 Event dump parameters

      Parameter

      Description

      Dump At

      Time when events are dumped.

      The dump time is usually set to a time when the system is idle.

      Dump Cycle

      Interval for the system to automatically dump event information.

      [Value range]

      7 to 120 days.

      For example, if Dump Cycle is set to 7, the system automatically dumps data every seven days.

      Keep Database Records Within the Latest

      Days within which generated event records in the database are retained.

      [Value range]

      7 to 120 days.

      For example, if this parameter is set to 7, the event records of the last seven days are retained in the database of the storage system.

      File Type

      Format of the event dump file.

      The value can be Excel or CSV.

      Language

      Language of the event information.

      The value can be simplified Chinese or English.
    3. Click Save.

  3. Configure and enable the SFTP server.

    1. In the SFTP Server area, click Modify.
    2. Click below Status to enable and set SFTP server parameters. Table 2 describes related parameters.
      Table 2 SFTP server parameters

      Parameter

      Description

      Username

      Username for the SFTP server used to store events.

      [Value range]

      • The value contains 1 to 255 characters.
      • The value cannot contain the following special characters: !':;|`$<>&-()#?"\* ;: !':;|`$<>&-()#?"\*.

      [Example]

      admin

      Password

      Password for logging in to the SFTP server.

      [Value range]

      The value contains 1 to 255 characters.

      [Example]

      user123456

      Service IP Address

      IP address of the SFTP server.

      [Value range]

      • When an IPv4 address is used:
        • The 32-bit IPv4 address is divided into four 8-bit fields that are expressed in dotted decimal notation.
        • Each field of the IPv4 address cannot be blank and must be an integer.
        • The value of the leftmost field ranges from 1 to 223 (excluding 127).
        • The values of other fields range from 0 to 255.
        • The IP address cannot be a special address such as the broadcast address.
      • When an IPv6 address is used:
        • The 128-bit IPv6 address is divided into eight 16-bit fields. Each 16-bit field consists of four colon-separated hexadecimal numbers.
        • In each 16-bit field, leading zeros can be omitted for simplicity. However, at least one digit must be reserved in each field.
        • For further simplicity, you can use a double colon (::) in place of a series of zeros. A double colon (::) can be used only once in an address. It can also represent the neighboring consecutive zeros at the beginning or end of an address.
        • The IP address cannot be set to a special address, such as a network address, loopback address, or multicast address.

      [Example]

      192.168.1.100

      fc00::1234

      Port

      Port number of the SFTP server.

      [Value range]

      1 to 65535

      [Example]

      22

      File Save Path

      File path for dumping event information.

      [Value range]

      • The value contains 1 to 1024 characters.
      • Special characters !':;|`$<>&-()#?"\* are not allowed. The first character cannot be a period (.). The first and last characters cannot be spaces. The first character after a slash (/) in the path cannot be a period (.).

      [Example]

      /test
    3. (Optional) Click Test to verify parameter values.
      • If a dialog box indicating an error is displayed, at least one parameter value is incorrect. Modify the parameter and try again.
      • If a dialog box indicating success is displayed, SFTP server parameters are correctly set.
    4. Click Save.

Follow-up Procedure

  • View and download dumped files:

    In the Dump History area, you can view the event dump history. In the Dump File area where the dumped event records are located, you can click the file name to download the file to a local directory for backup.

  • Delete dumped files:

    In the Dump History area, click More on the right of a dumped file and choose Delete to delete the dumped file.

  • Upload dumped events:

    In the Dump History area, click More on the right of a dumped file and choose Upload to upload the dumped file to the enabled SFTP server.
