使用Hot-Add传输模式备份虚拟机时，备份任务一直重试

现象描述

使用Hot-Add传输模式备份虚拟机时，备份任务详情重复显示“节点XX准备备份任务”，客户端日志中重复打印“==================stack list==================”堆栈日志，vCenter Server界面可查看到客户端所在虚拟机存在残留磁盘。

可能原因

虚拟机前一次备份或恢复任务未正常结束，导致客户端所在虚拟机上的磁盘残留异常。再次进行备份时，VDDK无法正常自动清理残留磁盘，导致备份任务一直重试。

处理步骤

  1. 终止正在重试的备份任务，确认客户端所在虚拟机不存在其他执行中的任务。
  2. 使用浏览器，登录vCenter Server。

    登录地址：https://vcenter_server_ip_address

  3. 在左侧导航树上选择对应的客户端所在虚拟机，右键单击“编辑设置”。
  4. 选择所有除系统盘和占位磁盘外的残留磁盘，单击右侧的，执行移除操作。注意不要勾选“从数据存储删除文件”。
  5. 单击“确定”。
