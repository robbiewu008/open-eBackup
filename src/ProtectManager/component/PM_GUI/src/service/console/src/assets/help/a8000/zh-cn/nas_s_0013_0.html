步骤1：添加存储设备

执行NDMP NAS文件系统备份恢复前，需要将待保护的NDMP NAS文件系统所在的存储设备添加至OceanProtect。

背景信息

OceanProtect支持对NDMP协议的NAS文件系统进行保护。

操作步骤

  1. 登录OceanProtect管理界面，具体操作请参见登录OceanProtect管理界面。
  2. 选择“保护 > 文件系统 > 存储设备”。
  3. 单击“添加设备”，添加存储设备。
  4. 选择NDMP Server设备类型。
  5. 按界面要求配置参数。参数说明如表1所示。

    表1 添加设备参数说明

    参数

    说明

    设备名称

    用户自定义设备名称。

    IP地址

    存储设备的管理IP地址。

    端口

    OceanProtect与存储设备的管理平面通信的端口号。

    默认端口号为10000。

    用户名

    存储设备NDMP管理员角色的用户名及密码。请确保该用户具有Restful的登录方式，且已完成密码初始化。

    说明：

    请使用管理员角色的用户，否则可能导致备份该设备下的资源失败。

    密码

    代理主机

    系统默认选择内置代理，当存在外置代理时，还可以选择外置代理。

    并发任务数

    默认并发任务数是8，可以输入范围1~64。

    说明：
    • 并发任务数不能超过存储厂商提供的流控限制。
    • 此“并发任务数”为当前存储设备可同时进行备份或恢复任务数的上限。

  6. 单击“确定”。

    添加成功后，已添加的存储设备信息将在“存储设备”页面显示。

    OceanProtect将自动扫描存储设备中的NDMP NAS文件系统，并在“保护 > 文件系统 > NDMP”页面展示。当注册的设备新增了文件系统时，可在设备所在行选择“更多 > 资源扫描”更新文件系统资源。
