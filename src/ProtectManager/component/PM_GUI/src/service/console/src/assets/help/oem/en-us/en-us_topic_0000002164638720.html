Step 2: Registering an openGauss/CMDB Cluster

Prerequisites

  • Before registering an openGauss/CMDB cluster, install the client on all nodes in the openGauss/CMDB cluster. For details, see the Client Installation Guide.
  • The deployment environment must contain required dependent tools (including the gs_om management tool).

Procedure

  1. Choose Protection > Databases > CMDB/openGauss.
  2. Click the Cluster tab.
  3. Click Register to register an openGauss/CMDB cluster.
  4. Configure the cluster authentication information.

    Table 1 describes the related parameters.

    Table 1 openGauss/CMDB cluster registration information

    Parameter

    Description

    Name

    Custom cluster name.

    Username

    Name of the OS user who runs the database.

    Type

    Cluster type, which can be Single Server, Distributed CMDB Cluster, or Active/Standby Replication.

    Environment Variable File Path

    During database installation, this path must be specified if separated deployment is used.

    For details about the query method, see Preparations for Backup.

    NOTE:

    This path is the environment configuration file path set by the user during separated deployment.

    Nodes

    Host where the openGauss/CMDB instance resides.

    NOTE:
    • The selected nodes must have the openGauss/CMDB database installed and belong to the same cluster.
    • All nodes in an active/standby replication cluster must be registered.

    DCS Address

    These two parameters are displayed when Type is set to Distributed CMDB Cluster.

    For details about the query method, see Preparations for Backup.

    DCS Port

    Database Account

    These two parameters are displayed when Type is set to Distributed CMDB Cluster.

    Enter the username and password of a database account that has the permission to back up or restore the database. For details about how to obtain the username and password, see Preparations for Backup.

    Database Password

  5. Click OK.
  • After the cluster is successfully registered, the system automatically scans instances and databases every 5 minutes. The built-in databases, such as template0, template1, and postgres, are not be scanned.
  • You can also manually scan the cluster. On the Cluster tab page, choose More > Resource Scan in the row of the target cluster to obtain instance and database information.
