Restoring HDFS Filesets

This section describes how to restore an HDFS fileset that has been backed up to the original location or a new location.

Context

Backup copies, replication copies, or imported copies can be used for restoration. Restoration to the original location or a new location is supported. Restoration to the original location using a replication copy is not supported.

Prerequisites

  • If you need to restore ACL information of HDFS files, ensure that the HDFS ACL configuration has been enabled on the big data platform before the restoration. For details, see Step 2: Checking the HDFS ACL Switch Status.
  • If you need to restore ACL information of HDFS files, you must have HDFS super user permissions. The user is configured in Step 3: Registering an HDFS Cluster.

Procedure

  1. Choose Explore > Copy Data > Big Data > HDFS.
  2. You can search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the fileset to be restored based on the fileset name and click the fileset name.

  3. Click Copy Data and select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in that month or day.

  4. In the row where the copy resides, choose More > Restore.

    Table 1 describes the related parameters.

    Table 1 Fileset restoration parameters

    Parameter

    Description

    Restore To

    The location to which the fileset is restored.

    • Original location: Restore the fileset to the original HDFS cluster.
    • New location: Restore the fileset to a new location. You must specify the target HDFS cluster and path.

    Target Cluster

    When restoring the fileset to a new location, you must specify the target HDFS cluster, which can be the original cluster or another cluster added to the management page.

    Path

    When restoring the fileset to a new location, you must specify the path for the restoration.

    Please select

    If files with the same names exist in the restoration path, you can select either of the following:

    • Skip existing files: Files with the same names are skipped and are not replaced.
    • Replace existing files

  5. Click OK.
