Restoring Files on an EVS Disk

This section describes how to restore files on an EVS disk by using the file-level restoration function.

Context

  • The OceanProtect supports file-level restoration using backup copies and replication copies. Note that data cannot be restored to the original location when a replication copy is used.
  • The types of Linux file systems that can be restored are ext2/ext3/ext4 and XFS. The types of Windows file systems that can be restored are NTFS, FAT, and FAT32.

Prerequisites

  • The IP address of the backup network plane has been configured for the cloud server to which files are to be restored.
  • If the target cloud server runs a Windows OS, ensure that ports 445 and 138 have been enabled for the cloud server. If the target cloud server runs a Linux OS, ensure that port 22 has been enabled for the cloud server.
  • If the target ECS runs a Windows OS, ensure that the CIFS service has been enabled for the target cloud server. For details, visit the Microsoft official website.
  • If the target ECS runs a non-Windows OS, ensure that rsync has been installed and started on the target cloud server and that the rsync version is 3.2.5 or later.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Huawei Cloud Stack.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    Click Resources to switch to the corresponding page, search for the VM to be restored by name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or date, copies exist in the month or on the date.

  4. Locate the copy to be restored and choose More > File-level Restoration.

    • For 1.5.0, only copies whose index status is Indexed can be used for file-level restoration. Copies whose index status is Indexed or Not indexed can be used for file-level restoration for 1.6.0 and later versions.
    • For 1.5.0, if automatic indexing has been enabled for the SLA associated with the VM corresponding to the copy, the index status of the copy is Indexed, and file-level restoration can be directly performed. If automatic indexing is disabled, click Manually Create Index and then perform file-level restoration.
    • A copy that does not contain system disks cannot be used for file-level restoration.
    • Reverse replication copies and archive copies cannot be used for file-level restoration.
    • Linked files do not support file-level restoration.
    • During Huawei Cloud Stack backup, the mount point information that is not written into the /etc/fstab file is stored in the memory. The OceanProtect does not back up the information and does not support file-level restoration. If new file systems need to support file-level restoration, write the mount point information of the new file systems to the /etc/fstab file.
    • If the name of a folder or file contains garbled characters, file-level restoration is not supported. Do not select folders or files of this type. Otherwise, the restoration job fails.
    • If the target cloud server runs a Windows OS and only the SMBv1 service is enabled, the time of the restored file is the current cloud server time.

  5. Select the files to be restored.

    • For 1.5.0: Select the files to be restored from the directory tree.
    • For 1.6.0 and later versions: Set File Obtaining Mode, which can be Select file paths from the directory tree or Enter file paths.
      • If you choose Select file paths from the directory tree for File Obtaining Mode but no file-level mounting task has been created for the current copy, click Create Mount. You can select files from the directory tree only after the mounting is successful.
      • For OceanProtect E1000, if there are more than 10,000 files in the selected directory, the system may time out, preventing you from selecting files or folders to be restored from the directory. In this case, you are advised to manually enter paths. For other models of this product, if there are more than one million files in the selected directory, the system may time out, preventing you from selecting files or folders to be restored from the directory. In this case, you are advised to manually enter paths.
      • When entering a file path, enter a complete file path, for example, /opt/abc/efg.txt or C:\abc\efg.txt. If you enter a folder path, for example, /opt/abc or C:\abc, all files in the folder are restored. The file name in the path is case sensitive.

  6. Select Original location or New location and set restoration parameters.

    • Select the original location for restoration, that is, restore data to the original directory on the original cloud server.
      Table 1 describes the related parameters.
      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Cloud Server IP Address
      Select the configured backup network plane IP address.
      NOTE:
      • If the cloud server IP address is not in the list, exit the current page. Then, choose Protection > Cloud Platforms > Huawei Cloud Stack and rescan the virtualization environment.
      • If the system cannot obtain the IP address of the backup network plane, manually enter the IP address.

      Cloud Server Username
      During file-level restoration, the target cloud server needs to verify the login credential of the OceanProtect. Enter a username and password for logging in to the target cloud server.
      • Windows OS: The default username is Administrator.
      • Linux OS: The default username is root.
      NOTE:
      • The login user must have the read and write permissions on the directory to which data is to be restored on the target cloud server.
      • After the restoration, the access permission on the file is the same as that of the login user.

      Cloud Server Password

      Overwrite Rule
      If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.
      • Replace existing files
      • Skip existing files: Files with the same names are skipped and are not replaced.
      • Only replace the files older than the restoration file
        NOTE:

        If the access to the file with the same name on the target VM is denied, the file fails to be restored during file replacement.
    • Select a new location for restoration, that is, restore data to a specified VM.

      Table 2 describes the related parameters.

      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Target Cloud Platform

      Select the cloud platform to which the data is to be restored.

      Cloud Platform Tenant

      Select a cloud platform tenant.

      Region

      Select the region to which the data is to be restored.

      Project/Resource Set

      Select the project/resource set to which the data is to be restored.

      Cloud Server

      Select the cloud server to which the data is to be restored.

      Cloud Server IP Address
      Select the configured backup network plane IP address.
      NOTE:
      • If the cloud server IP address is not in the list, exit the current page. Then, choose Protection > Cloud Platforms > Huawei Cloud Stack and rescan the virtualization environment.
      • If the system cannot obtain the IP address of the backup network plane, manually enter the IP address.

      Cloud Server Username
      During file-level restoration, the target cloud server needs to verify the login credential of the OceanProtect. Enter a username and password for logging in to the target cloud server.
      • Windows OS: The default username is Administrator.
      • Linux OS: The default username is root.
      NOTE:
      • The login user must have the read and write permissions on the directory to which data is to be restored on the target cloud server.
      • After the restoration, the access permission on the file is the same as that of the login user.

      Cloud Server Password

      Overwrite Rule
      If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.
      • Replace existing files
      • Skip existing files: Files with the same names are skipped and are not replaced.
      • Only replace the files older than the restoration file
        NOTE:

        If the access to the file with the same name on the target VM is denied, the file fails to be restored during file replacement.

  7. Click Test and ensure that the target VM to be restored is properly connected to the OceanProtect.
  8. Click OK.
