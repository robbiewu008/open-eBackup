Backup Fails Because the Number of Asynchronous I/O Requests in the Production Environment Reaches the Maximum

Symptom

A backup job fails. You can view the description of the failed event on the job page, as shown in the following figure.

Possible Cause

The number of asynchronous I/O requests in the production environment reaches the maximum. As a result, the backup fails.

Procedure

  1. Log in to the production environment and run the vi /etc/sysctl.conf command to check the value of fs.aio-max-nr in insert mode.
  2. Determine the cause of the problem according to the following scenarios:

    • If the fs.aio-max-nr parameter is abnormal (less than the reference value or not found), reconfigure the parameter based on the reference value 18446744073709551615 of the parameter in Linux Kernel Parameters, and then go to 3.
    • If the fs.aio-max-nr parameter exists and its value is normal (you can refer to the parameter value in Linux Kernel Parameters), collect logs and contact Huawei technical support engineers.

  3. Run the sysctl -p /etc/sysctl.conf command for the configuration to take effect. After the command is executed, perform the backup again.
