Viewing Environment Information of the SQL Server Database

After the SQL Server database is registered with the OceanProtect, you can view details about the database, the host where it resides, and the cluster or availability group to which it belongs on the management page.

Procedure

  1. Choose Protection > Databases > SQL Server.
  2. View the details.

    • On the Cluster tab page, view the SQL Server cluster information.

      If a database in a cluster has been registered, the cluster information is displayed on the page.

      Table 1 describes the detailed information.
      Table 1 Cluster information

      Parameter

      Description

      Name

      Name of a database cluster.

      • If you click the cluster name, host members and all authenticated databases in the cluster are displayed.

      Status
      Cluster status.
      • Online: When a host is powered on and ProtectAgent on the host is running properly, the Status of the host is Online.

        If there are online hosts in a cluster, the cluster status is Online.

      • Offline: When a host is shut down or ProtectAgent on the host is faulty, the Status of the host is Offline.

        If all hosts in a cluster are offline, the status of the cluster is Offline.

      Owner

      Data protection administrator of a cluster.

      If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no authorization is performed, -- is displayed.
    • On the Instances tab page, view the database instance information.
      Table 2 describes the database instance information.
      Table 2 Database instance information

      Parameter

      Description

      Name

      Name of a database instance.

      Owning Host/Cluster

      Name of the host or cluster where a database instance resides.

      Node IP Address

      IP address of the host where a database instance is located.

      Version

      Version of the database where an instance is located.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether an instance is protected.

      • Unprotected: The instance is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The instance is associated with an SLA and protection is activated.
      • Creating: The instance is associated with an SLA and protection is being created.
      1. Click the name and check the overview, copy information, and jobs.
        • Overview

          Displays the SLA information, job execution time, total number of copies, nodes, and databases.

        • Copy data

          You can search for copies by year, month, or day.

          If is displayed below the time, copies exist at that time.

        • Job

          Backup and restoration jobs are displayed.

    • On the Availability Group tab page, view the database availability group information.
      Table 3 describes details about a database availability group.
      Table 3 Availability group information

      Parameter

      Description

      Name

      Name of a database availability group.

      Owning Cluster

      Name of the cluster to which a database availability group belongs.

      SLAs

      If an availability group is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Whether an availability group is protected.

      • Unprotected: The availability group is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The availability group is associated with an SLA and protection is activated.
      • Creating: The availability group is associated with an SLA and protection is being created.
      1. Click the name and check the overview, copy information, and jobs.
        • Overview

          Displays SLA information, job execution time, total number of copies, and protected objects.

        • Copy data

          You can search for copies by year, month, or day.

          If is displayed below the time, copies exist at that time.

        • Job

          Backup and restoration jobs are displayed.

    • View the database information on the Databases tab.

      The information about the scanned database is displayed.

      Table 4 describes the database information.
      Table 4 Database information

      Parameter

      Description

      Name

      Database name.

      Owning Host/Cluster

      Name of the host/cluster where a database resides.

      Owning Instance

      Name of a single instance or cluster instance.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Indicates whether the database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is enabled.
      • Creating: The database is associated with an SLA and protection is being created.
      1. Click the name and check the overview, copy information, and jobs.
        • Overview

          The SLA information, job execution time, and total number of copies are displayed.

        • Copy data

          You can search for copies by year, month, or day.

          If is displayed below the time, copies exist at that time.

        • Job

          Backup and restoration jobs are displayed.
