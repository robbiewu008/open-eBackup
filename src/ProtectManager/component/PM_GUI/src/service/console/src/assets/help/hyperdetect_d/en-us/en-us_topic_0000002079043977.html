Managing File Systems

ProtectManager allows you to modify, remove, activate, or disable protection plans for local file systems.

You need to log in to ProtectManager, choose Protection > Storage > Local Resources > Local File Systems, and locate the desired local file system.

Table 1 describes the related operations.

Table 1 Operations related to protection plans for local file systems

Operation

Description

Navigation Path

Scanning resources

Context

File systems added to or deleted from a storage system can be discovered by ProtectManager after about 1 minute. You can manually scan for file systems in the current storage system.

Click Resource Scan in the upper right corner of the page.

Modifying protection

Context

You can modify the protection plan of a file system to change its associated SLA. After the modification, the next backup will be executed based on the new protection plan.

Precautions

The modification does not affect ongoing protection jobs. The modified protection plan will apply to the next protection period.

Locate the row that contains the desired file system and choose More > Modify Protection.

Removing protection

Context

This operation enables you to remove protection for a file system when it does not need to be protected. After the removal, the system cannot execute protection jobs for the file system. To protect the file system again, associate an SLA with the file system again.

Precautions

When a protection job of the file system is running, protection cannot be removed.

Locate the row that contains the desired file system and choose More > Remove Protection or click Remove Protection above the list.

Activating protection

Context

This operation enables you to activate a disabled protection plan of a file system to make the protection plan take effect again. After the protection plan is activated, the system backs up the file system based on the protection plan.

Precautions

This operation can be performed only for a file system that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection.

Disabling protection

Context

This operation enables you to disable the protection plan of a file system for which periodic protection is not required. After the protection plan is disabled, the system does not perform automatic backup jobs for the file system.

Precautions

Disabling protection does not affect ongoing backup jobs.

Choose More > Disable Protection.

Restoration

Context

This operation enables you to use a ransomware detection snapshot to restore data.

Locate the row that contains the desired file system and choose More > Restore.

Manual backup

Context

This operation enables you to perform a backup immediately.

Precautions

Copies generated by manual backup are retained for the duration defined in the SLA.

Locate the row that contains the desired file system and choose More > Manual Backup.
