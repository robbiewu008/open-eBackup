Manually Executing Backup

If you want to perform a backup immediately, you can manually perform the backup.

Context

  • During the execution of a manual backup job, parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, Job Failure Alarm, and Source Deduplication, take effect.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy is retained permanently. If Forever Incremental (Synthetic Full) Backup is not defined in the SLA but Forever Incremental (Synthetic Full) Backup is selected during manual backup, the copies generated in this manual Forever Incremental (Synthetic Full) Backup will be retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Prerequisites

  • If you want to back up data when the VM is started, ensure that Tools has been configured for the VM. Otherwise, backup cannot be performed. For details about how to install the tool, see the FusionCompute Online Help.
  • For FusionCompute 8.7.0 and later versions, if you want to use an independent backup network plane for LAN-based backup and restoration, configure an independent backup network plane through FusionCompute first. For details, see Configuring the Backup Network Plane (Applicable to FusionCompute 8.7.0 and Later Versions) (applicable to only 1.6.0 and later versions).

Procedure

  1. Choose Protection > Virtualization > FusionCompute.
  2. Click the Cluster, Hosts, VMs, or VM Groups tab based on the type of data to be backed up.

    • VM Groups is available only in 1.6.0 and later versions.
    • When you select Cluster or Hosts for backup, only VMs with the same SLA as the cluster or host are backed up. VMs with different SLAs from the cluster or host are not backed up.

  3. In the row of the cluster, host, VM, or VM group to be backed up, choose More > Manual Backup.

    You can select multiple clusters, hosts, or VMs and choose More > Manual Backup in the upper left corner of the page to manually back up data in batches.

  4. Set the name of the copy generated through manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  5. Select a protection policy, which can be Full backup or Forever Incremental (Synthetic Full) Backup.
  6. Click OK.
