Verifying the Restored User Mailbox Data (Applicable to Microsoft Exchange Server 2010)

After the restoration, you need to update the user mailbox status at the production end and create an Active Directory domain user to connect to the restored mailbox to verify the restored user mailbox data.

Precautions

After the restoration, you need to determine the user mailbox whose status needs to be updated based on whether a database with the same name exists in the target location.
  • If a database with the same name exists in the target location, you need to update the user mailbox status of the different mailbox in the database with the same name at the target production end.
  • If the database does not exist, you need to update the status of all user mailboxes in the database at the target production end.

Procedure

The following procedure uses mail2 as the original database name and mail2_test as the name of the database in the new location.

  1. After the restoration is complete, log in to the client where the target Exchange Server database resides as an administrator and open Exchange Management Shell.
  2. Run the following command to query the GUIDs of all mailboxes in database mail2_test:

    Get-MailboxStatistics -Database "mail2_test" | Select DisplayName,MailboxGuid

  3. Run the following command to update the mailbox status:

    Clean-MailboxDatabase "mail2_test"

  4. Log in to the Server Manager of the host where the client for the target database is installed. From the menu bar on the right of the dashboard, choose Tools > Active Directory Users and Computers.

  5. On the page that is displayed, select the Users directory under the Active Directory domain controller. Right-click the blank area on the right and choose New > User from the shortcut menu.

  6. After setting the name and login name of the new user, click Next to set the password. You can select Password never expires as required, click Next, and then click Finish. The Active Directory domain user is created.
  7. Enter the Exchange Management Console.
  8. Choose Microsoft Exchange On-Premises > Recipient Configuration > Disconnected Mailbox.
  9. On the Disconnected Mailbox page, right-click the mailbox to be reconnected and choose Connect from the shortcut menu.
  10. In the displayed Connect Mailbox dialog box, select the type of the mailbox to be connected and click Next.
  11. Select Existing user, click Browse to select the user to be connected, enter the alias, and click Next.
  12. Confirm the mailbox information and click Connect.
  13. Click Finish.
