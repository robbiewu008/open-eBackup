Step 1: Enabling the Snapshot Function for the Directory Where the Database Table Resides

Before performing protection, enable the snapshot function for the directory where the Hive database table to be protected resides.

Prerequisites

Before enabling this function, ensure that nodes in the big data cluster are not in the safe mode. For details about how to query whether a node is in safe mode and how to exit the safe mode, see Exiting the Cluster Safe Mode.

Procedure

  1. Use PuTTY to log in to any NameNode in the big data cluster.
  2. Log in to the Hive client based on the cluster authentication mode. Assume that MRS is used. For details, see "Using a Hive Client" in MapReduce Service Product Documentation of the corresponding version.
  3. Run the following command to query the parent directory of the database table (replace the database name and table name with the actual ones):

    SHOW CREATE TABLE Database name.Table name;

    For example, in command output hdfs://hacluster/user/hive/warehouse/partion_test.db/OrderInfo, /user/hive/warehouse/partion_test.db is the parent directory of the database table.

  4. For MRS, run the following command to exit the Beeline client:

    !q

  5. Run the following command to enable the snapshot function for the directory:

    hdfs dfsadmin -allowSnapshot directory_name

    In the command, directory_name indicates the parent directory of the database table to be protected.
