Step 7: Creating a Replication SLA

This section describes how to configure the SLA of replication policies so that you can replicate local backup copies.

Prerequisites

  • A backup SLA has been created. For details, see Step 7: Creating a Backup SLA.

Procedure

  1. Choose Protection > Protection Policies > SLAs.
  2. Locate the SLA for which you want to create a replication policy and choose More > Modify.

    The preset SLAs cannot be modified.

  3. Click the Replication Policy icon.
  4. Configure replication policy information and click OK.

    Click on the right of the page to create a maximum of four replication policies.

    Table 1 describes the related parameters.

    Table 1 Replication policy parameters

    Parameter

    Description
    Data Type
    NOTE:

    This parameter is available only in 1.6.0 and later versions.
    Type of the copy to be replicated. The value can be Data copy or Log copy.
    NOTE:
    • The Log copy option is displayed only for the Oracle or SQL Server application.
    • Restoration using log copies depends on data copies. Therefore, create a data copy replication policy before creating a log copy replication policy. Otherwise, log replication copies cannot be used for restoration.

    Name

    Replication policy name, which can be customized.

    Replication Mode
    Supported replication mode.
    • Cross-domain: Replication is performed between clusters.
    • Intra-domain: Replication is performed among multiple nodes in the same cluster.
    NOTE:

    You can select Intra-domain only when there are multiple nodes in a cluster in the cluster HA scenario.

    Replication Target Cluster

    Target cluster to which data is replicated.

    Select the target cluster, or click Add to add a new cluster.

    NOTE:

    This parameter is available only when Replication Mode is set to Cross-domain.

    Replication Target User

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    Select the system administrator or data protection administrator of the target end for replication.

    Password

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    Enter the password of the system administrator or data protection administrator of the target end for replication. After entering the password, click Authenticate to verify the password.
    Local Storage Type
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    Select the local storage type. The value is fixed at OceanProtect.

    Specify Target Location
    A backup storage unit or backup storage unit group can be selected as the target location for replication. If a backup storage unit group is selected, the system will select a backup storage unit from the backup storage unit group as the target location for first replication of a backup copy and preferentially select the backup storage unit used for the first replication in subsequent replications.
    NOTE:
    • For version 1.5.0, this parameter is available only when Replication Mode is set to Intra-domain.
    • A backup storage unit that has been specified as the target location for a backup policy cannot be used as the target location for a replication policy.

    Interval
    Interval at which replication is performed.
    • Perform replication immediately after a backup is complete.
    • Replication is performed every xx minutes, hours, days, or weeks.

    Replication and Retention Rules

    Select the backup copy to be replicated and set the retention period of replication copies.

    Once the retention period expires, the system automatically deletes the expired copies. The retention period must be longer than the replication interval.
    • Replicate all copies: replicates copies generated since a specified time and set the copy retention period. The copy retention period can be as follows:
      • 1 to 25550 days
      • 1 to 3650 weeks
      • 1 to 840 months
      • 1 to 70 years
      • Permanently
    • Replicate copies on the specified date: specifies the date and set the copy retention period. You can select multiple options at the same time:
      • Select the backup copies generated in a specified month of each year for replication. The replication copies are retained for x years (the retention period can be 1 to 70 years).
      • Select the first or last backup copy of each month for replication. The replication copies are retained for x months (the retention period can be 1 to 840 months).
      • Select the backup copies generated on a specified day of each week for replication. The replication copies are retained for x weeks (the retention period can be 1 to 3650 weeks).
      NOTE:
      • You can select Replicate copies on the specified date only for replication of full and forever incremental (synthetic full) backup copies on the specified date.
      • If multiple options are selected, copies on any date specified by these options are replicated, and the obtained replication copies are retained for the longest period specified in these options.
      • If you choose to replicate the last full backup copy in a month for 1.6.0 and later versions, the job for replicating the to-be-replicated full backup copy in the current month will be executed in the next month.

    First Run At

    Start time of the first replication job.

    If First Run At is later than the current time, data replication starts based on First Run At. If First Run At is earlier than the current time, data is replicated when the next replication period arrives.
    NOTE:
    • Ensure that the time of the host where the client is installed, local cluster, and target cluster is consistent.
    • If Air Gap is planned, ensure that the first execution time is within the Air Gap replication time window of the target cluster and that the start time of each replication job is within the Air Gap replication time window. If a replication job does not end within the Air Gap replication time window and Forcibly Suspend Replication is enabled for the Air Gap policy, the replication job will be forcibly terminated and fail immediately. To complete the replication job, disable Forcibly Suspend Replication for the Air Gap policy of the target cluster.

    Rate Limiting Policy

    Average maximum amount of data that can be transmitted per second in a replication job.

    Select the created rate limiting policy. For details about how to create a rate limiting policy, see Step 5: (Optional) Creating a Rate Limiting Policy.
    Link Deduplication
    NOTE:

    For 1.5.0, the parameter name is Data Deduplication.
    After this function is enabled, the system deduplicates data copies during remote replication to improve replication efficiency. You are advised to enable this function.
    NOTE:
    • Before enabling this function, ensure that target deduplication has been enabled.
      • If Target Deduplication exists in the backup policy of the SLA, enable Target Deduplication. Otherwise, link deduplication does not take effect.
      • If Target Deduplication does not exist in the backup policy of the SLA, target deduplication is enabled by default.
    • This parameter is available only when Local Storage Type is set to OceanProtect.
    Link Compression
    NOTE:

    For 1.5.0, the parameter name is Data Compression.

    After this function is enabled, the system compresses data copies during remote replication to improve replication efficiency. You are advised to enable this function.
    WORM Setting
    NOTE:

    This parameter is available only in 1.5.0.

    After this function is enabled, generated copies are WORM copies. WORM copies cannot be deleted, and their retention periods cannot be shortened.

    Job Failure Alarm

    After this function is enabled, an alarm is automatically sent if a job fails and is automatically cleared after successful execution.

  5. Click OK and confirm as prompted.
  6. Associate a resource with a replication SLA policy to periodically replicate copies.

    • If an SLA policy has been associated with resources, the system protects the resources based on the SLA policy.
    • If the SLA policy is not associated with any resource, associate a resource with the SLA policy for protection. For details, see Step 8: Performing Backup.
      • When executing a replication job, ensure that the used capacity of storage pools on the local and target clusters does not exceed Data Backup Capacity specified in the license. Otherwise, the replication job fails. Perform the following steps to check whether the capacity is sufficient:
        1. Log in to DeviceManager of the backup storage devices in the local and target clusters. For details, see Logging In to DeviceManager.
        2. Choose System > Storage Pools and view the used storage pool capacity.
        3. Choose Setting > License Management and check Data Backup Capacity.

          The used storage pool capacity cannot exceed Data Backup Capacity.

      • You cannot view the replication copies generated by replication jobs in the local cluster, but can only view them or use them for restoration in the target cluster.
