Viewing Hyper-V Copy Information

This section describes how to view Hyper-V copy information.

Procedure

  1. Choose Explore > Copy Data > Virtualization > Hyper-V.
  2. View the copy information.

    • Replication copies can be viewed only in the target cluster.
    • Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.
    • Click the Resources tab to view VM copy data by VM. When you need to restore a faulty VM, you are advised to search for the target copy through this portal.

      Table 1 describes the related parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Name

      Name of a VM.

      Location

      Compute location of a VM.

      Status

      Whether the VM is registered with the product.

      • If yes, Status is Existing.
      • If no, Status is Not existing.

      Copies

      Quantity

      Number of copies generated based on the VM. The number of replication copies is displayed only in the target cluster.

      Replication Copy SLA

      SLA associated with a replication copy.

      If an SLA has been configured for replication copies generated for the resource, the SLA name is displayed.

      Replication Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is enabled.
    • Click the Copies tab to view information about all copies by copy. When you need to restore data to the original host or a different host based on a specified copy, you are advised to search for the copy through this portal.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copies

      Backup Copy Generation Time

      Time when a backup copy is generated.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being restored or instantly recovered.
      • Mounting: The copy is being mounted.
      • Unmounting: The copy is being unmounted.
      • Mounted: The copy has been mounted.
      • Deletion failed: The copy fails to be deleted.

      Location

      Location of a copy.

      • If Generation Mode of a copy is Backup, Cascaded replication, Reverse replication, or Replicate, the copy is stored in the corresponding backup storage unit. In this case, the name of the corresponding backup storage unit is displayed.
      • If Generation Mode of a copy is Archive to tape, the copy is stored in a tape library. In this case, the media set name is displayed.

      Storage Media Status
      Status of the storage medium where the copy resides.
      • Online: The copy can be managed.
      • Offline: The copy can be deleted only.
      NOTE:

      Storage Media Status is Online for backup storage units whose Health Status is Normal or Degraded and Running Status is Pre-Copy, Rebuilt, Online, Balancing, or Degraded.

      Generation Mode
      Mode in which a copy is generated.
      • Replicate: copies generated by replication jobs.
      • Backup: copies generated by backup jobs.
      • Archive to object storage: copies generated by object storage archiving jobs.
      • Archive to tape: copies generated by tape archiving jobs.
      • Cascaded replication: refers to a copy generated by a cascaded replication job.
      • Reverse replication: refers to a copy generated by a reverse replication job.

      Copy Generation Time

      Time when a copy is generated. Click the copy generation time to view the details about the copy.

      Copy Type

      Backup type of a generated copy. The option can be Full or Forever Incremental (Synthetic Full).

      Copy Expiration Time
      Copy expiration time. If a copy is permanently valid, -- is displayed.
      NOTE:

      The actual copy expiration time is subject to the latest time between Copy Expiration Time and WORM Expiration Time.

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.
      • : The WORM copy has expired. Copies in this state can be deleted.

      WORM Expiration Time
      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.
      NOTE:

      If the WORM expiration time is earlier than the WORM file system compliance clock time, the WORM validity period has expired. You can click the copy generation time to view the WORM file system compliance clock time. If the WORM file system compliance clock time is inconsistent with the system time, check the WORM status after the WORM file system compliance clock reaches the set expiration time.

      Copies

      File-level Mount Status
      File-level mounting status of a copy.
      • Mounted: A file-level mount point has been created.
      • Unmounted: No file-level mount point is created.
      • Mounting: A file-level mount point is being created.
      • Mounting failed: A file-level mount point fails to be created.
      • Unmounting: A file-level mount point is being deleted.
      • Unmounting failed: A file-level mount point fails to be deleted.

      Copies

      Index
      If automatic indexing is enabled in the SLA or indexes are manually created, the system creates indexes for all files in the copy, helping you quickly search for the required files for restoration. The options are as follows:
      • Indexed: The system has created indexes for a copy.
      • Indexing: The system is creating indexes for a copy after it is generated.
      • Not indexed: Automatic indexing is not enabled in the SLA or indexes are not manually created.
      • Index deleting: The index is being deleted.
      • Index failed: The index creation fails.

      Resources

      Name

      Name of a VM.

      Location

      Compute location of a VM.

      Status

      Whether the VM is registered with the product.

      • If yes, Status is Existing.
      • If no, Status is Not existing.

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
