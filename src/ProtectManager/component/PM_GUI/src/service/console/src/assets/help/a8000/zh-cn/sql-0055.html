步骤6：添加复制集群

在本端集群侧添加目标集群，建立主端设备和从端设备之间的逻辑连接，用于设备之间的数据传输。

前提条件

已准备用于复制的目标集群。

操作步骤

  1. 选择“系统 > 基础设施 > 集群管理”。
  2. 在“复制集群”页签，单击“添加”。
  3. 配置目标集群参数。

    相关参数说明如表1所示。
    表1 目标集群参数说明

    参数

    说明

    名称

    自定义目标集群的名称。

    角色

    固定为“复制目标集群”。

    IP地址
    输入目标集群任一控制器的管理、备份或复制IP地址。
    • 如果输入目标集群的管理IP地址，请确保本端与目标端的管理网络连通。
    • 如果输入目标集群的备份网络IP地址，请确保本端与目标端的备份网络连通。
    • 如果输入目标集群的复制网络IP地址，请确保本端与目标端的复制网络连通。
    说明：
    • 管理IP地址即为登录目标集群管理界面的IP地址。
    • 备份或复制网络IP请登录目标集群的管理界面，选择“系统 > 基础设施 > 集群管理”，在“备份集群”页签单击集群名称，获取任一控制器的备份网络IP地址（数据备份引擎）或复制网络IP地址。

    端口

    目标集群的端口。取值范围为：1~65535。

    IP地址框输入管理IP地址时，默认端口为25081；IP地址框输入复制网络IP地址或备份网络IP地址时，默认端口为30068。

    用户名

    步骤5：创建远端设备管理员中创建的用户及其密码。

    密码

  4. 单击“确定”。
