UIDs of TiDB Cluster Users Are Inconsistent

Symptom

The TiDB subjob fails to be executed. The cause is displayed in the job event, indicating that UIDs of users running the TiDB cluster are inconsistent.

Possible Causes

UIDs of users running the TiDB cluster are inconsistent. You need to change UIDs of users.

Troubleshooting

You are advised to perform the following operations on all nodes in the TiDB cluster.

  1. Run the following command to query the name and current UID of the user whose UID needs to be changed (username indicates the name of the user whose UID is to be changed):

    $ id username

  2. Run the following command to change the UID to a new one. In the command, new_uid indicates the new UID, and username indicates the name of the user whose UID is to be changed. Ensure that the new UID is unique in the system.

    $ sudo usermod -u new_uid username

  3. Execute 1 again to check whether the user UID is successfully changed.
