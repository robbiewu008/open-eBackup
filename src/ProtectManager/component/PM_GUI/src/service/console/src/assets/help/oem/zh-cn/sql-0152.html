恢复SQL Server实例中的单个或多个数据库

当需要对已备份的SQL Server实例中的单个或多个数据库恢复时，可以参考本节使用副本恢复数据库至原机或异机。

前提条件

  • 在进行异机恢复时，请确保目标主机上存在数据库实例。
  • 恢复前请确保恢复目标位置的数据目录剩余空间大于恢复副本缩减前大小，否则恢复将失败。

操作步骤

  1. 选择“数据利用 > 副本数据 > 数据库 > SQL Server”。
  2. 在“副本”页签，根据副本的信息找到需要恢复的副本。

    数据库级恢复支持恢复实例的备份副本或复制副本。

  3. 单击副本所在行的“更多 > 数据库级恢复”。
  4. 在“数据库级恢复”页签左侧框勾选需要恢复的数据库对象，在右侧框选择需要恢复到的目标位置。
  5. 设置数据库级恢复的相关参数，相关参数说明如表1所示

    表1 数据库级恢复参数说明

    参数

    说明

    恢复至

    选择恢复至“原位置”或“新位置”。

    位置

    当选择恢复至“原位置”时，默认显示原位置。

    标签

    说明：

    仅1.6.0及后续版本存在此参数。

    选择标签名，根据标签名筛选目标主机/集群。

    目标主机/集群

    当选择恢复至“新位置”时，选择目标主机上的实例。

    说明：

    恢复前确保已退出Microsoft SQL Server Management Studio，恢复过程中，系统将关闭目标数据库。

  6. 单击“确定”。
