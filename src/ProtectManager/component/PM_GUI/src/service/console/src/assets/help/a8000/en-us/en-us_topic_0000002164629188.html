Viewing Client Information

After the client is installed on a host, you can view details about the client on the management page.

Precautions

The GUI varies depending on the version.

Procedure

  1. Choose Protection > Clients > Clients.
  2. View client information.

    Table 1 describes the related parameters.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    Table 1 Client information

    Parameter

    Description

    Resource ID

    Client ID.

    Name

    Client name.

    IP Address

    IP address of the host where the client is installed.

    Status
    Client status.
    • Online: If the host where the client is installed is started and the client is running properly, Status is Online.
    • Offline: If the host where the client is installed is shut down or the client on the host is faulty, Status is Offline. For details about how to locate the offline cause, see How Do I Locate the Causse of a Client to Be Offline?.
    • Upgrading Queuing: The host where the client is installed is preparing to update the client.
    • Upgrading: The host where the client is installed is updating the client.
    • Modifying Application Type: The host where the client is installed is modifying the resource types that can be protected.

    CPU Usage

    Percentage of the host CPU occupied by the client.

    Memory Usage

    Percentage of the host physical memory occupied by the client.

    OS

    OS type of the host where the client is installed.
    Client Type
    NOTE:

    For version 1.5.0, the parameter name is Agent Type.

    Client type, which can be VMware backup client, General backup client, or SAN Client.

    Current Version

    Version of the client installed on the host.
    Owner
    NOTE:

    This parameter is available only in 1.5.0.

    Data protection administrator of the client.

    If another data protection administrator has been authorized for the client, the name of the data protection administrator will be displayed. If no data protection administrator is authorized, -- is displayed.
    SourceDedupe Supported or Not
    NOTE:

    The OceanProtect E6000 Appliance does not support this parameter.

    Some host OSs support source deduplication.
    • Yes: The DataTurbo service exists when the client is being installed or upgraded on the host.
    • No: The DataTurbo service does not exist when the client is being installed or upgraded on the host.
    NOTE:
    • For details about how to configure the source deduplication function, see the SourceDedupe User Guide of the corresponding version.
    • After source deduplication is enabled, add the source deduplication user to sudoers before the first backup. For details, see How Can I Add a Source Deduplication User to sudoers? (Applicable to the Linux OS).

    Log Level

    Level of a client log. Possible values are Info, Warn, Error, and Debug.

    Sharing

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    Sharing status of the client.

    LAN-Free

    NOTE:

    The OceanProtect E6000 Appliance does not support this parameter.

    LAN-free configuration status of the client.

    Installation Directory

    Installation directory of the client.

    Remarks

    Remarks of the client.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added to the client, the tag name is displayed. Otherwise, -- is displayed.
