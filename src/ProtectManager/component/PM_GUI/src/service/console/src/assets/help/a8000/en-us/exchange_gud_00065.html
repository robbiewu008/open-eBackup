Email-Level Restoration

This section describes how to use a mailbox backup copy to restore an email to the original mailbox.

Prerequisites

  • Before the restoration, ensure that the original mailbox exists and the database where the mailbox is located has been mounted.
  • Before the restoration, ensure that the Exchange Server service and related dependent services are running properly. For details about the corresponding services, see Overview of Exchange services on Exchange servers.
  • A mailbox backup copy is available.

Precautions

Before email restoration, if you delete a folder and emails in the folder, you need to delete the folder and emails from Deleted Items in the Exchange mailbox. Otherwise, the restored data will be incomplete.

Procedure

  1. Choose Explore > Copy Data > Applications > Exchange.
  2. Search for copies by mailbox resource or copy. This section describes how to search for copies by mailbox resource.

    On the Resources tab page, search for the mailbox where the email to be restored resides based on the mailbox name and click the mailbox name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed under a month or date, copies exist in the month or date.

  4. In the row where the Exchange mailbox copy resides, choose More > Email-Level Restore.
  5. Select an overwrite rule to replace or ignore existing emails.
  6. Select the email to be restored. You can select one or more emails for restoration.
  7. Click OK.

Follow-up Procedure

After the restoration, you need to update the user mailbox status at the production end and create an Active Directory domain user to connect to the restored mailbox to verify the restored user mailbox data. For details, see Verifying the Restored Mailbox Data (Applicable to Microsoft Exchange Server 2010) or Verifying the Restored Mailbox Data (Applicable to Microsoft Exchange Server 2013 and Later Versions).
