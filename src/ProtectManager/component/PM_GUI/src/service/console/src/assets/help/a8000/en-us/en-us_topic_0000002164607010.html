Restoring an NDMP NAS File System or File Directory

This section describes how to restore an NDMP NAS file system or file directory (that has been backed up) to its original location or a new location.

Context

The OceanProtect supports restoration to the original location or a new location using backup copies.

Prerequisites

  • To restore an NDMP NAS file system or file directory to a new location, a storage device residing at the new location needs to be added to the OceanProtect. For details about how to add a storage device, see Step 1: Adding a Storage Device.
  • Before the restoration, ensure that the remaining space of the target location is larger than that of the NDMP NAS file system or file directory to be restored. If the space is insufficient, expand the capacity of the target location or release sufficient space to complete the restoration.

    If the target location does not have sufficient space for restoration, the restoration job will be partially successful but the failed files cannot be restored. The restoration event contains messages similar to the following:

    RESTORE: Warning: We recommend that 13547742 inodes and 6991617042 kbytes of disk space be available on the target volume in order to restore this dump. You have 8348418 inodes and 271447472 kbytes of disk space on volume ndmp_shu8t.

    RESTORE: Warning: This restore will proceed, but may fail when it runs out of inodes and/or disk space on this volume.

Precautions

If you change the name of a protected NDMP NAS file system or file directory in the OceanProtect at the production end and scan for resources again, the OceanProtect will identify the file system or file directory as a new resource and the file system or file directory does not inherit the backup policy and backup copy of the original file system or file directory. If a file system or file directory with a new name needs to be backed up, you need to configure protection again.

Procedure

  1. Choose Explore > Copy Data > File Systems > NDMP.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, find the file system or file directory to be restored based on the name and click the name.

  3. On the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or day.

  4. In the row of the target copy, choose More > Restore.
  5. Restore an NDMP NAS file system or file directory to its original location or a new location.

    • Restore data to the original location.

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Location

      Location of a file system, which cannot be modified.

      Clients

      By default, the built-in client is selected. If an external client exists, you can also select the external client.
    • Restore data to a new location.

      Table 2 describes the related parameters.

      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Storage Device

      Select the storage device that resides at a new location.

      File System

      Select the file system to be restored.

      FQDN/IP

      Domain name or service IP address for accessing a NAS share.

      NOTE:

      If you need to enter an FQDN, configure the DNS service on the management page of the OceanProtect so that the OceanProtect can access external domain names. For details about how to configure the DNS service, see Configuring the DNS Service.

      Clients

      By default, the built-in client is selected. If an external client exists, you can also select the external client.

      Overwrite Rule

      Replace existing files.

  6. Click OK.
