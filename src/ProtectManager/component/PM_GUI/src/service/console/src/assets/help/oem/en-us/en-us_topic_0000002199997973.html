(Optional) Synchronizing Trap Configurations to an Oracle Host

If you have configured SNMP trap notification on the management page and want the alarm information of an Oracle host to be reported to the trap server through SNMP, synchronize the trap configuration to the Oracle host by referring to this section.

Context

For Oracle database protection, you need to enable the archive log mode. In this mode, you need to set the archive space. If Delete Archived Logs Immediately After Backup is enabled during protection creation, the system periodically deletes archive logs to release archive space after the product performs log backup for the database. However, if the archive log space is full because backup jobs are not properly performed for a long time or due to other reasons, production services will be affected.

The client monitors whether the archive space usage of the Oracle host exceeds the threshold (80% by default). When detecting that the usage exceeds the threshold, the client instructs users to handle the exception in a timely manner in either of the following ways to prevent impact on production services:

  • By default, the alarm is reported to the product, and the product reports the alarm to the trap server (only when SNMP trap notification is configured).
  • If the alarm cannot be reported to the product in a timely manner because the product is abnormal or the network between the product and the host is abnormal, the Oracle host reports the alarm to the trap server through SNMP. This section describes how to synchronize trap information to the Oracle host so that the trap server can receive the alarm reported by the Oracle host, indicating that the archive space usage exceeds the threshold.

Prerequisites

  • You have configured SNMP trap notification on the management page.
  • The Oracle host has executed backup once. Otherwise, the alarm will not be reported even if the archive space usage exceeds the threshold after trap information is synchronized to the host.

Procedure

  1. Choose Protection > Clients > Clients.
  2. Choose More > Sync Trap Info in the row where the Oracle host is located.

    If trap information needs to be synchronized for multiple Oracle hosts, repeat this step.

Related Operations

If SNMP authentication information or trap server information is modified on the management page, you need to synchronize the updates to the Oracle host. The procedure is the same as the preceding procedure.
