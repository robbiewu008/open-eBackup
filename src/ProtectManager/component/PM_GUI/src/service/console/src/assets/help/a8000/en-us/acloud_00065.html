Viewing Alibaba Cloud Resource Information

After Alibaba Cloud is registered with the OceanProtect, you can view information of Alibaba Cloud projects and cloud servers on the management page.

Procedure

  1. Choose Protection > Cloud Platforms > Alibaba Cloud.
  2. On the AZs tab page, view project information.

    Table 1 describes project details.
    Table 1 Project information

    Parameter

    Description

    Name

    Availability zone (AZ) name.

    Click an AZ name to view the AZ overview and jobs.

    • Overview

      Displays the name, SLA information, and cloud server information.

    • Jobs

      Displays all jobs, such as backup and restoration jobs.

    Location

    AZ location.

    SLAs

    If an AZ is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
    • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether an AZ is protected.

    • Unprotected: The AZ is not associated with any SLA, or the AZ is associated with an SLA but protection is disabled.
    • Protected: The AZ is associated with an SLA and protection is activated.
    • Creating: The AZ is associated with an SLA and protection is being created.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

  3. On the Resource Sets tab page, view cloud server information.

    Table 2 describes cloud server details.
    Table 2 Resource set information

    Parameter

    Description

    Name

    Resource set name.

    Click the name of a resource set to view the resource set overview and jobs.

    • Overview

      Displays the name, SLA information, and cloud server information in the resource set.

    • Jobs

      Displays all jobs, such as backup and restoration jobs.

    SLAs

    SLA associated with a resource set.

    SLA Compliance

    • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a resource set is protected.

    • Unprotected: The cloud server is not associated with any SLA, or the cloud server is associated with an SLA but protection is disabled.
    • Protected: The cloud server is associated with an SLA and protection is activated.
    • Creating: The cloud server is associated with an SLA and protection is being created.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

  4. On the Cloud Server tab page, view cloud server information.

    Table 3 describes cloud server details.
    Table 3 Cloud server information

    Parameter

    Description

    Name

    Cloud server name.

    Click the name of a cloud server to view the cloud server overview, copy data, and jobs.

    • Overview

      The name, SLA information, job execution time, total number of copies, and disk information are displayed.

    • Copy Data

      You can search for copies by year, month, or day.

      If is displayed below the time, copies exist at that time. Click a date to go to the copy data page.

    • Jobs

      Displays all jobs, such as backup and restoration jobs.

    Location

    Cloud server location. The format is Domain/Project/Cloud server.

    Status

    Cloud server status. Available options include Online and Offline.

    SLAs

    SLA associated with a cloud server.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup frequency requirements and the backup job is executed successfully, SLA compliance is met.
    • Missed: If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a cloud server is protected.

    • Unprotected: The cloud server is not associated with any SLA, or the cloud server is associated with an SLA but protection is disabled.
    • Protected: The cloud server is associated with an SLA and protection is activated.
    • Creating: The cloud server is associated with an SLA and protection is being created.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

    Cloud Server Group

    Cloud server group to which a cloud server has been added. A cloud server that has been added to a cloud server group does not support related protection operations any more. To directly protect the cloud server, remove it from the cloud server group.
