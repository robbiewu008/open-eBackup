Performing Periodic Backup

After resources to be protected are associated with an SLA, the system protects the resources based on the SLA.

Prerequisites

The prerequisites for the GaussDB T database backup in a single-node system are as follows:

  • You need to ensure that the GaussDB T database is running properly.
  • If full backup or log backup is required, you need to ensure that log archiving has been enabled before backup.

The prerequisites for backing up the GaussDB T cluster database are as follows:

The database needs to run in ARCHIVELOG mode and archive redo logs to multiple locations. For details about how to query and change the log archive mode, see Step 2: Setting the Redo Log Mode.

Precaution

After a full backup job fails, if the next backup is a log backup, the log backup job will fail because log backup depends on the copy generated in the last successful full backup. In this case, manually perform a full backup once.

Procedure

  1. Choose Protection > Databases > GaussDB T.
  2. Select the database instance that you want to protect and click Protect.

    You can also select multiple database instances for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

  1. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
