Syslog Notification

You can modify the recipient server address, notification type, and alarm severity of Syslog notifications based on service requirements. The generated alarms are reported to the third-party alarm platform of Splunk.

Procedure

  1. Choose System > Settings > Alarm Notification.
  2. Enable Syslog Notification: Click Modify to modify Severity, Notification Type, Language, or Send Device Name. For details, see Table 1.

    Table 1 Syslog notification parameter settings

    Parameter

    Description

    Severity

    Indicates the lowest severity of a Syslog alarm that can be sent, including Info, Warning, Major, and Critical.

    Notification Type

    Indicates the notification type, including Alarm, Alarm Recovery, and Events.

    Language

    The language for sending Syslog notifications can be simplified Chinese or English.

    Send Device Name

    Indicates whether the device name should be sent to the Syslog server.

    If you select Enable, the system sends the device name to the Syslog server.

  3. Click Save to complete the Syslog notification settings.
  4. Configure the address of the receiving server.

    1. Click Add. In the dialog box that is displayed, add a recipient server address. For details about parameter settings, see Table 2.
      Table 2 Receiving server address settings

      Parameter

      Description

      Server Address

      Enter the IP address of the server for receiving device alarm notifications.

      Port

      Port number of the Syslog notification. The default value is 514.

      NOTE:
      • The value of the port ID ranges from 1 to 65535.
      • The port number configured on the storage system must be the same as that configured on the Syslog server.

      Protocol

      Protocol through which the Syslog notification is sent, including UDP, TCP, and TCP+SSL/TLS. The default value is TCP+SSL/TLS.

      Certificate

      Select an external certificate of the Syslog type that has been added to the OceanCyber 300 Data Security Appliance. For details about how to add an external certificate, see Adding an External Certificate in the OceanCyber 300 1.2.0 Security Configuration Guide.
    2. Click OK.
    3. (Optional) Click Test to verify the connectivity between the storage system and the Syslog server.
    • A maximum of four receiving server addresses are supported.
    • To modify the added receiving server address, locate the row that contains the target server, and choose More > Modify.
    • To delete a receiving server address, locate the row that contains the server to be deleted, and choose More > Delete. Alternatively, select the recipient server address to be deleted and click Delete.
