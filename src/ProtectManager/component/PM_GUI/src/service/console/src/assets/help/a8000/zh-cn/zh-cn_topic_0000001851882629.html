恢复CNware虚拟机中的文件

当需要恢复虚拟机中的某些文件时，可以通过文件级恢复功能实现。

背景信息

  • OceanProtect支持使用备份副本、复制副本（不支持恢复至原位置）、反向复制副本（不支持恢复至原位置）、级联复制副本（不支持恢复至原位置）进行文件级恢复。
  • 支持恢复的Linux文件系统类型为ext 2/3/4、XFS。支持恢复的Windows文件系统类型为NTFS、FAT、FAT32。

前提条件

  • 已为待恢复到的目标虚拟机配置业务网络平面IP地址。

操作步骤

  1. 选择“数据利用 > 副本数据 > 虚拟化 > CNware”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 找到要恢复的副本，选择“更多 > 文件级恢复”。
  5. 勾选要恢复的文件后，进行恢复配置。

    • 副本的索引状态必须为“已索引”才支持文件级恢复。
    • 如果副本对应的虚拟机关联的SLA已开启自动索引，则副本的索引状态为“已索引”，可以直接进行文件级恢复。如果未开启自动索引，请先单击“手动创建索引”，再进行文件级恢复。
    • 如果副本中没有OS，则不能为其建立索引，因此该副本无法进行文件级恢复。
    • 备份CNware虚拟机时，未写入/etc/fstab文件的挂载点信息存储在内存中，OceanProtect不会备份该内容。因此，不会对该挂载点的文件生成索引，也不支持文件级恢复。如需对新增的文件系统生成索引并支持文件级恢复，请将新增的文件系统挂载点信息写入/etc/fstab。
    • 当文件夹或文件名称存在乱码时，不支持文件级恢复。请勿勾选该类型的文件夹或文件，否则恢复失败。
    • 当目标虚拟机为Windows OS时，且仅开启了SMBv1服务，文件恢复后的时间为虚拟机的当前时间。
    • 选择恢复至原位置，即恢复至原虚拟机的原目录。
      相关参数说明如表1所示。
      表1 原位置恢复参数说明

      参数

      说明

      虚拟机IP地址
      选择已配置的备份网络IP地址。
      说明：
      • 执行文件级别的恢复前，需要事先为虚拟机配置备份网络IP地址。文件级恢复过程中，系统会使用该地址连接目标虚拟机进行恢复。
      • 如果虚拟机IP地址未在列表内，请退出当前页面，在“保护 > 虚拟化 > CNware”页面重新扫描虚拟化环境。

      虚拟机用户名
      文件级恢复时，目标虚拟机需要验证OceanProtect的登录凭据，输入目标虚拟机的登录用户名及密码。
      • Windows OS：默认用户名为Administrator。
      • Linux OS：默认用户名为root。
      说明：

      该登录用户需要有目标虚拟机待恢复目录的读写权限。

      虚拟机密码

      覆盖规则
      当恢复路径中存在同名文件时，选择替换或跳过已存在的文件。
      • 直接替换已经存在的文件
      • 忽略已存在的文件，即跳过这些同名文件不替换。
      • 只替换比恢复文件更旧的文件
        说明：

        如果目标虚拟机的同名文件拒绝访问，则在替换文件进行恢复时会恢复失败。
    • 选择恢复至新位置，即恢复至指定虚拟机。

      相关参数说明如表2所示。

      表2 新位置恢复参数说明

      参数

      说明

      目标计算位置
      选择恢复到的目标计算资源。
      说明：
      • 指定虚拟机为Windows OS时，恢复到的目录与文件的源目录一致，如果虚拟机中不存在相应的盘符，则恢复失败。如：文件所在路径为D:\a\b.txt，则在指定虚拟机的D盘下新建a目录，再进行文件恢复，如果指定虚拟机不存在D盘，则恢复失败。
      • 指定虚拟机为Linux OS时，恢复到的目录与文件的源目录一致，恢复时在指定虚拟机中新建相应的目录，再进行文件恢复。

      虚拟机IP地址
      选择已配置的备份网络IP地址。
      说明：
      • 执行文件级别的恢复前，需要事先为虚拟机配置备份网络IP地址。文件级恢复过程中，系统会使用该地址连接目标虚拟机进行恢复。
      • 如果虚拟机IP地址未在列表内，请退出当前页面，在“保护 > 虚拟化 > CNware”页面重新扫描虚拟化环境。

      虚拟机用户名
      文件级恢复时，目标虚拟机需要验证OceanProtect的登录凭据，输入目标虚拟机的登录用户名及密码。
      • Windows OS：默认用户名为Administrator。
      • Linux OS：默认用户名为root。
      说明：

      该登录用户需要有目标虚拟机待恢复目录的读写权限。

      虚拟机密码

      覆盖规则
      当恢复路径中存在同名文件时，选择替换或跳过已存在的文件。
      • 直接替换已经存在的文件
      • 忽略已存在的文件，即跳过这些同名文件不替换。
      • 只替换比恢复文件更旧的文件
        说明：

        如果目标虚拟机的同名文件拒绝访问，则在替换文件进行恢复时会恢复失败。

  6. 单击“测试”，确保待恢复的目标虚拟机与OceanProtect网络连接正常。
  7. 单击“确定”。
