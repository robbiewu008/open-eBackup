Querying DB2 Information

Procedure

  1. Choose Protection > Databases > DB2.
  2. On the Cluster tab page, query the DB2 cluster information.

    1. View information about DB2 clusters registered with the OceanProtect.

      Table 1 describes the related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 DB2 cluster information

      Parameter

      Description

      Name

      DB2 cluster name.

      Status

      DB2 cluster status.

      • Online: If the connections between the OceanProtect and all nodes in the DB2 cluster are normal, Status is Online.
      • Offline: If the connections between the OceanProtect and all nodes in the DB2 cluster are abnormal, Status is Offline.

      Type

      Cluster type.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of a host or cluster.

      If a host or cluster has been authorized to a data protection administrator, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
    2. Click a cluster name to view the cluster basic information and the information about databases related to the cluster.

  3. On the Instance tab page, query the DB2 instance information.

    1. View details about DB2 instances.

      Table 2 describes the related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 2 DB2 instance parameters

      Parameter

      Description

      Name

      Instance name.

      Status

      Instance status.

      Owning Host/Cluster

      Host or cluster to which a instance belongs.

      Version

      Instance version.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    1. Click a name and check the basic information and databases.
      • Basic information

        Displays the instance name, owing host/cluster, version, and node information.

      • Databases

        Displays the database name, SLAs, SLA compliance, and protection status.

  4. On the Databases tab page, query the DB2 database information.

    1. View details about DB2 databases.

      Table 3 describes the related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 3 DB2 database parameters

      Parameter

      Description

      Name

      Database name.

      Owning Host/Cluster

      Host or cluster to which a database belongs.

      Owning Instance

      Instance to which a database belongs.

      SLAs

      If an SLA has been associated with a DB2 database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether a DB2 database has been protected.

      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click a name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Operation

        Displays all jobs related to a database, such as backup, restoration, and replication jobs. You can choose to display ongoing or historical jobs.

  5. On the Tablespace Set tab page, query the DB2 tablespace set information.

    1. View details about DB2 tablespace sets.

      Table 4 describes the related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 4 DB2 tablespace set parameters

      Parameter

      Description

      Name

      Name of a DB2 tablespace set.

      Owning Host/Cluster

      Host or cluster to which a DB2 tablespace set belongs.

      Owning Instance

      Instance to which a DB2 tablespace set belongs.

      Owning Database

      Database to which a DB2 tablespace set belongs.

      SLAs

      If an SLA has been associated with a DB2 tablespace set, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether a DB2 tablespace set has been protected.

      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click a name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Operation

        Displays all jobs related to a database, such as backup, restoration, and replication jobs. You can choose to display ongoing or historical jobs.
