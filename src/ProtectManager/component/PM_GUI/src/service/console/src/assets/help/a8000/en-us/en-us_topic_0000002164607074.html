Viewing NDMP NAS File Systems

After a storage device is added to the OceanProtect, it is automatically scanned by the OceanProtect and the detected file systems are displayed.

Procedure

  1. Choose Protection > File Systems > NDMP.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    Table 1 describes NDMP NAS file system information.

    Table 1 NDMP NAS file system information

    Parameter

    Description

    Name

    Name of an NDMP NAS file system.

    Click the name of an NDMP NAS file system to view its overview, copy data, jobs, and file system information.

    • Overview

      The name, SLA information, job execution time, and total number of copies are displayed.

    • Copy data

      You can search for copies by year, month, or day.

      If is displayed below the time, a copy at that time exists.

      Click Clear Index to delete all indexes of a resource.

      NOTE:
      • If a copy is being indexed, it cannot be deleted until the indexing is complete.
      • If Automatic Indexing is enabled for the SLA associated with a resource and Automatic Indexing is enabled for the backup policy, you need to disable Automatic Indexing or disassociate the resource from the SLA before deleting the index. If Automatic Indexing is enabled for the archiving policy, you need to disassociate the resource from the SLA before deleting the index.
    • Job

      All jobs, including backup and restoration jobs, are displayed.

    • File system information

      The owning device, tenant, and capacity of file systems are displayed.

    Storage Device

    Name of the owning storage device of the NDMP NAS file system.

    vStore

    Tenant to which the NDMP NAS file system belongs.

    SLAs

    Name of the SLA associated with the NDMP NAS file system.

    Click an SLA name to view its basic information and associated resources.

    • Basic information

      The backup policy and advanced settings are displayed.

    • Associated resources

      Resources associated with the SLA are displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
    • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

    Protection Status

    Indicates whether the NDMP NAS file system is protected.

    • Unprotected: The NAS file system is not associated with any SLA, or the NAS file system is associated with an SLA but protection is disabled.
    • Protected: The NAS file system is associated with an SLA and protection is enabled.
    • Creating: The NAS file system is associated with an SLA and protection is being created.

    Tag

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
