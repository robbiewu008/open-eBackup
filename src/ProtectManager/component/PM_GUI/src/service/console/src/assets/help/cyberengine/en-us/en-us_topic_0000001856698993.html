Creating a Real-Time Detection Policy

This section describes how to create a real-time detection policy.

Procedure

  1. On the top menu bar, choose Data Security > Ransomware Protection > Real-Time Detection.
  2. Click the Real-Time Detection Policies tab.
  3. Click Create.
  4. Enter the name of the real-time detection policy.
  5. Configure a snapshot locking policy. The default snapshot locking duration is 2 days. The value ranges from 1 to 14 days. Snapshots generated during real-time detection are secure snapshots. During the snapshot locking period, you cannot perform operations on the snapshot. After the lock period ends, you can manually delete the snapshot on the Snapshot Data page.
  6. Determine whether to enable False Alarm Analysis. False Alarm Analysis is enabled by default. Enabling this function improves the detection accuracy but prolongs the detection duration.

    • If you have higher requirements on the detection accuracy, you are advised to enable False Alarm Analysis.
    • If you have higher requirements on the detection duration, you are advised to disable False Alarm Analysis.

  7. (Optional) Enable honeypot detection. After the honeypot detection function is enabled, honeyfiles are generated in the directory of the selected file system. When an exception is detected, the system analyzes the honeyfiles to determine whether the file system is under ransomware attacks or the exception is caused by misoperations of users, so as to detect ransomware attacks in real time and reduce false detections.

    Set the honeyfile update frequency.

    • Periodic update: Set the frequency for updating the honeyfile. The value ranges from 1 to 30 days. When Periodic update is selected, honeyfiles can be adaptively updated based on the latest file system information, improving effectiveness in ransomware detection of honeyfiles.
    • No update: honeyfiles cannot be adaptively updated, which may reduce effectiveness in ransomware detection of honeyfiles. You are advised to update the honeyfile periodically.
      • After the honeypot detection function is enabled, a small number of honeyfiles are placed in the shared directory for ransomware detection. Do not encrypt or delete these honeyfiles, or change their file name extensions. Otherwise, false detection may occur.
      • The honeyfile detection function scans the metadata of files in the shared directory.
      • After the WORM function of the storage device is enabled, there may be residual honeyfiles in the shared directory. Honeypot detection is not recommended when WORM is enabled because WORM provides a strict data protection mechanism.
      • After the ransomware protection alarm 0x5F025D000F is cleared, the system automatically redeploys honeyfiles to ensure that there are available honeyfiles.

  8. Click OK.
