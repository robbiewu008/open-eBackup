Managing Informix Copies

This section describes operations that you can perform on copies on the product.

You need to log in to the management page and choose Explore > Copy Data > Databases > Informix to go to the Informix instance copy page.

  • Click Copy to perform operations by copy.

    Table 1 describes the related operations.

    Table 1 Operations on Informix copies

    Operation

    Description

    Navigation Path

    Modifying a copy retention policy

    Scenario

    This operation enables you to modify the retention period of a copy.

    Note

    The new retention period of a copy starts from the time when the modification takes effect.

    Choose More > Modify Retention Policy in the row where the copy resides.

    You can set the retention period to be permanent or to xx days, xx weeks, xx months, or xx years.

    Deleting a copy

    Scenario

    This operation enables you to delete an unneeded or unusable copy to free up storage space.

    Note

    • Before deleting a copy, ensure that the copy is no longer used for restoration.
    • Only copies in the Invalid or Normal status can be deleted.
    • If a backup set has multiple copies, the latest copy cannot be deleted.
    • If a resource has multiple copies and only one full replication copy, the full replication copy cannot be deleted.
    • Copies generated by log backup cannot be deleted.

    Choose More > Delete in the row where the copy resides.

    Note: The configuration for restoring an Informix instance is not described in this section since it is similar with Restoring the Informix Database.
  • Click Resources to perform operations by resource.

    Table 2 describes the related operations.

    Table 2 Operations related to replication copies

    Operation

    Description

    Navigation Path

    Creating protection for replication copies

    Scenario

    This operation enables you to periodically replicate replication copies.

    Locate the row that contains the target resource, choose More > Protect, and associate a replication SLA to replicate replication copies.

    Modifying the SLA associated with replication copies

    Scenario

    This operation enables you to modify the SLA associated with replication copies.

    In the row that contains the target resource, choose More > Modify Protection to associate a new SLA with the replication copy.

    Removing protection for replication copies

    Scenario

    You can remove the protection plan if the replication copies no longer need to be protected. After removal, the system cannot protect these replication copies.

    In the row that contains the target resource, choose More > Remove Protection.

    Activating protection for replication copies

    Scenario

    After the protection plan for replication copies is disabled, you can enable it again. Then, the system protects replication copies based on the protection plan.

    Choose More > Activate Protection.

    Disabling protection for replication copies

    Scenario

    This operation enables you to disable a protection plan when periodic protection on the replication copies is not required. After the protection plan is disabled, the system does not execute automatic protection jobs of these replication copies.

    Choose More > Disable Protection.

    Performing manual replication for a replication copy

    Scenario

    This operation enables you to replicate a replication copy immediately.

    Note

    Before performing this operation, ensure that a replication copy-based SLA has been created for the current resource and the SLA is a replication SLA.

    Choose More > Manually Replicate.
