Viewing Quotas and Functions

This section describes how to view quotas and functions of data protection administrators.

Procedure

  1. Choose System > Security > Quotas and Functions.
  2. View quota and function information. Table 1 describes the parameters.

    Table 1 Quota and function parameters

    Parameter

    Description

    Name

    Username.

    Type

    Type of a user. The value can be Local user, LDAP user, LDAP user group, SAML user, and HCS user.

    NOTE:

    Quotas and functions can be set only for data protection administrators.

    Backup Quota

    Quota used by backup and replication services. If the backup quota is set, the total quota and used quota are displayed. If the backup quota is not limited, Unlimited is displayed.

    NOTE:
    • After a backup job is successfully executed, the backup copy size is counted in the used quota. After a replication job is successfully executed, the replication copy size is counted in the used quota of the target end.
    • For replication services, only SAML users are restricted by the backup quota. Non-SAML users are not restricted by the backup quota.
    • The system verifies the backup quota before executing a backup or replication job.
      • If the verification is successful, the system executes the backup or replication job. After the job is complete, the used quota may be greater than the total quota.
      • If the verification fails, the backup or replication job fails to be executed.

    Archive Quota

    Quota used by cloud archive services. If the archive quota is set, the total quota and used quota are displayed. If the archive quota is not limited, Unlimited is displayed.

    NOTE:
    • There is no quota limit for tape archive.
    • After an archive job is successfully executed, the archive copy size is counted in the used quota.
    • The system verifies the backup quota before executing an archive job.
      • If the verification is successful, the system executes the archive job. After the job is complete, the used quota may be greater than the total quota.
      • If the verification fails, the archive job fails to be executed.

    Functions

    Data protection functions supported for a data protection administrator, including Backup, Replicate, Archive, and Restore. For example, when Backup is enabled, backup jobs can be executed successfully. Otherwise, backup jobs fail.
