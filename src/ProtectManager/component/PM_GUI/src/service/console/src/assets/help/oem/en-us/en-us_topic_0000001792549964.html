Uninstalling ProtectAgent (Applicable to the Linux OS/AIX OS)

After ProtectAgent is uninstalled, the product does not back up or restore resources.

Context

The following operations use the ProtectAgent installation directory /opt as an example.

Prerequisites

If you have set aliases for rm, cp, mv, and other commands that need to be input in interactive mode on the agent host (for example, alias setting mv=mv -i, indicating that the mv command is executed with the -i parameter by default), cancel the alias setting first. Otherwise, the uninstallation will fail.

Procedure

  1. Use PuTTY to log in to the host where ProtectAgent needs to be uninstalled as a system administrator.
  2. Run the following command to go to the ProtectAgent installation directory:

    cd /opt/DataBackup/ProtectClient

  3. Run the following command to uninstall ProtectAgent:

    sh uninstall.sh

    If information similar to the following is displayed, enter y.

    Are you sure you want to uninstall DataBackup ProtectAgent? y/n:

    If message "DataBackup ProtectAgent has been uninstalled successfully." is displayed, the uninstallation is successful.

  1. If ProtectAgent needs to be installed on the host subsequently, delete the /etc/HostSN/HostSN file from the host. Otherwise, skip this step.

    rm -rf /etc/HostSN/HostSN

  1. If ProtectAgent needs to be installed on the host subsequently and the OS has been reinstalled on the host or the /etc/HostSN/HostSN file has been deleted from the host before ProtectAgent is installed, perform the following operations to delete the host on the management page. Otherwise, skip this step.

    1. Choose Protection > Clients > Clients.
    2. Search for the host from which ProtectAgent has been uninstalled based on the IP address and choose More > Delete Host.

      Before deleting the host, ensure that the host is not associated with any SLA.
