Enabling File Interception

This section describes how to enable file interception.

Prerequisites

NFS or CIFS sharing has been enabled for a file system.

Precautions

File name extension filtering rules do not take effect on the secondary file system of a HyperMetro domain or remote replication vStore pair.

Procedure

  1. On the navigation pane, choose Data Security > Ransomware Protection > Ransomware Detection Settings.
  1. On the NAS Ransomware Detection tab page, locate the row that contains the vStore for which you want to enable file interception and choose More > Enable File Interception.

    • You can also select multiple vStores for which you want to enable file interception and click Enable File Interception above the vStore list.
    • If file interception is not required, choose More > Disable File Interception. Alternatively, select multiple vStores for which you want to disable file interception and choose More > Disable File Interception above the vStore list.

  2. Click the number in the column of Number of File Name Extension Filtering Rules of the vStore to view the association status of the file name extension filtering rules. Table 1 describes the association status. If the association status is Applied, the file name extension filtering rule has taken effect.

    Table 1 Association status of file name extension filtering rules

    Association Status

    Description

    Associated

    The file name extension filtering rule has been associated with the vStore.

    Applied

    The file name extension filtering rule has been applied to the vStore's file system. You are not allowed to create, open, and rename files with the same extension in the file system.

    NOTE:

    When accessing a file system through a CIFS share, you must open a file before deleting it. Therefore, the deletion operation on files with the same extension in the file system is also intercepted.

    Applying

    The file name extension filtering rule is being applied to the vStore's file system.

    Disassociating

    The file name extension filtering rule is being disassociated from the vStore. Its application to the file system will also be canceled.

    Canceling Application

    The application of the file name extension filtering rule to the vStore's file system is being canceled.

    Exception

    The applied file name extension filtering rule does not take effect for some file system shares.
