Creating Anonymization Policies

Common anonymization policies have been preset in the product. If the preset anonymization policies do not meet your requirements, you can customize some policies.

After an anonymization policy is created, you can select it for a database on the Data Anonymization page. The product processes sensitive data in the database based on the identification rules in the anonymization policy.

Procedure

  1. In the navigation pane, choose Data Security > Data Anonymization > Anonymization Policies.

    For 1.5.0, choose Explore > Data Utilization Policies in the navigation tree and click Anonymization Policies.

  2. On the Anonymization Policies tab page of the Anonymization Policies page on the right, click Create, and enter the identification rule as prompted.

    Table 1 describes the parameters.

    Table 1 Parameters related to the anonymization policy

    Parameter

    Description

    Name

    Name of an anonymization policy. It contains a maximum of 64 characters in length.

    Description

    Description of an anonymization policy. You are advised to enter the description based on the content to be anonymized to facilitate subsequent management.

  3. In the Select Identification Rule area, select one or multiple identification rules to be associated with.

    If the existing identification rules do not meet your requirements, you can click Add to create some rules. For details about the identification rule parameters, see Table 1.

  4. Click OK.
