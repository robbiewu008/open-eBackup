What Should I Do If a Message Indicating that the CA Certificate Has Expired or Is Invalid Is Displayed When I Add Archive Storage?

Question

When I add archive storage and the parameter configurations meet requirements, a message is displayed, indicating that the CA certificate has expired or is invalid.

Answer

  1. Enter https://DeviceManager management IP address of the Storage System:8088 in the address box of the browser and press Enter to switch to the DeviceManager login page.
  2. Choose Settings > Certificate > Certificate Management.
  3. Click Object storage service security certificate. On the displayed page, click CA Certificate to check whether the certificate is a root certificate.

    • If CN=RootCA is displayed, as shown in the following figure, the certificate is a root certificate. In this case, perform 8.

    • Otherwise, perform 4 to 8.

  4. Export the certificate request file.

    1. Choose Settings > Certificate > Certificate Management. On the displayed page, select Object storage service security certificate and click Export Request File.
    2. On the displayed page, specify Certificate Key Algorithm to encrypt the request file to be exported.
    3. Select Advanced in the upper right corner and set advanced parameters related to the certificate request file. Table 1 describes the related parameters.
      Table 1 Advanced request file parameters

      Parameter

      Description

      Country

      Value of the country parameter in the subject field when a CSR file is generated.

      State or Province

      Value of the state or province name parameter in the subject field when a CSR file is generated.

      City or Locality

      Value of the locality parameter in the subject field when a CSR file is generated.

      Organization

      Value of the organization parameter in the subject field when a CSR file is generated.

      Organization Unit

      Value of the organizational unit parameter in the subject field when a CSR file is generated.

      Common Name

      Value of the common name parameter in the subject field when a CSR file is generated.

      NOTE:

      If the exported certificate is Object storage service security certificate, the value of common name in the subject field is the global domain name of the object storage service.

      Subject Alternative Name

      Subject Alternative Name (SAN) is an extensible field (defined in the SSL standard X.509) during CSR file generation. The SSL certificate that uses the Subject Alternative Name field can extend the domain names supported by the certificate so that one certificate can support the resolution of multiple domain names. In the scenario where multiple clusters are deployed for the object service, you can configure global, region, and cluster domain names at the same time, or use wildcards (*).     4. Click OK.

  5. Use the built-in CA to issue digital certificates.

    1. Choose Settings > Certificate > Built-In CA.
    2. Select Service and click Issue Digital Certificate.
    3. On the Issue Digital Certificate page that is displayed on the right, set certificate parameters. Table 2 describes the related parameters.
      Table 2 Parameters for issuing a certificate

      Parameter

      Description

      Name

      Name of the digital certificate to be issued.

      NOTE:
      • The name consists of 4 to 63 characters.
      • The name can contain only letters, digits, underscores (_), and hyphens (-).

      Import Request File

      Select whether to import the request file as required.

      Request File

      If Yes is selected in Import Request File, you need to select and import a request file.

      Request Content

      If No is selected in Import Request File, you need to enter the request file content.

      Signature Hash Algorithm

      Select the signature algorithm to be used by the digital certificate. The options include SHA256 or SHA512.

      Certificate Validity Period (days)

      Set the validity period of the digital certificate. The value ranges from 30 to 25,550 days.
    4. Click OK.
    5. Click Export Issued Certificate to download the certificate.
    6. Click Close.

  6. Choose Settings > Certificate > Built-In CA. Select Service and click Export CA Certificate.
  7. Import and activate the certificate.

    1. Choose Settings > Certificate > Certificate Management, select Object storage service security certificate, and click Import and Activate.
    2. On the Import and Activate page that is displayed on the right, upload the certificate downloaded in 5 for Certificate File and the CA certificate exported in 6 for CA Certificate File.
    3. Click OK.

  8. Export the built-in CA certificate.

    1. Choose Settings > Certificate > Built-In CA.
    2. Select Service and click Export CA Certificate. Change the certificate file name extension to .pem.
