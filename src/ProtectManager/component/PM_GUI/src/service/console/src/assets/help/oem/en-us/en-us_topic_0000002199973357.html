Some of the Concurrent Jobs on the Same NDMP Server Failed

Symptom

When multiple file system backup jobs are concurrently executed on the same NDMP server, some of these jobs fail to be executed.

Figure 1 Backup jobs

Figure 2 Failed jobs

Possible Causes

The maximum number of concurrent backup jobs supported by the NDMP service varies depending on the storage vendor and the flow control of the storage vendor may be triggered.

Troubleshooting

  1. Adjust the number of concurrent backup jobs based on the flow control threshold of each storage vendor. OceanStor Dorado is used as an example. You can obtain the flow control threshold of a specific model from Specifications Query.
  2. Choose Protection > File Systems > Storage Devices.
  3. In the row of the target device, choose More > Modify.
  4. Change Number of Concurrent Tasks to a value that is less than or equal to the flow control threshold provided by the corresponding vendor.
  5. Click OK.
