Configuring a Database Backup and Restoration Account

Procedure

  1. Create a database backup and recovery account, as well as create an available mailbox. After the account is created, it automatically becomes the new account of the domain user.

    • Exchange 2010: Log in to the Exchange agent host, open the Exchange Management Console, expand Recipient Configuration, right-click Mailbox, choose New Mailbox from the shortcut menu, and create an Exchange mailbox as prompted. For example, the username can be OceanProtectUser.
    • Exchange 2013/2016/2019: Log in to Exchange admin center, click Mailboxes on the Recipients page, and create an Exchange mailbox as prompted. For example, the username can be OceanProtectUser.

  2. Add the newly created user as a member of the domain administrator.

    On the domain controller, open Active Directory Users and Computers, double-click the created account, click the Member Of tab and Add, add the user to Domain Admins, and click OK.

  3. Provide credentials for this user in the Exchange agent host properties, create a role group, and set the user as a member of the role group.

    1. Open Exchange Management Shell and run the following command to create a role group named OceanProtectRoles:
      New-RoleGroup -Name OceanProtectRoles -Roles @("Database Copies", "Databases", "Exchange Servers", "Monitoring", "Mail Recipient Creation", "Mail Recipients", "Recipient Policies")

      Parameters:

      • DatabaseCopies: enables administrators to manage database copies on each server.
      • Databases: enables administrators to create, manage, mount, and unmount mailboxes and public folder databases on each server.
      • Exchange Servers: enables administrators to manage Exchange Server configurations on each server.
      • Monitoring: enables administrators to monitor the availability of Exchange services and components in an organization. In addition to administrators, other roles associated with administrators, together with the service account that monitors applications, can collect information about the status of the Exchange Server.
      • Mail Recipient Creation: enables administrators to create mailboxes, mail users, mail contacts, communication groups, and dynamic communication groups in an organization.
      • Mail Recipients: enables administrators to manage existing mailboxes, mail users, mail contacts, communication groups, and dynamic communication groups in an organization.
      • Recipient Policies: enables administrators to manage recipient policies in an organization, for example, configuration policies.
    2. Add the OceanProtectUser user to the OceanProtectRoles role group.
      Add-RoleGroupMember -Identity OceanProtectRoles -Member OceanProtectUser
