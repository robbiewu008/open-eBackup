During Active Directory Object-Level Restoration, an Error Message Is Displayed, Indicating that the DistinguishedName of the Object Already Exists

If you create an object with the same name as an object that has been deleted but is not in the recycle bin, errors will be reported when you restore the deleted object because this operation violates the object uniqueness constraint. In this case, you need to choose whether to restore the deleted object or retain the created same-name object. If you choose the former, you need to delete the created same-name object by referring to this section and then perform object-level restoration again. If you choose the latter, object-level restoration is not required.

Symptom

During Active Directory object-level restoration, an error message is displayed, indicating that the DistinguishedName of the object already exists.

Possible Causes

The DistinguishedName of the object already exists.

Procedure

  1. Open Active Directory Users and Computers.

    1. Log in to the remote host as Administrator.
    2. Press Win+R.
    3. Enter dsa.msc and press Enter.

  2. Find the object to be deleted.
  3. Right-click the object and choose Delete from the shortcut menu.
  4. Click Yes.
