Step 8: (Optional) Creating a Reverse Replication/Cascaded Replication SLA

If you need to configure reverse replication (replicating replication copies to the source end) or cascaded replication (replicating replication copies from the target end to another cluster), configure the replication policy SLA by referring to this section. Otherwise, skip this step.

Procedure

  1. Log in to the management page of the cluster where the replication copies reside.

    You can view and perform operations on replication copies only on the target cluster.

  2. Choose Protection > Protection Policies > SLAs.
  3. Click Create.
  4. Enter an SLA name.
  5. Select an application type for the SLA to be created.

    1. Click the Applications icon.
    2. Choose Specific to application > Replicated Copy.
    3. Click OK.

  6. Click the Replication Policy icon.
  7. Configure the replication policy.

    Click Add on the right of the page to create a replication policy. A maximum of four replication policies can be created.

    Table 1 describes the related parameters.

    Table 1 Replication policy parameters

    Parameter

    Description

    Name

    Replication policy name, which can be customized.

    Replication Mode

    Only cross-domain replication, that is, replication between clusters, is supported.

    Replication Target Cluster

    Target cluster to which data is replicated.

    Select the target cluster, or click Add to add a new cluster.

    • If Replication Target Cluster is the source cluster of the replication copies, reverse replication is performed.
    • If Replication Target Cluster is another cluster, cascaded replication is performed.

    Replication Target User

    Select the system administrator or data protection administrator of the target end for replication.

    Password

    Enter the password of the system administrator or data protection administrator of the target end for replication. After entering the password, click Authenticate to verify the password.

    Specify Target Location
    The replication target location can be a backup storage unit or a backup storage unit group. If a backup storage unit group is selected, the system selects a backup storage unit from the backup storage unit group as the replication target location when a backup copy is replicated for the first time. In subsequent replication, the backup storage unit used for the first replication is preferentially selected.
    NOTE:
    • For version 1.5.0, this parameter is available only when Replication Mode is set to Intra-domain.
    • A backup storage unit that has been specified as the target location for a backup policy cannot be used as the target location for a replication policy.

    Frequency

    Replication is performed every xx minutes, hours, days, or weeks.

    Replication and Retention Rules

    Select the replication copy that you need to replicate and set the retention period of the copy.

    Once the retention period expires, the system automatically deletes the expired copies. The retention period must be longer than the replication interval.

    • Replicate all copies: Specify the time when the replication copy starts to be replicated and set the copy retention period.
    • Replicate copies on the specified date: Specify the date and set the copy retention period. Only full copies and forever incremental (synthetic full) copies on a specified date can be replicated. You can select multiple options at the same time:
      • Select the full replication copies generated in month x of each year for replication. The replicated copies are retained for x years (the retention period can be 1 to 70 years).
      • Select the first or last full replication copies of each month for replication. The replicated copies are retained for x months (the retention period can be 1 to 840 months).
      • Select the full replication copies generated on a specified day of each week for replication. The replication copies are retained for x weeks (the retention period can be 1 to 3650 weeks).
      NOTE:
      • For applications that do not support forever incremental (synthetic full) backup, only full replication copies on a specified date can be replicated.
      • If multiple options are selected, copies on any date specified by these options are replicated, and the copies are retained for the longest period specified in these options.

    Initial Execution Time

    Select a created rate limiting policy.

    If First Run At is later than the current time, data replication starts based on First Run At. If First Run At is earlier than the current time, data is replicated when the next replication period arrives.
    NOTE:

    Ensure that the device time of the agent host, local cluster, and target cluster is consistent.

    Rate Limit Policy

    Average maximum amount of data that can be transmitted per second in a replication job.

    Select a created rate limiting policy.

    Data Deduplication

    After this function is enabled, the system deduplicates data copies during remote replication to improve replication efficiency. You are advised to enable data deduplication.

    Data Compression

    After this function is enabled, the system compresses data copies during remote replication to improve replication efficiency. You are advised to enable data compression.

    Job Failure Alarm

    After this function is enabled, an alarm is automatically sent if a job fails and is automatically cleared after successful execution.

  8. Click OK.
  9. Associate the resource with a created SLA and perform reverse replication or cascaded replication on the replication copies.

    1. Choose Explore > Copy Data and select the corresponding application type.
    2. Locate the row that contains the resource for which reverse replication or cascaded replication will be performed, and choose More > Protect.
    3. Select a created SLA.
    4. Click OK.
