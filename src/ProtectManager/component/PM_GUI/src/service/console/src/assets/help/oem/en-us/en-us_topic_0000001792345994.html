Performing Manual Backup

If you want to perform a backup immediately, you can manually perform the backup.

Context

  • During manual backup, parameters (Automatic Retry) defined in the SLA are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected for a manual backup job is not defined in the SLA, the generated backup copy is retained permanently. If full backup is not defined in the SLA but selected during manual backup, the copies generated by the manual full backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Precautions

For version 1.5.0, there is a hole area where data files are not retained between the log backup and the next data backup.

Prerequisites

  • Backup operations can be performed only when the database is in the normal or degraded state. Only full backup or incremental backup can be performed, if the database is in the degraded state.
  • After the agent host is replaced or the project information is modified, you need to scan resources again.
  • For OLTP OPS, before a backup is initiated, ensure that the default backup root path has been configured for the instance on the OLTP OPS management page.
  • For OLTP OPS, before a backup is initiated, ensure that monitoring has been enabled for instances managed by OLTP OPS.
  • Before a backup is performed, you need to start backUpAgent on the agent host. For details about how to start the backUpAgent, see "Starting backUpAgent" in the ProtectAgent Installation Guide.
  • Before a backup is initiated, ensure that the XBSA backup whitelist has been enabled on TPOPS. For details about how to enable the whitelist, see Step 1: Enabling the XBSA Backup Whitelist on the TPOPS Node.

Procedure

  1. Choose Protection > Databases > GaussDB.

    For 1.5.0, choose Protection > Cloud Platforms > GaussDB.

  2. Click the Instance tab.
  3. In the row that contains the target instance, choose More > Manual Backup.

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

  4. Set the name of the copy generated through manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  5. Select a protection policy, which can be Full backup, Incremental backup, or Log Backup.
  6. Click OK.
