Step 5: (Optional) Enabling Backup Link Encryption

You need to enable backup link encryption if required.

Prerequisites

Ensure that the NFS Kerberos service has been configured for the storage system. Otherwise, backup jobs may fail after backup link encryption is enabled. For details about how to configure the NFS Kerberos service, see "(Optional) Configuring the NFS Kerberos Service" in the Installation Guide. When adding users to the AD domain, add two users whose usernames are rdadmin and the OS username used for database installation. You can contact the application administrator to obtain the OS username and ID. If the user already exists in the AD domain, you do not need to add it.

Precautions

After backup link encryption is enabled, the ticket between the agent host and the KDC domain server has a validity period. If the ticket expires, backup jobs may fail. You are advised to prolong the validity period of the ticket in the Kerberos policy on the KDC domain server or configure the ticket to never expire. If the permanent validity period is not configured, you must run the kinit or net command on the agent host to re-obtain tickets for users added to the AD domain before the tickets are about to expire. For details, see "Configuring the Client" in the Installation Guide.

Creating a Local UNIX Authentication User Group

Before enabling backup link encryption, if the current environment is not added to any domain environment, you need to add the user group to the local authentication user group.

  1. Log in to DeviceManager.

    1. Choose System > Infrastructure > Cluster Management.
    2. On the Backup Cluster tab page, click the node name in the Local Cluster Nodes area.
    3. In the Node Details dialog box that is displayed, click Open the device management platform to go to DeviceManager.

  2. Choose Services > File Service > Authentication Users.
  3. On the UNIX Users tab page, select Local Authentication User Groups.
  4. Click Create.
  5. Configure local UNIX authentication user group parameters.

    • Name: Group to which the user belongs, which is set during database installation. You can run the cat /etc/group command on the database host to view the group to which the database user belongs.
    • ID: Run the cat /etc/group command on the database host to view the group ID.

  6. Click OK.

Procedure

  1. Choose System > Security > Data Security.
  2. In the Encryption Settings area, click Modify on the right of the page and enable Backup Link Encryption.
  3. Click Save.
