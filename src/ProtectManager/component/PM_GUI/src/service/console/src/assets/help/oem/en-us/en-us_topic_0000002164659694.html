Step 2: Enabling Local Archiving for the Database

Before performing a backup job, you need to enable local archiving for the Dameng database. Otherwise, the backup will fail.

Precautions

  • Unless otherwise specified, the operations in this section use Dameng 8 as an example. The operations may vary according to the database version.
  • If the Dameng database is deployed in a cluster, you need to log in to all hosts where the cluster instances reside and perform the following operations unless otherwise specified.
  • If the value of configuration item ARCH_INI in configuration file dm.ini is 1 and configuration file dmarch.ini contains the content in 7, the archive mode has been enabled on the database host.

Procedure

  1. Log in to the Dameng database host.
  2. Switch to user root and run the following command to query the path to the dm.ini configuration file:

    ps -ef | grep dm.ini

  3. Go to the directory where the configuration file is stored and run the following command to open and edit the configuration file:

    vim dm.ini

  4. In the #configuration file section of the configuration file, change the value of ARCH_INI to 1.

    ARCH_INI = 1

  5. Run the following command to query the path to the dmarch.ini configuration file:

    find / -name dmarch.ini

  6. Go to the directory where the configuration file is stored and run the following command to open and edit the configuration file:

    vim dmarch.ini

  7. Change the following items in the configuration file:

    [ARCHIVE_LOCAL1]
ARCH_TYPE = LOCAL 
ARCH_DEST = Archive file path
ARCH_FILE_SIZE = 128 
ARCH_SPACE_LIMIT = 0

    Replace Archive file path with the actual path.
