After Data Is Recovered from the Oracle 12.1 Cluster Installed in ASM Mode to the Oracle 12.1 Single-Node System Installed in Non-ASM Mode, Data Backup of the Oracle 12.1 Single-Node System Is Suspended

Symptom

After Oracle backup is performed, the following error information is displayed in the Oracle trace logs:

Starting background process ASMB
Thu Jul 20 04:52:37 2023
ASMB started with pid=89, OS id=13500518 
WARNING: failed to start ASMB (connection failed) state=0x1 sid=''
WARNING: ASMB exiting with error

Possible Causes

According to Oracle official documents, this error is related to migrating database from ASM to file systems.

The production database is deployed in RAC mode and ASM disk groups are used. However, the target database for recovery is a non-ASM database and file systems are used. Therefore, this error is related to Oracle Bug 19503821, that is, when the database is migrated from ASM to the file system, the RMAN catalog runs extremely slow.

Troubleshooting

  1. Stop the Oracle database backup job.
  2. Install the 19503821 patch of the Oracle database.
