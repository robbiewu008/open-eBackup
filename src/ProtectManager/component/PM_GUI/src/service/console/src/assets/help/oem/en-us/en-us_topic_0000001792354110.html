Step 2: Creating and Configuring a Mount Directory

During backup and restoration, the product needs to mount the file system for storing backup data to the corresponding directory. This section describes how to create and configure the mount directory.

Creating a Mount Directory

  1. Log in to all nodes in the Elasticsearch cluster in sequence.
  2. Create the /mnt/databackup/ElasticSearchBackupSet directory on the node.

    The file system for storing backup data is mounted to this directory.

Configuring the Mount Directory

This section uses FusionInsight Manager as an example to describe related operations. The operations vary depending on the big data platform. For details, see the product documentation of the corresponding big data platform.

  1. Log in to FusionInsight Manager using a browser.
  2. Choose Cluster > Elasticsearch.
  3. Choose Configurations > All Configurations.
  4. Select Self-Definition for Elasticsearch service and Elasticsearch roles in sequence, and set the names and values to the self-defined configuration.

    Name: path.repo. Value: /srv/BigData/elasticsearch/nas,/mnt/databackup/ElasticSearchBackupSet.

    path.repo indicates the snapshot repository of the Elasticsearch cluster. /srv/BigData/elasticsearch/nas is the default snapshot repository, and /mnt/databackup/ElasticSearchBackupSet is the snapshot repository added in this configuration.

  5. After the configuration is complete, ensure that no service is running on the production end and restart the Elasticsearch service. The configuration takes effect after the restart is successful.
