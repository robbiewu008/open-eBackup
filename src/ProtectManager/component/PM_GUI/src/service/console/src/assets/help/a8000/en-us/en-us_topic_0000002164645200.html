Restoring Files in a NAS Share

This section describes how to restore a NAS share file to its original location, a new location, or a local location.

Context

  • The OceanProtect supports file-level restoration using backup copies, replication copies, and copies archived to object storage.
  • To query the compatibility of file systems that support indexes, you can log in to the OceanProtect Compatibility Query tool.

Precautions

  • File-level restoration is not supported if a folder or file name contains garbled characters. Do not select folders or files of this type. Otherwise, file-level restoration fails.
  • For the OceanProtect E6000 Appliance (with OceanStor Pacific used as backup storage), a replication copy cannot be used for file-level restoration.

Prerequisites

  • For 1.5.0, file-level restoration is supported only when the index status of a copy is Indexed. For 1.6.0 and later versions, copies whose index status is Not indexed also support file-level restoration.
  • To restore a NAS share to a new location, register the NAS share at the new location or the storage device where the NAS share resides with the OceanProtect. The following lists the supported storage device types.
    • OceanStor Dorado V700R001
    • OceanStor V700R001
    • OceanStor Dorado 6.x
    • OceanStor 6.x
    • OceanProtect 1.3.0 and later versions
    • OceanStor V5
    • OceanStor Pacific
    • NetApp ONTAP
  • If you want to restore a NAS share to a new location, you need to configure the access permission for the OceanProtect on the new storage device. For details, see Step 4: Configuring Access Permissions (Applicable to OceanStor V5/OceanStor Pacific/NetApp ONTAP). During the operation, configure a client or add a user for the target NAS share.

Procedure

  1. Choose Explore > Copy Data > File Systems > NAS Shares.

    For 1.5.0, choose Explore > Copy Data > File Services > NAS Shares.

  2. A copy can be searched by NAS share resource or copy. This section uses the NAS share resource as an example.

    On the Resources tab page, locate the NAS share to be recovered by NAS share name and click the name.

  3. On the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed under a month or date, copies exist in the month or on the date.

  4. Locate the copy for restoration and click More > File-level Restoration on the right of the row where the copy is located.

    • For 1.5.0: Select the files to be restored from the directory tree.
    • For 1.6.0 and later versions: Set File Obtaining Mode, which can be Select file paths from the directory tree or Enter file paths.
      • When entering a file path, enter a complete file path, for example, /opt/abc/efg.txt or C:\abc\efg.txt. If you enter a folder path, for example, /opt/abc or C:\abc, all files in the folder are restored. The file name in the path is case sensitive.

  5. Select the object to be restored and the restoration target location.

    The restoration target location can be Original location, New location, or Local Host Location.

    • Original location: Restores data to the original file system.
    • New location: Restores data to the file system of another device.
    • Local Host Location: Restores data to the OceanProtect.

    For details about the parameters of the original location, see Table 1. For details about the parameters of a new location, see Table 3 and Table 4. Table 1 describes the local host location parameters.

    Table 1 Parameters for restoring data to the local location

    Parameter

    Description

    Channels

    Number of subjobs that can be concurrently executed on the clients when restoration is executed. The default number of channels for the NAS share is 10, and the maximum number is 40.

    More channels indicate better backup performance and a higher NAS share access rate. However, more CPUs and memory of the client will be occupied. Each channel occupies about 200 MB memory.

    For 1.5.0, you are advised to reduce the number of channels due to limited container resources if the client is deployed on the OceanProtect.

    Clients

    Select clients for performing backup jobs.

    Nodes

    Local location of the cluster where the copy resides.

    File Systems

    Select the file system to be restored to the local device. When restoring data to the local location for the first time, you can click Create to create a file system. For details about how to create a file system, see Creating a File System.

    Overwrite Rule

    If files with the same names exist in the restoration path, you can choose to replace or skip existing files.

    • Replace existing files
    • Skip existing files
    • Only replace the files older than the restoration file

  6. Click OK. Before you restore data using the copy, ensure that the target location for restoration has sufficient free space for restoration. The confirmation method is as follows:

    • Requirements for restoration using a full copy: Remaining space of the target location ≥ Data volume of the full copy before reduction. To obtain the data volume of the full copy before reduction, view Data Volume Before Reduction of the backup job on the Job Information tab page.
    • Requirements for restoration using forever incremental copies or incremental copies: Remaining space of the target location ≥ Total amount of data of all copies generated in the following period before reduction: from the current permanent incremental copy or incremental copy to the last full copy. To obtain the total amount of data before reduction, add the value of Data Volume Before Reduction of each backup job on the Job Information tab page.
