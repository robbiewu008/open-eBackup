Restoration Using Multiple Clients Based on Log Copies Failed (Applicable to OLTP OPS)

Symptom

The job of restoration using multiple clients based on GaussDB log copies failed.

Possible Causes

In GaussDB 2.23.01.275 and earlier, the logic for processing the port numbers of multiple clients transferred during restoration using log copies is incorrect due to GaussDB defects. As a result, port numbers cannot be correctly parsed.

Troubleshooting

Use a single host where the client is installed to perform restoration using log copies.
