Adding an External Cluster

Prerequisites

You have imported the certificate of the external cluster. Specifically, you have downloaded the CA certificate of the Server Certificate type from the external cluster and imported the certificate to the local cluster, with Type set to Backup/Multi-domain/Replication Cluster (the OceanProtect E6000 Appliance does not support multiple-domain clusters). For details, see External Certificates in Downloading a Certificate and Importing a Certificate in the OceanProtect DataBackup 1.5.0-1.6.0 Security Configuration Guide.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  2. Click the Replication Clusters tab. On the replication cluster page that is displayed, click Add.
  3. Set external cluster parameters.

    Table 1 describes the related parameters.
    Table 1 External cluster parameters

    Parameter

    Description

    Name

    User-defined name of the external cluster.

    IP Address

    Management IP address of the external cluster. You can also enter the backup IP address.

    Port
    Management port of the external cluster.
    • Use 25081 when a management IP address is entered.
    • Use 30068 when a replication IP address (recommended) or a backup IP address is entered.

    Role

    Add a replication target cluster. After such a cluster is added, a replication relationship is established between the local and target clusters. Data can be transmitted between the local cluster and the replication target cluster.

    Username

    Enter the username and password of the remote device administrator For details about how to create a remote device administrator on the replication target cluster, see Creating a User.

    Password

  4. Click OK.
