Managing Logs

By viewing debug logs, you can understand the error information and prompt information that are triggered when you operate or design a template.

Procedure

  1. Choose System > Log > Log Management.
  2. In the Set Log area, select the node to be modified, click the drop-down list to the right of Log Level, and select the desired log level. You can export system debug logs at the selected or higher levels.

    You can set log levels in batches. After selecting the levels of logs to be exported, click OK to complete the setting.

  3. Set parameters in the Info Export area.

    Table 1 describes the related parameters.

    Table 1 Parameters in the Info Export area

    Parameter

    Description

    Export Content

    NOTE:
    • The Export Content parameter is available in 1.6.0 and later versions.
    • The Export Content parameter is unavailable for this product.

    Available options are Log File and Config File. You can select multiple options.

    NOTE:
    • The exported logs include system debug logs and system running parameters.
    • For this product, the exported logs include only system debug logs.

    Node

    Select the node whose information you want to export.

    Controller

    NOTE:

    The Controller parameter is unavailable for this product.

    Select a controller.

    Software Module Name

    Available options are Infrastructure, Data protection engine, and Management service. You can select multiple options.

    Export Range

    NOTE:

    The Export Range parameter is available in 1.6.0 and later versions.

    Available options are Based on custom time range and All. The custom time ranges from 1 to 10,000 days.

    Log File Name

    Enter a log file name.

    Config File Name

    NOTE:

    The Config File Name parameter is unavailable for this product.

    Enter a configuration file name.

  4. Click Export to export debug logs.
