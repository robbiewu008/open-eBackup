Restoring Cloud Disks

This section describes how to restore a cloud server or cloud disk to the original or a new location after the cloud server has been backed up, when the cloud server disks are damaged.

Context

The OceanProtect allows you to restore cloud disks to the original or a new location using backup copies, reverse replication copies, archive copies (data cannot be restored to the original location using archive copies of replication copies), or replication copies (data cannot be restored to the original cloud server using replication copies). Cascaded replication copies and their archive copies cannot be used for restoration to the original cloud server.

  • Restoration to the original location

    Restores data to cloud server disks on the current cloud platform, including existing disks and new disks.

  • Restoration to a new location

    Restores data to existing disks of another cloud server.

Precautions

A maximum of 24 disks can be mounted to a host where the client is installed. Therefore, if 24 disks have been mounted to a host where the client is installed during restoration, the system will execute the restoration job when there are idle slots.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > OpenStack.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the cloud server to be restored by cloud server name and click the name.

  3. Click the Copy Data tab and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. Find the copy to be restored and choose More > Disk Restoration on the right.

    For the OceanProtect E6000 Appliance, when a replication copy is used for restoration, other copies of the resource cannot be replicated at the remote end.

  5. Select the cloud disk to be restored and configure the restoration.
  6. Restore data to the original location or a new location.

    • Restoration to the original location

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Target Location

      Cloud server to which the backup copy is restored.

      Name of Disk to Be Restored

      Name of the disk to be restored in the copy.

      Capacity

      Capacity of the disk to be restored in the copy.

      Type

      Select the type of the disk to be restored. You can select Existing or New.

      • Existing: The existing disks of the target VM for restoration are overwritten.
      • New: The backup software creates disks and mounts them to the target VM for restoration.

      Target Disk Name
      Select the disk name of the target cloud server.
      NOTE:

      For a new disk, enter the target disk name and select a disk type.

      Advanced

      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      Set the host where the client is installed, automatic power-on after restoration, and copy verification before restoration.

      Clients

      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      Select the host where the client is installed for copy-based restoration.

      If no host is selected, the system selects one or more hosts from the hosts where the client is installed configured during resource registration to perform restoration.

      Power on Automatically After Restoration

      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If this option is enabled, the cloud server automatically starts after the restoration is complete. This option is disabled by default.

      Copy Verification Before Restoration

      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If this option is enabled, the integrity of a copy is verified before the copy is used for restoration. This operation affects the restoration performance. This option is disabled by default. This function cannot be enabled when the copy verification status is Invalid or --.

  7. Click OK.
  8. Select a target cloud disk for each cloud disk to be restored.

    • Data on data disks cannot be restored to the system disk, but data on the system disk can be restored to data disks.
    • The restoration can be successful only when the target cloud disk is in the mounting state and its capacity is greater than or equal to that of the cloud disk to be restored.

  9. Click OK.
