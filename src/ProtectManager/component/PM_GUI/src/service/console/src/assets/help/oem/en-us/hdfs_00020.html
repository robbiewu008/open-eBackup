Step 8: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • During manual backup, the parameters defined in the SLA (for example, Automatic Indexing) are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy is retained permanently. If incremental backup is not defined in the SLA but is selected during manual backup, the copies generated by the manual incremental backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Big Data > HDFS.
  2. On the Filesets tab page, select the fileset that you want to protect, and click Protect.

    You can choose multiple filesets for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

    Table 1 Fileset protection advanced parameters

    Parameter

    Description
    Auto Indexing of Backup Copies
    NOTE:

    Only 1.6.0 and later versions support this parameter.

    After this function is enabled, you can select one or more files in a copy for restoration and search for files, directories, or links in copies on the management page.

    If you do not enable this function, you can create indexes for a specific copy on its page.

    This function cannot be disabled after being enabled.
    Automatic Indexing of Object Storage Archive Copies
    NOTE:

    Only 1.6.0 and later versions support this parameter.

    After this function is enabled, you can select one or more files in a copy for restoration and search for files, directories, or links in copies on the management page. If a tape library serves as the archive storage, archive copy indexing is not supported.

    This function cannot be disabled after being enabled.

  4. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  5. Perform manual backup.

    If you want to perform a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row where the fileset you want to back up is located, choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

    2. Set the name of the copy generated through manual backup.

      If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup or Forever Incremental (Synthetic Full) Backup.
    4. Click OK.
