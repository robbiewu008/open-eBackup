Adding an External Cluster

The external cluster can be added to the local cluster to establish the relationship between these clusters. You can also add a managed cluster for unified management of multiple clusters.

Prerequisites

  • An external cluster is ready, and the management network between the local and external clusters is normal.
  • The external cluster certificate has been imported. Download the CA certificate of the Server Certificate type in the external cluster, import the certificate to the local cluster, and set the certificate type to Backup/Multi-domain/Replication Cluster. For details, see External Certificates in Downloading a Certificate and Importing a Certificate in the OceanProtect DataBackup 1.5.0-1.6.0 Security Configuration Guide.

Context

The OceanProtect supports 1:1 replication, 1:N replication, and N:1 replication.

  • For 1:N replication, the maximum value of N is 4.
  • For N:1 replication, the maximum value of N is 32.

Precautions

If another management cluster has been added to the local cluster, ensure that the CA certificate of the Server Certificate type of the external cluster has been imported to the management cluster when adding the external cluster to the local cluster.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  2. In the Multi-domain Clusters area, click Add.
  3. Set multi-domain cluster parameters.

    Table 1 describes the related parameters.
    Table 1 External cluster parameters

    Parameter

    Description

    Name

    User-defined name of the external cluster.

    IP Address

    Management IP address of the external cluster.

    Port

    Management port of the external cluster. The default port number is 25081.

    Role

    Add a managed cluster. After a managed cluster is added, you can view data about the managed cluster in the local cluster and switch from the local cluster to the managed cluster.

    Username

    Enter the username and password of the system administrator.

    Password

  4. Click OK.
