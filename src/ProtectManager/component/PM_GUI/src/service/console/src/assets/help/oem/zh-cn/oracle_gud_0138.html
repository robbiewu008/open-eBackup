备份任务长时间没有进度

现象描述

执行RMAN备份操作时，备份任务长时间没有进度，且安装客户端的主机操作系统及内核版本为Redhat7.6 3.10.0-957.el7.x86_64。

可能原因

Redhat7.6 3.10.0-957.el7.x86_64版本内核问题导致本产品进程重启，进而导致备份存储上文件系统不可写，备份任务卡住。

处理步骤

  1. 使用PuTTY，登录已安装客户端的主机。
  2. 执行以下命令查看主机操作系统及内核版本是否为Redhat7.6 3.10.0-957.el7.x86_64。

    cat /etc/redhat-release

    uname -r

    • 是，执行3。
    • 否，联系技术支持协助处理。

  3. 用户自行下载Redhat内核包。
  4. 将下载到的内核包上传至数据库主机。
  5. 执行以下命令安装内核包。

    rpm -ivh 内核包

    以kernel-3.10.0-957.10.1.el7.x86_64.rpm为例，如： 
    rpm -ivh kernel-3.10.0-957.10.1.el7.x86_64.rpm

  6. 重启操作系统后，再次执行以下命令查看内核版本是否为新安装的版本。

    uname -r
