Managing Filesets

The product allows you to perform operations on databases, such as modifying, removing, activating, and disabling protection.

You need to log in to the management page, choose Protection > File Systems > Filesets, find the fileset to be operated, and click More to select an operation to be performed.

You can also select and manage filesets in batches for protection.

Table 1 describes the detailed operations.

Table 1 Operations related to fileset protection

Operation

Description

Navigation Path

Protect Fileset

Scenario

This operation enables you to protect a fileset.

Choose More > Protection in the row where the fileset is located.

Modify Protection

Scenario

You can modify a protection plan to associate another SLA with a fileset or modify a user-defined script. After the modification, the next backup will be executed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

Choose More > Modify Protection in the row where the fileset is located.

Remove Protection

Scenario

You can remove the protection plan if the fileset no longer needs to be protected. After the removal, the system cannot execute protection jobs for the fileset. To protect the fileset again, you need to associate an SLA with the fileset.

Note

  • When the protection job of the fileset is running, the protection cannot be removed.
  • After you remove protection and associate the resource with another SLA, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

Choose More > Remove Protection in the row where the fileset is located.

Activate Protection

Scenario

This operation enables you to re-activate a disabled protection plan of a fileset. After the protection plan is activated, the system backs up the fileset based on the protection plan.

Note

This operation can be performed only for a fileset that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection in the row where the fileset is located.

Disable Protection

Scenario

This operation allows you to disable the protection plan when periodic protection on a fileset is not needed. After the protection plan is disabled, the system does not execute the automatic backup job of the fileset.

Note

Disabling protection does not affect ongoing backup jobs.

In the row of the target fileset, choose More > Disable Protection.

Restoring a fileset

Scenario

This operation enables you to restore data using a fileset copy.

Choose More > Restore in the row where the fileset is located.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup in the row where the fileset is located.

Modify Fileset

Scenario

This operation enables you to modify the information about a fileset, such as the name of the fileset, the files or directories contained in the fileset, and the filtering rules of the files or directories.

Choose More > Modify in the row where the fileset is located.

Delete Fileset

Scenario

You can delete a fileset when you do not need to protect it.

Note

You are not allowed to delete a fileset that has been associated with an SLA.

Choose More > Delete in the row where the fileset is located.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
