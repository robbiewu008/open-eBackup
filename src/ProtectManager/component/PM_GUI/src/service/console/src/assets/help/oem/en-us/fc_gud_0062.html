Managing FusionOne Compute Registration Information

After the FusionOne Compute virtualization environment is registered with the product, you can perform operations on the FusionOne Compute virtualization environment on the management page.

Log in to the management page and choose Protection > Virtualization > FusionOne Compute to go to the FusionOne Compute page. On the left of the FusionOne Compute page, select the desired virtualization environment.

Table 1 describes the related operations.

Table 1 Operations related to the FusionOne Compute virtualization environment

Operation

Description

Navigation Path

Viewing FusionOne Compute environment information

Scenario

This operation enables you to view FusionOne Compute environment information, such as the name, port number, IP address, and username.

Choose More > Environment Info.

Resource Scan

Scenario

The product automatically scans the FusionOne Compute virtualization environment every hour. If VMs are added, deleted, or modified in the registered virtualization environment, the VM information is automatically updated to the product after the scan. This operation allows you to immediately update the changed VM information to the product.

Note

If you do not select to apply the SLA policy to the newly created VM when creating a periodic backup job, the system will not associate the newly detected VM with an SLA after scanning.

Choose More > Resource Scan.

Modify Registration

Scenario

This operation enables you to change the name of FusionOne Compute registration information. If the IP address, port number, or CA certificate of the registered FusionOne Compute virtualization environment is changed, you need to perform this operation to synchronize the change.

Note

Modify the FusionOne Compute registration information when no backup or restoration job is running. Otherwise, the job will fail.

Choose More > Modify Registration.

Unregister

Scenario

If you do not need to protect the FusionOne Compute virtualization environment or use it as a restoration target, you can perform this operation to remove the FusionOne Compute virtualization environment.

Note

If a backup job is being executed, the unregistration will fail. Wait until the backup job is complete and try again.

Choose More > Unregister.
