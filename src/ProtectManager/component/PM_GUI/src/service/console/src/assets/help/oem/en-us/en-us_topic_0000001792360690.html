Restoring a NAS Share

This section describes how to restore a NAS share that has been backed up to the original location or a new location.

Context

The product supports restoration to the original location or a new location using backup copies or replication copies (data cannot be restored to the original location).

Prerequisites

  • To restore a NAS share to a new location, register the NAS share at the new location or the storage device where the NAS share resides with the product. The following lists the supported storage device types.
    • OceanStor Dorado 6.x
    • OceanStor 6.x
    • OceanStor V5
    • OceanStor Pacific
    • NetApp ONTAP
    • OceanProtect 1.3.0 and later versions
  • If you want to restore a NAS share to a new location, you need to configure the access permission for the product on the new storage device. For details, see Step 5: Configuring Access Permissions. During the operation, configure a client or add a user for the target NAS share.

Procedure

  1. Choose Explore > Copy Data > File Systems > NAS Shares.

    For 1.5.0, choose Explore > Copy Data > File Services > NAS Shares.

  2. A copy can be searched by NAS share resource or copy. This section uses the NAS share resource as an example.

    On the Resources tab page, locate the NAS share to be recovered by NAS share name and click the name.

  3. On the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or day.

  4. In the row where the copy resides, choose More > Restore.
  5. Restore a NAS share to its original location or a new location.

    • Restore data to the original location.

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Location

      Location of a NAS share, which cannot be modified.

      Overwrite Rule

      If a file with the same name exists in the restoration path, you can select replacing or skipping the existing file.

      • Replace existing files
      • Skip existing files
      • Only replace the files older than the restoration file
    • Restore data to a new location.

      Table 2 describes the parameters for restoring data to OceanStor V5, OceanStor Pacific, or NetApp ONTAP.

      Table 2 Parameters for restoring data to OceanStor V5, OceanStor Pacific, or NetApp ONTAP

      Parameter

      Description

      Storage Device

      Select the storage device that resides at a new location.

      Share Name

      Select the name of the share that accesses a file system. If no share is available, click Register to add a NAS share. For details about NAS share registration parameters, see Table 1.

      NOTE:
      • For an NFS share, ensure that the current device has the read and write permissions on the NFS share directory and the no_root_squash option is enabled. For details about how to configure NFS share access permissions, see Step 5: Configuring Access Permissions.
      • For a CIFS share, ensure that the current device has the read and write permissions on the CIFS share directory. You are advised to use a username in the Administrator group for backup. Otherwise, only files and directories that have granted the read and write permissions to the device can be backed up. For details about how to configure CIFS share access permissions, see Step 5: Configuring Access Permissions.

      Overwrite Rule

      If a file with the same name exists in the restoration path, you can select replacing or skipping the existing file.

      • Replace existing files
      • Skip existing files
      • Only replace the files older than the restoration file

      Table 3 describes the parameters for restoring data to OceanStor Dorado 6.x, OceanStor 6.x, or OceanProtect 1.3.0 or later.

      Table 3 Parameters for restoring data to OceanStor Dorado 6.x, OceanStor 6.x, or OceanProtect 1.3.0 or later

      Parameter

      Description

      Storage Device

      Select the storage device that resides at a new location.

      File Systems

      Select the file system to be restored.

      Share Name

      Select the share name of the file system to be restored.

      NOTE:
      • For an NFS share, ensure that the current device has the read and write permissions on the NFS share directory and the no_root_squash option is enabled. For details about how to configure NFS share access permissions, see Step 5: Configuring Access Permissions.
      • For a CIFS share, ensure that the current device has the read and write permissions on the CIFS share directory. You are advised to use a username in the Administrator group for backup. Otherwise, only files and directories that have granted the read and write permissions to the device can be backed up. For details about how to configure CIFS share access permissions, see Step 5: Configuring Access Permissions.

      FQDN/IP

      FQDN name or service IP address for accessing a NAS share.

      The FQDN format is Name of the storage device where a NAS share resides.Name of the domain to which the storage device is added. For example, if the name of the storage device where a NAS share resides is Huawei.Storage and the name of the domain to which the storage device is added is huawei.com, the name of the FQDN is Huawei.Storage.huawei.com.

      NOTE:

      If the FQDN name is required, configure the DNS service on the device management page of the product so that the product can access external domain names. For details about how to configure the DNS service, see Configuring the DNS Service.

      Authentication Method

      Authentication mode of a file system. This parameter is available when the share protocol of a backup copy is CIFS. For details about this parameter, see Table 1.

      Overwrite Rule

      If a file with the same name exists in the restoration path, you can select replacing or skipping the existing file.

      • Replace existing files
      • Skip existing files
      • Only replace the files older than the restoration file

  6. Click OK.
