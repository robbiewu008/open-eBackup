Managing GaussDB Projects

After GaussDB is registered with the product, you can perform operations on GaussDB.

Choose Protection > Databases > GaussDB. On the Project tab page, locate the target project.

For 1.5.0, choose Protection > Cloud Platforms > GaussDB.

Table 1 describes the related operations.

Table 1 Operations related to GaussDB projects

Operation

Description

Navigation Path

Authorizing a GaussDB project

NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only a system administrator has the permission to protect resources. However, a system administrator can grant this permission to a data protection administrator.

Note

The permission on a single resource can be granted to only one data protection administrator.

Choose More > Authorize Resource and select a data protection administrator to be authorized.

Reclaiming a GaussDB project

NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to a data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Choose More > Reclaim Resource.

Scanning GaussDB

Scenario

The product automatically scans the GaussDB environment every 5 minutes. If the projects in the registered GaussDB environment change, the change will be automatically updated to the product after scanning. This operation enables you to immediately update the changed information to the product.

Choose More > Resource Scan.

Testing GaussDB connectivity

Scenario

Manual connectivity test of the registered GaussDB environment is used to check the connectivity between the GaussDB environment and the agent host and between the GaussDB environment and the production environment.

Choose More > Test Connectivity.

Allowing restoration

Scenario

If you want to select a project or an instance in the project as the target project or instance for restoration, you can use the Allow Restoration option.

In the row of the target project, choose More > Allow Restoration.

Disallowing restoration

Scenario

If you do not allow the selected project or instance in the project to be used as the target project or instance for restoration, you can use the Disallow Restoration option.

In the row of the target project, choose More > Disallow Restoration.

Modifying a GaussDB project

Scenario

Perform this operation when you need to modify a GaussDB project.

Choose More > Modify.

Deleting a GaussDB project

Scenario

Perform this operation when instances in a GaussDB project do not need to be protected.

Note

If instances in the GaussDB project are protected, remove the protection before performing this operation.

Choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for registering a GaussDB project is similar to that in Step 3: Registering the GaussDB Project and is not described here.
