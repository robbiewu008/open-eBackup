Global Search for Copies

This section describes how to perform a global search for copies.

Context

The global search for copies supports VMware, FusionCompute, Huawei Cloud Stack, HDFS, NAS file system, NAS share, fileset, and CNware. If only one of the applications is involved, you can ignore the description of the other one.

Prerequisites

  • When you need to browse or search for copies, automatic indexing must be enabled for the SLA that generates copies or indexes must be manually created when copies are generated.
  • CNware, FusionCompute, and Huawei Cloud Stack copies can be searched if they are in the ext2/ext3/ext4 or XFS file systems in Linux OSs, and if they are in the NTFS, FAT, or FAT32 file systems in Windows OSs.

Procedure

  1. Click in the upper right corner to go to the Global Search page.

    1. Select Copy Data in the search box.
    2. Enter the name or path of the file object (file, directory, or link) in the copy and click . Fuzzy search is supported.

      File objects that meet the search criteria are displayed.

      When multiple search criteria are entered at the same time, the search results that meet at least one of the search criteria are displayed. The most closely-matched search result is displayed at the top.

  2. In the navigation tree on the left, filter the displayed results again as required.

    The logic relationship between parameters is OR. That is, the search results filtered based on either one or more selected conditions will be displayed.

    Table 1 Parameter description

    Parameter

    Description

    Cluster

    Select the cluster whose copies are to be searched for.

    Resource Type

    Select the resource type to be searched for.

    File Object Type

    Select All, File, Directory, or Link for filtering.

    Resource Name

    Enter a resource name.

    Absolute path to file object

    Enter the absolute path to the file object.

  3. After the search is complete, the resource search result is displayed on the right of the page. Table 2 describes the related parameters.

    During the backup of FusionCompute, VMware, CNware, or Huawei Cloud Stack VMs, the mount point information that is not written into the /etc/fstab file is stored in the memory. The OceanProtect does not back up the information. Therefore, no index is generated for the files at the mount point, and file-level restoration is not supported for these files. To generate indexes for new file systems and support file-level restoration, write the mount point information of the new file systems to the /etc/fstab file.

    Table 2 Parameters related to copy data

    Parameter

    Description

    File Object

    Name

    Name of a file, directory, or link in a copy.

    Absolute Path

    Absolute path of a file, directory, or link.

    Type

    File object type.

    • : Directory
    • : File
    • : Link

    Modified At

    Last modification time of a file, directory, or link in a copy when the copy is generated.

    Resources

    Name

    Name of the source resource corresponding to the copy data.

    Type

    Resource type.

    Index Copies

    -

    Number of index copies of files, directories, or links in copy data.

    • If automatic indexing has been enabled for the SLA that is used to generate a copy, the system checks whether there are updates for the files or directories in the backup copy during each automatic backup job. If there are updates, a new index is created for the files or directories. The number of index copies is 1. If there are no updates, the number of index copies is increased by 1.
    • If the index is manually created for CNware on the management page, the system creates another index for the files or directories in the backup copy. The number of index copies is 1.

    Click Index Copies to view copy data of the resource. The copies displayed on the page are the ones for which indexes have been created for the object.

  4. In the row where the search result is located, download the file or perform file-level restoration.

    • HDFS does not support copy file download.
    • For NAS shares and filesets, copy files in aggregation format cannot be downloaded.
