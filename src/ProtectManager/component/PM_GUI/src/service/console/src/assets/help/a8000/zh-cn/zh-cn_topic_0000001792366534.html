恢复SQL Server数据库

当需要对已备份的数据库进行恢复时，可以参考本节使用副本恢复数据库至原机或异机。

前提条件

  • 在进行异机恢复时，请确保目标主机上存在数据库实例。
  • 恢复前请确保恢复目标位置的数据目录剩余空间大于恢复副本缩减前大小，否则恢复将失败。

操作步骤

  1. 选择“数据利用 > 副本数据 > 数据库 > SQL Server”。
  2. 您可以以数据库资源维度或以副本维度查找副本，本节以资源维度为例进行说明。

    在“资源”页签，根据资源名称查找到需要恢复的资源，并单击资源名称。

  3. 单击“副本数据”，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 单击时间轴上，弹出相应副本，在副本所在行，单击“更多 > 恢复”。

    相关参数说明如表1所示。

    表1 恢复SQL Server数据库参数说明

    参数

    说明

    恢复至

    选择恢复至“原位置”或“新位置”。

    位置

    当选择恢复至“原位置”时，默认显示原位置。

    目标主机/集群

    当选择恢复至“新位置”时，选择原主机或异主机恢复。

    目标实例

    当选择恢复至“新位置”时，选择目标主机上的实例。

    数据库重命名

    开启后，可对恢复后的数据库重命名。

    说明：

    当数据库恢复至原位置时，需选择数据库所属实例进行资源扫描。

    恢复前执行脚本

    您可以根据实际需求，在恢复任务执行前、执行成功后、执行失败后，执行自定义脚本。

    此处请输入脚本的绝对路径和名称，如输入：C:\script.bat。

    说明：

    当配置了“恢复成功执行脚本”时，即使该脚本执行失败，OceanProtect上也会显示恢复任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。
