Adding Storage Devices

After a storage device is added, the OceanCyber 300 Data Security Appliance can secure the storage device against ransomware.

Prerequisites

For details about the models and versions of the storage devices to be added, use OceanCyber Compatibility Query.

  • The storage devices to be discovered are running properly.
  • The network connection of the storage devices to be discovered is normal.
  • If the detection type of the storage device to be added is OceanCyber Detection, ensure that the remote device connected to the IP link of the storage device to be added is normal. This ensures that the alarm suppression instruction is successfully sent to the discovered storage device and its peer storage device (no alarm is sent when the replication port is disabled).
  • If the detection type of the added storage device is OceanCyber Detection, ensure that the NFSv3 service has been enabled on the storage device.

Procedure

  1. On the top menu bar, choose Data Security > Storage Device.
  2. On the Storage Device page, click Add Device
  3. Set Detection Type and Device Type.

    The detection type can be OceanCyber Detection or In Device Detection.
    • OceanCyber Detection: The OceanCyber Appliance provides detection capabilities for third-party devices, including storage devices of OceanProtect 1.3.0 and later and OceanStor Pacific 8.1.5 and later, as well as NAS file system storage devices of OceanStor Dorado 6.1.6 and later and OceanStor 6.1.6 and later versions.
    • Device Detection: The device automatically performs detection and the OceanCyber Appliance monitors the detection. Detection is supported for storage devices of OceanProtect 1.3.0 and later, and NAS file system storage devices of OceanStor Dorado 6.1.5 and later and OceanStor 6.1.5 and later versions.

  4. Set storage device parameters. Table 1 describes the parameters.

    Table 1 Parameters for adding storage devices

    Parameter

    Description

    Value

    Device Name

    Name of the storage device.

    [Example]

    Dorado1

    IP Address

    Management IPv4 or IPv6 address of the storage device.

    [Example]

    192.168.10.11

    Port

    Communication port of the storage device.
    • When OceanCyber Detection is selected, the default value is 8088.
    • When In Device Detection is selected, the default value is 25081.

    Username

    Username for logging in to the storage device.

    Obtain the value from the storage device administrator.

    NOTE:

    Log in to the system as a system administrator and ensure that the RESTful login mode is available for the user and the password has been initialized. Otherwise, the generated snapshot data may be displayed abnormally.

    Password

    Password for logging in to the storage system.

    Obtain the value from the storage device administrator.

    NOTE:

    After the storage device administrator changes the password, notify the OceanCyber Appliance administrator in a timely manner.

  5. Verify a certificate. Select the CA certificate file and CRL of the storage device. If Certificate verification is disabled, security risks may exist in the communication between the system and the storage device. You are advised to enable Certificate verification.
  6. Click OK.

    The system supports up to eight storage devices.
