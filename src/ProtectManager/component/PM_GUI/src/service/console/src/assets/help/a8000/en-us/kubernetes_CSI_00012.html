Step 4: Registering a Dataset

A dataset is a collection of resources with specified labels in a Kubernetes namespace. You can register a dataset to protect data of resources added with the same labels.

Precautions

During the dataset registration, ensure that the entered label contains all information about the application. Otherwise, the restoration may fail.

Procedure

  1. Choose Protection > Containers > Kubernetes CSI.
  2. Click the Dataset tab and click Register to register the Kubernetes CSI dataset information.

    Table 1 describes the Kubernetes CSI dataset registration information.

    Table 1 Dataset registration information

    Parameter

    Description

    Name

    Customize the dataset name.

    Namespace

    Name of the namespace where the dataset to be registered is located.

    NOTE:

    Before a namespace is selected, ensure that the namespace already exists in the cluster registered with the OceanProtect.

    Include Selectors

    You can add multiple labels to specify resources that contain the labels in the namespace. You can query resource labels on the Kubernetes management platform. Ensure that all resources in the dataset to be registered contain the added labels.

    NOTE:

    Workloads depend on the resources to which labels are added. Such resources include Secret, DaemonSet, Deployment, StatefulSet, ReplicaSet, and more.

    The following is an input example of the Key and Value of a label:

    Label:app.kubernetes.io/name=mysql
Key:app.kubernetes.io/name
Value:mysql

    Exclude Selectors

    You can add multiple labels to exclude resources that contain the labels in the namespace. You can query resource labels on the Kubernetes management platform. Ensure that no resource in the dataset to be registered contains the added labels.

    The following is an input example of the Key and Value of a label:

    Label:app.kubernetes.io/name=mysql
Key:app.kubernetes.io/name
Value:mysql

    A maximum of 16 datasets can be created in a namespace.

  3. Click OK.
