Managing Database Protection

The product allows you to modify, remove, activate, or disable database protection.

Related Operations

You need to log in to the management page, choose Protection > Databases > Oracle, click the Databases tab, and find the database on which you want to perform related operations.

Table 1 describes the related operations.

Table 1 Database protection operations

Operation

Description

Navigation Path

Modifying protection

Scenario

When you need to reselect the SLA associated with a database and modify the execution scripts before and after database protection, you can modify the protection plan. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

Choose More > Modify Protection in the row where the database is located.

Removing protection

Scenario

If you do not need to protect a database, you can remove the protection from the database. After the removal, the system cannot execute protection jobs for the database. To protect the database again, you need to associate an SLA with the database again.

Note

When a protection job of the database is running, the protection cannot be removed.

Choose More > Remove Protection in the row where the database is located.

Manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup in the row where the database is located.

Disabling protection

Scenario

This operation enables you to disable a protection plan when periodic protection on the database is not required. After the protection plan is disabled, the system does not execute the automatic backup job of the database.

Note

Disabling protection does not affect ongoing backup jobs.

Choose More > Disable Protection in the row where the database is located.

Activating protection

Scenario

This operation enables you to re-activate the disabled protection plan of a database. After the protection plan is activated, the system backs up the database based on the protection plan.

Choose More > Activate Protection in the row where the database is located.

Scanning Resources

Scenario

If the information about a registered database changes, for example, the database version changes, you can perform this operation to scan the database and update the information. Otherwise, the execution of backup and recovery jobs may be affected.

Choose More > Resource Scan in the row where the database resides.

Testing connectivity

Scenario

This operation enables you to test the connectivity between the product and the database.

Choose More > Test Connectivity in the row where the database resides.

Modifying database information

Scenario

This operation enables you to modify information such as the database name and database authentication mode.

Choose More > Modify in the row where the database resides.

Deleting a database

Scenario

This operation enables you to delete a database that is no longer used.

Note

Before deleting a database, remove protection from the database.

Choose More > Delete in the row where the database resides.

Note: The procedure for restoring Oracle data is the same as that described in Restoration.
