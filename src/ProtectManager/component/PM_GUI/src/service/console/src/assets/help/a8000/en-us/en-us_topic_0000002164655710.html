Managing DB2 Databases, Tablespace Sets, or Instances

The OceanProtect allows you to modify, remove, activate, or disable protection for DB2 databases or tablespace sets.

Related Operations

Log in to the management page, choose Protection > Databases > DB2, click the Databases, Tablespace Set, or Instance tab, and locate the database, tablespace set, or instance on which you want to perform related operations.

Table 1 describes the related operations.

Table 1 Operations performed on database/tablespace set/instance protection plans

Operation

Description

Navigation Path

Modifying protection

Scenario

If you need to reselect an SLA associated with a database or tablespace set, you can modify the protection plan. After the modification, the next backup will be performed based on the new protection plan.

Note

The modification does not affect the protection job that is being performed. The modified protection plan will take effect in the next protection period.

In the row of the target database or tablespace set, choose More > Modify Protection.

Removing protection

Scenario

This operation enables you to remove protection for a database or tablespace set when it does not need to be protected. After the removal, the system cannot perform protection jobs for the database or tablespace set. To protect the database or tablespace set again, associate an SLA with it again.

Note

Protection cannot be removed when a protection job of the database or tablespace set is running.

In the row of the target database or tablespace set, choose More > Remove Protection.

Activating protection

Scenario

This operation enables you to activate a disabled protection plan of a database or tablespace set. After the protection plan is activated, the system backs up the database or tablespace set based on the protection plan.

Note

You can perform this operation only on a database or tablespace set that has been associated with an SLA and is in the Unprotected state.

In the row of the target database or tablespace set, choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable the protection plan for a database or tablespace set when its periodic protection is not needed. After the protection plan is disabled, the system does not perform the automatic backup job for the database or tablespace set.

Note

Disabling protection does not affect ongoing backup, archive, or replication jobs.

In the row of the target database or tablespace set, choose More > Disable Protection.

Performing manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target database or tablespace set, choose More > Manual Backup.

Performing resource scan

Scenario

The OceanProtect automatically scans databases in a registered instance every hour. If a database is added, deleted, or modified in the instance, the database is automatically updated to the OceanProtect after the scanning. This operation enables you to immediately update the changed database information to the OceanProtect.

Note

If you do not select to apply the SLA policy to the newly created VM when creating a periodic backup job, the system will not associate the newly discovered VM with an SLA after scanning.

In the row of the target instance, choose More > Resource Scan.

Testing connectivity

Scenario

Manual connectivity test of the registered DB2 instance is used to check the connectivity between the instance and the production environment.

In the row of the target instance, choose More > Test Connectivity.

Modification

Scenario

This operation enables you to modify resource information.

In the row of the target instance or tablespace set, and choose More > Modify.

Deletion

Scenario

This operation enables you to delete a resource when you no longer need to protect the resource or use the resource as the restoration target. After the resource is deleted, you cannot view information about it on the management page.

Note

Before deleting a tablespace set or an instance set, remove the protection for them.

In the row of the target instance or tablespace set, and choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for restoring DB2 databases or tablespace sets is similar to that in Restoring DB2 Databases or Tablespace Sets and is not described here.
