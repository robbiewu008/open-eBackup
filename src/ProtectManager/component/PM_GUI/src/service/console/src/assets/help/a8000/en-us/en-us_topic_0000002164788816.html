Configuring Management Data Backup

Before performing an upgrade or adjusting key data, you can back up management data on the OceanProtect management page. The system then can use the backup data to restore configuration data (such as user, SLA, resource, and alarm notification data) in a timely manner if the data is deleted due to misoperations.

Context

The management data that can be backed up using the OceanProtect includes the KMC keystore file, KMC master key file, database file, ConfigMap, and certificate file.

Precautions

  • If a job is being executed on the OceanProtect, you are advised not to initiate management data backup.
  • If a management data backup or restoration job is being executed, you are not allowed to execute the job again. If a manual job is being executed, the scheduled management data backup job will be canceled.
  • Backup can be performed only after a backup policy is configured.
  • The OceanProtect E6000 Appliance does not support management data backup after the backup storage is reinstalled.
  • After the corresponding backup storage is reinstalled, the OceanProtect E6000 Appliance does not support restoration using the management data backup copies before the re-installation.
  • The following descriptions are applicable to the OceanProtect X series backup appliances:
    • When configuring management data backup, you are not allowed to enable the SFTP service. You can enable the SFTP service only after the backup is complete.
    • Do not perform operations related to management data backup configuration when the SFTP service is being enabled. Perform these operations after the SFTP service is enabled.
    • The system management data backup policy has been configured before the SFTP service is enabled. After the SFTP service is enabled, system management data backup is automatically triggered.
    • If management data exists before the SFTP service is enabled, the existing management data files will become invalid after the SFTP service is enabled because the data files do not meet system restoration requirements. Invalid files can only be deleted.

Procedure

  1. In the navigation pane, choose System > Settings > Management Data Backup.
  2. Configure the management data backup policy.

    In the Management Data Backup area, click Backup Policy. On the Backup Policy page, set the backup policy parameters as described in Table 1. Click OK.
    Table 1 Backup policy parameters

    Parameter

    Description

    Backup Time

    Time for backing up data by day

    If a manual backup job is being executed at the specified time, the automatic backup job will be automatically canceled.

    Retained Backups

    Number of latest available management data backups.

    The value must range from 3 to 20.

    Default value: 7

    SFTP IP Address

    Information about the SFTP server for storing backup files

    Ensure that the SFTP user has the permission to read and write the /var directory.

    After the SFTP IP address is changed, the backup records of the original SFTP server become invalid.

    Default SFTP port: 22.

    SFTP User Name

    SFTP Password

    SFTP Port

    Backup Path

    Set a path for backing up management data. After a backup path is set, the path is automatically generated on the SFTP server. If this parameter is not set, the default value is /var/sysbackup.

    NOTE:
    • Ensure that the SFTP user has the execute permission on your set backup path.
    • If the SFTP user is user root, enter an absolute path. Otherwise, enter a relative path.

    Data Encryption Password

    Enter the password. The password is used to encrypt the system management data to ensure the security of the backup data. This password is also used for encryption in manual backup. Keep this password secure. When the system configurations need to be restored after a system fault, the password needs to be used to decrypt the backup system management data.

    Confirm Data Encryption Password

    Enter the data encryption password again.

  3. Click Manual Backup to manually back up management data immediately.

    During a manual backup, the system also uses the Data Encryption Password to encrypt management data. When the system recovers from a fault, the data encryption password is also required for decryption.

  4. On the Manual Backup page, enter the description and click OK.
