Creating a Media Set (When the Archive Storage Is a Tape Library)

This section describes how to create a media set when a tape library is used as the archive storage. If the archive storage is not a tape library, skip this section.

Context

  • A media set is created based on tapes and is a tape storage pool for storing archive data. Therefore, ensure that a tape library has been scanned and updated to the product before creating a media set.
  • The system automatically scans tape libraries every 10 minutes. When detecting that a tape library is connected to the product, the system automatically updates the tape library to the product.

    If a tape library is not updated to the product, you can manually scan the tape library. For details, see "Scanning Tape Libraries" in the Administrator Guide.

Procedure

  1. Choose System > Infrastructure > Archive Storage Library.
  2. On the Tape tab page, select a node and click Media Set.
  3. Click Create.
  4. Enter basic information in the displayed dialog box.

    1. Specify a name for the medium set.
    2. Select the driver block size. The value can be 64 KB, 128 KB, or 256 KB. Data is written to tapes block by block. Data is written to tapes in a media set based on the set block size.

      You are advised to select the driver block size based on the file size.

      • For large files, select a large driver block size to ensure read and write efficiency.
      • For small files, select a small drive block size to make full use of tape space.
    3. Determine whether to enable data compression.
      • Enable: enables the hardware compression function of the tape library to improve the tape capacity and the write speed.
      • Disable: disables the hardware compression function of the tape library.

  5. Select tapes.

    1. Select the controller to which the tapes belong.
    2. Select tapes.

      In the tape library list on the left, select a tape library and select tapes in the tape library on the right.

      • Tapes in multiple tape libraries can be added. Tapes across controllers cannot be added.
      • Tapes that have been added to other media sets cannot be added to the media set.
      • The tape must be in the tape library and its Write Status cannot be Error or Unknown.

  6. Determine whether to enable the alarm of insufficient available tapes.

    • Enable the alarm. In this case, you need to set the threshold of tapes. If the number of empty tapes and expired tapes that can be overwritten is smaller than the threshold, an alarm is generated. To clear the alarm, you can insert empty tapes or mark tapes as empty.
      • If data compression is enabled, the maximum amount of data that can be stored on a tape is unknown. For example, if the nominal capacity of a tape is 50 GB, the tape may actually be able to store 60 GB, 80 GB, or 100 GB. Therefore, alarms cannot be generated based on the tape capacity, but can be generated only based on the number of empty tapes.
      • The alarm threshold cannot exceed the total number of selected tapes.
    • Disable the alarm. After the alarm for insufficient available tapes is disabled, the number of available tapes becomes unknown. You are advised to enable this alarm.

  7. Select a tape retention policy.

    Table 1 describes the parameters related to the tape retention policy.
    Table 1 Parameters related to the tape retention policy

    Parameter

    Description

    No Data Protection (Overwrite Tape as Needed)

    Data in the tape is not protected. When new data is written into the tape, the original data is overwritten.

    To select this retention policy, ensure that there are at least two tapes in the current media set.

    Permanent Data Retention

    Data in the tape is permanently retained.

    Temporary Data Protection

    Data in the tape is protected for a specified period, which can be xx days, months, or years.

    For example, if the period for temporary data protection is set to 10 days, the tape is marked as empty after 10 days since the last data is written into the tape. Then, the tape can be overwritten.

  8. Click OK. After a media set is created, the system updates the status and capacity information of the media set every hour. Before the system updates data, the status and capacity of a media set may be inconsistent with the actual situation.
