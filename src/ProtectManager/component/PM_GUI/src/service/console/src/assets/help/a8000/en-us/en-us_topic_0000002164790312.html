How Do I Locate the Causse of a Client to Be Offline?

This section describes how to locate the cause when a client is offline.

Linux OS/AIX OS/Solaris OS

  1. Use PuTTY to log in to the host where the client is to be installed as user root.
  2. Check whether the firewall of the host where the client is installed is disabled.

    If the firewall is enabled, the communication requests between the OceanProtect and host where the client is installed may be intercepted. If the firewall is enabled and cannot be disabled directly, enable the corresponding ports in the OceanProtect DataBackup 1.5.0-1.6.0 Communication Matrix.

    To ensure that the client status is Online, ensure that port 25082 of the OceanProtect and ports 59520–59559 on the host where the client is installed are enabled.

  3. Run the following command to check whether the client processes are running properly:

    ps aux | grep rdadmin

    If information about four client processes is displayed, the client processes are running properly. Otherwise, run the start.sh script in the Client installation directory/DataBackup/ProtectClient path to start the client processes.

Windows OS

  1. Log in to the host where the client is installed as user Administrator.
  2. Check whether the firewall of the host where the client is installed is disabled.

    If the firewall is enabled, the communication requests between the OceanProtect and host where the client is installed may be intercepted. If the firewall is enabled and cannot be disabled directly, enable the corresponding ports in the OceanProtect DataBackup 1.5.0-1.6.0 Communication Matrix.

    To ensure that the client status is Online, ensure that port 25082 of the OceanProtect and ports 59520–59559 on the host where the client is installed are enabled.

  3. Check whether the client processes are running properly.

    1. Right-click on the task bar.
    2. Choose Task Manager from the shortcut menu.
    3. On the Processes tab page, check whether the client-related processes (monitor, rdagent, and rdnginx) are running properly.

      If the client processes are not running properly, run the start.sh script in the Client installation directory\DataBackup\ProtectClient path to start the client processes.
