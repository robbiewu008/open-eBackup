Performing Periodic Backup

After the VMs to be protected are associated with a specified SLA, the system periodically backs up the VMs based on the SLA.

Prerequisites

  • If you want to back up data when the VM is started, ensure that Tools has been configured for the VM. Otherwise, backup cannot be performed. For details about how to install the tool, see the FusionCompute Online Help.
  • For FusionCompute 8.7.0 and later versions, if you want to use an independent backup network plane for LAN-based backup and restoration, configure an independent backup network plane through FusionCompute first. For details, see Configuring the Backup Network Plane (Applicable to FusionCompute 8.7.0 and Later Versions) (applicable to only 1.6.0 and later versions).

Procedure

  1. Choose Protection > Virtualization > FusionCompute.
  2. On the left of the page, select the VM that you want to protect.

    • When you select a cluster on the Cluster tab page, the system protects all VMs in the cluster.
    • When you select a host on the Hosts tab page, the system protects all VMs on the host.
    • When you select a VM on the VMs tab page, the system protects the VM.
    • When you select a VM group on the VM Groups tab page, the system protects all VMs in the VM group.
      • VM Groups is available only in 1.6.0 and later versions.
      • A maximum of 100 FusionCompute VMs can be protected in a batch.

  3. Click Protect.
  4. When you select a VM for protection on the VMs tab page, determine whether to enable automatic disk protection. In other scenarios, skip this step.

    Automatic disk protection is enabled by default. After this function is enabled, all disks of the VM, including newly detected disks, are protected. After automatic disk protection is disabled, you need to manually select disks to be protected.

    If the system disk is deselected, the generated copy can only be used for disk restoration.

  5. Select an SLA. You can select a preset SLA or click Create to create an SLA.
  6. Set the following parameters in Advanced.

    Table 1 Advanced parameters

    Parameter

    Description

    Consistency Snapshot

    If this function is enabled, the cache data that is not saved on a VM is saved before a consistency snapshot is created. If the VM does not support consistency snapshots, the job is automatically executed using common snapshots.

    Whether the VM supports consistency snapshots needs to be confirmed on the FusionCompute platform.

    Snapshot Deletion Rate (MB/s)

    After the backup is complete, the system deletes the snapshots generated during the backup. Snapshot deletion occupies storage resources. You can configure this parameter to specify the snapshot deletion speed. The default value is 30 MB/s.

    Storage resources are occupied. A higher rate has greater impact on performance of VMs running on the storage system. The snapshot deletion rate is invalid when Huawei scale-out block storage is used as the datastore.

    Agent Host

    Select the agent host for data protection.

    SLA Overwrite Policy
    When you select a cluster or host for protection, you can choose whether to overwrite the existing VM SLAs.
    NOTE:

    If you select Do not overwrite existing VM SLAs when modifying the SLA associated with the cluster or host, the VM SLAs are still overwritten when they are the same as the SLA of the cluster or host before the modification.

    SLA Application Policy
    When you select a cluster or host for protection, you can choose the application scope of the SLA policy as follows:
    • Applying the cluster or host SLA policy to all VMs that are not associated with any SLA
    • Applying the cluster or host SLA policy to newly created VMs

      After the virtualization environment is scanned, the newly discovered VMs are associated with the same SLA as the cluster or host.

    VM Filter
    If you want to specify the VMs to be excluded or included in the protection plan when selecting a cluster or host for protection, enable this function and perform the following operations:
    1. Select a filtering mode.
    2. Enter a VM name or keyword in the text box to filter VMs based on the selected filtering mode.
      • Use commas (,) to separate multiple names or keywords.
      • Exact match and fuzzy match of VM names are supported. Fuzzy match rules include prefix match (for example, abc*), suffix match (for example, *abc), and middle match (for example, *abc*).

  7. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
