What Should I Do If a Backup Job Fails After the Capacity of the Production Storage Is Expanded?

If the capacity of a LUN corresponding to the datastore where a VMware VM resides is expanded, you need to scan the LUN again on the host where the client is installed by referring to the following steps. Otherwise, the backup job will fail.

Procedure

  1. Use PuTTY to log in to the host where the client is to be installed as user root.
  2. Run the following command to scan for LUNs:

    for host in $(ls -1d /sys/class/scsi_host/*); do echo "- - -" > ${host}/scan ; done

  3. Run the following command to check whether the LUN capacity is expanded:

    fdisk -l

  4. Restart the host where the client is installed.

    Perform this step only when the LUN capacity obtained in 3 is different from the actual LUN capacity. After the restart, run the fdisk -l command to check the LUN capacity again.
