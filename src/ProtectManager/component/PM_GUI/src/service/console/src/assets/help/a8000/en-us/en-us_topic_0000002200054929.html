VM Restoration Is Successful but VM Startup Fails

Symptom

Power on Automatically After Restoration is selected for a job of restoring a VM to the original location. The restoration job is successfully executed, but the VM fails to be started. The restoration job uses the Hot-Add transmission mode, which can be queried in the restoration job details.

Possible Causes

When a VM is restored in Hot-Add transmission mode, the VM disks are unmounted from the host where the client is installed after data restoration is complete. However, the disks fail to be unmounted due to an internal error of the VDDK library and then get locked. Consequently, the VM cannot be started.

Troubleshooting

  1. Log in to the vCenter Server on a browser.

    Login address: https://vCenter Server IP address

  2. Manually delete the VM disks from the host where the client is installed.

    1. Locate the host where the client is installed.
    2. Choose ACTIONS > Edit Settings.
    3. Locate the disks to be deleted based on the restoration job ID.

      The restoration job ID can be obtained from the job details on the management page, for example, b6f83bfa-c42d-476f-854a-f4f9f6b2e427.

    4. Click on the right of the disk to unmount the disk. When unmounting the disk, deselect Delete files from datastore.
    5. Click OK.

  3. Find the restored VM on the vCenter Server.
  4. Choose ACTIONS > Snapshots > Consolidate.
  5. After the consolidation is complete, choose ACTIONS > Power > Power On to start the VM.
