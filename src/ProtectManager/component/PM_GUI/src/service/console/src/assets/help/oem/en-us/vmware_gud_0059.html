Restoring Hyper-V VM Disks

This section describes how to restore disks of a VM that has been backed up to the original VM or a new VM.

Context

The product allows you to restore disks to the original or a new VM using backup copies, replication copies (restoration to the original VM is not supported).

  • Restoring disks to the original VM

    When restoring disks to the original VM, the disks will be replaced if disks exist in the corresponding slot of the original VM.

  • Restoring disks to a new VM

    When restoring disks to a new VM, the disks will be replaced if disks exist in the corresponding slot of the new VM.

    When restoring disks to a VM, ensure that the target VM has disks. Otherwise, the disks cannot be selected.

Procedure

  1. Choose Explore > Copy Data > Virtualization > Hyper-V.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    Click Resources to switch to the corresponding page, search for the VM to be restored by name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. In the row of the target copy, choose More > Disk Restoration.
  5. Select the disks to be restored and click Select.
  6. Restore the VM disks to the original VM or a new VM.

    • Restore the VM disks to the original VM.
      1. Select Original location.
      2. Select whether to automatically start the VM after the restoration.
      3. Select whether to delete the original VM after the restoration.
    • Restore the VM disks to a new VM.
      1. Select New location.
      2. Set information about a new location to which the VM is restored.
        Table 1 describes the related parameters.
        Table 1 New VM information

        Parameter

        Description

        Compute Location

        Target VM to which the backup disk is restored.

        NOTE:

        The OS type of the target VM must be the same as that of the source VM. For example, both of them use the Windows or Linux OS.

        Target VM

        Path for restoring the backup disks to the Hyper-V server.
      3. Select whether to automatically start the VM after the restoration.

  7. Click OK.
  8. Select the disk corresponding to the target datastore.
  9. Click OK.
