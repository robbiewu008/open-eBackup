Step 1: Registering the SAP HANA Database (Backint Backup Mode)

The Backint backup mode writes backup data through third-party backup tools. Each SAP HANA service starts Backint for SAP HANA agent and sends the data to be backed up to third-party backup tools.

Procedure

  1. Choose Protection > Applications > SAP HANA.
  2. Register an SAP HANA instance.

    On the Instance tab page, click Register. Table 2 describes the parameters for registering an SAP HANA instance.
    Table 1 SAP HANA instance registration information

    Parameter

    Description

    Name

    User-defined instance name.

    Instance ID

    ID set during database installation. For details about how to query the instance ID, see Preparations for Backup.

    Hosts

    Host where the database resides.

    If the database is deployed in a cluster, select all hosts where the database resides.

    Authentication Mode

    Select Database authentication or Hdbuserstore mode.

    NOTE:
    • When the Database authentication mode is selected, you need to enter the database username and password during registration.

    Hdbuserstore KEY
    If the Hdbuserstore mode is selected in cluster deployment mode, you need to set the Hdbuserstore KEY parameter on all hosts where the instance resides.
    1. Use PuTTY to log in to the host where the SAP HANA database is located as user root.
    2. Run the following command to switch to the SAP HANA database user, for example, s00adm:
      su - s00adm
    3. Run the following command to set the Hdbuserstore KEY parameter:
      hdbuserstore set S00SAPDB "hana1:30013" SYSTEM 1234

      In the preceding command, S00SAPDB is the KEY parameter set by the user, hana1 is the host name, 30013 is the default port number, and 1234 is the database password. Replace them with the actual values.

    4. Run the following command to check the Hdbuserstore KEY parameter:
      hdbuserstore list

      In the following command output, the line where KEY is located is the Hdbuserstore KEY parameter.

    System database username

    When Authentication Mode is set to Database authentication, enter the username and password of the system database.

    System database password

    System database port

    Port number set during database installation. The default port number is 30013.

    Log Backup

    Logs can be backed up only after the function is enabled.

    Regular log backup interval

    Set the interval for backing up logs.

    NOTE:

    It is recommended that the interval be set to 50% to 100% of the log backup interval set in Table 1. For example, if the log backup interval set in the backup policy is 60 minutes, you are advised to set the interval to a value ranging from 30 minutes to 60 minutes.

    Log backup path

    Set the path for storing backup data logs.

    NOTE:
    • Ensure that the available space in the log backup path is greater than the space to be occupied by all log backup copies, and ensure that the SAP HANA log backup path has at least the permission 755.
    • If the path for backing up logs has been set in the production environment, the original log backup path will be changed after this parameter is set.
    • The log backup path takes effect only after a backup job is executed.

  3. Click OK.
  4. Register the SAP HANA database.

    On the Databases tab page, click Register. Table 2 describes the parameters for registering the SAP HANA database.
    Table 2 SAP HANA database registration information

    Parameter

    Description

    Type

    Select a database type.

    • System Database
    • Tenant Database
    NOTE:

    When registering a system database, tenant database, or cluster, you must have the backup administrator permission. You are advised to use SYSTEM as the system database username.

    Name
    Name of the database to be protected, which must be the same as that of the database in the production environment. For details about how to query the name, see Preparations for Backup.
    NOTE:
    • If Type is set to System Database, select the database name that is set when the database is registered and installed.
    • If Type is set to Tenant Database, select the tenant database name set during tenant database registration.

    Instance

    Selects the corresponding registered instance.

    Hosts

    Host where the database resides.

    If the database is deployed in a cluster, select all hosts where the database is distributed.

    Authentication Mode

    Select Database authentication or Hdbuserstore mode.

    NOTE:
    • When the Database authentication mode is selected, you need to enter the database username and password during registration.

    Hdbuserstore KEY
    If the Hdbuserstore mode is selected in cluster deployment mode, you need to set the Hdbuserstore KEY parameter on all hosts where the database resides.
    1. Use PuTTY to log in to the host where the SAP HANA database is located as user root.
    2. Run the following command to switch to the SAP HANA database user, for example, s00adm:
      su - s00adm
    3. Run the following command to set the Hdbuserstore KEY parameter:
      hdbuserstore set S00SAPDB "hana1:30013" SYSTEM 1234

      In the preceding command, S00SAPDB is the KEY parameter set by the user, hana1 is the host name, 30013 is the default port number, and 1234 is the database password. Replace them with the actual values.

    4. Run the following command to check the Hdbuserstore KEY parameter:
      hdbuserstore list

      In the following command output, the line where KEY is located is the Hdbuserstore KEY parameter.

    System database username

    When Authentication Mode is set to Database authentication, enter the username and password of the system database.

    System database password

    System database port

    Port number set during database installation. The default port number is 30013.

  5. Click OK.
