Managing Databases

The product allows you to modify, remove, activate, or disable protection for databases.

Related Operations

You need to log in to the management page, choose Protection > Databases > Oracle, click the Databases tab, and find the database on which you want to perform related operations.

Table 1 describes the related operations.

Table 1 Database protection operations

Operation

Description

Navigation Path

Modifying protection

Scenario

When you need to reselect the SLA associated with a database and modify the execution scripts before and after database protection, you can modify the protection plan. After the modification, the next backup will be executed based on the new protection plan.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

In the row of the target database, choose More > Modify Protection.

Removing protection

Scenario

If you do not need to protect a database, you can remove the protection from the database. After the removal, the system cannot execute protection jobs for the database. To protect the database again, you need to associate an SLA with the database again.

Note

When a protection job of the database is running, the protection cannot be removed.

In the row of the target database, choose More > Remove Protection.

Manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target database, choose More > Manual Backup.

Disabling protection

Scenario

This operation enables you to disable the protection plan for a resource for which periodic protection is not required. After the protection plan is disabled, the system does not perform automatic backup or replication for the resource.

Note

Disabling protection does not affect ongoing backup or replication jobs.

In the row of the target database, choose More > Disable Protection.

Activating protection

Scenario

This operation enables you to activate a disabled protection plan for a resource. After the protection plan is activated, the system backs up or replicates the resource based on the protection plan.

In the row of the target database, choose More > Activate Protection.

Scanning Resources

Scenario

If the information about a registered database changes, for example, the database version changes, you can perform this operation to scan the database and update the information. Otherwise, the execution of backup and recovery jobs may be affected.

In the row of the target database, choose More > Resource Scan.

Testing connectivity

Scenario

This operation enables you to test the connectivity between the product and the database.

In the row of the target database, choose More > Test Connectivity.

Modifying database information

Scenario

This operation enables you to modify information such as the database name and database authentication mode.

In the row of the target database, choose More > Modify.

Deleting a database

Scenario

This operation enables you to delete a database that is no longer used.

Note

Before deleting a database, remove protection from the database.

In the row of the target database, choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for restoring Oracle data is the same as that described in Restoration.
