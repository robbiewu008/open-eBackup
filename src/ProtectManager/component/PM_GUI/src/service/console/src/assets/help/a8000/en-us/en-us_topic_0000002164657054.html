A Migrated Disk Fails to Be Mounted When It Is Overwritten During Restoration

Symptom

After a FusionSphere OpenStack disk is migrated, the disk fails to be mounted when it is overwritten during restoration. As a result, the restoration job fails.

Possible Causes

In FusionSphere 22.1.0 and earlier versions, after a disk is migrated across storage devices, the mapping information fails to be queried.

Procedure

  1. Use PuTTY to log in to the OpenStack node using the management IP address as user fsp.
  2. Run the following command to switch to user root:

    su - root

  3. Run the source set_env command to import environment variables, select cloud_admin, and enter its password (obtained from the environment administrator).
  4. Run the following command to check the FusionSphere version:

    get-fs-version

    If the version is NFV_FusionSphere 22.1.0 and earlier versions, upgrade it to NFV_FusionSphere 22.1.0.SPH12 and later versions.
