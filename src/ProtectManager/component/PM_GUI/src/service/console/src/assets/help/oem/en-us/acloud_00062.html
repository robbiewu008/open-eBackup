Viewing Alibaba Cloud Copy Information

This section describes how to view Alibaba Cloud copy information.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Alibaba Cloud.
  2. Select a viewing mode to view copy information.

    Replication copies can be viewed only in the target cluster.

    • Resources: View resources and copy information by cloud server. When you need to restore a faulty disk on the cloud server, you are advised to click this icon and then search for the desired copy.

      Table 1 describes the related parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Name

      Cloud server name.

      Click the name of a cloud server to view its copy information.

      You can search for copies by year, month, or day.

      If is displayed below the time, a copy at that time exists.

      Location

      Cloud server location. The format is Domain/Project/Cloud server.

      Status

      Whether a cloud server is registered with the current management page.

      • If yes, Status is Existing.
      • If no, Status is Not existing.

      Copies

      Quantity

      Number of copies generated based on a cloud server. The number of replication copies is displayed only in the target cluster.

      Replicated Copy SLA

      Archive SLA associated with a replication copy.

      If an archive SLA has been configured for the replication copy generated by the resource, the SLA name is displayed.

      Replicated Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: A replication copy is not associated with an SLA, or a replication copy is associated with an SLA but protection is disabled.
      • Protected: A replication copy is associated with an SLA and protection is enabled.
    • Copy: View information about all copies by copy. When you need to restore data to the original location or a new location using a specified copy, you are advised to click this icon and then search for the desired copy.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copies

      Backup Time

      Time when a backup copy is generated.

      Copy Time

      Time when a copy is generated.

      Click Copy Time to view the copy information such as expiration time and verification. If -- is displayed for the expiration time, the copy is retained permanently.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being restored or in instant recovery.
      • Deletion failed: The copy fails to be deleted.

      Location
      Location of a copy.
      • For 1.5.0: If Generation Mode of a copy is Backup, Cascaded Replication, Reverse Replication, or Replicate, the copy is stored in the local cluster. In this case, Local is displayed.
      • For 1.6.0 and later versions, if Generation Mode of a copy is Backup, Cascaded Replication, Reverse Replication, or Replicate, the copy is stored in the corresponding backup storage unit. In this case, the name of the corresponding backup storage unit is displayed.
      • If Generation Mode of a copy is Object Storage Archive or Import, the copy is stored in an object storage bucket. In this case, the bucket name is displayed.
      • If Generation Mode of a copy is Tape Archive, the copy is stored in a tape library. In this case, the media set name is displayed.

      Generation Mode

      Mode in which a copy is generated.

      • Replicate: copies generated by a replication job.
      • Backup: copies generated by backup jobs.
      • Object Storage Archive: copies generated by object storage archive jobs.
      • Tape Archive: copies generated by tape archiving jobs.
      • Cascaded Replication: copies generated by cascaded replication jobs.
      • Reverse Replication: copies generated by reverse replication jobs.

      Copy Type

      Backup type of a generated copy.

      It can be Full or Forever Incremental (Synthetic Full).
      Expired At
      NOTE:

      This parameter is available only in 1.5.0SPC26 and later versions.

      Copy expiration time. If a copy is permanently valid, -- is displayed.

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.
      • : The copy has expired. This state is available only in 1.6.0 and later versions. It indicates that the WORM copy has expired. Copies in this state can be deleted.

      WORM Expiration Time
      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.
      NOTE:

      The WORM expiration time is based on the WORM file system compliance clock. If the WORM file system compliance clock time is inconsistent with the system time, check the WORM status after the WORM file system compliance clock reaches the set expiration time.

      Index

      After a copy is generated, the system creates indexes for all files and directories in the copy to quickly search for the corresponding files for restoration. The options are as follows:

      • Indexed: The system has created an index for a copy.
      • Indexing: The system is creating an index for a copy after it is generated.
      • Not indexed: The index function is not enabled in the SLA.
      • Index deleting: The index is being deleted.
      • Index failed: The system fails to create an index for a copy.

      Resources

      Name

      Name of the cloud server where a copy is located.

      Location

      Location of the cloud server where a copy is located.

      Status

      Whether the cloud server where a copy is located is registered with the current management page.

      • If yes, Status is Existing.
      • If no, Status is Not existing.

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
