In GaussDB (DWS) 8.1.3, a Backup Job Fails Due to GaussDB (DWS) Software Processing Exceptions

Fault Symptom

In GaussDB (DWS) 8.1.3, a backup job fails to be executed when the client is installed on the production host and an independent physical server, and a message is displayed indicating that the Roach command fails to be executed.

Possible Cause

After data backup is complete, the upload process is started before the GaussDB (DWS) backup process exits completely. As a result, service processing conflicts.

Suggestion

  1. Log in to the node where the backup command is delivered.
  2. Run the following command to go to the log file directory.

    cd /opt/DataBackup/ProtectClient/ProtectClient-E/slog/Plugins/GeneralDBPlugin/

  3. Run grep {Job ID} dws.log | grep "Backup failed" command to check whether the error log file contains the following error information:

    Failed to do gs roach. Roach operation backup failed

    Example:

    • If the error information is contained, perform the backup job again.
    • If no error information is contained, go to 4.

  4. Log in to other CNs of GaussDB (DWS) where the client has been deployed and perform 2 to 3. If no node contains the error information, collect logs and contact Huawei technical support engineers.
