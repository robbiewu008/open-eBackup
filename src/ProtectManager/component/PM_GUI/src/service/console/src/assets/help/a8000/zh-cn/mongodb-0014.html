步骤5：执行备份

执行备份前将需要保护的资源关联到指定SLA策略，系统将对这些资源按照SLA策略进行保护，并根据SLA策略周期性执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

注意事项

手动备份产生的副本保留时长按照SLA中的定义进行保留。

操作步骤

  1. 选择“保护 > 大数据 > MongoDB”。
  2. 选择需要保护的实例，单击“保护”。

    您也可以勾选多个实例进行批量保护。

  3. （可选）在高级配置区域设置备份任务中创建快照逻辑卷所需空间在原逻辑卷中的占比。
  4. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

    如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  5. 在“高级配置”中配置创建逻辑卷空间比例参数。

    备份任务中创建快照逻辑卷所需空间在原逻辑卷中的占比，请确保原逻辑卷中空闲空间的占比大于该值。

  6. 单击“确定”。

    如果当前系统时间已晚于该SLA首次备份的开始时间，则关联备份SLA后可以选择在当前立即执行备份或者根据SLA中设置的备份策略等待下次调度。

  7. 执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。

    1. 在需要备份的实例所在行单击“更多 > 手动备份”。
    2. 设置手动备份产生的副本名称。
    3. 如果您不设置，系统默认设置副本名称为：backup_时间戳。
    4. 选择保护策略：全量备份或日志备份。
      • MongoDB单实例未开启Oplog不支持日志备份。
      • 对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。
    5. 单击“确定”。
