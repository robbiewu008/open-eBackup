步骤5：注册VMware虚拟化环境

执行VMware备份前，需要将VMware虚拟化环境注册至OceanProtect。

背景信息

  • 注册VMware虚拟化环境至OceanProtect时，支持注册vCenter Server或ESXi主机。如果有多个独立的ESXi主机则需要注册多次。
  • 注册VMware虚拟化环境后，可以在管理界面浏览vCenter Server、ESXi主机或虚拟机的信息并对其进行保护。

操作步骤

  1. 选择“保护 > 虚拟化 > VMware”。
  2. 单击“注册”，注册VMware虚拟化平台信息。

    VMware注册信息如表1所示。
    表1 VMware注册信息

    参数

    说明

    名称

    用户自定义虚拟化平台名称。

    IP地址
    • 当由vCenter Server管理ESXi主机时，请输入vCenter Server的管理IP地址。
    • 当ESXi主机未被vCenter Server管理时，请输入ESXi主机的管理IP地址。

    端口

    OceanProtect与vCenter Server或者ESXi主机管理平面通信的端口号。

    默认端口号为443。

    用户名
    • 当注册的是vCenter Server时，输入登录VMware vSphere web Client的用户名及其密码。
    • 当注册的是ESXi时，输入登录ESXi主机的用户名及其密码。

    密码

    资源扫描频率

    系统自动扫描资源的间隔时间。

    设置后系统将按照该频率自动扫描VMware虚拟化环境，将变更后的虚拟机更新至OceanProtect。

    校验证书

    默认开启，注册VMware受保护环境时，系统将对其所访问设备的信息进行校验。关闭后，系统与VMware通信可能存在安全风险，请谨慎操作。

    证书

    开启“校验证书”后，导入步骤4：获取VMware证书中获取的证书。

    吊销列表

    吊销列表由证书颁发机构发布，导入证书吊销列表后，系统将校验客户端证书是否被吊销，如果证书已被吊销，则注册集群失败。

    安全tls版本与算法
    默认关闭，关闭后默认使用TLS 1.3安全协议，开启后通信双方自主协商。
    说明：

    开启后，系统连接外部服务端可能会使用不安全的加密算法，存在安全风险，请谨慎操作。

  3. 对于1.6.0及后续版本，如需执行存储层备份，请执行本步骤添加存储资源。否则，请跳过。

    1. 在“存储资源”区域，单击“添加”。
    2. 按界面要求配置参数。参数说明如表2所示。
      表2 添加存储资源参数说明

      参数

      说明

      类型

      存储资源的类型，包括：OceanStor Dorado 6.x和NetApp ONTAP。

      说明：

      仅1.6.0及后续版本支持选择NetApp ONTAP。

      管理IP地址

      存储设备的管理IP地址。

      单击“添加”可以填写存储设备的多个管理IP地址。当某个管理IP地址无法访问时，系统将自动切换到其他可访问的管理IP地址。

      端口

      OceanProtect与存储设备的管理平面通信的端口号。

      用户名

      存储设备的用户名及密码，用户角色必须是管理员或超级管理员。

      密码

      校验证书

      默认开启，添加存储设备时会对其证书进行校验，确保OceanProtect与存储设备交互的安全性。关闭后，系统与存储设备通信可能存在安全风险，请谨慎操作。

      证书

      导入存储设备的CA证书。获取存储设备的CA证书具体操作可参考《OceanProtect DataBackup 1.5.0-1.6.0 数据备份特性指南(适用于NAS文件)》中的“步骤1：获取存储设备CA证书”章节。

      吊销列表

      吊销列表由证书颁发机构发布，导入证书吊销列表后，系统将校验客户端证书是否被吊销，如果证书已被吊销，则注册存储设备失败。

  4. 单击“确定”。

    注册成功后，已注册的VMware虚拟化环境信息将在页面左侧显示。您可以浏览vCenter Server或ESXi主机，及其虚拟机列表。
