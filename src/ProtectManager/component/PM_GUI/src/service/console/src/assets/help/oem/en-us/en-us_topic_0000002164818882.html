How Do I Manually Stop a TDSQL Restoration Job?

This section describes how to manually stop a TDSQL restoration job.

Precautions

Stopping a running restoration job may affect the database environment or data on the production end. Exercise caution when performing this operation.

Procedure

  1. Log in to all data nodes of the target instance for restoration as user root.
  2. Run the following commands to stop the restoration job:

    cd /opt/DataBackup/ProtectClient

    sh stop.sh

    In the preceding commands, /opt indicates the client installation directory. Replace it with the actual directory.

  3. Choose Jobs.

    For 1.5.0, choose .

  4. Check the status of the restoration job. After the job status changes to Failed, log in to all data nodes of the target instance for restoration again and check whether the 4002_deploy.conf file exists.

    4002 in 4002_deploy.conf is the port used by the TDSQL storage node. Replace it with the actual port. The following operations use 4002_deploy.conf as an example.

    cd /data/oc_agent/log/

    ll |grep deploy

    • If the 4002_deploy.conf file exists, run the rm 4002_deploy.conf command to delete the file, and then go to 5.
    • If the 4002_deploy.conf file does not exist, go to 5.

  5. Run the following commands on all data nodes of the target instance for restoration to start the restoration job:

    cd /opt/DataBackup/ProtectClient

    sh start.sh
