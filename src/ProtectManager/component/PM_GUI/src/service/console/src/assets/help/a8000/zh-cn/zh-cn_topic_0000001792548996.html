检查集群数据的存放位置

Oracle集群的备份恢复场景，要求集群数据必须存放在ASM内，否则可能导致备份恢复任务失败。参考本节检查集群数据的存放位置。非集群场景请跳过本章节。

操作步骤

  1. 登录Oracle数据库主机。

    • 如果是Linux操作系统，请使用PuTTY登录Oracle数据库主机，并执行su - oracle命令，切换到oracle账户。
    • 如果是Windows操作系统，请使用系统管理员登录Oracle数据库主机。

  2. 执行以下命令，登录Oracle数据库实例。

    • OS认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus / as sysdba

      • 如果是Windows操作系统，请执行以下操作：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus / as sysdba

    • 数据库认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus

        根据提示输入具备sysdba权限的用户：

        username as sysdba

        根据提示输入密码。

      • Windows OS执行以下命令：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus

          根据提示输入具备sysdba权限的用户：

          username as sysdba

          根据提示输入密码。

  3. 执行以下命令查看集群数据的存放位置。

    select name from v$datafile;

    如果所有路径均以+开头，则集群数据存放在ASM内，可以进行备份恢复。否则，不支持备份恢复。
