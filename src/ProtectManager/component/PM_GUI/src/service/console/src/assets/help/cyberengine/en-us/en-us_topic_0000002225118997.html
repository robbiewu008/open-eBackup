Managing SNMP Trap Notification

By configuring SNMP, you can query and set information about the OceanCyber 300 Data Security Appliance, or receive alarms of the OceanCyber 300 Data Security Appliance. The OceanCyber 300 Data Security Appliance supports SNMPv2c and SNMPv3. This section describes SNMP configuration methods when different protocols are used.

Context

Trap is a type of SNMP message that indicates the occurrence of an event. It is supported only by the Data Security Appliance. To send alarm notifications by SNMP, you must configure a trap server IP address. After the trap server address is set, alarms of the OceanCyber 300 Data Security Appliance will be sent to the third-party network management tool specified by the trap.

Precautions

A third-party network management tool must meet the following requirements:

  • If SNMPv2c is used, the community name used by the third-party network management tool must be the same as that configured on the OceanCyber 300 Data Security Appliance.
  • If SNMPv3 is used, the username, authentication protocol, and data encryption protocol of SNMPv3 must be the same as those configured on the OceanCyber 300 Data Security Appliance.

Prerequisites

You have logged in to the OceanCyber WebUI as a system administrator.

Procedure

  1. Choose System > Settings > SNMP Trap.
  2. In the Trap Parameter Settings area, set SNMPv2c or SNMPv3 parameters.

    There will be security risks in enabling SNMPv2c. You are advised to use the secure SNMPv3 protocol.

    • Setting SNMPv3 parameters
      1. Click Modify.
      2. Set Protocol Version to SNMPv3.

        If you need to configure an engine ID for SNMP of a third-party NMS, you can configure it based on the controller SNMPv3 engine ID displayed on the page.

      3. Set SNMPv3 parameters. Table 1 describes related parameters.
        Table 1 Parameter description

        Parameter

        Description

        Username

        Username.

        The value contains 1 to 64 characters.

        Authentication Protocol

        Authentication algorithm of the user, including None, HMAC_MD5, HMAC_SHA1, and HMAC_SHA2.

        Default value: HMAC_SHA2.

        NOTE:

        HMAC_SHA is more secure than HMAC_MD5. You are advised to use HMAC_SHA2.

        Authentication Password

        Authentication password of the user.

        Confirm Authentication Password

        Confirms the authentication password of the user.

        Data Encryption Protocol

        Encryption protocol of the user, including None, DES, and AES.

        Default value: AES.

        NOTE:

        AES is more secure than DES. For security purposes, you are advised to use AES.

        Data Encryption Password

        Data encryption password used by the user.

        NOTE:

        The data encryption password and authentication password must be different.

        Confirm Data Encryption Password

        Confirms the data encryption password used by the user.

        Environment Name

        Name of the environment engine.

        Environment Engine ID

        Unique identifier of an SNMP engine. This ID is used together with Environment Name to identify the environment of an SNMP entity.

        An SNMP message packet is processed only when the environment of the transmitter fully matches that of the receiver. Otherwise, the SNMP message packet is discarded.
      4. Click Save.
    • Setting SNMPv2c parameters
      1. Click Modify.
      2. Set Protocol Version to SNMPv2c.
      3. Set the community name. Table 2 describes related parameters.
        Table 2 Community name parameters

        Parameter

        Description

        Community Name

        Name of the community that has the read and write permissions.
      4. Click Save.

  3. In the Trap Server area, click Add to configure a trap server. For details about related parameters, see Table 3.

    Table 3 Trap server configuration parameters

    Parameter

    Description

    IP Address

    Currently, only IPv4 addresses are supported.

    Port

    Port used by the trap server to listen to trap messages.

    Description

    Trap server description.
