Managing Ransomware Detection Snapshots

You can perform operations on ransomware detection snapshots on ProtectManager.

You need to log in to ProtectManager and choose Data Security > Intelligent Ransomware Detection Results > Intelligent Ransomware Detection Results. Table 1 describes the related operations.

Table 1 Operations related to ransomware detection snapshots

Operation

Description

Navigation Path

Viewing information about local file systems or local LUNs

Context

This operation enables you to view the number of ransomware detection snapshots created for local file systems or local LUNs and the detection status.

Table 2 and Table 3 describe the related parameters.

Viewing ransomware detection snapshot information

Context

This operation enables you to view ransomware detection snapshot details.
  1. Click the number of ransomware detection snapshots to display the Ransomware Detection Snapshot page.
  2. Table 4 describes the related parameters.

Manual detection

Context

This operation enables you to perform manual detection for a file system or a LUN.
Locate the row that contains the desired file system or LUN and choose More > Manually Detect.
NOTE:

You can also click the number of the ransomware detection snapshots to go to the Ransomware Detection Snapshot page, locate the row that contains the target ransomware detection snapshot, and choose More > Manually Detect.

Handling false detection

Context

If an uninfected copy is detected as Infected, you can correct the copy status through false detection handling.

Precautions

Only copies in Infected status support false detection handling. After the false detection is handled, no alarm will be generated for subsequent similar copies.
Locate the row that contains the desired file system or LUN and choose More > Handle False Detection.
NOTE:

You can also click the number of the ransomware detection snapshots to go to the Ransomware Detection Snapshot page, locate the row that contains the target ransomware detection snapshot, and choose More > Handle False Detection.

Viewing the detection report

Context

This operation enables you to view the detection report of a ransomware detection snapshot.
  1. Click the number of ransomware detection snapshots to display the Ransomware Detection Snapshot page.
  2. Locate the row that contains the target ransomware detection snapshot and choose More > Detection Report.
    NOTE:

    As for block services, if an exception occurs during the detection of a snapshot and the detection report shows that "The number of created objects has reached the upper limit. Suggestion: Delete unused objects and then create new ones.", the number of LUNs, hosts, initiators, or mapping views has reached the upper limit. Delete the LUNs, hosts, initiators, or mapping views that are no longer used on DeviceManager and perform detection on the snapshot again.

Restoration

Context

This operation enables you to use a ransomware detection snapshot to restore data.

Precautions

You are advised to select a ransomware detection snapshot in Uninfected status for restoration.
  1. Click the number of ransomware detection snapshots to display the Ransomware Detection Snapshot page.
  2. Locate the row that contains the target ransomware detection snapshot and choose More > Restore.

Modifying a retention policy

Context

This operation enables you to reset the retention period of a ransomware detection snapshot.

Precautions

After modification, the retention period of the ransomware detection snapshot is calculated from the current time.
  1. Click the number of ransomware detection snapshots to display the Ransomware Detection Snapshot page.
  2. Click the snapshot time. On the basic information page of the ransomware detection snapshot, choose More > Modify Retention Policy.
    NOTE:

    You can set the retention period to be permanent or to xx days, xx weeks, xx months, or xx years.

Deleting a ransomware detection snapshot

Context

This operation enables you to delete a ransomware detection snapshot that is unavailable or no longer needed.

Precautions

  • Before deleting a ransomware detection snapshot, ensure that the snapshot is no longer used for restoration.
  • A ransomware detection snapshot that is in Uninfected status and configured with Uninfected Snapshot Lock cannot be deleted.
  • A ransomware detection snapshot in Detecting status cannot be deleted.
  • If the last uninfected copy of a resource is deleted, the accuracy of subsequent ransomware detection for copies will be affected. Exercise caution.
  1. Click the number of ransomware detection snapshots to display the Ransomware Detection Snapshot page.
  2. Locate the row that contains the target ransomware detection snapshot and choose More > Delete.
