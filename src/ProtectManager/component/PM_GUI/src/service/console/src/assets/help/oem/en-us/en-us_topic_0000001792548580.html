Restoring One or Multiple Tables in an HBase Backup Set

This section describes how to restore one or multiple tables in an HBase backup set.

Context

Data can be restored to the original or a new location using a backup copy. If you use a replication copy for restoration, the data can only be restored to a new location.

Procedure

  1. Choose Explore > Copy Data > Big Data > HBase.
  2. You can search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the backup set to be restored based on the backup set name and click the backup set name.

  3. Click Copy Data and select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in that month or day.

  4. Click . In the row where the copy resides, choose More > Table-level Restoration.
  5. On the Table-level Restoration page that is displayed, select the table to be restored on the left.
  6. On the right of the Table-level Restoration page, click Please select and select the target location to which you want to restore data.

    Table 1 describes the related parameters.

    Table 1 Restoring HBase tables

    Parameter

    Description

    Restore To

    Selects either Original location or New location.

    Target Cluster

    If you choose to restore the HBase table to New location, choose the target HBase cluster.

    Namespace

    If you choose to restore the HBase table to New location, you must choose the target namespace.

    Please select

    If a table exists in the restoration path, replace or skip the existing table.

    • Ignore the existing table: Tables with the same names are skipped and are not replaced.
    • Replace the existing table

  7. Click OK.
