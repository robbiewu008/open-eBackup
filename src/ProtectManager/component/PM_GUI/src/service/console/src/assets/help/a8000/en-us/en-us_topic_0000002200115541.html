Authorization

After a managed cluster is added to a local cluster, the system administrator of the local cluster has the system administrator permissions of the managed cluster. This section describes how to grant user permissions of a managed cluster to a local cluster user.

Local cluster users that support the authorization function include users of the Data Protection Administrator or custom roles.

Users of managed clusters include those of the Data Protection Administrator or custom roles.

Precautions

  • Batch authorization is not supported for local cluster users.
  • A local cluster user can be granted permissions of multiple managed clusters. Each managed cluster can grant the permissions of only one user of the Data Protection Administrator role to a local cluster user.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  1. In the multi-domain cluster area, choose More > Authorize in the row that contains the desired managed cluster.
  2. Select the local cluster user of the Data Protection Administrator or custom role for authorization.
  3. In the Authorization Information area, enter the name and password of the user of the Data Protection Administrator or custom role in the managed cluster.
  4. Click OK.
  5. In the displayed dialog box, confirm the information and click OK.

    After authorization is complete, you can click the authorized user of the managed cluster in the cluster list to view the authorization information.
