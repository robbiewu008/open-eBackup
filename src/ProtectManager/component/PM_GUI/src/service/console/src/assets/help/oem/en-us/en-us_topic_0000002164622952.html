Anonymization Rule Types

This section describes anonymization rule types supported by the product.

Table 1 describes the preset anonymization rule types.

Table 1 Preset anonymization rule types

Anonymization Rule Type

Description

Shuffling

Mixes up source data lines. This rule applies to any type of data.

Format-Preserving

Uses the format-preserving encryption algorithm to encrypt a character string. The range of ciphertext is specified by related parameters. This rule applies only to character data.

Full-Mask

Masks all sensitive information. The mask characters and mask length are specified by related parameters. This rule applies only to character data.

Partial-Mask

Partially masks sensitive information. The mask characters, locations of start and end characters, and mask length are specified by related parameters. This rule applies only to character data.

PII-Type

Replaces sensitive data with identified sensitive data name. This rule applies only to character data.

Numeric-Range

Replaces source data with a number randomly selected from a specific number range. The number range is specified by related parameters. This rule applies only to numeric data.

Fixed-Number

Replaces source data with a fixed number. The fixed number is specified by related parameters. This rule applies only to numeric data.

Noise-Adding

Adds or subtracts a random number to the source data. This rule applies only to numeric data.
