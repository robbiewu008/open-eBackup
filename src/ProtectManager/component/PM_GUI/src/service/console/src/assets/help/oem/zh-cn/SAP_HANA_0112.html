备份SAP HANA数据库时，备份子任务长时间无进度且已备份数据量长时间无变化

现象描述

备份SAP HANA数据库时，备份子任务长时间无进度，长时间停留在该子任务，且已备份数据量长时间未发生变化：“数据保护代理主机（xxx.xxx.xxx.xx）正在执行备份子任务（xxx-xxx-xxx-xx），已备份数据量：（xx）。”或“客户端（xxx.xxx.xxx.xxc）正在执行备份子任务（xxx-xxx-xxx-xx），已备份数据量：（xx）。”

可能原因

客户端所在主机硬件资源（内存、CPU、存储等）不足。

处理步骤

  • 方案一：

    增加硬件资源。您可以使用安装SAP HANA数据库后系统自带的Quick Sizer工具，评估主机运行时的硬件需求。通过分析用户输入的业务场景数据，生成对内存、CPU、存储等关键资源的建议配置后，根据分析结果补充所需的硬件资源。您可以访问quick-sizer获取相关的指导。

  • 方案二：

    调整global.ini的相关参数，优化系统的运行效率。相关的参数说明参见表1。

    在不改变硬件资源的条件下，仅通过调整global.ini的相关参数优化系统运行效率，无法确保新的备份任务能够成功。建议您通过方案一、增加所需的硬件资源后再重新进行备份。

    表1 参数说明

    名称

    说明

    默认值

    取值范围

    parallel_data_backup_backint_channels

    备份/恢复时并行使用的备份通道数。备份/恢复期间，此数量的通道将并行写入数据。该参数值越大，对硬件资源的要求越高。

    1

    1~32

    data_backup_buffer_size

    备份缓冲区大小（MB）。该参数值越大，占用的内存越多；该参数值越小，占用的CPU资源越多。

    512MB

    128~4096

    backint_pipe_capacity

    备份管道容量（Byte）。如果该值为0，则使用操作系统的默认容量。该参数值越大，占用的内存越多；该参数值越小，占用的CPU资源越多。

    0

    -
    1. 使用PuTTY，登录执行当前备份任务的客户端所在主机。
    2. 执行以下命令，切换到SAP系统用户。
      su - <sid>adm

      其中，“sid”为SAP系统ID。例如：

      su - s00adm
    3. 执行以下命令，登录系统数据库。
      hdbsql -i 实例id -n 系统数据库主节点的hostname:系统数据库的SQL Port -d SYSTEMDB -u 系统数据库用户名 -p 系统数据库密码

      如：

      hdbsql -i 00 -n localhost:30013 -d SYSTEMDB -u SYSTEM -p xxxxxxx
    4. 根据需要调整单个或多个参数的取值。
      • 设置parallel_data_backup_backint_channels：
        alter system alter configuration ('global.ini', 'SYSTEM') set ('backup', 'parallel_data_backup_backint_channels') = 'parallel_data_backup_backint_channels的值' with reconfigure;
      • 设置data_backup_buffer_size：
        alter system alter configuration ('global.ini', 'SYSTEM') set ('backup', 'data_backup_buffer_size') = 'data_backup_buffer_size的值' with reconfigure;
      • 设置backint_pipe_capacity：
        alter system alter configuration ('global.ini', 'SYSTEM') set ('backup', 'backint_pipe_capacity') = 'backint_pipe_capacity的值' with reconfigure;
