Managing CNware Registration Information

After the CNware virtualization environment is registered with the OceanProtect, you can perform operations on the CNware virtualization environment on the management page.

Log in to the management page and choose Protection > Virtualization > CNware. On the CNware page that is displayed, select the target CNware platform in the left pane.

Table 1 describes the related operations.

Table 1 Operations related to the CNware virtualization environment

Operation

Description

Navigation Path

Manually scanning the CNware virtualization environment

Scenario

This operation enables you to immediately update the changed VM information to the OceanProtect.

Note

If you do not select to apply the SLA policy to the newly created VM when creating a periodic backup job, the system will not associate the newly discovered VM with an SLA after scanning.

Choose More > Resource Scan.

Authorizing to protect CNware resources

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Note

  • Users cannot be authorized to protect resources that have been associated with SLAs. You need to remove the protection of resources before performing authorization.
  • A single resource can be authorized only to a single data protection administrator.

Choose More > Authorize Resource and select a data protection administrator to authorize.

Reclaiming CNware resources

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

Users are not allowed to reclaim resources that have been associated with SLAs. You need to remove the protection of resources before reclaiming them.

Choose More > Reclaim Resource.

Modifying the CNware registration information

Scenario

This operation enables you to modify CNware registration information, such as the name, port number, agent host, and certificate.

Note

Modify the CNware registration information when no backup or restoration job is running. Otherwise, the job will fail.

Choose More > Modify Registration.

Table 1 describes the parameters related to CNware registration information.

Unregistering CNware

Scenario

This operation enables you to remove a CNware virtualization environment when a host does not need to be protected or used as a restoration target.

Note

If a backup job is being executed, the unregistration will fail. Wait until the backup job is complete and try again.

Choose More > Unregister.
