即时挂载Oracle数据库

当需要将数据库的副本挂载至一个新位置以快速启动一个新数据库时，可参考本节配置即时挂载。

背景信息

  • 支持使用备份副本、复制副本、即时挂载产生的副本进行即时挂载。
  • 不支持挂载副本至原位置。

前提条件

挂载至的目标主机已安装ProtectAgent及其他相关软件。

具体操作可参考《ProtectAgent安装指南》。

操作步骤

  1. 如果存在同名数据库，请先关闭该同名数据库再执行挂载，否则会导致挂载失败。关闭数据库操作如下：

    1. 登录Oracle数据库主机。

      请使用PuTTY登录Oracle数据库主机，并执行su - oracle命令，切换到oracle账户。

    2. 执行以下命令，登录Oracle数据库实例。
      • OS认证：

        export ORACLE_SID=数据库实例名称

        sqlplus / as sysdba

      • 数据库认证：

        export ORACLE_SID=数据库实例名称

        sqlplus

        根据提示输入具备sysdba权限的用户：

        username as sysdba

        根据提示输入密码。

    3. 执行以下命令关闭同名数据库。

      shutdown immediate;

  2. 选择“数据利用 > 即时挂载 > Oracle”。
  3. 单击“创建”。
  4. 在“选择资源”页签，选择要挂载的副本对应的源数据库，单击“下一步”。

    • 如果资源类型是集群，则此处仅展示集群的管理IP，集群成员不会展示。
    • 如果主机或集群处于离线状态，则不会进行即时挂载，此处不展示离线状态的主机或集群。

  5. 在“选择副本”页签：

    1. 选择要挂载的副本。

      创建即时挂载的副本必须是“正常”状态。

    2. 选择挂载更新策略。

      如果未在此处选择挂载更新策略，后续可通过修改操作配置挂载更新策略。

    3. 单击“下一步”。

  6. 在“挂载选项”页签，配置挂载参数，单击“下一步”。

    1. 配置即时挂载基本参数和高级参数。
      相关参数说明如表1所示。
      • 带宽与IOPS可以任意配置一个或者两个都配置，如果是带宽型业务，建议只配置带宽，如果是IOPS型业务，建议只配置IOPS，如果不确定或者有时是带宽型、有时是IOPS型，则建议两个都配置，此时流量控制以小的一个为准。
      • 标准IOPS(8KB)与真实IO模型的换算关系可以通过界面上的换算表查看。
      • 通过限制挂载副本的最大流量，以保证备份、恢复任务的性能；设置挂载副本的下限保障，以保证挂载副本的最低性能。

      参数

      说明

      目标位置

      数据库副本挂载位置。

      您可以单击“添加目标位置”，同时挂载副本至多个主机或集群。

      说明：
      • 如果目标主机中存在同名数据库，但该数据库未处于运行状态，挂载后将覆盖原数据库。
      • 如果挂载副本是由主机产生，则不支持挂载至集群。
      原实例
      说明：

      仅1.6.0及后续版本存在此参数。

      目标实例
      说明：

      仅1.6.0及后续版本存在此参数。
      支持配置恢复到目标位置的实例名，如不配置，则与恢复前的原实例名保持一致。
      说明：

      如果配置了目标实例名，恢复后请使用目标实例名登录数据库。

      启动数据库

      即时挂载后是否启动数据库。

      如果选择不启动，当需要访问数据库时，请手动启动数据库。

      BCT功能
      • 如果选择开启BCT功能，即时挂载后会自动开启BCT功能。
      • 如果选择关闭BCT功能，即时挂载后会自动关闭BCT功能。

      带宽（MB/s）

      Min

      即时挂载副本的最小带宽。

      Max

      即时挂载副本的最大带宽。

      Burst

      即时挂载副本的最大突发带宽，Burst的取值必须大于Max。

      标准IOPS（8KB）

      Min

      即时挂载副本平均每秒标准I/O的最小请求个数。

      Max

      即时挂载副本平均每秒标准I/O的最大请求个数。

      Burst

      即时挂载副本的最大突发IOPS，Burst的取值必须大于Max。

      最大Burst时间(秒)

      当设置了带宽或标准IOPS的Burst参数，才需要设置该参数。表示突发带宽或突发IOPS的最大持续时间。

      标准化时延（8KB）

      Max

      在本地阵列的最大时延。

      挂载前执行脚本

      请输入脚本名称，脚本名称以.sh结尾。确保脚本已存放在数据库主机安装目录下的“DataBackup/ProtectClient/ProtectClient-E/sbin/thirdparty”目录中，且登录数据库主机的用户（默认为root）对该脚本有可执行权限。如果无执行权限请依次执行以下命令授权：

      chown root:root 脚本名称

      chmod 500 脚本名称

      说明：

      当配置了“挂载成功执行脚本”时，即使该脚本执行失败，管理界面上也会显示挂载任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。

      修改数据库配置

      根据挂载主机的实际配置修改数据库参数，参数错误将导致挂载失败。

      当Oracle集群的副本挂载至Oracle单机ASM时，请修改“*.cluster_database”的配置值为“false”。

      说明：

      恢复时，要求目标主机的空闲内存大小必须大于数据库SGA和PGA的大小总和，否则可能导致恢复失败。

      1. 查看目标主机空闲内存的大小。

        具体操作可参考2。

      2. 开启“修改数据库配置”开关后，搜索“*.pga_aggregate_target”和“*.sga_target”参数。

        检查空闲内存大小是否大于“*.pga_aggregate_target”与“*.sga_target”的总和。

        • 是，结束处理。
        • 否，执行3。
      3. 修改“*.pga_aggregate_target”与“*.sga_target”其中一个参数或两个参数都修改，使得空闲内存大小大于“*.pga_aggregate_target”与“*.sga_target”的总和。

        如果搜索到多个相同的参数，相同的参数均需修改。

      4. 如果修改了“*.pga_aggregate_target”，需要同步修改“实例名称.__pga_aggregate_target”的值，修改“实例名称.__pga_aggregate_target”为：

        实例名称.__pga_aggregate_target = *.pga_aggregate_target * 1024 *1024

        如果修改了“*.sga_target”，需要同步修改“实例名称.__sga_target”的值，修改“实例名称.__sga_target”为：

        实例名称.__sga_target = *.sga_target * 1024 *1024
    2. 单击“下一步”。

  7. 预览挂载参数，确认无误后，单击“完成”。

    • 如果需要对即时挂载的数据库进行远程访问，请先配置Oracle数据库的侦听参数。如：对即时挂载产生的副本进行数据脱敏前，需要先配置数据库的侦听参数。

      具体操作请参考配置Oracle数据库侦听参数。

    • 副本生成后立即用该副本执行即时挂载，可能会由于副本仍处于初始化状态而导致挂载任务失败，请3分钟后重试。
    • 如果目标数据库版本高于副本数据库版本，即时挂载过程中会自动触发升级数据库，请确保目标数据库归档日志路径有剩余空间，否则挂载任务将无法运行。
