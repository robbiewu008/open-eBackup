Object Attribute Restoration Fails When the Active Directory Recycle Bin Is Disabled

Some attributes may fail to be restored when the Active Directory Recycle Bin is disabled. Manual attribute modification has various impacts on the system and users.

Table 1 Attributes that fail to be restored and the impact of attribute modification

Attribute

Description

Impact

Common Name (CN)

Common name of an object, which is a part of the object RDN. RDN is short for relative distinguished name.

Modifying this attribute will change the name of the object, which may affect the applications or services associated with the object.

memberof

List of groups to which an object belongs.

If the groups to which an object belongs are not deleted, the memberof attribute of the object can be restored successfully. If the memberof attribute of a user changes, the permissions and access control of the user may be affected. Removing the group to which a user belongs may cause the user to lose permissions.

userParameters

User parameters.

Modifying this attribute affects user experience or operation methods. Exercise caution when performing this operation.

member

List of groups to which an object belongs. This attribute is the same as memberof.

Modifying this attribute will change the membership of an object.

msExchWhenMailboxCreated

Time when an Exchange mailbox was created.

Modifying this attribute during mailbox policy adjustment or data migration may affect the timestamps of subsequent operations.

mS-DS-CreatorSID

Security identifier (SID) of the user who creates an object, indicating which user or security principal (such as a computer account) creates the object.

This attribute is automatically managed by the system. Manual modification may cause incorrect information about the object creator. This may affect audit and permission management.

userAccountControl

User account management.

Modifying this attribute can change the behavior of a user account, for example, enabling or disabling an account, setting a password expiration policy, and allowing or forbidding login. An incorrect setting may cause user login failures or account locking.

badPasswordTime

Time when a user entered an incorrect password for the last time.

The attributes are automatically managed by the system. No impact is involved.

badPwdCount

Number of consecutive incorrect password attempts.

CanonicalName

Canonical name of an object, which is usually used for Lightweight Directory Access Protocol (LDAP) operations.

Created

Time when an object was created.

whenChanged

Time when an object was last modified.

ObjectGuid

Globally unique identifier (GUID) of an object.

groupType

Type of a group, such as a security group or a distribution group.

createTimeStamp

Timestamp when an object was created.

Deleted

Whether an object has been deleted.

DistinguishedName

Full path name of an object, which uniquely identifies the object.

dSCorePropagationData

Internal attribute used for replication. It records the replication metadata of an object.

instanceType

Object instance type, which is used to distinguish different Active Directory object types.

ObjectGUID

Unique GUID of an object, which is the same as ObjectGuid.

ou (OrganizationalUnit)

Organization unit to which an object belongs.

pwdLastSet

Last time when a user password was set.

isDeleted

Whether an object has been deleted. This attribute is the same as Deleted.

LastKnownParent

Last known parent container of an object.

lastLogoff

Last time when a user logged off.

lastLogon

Last time of user login.

logonCount

Number of user logins.

Modified

Time when an object was modified.

whenCreated

Time when an object was created. This attribute is the same as Created.

modifyTimeStamp

Time when an object was last modified. This attribute is the same as whenChanged.

nTSecurityDescriptor

Security descriptor of an object.

ObjectCategory

Object category, which is used to define the object type.

PropertyNames

List of object attribute names.

objectSid

Security identifier (SID) of an object.

primaryGroupID

Primary group SID of a user.

PropertyCount

Number of object attributes.

sAMAccountType

User account type.

sDRightsEffective

Permission effect of an object.

msNPAllowDialin

Whether dial-in is allowed.

ProtectedFromAccidentalDeletion

Whether to prevent an object from being deleted.

msDS-LastKnownRDN

Last known RDN of an object.

uSNChanged

Update sequence number, which records the number of times an object is modified.

lockoutTime

User lockout time.

Name

Name of an object.

lastLogonTimestamp

Timestamp of the last login of a user.

msExchMailboxGuid

GUID of an Exchange mailbox.

msExchMailboxSecurityDescriptor

Security descriptor of an Exchange mailbox.

mDBUseDefaults

Whether the default settings are used for the email database.

Procedure

  1. Open Active Directory Users and Computers.

    1. Log in to the remote host as Administrator.
    2. Press Win+R.
    3. Enter dsa.msc and press Enter.

  2. Modify attributes.

    1. Find the target user or group.
    2. Right-click the object and choose Properties from the shortcut menu.

    3. In the displayed dialog box, click the Attribute Editor tab.

    4. Double-click the attribute to be modified.
    5. Modify the attribute value and click OK.
