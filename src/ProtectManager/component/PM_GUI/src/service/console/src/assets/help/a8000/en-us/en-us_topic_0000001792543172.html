Global Search for Resources

This section describes how to perform global search for different resources.

Context

Global search for resources supports all applications connected to the OceanProtect. If only one of the applications is involved, you can ignore the description of the other one.

Procedure

  1. Click in the upper right corner of the page. The Global Search page is displayed.

    1. Select Resources in the search box.
    2. Enter a resource name and click . The resource details are displayed.

  2. In the navigation tree on the left, you can filter the displayed results again as required.

    The logic relationship between parameters is OR. That is, the search results filtered based on either one or more selected conditions will be displayed.

    Table 1 Parameter description

    Parameter

    Description

    Resource Type

    Select the resource type to be searched for.

    Protection Status

    Whether a resource is currently protected.

    • Unprotected: The resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
    • Protected: The resource has been associated with an SLA and protection has been enabled.
    • Creating: The resource has been associated with an SLA and protection is being created.

    SLA Compliance

    Filter data based on whether the SLA is complied with.

    Resource Location

    Enter the location of the resource.

  3. After the search is complete, the resource search result is displayed on the right of the page. Table 2 describes the related parameters.

    Table 2 Resource parameters

    Parameter

    Description

    Name

    Resource name.

    Type

    Resource type.

    Location

    Location of a resource.

    SLAs

    Whether a resource has been associated with an SLA.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a resource is currently protected.

    • Unprotected: The resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
    • Protected: The resource has been associated with an SLA and protection has been enabled.
    • Creating: The resource has been associated with an SLA and protection is being created.

  4. Locate the row that contains the target resource, and click More to perform operations such as protection and modification on the resource.
