Managing File Name Extension Filtering Rules

ProtectManager allows you to associate with vStores, disassociate from vStores, and delete file name extension filtering rules. When an operation is performed, the system scans the file systems of the vStore related to the operation to ensure that the operation is applied to all file systems of the vStore.

You need to log in to ProtectManager, choose Data Security > Ransomware Protection > File Interception, and locate the desired file name extension.

Table 1 describes the related operations.

Table 1 Operations related to file name extension filtering rules

Operation

Description

Navigation Path

Viewing file name extension filtering rules

Context

This operation enables you to view information such as file name extensions and vStores associated with file name extensions.

--

Associating with a vStore

Context

This operation enables you to apply a file name extension filtering rule to a vStore. After a vStore is associated, the associated file name extension filtering rule is selected by default when you set a file name extension filtering rule for the vStore.

Choose More > Associate with vStore.

Disassociating from a vStore

Context

This operation enables you to disassociate a file name extension filtering rule from a vStore.

Choose More > Disassociate from vStore.

Deleting a file name extension filtering rule

Context

This operation enables you to delete a file name extension filtering rule that is no longer needed.

Precautions

Preset file name extension filtering rules cannot be deleted.

Choose More > Delete.
