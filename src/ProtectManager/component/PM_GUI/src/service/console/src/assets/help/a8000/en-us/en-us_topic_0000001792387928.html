About Global Search

Definition

The OceanProtect provides the function of searching for protected resources to quickly find all objects (such as VMware VMs) in the protected environment. It also provides the function of searching for files and directories in backup copies. In addition, the global copy data search capability provides basic capabilities for file-level restoration and file download.

Global Search for Copy Data

Global resource search is to search for resources based on the recorded resource metadata. This section describes the service process of global search for copy data.

The OceanProtect supports global search for backup copies of FusionCompute/VMware/Huawei Cloud Stack/HDFS/NAS file systems/NAS shares/object storage/filesets/OpenStack/volumes/CNware. Indexes can be created either online or offline for search content to implement quick search. Online indexing indicates that an index is created immediately after a backup copy is generated. Offline indexing indicates that users can manually select a copy to create an index after the backup is complete.

The following part describes the process of creating an index for a VMware backup copy (as shown in Figure 1).

Figure 1 Service flow of global search for VMware copy data

  1. After a VMware backup job is completed or an index creation job is manually triggered on the management page, the management system schedules the job and sends it to the data enablement engine.
  2. The data enablement engine creates a clone for the file system snapshot where the backup copy resides and mounts the clone to the OceanProtect.
  3. The data enablement engine scans the directories and file lists of the virtual disk files in the clone.
  4. The data enablement engine generates an index based on the scanning result and inserts the index into the search engine Elasticsearch.

    After the index is created, users can search for and display backup copies or files in the copies on the GUI.
