步骤4：MySQL/MariaDB数据库开启日志模式

MySQL/MariaDB在执行备份任务前，需要开启MySQL/MariaDB数据库日志模式，否则会导致资源接入失败。

操作步骤

  1. 使用root用户，登录MySQL/MariaDB数据库。
  2. 执行以下命令查看MySQL/MariaDB数据库是否开启归档模式。

    show variables like 'log_bin';

    回显如下，“Value”为“ON”，则表示已开启日志模式。

    +-------------+-----+
|Variable_name|Value|
+-------------+-----+
|   log_bin   |  ON |
+-------------+-----+
1 row in set(0.00 sec)

  3. 执行以下命令查看MySQL/MariaDB数据库的日志模式是否为“ROW”。

    show variables like 'binlog_format';

    回显如下，“Value”为“ROW”，则表示日志模式为“ROW”。

    +-------------+-----+
|Variable_name|Value|
+-------------+-----+
|binlog_format| ROW |
+-------------+-----+
1 row in set(0.00 sec)

  4. 如果未开启日志模式或日志模式未设置为“ROW”，请执行该步骤，否则请忽略。

    1. 登录MySQL/MariaDB数据库实例所在主机。
    2. 执行vi /etc/my.cnf命令，打开my.cnf配置文件，并在配置文件末尾添加将以下内容。
      log_bin = mysql_bin
binlog_format = ROW
server-id=1

      在集群场景下，请确保各个节点server-id取值不同。

    3. 执行重启命令，重启MySQL服务。

      例如，执行service mysqld restart命令，重启MySQL服务。
