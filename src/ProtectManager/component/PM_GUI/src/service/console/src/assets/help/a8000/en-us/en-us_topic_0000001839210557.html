Manually Executing Backup

If you want to perform a backup immediately, you can manually perform the backup.

Context

Copies generated by manual backup are retained for the period defined in the SLA.

Prerequisites

  • The TDSQL database has been installed in the /data/tdsql_run directory.
  • The NTP service has been enabled for the cluster and the clock source configurations are the same.
  • On the Instance management page of the TDSQL CHITU management console, select an instance and choose Backup & Recovery > Backup settings. Set Number of days to save binlog and cold standby of the instance to be backed up to the maximum value.
  • The database vendor does not support the backup of distributed instances whose DR mode is 1+0 active/standby. Therefore, ensure that the cold standby node has been selected for the TDSQL distributed instance to be backed up before the backup. For configuration details, select an instance on the Instance management page of the TDSQL CHITU management console, go to the Set management page, and click the set name to view the current cold standby node.

Precautions

  • The following DDL operations cannot be performed during the backup of TDSQL distributed instances: CREATE, DROP, ALTER, and TRUNCATE.
  • To back up the TDSQL distributed instances, do not mount the /tdsqlbackup directory.

Procedure

  1. Choose Protection > Databases > TDSQL.
  2. On the Non-distributed Instances or Distributed Instances tab page, select the instance to be backed up.
  3. Choose more > manual backup.
  4. Set the name of the copy generated during manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  5. Select a protection policy, which can be Full backup, Forever Incremental (Synthetic Full) Backup, or Log Backup.

    TDSQL distributed instances do not support Forever Incremental (Synthetic Full) Backup.

  6. Click OK.
