恢复Exchange数据库

当需要对已备份的数据库进行恢复时，可以参考本节使用副本恢复数据库至原位置或新位置。

前提条件

  • 执行恢复前，请确保Exchange Server服务和相关的依赖服务正常运行。对应服务可参考Exchange 服务器上的 Exchange 服务概述。
  • 执行恢复前，请确保本产品系统时间与客户端时间保持一致，否则可能导致日志恢复后数据丢失。

注意事项

  • 恢复至原位置或新位置时，请确保目标位置空间足够，否则会恢复失败。
  • 恢复前需要对待恢复的数据库进行卸载（支持人工或系统自动卸载）。
  • 日志副本不支持数据库用户级恢复。

数据库恢复操作步骤

  1. 选择“数据利用 > 副本数据 > 应用 > Exchange”。
  2. 以数据库资源维度或以副本维度查找副本。本节以数据库资源维度为例进行说明。

    在“资源”页签，根据数据库名称查找到需要恢复的数据库，并单击数据库名称。

  3. 依次选择年、月、天找到副本。

    当日历上显示，即表示该月、该天存在副本。

  4. 指定时间点。
  5. 在副本所在行，单击“更多 > 恢复”。

      恢复完成后，目标数据库将处于卸除状态。开启该开关后，会自动装入目标数据库。

      数据库名称

      设置恢复后的数据库新名称。

      恢复前执行脚本

      仅对于单机的数据库恢复，可以设置“恢复前执行脚本”、“恢复成功执行脚本”、“恢复失败执行脚本”。

      您可根据实际情况，在恢复任务执行前、或恢复任务执行成功后、或恢复任务执行失败后，执行用户自定义的脚本，脚本类型为bat。若将脚本放置在DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty目录下，则只需填写脚本名称，若将脚本放置在其他目录下，则需要填写脚本绝对路径。
      说明：

      当配置了“恢复成功执行脚本”时，即使该脚本执行失败，本产品上也会显示恢复任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。

数据库用户级恢复

  1. 选择“数据利用 > 副本数据 > 应用 > Exchange”。
  2. 支持以数据库资源维度或以副本维度查找副本，本节以数据库资源维度为例进行说明。

    在“资源”页签，根据数据库名称查找到需要恢复的数据库，并单击数据库名称。

  3. 依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在行，单击“更多 > 用户级恢复”。

    • 恢复至原位置
      1. 查看并确认目标单机/可用性组。
      2. 选择需要恢复的邮箱。
      3. 设置高级配置参数。

        您可根据实际情况，在恢复任务执行前、或恢复任务执行成功后、或恢复任务执行失败后，执行用户自定义的脚本，脚本类型为bat。若将脚本放置在DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty目录下，则只需填写脚本名称，若将脚本放置在其他目录下，则需要填写脚本绝对路径。

后续操作

恢复后，需要在生产端更新用户邮箱状态，并创建新的Active Directory域用户与恢复后的邮箱建立连接，以验证恢复后的用户邮箱数据。具体操作请参考验证恢复后的用户邮箱数据（适用于Microsoft Exchange Server 2010）或验证恢复后的用户邮箱数据（适用于Microsoft Exchange Server 2013及后续版本）。
