Managing Clients

You can perform operations on clients, such as modifying, deleting, or authorizing clients on the management page.

You need to log in to the management page, choose Protection > Clients > Clients, and find the desired client.

Table 1 describes the related operations.

The GUI varies depending on the version.

Table 1 Client-related operations

Operation

Description

Navigation Path

Updating the client

Scenario

This operation enables you to install the client of another version on a host.

Note

  • Before updating the client, collect client logs for future system maintenance.
    Assume logs are stored in the default installation directory of the client. The log storage path varies with the protected resource.
    • Window OS

      The log file is stored in the C:\DataBackup\ProtectClient\ProtectClient-E\log\Plugins\GeneralDBPlugin directory. Copy the log file to a directory other than C:\DataBackup\ProtectClient.

      NOTE:

      Before updating the client, ensure that the log files related to the old package are closed.

    • Non-Windows OS

      Run the sh /opt/DataBackup/ProtectClientcollectlog.sh command to collect client logs.

      By default, the log files whose names start with sysinfo are stored in /opt/DataBackup/ProtectClient/ProtectClient-E/stmp in the client installation directory. Move the log files to a non-/opt directory.

  • During the client update, the installation directory of the client cannot be changed. To change the installation directory, uninstall the client and register it again.
  • During the client update, the client cannot execute protection jobs.
  • If the client update fails, the system automatically performs rollback.
  • If the rollback upon a client update failure fails, you can download the client software package from the management page and manually install it.

    If the client status remains Offline 5 minutes after the client update is complete, the update and rollback both fail.

  • If the firewall is enabled for the protected resource and the Nginx listening port is enabled, and the client is offline after the client is updated, the port may be occupied by another process during the client update and the system allocates another Nginx listening port.

    In this case, check and enable the Nginx listening port by referring to Enabling Input and Output Functions of the Nginx Port (Applicable to Some Resources).

  • Before updating the client, if the protected resource is OpenStack, ensure that the domain name resolution configuration is correct. For details, see Configuring Domain Name Resolution (Applicable to OpenStack).
  • If you have set aliases for rm, cp, mv, and other commands that need to be input in interactive mode on the Linux host where the client is to be installed, cancel the alias setting first. Otherwise, the installation will fail.

    For example, after you run the alias command on the host to view the configured command aliases, an alias is configured for the mv command if command output similar to alias mv='mv -i' is displayed. In this case, run the unalias mv command to cancel the alias configuration of the mv command.

In the row of the target client, choose More > Update Client.

NOTE:

For 1.6.0 and later versions: You can click Pending Update above the client list to filter all clients to be updated.

Synchronizing trap information

Scenario

If you have configured SNMP trap notification on the management page and want the alarm information of a host where the client is installed to be reported to the trap server through SNMP, you can synchronize the SNMP trap configuration to this host. If the SNMP authentication information or trap server information is modified on the management page, you need to synchronize the information.

Note

  • Only Oracle hosts support trap information synchronization.
  • For details about this operation, see Synchronizing Trap Configuration to an Oracle Host in the OceanProtect DataBackup 1.5.0-1.6.0 Data Backup Feature Guide (for Oracle).

In the row of the target client, choose More > Sync Trap Info.

Configuring LAN-free mode

NOTE:

Only OceanProtect X series backup appliances support this parameter.

Scenario

Fibre Channel/IP hybrid networks are supported between the host where the client is installed and the OceanProtect. After configuration, the backup and restoration data is transmitted over the Fibre Channel network to improve backup and restoration efficiency. This operation enables you to use the Fibre Channel network between the host where the client is installed and the OceanProtect.

Note

  • This function is supported only when the host where the client is installed runs the Linux OS and the IP network is used as the management network.
  • This function can be configured only when source deduplication is enabled for the client.
  • The WWPN of the host where the client is installed and that of the OceanProtect must be in the same zone. For details about how to configure zones, see "Configuring Zones" in the Installation Guide specific to your product model.
  • A Fibre Channel card has been installed on the OceanProtect. For details, see "Connection Planning Between the Controller Enclosure and Application Server" in the Installation Guide specific to your product model.
  • A Fibre Channel card must have been installed on the host where the client is installed. A maximum of two Fibre Channel ports can be configured in LAN-free mode. For details about the supported Fibre Channel card models, access Specifications Query.
  • If all WWPNs of the hosts where the client is installed are offline or no Fibre Channel port of the OceanProtect is connected, backup and restoration jobs will fail.

In the row that contains the target client, choose More > Configure LAN-Free.

  1. Enable LAN-Free.
  2. Select the target node.
  3. Select the WWPNs and FC ports of the detected hosts where the client is installed.
    NOTE:

    You can click Add to manually add WWPNs of the hosts where the client is installed.

  4. Click OK.
Performing resource authorization
NOTE:

Only 1.5.0 supports this function.

Scenario

You can grant the data protection permission to another data protection administrator through the system administrator.

Note

  • Permissions on resources that have been associated with SLAs cannot be granted. Remove protection before authorization.
  • The permission on a single resource can be granted to only one data protection administrator.

In the row of the target client, choose More > Authorize Resource, and select the data protection administrator to grant resource permissions.

Modifying an application type

Scenario

This operation enables you to modify the application type that can be protected by the client currently.

Note
  • The application type can be modified only when the client status is Online and the client type is General backup client.
  • The client version must be the same as the version of the OceanProtect.
In the row of the target client, choose More > Modify Application Type.
NOTE:

Click the client name. On the Overview page that is displayed, you can view the application types that can be protected by the client currently.
Performing resource reclamation
NOTE:

Only 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

The protection permissions on a resource that has been associated with an SLA cannot be reclaimed. Before reclamation, remove the protection.

In the row of the target client, choose More > Reclaim Resource.

Performing resource scan

Scenario

The OceanProtect automatically triggers resource scan every hour. If you need to immediately update the information about a plug-in that is running properly on the host where the client is installed so that the client can be selected for the corresponding resources for protection, you can manually scan resources.

Note

Client scan is supported only for clients whose status is Online.

Choose More > Resource Scan.

Modifying a host

Scenario

This operation allows you to change the status of Auto-Sync Host Name and the host name.

Note

The host name cannot be changed when Auto-Sync Host Name is enabled.

In the row of the target client, choose More > Modify Host.

Deleting a host

Scenario

If you do not need to use the host where the client is installed to perform protection jobs, you can delete it.

Note

  • A host that has been associated with an SLA cannot be deleted.
  • Only hosts in the Offline state can be deleted.

In the row of the target client, choose More > Delete Host.

Setting remarks

Scenario

This operation enables you to set remarks for the client.

Choose More > Set Remarks.

Exporting logs

Scenario

This operation enables you to export client logs.

Note

  • The system and data protection administrators cannot export logs of offline clients.
  • Logs of a maximum of 10 clients can be exported at a time. A maximum of 1.5 GB log files of a single client can be exported.
  • The new log export record overwrites the old one.
    • For 1.5.0: The system retains only one client log export record for each user.
    • For 1.6.0 and later versions: The system retains the latest 10 client log export records for each user.
  • An auditor cannot export logs or download collected logs.

Select the clients whose logs need to be exported and choose More > Export Logs in the upper area of the page.

Setting the log level

Scenario

This operation enables you to configure the client log level.

Choose More > Configure Log Level.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to add a tag to the client for quick filtering and management.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Client management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.

Choose More > Add Tag.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove a client tag.

Choose More > Remove Tag.
Synchronizing backup network configuration
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

After you modify the backup network configuration of the OceanProtect, this operation allows you to synchronize the backup network configuration to all clients.

In the upper part of the client list, click Synchronize Backup Network Configuration.
