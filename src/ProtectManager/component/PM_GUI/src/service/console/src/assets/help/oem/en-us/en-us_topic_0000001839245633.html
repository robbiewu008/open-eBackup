Viewing Environment Information of the SQL Server Database

After a SQL Server database is registered with product, you can view details about the database, the host where it resides, and the cluster or availability group to which it belongs on the management page.

Procedure

  1. Choose Protection > Databases > SQL Server.
  2. View the database information on the Databases tab.

    The information about the scanned database is displayed.
    1. Table 1 describes the database information.
      Table 1 Database information

      Parameter

      Description

      Name

      Database name.

      Owning Host/Cluster

      Name of the host or cluster where the database resides.

      Owning Instance

      Name of a single instance or cluster instance.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
      • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

      Protection Status

      Indicates whether the database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is enabled.
      • Creating: The database is associated with an SLA and protection is being created.
    2. Click the name and check the overview, copy information, and jobs.
      • Overview

        The SLA information, job execution time, and total number of copies are displayed.

      • Copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, a copy at that time exists.

      • Job

        Backup and recovery jobs are displayed.

  3. On the Cluster tab page, view the SQL Server cluster information.

    If a database on a cluster has been registered, the cluster information is displayed on the page.

    Table 2 describes the displayed cluster information.
    Table 2 Cluster information

    Parameter

    Description

    Name

    Name of a database cluster.

    • If you click the cluster name, host members and all authenticated databases in the cluster are displayed.

    Status
    Cluster status.
    • Online: When a host is powered on and ProtectAgent on the host is running properly, the Status of the host is Online.

      If there are online hosts in a cluster, the cluster status is Online.

    • Offline: When a host is shut down or ProtectAgent on the host is faulty, the Status of the host is Offline.

      If all hosts in a cluster are offline, the status of the cluster is Offline.

    Owner

    Data protection administrator of a cluster.

    If a data protection administrator is authorized for the cluster, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
