Step 5: Registering the VMware Virtualization Environment

Before performing VMware backup and restoration, you must register the VMware virtualization environment with the OceanProtect.

Context

vCenter Servers or ESXi hosts can be registered when you register the VMware virtualization environment with the OceanProtect. If there are multiple independent ESXi hosts, you need to register them one by one.

After registering the VMware virtualization environment, you can view information about vCenter Servers, ESXi hosts, or VMs on the management page and protect or restore them.

Procedure

  1. Choose Protection > Virtualization > VMware.
  2. Click Register, and register the VMware virtualization platform.

    Table 1 lists the VMware registration information.
    Table 1 VMware registration information

    Parameter

    Description

    Name

    User-defined virtualization platform name.

    IP Address
    • If the ESXi host is managed by a vCenter Server, enter the management IP address of the vCenter Server.
    • If the ESXi host is not managed by a vCenter Server, enter the management IP address of the ESXi host.

    Port

    Port number used by the OceanProtect to communicate with the management plane of the vCenter Server or ESXi host.

    The default port number is 443.

    Username
    • When a vCenter Server is registered, enter the username and password for logging in to the VMware vSphere Web Client.

      For details about minimum permission requirements, see Minimum Permissions for Users Performing VMware Virtualization Environment Registration.

    • When an ESXi host is registered, enter the username and password for logging in to the ESXi host.

    Password

    Resource Scan Interval

    Interval for the system to automatically scan resources

    After the frequency is set, the system automatically scans the VMware virtualization environment based on the frequency and updates the changed VM to the OceanProtect.

    Certificate Verification

    This function is enabled by default. When you register a VMware protected environment, the system verifies the information about the accessed devices. After this function is disabled, security risks may exist in the communication between the system and VMware. Exercise caution when performing this operation.

    Certificate

    After Certificate Verification is enabled, import the certificate obtained in Step 4: Obtaining the VMware Certificate.

    Revocation List

    The CRL is issued by the certificate authority. After the CRL is imported, the system checks whether the client certificate has been revoked. If the certificate has been revoked, cluster registration fails.

    Secure TLS Versions and Algorithms

    This function is disabled by default and TLS 1.3 is used by default. After this function is enabled, communication parties negotiate with each other to determine the protocol to be used.

  3. To perform storage-layer backup, perform this step to add storage resources. Otherwise, skip this step.

    1. In the Storage Resources area, click Add.
    2. Configure parameters as required. Table 2 describes the parameters.
      Table 2 Parameters for adding storage resources

      Parameter

      Description

      Type

      Type of a storage resource. The value can be OceanStor Dorado 6.x or NetApp ONTAP.

      NOTE:

      Only 1.6.0 and later versions support NetApp ONTAP.

      Management IP Address

      Management IP address of the storage device.

      You can click Add to enter multiple management IP addresses of the storage device. If a management IP address cannot be accessed, the system automatically switches to another accessible management IP address.

      Port

      Port number for communication between the OceanProtect and the management plane of the storage device.

      Username

      Username and password of the storage device. The user role must be an administrator or a super administrator.

      Password

      Certificate Verification

      This function is enabled by default. Certificate verification is performed when a storage device is added, ensuring the security of interactions between the OceanProtect and storage device. After this function is disabled, security risks may exist in the communication between the system and the storage device. Exercise caution when performing this operation.

      Certificate

      To-be-imported CA certificate of the storage device. For details about how to obtain the CA certificate of a storage device, see "Step 1: Obtaining the CA Certificate of a Storage Device" in the OceanProtect Appliance 1.5.0-1.6.0 Data Backup Feature Guide (for NAS Files).

      Revocation List

      The CRL is issued by the certificate authority. After the CRL is imported, the system checks whether the client certificate has been revoked. If the certificate has been revoked, storage device registration fails.

  4. Click OK.

    After the registration is successful, information about the registered VMware virtualization environment is displayed on the left of the page. You can view vCenter Servers or ESXi hosts and their VM lists.
