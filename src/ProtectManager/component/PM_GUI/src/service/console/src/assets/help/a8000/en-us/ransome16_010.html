Hyper-V VM Copies

This section describes how to perform ransomware detection on Hyper-V VM copies.

Prerequisites

An anti-ransomware policy has been created for the resource.

Precautions

  • Linux and Windows OSs are supported.
  • The following can be detected: ext2, ext3, ext4, and XFS Linux file systems, and NTFS, FAT, and FAT32 Windows file systems.
  • Encrypted volumes do not support detection.
  • If the storage device is upgraded from 1.2.0 through a fast upgrade, upgrade the kernel version of the storage device to its current version by following instructions in the OceanProtect Backup Storage X.X.X Upgrade Guide. X.X.X indicates the product version.

Performing Detection

In addition to copy detection based on scheduled ransomware protection policies, you can perform manual detection on all copies of individual resources. To perform manual detection, perform the following steps:

  1. Choose Data Security > Ransomware Protection > Detection Overview.
  2. On the Detection Overview page, select Hyper-V VM from the Select Resource Type drop-down list.
  3. Locate the row of the target resource and choose More > Manually Detect.

    In the displayed dialog box, select the copies to be detected and click OK.

    Alternatively, you can locate the row of the target resource and click the number of copies. In the displayed window on the right, choose More > Manually Detect on the right of the target copy.

Viewing the Detection Report

  1. Choose Data Security > Ransomware Protection > Detection Overview.
  2. On the Detection Overview page, select Hyper-V VM from the Select Resource Type drop-down list.
  3. Locate the row that contains the target resource and click the number of copies.
  4. In the displayed window, choose More > Detection Report on the right of a copy. The detection report displays the detection details and suspicious file list.
  5. (Optional) Handle false detection.

    If an uninfected copy is detected as Infected, check whether file information, such as the file owner, changes based on the suspicious file list in the detection report. If the file information does not change, false detection occurs. In this case, you can correct the ransomware detection status of the copy by handling false detection.

    1. Locate the row that contains the target resource and choose More > Handle False Detection in the Operation column on the right.
    2. In the displayed dialog box, confirm the information and click OK. The system changes the copy state from Infected (incorrect detection) to Uninfected.

    False detection handling is supported only for copies in the Infected state.
