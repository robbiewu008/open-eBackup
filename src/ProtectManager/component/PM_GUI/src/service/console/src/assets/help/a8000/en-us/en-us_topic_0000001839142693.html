Step 1: Enabling MySQL/MariaDB Database Permission

Before executing a MySQL/MariaDB backup job, you need to enable the permission of the corresponding MySQL/MariaDB version. Otherwise, registration, backup, or restoration will fail. The MySQL 8.0.11 database and openEuler 22.03 are used as examples in this section. The commands may vary depending on the database and OS versions. Contact O&M personnel or database administrators for the commands.

Procedure

  1. Log in to the MySQL/MariaDB host as user root and run the mysql -uroot -pPassword command to log in to the MySQL/MariaDB database.
  2. Run the CREATE USER 'name'@'127.0.0.1' IDENTIFIED BY 'password'; command to create a user.
  3. Run the following commands in sequence to enable the remote login permission of 127.0.0.1 of the MySQL/MariaDB database:

    1. Run the GRANT ALL PRIVILEGES ON *.* TO 'name'@'127.0.0.1' WITH GRANT OPTION; command to enable the remote login permission of 127.0.0.1.
    2. Run the FLUSH PRIVILEGES; command to update the remote login permission of MySQL/MariaDB.

  4. Run the following command to enable the permission of the corresponding database version for the user:

    • MySQL 5.x:
      GRANT RELOAD, PROCESS, REPLICATION CLIENT, SUPER, LOCK TABLES  ON *.* TO 'name'@'127.0.0.1';
FLUSH PRIVILEGES;
    • MySQL 8.x:
      GRANT BACKUP_ADMIN, SELECT, RELOAD, PROCESS, SUPER, LOCK TABLES  ON *.* TO 'name'@'127.0.0.1';
FLUSH PRIVILEGES;
    • MariaDB 10.5/10.6/10.7:
      GRANT RELOAD, PROCESS, BINLOG MONITOR, SUPER, LOCK TABLES ON *.* TO 'name'@'127.0.0.1';
FLUSH PRIVILEGES;
    • MariaDB 10.3/10.4:
      GRANT RELOAD, PROCESS, REPLICATION CLIENT, SUPER, LOCK TABLES ON *.* TO 'name'@'127.0.0.1';
FLUSH PRIVILEGES;
    • MariaDB 10.2:
      GRANT USAGE ON *.* TO 'name2'@'127.0.0.1' IDENTIFIED BY 'password';
GRANT RELOAD,PROCESS,SUPER,REPLICATION CLIENT,LOCK TABLES ON *.* TO 'name'@'127.0.0.1';
FLUSH PRIVILEGES;
