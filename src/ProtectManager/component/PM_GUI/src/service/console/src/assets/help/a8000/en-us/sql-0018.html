Step 9: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA. The system protects the resources and periodically executes backup jobs based on the SLA. You can perform a backup job immediately through manual backup.

Precautions

  • During a manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies and , are applied to the manual backup job.
  • Copies generated by manual backup are retained based on the SLA.

Procedure

  1. Choose Protection > Databases > SQL Server.
  2. On the Databases tab page, select the database to be protected and click Protect.

    You can also select multiple databases for batch protection.

  3. Select an SLA created by a user.

    You can also click Create to create an SLA.

    • SQL Server does not support the preset SLAs (including Gold, Silver, and Bronze).
    • If a WORM policy has been configured for the resource to be protected, select an SLA without a WORM policy to create protection to avoid WORM policy conflicts.

  4. Set parameters in Advanced by referring to Table 1.

    Table 1 Advanced backup parameters

    Parameter

    Description

    Script to Run Before Backup

    You can configure Script to Run Before Backup, Script to Run upon Backup Success, and Script to Run upon Backup Failure based on actual requirements. When a message indicating that the pre-processing task starts to be executed is displayed during a backup job, the configured script is executed in the corresponding scenario.

    Enter a script name that ends with .bat. Ensure that the script has been stored in Installation directory\DataBackup\ProtectClient\ProtectClient-E\bin\thirdparty and the user who logs in to the database host has permissions to execute the script.

    NOTE:

    If Script to Run upon Backup Success is configured, the status of the backup job is displayed as Succeeded on the management page even if the script fails to be executed. Check whether the job details contain a message indicating that a post-processing script fails to be executed. If yes, modify the script in a timely manner.

  5. Click OK.

    If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  6. Perform manual backup.

    If you want to perform a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. Choose More > Manual Backup in the row where the target database is located.

      You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

    2. Set the name of the copy generated by manual backup.

      If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup, Differential Backup, or Log Backup.

      For 1.6.0 and later versions, if the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    4. Click OK.
