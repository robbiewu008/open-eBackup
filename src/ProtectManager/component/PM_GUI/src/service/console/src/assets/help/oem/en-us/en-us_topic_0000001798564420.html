Live Mounting Volumes

This section describes how to live mount a volume copy to the original or a new location to quickly start a new volume.

Context

ProtectAgent and other related software have been installed on the target host to which the copy is mounted.

For details, see the ProtectAgent Installation Guide.

Procedure

  1. Choose Explore > Live Mount > Volume.
  2. Click Create.
  3. On the Select Resource tab page, search for or select the resource corresponding to the copy to be mounted, and click Next.
  4. On the Select Copy tab page, perform the following operations:

    1. Select the copy to be mounted.
      • The copy to be live mounted must be in the Normal status.
      • The copy generated after part of disks of a VM are backed up cannot be live mounted.
    2. Select the created mount update policy.

      If the mount update policy is not selected in this step, you can configure the mount update policy by modifying mount information later.

    3. Click Next.

  5. On the Mount Options tab page, set mounting parameters and click Next.

    1. Configure basic and advanced parameters for live mount.
      Table 1 describes the related parameters.
      • You can configure either or both of Bandwidth (MB/s) and Normalized IOPS (8 KB). You are advised to configure only Bandwidth (MB/s) for bandwidth-intensive services and only Normalized IOPS (8 KB) for IOPS-intensive services. If you cannot determine the service type or the service type changes frequently, you can configure both parameters and the system will control the traffic using the parameter with a smaller value.
      • You can query the conversion between the standard IOPS (8 KB) model and real I/O model in the conversion table on the GUI.
      • The maximum traffic of mounted copies is limited to ensure the performance of backup and restoration jobs. In addition, the minimum traffic of mounted copies is set to ensure the minimum performance of mounted copies.
      Table 1 Live mount parameters

      Parameter

      Description

      Target Host

      Target host to which the copy is mounted.

      You can search for a host name or IP address to select the host.

      NOTE:
      • The live mount path cannot be selected, and is /mnt/databackup/volumelivemount/Copy ID/volumes by default.
      • Ensure that the mount directory is not a system directory or directory of another application. Otherwise, the system cannot run properly.
      • Ensure that the mount directory is not mounted. Otherwise, the live mount job will fail.

      Bandwidth (MB/s)

      Min

      Minimum bandwidth for copy live mount

      Max

      Maximum bandwidth for copy live mount

      Burst

      Maximum burst bandwidth for copy live mount. The value of Burst must be greater than that of Max.

      Burst can be set only after Max is set.

      Standard IOPS (8 KB)

      Min

      Average minimum number of standard I/O requests per second for copy live mount.

      Max

      Average maximum number of standard I/O requests per second for copy live mount

      Burst

      Maximum burst IOPS for copy live mount. The value of Burst must be greater than that of Max.

      Burst can be set only after Max is set.

      Max. Burst Duration (s)

      This parameter needs to be set only when the Burst parameter of the bandwidth or standard IOPS is set. It indicates the maximum duration of burst bandwidth or burst IOPS.

      Normalized Latency (8 KB)

      Max

      Maximum latency of the local storage system
    2. Click Next.

  6. Preview and confirm the mounting parameters, and click Finish.

    If a copy is used for live mount immediately after it is generated, the live mount job may fail because the copy is still in the initialization status. In this case, try again in 3 minutes.
