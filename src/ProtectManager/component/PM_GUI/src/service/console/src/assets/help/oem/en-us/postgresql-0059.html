Managing PostgreSQL

The product allows you to modify, remove, activate, or disable protection for PostgreSQL databases.

Related Operations

You need to log in to the management page, choose Protection > Databases > PostgreSQL, click the Instance tab, and locate the target database.

Table 1 describes the related operations.

Table 1 Database protection operations

Operation

Description

Navigation Path

Protect

Scenario

This operation enables you to protect a database instance.

In the row of the target database, choose More > Protection.

Modifying protection

Scenario

This operation enables you to modify a custom script. After the modification, the next backup will be executed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • If the SLA associated with a resource changes, you are advised to modify the current SLA by choosing Protection > Protection Policies > SLAs. You are advised not to associate the resource with another SLA by modifying the protection.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

Choose More > Modify Protection in the row where the database is located.

Removing protection

Scenario

This operation enables you to remove the protection plan when protection on the database is not required. After the removal, the system cannot execute protection jobs for the database.

Note

  • When a protection job of the database is running, the protection cannot be removed.
  • If the SLA associated with a resource changes, you are advised to modify the current SLA by choosing Protection > Protection Policies > SLAs. You are advised not to associate the resource with another SLA after removing the protection.

    After you remove protection and associate the resource with another SLA, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

Choose More > Remove Protection in the row where the database is located.

Manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup in the row where the database is located.

Disabling protection

Scenario

This operation enables you to disable a protection plan when periodic protection on the database is not required. After the protection plan is disabled, the system does not execute the automatic backup job of the database.

Note

Disabling protection does not affect ongoing backup jobs.

Choose More > Disable Protection in the row where the database is located.

Activating protection

Scenario

This operation enables you to activate the disabled protection plan of a database. After the protection plan is activated, the system backs up the database based on the protection plan.

Note

You can perform this operation only on a database that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection in the row where the database is located.

Scanning resources

Scenario

If the information about a registered database changes, for example, the database version changes, you can perform this operation to scan the database and update the information. Otherwise, the execution of backup and restoration jobs may be affected.

In the row of the target database, choose More > Resource Scan.

Testing connectivity

Scenario

This operation enables you to test the connectivity between the product and database.

In the row of the target database, choose More > Test Connectivity.

Modifying database information

Scenario

This operation enables you to modify information such as the database name and database authentication method.

In the row of the target database, choose More > Modify.

Deleting database information

Scenario

This operation enables you to delete a database that is no longer used.

Note

Before deleting a database, remove protection from the database.

In the row of the target database, choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for restoring Elasticsearch backup sets or a table in a backup set is similar to that in Restoration and is not described here.
