Step 5: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Prerequisites

  • Before the backup, you need to ensure that the pexpect package (pexpect 3.2 recommended) has been installed on the agent host. For details about how to download and install pexpect, see "Installing pexpect" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.
  • Before the backup, ensure that GBase 8a databases are running properly and the network connection of each node in the cluster is normal.
  • During the backup, the passwords of the GBase 8a agent host and database users cannot be changed.

Context

  • During the execution of a manual backup job, parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, Job Failure Alarm, and Source Deduplication, take effect.
  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Databases > General Databases.
  2. Select the database to be protected and click Protect.
  3. Select an SLA. You can select a preset SLA or click Create to create one.

    If a WORM policy has been configured for the resource to be protected, select an SLA without a WORM policy to create protection to avoid WORM policy conflicts.

  4. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  5. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row of the target database, choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup.

    2. Set the name of the copy generated through manual backup.

      If this parameter is left empty, the system sets the name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full backup or Incremental backup.

      For 1.6.0 and later versions: If the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    4. Click OK.
