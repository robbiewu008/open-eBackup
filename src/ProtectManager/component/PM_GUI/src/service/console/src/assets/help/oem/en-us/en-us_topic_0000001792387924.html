Managing VMware Registration Information

After the VMware virtualization environment is registered with the product, you can perform operations on the VMware virtualization environment on the management page.

Log in to the management page and choose Protection > Virtualization > VMware. On the VMware page that is displayed, select the desired vCenter Server or ESXi in the left pane.

Table 1 describes the related operations.

Table 1 Operations related to the VMware virtualization environment

Operation

Description

Navigation Path

Modifying the VMware registration information

Scenario

You can perform this operation to change the name of VMware registration information. If the IP address, port number, or CA certificate of a registered vCenter Server or ESXi host is changed, you need to perform this operation to synchronize the change.

Note

Modify the VMware registration information when no backup or restoration job is running. Otherwise, the job will fail.

Choose More > Modify Registration.

Table 1 describes the parameters related to VMware registration information.

Canceling VMware registration

Scenario

If you do not need to protect a vCenter Server or ESXi host or use it as a restoration target, you can perform this operation to remove the VMware virtualization environment.

Note

If a backup job is being executed, the VMware virtualization environment fails to be deregistered. Wait and try again until the backup job is complete.

Choose More > Unregister.

Manually scanning the VMware virtualization environment

Scenario

This operation allows you to immediately update the changed VM information to the product.

Note

If you do not select to apply the SLA policy to the newly created VM when creating a periodic backup job, the system will not associate the newly discovered VM with an SLA after scanning.

Choose More > Resource Scan.
