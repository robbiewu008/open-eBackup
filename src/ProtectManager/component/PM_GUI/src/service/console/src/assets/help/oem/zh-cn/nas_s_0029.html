步骤9：执行备份

执行备份前将需要保护的资源关联到指定SLA策略，系统将对这些资源按照SLA策略进行保护，并根据SLA策略周期性自动执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

注意事项

  • 对于通过设备注册发现的NAS共享，如果NAS共享有认证信息，已参考（可选）步骤3：配置NAS共享信息（适用于OceanStor V5/OceanStor Pacific/NetApp ONTAP）完成配置，否则将导致备份失败。
  • 对于NFS共享，请确认当前设备具备NFS共享目录的读写权限，并且开启了no_root_squash选项。配置NFS共享访问权限的详细操作，请参见步骤4：配置访问权限（适用于OceanStor V5/OceanStor Pacific/NetApp ONTAP）。
  • 对于CIFS共享，请确认当前设备具备CIFS共享目录的读写权限，推荐使用Administrator组中的用户进行备份，否则只能备份具有读写权限的文件和目录。配置CIFS共享访问权限的详细操作，请参见步骤4：配置访问权限（适用于OceanStor V5/OceanStor Pacific/NetApp ONTAP）。
  • 手动备份时，SLA中定义的参数（“失败后自动重试”除外），如“限速策略”会应用到手动备份任务中。
  • 手动备份产生的副本保留时长按照SLA中定义的保留时长进行保留。
  • 如果SLA定义了复制策略或归档策略，则执行手动备份时，系统会按照SLA的定义执行一次复制或归档。

操作步骤

  1. 选择“保护 > 文件系统 > NAS共享”。

    对于1.5.0版本，请选择“保护 > 文件服务 > NAS共享”。

  2. 在需要保护的NAS共享所在行，单击“更多 > 保护”。
  3. 选择已创建的SLA。

    您也可以单击“创建”，创建新的SLA。

    如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  4. 设置高级配置参数。相关参数说明如表1所示。

    表1 高级配置参数说明

    参数

    说明

    通道数

    执行恢复任务时，在客户端上能够同时执行的子任务数，NAS共享的通道数默认为10，最大可设置为40。

    通道数越大，备份性能越好，相应占用客户端的CPU和内存越多，每个通道占用约200MB内存，同时也会提高对NAS共享的访问速率。

    对于1.5.0版本，客户端部署到本产品时，由于容器资源有限，建议适当调低通道数的配置。

    小文件聚合

    开启后，系统将同一目录下的多个小文件聚合为大文件，提升备份效率。在SLA包含了增量备份的场景下，若SLA未设置永久增量（合成全量）备份，则不支持关闭小文件聚合；若SLA设置了永久增量（合成全量）备份，则不支持开启小文件聚合。若资源开启了小文件聚合，则不支持对该资源的副本执行勒索软件检测。单目录下子目录数+文件数超过100万时，不建议开启聚合。

    需要设置以下两个参数。

    • 聚合后文件大小：文件聚合后的大小，可以设置为1024KB和4096KB，推荐值为4096KB。
    • 最大的被聚合文件大小：被聚合文件的最大值，可以设置为128KB、256KB、1024KB和4096KB，推荐值为1024KB。

    小文件聚合应满足：聚合后文件大小 >= 最大的被聚合文件大小

    SMB 硬链接保护

    默认开启。当待保护的NAS共享数据集同一目录下的文件数量过多时，备份任务可能失败。在此场景下，建议关闭此开关。

    SMB ACL保护

    默认开启。当待保护的NAS共享数据集同一目录下的文件数量过多时，备份任务可能失败。在此场景下，建议关闭此开关。

    热数据备份

    开启后，仅备份文件修改时间在“指定时间点”之后的文件。指定时间点：根据备份时间点向前推进设置的时间后得到的历史时间点。

    冷数据备份

    开启后，仅备份文件修改时间在“指定时间点”之前的文件。指定时间点：根据备份时间点向前推进设置的时间后得到的历史时间点。

    安全归档

    说明：
    • 仅1.6.0及后续版本支持此功能。
    当选择的SLA包含归档策略时，显示该参数。勾选后仅归档防勒索检测未感染的备份副本。
    说明：

    如果SLA归档策略中“归档时间”未选择“备份成功后立即执行归档”或“归档策略”未选择“仅归档最新的副本”，以及并未设置防勒索策略时，请单击“前往配置”为待保护资源设置防勒索策略。具体操作请参考《数据备份特性指南(副本防勒索)》中的“创建防勒索&WORM策略”章节。

    备份副本自动索引

    设置备份副本自动索引开关。

    默认关闭，当选择虚拟机或虚拟机组进行保护，且备份的磁盘包含了系统盘时，开启后，系统会对备份副本自动创建索引。使用该副本进行恢复时，支持选择副本中的单个或多个文件进行恢复，支持搜索副本中的文件、目录或链接。

    说明：
    • 仅1.6.0及后续版本支持此参数。

    对象存储归档副本自动索引

    设置对象存储归档副本自动索引开关。

    默认关闭，当归档存储类型选择为“对象存储”时，显示该参数。开启后不支持关闭。开启后，系统自动为副本创建文件索引，创建索引后，支持恢复单个或多个文件。您也可以为副本手动创建索引。

    说明：
    • 仅1.6.0及后续版本支持此参数。

    磁带归档副本自动索引
    • 设置磁带归档副本自动索引开关。

      默认关闭，当归档存储类型选择为“磁带库”时，显示该参数。当需要查看归档副本内的详细数据，或者需要节省恢复时占用的存储空间，此时可以开启开关。开启后，系统自动为副本创建文件索引，创建索引后，支持恢复单个或多个文件。

      说明：
      • 仅1.6.0及后续版本支持此参数。
      • 关闭磁带归档副本自动索引，磁带副本将无法直接恢复到生产存储，会额外在本产品创建临时文件系统，占用备份存储空间，当恢复任务结束后，该临时文件系统会被自动清理回收。

  5. 单击“确定”。

    如果当前系统时间已晚于SLA首次备份的开始时间，可以在弹出的对话框中选择立即执行手动备份或者根据SLA中设置的备份策略周期性自动执行备份。

  6. 可选：执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。
    1. 在需要备份的资源所在行单击“更多 > 手动备份”。

      支持选择多个资源批量进行手动备份。勾选多个已保护的资源，在资源列表左上角选择“更多 > 手动备份”。

    2. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    3. 选择保护策略。
      • 如果NAS共享保护未配置“小文件聚合”，保护策略可设置为：全量备份或永久增量（合成全量）备份。
      • 如果NAS共享保护已设置“小文件聚合”，保护策略可设置为：全量备份或增量备份。

      对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    4. 单击“确定”。
