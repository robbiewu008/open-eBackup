管理Windows ADFS配置（适用于1.6.0及后续版本）

本产品支持使用ADFS登录的方式登录管理界面。

前提条件

  • 已配置DNS服务，具体操作请参考配置DNS服务。
  • 已登录配置了ADFS服务器的Windows主机，获取配置Windows ADFS所需的相关参数，如表1所示。

操作步骤

  1. 选择“系统 > 安全 > Windows ADFS配置”。
  2. 单击Windows ADFS配置右侧按钮，开启“Windows ADFS配置”。

    当“Windows ADFS配置”已开启时，单击右上角“修改”进行设置。

  3. 设置Windows ADFS配置参数，相关参数说明如表1所示。

    表1 Windows ADFS配置参数

    参数名称

    参数说明

    参数获取

    提供商

    提供商名称，默认填写且不可修改。

    默认参数

    名称

    自定义Windows ADFS配置名称。

    自定义

    提供商URL

    配置ADFS服务后生成的ADFS登录页面地址。

    提供商的URL格式为https://ADFS域名/adfs/ls/IdPInitiatedSignonPage.htm

    在ADFS的生产主机上获取ADFS域名和ADFS的Local地址。

    1. 启动Server Manager工具。
    2. 在Server Manager界面上选择“Local Server”。
    3. 在PROPERTIES界面上查看对应的“computer name”、“domain”和“Ethernet0”的值。ADFS域名为“Computer name”.“domain”，ADFS的Local地址为“Ethernet0”。

    在本地登录的本产品的PC上配置hosts文件，如果本地PC能访问提供商的URL就无需配置。

    1. 进入目录“C:\Windows\System32\drivers\etc”。
    2. 打开“hosts”文件。
    3. 在文件中配置以下内容后保存。
      ADFS的Local地址 ADFS域名

    在本地登录的本产品的PC上访问ADFS服务器供应商URL，如果能正常访问无需进行以下配置。

    1. 打开本地PC的“设置”。
    2. 选择“网络和Internet”>“代理”。
    3. 如果“使用代理服务器”的状态为“开”，需要设置代理；状态为“关”，则不执行4和5。
    4. 在“手动设置代理”区域中添加以下内容，域名间用英文分号分隔开。
      ADFS域名
    5. 单击“保存”。

    回调URL

    用于供应商认证后通过回调此地址登录本产品。

    默认参数

    在ADFS的生产主机上配置“回调URL”。

    1. 启动Server Manager工具。
    2. 单击菜单“Tools”>“AD FS Management”。
    3. 在"AD FS"界面上，单击“AD FS”>“Application Groups”。
    4. 在“Application Groups”区域中双击对应的应用组进入应用组的属性界面。
    5. 在“Applications”区域中选择对应服务器应用程序，单击"Edit"。
    6. 在“Server application Properties”的对话框中将“回调URL”输入到“Redirect URI”，点击"Add"，单击"OK"。

    客户端ID

    ADFS中生成的客户端ID。

    在ADFS的生产主机上查询客户端ID。

    1. 启动Server Manager工具。
    2. 单击菜单“Tools”>“AD FS Management”。
    3. 在"AD FS"界面上，单击“AD FS”>“Application Groups”。
    4. 在“Application Groups”区域中双击对应的应用组进入应用组的属性界面。
    5. 在“Applications”区域中选择对应服务器应用程序，单击"Edit"。在“Server application Properties”的对话框中“Client Id”的值 为客户端ID。

    客户端密钥

    ADFS中生成的客户端密钥。

    联系ADFS服务器管理员获取。

    说明：

    请妥善保管添加应用组时设置的客户端密钥，Windows ADFS配置时需要使用到该客户端密钥。

    CA证书

    单击，选择需要添加的证书对应的CA证书。

    在ADFS的生产主机上下载CA证书。

    1. 在键盘上按“Win+R”，打开运行窗口。
    2. 输入“mmc”，单击“OK”，进入“Console1”控制台界面。
    3. 选择“File”>“Open”，打开证书文件。
    4. 选择“Console Root”>“Certificates(Local Computer)”>“Personal”>“Certificates”，找到对应的CA证书，证书为xx-xx-CA。
    5. 双击CA证书，在Certificate界面选择“Details”。
    6. 选择“Copy to File...”>“Next”，在Certificate Export Wizard界面上选择“No,do not export the private key”，单击“Next”。
    7. 选择“Base-64 encoded X.509(.CER)”，单击“Next”。
    8. 单击“Browse...”，选择下载的本地路径，并输入文件名，单击“保存”。
    9. 选择“Next”>“Finish”。
    10. 在本地保存的路径下找到CA文件并修改证书的后缀为“.pem”。
      说明：

      修改Windows的文件后缀需要关闭“隐藏已知文件类型的扩展名”。

  4. 单击“测试”，测试Windows ADFS配置与本产品的连通性。
  5. 单击“保存”。

    在使用Windows ADFS登录前，需创建与ADFS服务器上同名的ADFS用户（用户名@“domain”）用于后续登录，参见创建用户。
