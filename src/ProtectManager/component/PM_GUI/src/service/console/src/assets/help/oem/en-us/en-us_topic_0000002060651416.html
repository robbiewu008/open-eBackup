Configuring a Windows Server to Exit the Directory Services Repair Mode

After the system is restored, the Windows server where the domain controller resides needs to exit the DSRM.

Procedure

  1. Log in to the Windows server where the domain controller resides.
  2. In the cmd window, run the bcdedit /deletevalue safeboot command as the system administrator. After the command output indicating successful command execution is displayed, run the shutdown/r command to restart the Windows server to exit the DSRM.
