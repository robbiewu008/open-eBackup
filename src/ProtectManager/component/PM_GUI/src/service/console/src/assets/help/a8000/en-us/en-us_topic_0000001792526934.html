Importing a Certificate

This section describes how to update certificates through certificate importing.

Context

The ProtectAgent certificate is used to ensure communication security between ProtectAgent and the OceanProtect. The server certificate is used to ensure communication security when the OceanProtect receives external access requests. The internal communication certificate and internal database certificate are used only to ensure internal communication security among internal components of the OceanProtect.

Important Notes

To ensure the security when the management IP addresses or domain names of multiple controllers are used to access the storage system, ensure that the CSR file contains the management IP addresses or domain names of these controllers. After obtaining the certificate file from the CA, open the certificate file in the Windows operating system. On the Details tab page, check whether the value of Subject Alternative Name contains the domain names or management IP addresses of multiple controllers. The following example indicates that thtest.spe02.com and thtest2.spe02.com can be used to access the storage system securely.

Procedure

  1. Choose System > Security > Certificate.
  2. Click More on the right of a certificate.

    • ProtectAgent Certificate/Server Certificate/Internal communication Certificate/Internal database Certificate
      1. Select Import Certificate.
      2. Set certificate information.

        Table 1 lists the parameters.

        Table 1 Certificate parameters

        Parameter

        Description

        Remarks

        CA Certificate

        Click and select the CA certificate file corresponding to the certificate to be imported.

        NOTE:
        • The size of the CA certificate file to be imported cannot exceed 1 MB.
        • The content of the CA certificate file to be imported must be in x.509 format, and the file name extension must be .pem.
        • If the CA certificate file is not the root one, add a certificate chain file. For details about how to create a certificate chain file, see Creating a Certificate Chain File. A maximum of three levels of CAs are supported for the server certificate and ProtectAgent certificate.
        • The internal communication certificate or internal database certificate supports only one level of CA.

        Applicable certificate types:

        • Server Certificate
        • ProtectAgent Certificate
        • Internal communication Certificate
        • Internal database Certificate

        Server Certificate

        Click and select the server certificate file to be imported.

        NOTE:
        • The size of the certificate file to be imported cannot exceed 1 MB.
        • The content of the certificate file to be imported must be in x.509 format, and the file name extension must be .pem.
        • The common name (CN) of the server certificate must be different from that of the CA certificate.

          Check method: Open the server certificate in CER format and check whether the value of Issued to (CN of the server certificate) is different from the value of Issued by (CN of the CA certificate) on the General tab page.

        • The common name (CN) of the ProtectAgent server certificate must be OceanProtect-AGENT.
        • For the internal communication certificate and internal database certificate, SubjectAltName (SAN) must be configured for the server certificate, and SAN must contain DNS:*.dpa.svc.cluster.local.

        Applicable certificate types:

        • Server Certificate
        • ProtectAgent Certificate
        • Internal communication Certificate
        • Internal database Certificate

        Server Private Key

        Click and select the private key file corresponding to the server certificate file to be imported.

        If the server certificate is generated by the CA based on the request file exported from the OceanProtect, you do not need to set this parameter.

        NOTE:
        • The size of the private key file cannot exceed 1 MB.
        • The extension of the private key file must be .pem.
        • The private key file must be encrypted. If your private key file is in plaintext, perform operations by referring to Encrypting the Plaintext Private Key File.

        Applicable certificate types:

        • Server Certificate
        • ProtectAgent Certificate
        • Internal communication Certificate
        • Internal database Certificate

        Server Private Key Password

        Import the password of the server private key file.

        If the server certificate is generated by the CA based on the request file exported from the OceanProtect, you do not need to set this parameter.

        [Value range]

        The value contains 1 to 512 characters.

        NOTE:

        For the internal communication certificate, the password must contain 8 to 64 characters, including digits, uppercase letters, lowercase letters, and special characters.

        Applicable certificate types:

        • Server Certificate
        • ProtectAgent Certificate
        • Internal communication Certificate
        • If the certificate to be imported is a server certificate or ProtectAgent certificate, the system will push the CA certificate to all member nodes after the certificate is imported.
        • In the remote replication scenario, after replacing the server certificate of the source or target end, you need to use the certificate issued by the same CA certificate to replace the server certificate of the other end. Otherwise, remote replication will fail.
        • If ProtectAgent has been installed and the new server certificate and client certificate are not issued by the same CA certificate, when you update the server certificate, you need to replace the client certificate after the server certificate is updated. For details, see Replacing the SSL Certificate of ProtectAgent on the Client (Non-Windows OS).
        • Only certificates whose signature algorithm is SHA-256, SHA-384, or SHA-512 can be imported.
    • External certificate
      External certificate types include Email, Object Storage, External Storage, Backup/Multi-domain/Replication Cluster, LDAP, and HCS IAM.

      OceanProtect E6000 does not support the Backup/Multi-domain/Replication Cluster certificate type.

      1. Select Import Certificate.
      2. Set certificate information.

        Table 2 lists the parameters.

        Table 2 Certificate parameters

        Parameter

        Description

        CA Certificate

        Click and select the CA certificate corresponding to the certificate to be imported.

        NOTE:
        • The size of the CA certificate to be imported cannot exceed 1 MB.
        • The content of the CA certificate to be imported must be in x.509 format, and the file name extension must be .pem.
        • If the CA certificate is not the root one, add a complete certificate chain file. For details about how to create a certificate chain file, see Creating a Certificate Chain File. The external certificate supports a maximum of ten levels of CAs.

  3. Click OK.
