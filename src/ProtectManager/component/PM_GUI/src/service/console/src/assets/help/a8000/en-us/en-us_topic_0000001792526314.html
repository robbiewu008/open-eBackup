Step 3: Registering the SQL Server Database

After ProtectAgent is installed on the host where the SQL Server database resides, the OceanProtect automatically discovers the SQL Server database host. You need to register the database on the management page to protect it.

Prerequisites

Before backing up or restoring a database, you need to add the login credential of the database to the OceanProtect. Windows Authentication and SQL Server Authentication modes are supported.

  • When Windows Authentication is enabled on the database, the SQL Server database is automatically registered with the OceanProtect, and the authentication status of the database is automatically displayed as Authenticated on the management page.
  • If Unauthenticated is displayed on the management portal, the authentication mode of the newly registered database is Database authentication. In this case, you need to enter the authentication information.

Registering a Single SQL Server Database Instance

  1. Choose Protection > Databases > SQL Server.
  2. Click the Instance tab.
  3. Click Register to register the SQL Server database.
  4. Set Type to Single instance.
  5. Configure the database instance and its authentication information.

    Table 1 describes the related parameters.

    Table 1 SQL Server single-instance registration information

    Parameter

    Description

    SQL Server Network Name

    Network name of the SQL Server. The default value is the host name.

    Named Instance

    Instance name.

    Hosts

    Database host to be registered.

    Authentication Method

    The instance authentication mode can be OS authentication or database authentication.

    Database Username

    Name of the database administrator.

    Database Password

    Password of the database administrator.

  6. Click OK.

Registering a SQL Server Cluster Instance

Prerequisites

Before registering a SQL Server cluster instance, ensure that each database host in the cluster has been registered.

The names of databases with which all hosts in the cluster are registered must be the same.

Procedure

  1. Create a cluster.

    1. On the SQL Server page, click the Cluster tab.
    2. Click Register.
    3. On the Register page, customize the cluster name and select the cluster node.
    4. Click OK.

  2. Register a cluster instance.

    1. On the SQL Server page, click the Instance tab.
    2. Click Register.
    3. Set Type to Cluster instance.
    4. Configure the database instance and its authentication information.

      Table 2 describes the related parameters.

      Table 2 SQL Server cluster instance registration information

      Parameter

      Description

      SQL Server Network Name

      Network name of the SQL Server. The default value is the host name.

      The SQL Server network name of a Windows Server Failover Cluster (WSFC) is customized during cluster creation.

      Named Instance

      Cluster instance name

      Cluster

      Database cluster to be registered.

      Nodes

      Host where the database instance resides.

      Authentication Method

      The cluster instance authentication mode can be OS authentication or database authentication.

      Database Username

      Name of the database administrator.

      Database Password

      Password of the database administrator.
    5. Click OK.

Related Operations

  • If the cluster is of the WSFC type, select all nodes.

  • If the cluster is of the availability group type, register cluster instances one by one.
