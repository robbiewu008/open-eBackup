Restoring Files in a VMware VM

This section describes how to restore files on a VM by using the file-level restoration function.

Context

  • The OceanProtect supports file-level restoration using backup copies and replication copies. Note that data cannot be restored to the original location when a replication copy is used.
  • The types of Linux file systems that can be restored are ext2/ext3/ext4 and XFS. The types of Windows file systems that can be restored are NTFS, FAT, and FAT32.

Prerequisites

  • The IP address of the backup network plane has been configured for the VM to which files are to be restored.
  • If the target VM runs the Windows OS, ensure that ports 445 and 138 have been enabled on the target VM. If the target VM runs the Linux OS, ensure that port 22 has been enabled on the target VM.
  • If the target VM runs on the Windows OS, ensure that the CIFS service has been enabled on the VM. For details, see the Microsoft official website.
  • If the target VM runs on a non-Windows OS, ensure that rsync has been installed and started on the target VM.
  • Ensure that VMware Tools has been installed on the target VM. For details, see Step 1: Checking and Installing VMware Tools.

Procedure

  1. Choose Explore > Copy Data > Virtualization > VMware.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    Click Resources to switch to the corresponding page, search for the VM to be restored by name, and click the VM name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. Find the copy used for restoration and click Copy Time.
  5. In the dialog box that is displayed, click the File tab.
  6. Select the files to be restored and click Restore File.

    • File-level restoration is supported only when the index status of the copy is Indexed.
    • If automatic indexing has been enabled for the SLA associated with the VM corresponding to the copy, the index status of the copy is Indexed, and file-level restoration can be directly performed. If automatic indexing is disabled, click Manually Create Index and then perform file-level restoration.
    • If the copy does not contain OS data, no index can be created for the copy. Therefore, the copy cannot be used for file-level restoration.
    • Reverse replication copies and archive copies do not support indexes.
    • During VMware VM backup, the mount point information that is not written into the /etc/fstab file is stored in the memory. The OceanProtect does not back up the information. Therefore, no index is generated for the files at the mount point, and file-level restoration is not supported. To generate indexes for new file systems and support file-level restoration, write the mount point information of the new file systems to the /etc/fstab file.
    • You can also click Export to download files to the local host. The size of the file to be downloaded cannot exceed 1 GB. Linked files cannot be downloaded.
    • If the name of a folder or file contains garbled characters, file-level restoration is not supported. Do not select folders or files of this type. Otherwise, the restoration job fails.
    • If the target VM runs the Windows OS and only the SMBv1 service is enabled, the time of the restored file is the current VM time.
    • Select Original Location to restore data to, that is, restore data to the original directory of the original VM.
      Table 1 describes the related parameters.
      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      VM IP Address
      Select the configured backup network IP address.
      NOTE:
      • Before performing file-level restoration, you need to configure a backup network IP address for the VM. During file-level restoration, the system uses this IP address to connect to the target VM.
      • If the VM IP address is not in the list, exit the current page. Then, choose Protection > Virtualization > VMware and rescan the virtualization environment.

      VM Username
      During file-level restoration, the target VM needs to verify the login credential of the OceanProtect. Enter a username and its password for logging in to the target VM.
      • Windows OS: The default user name is Administrator.
      • Linux OS: The default user name is root.
      NOTE:

      The login user must have the read and write permissions on the directory to which data is to be restored on the target VM.

      VM Password

      Overwrite Rule
      If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.
      • Replace existing files
      • Skip existing files: Files with the same names are skipped and are not replaced.
      • Only replace the files older than the restoration file
        NOTE:

        If the access to the file with the same name on the target VM is denied, the file fails to be restored during file replacement.
    • Select a new location to restore data to, that is, restore data to a specified VM.

      Table 2 describes the related parameters.

      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Compute Location
      Select the target VM to which data is restored.
      NOTE:
      For 1.5.0 and earlier versions:
      • If the specified VM runs on a Windows OS, the directory to which data is restored is the same as the original directory of the file. If the VM does not have the corresponding drive letter, restoration will fail. For example, if the file path is D:\a, the a directory is created on drive D of the specified VM before file restoration is performed. If drive D does not exist on the specified VM, restoration will fail.
      • If the specified VM runs on a Linux OS, the directory to which data is restored is the same as the original directory of the file. During restoration, the corresponding directory is created on the specified VM before file restoration is performed

      Target Path

      NOTE:

      Only 1.6.0 and later versions support this parameter.
      • If no target path is not specified, data will be restored to the target VM based on the original path where the selected files are stored.
      • If the target path is specified, the files will be restored to the specified path on the target VM. Enter an absolute path.
        NOTE:
        • If the drive letter in the specified target path does not exist on the target VM running the Windows OS, the restoration will fail. For example, if you specify the file restoration path as D:\a, the a directory will be created on drive D of the specified VM before file restoration is performed. However, if drive D does not exist on the target VM, the restoration will fail.
        • If the target VM runs the Linux OS and the specified file restoration path is opt/b, the corresponding directory will be created on the target VM before file restoration is performed when the target path does not exist on the VM.

      VM IP Address
      Select the configured backup network IP address.
      NOTE:
      • Before performing file-level restoration, you need to configure a backup network IP address for the VM. During file-level restoration, the system uses this IP address to connect to the target VM.
      • If the VM IP address is not in the list, exit the current page. Then, choose Protection > Virtualization > VMware and rescan the virtualization environment.

      VM Username
      During file-level restoration, the target VM needs to verify the login credential of the OceanProtect. Enter a username and its password for logging in to the target VM.
      • Windows OS: The default user name is Administrator.
      • Linux OS: The default user name is root.
      NOTE:

      The login user must have the read and write permissions on the directory to which data is to be restored on the target VM.

      VM Password

      Overwrite Rule
      If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.
      • Replace existing files
      • Skip existing files: Files with the same names are skipped and are not replaced.
      • Only replace the files older than the restoration file
        NOTE:

        If the access to the file with the same name on the target VM is denied, the file fails to be restored during file replacement.

  7. Click Test and ensure that the target VM to be restored is properly connected to the OceanProtect.
  8. Click OK.
