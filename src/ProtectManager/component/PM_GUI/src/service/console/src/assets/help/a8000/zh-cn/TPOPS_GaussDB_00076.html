执行DN/CN重启或主备切换等操作，导致日志备份失败

现象描述

日志备份过程中，执行了DN/CN重启或主备切换等操作，日志备份失败。

可能原因

DN/CN重启或主备切换等操作导致xlog不完整，日志备份失败。

处理步骤

  1. 登录客户端所在的主机。
  2. 执行cd /opt/DataBackup/ProtectClient/ProtectClient-E/slog/Plugins/GeneralDBPlugin命令进入客户端所在的主机的日志目录（本步骤命令以客户端安装在“/opt”目录下为例，请按照实际客户端安装目录进行修改）。
  3. 执行以下命令搜索关键报错信息。

    grep -nr "GS-53429 : The recovery time is invalidxlog has been archived"

  4. 如果3能搜索到报错信息，表示xlog不完整，需要重新进行一次全量备份。
