Restoration of a Cluster Copy to a New Cluster Fails Because the Node that Delivers the Restoration Command Fails to Parse the Result.

Fault Symptom

A cluster copy fails to be restored to a new cluster.

Possible Cause

All GaussDB (DWS) node data has been successfully restored and the result has been returned to the node that delivers the restoration command. However, the node that delivers the restoration command fails to parse the result. As a result, the restoration job fails.

Suggestion

  1. Log in to the GaussDB (DWS) node that delivers the restoration command.
  2. Run the following commands in sequence to switch to the cluster user and load environment variables:

    su - Cluster username
    source Environment variable file path

  3. Run the cd $GAUSSLOG/ roach/controller/ command to go to the log file directory and check whether the log file whose name starts with roach_controller contains the following error information:

    Error: The return code for failed subprocess on xxx.xxx.xxx.xxx is 1

    Example:

    • If error information is displayed, perform the restoration job again.
    • If no error information is displayed, collect logs and contact technical support for assistance.
