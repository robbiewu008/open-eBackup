Managing Hive Clusters

This section describes how to modify or delete Hive clusters registered with the OceanProtect.

Related Operations

Log in to the management page, choose Protection > Big Data > Hive, click the Cluster tab, and locate the desired cluster.

Table 1 describes the related operations.

Table 1 Cluster-related operations

Operation

Description

Navigation Path

Authorizing resources

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Note

  • Users cannot be authorized to protect resources that have been associated with SLAs. You need to remove the protection of resources before performing authorization.
  • A single resource can be authorized only to a single data protection administrator.

In the row of the target resource, choose More > Authorize Resource. Then, select the desired data protection administrator to whom the resource is authorized.

Reclaiming resources

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

Users are not allowed to reclaim resources that have been associated with SLAs. You need to remove the protection of resources before reclaiming them.

In the row of the target resource, choose More > Reclaim Resource.

Performing connectivity tests

Scenario

This operation tests the connectivity between the OceanProtect and the protected environment.

In the row of the target resource, choose More > Connectivity Test.

Modifying cluster information

Scenario

This operation enables you to change the authentication mode, name and configuration file of a Hive cluster.

In the row where the cluster resides, choose More > Modify.

For details about Hive cluster parameters, see Step 3: Registering a Hive Cluster.

Deleting a cluster

Scenario

This operation enables you to delete a Hive cluster if you no longer need to protect files in it or restore data to it.

Note

  • A cluster cannot be deleted if a backup or restoration job is being executed.
  • A cluster cannot be deleted if its backup sets have been associated with an SLA.
  • Deletion of a cluster will delete all its backup sets.

In the row where the cluster resides, choose More > Delete.
