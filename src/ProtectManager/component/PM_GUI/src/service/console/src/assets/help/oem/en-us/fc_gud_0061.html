Querying Information About the FusionOne Compute Virtualization Environment

After the FusionOne Compute virtualization environment is registered with the product, you can view details about FusionOne Compute clusters, hosts, VMs, or VM groups on the management page.

Context

GUIs may vary with versions.

Procedure

  1. Choose Protection > Virtualization > FusionOne Compute.

    Figure 1 shows the overall information about the FusionOne Compute virtualization environment.

    Figure 1 Overall information about the FusionOne Compute virtualization environment

  2. On the left of the page, select the FusionOne Compute virtualization environment to be viewed and choose More > Environment Info to view the basic information and agent host of the FusionOne Compute virtualization environment.
  3. On the Clusters tab page, view the cluster information.

    1. For details about the cluster, see Table 1.
      Table 1 Cluster information

      Parameter

      Description

      Name

      Cluster name.

      SLAs

      If the entire cluster is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status
      Indicates whether a cluster is protected.
      • Unprotected: The cluster is not associated with an SLA, or it is associated with an SLA but protection is disabled.
      • Protected: The cluster is associated with an SLA and protection is enabled.
      • Creating: The cluster is associated with an SLA and protection is being created.

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the cluster name to view the cluster overview and related jobs.
      • Cluster overview

        Displays the cluster name, SLA information, host list, and VM list.

        The host list contains the host name, SLA, SLA compliance, and protection status.

        The VM list contains the VM name, status, SLA, SLA compliance, and protection status.

        Click Operation on the right of the host or VM name to protect or manually back up the host or VM or to modify, remove, activate or disable the protection for the host or VM.

      • Cluster job

        Displays all cluster-related jobs, whose types include Protect Resource, Modify Protection, and Copy Expiration. You can choose to display ongoing or historical jobs.

    2. Click the host name to view the host information and related jobs.
      • Host overview

        Displays the host name, SLA information, and VM list.

        The VM list contains the VM name, status, SLA, SLA compliance, and protection status.

        Click Operation on the right of the VM to protect or manually back up the VM or to modify, remove, activate or disable the protection for the VM.

      • Host task

        Displays all host-related jobs, whose types include Protect Resource, Modify Protection, and Copy Expiration. You can choose to display ongoing or historical jobs.

    2. Click the VM name to view the overall VM information, copy quantity, and related jobs.
      • VM overview

        You can view the name, SLA information, job execution time, total number of copies, and disk information of a VM.

      • VM copy data

        You can search for copies by year, month, or day.

        If is displayed below the time, a copy at that time exists.

      • VM job

        Displays all jobs related to the VM, including backup and restoration. You can choose to display jobs being executed or historical jobs.
