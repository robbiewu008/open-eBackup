Viewing MySQL/MariaDB/GreatSQL Database Environment Information

After a MySQL/MariaDB/GreatSQL database is registered with the OceanProtect, you can view details about the databases, clusters, and instances on the management page.

Procedure

  1. Choose Protection > Databases > MySQL/MariaDB/GreatSQL.
  2. View the details.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    • On the Clusters tab page, view the database cluster information.

      Information of registered clusters is displayed on the page.

      Table 1 describes the detailed information.
      Table 1 Host and cluster information

      Parameter

      Description

      Name

      Name of a database cluster.

      If you click the cluster name, all hosts in the current cluster and all authenticated databases in the cluster are displayed.

      Type

      Type of the cluster.

      Status
      Status of a host or cluster.
      • Online: If the host is powered on and the client on the host is running properly, Status is Online for the host.

        If there are online hosts in the cluster, the cluster status is Online.

      • Offline: If the host is shut down or the host where the client is installed is faulty, Status is Offline for the host.

        If all hosts in the cluster are offline, the cluster status is Offline.
      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of a host or cluster.

      If a host or cluster has been authorized to a data protection administrator, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
    • On the Instances tab page, view the database instance information.
      Table 2 describes the database instance information.
      Table 2 Database instance information

      Parameter

      Description

      Name

      Name of a database instance.

      Status

      Database instance status.

      Owning Host/Cluster

      Name of the host or cluster where the database instance resides.

      Database Type

      Type of the database where the instance is located.

      Version

      Version of the database where the instance is located.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether the database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      1. Click the name and check the overview, copy information, and jobs.
        • Overview

          Displays the SLA information, job execution time, and total number of copies.

        • Copy data

          You can search for copies by year, month, and day in sequence.

          If is displayed below the time, copies exist at that time.

        • Job

          Displays jobs of backup, restoration, and so on.

    • On the Databases tab page, view the database information.

      The registered databases are displayed on the page.

      Table 3 describes the database information.
      Table 3 Database information

      Description

      Owning host/cluster

      Instance

      Name of a database instance.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed.       • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      1. Click the name and check the overview, copy information, and jobs.
        • Overview

          Displays the SLA information, job execution time, and total number of copies.

        • Copy data

          You can search for copies by year, month, and day in sequence.

          If is displayed below the time, a copy at that time exists.

        • Jobs

          Displays jobs of backup, restoration, and so on.
