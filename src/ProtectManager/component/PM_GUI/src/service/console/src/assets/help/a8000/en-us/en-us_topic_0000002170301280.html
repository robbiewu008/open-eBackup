Step 2: Configuring Route Information for the Replication Network

If different replication ports of replication nodes are configured with IP addresses on the same network segment and the replication network uses multiple IP addresses for networking, you need to follow instructions in this section to configure policy-based routes for the replication network ports of all replication nodes in the local and remote storage systems to implement network connectivity and data balancing. Otherwise, skip this section.

Procedure

  1. Log in to DeviceManager of the local and remote storage systems separately.
  2. Choose Cluster > Network > Port and Route > Physical Port.
  3. Select the physical port of the replication node and click Configure Policy-based Route.
  4. On the Configure Policy-based Route page, configure the gateway of the selected physical port IP address.
  5. Click OK.
  6. After the configuration is complete, check whether the policy-based route is configured successfully.

    1. On the Physical Port page, click the name of the port for which a policy-based route has been configured.
    2. On the page that is displayed, click the Policy-based Route tab and check whether a policy-based route has been configured.
    3. Log in to the replication node using the management IP address, run the minisystem command to go to the minisystem view, and run the ping command to check whether each network port can properly communicate with the remote replication plane.
      Example:
      ping -I 192.168.10.31 XXX.XXX.XXX.XXX 
ping -I 192.168.10.32 XXX.XXX.XXX.XXX

      In the command, 192.168.10.31 indicates the replication service IP address configured for the local replication plane, and XXX.XXX.XXX.XXX indicates the replication service IP address of the remote replication plane.

      If the data returned by the remote replication plane can be received, the policy-based route configuration takes effect. Otherwise, check whether the default gateway configuration of the management plane and the policy-based route configuration of the local and remote replication planes are correct.
