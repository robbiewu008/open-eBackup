如何手动停止TDSQL恢复任务

在TDSQL恢复任务执行过程中如需手动停止任务，可请参考本章节操作。

操作须知

在恢复任务执行过程中停止恢复任务可能对生产端的数据库环境或数据产生影响，请谨慎操作。

操作步骤

  1. 使用root用户登录恢复的目标实例所有的数据节点。
  2. 执行以下命令，停止恢复任务。

    cd /opt/DataBackup/ProtectClient

    sh stop.sh

    命令中的/opt为客户端的安装目录，请根据实际替换。

  3. 选择“任务”。

    对于1.5.0版本，请选择。

  4. 查看该恢复任务的任务状态，待任务状态显示为“失败”后，重新登录恢复的目标实例所有的数据节点，查询是否存在4002_deploy.conf文件。

    4002_deploy.conf中的4002为TDSQL存储节点使用的端口，请以实际为准。以下操作以4002_deploy.conf为例进行说明。

    cd /data/oc_agent/log/

    ll |grep deploy

    • 如果存在4002_deploy.conf文件，则执行rm 4002_deploy.conf命令删除该文件，再继续执行5。
    • 如果不存在4002_deploy.conf文件，则继续执行5。

  5. 在恢复的目标实例所有的数据节点执行以下命令，启动恢复任务。

    cd /opt/DataBackup/ProtectClient

    sh start.sh
