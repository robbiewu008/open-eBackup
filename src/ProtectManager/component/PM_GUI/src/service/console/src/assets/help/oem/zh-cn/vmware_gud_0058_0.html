恢复Hyper-V虚拟机

当需要对已备份的虚拟机进行整机恢复时，可以参考本节恢复虚拟机至原位置或新位置。

背景信息

本产品支持使用备份副本、复制副本（不支持恢复至原位置）进行虚拟机恢复，支持恢复至原位置或新位置。
  • 恢复至原位置

    恢复至原位置时，系统将在原虚拟机所在路径下（即原虚拟机所属的Hyper-V服务器）创建一个新的虚拟机，将备份数据全量恢复到新虚拟机。

  • 恢复至新位置

    恢复至新位置时，系统进行全量恢复，将备份数据恢复到指定的新路径下，即由用户指定新虚拟机所在的Hyper-V服务器对应的路径。

操作步骤

  1. 选择“数据利用 > 副本数据 > 虚拟化 > Hyper-V”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在的行，单击“更多 > 恢复”。
  5. 恢复虚拟机至原位置或新位置。

    • 恢复虚拟机至原位置。
      1. 选择恢复至“原位置”。
      2. 设置新虚拟机的名称。
      3. 恢复后是否启用虚拟机网络适配器。
        • 不启用：恢复后虚拟机的所有网络适配器都处于未连接状态，如需远程登录虚拟机，恢复后需要在Hyper-V 管理界面设置网络适配器的连接状态。
        • 启用：启用后，如果副本中的原虚拟机仍然存在，则恢复后可能导致新虚拟机与原虚拟机网络冲突，请恢复虚拟机后检查新虚拟机网络。
      4. 选择恢复后是否自动开机。
      5. 选择恢复成功后是否删除原虚拟机。

        开启该选项后，虚拟机恢复成功后会删除原虚拟机，否则原虚拟机将保留。

      6. 单击“确定”。
    • 恢复虚拟机至新位置。
      1. 选择恢复至“新位置”。
      2. 设置新虚拟机名称。
      3. 设置恢复虚拟机到新位置的位置信息。
        表1 新位置信息

        参数

        说明

        计算位置

        虚拟机恢复后所使用的计算资源。

        选择一个Hyper-V服务器作为目标主机。

        说明：

        目标主机的Hyper-V版本不能低于原Hyper-V的版本。

        存储位置

        虚拟机配置文件和磁盘数据恢复到的Hyper-V服务器文件路径。

        • 不同数据存储
          • VM配置文件：虚拟机配置文件在Hyper-V服务器上的路径。
          • VM磁盘文件：虚拟磁盘在Hyper-V服务器上的路径。
            说明：

            为提高数据安全性，用户可选择将虚拟机的配置文件单独存放在一个数据存储上，而将磁盘数据统一放在另一个数据存储上。

        • 相同数据存储

          虚拟机配置文件和所有虚拟磁盘数据存储在Hyper-V服务器上的同一路径。
      4. 恢复后是否启用虚拟机网络适配器。
        • 不启用：恢复后虚拟机的所有网络适配器都处于未连接状态，如需远程登录虚拟机，恢复后需要在Hyper-V 管理界面设置网络适配器的连接状态。
        • 启用：启用后，如果副本中的原虚拟机仍然存在，则恢复后可能导致新虚拟机与原虚拟机网络冲突，请恢复虚拟机后检查新虚拟机网络。
      5. 选择恢复后是否自动开机。
      6. 单击“确定”。

  6. 单击“关闭”。
