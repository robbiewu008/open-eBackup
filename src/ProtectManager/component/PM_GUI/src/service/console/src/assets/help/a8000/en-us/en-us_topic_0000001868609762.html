Logging In to DeviceManager

DeviceManager is a piece of integrated storage management software developed by Huawei. On any management terminal connected to the storage system, you can use a browser to log in to DeviceManager as a local user using the management network port IP address of the controller of the storage system.

Prerequisites

Before using DeviceManager, check whether the management terminal meets the following requirements:

  • Compatibility of the operating system and browser running on the management terminal

    DeviceManager supports multiple operating systems and browsers. For details about the compatibility information, visit Compatibility Query.

  • The communication between the management terminal and storage device is normal.

Context

  • This section uses the Windows operating system as an example to describe how to log in to DeviceManager. The DeviceManager login operations on other operating systems need to be adjusted accordingly.
  • If the current user does not perform any operation within the session timeout period (30 minutes by default, which can be modified), the system automatically exits and the user needs to log in again.
  • If a non-super administrator has not logged in to the storage system for a period of time (60 days by default, which can be modified), the account will be locked and can be unlocked only by the super administrator. The unlocking method is as follows:

    Log in to DeviceManager as the super administrator and choose Settings > User and Security > Users and Roles > User Management to go to the User Management page. In the row where the name of the user to be unlocked resides, choose More > Unlock.

    If the super administrator chooses Settings > User and Security > Security Policies and selects Lock Account When Idle, a non-super administrator account will be locked only when the account has not logged in to the system for a specified period of time.

Procedure

  1. Open a browser on the management terminal.
  2. Type https://IPv4 address:8088 or https://[IPv6 address]:8088 in the address box and press Enter.

    • IPv4 address and IPv6 address indicate the DeviceManager management IP address of the storage system.
    • If a firewall is configured, you must enable port 8088 for the system to provide web services.
    • The browser may prompt that the website has a security certificate issue. If the IP address is correct, you can neglect the prompt and continue accessing the storage system.
    • If you have an available security certificate, you are advised to run related commands to import the certificate to further improve system security.

  3. (Optional) Click the language switching button in the upper right corner to switch the language.

    DeviceManager supports simplified Chinese and English.

  4. Select Local user or LDAP user in Authentication Mode, and enter the username and password in Username and Password.

    • The username and password are case-sensitive.
    • If the default password has been changed, use the new password to log in.
    • If Account Lockout is enabled (enabled by default) for an account in the security policy, the account will be locked if the number of consecutive incorrect password attempts (Lockout Threshold) reaches the upper limit (3 by default) within 5 minutes. (By default, the account is automatically unlocked after being locked for 5 minutes.)

    To ensure security of a user account, you are advised to change the account password periodically. You can log in to DeviceManager, choose Settings > User and Security > Users and Roles > User Management, and change the login password.

  5. Click Log In.

    The DeviceManager WebUI is displayed.

    • When you log in to DeviceManager for the first time, you need to sign the Huawei Enterprise End User License Agreement (EULA) as a user with administrator permissions to access DeviceManager. When signing the agreement, select I have read and agree to sign the EULA and click OK. The agreement takes effect after you log in to DeviceManager for the first time.
    • After the operation, click the current login user in the upper right corner of the page, and then click Log Out in the drop-down list box to log out of DeviceManager.
