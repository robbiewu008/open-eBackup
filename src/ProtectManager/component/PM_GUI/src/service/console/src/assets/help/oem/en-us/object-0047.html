Restoring Object Storage

This section describes how to restore the object storage that has been backed up to the original location or a new location.

Prerequisites

  • Before the restoration, ensure that the bucket with the same name exists in the target location or the conditions of creating the target bucket for restoration are met. Otherwise, the restoration will fail.
  • Before restoring data to a new location, ensure that users have the read and write permissions on the AK/SK of the target object storage for restoration.

Procedure

  1. Choose Explore > Copy Data > File Systems > Object Storage.
  2. You can search for copies by object storage resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, find the resource to be restored by resource name and click the resource name.

  3. Click Copy Data and choose the year, month, and day to find the copy.

    If is displayed below a month or date, copies exist in the month or on the day.

  4. Choose More > Restore in the row that contains the copy.

    Table 1 describes the related parameters.

    Table 1 Parameters for restoring object storage

    Parameter

    Description

    Restore To

    Location where the object storage is to be restored.

    • Original location: indicates to restore data to the original object storage location.
    • New location: indicates to restore data to a new location. In this case, you need to specify the target location and target object storage for restoration.

    Location

    This parameter is displayed only when data is restored to the original object storage location.

    Target Object Storage

    This parameter is displayed only when data is restored to a new location. You need to select the target object storage.

    You can search for the name of object storage that has been registered with the product to select the object storage.

    Target Bucket

    This parameter is displayed only when data is restored to a new location.

    • Select an existing bucket: indicates to select an existing bucket of the current user.
    • Create a bucket: indicates to create a bucket for the current user. In this case, data will be restored to the created bucket.
    NOTE:
    • If you select an existing bucket and the ACL data has been backed up in the copy, the ACL of the target bucket will be overwritten during restoration.
    • If multiple buckets to be restored have objects with the same name and prefix, only one object can be randomly restored.
    • Enter a name for the bucket to be created based on the naming rule of Type of Owning Object Storage.
      • OceanStor Pacific: The namespace name consists of 1 to 255 characters, including only letters, digits, hyphens (-), underscores (_), and periods (.). It must contain at least one letter or digit.
      • Huawei Cloud Stack Object Storage Service (OBS): The bucket name must be globally unique and cannot be the same as any existing bucket name, including the names of buckets created by other users. Once a bucket is created, its name cannot be changed. Make sure that the bucket name you set is appropriate.

        Name the bucket in the OBS according to the globally applied DNS naming rules as follows:

        The name of a deleted bucket can be reused for another bucket or a parallel file system at least in 30 minutes after the deletion.

        Must contain 3 to 63 characters and supports only lowercase letters, digits, hyphens (-), and periods (.)

        Cannot start or end with a period (.) or hyphen (-). A period (.) is not allowed to be adjacent with another period (.) or any hyphens (-).

        Cannot be an IP address.

        When you access OBS through HTTPS using a virtual host and if the bucket name contains a period (.), the certificate verification will fail. Therefore, you are advised not to use periods (.) in bucket names.

      • Alibaba Cloud Object Storage Service (OSS): The bucket name must contain 3 to 63 characters, including only lowercase letters, digits, and hyphens (-); must start and end with a lowercase letter or digit.

    Prefix

    Prefix added to the object to be restored.

    Overwrite Rule

    If a file with the same name exists in the restoration path, you can choose to replace or skip the existing file.

    • Replace existing files
    • Skip existing files: Files with the same names are skipped and are not replaced.
    • Only replace the files older than the restoration file: The latest file with the same name in the target path is retained.

    Select Objects to Be Restored

    This parameter is displayed only for object-level restoration.

    NOTE:

    If there are no data changes after the incremental backup, no data will be displayed on the GUI.

  5. Click OK.
