步骤5：注册GaussDB项目

执行GaussDB备份恢复前，需要将GaussDB注册至OceanProtect。

前提条件

  • 生产环境已安装TPOPS，并将数据库实例纳管到TPOPS平台。
  • TPOPS平台和OceanProtect网络互通。
  • 如果需要开启XBSA SSL 证书管理，需确保安装ProtectAgent时已获取SSL证书，详细操作请参见《OceanProtect 备份一体机 1.5.0-1.6.0 ProtectAgent安装指南》中的“配置SSL双向认证证书文件”章节。

操作步骤

  1. 选择“保护 > 数据库 > GaussDB”。

    对于1.5.0版本，请选择“保护 > 云平台 > GaussDB”。

  2. 在“项目”页签下，单击“注册”，注册GaussDB信息。

    GaussDB注册信息如表1所示。
    表1 GaussDB注册信息

    参数

    说明

    名称

    用户自定义GaussDB项目名称。

    管理面地址

    TPOPS对外提供服务的地址。如果TPOPS为HA形态部署，可使用单个或多个管理面地址进行注册，多个地址以英文逗号分隔。

    地址输入示例：192.168.0.0,192.168.0.1。

    管理面端口

    TPOPS对外提供服务的端口。

    端口输入示例： 7070或8002。

    用户名

    用户在TPOPS中创建的用户名。

    密码

    用户在TPOPS中创建的用户密码。

    代理主机

    已注册到OceanProtect的数据保护代理。

    XBSA配置路径

    用户配置文件xbsa.conf在代理主机中的绝对路径。

    XBSA SSL 证书管理

    默认开启。开启后，注册GaussDB时会对其证书进行校验，确保OceanProtect与外部设备交互的安全性。关闭后，系统与云平台通信可能存在安全风险，请谨慎操作。

    根证书

    后缀为“.pem”的根证书文件，获取方式参见表1中的准备项“证书”。当选择“XBSA SSL 证书管理”时，需要配置该参数。

    用户证书

    后缀为“.crt”的客户端用户证书文件，获取方式参见表1中的准备项“证书”。当选择“XBSA SSL 证书管理”时，需要配置该参数。

    用户私钥

    后缀为“.key”的客户端用户私钥文件，获取方式参见表1中的准备项“证书”。当选择“XBSA SSL 证书管理”时，需要配置该参数。

    私钥密码

    创建证书文件时设置的客户端私钥密码字符串。当选择“XBSA SSL 证书管理”时，需要配置该参数。

  3. 单击“确定”。
