步骤10：创建备份SLA

系统中预置了Gold、Silver、Bronze三种SLA（Service Level Agreement），当预置SLA已满足您的备份需求时，请跳过本节。您也可以根据自身需求自定义SLA以对资源进行保护。

背景信息

关于系统预置SLA和自定义SLA的详细说明可参考关于SLA。

操作步骤

  1. 选择“保护 > 保护策略 > SLA”。
  2. 单击“创建”。
  3. 自定义SLA的名称。当使用系统默认名称时，系统默认名称为：SLA_当前时间戳。
  4. 选择需要创建SLA的应用类型。

    1. 单击“应用”图标，选择要创建SLA的应用。
      • 选择“通用SLA”页签，单击“所有资源”。

        通用SLA可以应用到除文件集、SQL Server、PostgreSQL、Kingbase、Redis、ClickHouse、MongoDB、NAS共享、NAS文件系统、华为云Stack GaussDB、TDSQL、Kubernetes CSI、TiDB、卷、通用共享、Active Directory、GaussDB T 单机、对象存储、GaussDB（1.6.0及后续版本）、GaussDB(DWS)、应用入口的SAP HANA（1.6.0及后续版本）、SAP on Oracle、AntDB、VMware、Hyper-V以外的资源。

      • 选择“指定应用”页签，选择待创建SLA的应用。

        本节以指定应用创建SLA为例进行说明。

      对于1.5.0.SPC16及后续版本，使用通用SLA，系统会在备份前，检查待备份虚拟机所在生产存储的剩余容量百分比是否高于默认阈值（固定为20%），若剩余容量低于20%，会导致备份失败。使用指定应用SLA，则可在备份策略的高级配置中修改此阈值。

    2. 单击“确定”。

  5. 设置备份策略。

    1. 单击“备份策略”图标。
    2. 配置备份策略基础参数。

      相关参数说明如表1所示。

      请根据业务情况合理设置备份周期、副本保留时长以及允许备份的时间段，建议如下：
      • 备份频率至少大于备份时长。
      • 全量副本的保留时长大于永久增量（合成全量）备份副本的保留时长。
      • 至少需要创建一个备份策略，如需添加其他备份策略，请单击界面右侧的。
      • 每种备份类型最多支持添加4个备份策略。
      表1 备份策略基础参数说明

      参数

      说明

      全量备份

      名称

      设置备份策略名称。

      -
      配置全量备份的备份频率、副本保留时长以及允许备份的时间段。
      • 配置全量备份的备份频率以及副本的保留时长。
        • 按年

          设置每年xx月xx日执行一次，副本保留xx天/周/月/年或永久保留。

          如果当年不存在该日期，则不产生副本。

        • 按月

          设置每月xx日（可选择多日）或最后一天执行一次，副本保留xx天/周/月/年或永久保留。

          如果设置为每月xx日执行一次备份，当月不存在该日期时，则不产生副本。

        • 按周

          设置每周一/二/三/四/五/六/日执行一次（可多选），副本保留xx天/周/月/年或永久保留。

        • 按天

          设置从xx年xx月xx日开始，每xx天执行一次备份，副本保留xx天/周/月/年或永久保留。

        • 按小时

          设置从xx年xx月xx日开始，每xx小时执行一次备份，副本保留xx天/周/月/年或永久保留。

      • 设置允许执行全量备份的时间段，该时间范围以外的时间不会进行全量备份任务调度。
        说明：
        • 如果结束时间小于或等于开始时间，则结束时间实际为跨日的结束时间。
        • 如果备份任务在指定的时间窗口内未执行完成，系统不会中止本次备份任务，但会上报事件。
        • 一旦副本超过设定的保留时长，系统将自动删除过期的副本。

      永久增量（合成全量）备份

      名称

      设置备份策略名称。

      -
      配置永久增量（合成全量）备份的首次执行时间、备份频率、副本保留时长以及允许备份的时间段。
      • 配置永久增量（合成全量）备份的备份频率以及副本的保留时长。
        • 按年

          设置每年xx月xx日执行一次，副本保留xx天/周/月/年或永久保留。

          如果当年不存在该日期，则不产生副本。

        • 按月

          设置每月xx日（可选择多日）或最后一天执行一次，副本保留xx天/周/月/年或永久保留。

          如果设置为每月xx日执行一次备份，当月不存在该日期时，则不产生副本。

        • 按周

          设置每周一/二/三/四/五/六/日执行一次（可多选），副本保留xx天/周/月/年或永久保留。

        • 按天

          设置从xx年xx月xx日开始，每xx天执行一次备份，副本保留xx天/周/月/年或永久保留。

        • 按小时

          设置从xx年xx月xx日开始，每xx小时执行一次备份，副本保留xx天/周/月/年或永久保留。

        • 按分钟

          设置从xx年xx月xx日开始，每xx分钟执行一次备份，副本保留xx天/周/月/年或永久保留。

      • 设置允许执行永久增量（合成全量）备份的时间段，该时间范围以外的时间不会进行永久增量（合成全量）备份任务调度。
        说明：
        • 如果结束时间小于或等于开始时间，则结束时间实际为跨日的结束时间。
        • 如果备份任务在指定的时间窗口内未执行完成，系统不会中止本次备份任务，但会上报事件。
        • 一旦副本超过设定的保留时长，系统将自动删除过期的副本。
    3. 对于1.6.0及后续版本，如果需生成WORM副本，请打开WORM开关，并设置WORM有效期。否则，请跳过本步骤。
      “WORM有效期”支持设置为“与副本留存期一致”或“自定义有效期”。
      • 仅OceanProtect X系列备份一体机和OceanProtect E1000支持WORM。
      • WORM副本不支持删除或缩短有效期。
      • 自定义WORM有效期不能大于副本保留时长。
      • 在WORM有效期内不支持修改为非WORM副本。
      • 如果“WORM有效期”设置为“与副本留存期一致”且副本为永久保留，由于备份存储对快照格式的副本最长保留20年，目录格式的副本最长保留70年，因此该场景下快照格式的副本WORM有效期实际为20年，目录格式的副本WORM有效期实际为70年。
    4. 配置备份策略其他参数。
      相关参数说明如表2所示。
      表2 备份策略其他参数说明

      参数

      说明

      指定目标位置
      请指定备份副本的目标存储位置。
      • “备份存储单元”：对应于备份集群节点。选择备份存储单元后，副本将存储至该存储单元。存储单元容量不足时，备份任务将会失败。

        创建备份存储单元的操作请参考《OceanProtect DataBackup 1.5.0-1.6.0 集群高可用特性指南》中的“创建备份存储单元”章节。

      • “备份存储单元组”：是备份存储单元的集合。选择备份存储单元组后，系统根据存储单元组的存储策略自动选择备份目标存储单元。

        创建备份存储单元组的操作请参考《OceanProtect DataBackup 1.5.0-1.6.0 集群高可用特性指南》中的“（可选）创建备份存储单元组”章节。

      说明：
      • 对于1.5.0版本，如果您部署了集群高可用特性，该参数才显示。您也可以不指定目标位置，如不指定，则系统通过智能均衡自动选择备份目标存储单元。
      • 对于1.6.0及后续版本，则必须指定目标位置。

      限速策略

      备份任务平均每秒最大传输的数据量。

      选择已创建的限速策略。
      自动索引
      说明：

      仅1.5.0版本存在此参数。
      当备份的磁盘包含了系统盘时，开启后，支持选择副本中的单个或多个文件进行恢复，支持搜索副本中的文件、目录或链接。
      说明：

      索引需要占用备份存储空间，当系统中的索引文件总数超过10亿时，请按照《OceanProtect DataBackup 1.5.0-1.6.0 故障处理》中的“创建副本索引失败”章节进行处理。

      应用一致性备份

      开启后，支持应用一致性备份。

      如果您的虚拟机中包含数据库、电子邮件服务等事务型应用程序，则应开启应用一致性备份以保证应用数据的一致性。
      任务超过时间窗告警
      说明：

      仅1.6.0及后续版本支持设置该参数。

      开启后任务执行超过时间窗将发送告警，告警需手动清除。

      任务失败告警

      开启后，任务执行失败后自动发送告警，下次任务成功后告警自动清除。
      源端重删
      说明：

      仅OceanProtect X系列备份一体机和OceanProtect E1000（备份存储为OceanProtect）支持配置此参数。
      开启后，可以减少传输的数据量，但会增加客户端的资源占用。
      说明：
      • 由于同一个客户端只能和一个存储单元建链，如果开启源端重删，“指定目标位置”请选择“备份存储单元”。否则，由于智能均衡可能会选择到未建链的存储单元，导致源端重删启用失败。如果不同受保护对象关联了不同SLA，且SLA中指定不同的备份存储单元，请为受保护对象选择不同客户端，否则将会导致部分备份任务源端重删启用失败。
      • 如果客户端已经与一个存储单元建链，当该客户端未执行源端重删备份任务时，可以在客户端所在主机上执行dataturbo delete storage_object storage_name=?命令（命令中的storage_name=?可以通过dataturbo show storage_object命令查询），删除与存储单元的源端重删链接。再选择该客户端执行源端重删备份任务，则可以成功与新的存储单元建链。
      • 请参考OceanProtect兼容性查询工具查看客户端所在主机的操作系统类型是否支持源端重删。若不支持，配置源端重删后将不会生效。
      • 若使用内置客户端（即客户端合并部署到OceanProtect）进行备份，配置源端重删后将不会生效。
      通用压缩
      说明：

      仅OceanProtect E6000备份一体机支持设置此参数。

      开启通用压缩可以将业务数据直接进行压缩后再写入存储空间，达到节约存储空间提升备份效率的目的。

      存储层备份
      • 对于1.5.0版本，开启后，当ESXi主机数据存储对接OceanStor Dorado 6.1.3及后续版本的LUN时，优先通过存储层备份模式备份。
      • 对于1.6.0及后续版本，如果添加了OceanStor Dorado 6.1.3及后续版本或NetApp ONTAP 9.4及后续版本的存储资源，则支持对满足要求的虚拟机磁盘优先通过存储层备份模式进行备份。
      说明：
      • 开启前请确保满足以下条件。
        • OceanStor Dorado设备已注册到OceanProtect。
        • 客户端所在主机与存储设备能正常通信。
        • 如果虚拟机磁盘是基于OceanStor Dorado LUN建立的Datastore，请确保已登录iSCSI启动器，具体操作请参见步骤9：登录iSCSI启动器。
      • 对于1.6.0及后续版本，此功能与排除已删除文件数据功能或指定数据传输模式功能互斥。

      排除已删除文件数据

      说明：

      仅1.6.0及后续版本存在此参数。

      如果虚拟机磁盘文件系统类型为EXT4、XFS或NTFS，开启此开关，系统会在备份前识别磁盘中的已删除文件数据，并将其排除在备份数据范围外，以减少备份数据量。

      说明：

      此功能与存储层备份功能互斥。

      指定数据传输模式

      说明：

      仅1.6.0及后续版本存在此参数。

      开启后，OceanProtect将优先使用指定的数据传输模式进行备份，如果不满足指定的数据传输模式要求，则通过其他可用的数据传输模式进行备份。

      说明：

      此功能与存储层备份功能互斥。

      添加备份记录

      默认关闭。开启后，在备份vCenter Server上的虚拟机完成后，系统会在vCenter Server上对应虚拟机的“自定义属性”页面添加本次备份的信息。

      如果注册的是ESXi主机，该开关不生效。

      生产存储剩余容量阈值

      说明：

      仅1.5.0.SPC16及后续版本存在此参数。

      此参数会用于在备份前，检查待备份虚拟机所在生产存储的剩余容量百分比是否高于指定阈值。

      当生产存储的剩余容量百分比低于指定阈值时，会导致备份失败。

      失败后自动重试

      开启后，周期性备份任务失败后系统自动重试。

      支持重试次数为1~5次，等待时长为1~30分钟。如设置为重试3次，等待5分钟，即每5分钟后重试一次，总共重试3次。
      说明：
      • 自动重试时系统会创建一个新的备份任务，如果该备份任务不在设定的时间窗内则不会执行，此次备份失败。
      • 该功能仅在周期性备份时生效。手动发起备份任务，任务失败无法自动重试。
    5. 单击“确定”。

  6. 可选：配置归档策略。

    OceanProtect自带存储空间有限，如果您需要长期保留副本数据，可以将副本数据归档到外部存储。

    您可以单击“归档策略”图标配置归档策略或随后通过修改SLA的方式配置归档策略。

    配置归档策略的具体操作请参考归档。

  7. 可选：配置复制策略。

    OceanProtect支持本地数据复制到异地，当灾难发生后在异地数据中心通过副本数据恢复生产，进而承接业务，实现异地数据保护。

    您可以单击“复制策略”图标配置复制策略或随后通过修改SLA的方式配置复制策略。

    配置复制策略的具体操作请参考复制中对应产品型号的“创建复制SLA”章节。

    • 对于OceanProtect E1000，如果备份策略中指定的“备份存储单元”或“备份存储单元组”的设备类型为“服务器”，则不支持创建复制策略。
    • 对于1.6.0及后续版本：
      • SLA中备份策略的WORM有效期需小于复制策略中副本保留时长。
      • 如果本端未设置WORM，目标端设置WORM，则WORM有效期与复制副本保留时长一致。
      • 如果本端设置了WORM，目标端未设置WORM，则复制副本的WORM有效期与本端WORM副本有效期一致，复制副本保留时长>=本端的备份副本WORM有效期。
      • 如果本端和目标端都设置WORM，且目标端在“数据安全”WORM配置中按资源粒度设置WORM，则所有复制副本均会有WORM属性，WORM有效期和复制副本保留时长一致。
      • 本端备份副本为永久保留，且“WORM有效期”设置为“与副本留存期一致”，则WORM有效期为永久有效。当生成了备份副本后，修改SLA策略（添加复制策略并设置“复制规则&保留规则”为“复制所有副本”），目标端WORM有效期与复制副本保留时长一致。

  8. 单击“确定”。
