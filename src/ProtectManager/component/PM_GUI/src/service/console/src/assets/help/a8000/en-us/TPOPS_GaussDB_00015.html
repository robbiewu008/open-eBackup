Step 3: Registering the GaussDB Project

Before GaussDB backup and restoration, register GaussDB with the OceanProtect.

Prerequisites

  • TPOPS has been installed in the production environment, and database instances have been managed on TPOPS.
  • The network communication between TPOPS and the OceanProtect is normal.
  • If you need to enable XBSA SSL Certificate Management, ensure that the SSL certificate has been obtained during ProtectAgent installation. For details, see OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide"Configuring Certificate Files for Mutual SSL Authentication" in the .

Procedure

  1. Choose Protection > Databases > GaussDB.

    For 1.5.0, choose Protection > Cloud Platforms > GaussDB.

  2. On the Project tab page, click Register to register the GaussDB information.

    Table 1 describes GaussDB registration information.
    Table 1 GaussDB registration information

    Parameter

    Description

    Name

    User-defined name of the GaussDB project.

    Management Plane Address

    Address used by TPOPS to provide services for external systems. If TPOPS is deployed in high availability (HA) mode, you can use one or more management plane addresses for registration. Use commas (,) to separate multiple addresses.

    Example: 192.168.0.0, 192.168.0.1.

    Management Plane Port

    Port used by TPOPS to provide services for external systems.

    Example: 7070 or 8002.

    Username

    Username for logging in to the TPOPS management page.

    Password

    Password for logging in to the TPOPS management page.

    Agent Host

    Agent host registered with the OceanProtect.

    XBSA Configuration Path

    Absolute path of the user configuration file xbsa.conf on the agent host.

    Example: /home/myuser/xbsa.conf

    XBSA SSL Certificate Management

    This function is enabled by default. After this function is enabled, when GaussDB is registered, its certificate will be verified during registration to ensure security of interaction between the OceanProtect and external devices. After this function is disabled, security risks may exist in the communication between the system and TPOPS. Exercise caution when performing this operation.

    Root Certificate

    Root certificate file with the file name extension .pem. For details about how to obtain the file, see the preparation item Certificate in Table 1. This parameter is mandatory only when XBSA SSL Certificate Management is selected.

    User Certificate

    Client user certificate file with the file name extension .crt. For details about how to obtain the file, see the preparation item Certificate in Table 1. This parameter is mandatory only when XBSA SSL Certificate Management is selected.

    User Private Key

    Client user private key file with the file name extension .key. For details about how to obtain the file, see the preparation item Certificate in Table 1. This parameter is mandatory only when XBSA SSL Certificate Management is selected.

    Private Key Password

    Client private key password character string set when the certificate file is created. This parameter is mandatory only when XBSA SSL Certificate Management is selected.

  3. Click OK.
