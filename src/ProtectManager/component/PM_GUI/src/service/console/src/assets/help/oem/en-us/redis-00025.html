Step : Adding a Target Cluster

This section describes how to add a target cluster to the local cluster so that a logical connection is established between the local and remote devices for data transmission.

Prerequisites

The target cluster for replication has been prepared.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  2. On the Replication Clusters tab page, click Add.
  3. Set parameters related to the target cluster.

    Table 1 describes the related parameters.
    Table 1 Parameters related to the target cluster

    Parameter

    Description

    Name

    Name of the target cluster, which can be customized.

    Role

    The value is fixed at Replication Target Cluster.

    IP Address
    Management, backup, or replication IP address of any controller in the target cluster.
    • If you enter the management IP address of the target cluster, ensure that the management network of the local end is connected to that of the target end.
    • If you enter the backup network IP address of the target cluster, ensure that the backup network of the local end is connected to that of the target end.
    • If you enter the replication network IP address of the target cluster, ensure that the replication network of the local end is connected to that of the target end.
    NOTE:
    • The management IP address is the IP address for logging in to the management page of the target cluster.
    • To obtain the backup or replication network IP address, perform the following operations: Log in to the management page of the target cluster and choose System > Infrastructure > Cluster Management. On the Backup Cluster tab page, click the cluster name to obtain the backup network IP address (data backup engine) or replication network IP address of any controller.

    Port

    Port of the target cluster.

    The default port number is 25081 if IP Address is set to a management IP address, and 30068 if IP Address is set to the IP address of the replication network or backup network.

    Username

    User and password created in Step 5: Creating a Remote Device Administrator or Step 5: Creating a Remote Device Administrator (Applicable to 1.6.0 and Later Versions).

    Password

  4. Click OK.
