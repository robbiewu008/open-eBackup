Viewing TDSQL Database Environment Information

After a TDSQL database is registered with the OceanProtect, you can view the database cluster and its instances on the management page.

Procedure

  1. Choose Protection > Databases > TDSQL.
  2. On the Cluster tab, view the database cluster information.

    If a database in a cluster has been registered, the cluster information is displayed on the page.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    Table 1 describes the detailed information.
    Table 1 Host and cluster information

    Parameter

    Description

    Name

    Name of a database cluster.

    If you click the cluster name, all hosts in the current cluster and all authenticated databases in the cluster are displayed.

    Status
    Status of a host or cluster.
    • Online: If the host is powered on and the client on the host is running properly, Status is Online for the host.

      If there are online hosts in the cluster, the cluster status is Online.

    • Offline: If the host is shut down or the host where the client is installed is faulty, Status is Offline for the host.

      If all hosts in the cluster are offline, the cluster status is Offline.
    Owner
    NOTE:

    This parameter is available only in 1.5.0.

    Data protection administrator of a host or cluster.

    If a host or cluster has been authorized to a data protection administrator, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
    Tag
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.

  3. On the Non-distributed Instances or Distributed Instances tab page, view the database instance information.

    The registered databases are displayed on the page.

    Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

    1. Table 2 describes the detailed instance information.
      Table 2 Instance information

      Parameter

      Description

      Name

      Database instance name.

      Status
      Database instance status.
      • Online: When the host where the client is installed is properly connected to all nodes in the TDSQL cluster, Status is Online.
      • Offline: When the connection between the host where the client is installed and all nodes in the TDSQL cluster is abnormal, Status is Offline.

      Owning Cluster

      Name of the cluster where the instance resides.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status
      Whether the current resource is protected.
      • Unprotected: The current resource is not associated with an SLA, or the resource is associated with an SLA but protection is disabled.
      • Protected: The current resource is associated with an SLA and protection is activated.
      • Creating: The current resource is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name and check the overview, copy information, and related jobs.
      • Overview

        Display the SLA information, job execution time, and total number of copies.

      • Copy data

        You can search for copies by year, month, and day.

        If is displayed below the time, a copy exists at the time.

      • Related jobs

        Displays backup and restoration jobs.
