Functions

This section describes the Nutanix data protection functions provided by the OceanProtect.

Backup

  • Full backup: backs up the entire set of data at a specific point in time, regardless of whether the data has been modified since the last backup. In the case of large data volumes, this backup type may take a long time and occupy a large amount of backup storage space.
  • Forever incremental (synthetic full) backup: backs up only the data that is added or modified since the first full backup. Compared with full backup, forever incremental (synthetic full) backup significantly improves the backup efficiency in scenarios with large data volumes. If no full backup copy exists, the first forever incremental (synthetic full) backup will be a full backup, and incremental backups are performed subsequently. Copies generated by forever incremental (synthetic full) backups are forever incremental (synthetic full) copies, which improve the restoration performance and do not occupy extra space.
  • Copy verification: verifies the integrity of a copy after the backup is complete or before the copy is used for restoration.
  • Source deduplication: deduplicates backup data at the source end before the data is sent to the target end. This technology saves network transmission bandwidth and storage space. Only OceanProtect X series backup appliances and OceanProtect E1000 (with the OceanProtect used as backup storage) support source deduplication.
  • Deduplication and compression: allows the OceanProtect to automatically deduplicate and compress data after the data is backed up. Only OceanProtect X series backup appliances and OceanProtect E1000 support deduplication and compression.
  • Rate limiting policy: limits the bandwidth for backup, replication, and archive.
  • Automatic retry upon failure: allows the system to automatically retry if a periodic backup job fails. The number of retries ranges from 1 to 5, and the wait time ranges from 1 to 30 minutes. For example, if the number of retries is set to 3 and the wait time is set to 5 minutes, the system retries for a total of three times at an interval of 5 minutes.
  • Backup link encryption: encrypts transmitted data during data backup to ensure data security. Only the OceanProtect X series backup appliances and OceanProtect E1000 (OceanProtect is used as backup storage) support backup link encryption.

Replication

  • Backup copy replication: replicates backup copies from the source end to the target end. If a disaster occurs in the local data center, the replication copies can be used to restore service data.
  • Replication link encryption: encrypts data transmitted during remote replication based on IPsec policies created for the replication network logical ports on the storage devices at both ends of a replication link, ensuring data security. Only the OceanProtect X series backup appliances and OceanProtect E1000 support replication link encryption.
  • Reverse replication: replicates the replication copy generated after replication of a backup copy back to the backup system where the backup copy resides. Only the OceanProtect X series backup appliances and OceanProtect E1000 support reverse replication.
  • Cascaded replication: replicates the replication copy generated after replication of a backup copy to another backup system. Only the OceanProtect X series backup appliances and OceanProtect E1000 support cascaded replication.

Archiving

  • Manual archive (supported only in 1.6.0 and later versions) or SLA-based periodic archive is supported. SLA is short for service level agreement.
  • Backup/Replication copies that need to be retained for a long time can be archived to object storage.
  • Backup/Replication copies that need to be retained for a long time can be archived to tape libraries. Only OceanProtect X series backup appliances support archive to tape libraries.

Restoration

  • Copy-based restoration to the original or a new location: can be implemented using backup copies, reverse replication copies (data cannot be restored to the original location), cascaded replication copies (data cannot be restored to the original location), archive copies (data cannot be restored to the original location using the archive copies of replication copies and cascaded replication copies), and replication copies (data cannot be restored to the original location).
