Step 6: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Context

  • Copies generated by manual backup are retained for the period defined in the SLA.

    If the protection policy selected during manual backup is not defined in the SLA, the generated backup copy is retained permanently. If incremental backup is not defined in the SLA but is selected during manual backup, the copies generated by the manual incremental backup are retained permanently.

  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform a manual backup.
  • During manual backup, parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, are applied to the manual backup job.

Procedure

  1. Choose Protection > Databases > DB2.
  1. Click the Databases or Tablespace Set tab.
  2. In the row where the database or table set to be protected resides, choose More > Protect.

    You can select multiple databases or table sets for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

  4. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  1. Optional: Execute manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row where the database or table set to be backed up resides, choose More > Manual Backup.
    2. Set the name of the copy generated by manual backup.

      If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.

    3. Click OK.

      You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup.
