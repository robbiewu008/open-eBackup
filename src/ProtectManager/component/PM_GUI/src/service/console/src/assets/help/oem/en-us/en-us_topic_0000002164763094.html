The mysql, mysqlbinlog, mysqldump, or mysqladmin Tool Cannot Be Found in the /usr/bin Directory of the MySQL/GreatSQL Host

Symptom

The MySQL/GreatSQL database fails to be backed up, and the following message is displayed: "No available configuration file is found, or the mysql, mysqlbinlog, mysqldump, or mysqladmin tool cannot be found in the /usr/bin directory of the MySQL/GreatSQL host."

Possible Causes

The MySQL/GreatSQL database may not be installed in the /usr/bin directory, and the mysql, mysqlbinlog, mysqldump, and mysqladmin tools are not configured through soft links.

Troubleshooting

  1. Log in to the MySQL/GreatSQL database host node.
  2. Run the which mysql command to find the MySQL/GreatSQL database installation path. /mysqlpath is used as an example.
  3. Run the following commands in sequence to establish MySQL command soft links:

    ln -s /mysqlpath/bin/mysql /usr/bin/mysql
ln -s /mysqlpath/bin/mysqlbinlog /usr/bin/mysqlbinlog
ln -s /mysqlpath/bin/mysqldump /usr/bin/mysqldump
ln -s /mysqlpath/bin/mysqladmin /usr/bin/mysqladmin

  4. Run the following commands and check the command output. If no error message is displayed, the configuration is successful.

    mysql --version
mysqlbinlog --version
mysqldump --version
mysqladmin --version
