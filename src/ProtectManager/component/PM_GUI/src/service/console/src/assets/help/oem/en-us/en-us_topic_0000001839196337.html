Restoring a Kingbase Instance

This section describes how to restore a Kingbase instance that has been backed up to the original location or a new location.

Context

Backup and replication copies can be used for restoration. Restoration to the original or a new location is supported. If you use replication copies, the data cannot be restored to the original location.

Procedure

  1. Choose Explore > Copy Data > Databases > Kingbase.
  2. You can search for copies by Kingbase instance resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the resource to be recovered by resource name and click the name.

  3. Select the year, month, and day in sequence to find the copy.

    If is displayed under a month or date, a copy exists in the month or on the date.

  4. Recover data to a specified point in time or using a specified copy.

    Table 1 describes the related parameters.
    Table 1 Parameters for restoring a Kingbase instance

    Parameter

    Description

    Restore To

    Select either Original location or New location.

    Location

    If you set Restore To to Original location, the location of the current instance is displayed by default.

    Target Host

    Specify the target host, which is mandatory if you set Restore To to New location.

    Target Instance

    Specify the target instance, which is mandatory if you set Restore To to New location.

    Script to Run Before Restoration

    You can execute a self-defined script before a recovery job is executed or after it succeeds or fails, all based on your needs. You need to enter the absolute path of the script.

    • The Windows OS supports .bat scripts.
    • Non-Windows OSs support .sh scripts.
      NOTE:

      If Script to Run upon Restoration Success is configured, the status of the restoration job is displayed as Succeeded on the product even if the script fails to be executed. Check whether the job details contain a message indicating that the post-processing script fails to be executed. If yes, modify the script in a timely manner.

    Script to Run upon Restoration Failure

  5. Click OK.
