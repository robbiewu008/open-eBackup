Step 1: Registering a MongoDB Cluster

Prerequisites

Before registering a MongoDB cluster, ensure that each host in the cluster has been registered.

Procedure

  1. Choose Protection > Big Data > MongoDB.
  2. Click Register to register the MongoDB cluster.
  3. Select a cluster type in the Register dialog box.

    • Single instance
      1. Configure the database instance and its authentication information.

        Table 1 describes the related parameters.

        Table 1 Registration information of a single MongoDB instance

        Parameter

        Description

        Name

        Instance name.

        Hosts

        Select the database host to be registered.

        Port

        Port number of the MongoDB instance.

        Authentication Method

        The instance authentication mode can be OS authentication or database authentication.

        • OS authentication: authentication based on the OS.
        • Database authentication: authentication through the database administrator username and password.

        Database Username

        Name of the database administrator. This parameter is displayed only when authentication mode is set to database authentication.

        Database Password

        Password of the database administrator. This parameter is displayed only when authentication mode is set to database authentication.

        Database Installation Directory

        Set this parameter to the directory where the mongo/mongod/mongos file is located. If this parameter is not specified, the directory configured in the global variable is used by default.

        Database Tool Installation Directory

        Set this parameter to the directory where the mongodump/mongorestore tool is located. If this parameter is not specified, the tool directory configured in the global variable is used by default.
      2. Click OK.
    • Cluster instance
      1. Customize the instance name and select a cluster type, which can be Replica set or Sharded.
      2. (Optional) Configure the data installation directory or the database tool installation directory in batches. After the configuration is complete, you can configure the database installation directory or database tool installation directory for all added nodes.
      3. Click Add. In the Add dialog box, add all nodes of the cluster instance.

        Table 2 describes the related parameters.

        Table 2 Node information of a MongoDB cluster instance

        Parameter

        Description

        Hosts

        Select the database host to be registered.

        Port

        MongoDB port number.

        Authentication Method

        The instance authentication mode can be OS authentication or database authentication.

        Database Username

        Name of the database administrator. This parameter is displayed only when authentication mode is set to database authentication.

        Database Password

        Password of the database administrator. This parameter is displayed only when authentication mode is set to database authentication.

        Database Installation Directory

        Set this parameter to the directory where the mongo/mongod/mongos file is located. If this parameter is not specified, the directory configured in the global variable is used by default.

        Database Tool Installation Directory

        Set this parameter to the directory where the mongodump/mongorestore tool is located. If this parameter is not specified, the tool directory configured in the global variable is used by default.
      4. Click OK.
