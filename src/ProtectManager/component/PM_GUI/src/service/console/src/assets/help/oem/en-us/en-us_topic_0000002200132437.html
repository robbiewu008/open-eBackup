Restoring Cloud Disks

This section describes how to restore a cloud server or cloud disk to the original or a new location after the cloud server has been backed up, when the cloud server disk is damaged.

Context

The product allows you to restore cloud disks to the original or a new location using backup copies or replication copies (data cannot be restored to the original cloud server using replication copies).

  • Restoration to the original location

    Restores data to cloud server disks on the current cloud platform, including existing disks and new disks.

  • Restoration to a new location

    Restores data to existing disks of another cloud server.

Precautions

A maximum of 24 disks can be mounted to a host where the client is installed. Therefore, if 24 disks have been mounted to a host where the client is installed during backup and restoration, the system performs the restoration job when there are idle slots.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Alibaba Cloud.
  2. Search for copies by resource or copy. This section describes how to search for copies by resource.

    On the Resources tab page, locate the cloud server to be restored by cloud server name and click the name.

  3. On the Copy Data page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, a copy is generated in the month or on the day.

  4. Find the copy to be restored and choose More > Disk Restoration on the right.
  5. Select the cloud disk to be restored and configure the restoration.
  6. Restore data to the original location or a new location.

    • Restoration to the original location

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring data to the original location

      Parameter

      Description

      Target Location

      Select the cloud server to which the backup copy is restored.

      Type

      Select the type of the disk to be restored. You can select Existing or New.

      Target Disk Name

      Select the name of the disk to be restored.

      Password

      Password of the VM to be restored. This parameter is available when you select a system disk.

      Clients

      Select the client for copy-based restoration.

      If no client is selected, the system selects one or more clients from those configured to perform restoration.

      Power on Automatically After Restoration

      If this function is enabled, the cloud server is automatically started after the restoration is complete. This function is disabled by default.

      Copy Verification Before Restoration

      If this option is enabled, the integrity of a copy is verified before the copy is used for restoration. This operation affects the restoration performance. This option is disabled by default. This function cannot be enabled when the copy verification status is Invalid or --.
    • Restoration to a new location
      Table 2 describes the related parameters.
      Table 2 Parameters for restoring data to a new location

      Parameter

      Description

      Target Location

      Select the cloud server to which the backup copy is restored.

      Type

      Select the type of the disk to be restored. You can select Existing or New.

      Target Disk Name

      If Type is set to Existing, set Target Disk Name to the name of the disk to be restored.

      If Type is set to New, -- is displayed by default.

      Password

      Password of the VM to be restored. This parameter is available when you select a system disk.

      Clients

      Select the host where the client is installed for copy-based restoration.

      Power on Automatically After Restoration

      If this function is enabled, the cloud server is automatically started after the restoration is complete. This function is disabled by default.

      Copy Verification Before Restoration

      If this option is enabled, the integrity of a copy is verified before the copy is used for restoration. This operation affects the restoration performance. This option is disabled by default. If no copy verification file is generated, this option cannot be enabled.

  7. Click OK.
