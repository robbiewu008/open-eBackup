Step 1: Obtaining the Keystone V3 Address

This section describes how to obtain the Keystone V3 address of OpenStack when you need to register OpenStack using the address.

Procedure

For FusionSphere OpenStack, perform the following steps to obtain the Keystone V3 address:

  1. Use PuTTY to log in to any OpenStack node using the management IP address as user fsp.
  2. Run the following command to switch to user root:

    su - root

  3. Run the following command to import environment variables, select openstack environment variable (keystone v3), and enter the password:

    source set_env

  4. Run the following command to query the Keystone V3 address:

    openstack endpoint list | grep keystonev3
    When command output similar to the following is displayed, the keystonev3 address whose interface is public is the IP address to be obtained. In the example, https://identity.az236.dc236.huawei.com:443/identity/v3 is the Keystone V3 address to be obtained.
    host02:~ # openstack endpoint list | grep keystonev3
| 6894ac66481e46e493c09b76110ec474 | az236.dc236 | keystonev3    | identityv3     | True    | admin     | https://identity.az236.dc236.huawei.com:443/identity-admin/v3         |
| 70b95d31ae75419980442cfae7362d97 | az236.dc236 | keystonev3    | identityv3     | True    | internal  | https://identity.localdomain.com:8023/identity-admin/v3               |
| 25cb9711e63f420b944d048bddba0d3d | az236.dc236 | keystonev3    | identityv3     | True    | public    | https://identity.az236.dc236.huawei.com:443/identity/v3

For the OpenStack community version, perform the following steps to obtain the Keystone V3 address:

  1. Log in to the management page of the OpenStack community version.
  2. In the navigation tree on the left, choose Admin > System > System Information.
  3. On the Services tab of the System Information page, the value of Public for the keystone service address is the Keystone V3 address.
