安装客户端（自动推送方式，适用于1.6.0及后续版本）

客户端作为OceanProtect与生产环境之间的数据连接组件，在备份和恢复过程中进行流程控制和数据传输。在进行数据备份和恢复前，需在主机上安装客户端。本节介绍自动推送安装包至主机的方式为主机安装客户端。

注意事项

  • 如果待安装客户端的主机的操作系统为Windows Server 2012及之前版本，则不支持通过自动推送方式安装客户端，请使用手动方式安装。具体操作请参考安装客户端（手动方式，适用于Windows OS）。
  • 如果待安装客户端的Windows OS主机为域控制器，且需要在该域上的多台域控制器上安装客户端时，请使用手动方式安装。具体操作请参考安装客户端（手动方式，适用于Windows OS）。

    可在主机上执行wmic computersystem get domainrole /value命令查询主机是否为域控制器，如果回显为DomainRole=4或者DomainRole=5，则表示为域控制器。

  • 受保护资源为GaussDB(DWS)时，如果您不需要配置防火墙，可以执行systemctl stop SuSEfirewall2.service命令关闭防火墙。
  • 不同版本的界面可能存在差异，请以实际为准。

前提条件

  • 请确保主机上未安装客户端。
    • 对于Linux OS、AIX OS或Solaris OS主机，可在主机上执行vi /etc/profile命令，如果文件中不存在“DATA_BACKUP_AGENT_HOME”字段，则表示未安装客户端。
    • 对于Windows OS主机，可在主机上执行sc query rdagent命令查询，如果回显提示指定的服务未安装，则表示未安装客户端。
  • 对于Linux OS、AIX OS或Solaris OS主机，请确保主机上的“/mnt”目录未被挂载。可在主机上执行mount | grep mnt命令查看。
  • 对于Windows OS主机，请确保主机上的“C:\mnt”目录未被挂载。可在主机上执行sc query rdagent命令查询，如果回显提示指定的服务未安装，则表示未安装客户端。
  • 请确保使用的操作系统支持以自动推送方式安装客户端，具体可通过OceanProtect兼容性查询工具查询。
  • 已获取需要安装客户端的主机的IP地址，该IP地址必须与OceanProtect的备份网络连通。
  • 已获取登录主机的用户及其密码。
  • 如果待安装客户端的主机为Windows OS：
    • 请确保已开启Windows远程管理服务，具体操作可参考Windows远程管理的安装和配置。
    • 当使用非系统管理员用户进行客户端安装时，请确保该用户已加入Administrators用户组，具体操作可参考添加用户至Administrators用户组。
  • 部署OceanProtect时，系统会预置相应版本的客户端软件包至OceanProtect管理界面。如果预置的软件包版本不满足您的需求，那么安装客户端前请先上传其他版本的软件包，具体操作请参考上传客户端软件包。
  • 待安装客户端的主机为Linux OS或AIX OS时，如果用户修改了“/etc/ssh/sshd_config”配置文件中的“ListenAddress”参数的值，请确保配置值中包含主机的备份网络IP地址，否则检查连通性失败，无法推送安装。即：“ListenAddress”的值使用系统默认值，或使用用户自行设置的IP地址+主机的备份网络IP地址。
  • 对于待安装客户端的主机为Linux OS时，如果在主机上设置了rm、cp、mv等命令需要交互式输入的别名，请先取消别名设置，否则会导致安装失败。

    例如：在主机上执行alias命令查看设置的命令别名，如果回显提示类似alias mv='mv -i'，表示mv命令设置了别名，请执行unalias mv命令取消mv命令的别名设置。

操作步骤

  1. 待保护资源是否为HDFS、HBase、Hive、ElasticSearch、Redis或ClickHouse集群，且集群中配置了Kerberos认证。

    • 是，安装前要确保待安装客户端的主机、OceanProtect以及Kerberos服务器的时间一致。如果时间不一致，请使用同一NTP服务器，否则可能导致客户端安装失败。
    • 否，安装前要确保待安装客户端的主机与OceanProtect设备时间一致。
      • 对于OceanProtect X系列备份一体机，参考以下步骤查看OceanProtect设备时间：
        1. 选择“系统 > 基础设施 > 集群管理”。
        2. 在“备份集群”页签的“本地集群节点”区域，单击节点名称。
        3. 在弹出的“节点详情”界面，查看当前设备时间。

          如果时间不一致，请修改待安装客户端的主机的时间，否则可能导致客户端安装失败，具体操作请参考如何修改主机时区和时间。

      • 对于OceanProtect E6000备份一体机，参考以下步骤查看OceanProtect设备时间：
        1. 选择“系统 > 基础设施 > 集群管理”。
        2. 在“集群管理”界面，单击“查看集群详情”。
        3. 在弹出的“集群详情”界面，查看当前设备时间。

          如果时间不一致，请修改待安装客户端的主机的时间，否则可能导致客户端安装失败，具体操作请参考如何修改主机时区和时间。

  2. 选择“保护 > 客户端 > 客户端”。
  3. 单击“注册”，配置相关参数。

    参数说明如表1所示。

    表1 安装客户端参数说明

    参数

    说明

    软件包类型
    选择客户端软件包类型。
    • “客户端”：适用于所有待保护资源。
    • “SAN Client”：仅当待保护资源为DB2，且AIX生产环境的组网为FC组网时，需通过SAN Client转接来实现备份，才选择此类型。

    操作系统
    待安装客户端的主机的操作系统类型。
    说明：
    当“操作系统”选择为“Windows”时：
    • 请确保主机上的Windows Remote Management (WS-Management)服务状态为“正在运行”，具体操作可参考如何启动Windows Remote Management (WS-Management)服务。
    • 如果待安装客户端的主机的操作系统为Windows Server 2012及之前版本，则不支持通过自动推送方式安装客户端，请使用手动方式安装。具体操作请参考安装客户端（手动方式，适用于Windows OS）。

    应用类型

    当“软件包类型”选择“客户端”时，请选择客户端支持保护的应用类型。

    消息签名算法

    当“操作系统”选择“Linux”、“AIX”或“Solaris”时，请选择消息签名算法。

    消息签名算法用于校验待安装客户端的主机与本设备之间的连通性。hmac_sha2_256为安全性更高的算法，默认选择该算法，hmac_sha1为不安全算法。
    说明：

    所有参数配置完成后，单击“测试”，当提示待注册客户端不支持指定安全加密算法套件时，请重新选择其他消息签名算法。如果两类算法均不支持，请升级主机的OpenSSH、SSL。

    共享

    开启后将客户端应用于所有数据保护管理员。

    当“软件包类型”选择“客户端”，“操作系统”选择“Linux”，且“应用类型”选择HDFS、HBase、Hive、华为云Stack、华为云Stack GaussDB、GaussDB(DWS)或对象存储时，支持配置此参数。

    IP类型

    待安装客户端的主机的IP地址类型，支持IPv4和IPv6。

    源端重删

    源端重删是删除源端数据中的冗余数据后再进行传输，可以减少传输的数据量，但会增加客户端的资源占用（占客户端主机内存至少4GB，最大CPU使用率30%以上）。不建议开启源端重删。如果网卡带宽不足，即待备份数据量除以备份窗口大于网卡带宽，可以开启源端重删。源端重删功能的详细信息可参考对应版本的《源端重删用户指南》。

    如需开启，请确保待安装客户端的主机的空闲内存大于2GB。
    说明：
    • 如果后续会为该客户端配置LAN-Free，请开启此功能。
    • 开启源端重删后，系统会自动在客户端所在主机安装OceanStor DataTurbo并创建dataturbo用户及用户目录，请勿删除或修改此用户及用户目录。
      • 对于Linux OS、AIX OS，该用户的用户目录为“/home/dataturbo”。
      • 对于Solaris OS，该用户的用户目录为“/export/home/dataturbo”。
      • 对于Windows OS，该用户的用户目录为“C:\Users\dataturbo”。
    • 当主机操作系统为Linux时，开启源端重删功能后，请在首次备份前将源端重删用户加到sudoers中，具体操作请参见添加源端重删用户到sudoers（适用于Linux OS）。
    • 如果安装客户端时未开启源端重删功能，在客户端安装完成后也可手动安装OceanStor DataTurbo，具体操作请参考安装OceanStor DataTurbo。

    网络平面类型
    当“操作系统”选择“Linux”时，请选择待安装客户端的网络平面类型。支持选择为“管理网络”或“备份网络”。
    说明：
    • 选择“管理网络”时，OceanProtect将通过管理网络将客户端软件包推送至主机，并为主机自动安装客户端。
      • 如果填写了备份网络IP地址，则OceanProtect将通过备份网络为主机自动安装客户端。
      • 如果未填写备份网络IP地址，则OceanProtect将自动选择与OceanProtect连通的网络为主机自动安装客户端。
    • 选择“备份网络”时，OceanProtect将通过备份网络将客户端软件包推送至主机，并通过备份网络为主机自动安装客户端。
    • 如需指定备份资源时使用的网络，则建议使用主机的备份网络IP地址进行注册。

    管理网络IP地址

    当“网络平面类型”选择“管理网络”时，请输入待安装客户端的主机的管理网络IP地址。

    备份网络IP地址
    待安装客户端的主机的备份网络IP地址。填写后则使用该IP地址进行注册。如果不填写，则会检测业务端口IP连通性，并自动选择网络连通的IP地址进行注册。
    说明：

    当“网络平面类型”选择“管理网络”时，才显示此参数。

    备份网络IP地址（地址段）
    请输入待安装客户端的主机的备份网络IP地址或地址段。
    说明：
    • 对于Linux OS，当“网络平面类型”选择“备份网络”时，才显示此参数。
    • 仅IPv4支持输入地址段，格式为“起始IP地址-结束IP地址”。示例：10.10.10.1-10.10.10.10。

    端口
    待安装客户端的主机的SSH端口或SFTP端口。
    • 操作系统为“Windows”时，请使用Windows远程管理服务侦听端口，默认值为“5985”。
    • 其他操作系统类型，请使用SSH服务侦听端口，默认值为“22”。
    说明：
    • SSH服务侦听端口号与SFTP服务侦听端口号需保持一致，否则会导致注册失败。
    • 在待安装客户端的主机上执行winrm enumerate winrm/config/listener命令可查询Windows远程管理服务侦听端口，安装Windows远程管理服务的具体操作可参考Windows远程管理的安装和配置。

    是否作为业务IP
    当“操作系统”为“Windows”时，可以选择是否将待安装客户端的主机的IP地址作为业务IP。
    • 如果选择“是”，则使用该IP地址进行注册。默认为“是”。
    • 如果选择“否”，则会检测业务端口IP连通性，并自动选择网络连通的IP地址进行注册。

    用户类型
    当“操作系统”选择“Linux”或“AIX”时，请选择待安装客户端的主机的用户类型。
    • “超级管理员”：具有执行sudo命令权限的用户，注册主机时，只需要输入登录主机的用户名和密码。
    • “普通用户”：不具有执行sudo命令权限的用户，注册主机时，需要输入sudo密码、登录主机的用户名和密码。
    说明：

    当“用户类型”为“普通用户”时，需要先将该普通用户添加到sudoers中，否则安装客户端将失败，具体操作步骤可参考添加普通用户至sudoers（适用于Linux OS）。

    sudo免密

    当“用户类型”为“普通用户”时，可以选择是否使用sudo免密，即普通用户在使用sudo权限时是否需要输入当前用户的密码。

    sudo密码

    当“用户类型”为“普通用户”时，需要输入sudo密码。

    用户名
    待安装客户端的主机的用户名及密码。
    说明：
    • 当“操作系统”选择“Windows”时，输入的用户名请勿包含域名。

      例如：域账户为CHINA\Administrator，仅输入Administrator即可。

    • 当“操作系统”选择“AIX”，且“用户类型”为“普通用户”时，请确保输入的用户存在对应的用户目录。查询用户目录的操作如下：
      1. 使用PuTTY，登录待安装客户端的主机。
      2. 执行以下命令，查询是否存在用户目录。
        cd /home/用户名

        如果回显No such file or directory，则表示没有对应的用户目录，请执行3创建。否则表示存在对应的用户目录。

      3. 执行以下命令，创建用户目录。
        mkdir /home/用户名

    密码

    安装目录

    客户端的安装目录。

    • 操作系统为Linux、AIX或Solaris时，默认安装在“/opt”目录下。不支持安装在/tmp、/dev、/bin、/usr及根目录下，不能有[|;&$><`\!]特殊字符。
    • 操作系统为Windows时，默认安装在“C:\”目录下。
    说明：
    • 系统会自动校验填入的安装目录的有效性，选择的安装目录权限需为755，且用户和属组为root。
    • 安装目录会保存在“/etc/profile”配置文件中，环境变量为“DATA_BACKUP_AGENT_HOME”，请不要随意更改此环境变量。

    自动同步主机名称

    注册时客户端名称使用主机上报的名称。

    开启后，OceanProtect管理界面会自动同步主机名称的修改。关闭后，已注册的客户端的名称不自动同步主机名称但支持修改。

  4. 单击“测试”。

    系统验证填写的IP地址、用户名、密码是否正确，一次最多支持测试300条客户端信息。
    • 在“客户端信息”区域，单击某条客户端信息后的“修改”或“删除”，可以修改或删除对应的客户端信息。
    • 在“客户端信息”区域，勾选多条客户端信息，单击“批量修改用户名和密码”，可以批量修改用户名和密码。

  5. 测试成功后单击“确定”。

    系统开始自动安装客户端，一次最多支持为300个主机安装客户端。当“客户端”页面显示安装了客户端的主机后，表示该主机已成功注册至管理界面。
    • 主机注册成功后请等待5分钟再使用该客户端。
    • 客户端安装完成后，系统会自动在客户端所在主机创建rdadmin用户及用户目录，请勿删除或修改此用户及用户目录。
      • 对于Linux OS、AIX OS，该用户的用户目录为“/home/rdadmin”。
      • 对于Solaris OS，该用户的用户目录为“/export/home/rdadmin”。
      • 对于Windows OS，该用户的用户目录为“C:\Users\rdadmin”。
