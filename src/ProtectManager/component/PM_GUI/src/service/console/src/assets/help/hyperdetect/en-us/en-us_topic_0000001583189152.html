Replacing the NTP Certificate

When a storage device communicates with an NTP server, the storage device functions as a client and the NTP server functions as a server (NTP synchronization clock source). In this scenario, the CA certificate must be imported, but the client certificate cannot be imported.

Context

In the NTP scenario, no default CA certificate or certificate revocation list (CRL) exists, and CRL is not supported.

Procedure

  1. Obtain the certificate from the NTP server and copy it to the maintenance terminal.

    1. On the maintenance terminal, press Win+R.

      The Run dialog box is displayed.

    2. Enter \\NTP server IP address and click OK.

      The maintenance terminal attempts to remotely access the NTP server.

    3. Type the user name and password of the NTP server and click OK to enter the shared directory.
    4. Enter the ntp_config folder, select the NTP certificate file that contains the ntpkey_cert field, and press Ctrl+C to copy the certificate file.
    5. Go back to the maintenance terminal desktop and press Ctrl+V to copy the NTP certificate file to the maintenance terminal.
    6. Right-click the NTP certificate file and then select Rename from the shortcut menu. Add the .crt extension to the file name, and click Enter.

  2. Import the certificate to the storage device on DeviceManager.

    For details, see 3. Change the operation of selecting Management plane domain authentication certificate to selecting NTP certificate.
