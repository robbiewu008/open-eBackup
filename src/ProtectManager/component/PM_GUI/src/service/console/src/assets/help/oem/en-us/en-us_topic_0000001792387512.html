Managing Protection for Clusters/Hosts/VMs/VM Groups

The product allows you to modify, remove, activate, or disable protection plans for clusters, hosts, VMs, or VM groups. (Only 1.6.0 and later versions support management of protection for VM groups.) This section uses a VM as an example to describe the related operations. The operations for a cluster, host, or VM group are the same.

You need to log in to the management page, choose Protection > Virtualization > FusionCompute, and find the target VM.

Table 1 describes the related operations.

Table 1 Operations related to VM protection plans

Operation

Description

Navigation Path

Modify Protection

Context

You can modify a protection plan of a VM to associate the SLA with the VM again, add or remove a protected VM disk, or modify a ProtectAgent host. After the modification, the next backup will be executed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.
  • If a backup job has been performed, reconfigure the SLA policy (change to the SLA with copy verification enabled) when modifying the protection plan. After the modification, the initial backup job will be converted to a full backup job if it is a non-full backup job.
  • If the SLA parameter associated with the resource changes (from copy verification disabled to copy verification enabled) after a backup job has been performed, when the protection plan for the resource is implemented after the change, the initial backup job will be converted to a full backup job if it is a non-full backup job.

In the row where the VM resides, choose More > Modify Protection.

Remove Protection

Context

You can remove the protection if the VM no longer needs to be protected. After the removal, the system cannot execute protection jobs for the VM. To protect the VM again, you need to associate an SLA with the VM again.

Note

When the protection job of the VM is running, the protection cannot be removed.

In the row where the VM resides, choose More > Remove Protection.

Activate Protection

Context

If the protection plan of a VM is disabled, you can perform this operation to activate it again. After the protection plan is activated, the system backs up or replicates the VM data based on the protection plan.

In the row where the VM resides, choose More > Activate Protection.

Disable Protection

Context

This operation allows you to disable the protection plan of a VM when periodic protection on the VM is not required. After the protection plan is disabled, the system does not automatically back up or replicate the VM.

Note

Disabling a protection plan does not affect protection jobs that are running.

In the row where the VM resides, choose More > Disable Protection.

Manual Backup

Context

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row where the VM resides, choose More > Manual Backup.

Note: The procedure for restoring FusionCompute VMs is the same as that described in Restoration.
