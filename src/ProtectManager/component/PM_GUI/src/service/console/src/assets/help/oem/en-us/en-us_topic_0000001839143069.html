Step 5: Registering OpenStack

Before backing up and restoring OpenStack, register OpenStack with the product.

Procedure

  1. Choose Protection > Cloud Platforms > OpenStack.
  2. Click Register to register OpenStack.

    Table 1 describes the OpenStack registration information.
    Table 1 OpenStack registration information

    Parameter

    Description

    Name

    User-defined domain name.

    Keystone V3 Address

    Keystone V3 address of OpenStack For details about how to obtain the address, see Preparing for the Backup.

    Username

    User for interconnecting OpenStack with the product. Create a user by referring to Step 3: Creating a User for Interconnection.

    Password

    Agent Host

    Agent host used for OpenStack data protection.

    Multiple agent hosts can be selected for resource scanning, backup, and restoration. If no agent host is configured during backup and restoration, the system selects one from the agent hosts configured here for backup and restoration.

    Certificate Verification

    This function is enabled by default. When registering an OpenStack protected environment, you are advised to import the CA certificate of the protected environment. Otherwise, the system cannot verify the information about the accessed devices, resulting in security risks. After this function is disabled, security risks may exist in the communication between the system and the cloud platform. Exercise caution when performing this operation.

    Certificate

    After Certificate Verification is enabled, import the certificate obtained in Step 2: Obtaining the Certificate.

    Revocation List

    After the CRL is imported, the system checks whether the certificate on the production site has been revoked. If the certificate has been revoked, cluster registration fails.

  3. Click OK.
