注册数据库（适用于集群部署形态）

当要保护的数据库为集群部署时，请参考本节注册Oracle数据库集群。

前提条件

请确保客户端的安装用户能够正常的通过su - oracle命令切换到Oracle的安装用户。如果安装了grid组件，请确保能够通过su - grid命令切换到grid用户。

操作步骤

  1. 选择“保护 > 数据库 > Oracle”。
  2. 在“集群”页签，单击“注册”。

    相关参数说明如表1所示。

    表1 注册集群参数说明

    参数

    说明

    名称

    自定义集群名称。

    类型

    数据库集群类型。当前仅支持RAC集群。

    节点

    选择集群中的所有节点。

  3. 单击“确定”。
  4. 在“数据库”页签，单击“注册”。

    相关参数说明如表2所示。

    表2 注册数据库参数说明

    参数

    说明

    类型

    数据库类型，选择“单机”或“集群”。

    • 数据库部署形态为集群时，请选择“集群”。
    • 数据库部署形态为单机时，请选择“单机”。

    名称

    要保护的数据库名称。

    该名称与生产环境的数据库名称保持一致。一般数据库的name与db_unique_name一致，当两者不一致时，请配置为db_unique_name。可通过以下SQL语句查看name与db_unique_name。
    select name,db_unique_name FROM v$database;

    主机/集群

    数据库所在的主机或集群。

    运行数据库用户名

    当数据库所在主机的操作系统为Windows时，请联系数据库管理员获取数据库所在主机的系统管理员的用户名。

    运行数据库用户密码

    当数据库所在主机的操作系统为Windows时，请联系数据库管理员获取数据库所在主机的系统管理员用户的密码。

    数据库认证方式

    数据库认证方式，支持“数据库认证”和“OS认证”，与数据库侧的配置一致。

    查询数据库认证方式可参考备份前准备。

    说明：
    • 当要备份的数据库所在的主机为Oracle ADG或DG主备部署形态时，备端的数据库必须采用数据库认证方式。
    • 当在4中指定多个客户端所在主机执行备份任务时，则要求数据库必须采用数据库认证方式。
    • 在集群场景下，如果不同的数据库节点查询到的数据库认证方式有差异，此处请配置为“数据库认证”。

    数据库安装用户名

    安装数据库的用户，如oracle。获取方式可参考备份前准备。

    ORACLE_HOME路径

    说明：

    仅1.6.0及后续版本支持设置该参数。

    仅当数据库节点为Linux操作系统时，界面显示该参数。

    ORACLE_HOME为Oracle软件的环境变量，如果数据库节点安装用户未配置ORACLE_HOME环境变量，需手动配置，配置方法如下（当数据库类型为集群时，请在所有节点执行以下步骤）：

    1. 登录数据库节点，查询环境变量中是否配置ORACLE_HOME：
      su - oracle -c 'env | grep ORACLE_HOME'
      • 若回显不为空，表示已配置ORACLE_HOME环境变量，则无需在管理界面中配置该参数。
      • 若回显为空，请继续执行后续操作。
    2. 手动配置ORACLE_HOME环境变量。
      • 若支持修改环境变量配置文件，请执行以下操作：
        请联系数据库管理员获取ORACLE_HOME路径，在数据库安装用户所在目录下编辑环境变量配置文件（如.bash_profile），在最后一行增加该路径。例如：
        export ORACLE_HOME=/XXX

        保存环境变量配置文件，执行以下命令使环境变量生效。

        source ~/.bash_profile
      • 若不支持修改环境变量配置文件，请联系数据库管理员获取ORACLE_HOME路径并填写到管理界面。

    ORACLE_BASE路径

    说明：

    仅1.6.0及后续版本支持设置该参数。

    仅当数据库节点为Linux操作系统时，界面显示该参数。

    ORACLE_BASE为Oracle软件的环境变量，如果数据库节点安装用户未配置ORACLE_BASE环境变量，需手动配置，配置方法如下（当数据库类型为集群时，请在所有节点执行以下步骤）：

    1. 登录数据库节点，查询环境变量中是否配置ORACLE_BASE：
      su - oracle -c 'env | grep ORACLE_BASE'
      • 若回显不为空，表示已配置ORACLE_BASE环境变量，则无需在管理界面中配置该参数。
      • 若回显为空，请继续执行后续操作。
    2. 手动配置ORACLE_BASE环境变量。
      • 若支持修改环境变量配置文件，请执行以下操作：
        请联系数据库管理员获取ORACLE_BASE路径，在数据库安装用户所在目录下编辑环境变量配置文件（如.bash_profile），在最后一行增加该路径。例如：
        export ORACLE_BASE=/XXX

        保存环境变量配置文件，执行以下命令使环境变量生效。

        source ~/.bash_profile
      • 若不支持修改环境变量配置文件，请联系数据库管理员获取ORACLE_BASE路径并填写到管理界面。

    数据库用户名

    当“数据库认证方式”为“数据库认证”时，需要配置数据库的用户名和密码。

    输入具有sysdba权限的用户。

    说明：

    如果数据库侧配置为OS认证方式，但注册时“数据库认证方式”选择了“数据库认证”，由于数据库本身的机制，不会校验数据库的用户名和密码。

    数据库密码

    ASM认证

    如果数据库使用了ASM，则需要开启ASM认证。

    查看数据库是否使用了ASM的具体操作可参考备份前准备。

    认证方式

    ASM认证方式，与数据库侧的配置一致。

    查询数据库侧的ASM认证方式可参考备份前准备。

    说明：

    在集群场景下，如果不同的数据库节点查询到的ASM认证方式有差异，此处请配置为“ASM认证”。

    安装用户名

    ASM的安装用户名，如grid。

    ASM用户名

    当认证方式为ASM认证时，需要配置该参数。输入具有sysdba权限的用户。

    ASM密码

  5. 在“存储资源”区域，单击“添加”。存储资源参数说明如表3所示。

    • 仅1.6.0及后续版本支持此功能。
    • 当Oracle数据库对接了OceanStor Dorado 6.x存储设备，需使用存储层快照备份功能时，请添加存储资源。其他场景无需添加存储资源。
    • 当Oracle数据库对接OceanStor Dorado 6.x双活存储设备时，可添加2台存储资源，备份恢复任务优先使用最先添加的存储资源。
    表3 添加存储资源参数说明

    参数

    说明

    设备类型

    存储资源的类型。

    管理IP地址

    支持填写存储设备的多个管理IP地址。某个管理IP地址无法访问时，系统将自动切换到其他可访问的管理IP地址。

    端口

    存储资源的访问端口，默认是8088。

    用户名

    访问存储资源的用户名和密码。

    说明：

    请使用管理员角色的用户，该用户具有Restful的登录方式，且已完成密码初始化，否则可能导致备份该设备下的资源失败。

    密码

    协议

    选择Oracle数据库主机对接存储资源时使用的数据传输协议。

    校验证书

    默认开启，注册数据库时会对存储资源的证书进行校验，确保OceanProtect与外部设备交互的安全性。关闭后，系统与存储设备通信可能存在安全风险，请谨慎操作。

    证书

    请导入步骤2：获取存储资源CA证书（适用于存储层快照备份）章节中获取的存储资源的CA证书。

    吊销列表

    导入证书吊销列表后，系统将校验客户端证书是否被吊销，如果证书已被吊销，则注册存储设备失败。
