Registering a Database (Applicable to the Primary/Standby Deployment Mode)

Before backing up or restoring databases at both the primary and standby ends, you need to register the database clusters or single-server databases to be protected with the product.

Prerequisites

Ensure that the client installation user can be switched to the Oracle installation user by running the su - oracle command. If the grid component is installed, ensure that you can run the su - grid command to switch to user grid.

Procedure

  1. Choose Protection > Databases > Oracle.
  2. Register the databases to be protected at the primary end.

    • If a database to be protected at the primary end is deployed in cluster mode, register the database by referring to Registering a Database (Applicable to the Cluster Deployment Mode).
    • If a database to be protected at the primary end is deployed in a single-server system, register the database by referring to Registering a Database (Applicable to the Single-Server System Deployment Mode).

  3. Register the databases to be protected at the standby end.

    • If a database to be protected at the standby end is deployed in cluster mode, register the database by referring to Registering a Database (Applicable to the Cluster Deployment Mode).
    • If a database to be protected at the standby end is deployed in a single-server system, register the database by referring to Registering a Database (Applicable to the Single-Server System Deployment Mode).
