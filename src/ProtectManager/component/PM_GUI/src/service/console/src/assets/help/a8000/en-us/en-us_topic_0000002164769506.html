Domain Account Login Fails After Object-Level Restoration of a Computer in Active Directory Is Performed

Symptom

After object-level restoration of a computer in Active Directory, the domain account cannot be used for login, with the error message "The trust relationship between this workstation and the primary domain failed." displayed.

Possible Causes

After object-level restoration of a computer in Active Directory, you need to re-verify the trust relationship between the domain account and the local host.

Procedure

The following steps use Windows 11 as an example. The operations may vary depending on the version.

  1. Log in to the local host as Administrator.
  2. Open a folder and select This PC from the left navigation pane.
  3. Right-click and choose Properties from the shortcut menu.
  4. On the displayed tab page, click Change.
  5. Select Workgroup and enter WorkGroup.
  6. Click OK and restart the computer.
  7. Add the local host to the specified domain.

    1. Log in to the host as Administrator.
    2. Open a folder and select This PC.
    3. Right-click and choose Properties from the shortcut menu.
    4. On the displayed tab page, click Change.
    5. Select Domain and enter the specified domain.

  8. Log in to the local host again using the original domain account.
