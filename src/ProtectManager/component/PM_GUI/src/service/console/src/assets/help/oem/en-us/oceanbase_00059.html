Managing Clusters

The product allows you to modify, remove, activate, or disable protection for clusters.

Related Operations

Log in to the management page, choose Protection > Databases > OceanBase, click Clusters, and find the target cluster.

Table 1 describes the related operations.

Table 1 Cluster protection operations

Operation

Description

Navigation Path

Modify Protection

Scenario

This operation enables you to modify a protection plan to perform SLA association again for a cluster or modify user-defined script information. After the modification, the next backup will be performed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row of the target cluster, choose More > Modify Protection.

Remove Protection

Scenario

This operation enables you to remove the protection plan for a cluster when it does not need to be protected. After removal, the system cannot execute protection jobs for the cluster.

Note

  • When a protection job of the database is running, the protection cannot be removed.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the associated SLA on the Protection > Protection Policies > SLAs page, rather than perform SLA re-association after protection removal.

    After you remove protection and perform SLA re-association, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row of the target cluster, choose More > Remove Protection.

Disable Protection

Scenario

This operation enables you to disable the protection plan of a cluster for which periodic protection is not required. After the protection plan is disabled, the system does not perform automatic backup jobs for the cluster.

Note

Disabling protection does not affect ongoing backup jobs.

In the row of the target cluster, choose More > Disable Protection.

Activate Protection

Scenario

This operation enables you to activate a disabled protection plan of a cluster. After the protection plan is activated, the system backs up the cluster based on the protection plan.

Note

This operation can be performed only on instances that have been associated with an SLA and are in the Unprotected state.

In the row of the target cluster, choose More > Activate Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target cluster, choose More > Manual Backup.

Test Connectivity

Scenario

This operation enables you to test the connectivity between the product and cluster.

In the row of the target cluster, choose More > Test Connectivity.

Modifying cluster information

Scenario

This operation enables you to modify the information such as the cluster name, OBClient host, and OBServer host.

In the row of the target cluster, choose More > Modify.

Deleting a cluster

Scenario

This operation enables you to delete a cluster that is no longer used.

Note

Before deleting a cluster, remove protection from the cluster.

In the row of the target cluster, choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note:
