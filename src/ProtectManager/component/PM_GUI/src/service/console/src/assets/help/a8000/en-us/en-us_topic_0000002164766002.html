The OceanProtect Service Is Interrupted or FusionCompute Fails to Be Registered

Symptom

If the user type for connection between FusionCompute and the OceanProtect is set to Interface interconnection user or Local user, services are interrupted or registration fails.

Possible Cause

  • Cause 1: FusionCompute supports the configuration of Man-Machine and Machine-Machine Interface Access Isolation Policy. If the current value of the policy is Yes, the interface interconnection user cannot access FusionCompute through port 7443.
  • Cause 2: If a local user is used, you need to change the password upon the first login to FusionCompute.

Troubleshooting

  • For cause 1, perform the following steps:
    1. Log in to FusionCompute using a browser.

      Login address: https://Floating IP address of the VRM node of FusionCompute:8443

    2. Choose System Management > Rights Management > Rights Management Policy.
    3. In the Other Strategies area, click Modify.
    4. Change the value of Man-Machine and Machine-Machine Interface Access Isolation Policy to No.
    5. Click Save.
  • For cause 2, perform the following steps:
    1. Log in to FusionCompute as a local user.
    2. Change the login password as prompted.
    3. Use the new password to register FusionCompute with the OceanProtect again.
