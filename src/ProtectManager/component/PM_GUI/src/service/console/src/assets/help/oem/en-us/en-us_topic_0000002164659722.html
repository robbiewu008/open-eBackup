Preparations for Backup

Before the backup, prepare related information by following instructions in Table 1.

  • Unless otherwise specified, the operations in this section use Dameng 8 as an example. The operations may vary according to the database version.
  • If the Dameng database is deployed in a cluster, you need to log in to all hosts where the cluster instances reside and perform the following operations unless otherwise specified.

How to Obtain

To Be Used In

  • bin directory path under the database installation directory
  • Path of the dm.ini configuration file

Perform the following operations to query the name of the database installation user. Contact the database administrator to obtain the password of the installation user.

  1. Use PuTTY to log in to the host where the Dameng database is located and switch to user root.
  2. Run the following command to query the name of the database installation user, bin directory path under the database installation directory, and path to the dm.ini configuration file
    ps -ef | grep dm.ini

    Information similar to the following is displayed. In the command output, /dm8/data/DMServer/DAMENG/ indicates the path to the dm.ini configuration file, dmdba in the line of the dm.ini file path indicates the database installation user, and /dm8/bin indicates the bin directory path under the database installation directory.

    [root@dameng16543 ~]# ps -ef | grep dm.ini

Step 3: Registering the Dameng Database

Listening port of the database
  1. Use PuTTY to log in to the host where the Dameng database is located and switch to user root.
  2. Run the following command to query the port number:
    cat Path to the dm.ini configuration file/dm.ini | grep PORT_NUM
    NOTE:

    Replace Path to the dm.ini configuration file in the command with the query result in 2.

    Information similar to the following is displayed. The value of PORT_NUM is the listening port of the database. 
                    PORT_NUM                        = 5236                  #Port number on which the database server will listen

Database authentication method
  1. Use PuTTY to log in to the host where the Dameng database is located and run the following command to switch to the database installation user, for example, dmdba:
    su - dmdba
  2. Go to the bin directory under the database installation directory, for example, /dm8/bin.
    cd /dm8/bin
    NOTE:

    Replace the bin directory path under the database installation directory with that obtained in 2.

  3. Run the following command to log in to the Dameng database:
    ./disql Name of the database installation user/Password of the database installation user@LOCALHOST:Listening port of the database

    Example:

    ./disql SYSDBA/SYSDBA@LOCALHOST:5236
  4. Run the following command to check the value of VALUE:
    select name, value from v$parameter where name = 'ENABLE_LOCAL_OSAUTH';
    • If the value is 1, OS authentication is used.
    • If the value is 0, OS authentication is not used.
