Constraints

  • Backup and restoration jobs cannot be executed for a resource at the same time.
  • Live mount and instant recovery are not supported.
  • If an expiration or deletion job is triggered for a copy, the copy cannot be used for replication, archive, and restoration jobs.
  • Nutanix VMs cannot be registered using the Prism Central (PC) platform. They can be registered only using the Prism Element (PE) platform.
  • For the OceanProtect X3000/X6000 Appliance, copies with more than 20 million files cannot be indexed. For the OceanProtect X8000/X9000 Appliance, copies with more than 50 million files cannot be indexed. For OceanProtect E1000, copies with more than 30 million files cannot be indexed.
  • The constraints on file-level restoration are as follows:
    • The supported OSs are Linux and Windows. Disk backup files in RAW and QCOW2 formats are supported. External disks that cannot generate backup files are not supported.
    • The copy must contain the backup file of the disk where the OS resides. Otherwise, the mount point or disk drive information cannot be obtained, causing a mount failure. As a result, the directory and absolute path of a file cannot be obtained.
    • The supported Linux file system types are ext2/3/4 and XFS, and the supported Windows file system types are NTFS, FAT, and FAT32.
    • swap (Linux swap partition), BitLocker (Windows drive encryption), and ReFS (Windows recovery file system) cannot be parsed.
    • One logical volume manager can correspond to multiple physical volumes. Logical volumes can be parsed only when the disks where multiple physical volumes reside are backed up.
    • For Linux:
      • In Linux, only the file systems whose mount points are written into fstab can be parsed. If the device bus type is virtio, only the file systems whose mount points are written into fstab in UUID form can be parsed.
      • File systems that are not mounted persistently cannot be parsed. Paths starting with /proc, /sys, /dev, /lib/udev/devices, and /invisible are ignored and not parsed.
    • For Windows:
      • File systems can be parsed only when the disks or partitions where such file systems reside are mapped to a drive (for example, C:\). If the drive mapping cannot be determined, the related file systems cannot be parsed. Removable disks (such as DVD ROM) are ignored and not parsed.
      • Spanned, mirrored, striped, and RAID 5 volumes cannot be parsed.
    • Only copies of a single OS can be parsed.
    • The number of files to be restored at a time cannot exceed 200,000.
