Step1: Enabling the Archive Mode (Applicable Only to CMDB)

Before backing up a CMDB database, you must enable the archive mode. Otherwise, the backup fails.

Procedure

  1. Use PuTTY to log in to the CMDB database host.
  2. Run the following command to switch to the database installation user. The following uses user omm as an example.

    su - omm

  3. Run the following command to create a path for storing archive logs (WAL logs). The /database/panweidb/archive/ path is used as an example in subsequent operations.

    mkdir -p /database/panweidb/archive/

  4. Log in as the database administrator.

    1. On the host, open the postgresql.conf file. The /database/panweidb/data/postgresql.conf path is used as an example.
      vi /database/panweidb/data/postgresql.conf

      In the following command output, the queried port number is displayed in the row where port is located. Port number 17700 is used as an example.

    2. Run the following command to log in as database administrator postgres:
    gsql -d postgres -p 17700

  5. Run the show config_file; command to query the path to the postgresql.conf file.

    The following is an example of the command output:
    postgres=# show config_file;
              config_file
---------------------------------------
/database/panweidb/data/postgresql.conf
(1 row)

  6. On the database host, open the postgresql.conf file. The /database/panweidb/data/postgresql.conf path is used as an example.

    vi /database/panweidb/data/postgresql.conf

  7. Modify the wal_level, archive_mode, and archive_command parameters in the postgresql.conf file as follows:

    wal_level = hot_standby                 # minimal, archive, hot_standby or logical
archive_mode = on               # enables archiving; off, on, or always
archive_command = 'cp %p /database/panweidb/archive/%f'          # command to use to archive a logfile segment
    • Set wal_level to archive, hot_standby, or logical.
    • Set archive_mode to on.
    • Set archive_command to 'cp %p /mnt/server/archivedir/%f'.
    • When modifying the postgresql.conf file, change the values of existing fields in the file. Do not add the same fields to the file. Otherwise, the restoration job will be affected.
    • When enabling the archive mode, you need to set the same archive log path on all active and standby nodes of the database.

  8. Run the following command to restart the database:

    pw_ctl restart -D /database/panweidb/data

    Database services will be interrupted during the restart. Exercise caution when performing this operation.
