Step 4: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • During the execution of a manual backup job, the parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies and Automatic Indexing, will take effect.
  • Copies generated by manual backup are retained for the duration defined in the SLA.

Procedure

  1. Choose Protection > File Systems > NAS File Systems.
  2. In the row of the target NDMP NAS file system, choose More > Protect.
  3. Select the created SLA.

    You can also click Create to create an SLA.

  4. Select the NDMP share protocol.
  5. Select the agent host.

    By default, the built-in agent is selected. If an external agent exists, you can also select the external agent.

  6. Enter the files or directories to be filtered.

    The entered names of files or directories to be filtered during backup must meet the following requirements:

    • Complete paths are not allowed.
    • The asterisk (*) is a wildcard, which must be the first or last character of a file name or directory name.
    • Each character string can contain a maximum of two asterisks (*).
    • If a file name or directory name contains commas, ensure that a backslash (\) precedes each comma.
    • A maximum of 32 names are allowed.
    • A single file name or directory name can contain a maximum of 255 characters.
    Table 1 Advanced parameters of NDMP NAS file system protection

    Parameter

    Description
    Auto Indexing of Backup Copies
    NOTE:

    Only 1.6.0 and later versions support this parameter.
    After this function is enabled, you can search for files, directories, or links in copies.
    NOTE:

    Indexes occupy backup storage space. If the number of index files in the system exceeds 1 billion, perform operations by referring to "Failed to Create a Copy Index" in the OceanProtect Appliance 1.5.0-1.6.0 Troubleshooting.

  7. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  8. Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row of the target NAS file system, choose More > Manual Backup.
      1. You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup.
    2. Set the name of the copy generated by the manual backup.
    3. If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.
    4. Click OK.
