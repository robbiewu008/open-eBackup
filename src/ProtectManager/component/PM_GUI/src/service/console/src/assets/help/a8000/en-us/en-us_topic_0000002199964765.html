Viewing Copy Information About a Common Share Resource

This section describes how to view details about a common share resource copy.

Procedure

  1. Choose Explore > Copy Data > File Systems > Common Shares.
  2. View copy information.

    • Replication copies can be viewed only in the target cluster.
    • Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.
    • Click the Resources tab to view the common share copy information by common share resource. When you need to configure share information for a damaged common share resource, you are advised to search for the copies at the corresponding point in time through this entry.

      Table 1 describes the parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Name

      Name of a common share resource.

      Location

      Name of the host where the common share resource is located.

      Status

      Whether the common share resource has been deleted.

      • If yes, Status is Not existing.
      • If no, Status is Existing.

      Copies

      Quantity

      Number of backup copies, archive copies, or replication copies of a common share resource in the current cluster.

      Replication Copy SLA

      SLA associated with replication copies.

      If an SLA has been configured for replication copies generated by the resource, the SLA name is displayed.

      Replication Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is activated.
      • Creating: The replication copy is associated with an SLA and protection is being created. The status is available only in 1.5.0.
    • Click the Copies tab to view all copy information by common share copy. When you need to configure share information by using a specified copy, you are advised to search for the copy through this entry.

      Table 2 describes the parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copies

      Name

      Copy name.

      Backup Copy Generation Time

      Time when a copy is generated.

      Click Copy Time to view the copy information such as expiration time and verification status. If -- is displayed for the expiration time, the copy is retained permanently.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Deletion failed: The copy fails to be deleted.
      • Sharing: The copy is being shared.
      • Downloading: The copy is being downloaded.

      Location

      Location of the copy.

      • If Generation Mode of a copy is Backup, Cascaded replication, Reverse replication, or Replicate, the copy is stored in the corresponding backup storage unit. In this case, the name of the corresponding backup storage unit is displayed.
      • If Generation Mode of a copy is Archive to object storage or Import, the copy is stored in an object storage bucket. In this case, the bucket name is displayed.
      • If Generation Mode of a copy is Archive to tape, the copy is stored in a tape library. In this case, the media set name is displayed.

      Storage Media Status
      Status of the storage medium where the copy resides.
      • Online: The copy can be managed.
      • Offline: The copy can be deleted only.
      NOTE:

      Storage Media Status is Online for backup storage units whose Health Status is Normal or Degraded and Running Status is Pre-Copy, Rebuilt, Online, Balancing, or Degraded.

      Generation Mode
      Mode in which a copy is generated.
      • Replication: copies generated by replication jobs.
      • Backup: copies generated by backup jobs.
      • Archive to object storage: copies generated by object storage archive jobs.
      • Archive to tape: copies generated by tape archive jobs.
      • Live mount: copies generated by live mount jobs.
      • Download: copies generated by download jobs.
      • Snapshot: copies generated by snapshot jobs.
      • Import: copies imported by archive jobs.
      • Cascaded replication: copies generated by cascaded replication jobs.
      • Reverse replication: copies generated by reverse replication jobs.

      Copy Generation Time

      Time when a copy is generated. Click the copy generation time to view the details about the copy.

      Copy Type

      Backup type of a generated copy. The value can be Full or Forever Incremental (Synthetic Full).

      Copy Expiration Time

      Copy expiration time. If a copy is permanently valid, -- will be displayed.

      WORM
      Whether a copy has been set as a WORM copy.
      • : Disabled. The copy has not been set as a WORM copy, that is, the copy is a general copy.
      • : Setting. The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : Enabled. The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : Failed. The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.
      • : The copy has expired. The WORM copy has expired. Copies in this state can be deleted.

      WORM Expiration Time
      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.
      NOTE:

      If the WORM expiration time is earlier than the WORM file system compliance clock time, the WORM expiration time has expired. You can click the copy time to view the WORM file system compliance clock time. If the WORM file system compliance clock time is inconsistent with the system time, check the WORM status after the WORM file system compliance clock reaches the set expiration time.

      Resources

      Name

      VM name.

      Location

      Working location of the VM.

      Status
      Whether the VM is registered with the OceanProtect.
      • If yes, the status is Existing.
      • If no, the status is Not existing.

      Tag

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
