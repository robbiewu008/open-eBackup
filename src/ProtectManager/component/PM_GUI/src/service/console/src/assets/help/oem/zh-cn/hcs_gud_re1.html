恢复弹性云服务器中的文件

当需要恢复弹性云服务器中的某些文件时，可以通过文件级恢复功能实现。

背景信息

  • 本产品支持使用备份副本、复制副本（不支持恢复至原位置）进行文件级恢复。
  • 支持恢复的Linux文件系统类型为ext 2/3/4、XFS。支持恢复的Windows文件系统类型为NTFS、FAT、FAT32。

前提条件

  • 已为待恢复到的目标弹性云服务器配置备份网络平面IP地址。
  • 当目标虚拟机为Windows OS时，请确保目标虚拟机已开启CIFS服务，具体操作请参考Microsoft官方网站。
  • 当目标虚拟机为非Windows OS时，请确保目标虚拟机已安装并开启rsync。

操作步骤

  1. 选择“数据利用 > 副本数据 > 云平台 > 华为云Stack”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 找到待恢复的副本，单击其右侧的“更多 > 文件级恢复”。

    • 对于1.5.0版本，副本的索引状态必须为“已索引”才支持文件级恢复。对于1.6.0及后续版本，索引状态为“未索引”的副本也支持文件级恢复。
    • 对于1.5.0版本，如果副本对应的虚拟机关联的SLA已开启自动索引，则副本的索引状态为“已索引”，可以直接进行文件级恢复。如果未开启自动索引，请先单击“手动创建索引”，再进行文件级恢复。
    • 如果副本中没有包含系统盘，则该副本无法进行文件级恢复。
    • 反向复制副本和归档副本不支持文件级恢复。
    • 链接文件不支持文件级恢复。
    • 备份华为云Stack时，未写入/etc/fstab文件的挂载点信息存储在内存中，本产品不会备份该内容，也不支持文件级恢复。如新增的文件系统需支持文件级恢复，请将新增的文件系统挂载点信息写入/etc/fstab。
    • 当文件夹或文件名称存在乱码时，不支持文件级恢复。请勿勾选该类型的文件夹或文件，否则恢复失败。
    • 当目标弹性云服务器为Windows OS时，且仅开启了SMBv1服务，文件恢复后的时间为弹性云服务器的当前时间。

  5. 选择待恢复的文件。

    • 对于1.5.0版本，从目录树勾选待恢复的文件。
    • 对于1.6.0及后续版本，请先设置恢复文件的选择方式，再从目录树选择待恢复文件或手动输入待恢复文件的路径。
      • 如果副本中的文件数量较多，系统可能超时，无法从目录树上选择待恢复文件，建议手动输入恢复文件路径。
      • 手动输入文件路径时，请输入文件完整路径，例如：/opt/abc/efg.txt或者C:\abc\efg.txt。当输入文件夹路径时，表示恢复该文件夹下的所有文件，例如：/opt/abc或者C:\abc。路径中的文件名区分大小写。

  6. 选择恢复至“原位置”或“新位置”，并设置恢复参数。

    • 选择恢复至原位置，即恢复至原弹性云服务器的原目录。

      相关参数说明如表1所示。

      表1 原位置恢复参数说明

      参数

      说明

      云服务器IP地址
      选择已配置的备份网络平面IP地址。
      说明：

      如果弹性云服务器IP地址未在列表内，请退出当前页面，在“保护 > 云平台 > 华为云Stack”页面重新扫描虚拟化环境。

      云服务器用户名
      文件级恢复时，目标弹性云服务器需要验证本产品的登录凭据，输入目标弹性云服务器的登录用户名及密码。
      • Windows OS：默认用户名为Administrator。
      • Linux OS：默认用户名为root。
      说明：
      • 该登录用户需要有目标弹性云服务器待恢复目录的读写权限。
      • 恢复后该文件的访问权限与该登录用户一致。

      云服务器密码

      覆盖规则
      当恢复路径中存在同名文件时，选择替换或跳过已存在的文件。
      • 直接替换已经存在的文件。
      • 忽略已存在的文件，即跳过这些同名文件不替换。
      • 只替换比恢复文件更旧的文件。
        说明：

        如果目标虚拟机的同名文件拒绝访问，则在替换文件进行恢复时会恢复失败。
    • 选择恢复至新位置，即恢复至指定虚拟机。

      相关参数说明如表2所示。

      表2 新位置恢复参数说明

      参数

      说明

      目标云平台

      选择恢复到的云平台。

      云平台租户

      选择云平台租户。

      区域

      选择恢复到的区域。

      项目/资源集

      选择恢复到的项目/资源集。

      云服务器

      选择恢复到的弹性云服务器。

      • Windows OS：默认用户名为Administrator。

      • 恢复后该文件的访问权限与该登录用户一致。

  7. 单击“测试”，确保待恢复的目标虚拟机与本产品网络连接正常。
  8. 单击“确定”。
