Viewing Kubernetes FlexVolume Copy Information

This section describes how to view the details of a Kubernetes FlexVolume copy.

Procedure

  1. Choose Explore > Copy Data > Containers > Kubernetes FlexVolume.
  2. Select a copy viewing mode to view the copy information.

    Replication copies can be viewed only in the target cluster.

    • Resources: View resource and copy information by StatefulSet. If a StatefulSet is faulty and you need to restore the StatefulSet, you are advised to search for the copy of the corresponding time point through this entry.

      Table 1 describes the related parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Name

      StatefulSet name.

      Click the name to view its copy information.

      You can search for copies by year, month, or day.

      If is displayed below the time, a copy at that time exists.

      Location

      Location of the StatefulSet. Location format: cluster/namespace/StatefulSet

      Status

      Whether the StatefulSet is registered with the current Management Platform.

      • If yes, Status is Existing.
      • If no, Status is Not existing.

      Copy

      Quantity

      Number of copies generated based on the StatefulSet. The number of replication copies is displayed only in the target cluster.

      Replicated Copy SLA

      Archive SLA associated with a replication copy.

      If an archive SLA has been configured for the replication copy generated by the resource, the SLA name is displayed.

      Replicated Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is enabled.
      • Creating: The replication copy is associated with an SLA and protection is being created. The status is available only in 1.5.0 and earlier versions.
    • Copy: View information about all copies by copy. When you want to restore data to the original location or a new location using a specified copy, you are advised to click this icon and then search for the copy.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copy

      Backup Time

      Time when a backup copy is generated.

      Copy Time

      Time when a copy is generated.

      Click Copy Time to view the copy information such as expiration time and location. If -- is displayed for the expiration time, the copy is retained permanently.

      Status

      Copy status.

      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being restored or in instant recovery.
      • Deletion failed: The copy fails to be deleted.

      Location

      Location of a copy.

      • When Generation Mode of a copy is Backup ,Cascaded Replication, Reverse Replication, or Replicate, the copy is stored on the local OceanProtect. In this case, Local is displayed.
      • When Generation Mode of a copy is Object Storage Archive, the copy is stored in the object storage bucket. The bucket name is displayed.
      • When Generation Mode of a copy is Tape Archive, the copy is stored in the tape library. The media set name is displayed.

      Generation Mode

      Generation mode of a copy.

      • Replicate: Copies generated by replication jobs.
      • Backup: Copies generated by backup jobs.
      • Object Storage Archive: Copies generated by object storage archiving jobs.
      • Tape Archive: Copies generated by tape archiving jobs.
      • Cascaded Replication: Copies generated by cascaded replication jobs.
      • Reverse Replication: Copies generated by reverse replication jobs.

      Copy Type

      Type of backup used for generating copies.

      It can be full backup or forever incremental backup (synthetic full backup).

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.

      Copy Verification Status

      Verification status of the generated copy.

      If the copy verification status is --, the copy is not verified.

      • Available: The current copy has been verified and the data is correct.
      • Invalid: The current copy verification task fails or the copy data is inconsistent.

      Resources

      Name

      StatefulSet name.

      Location

      Location of the StatefulSet.

      Status

      Whether the StatefulSet is registered with the current Management Platform.

      • If yes, Status is Existing.
      • If no, Status is Not existing.
