步骤8：执行备份

执行备份前将需要保护的资源关联到指定SLA，系统将对这些资源按照SLA进行保护，并根据SLA周期性执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

注意事项

  • 手动备份产生的副本保留时长按照SLA中的定义进行保留。
  • 备份前请勿增加、修改、删除管理节点中数据库的数据。
  • 日志备份会占用本地存储空间。

操作步骤

  1. 选择“保护 > 数据库 > GoldenDB”。
  2. 在“实例”页签，选择需要保护的实例，单击“保护”。

    您也可以勾选多个实例进行批量保护。

  3. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

    如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  4. 配置备份高级参数。

    相关参数说明如表1所示。
    表1 备份高级参数说明

    参数

    说明

    备份前执行脚本

    您可以根据实际需求，配置“备份前执行脚本”、“备份成功执行脚本”和“备份失败执行脚本”，当备份任务事件中出现“开始执行前置任务”提示后，配置的脚本会在对应场景下执行。

    此处请输入脚本的绝对路径和名称，并确保登录数据库主机的用户对该脚本有可执行权限。
    • Windows OS支持的脚本类型为“.bat”，如输入：C:\script.bat。
    • 非Windows OS支持的脚本类型为“.sh”，如输入：/opt/script.sh。
      说明：

      当配置了“备份成功执行脚本”时，即使该脚本执行失败，OceanProtect上也会显示备份任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。

    备份失败执行脚本

  5. 单击“确定”。

    如果当前系统时间已晚于SLA首次备份的开始时间，可以在弹出的对话框中选择立即执行手动备份或者根据SLA中设置的备份策略周期性自动执行备份。

  6. 执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。
    1. 在需要备份的实例所在行单击“更多 > 手动备份”。
    2. 设置手动备份产生的副本名称。
    3. 如果您不设置，系统默认设置副本名称为：backup_时间戳。
    4. 选择保护策略：全量备份、增量备份或日志备份。

      对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    5. 单击“确定”。
