Step 1: Create a FusionCompute Interconnection User

Before performing FusionCompute backup, create a user for interconnecting FusionCompute with the product on the FusionCompute management page by referring to this section.

Precautions

This section uses FusionCompute 8.6.0 as an example. The GUI varies depending on the version.

Procedure

  1. Log in to FusionCompute using a browser.

    Login address: https://Floating IP address of the VRM node of FusionCompute:8443

  2. Create a role on the FusionCompute WebUI. This role is used to create an account for interconnecting with the product.

    1. Choose System Management > Rights Management > Role Management.
    2. Click Add Role.
    3. Enter the role information.
      Set Role Name to OceanProtect, as shown in Configuring FusionCompute Role Permissions.
      Figure 1 Adding a role

    4. Click OK.

  3. On the FusionCompute WebUI, create an account for interconnecting with the product.

    1. Choose System Management > Rights Management > User Management.
    2. Click Add User.
    3. Enter the user information.
      Set Permission Type to Interface interconnection user and Role to OceanProtect.
      Figure 2 Creating an interface interconnection user

    4. Click OK.
    5. Choose System Management > Rights Management > Rights Management Policy.
    6. In the Password Policy area, click Modify.
    7. Change the current rule of Interface interconnection user forcibly change passwords upon a reset or initial login. to No.
    8. Click Save.
