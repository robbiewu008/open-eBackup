Managing Copies of an NDMP NAS File System

This section describes operations on copy data.

Choose Explore > Copy Data > File Systems > NAS File Systems to go to the NDMP NAS file system copy page.

  • Click Copy to perform operations by copy.

    Table 1 describes the related operations.

    Table 1 Operations related to copies

    Operation

    Description

    Navigation Path

    Modifying a copy retention policy

    Scenario

    This operation enables you to modify the retention period of a copy.

    Note

    The new copy retention period is calculated from the time when the modification takes effect.

    In the row where the copy resides, choose More > Modify Retention Policy.

    You can set the retention period to xx days, xx weeks, xx months, xx years, or permanently.

    Deleting a copy

    Scenario

    This operation enables you to delete an unnecessary or unavailable copy to release storage space.

    Note

    • Before deleting a copy, ensure that the copy is no longer used for restoration.
    • Copies in the Invalid or Normal status can be deleted.
    • If a resource has multiple copies, the latest copy cannot be deleted.
    • If a resource has multiple copies among which only one copy is the full replication copy, the full replication copy cannot be deleted.
    • Copies in ransomware detection cannot be deleted.

    In the row of the target copy, choose More > Delete. Alternatively, select the copies to be deleted and click Delete above the table.

    The configurations for restoration are similar to those in Restoring an NDMP NAS File System.
