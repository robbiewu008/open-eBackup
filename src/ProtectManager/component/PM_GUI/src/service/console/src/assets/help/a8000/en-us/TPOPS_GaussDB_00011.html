Step 3: Configuring the Default Backup Root Path for the Instance on the TPOPS Management Page

Before the backup or restoration function is used, you need to set the default backup root path through Backup Parameter Management.

You can determine whether to perform operations in this section based on the management platform in use.
  • For the OLTP OPS management platform, perform operations in this section.
  • For the cloud database GaussDB management platform, skip this section.

Procedure

  1. Log in to the TPOPS management page.

    The TPOPS login URL varies according to the TPOPS version.

    • https://TPOPS management plane IP address:8443/gaussdb, for example, https://192.168.10.2:8443/gaussdb
    • https://TPOPS management plane IP address:7070/APW, for example, https://192.168.10.3:7070/APW

  2. Check whether the default backup root path has been configured for the instance to be backed up.

    1. In the area of all instances, click the name of the instance to be backed up.
    2. Choose Instance Management > Instance Backup/Restoration > Backup Parameter Management to go to the parameter management page and check whether the default backup root path has been configured.
      • If the default backup root path has been configured and you want to change the path, disable the archive status of the default root path and then reconfigure the path.

        After the default backup root path is switched, you need to perform a full backup before performing log backup or differential backup.

      • If not, go to 3.

  3. Configure the default backup root path.
