Restoring GoldenDB

This section describes how to use copies to restore a GoldenDB instance that has been backed up to the original or a different host.

Prerequisites

Before restoration, ensure that the remaining space of the data directory at the target location for restoration is greater than the size of the copy used for restoration before reduction. Otherwise, restoration will fail.

Procedure

  1. Choose Explore > Copy Data > Databases > GoldenDB.
  2. Search for copies by resource or copy.

    • On the Resources tab page, find the instance to be restored based on the name and click the instance name.
    • On the Copies tab page, find the target copy based on the copy information and choose More > Restore in the row where the copy is located.

    This section uses the database resource dimension as an example to describe the related operations.

  3. Specify a copy or a point in time for restoration.

    You can specify a copy or any point in time between two copies for restoration.

    • Specify a copy for restoration.
      1. Select the year, month, and day in sequence to find the copy.

        If is displayed under a month or date, copies exist in the month or on the date.

      2. On the copy data page, click . In the row of the target copy, choose More > Restore.
    • Specify a point in time for restoration.
      1. Select the year, month, and day in sequence to find the copy.

        If is displayed under a month or date, copies exist in the month or on the date.

      2. On the copy data page, click . In the blue part on the timeline, select a copy and choose More > Restore.

  4. In the row of the target copy, choose More > Restore.

    Table 1 describes the related parameters.

    Table 1 Parameters for restoring a GoldenDB instance

    Parameter

    Description

    Restore To

    Select either Original location or New location.

    NOTE:

    If you select New location, you are advised to select the location with the same number of shards for restoration. If the number of shards is inconsistent, the restoration may fail.

    To view the shard information:

    1. Choose Protection > Databases > GoldenDB.
    2. On the Instance tab page, click an instance name.
    3. In the Overview > Data Nodes > Owing Shard area, view the shard information.

    Location

    If you select Original location, the target location is displayed by default.

    Tag

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    If you set Restore To to New location, you can filter the resources to be restored by tag.

    Target Cluster

    New location: Data will be restored to the original cluster by default.

    Target Instance

    New location: Select the target instance for the restoration.

  5. Click OK.
