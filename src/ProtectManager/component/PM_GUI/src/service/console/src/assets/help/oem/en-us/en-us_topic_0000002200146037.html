Step 7: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • During manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Channels, Rate Limiting Policies, and Transmission and Data Encryption, are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication policy has been defined in the SLA, the product will perform replication once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Databases > Dameng.
  2. On the Databases tab page, select the database to be protected and click Protect.

    You can also select multiple databases for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resources to be protected, select an SLA without a WORM policy to avoid WORM policy conflicts.

  4. Set other parameters for the backup policy.

    Table 1 describes the related parameters.
    Table 1 Other parameters of the backup policy

    Parameter

    Description

    Specify Target Location
    Specify the target storage location of backup copies.
    • Backup Storage Units: A backup storage unit corresponds to a backup cluster node. After a backup storage unit is selected, copies are stored to the storage unit. If the capacity of the storage unit is insufficient, the backup job will fail.

      For details about how to create a backup storage unit, see "Creating a Backup Storage Unit" in the Cluster HA Feature Guide.

    • Backup Storage Unit Groups: A backup storage unit group is a collection of backup storage units. After a backup storage unit group is selected, the system automatically selects a target backup storage unit based on the storage policy of the storage unit group.

      For details about how to create a backup storage unit group, see "(Optional) Creating Backup Storage Unit Groups" in the Cluster HA Feature Guide.

    NOTE:
    • For 1.5.0, this parameter is displayed only when you have deployed the cluster HA feature. You can leave the target location unspecified. If you do not specify the target location, the system automatically selects the target storage unit through intelligent balancing.
    • For 1.6.0 and later versions, you must specify the target location.

    Rate Limiting Policy

    Average maximum amount of data that can be transmitted per second in a backup job.

    Concurrent Jobs

    Number of backup jobs that can be executed simultaneously.

    Source Deduplication

    NOTE:

    Some models of this product support this parameter. For details, see the product documentation released on the technical support website or consult the product provider.
    After this function is enabled, the amount of data to be transmitted can be reduced, but more client resources will be occupied.
    NOTE:
    • A client can only be linked with one storage unit. To enable source deduplication, select Backup Storage Unit for Specify Target Location. Otherwise, a storage unit that is not linked with the client may be selected through intelligent balancing. As a result, source deduplication fails to be enabled. If different protected objects are associated with different SLAs, where different backup storage units are specified, select different clients for the protected objects. Otherwise, source deduplication will fail to be enabled for some backup jobs.
    • If a client has been linked with a storage unit but is not performing a backup job with source deduplication enabled, you can run the dataturbo delete storage_object storage_name=? command on the host where the client is deployed to delete the source deduplication link with the storage unit. The value of storage_name in the command can be queried by running the dataturbo show storage_object command. This way, when the client performs a backup job with source deduplication enabled, the link between the client and a new storage unit can be established.
    • View the product compatibility to check whether source deduplication is supported by the OS of the host where the client is deployed. If not supported, source deduplication will not take effect after being configured.
    • If a built-in client (that is, the client is co-deployed with the product) is used for backup, source deduplication does not take effect after being configured.
    Standard SmartCompression
    NOTE:

    Some models of this product support this parameter. For details, see the product documentation released on the technical support website or consult the product provider.

    After standard SmartCompression is enabled, service data is compressed before being written to the storage space, saving storage space and improving backup efficiency.
    Job Timeout Alarm
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    After this function is enabled, an alarm is sent if the job execution time exceeds the time window. The alarm needs to be manually cleared.

    Job Failure Alarm

    After this function is enabled, an alarm is sent when a job fails. The alarm is automatically cleared when the next job is successful.

    Automatic Retry

    After this function is enabled, the system automatically retries a failed job based on the number of retries and time preset.

  5. Click OK.

    If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  6. Optional: Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row of the target resource, choose More > Manual Backup.

      You can select multiple resources to perform manual backup in batches. Select multiple protected resources and choose More > Manual Backup in the upper left corner of the resource list.

    2. Set the name of the copy generated during manual backup.

      If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full Backup, Incremental Backup, Differential Backup, or Log Backup.
      • The Dameng cluster does not support log backup.
      • If the selected protection policy is different from that configured in the associated SLA for 1.6.0 and later versions, the WORM configuration does not take effect.
    4. Click OK.
