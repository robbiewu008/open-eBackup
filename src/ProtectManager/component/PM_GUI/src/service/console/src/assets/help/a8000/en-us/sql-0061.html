Managing SQL Server

The OceanProtect allows you to perform operations on databases, such as modifying, removing, activating, and disabling protection.

  1. Choose Protection > Databases > SQL Server.
  2. Click the Instance, Availability Group, or Databases tab and find the target database, instance, or availability group. The following takes Databases as an example.

    Table 1 describes the related operations.

    Table 1 Operations related to database, instance, and availability group management

    Operation

    Description

    Navigation Path

    Modify Protection

    Scenario

    You can modify a protection plan to associate another SLA with a database or modify a user-defined script. After the modification, the next backup will be executed based on the new protection plan.

    Note

    • The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.
    • If the SLA associated with a resource changes, you are advised to modify the current SLA by choosing Protection > Protection Policies > SLAs. You are advised not to associate the resource with another SLA by modifying the protection.

      When you associate the resource with another SLA by modifying protection, the license capacity may be deducted repeatedly if the copy of the original SLA still exists.

    Choose More > Modify Protection in the row where the database is located.

    Remove Protection

    Scenario

    You can remove the protection plan if the database no longer needs to be protected. After removal, the system cannot execute protection jobs for the database.

    Note

    • Protection cannot be removed when a protection job of the database is running.
    • If the SLA associated with a resource changes, you are advised to modify the current SLA by choosing Protection > Protection Policies > SLAs. You are advised not to associate the resource with another SLA after removing the protection.

      When you associate the resource with another SLA after removing protection, the license capacity may be deducted repeatedly if the copy of the original SLA still exists.

    Choose More > Remove Protection in the row where the database is located.

    Manual Backup

    Scenario

    This operation enables you to perform a backup immediately.

    Note

    Copies generated by manual backup are retained based on the SLA.

    Choose More > Manual Backup in the row where the database is located.

    Activate Protection

    Scenario

    This operation enables you to activate a disabled protection plan of a database. After the protection plan is activated, the system backs up the database based on the protection plan.

    Note

    You can perform this operation only on a database that has been associated with an SLA and is in the Unprotected state.

    Choose More > Activate Protection in the row where the database is located.

    Disable Protection

    Scenario

    You can disable the protection plan of a database if it no longer needs periodical protection. After the protection plan is disabled, the system does not execute the automatic backup job of the database.

    Note

    Disabling protection does not affect ongoing backup jobs.

    Choose More > Disable Protection in the row where the database is located.
    Adding a tag
    NOTE:

    Only 1.6.0 and later versions support this function.

    Scenario

    This operation enables you to quickly filter and manage resources.
    Choose More > Add Tag.
    NOTE:
    • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
    • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
    Removing a tag
    NOTE:

    Only 1.6.0 and later versions support this function.

    Scenario

    This operation enables you to remove the tag of a resource when it is no longer needed.

    Choose More > Remove Tag.
