Managing Live Mounts

On the management page, you can perform operations related to live mount of VM copies.

Log in to the management page and choose Explore > Live Mount > CNware.

Table 1 describes the related operations.

Table 1 Operations related to live mount

Operation

Description

Navigation Path

Modifying mount information

Scenario

If parameters such as the bandwidth, IOPS, and mount update policy of a mount copy are not configured when you create a live mount job, perform this operation to configure the parameters or modify the configured parameters.

Choose More > Modify in the row where the target live mount information is located.

Unmounting a mount copy

Scenario

This operation enables you to unmount a copy from a VM.

Note

  • During unmounting, you need to determine whether to retain the copy generated by live mount. If you select to retain the copy, it is retained based on the retention period set in the mount update policy or permanently retained (in case no mount update policy is set). Otherwise, the mounted copy is directly deleted.
  • After the mount copy is unmounted, the mounted VM will be deleted.

Choose More > Destroy in the row where the target live mount information is located.

Updating a mount copy

Scenario

This operation enables you to update a mount copy for a VM immediately.

Note

A mount copy can be updated only when the live mount status is Available or Mounting failed.
  1. Choose More > Update in the row where the target live mount information is located.
  2. Select Update latest copy data or Update specified copy data.

Migrating data to a VM

Scenario

This operation enables you to migrate data to the target VM after live mount.

Note

During the migration, the live mount status must be Available.
  1. Choose More > Migration in the row where the target live mount information is located.
  2. Select the storage locations of the VM configuration file and virtual disk data.
    • Different Datastores: Select the storage locations of the VM configuration file and virtual disk data in sequence.
    • Same Datastore: Select VM Profile, indicating that the VM configuration file and the virtual disk data are stored in the same datastore.
  3. Click OK.

Disabling mount update

Scenario

Perform this operation if you do not need to update a mount copy based on the mount update policy or you want to prevent a mount copy from being manually updated or live mount parameters from being modified.

Choose More > Disable in the row where the target live mount information is located.

Activating mount update

Scenario

After live mount is disabled, it can be reactivated. After the activation, mount copies are updated based on the mount update policy.

Choose More > Activate in the row where the target live mount information is located.

Viewing mount details

Scenario

This operation enables you to view live mount details.

Choose More > View in the row where the target live mount information is located.

View the details about the resource, copy data, and mount update policy.
