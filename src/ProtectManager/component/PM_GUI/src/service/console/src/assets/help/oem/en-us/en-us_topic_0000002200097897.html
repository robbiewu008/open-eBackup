Viewing Huawei Cloud Stack Copy Information

This section describes how to view Huawei Cloud Stack copy information.

Procedure

  1. Choose Explore > Copy Data > Cloud Platforms > Huawei Cloud Stack.
  2. View copy information.

    • Replication copies can be viewed only in the target cluster.
    • Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.
    • Click the Resources tab to view resources and copy information by cloud server. When you need to restore a faulty cloud server, you are advised to search for the target copy by clicking this tab.

      Table 1 describes the related parameters.

      Table 1 Copy parameters

      Parameter

      Description

      Resources

      Resource ID

      Resource ID.

      Name

      Cloud server name.

      Click the name of a cloud server to view its copy information.

      You can search for copies by year, month, or day.

      If is displayed below the time, copies exist at that time.

      Location

      Cloud server location. The location format is as follows: Huawei Cloud Stack platform/Tenant/Region/Project (resource set)/Cloud server.

      Status

      Whether a cloud server is registered with the current management page.

      • Existing: The cloud server has been registered on the current management page.
      • Not existing: The ECS is not registered on the current management page.

      Copies

      Quantity

      Number of copies generated based on a cloud server. The number of replication copies is displayed only in the target cluster.

      Replication Copy SLA

      SLA associated with replication copies.

      If an SLA has been configured for replication copies generated by the resource, the SLA name is displayed.

      Replication Copy Protection Status
      Whether a replication copy is protected.
      • Unprotected: The replication copy is not associated with an SLA, or the replication copy is associated with an SLA but protection is disabled.
      • Protected: The replication copy is associated with an SLA and protection is activated.
      • Creating: The replication copy is associated with an SLA and protection is being created. The status is available only in 1.5.0.
    • Click the Copies tab to view all copy information by copy. When you need to restore data to the original location or new location based on a specified copy, you are advised to click this icon and then search for the copy.

      Table 2 describes the related parameters.

      Table 2 Copy parameters

      Parameter

      Description

      Copies

      Name

      Copy name.
      Backup Copy Generation Time
      NOTE:

      For 1.5.0, the parameter name is Backup Time.

      Time when a backup copy is generated.

      Status
      Copy status.
      • Normal: The copy is available.
      • Invalid: The copy is invalid.
      • Deleting: The copy is being deleted.
      • Restoring: The copy is being used for restoration. The state is available only in 1.5.0.
      • Verifying: The copy is under integrity check. This state is available only in 1.6.0 and later versions.
      • Deletion failed: The copy fails to be deleted.

      Location
      Location of a copy.
      • For 1.5.0: If Generation Mode of a copy is Backup, Cascaded replication, Reverse replication, or Replicate, the copy is stored in the local cluster. In this case, Local is displayed.
      • For 1.6.0 and later versions: If Generation Mode of a copy is Backup, Cascaded replication, Reverse replication, or Replicate, the copy is stored in the corresponding backup storage unit. In this case, the name of the corresponding backup storage unit is displayed.
      • If Generation Mode of a copy is Archive to object storage or Import, the copy is stored in an object storage bucket. In this case, the bucket name is displayed.
      • If Generation Mode of a copy is Archive to tape, the copy is stored in a tape library. In this case, the media set name is displayed.
      Storage Media Status
      NOTE:

      This parameter is available only in 1.6.0 and later versions.
      Status of the storage medium where the copy resides.
      • Online: The copy can be managed.
      • Offline: The copy can be deleted only.
      NOTE:

      Storage Media Status is Online for backup storage units whose Health Status is Normal or Degraded and Running Status is Pre-Copy, Rebuilt, Online, Balancing, or Degraded.

      Generation Mode

      Mode in which a copy is generated.

      • Replication: refers to a copy generated by a replication job.
      • Backup: refers to a copy generated by a backup job.
      • Archive to object storage: refers to a copy generated by an archive job (with object storage used as the archive storage).
      • Archive to tape: refers to a copy generated by an archive job (with tape libraries used as the archive storage).
      • Cascaded replication: refers to a copy generated by a cascaded replication job.
      • Reverse replication: refers to a copy generated by a reverse replication job.
      Copy Generation Time
      NOTE:

      For 1.5.0, the parameter name is Copy Time.

      Time when a copy is generated. Click the copy generation time to view the details about the copy.

      Copy Type

      Backup type of a generated copy. The option can be Full or Forever Incremental (Synthetic Full).
      Copy Expiration Time
      NOTE:
      • This parameter is available only in 1.5.0SPC26 and later versions.
      • For 1.5.0, the parameter name is Expired At.
      Copy expiration time. If a copy is permanently valid, -- is displayed.
      NOTE:

      The actual copy expiration time is subject to the latest time between Copy Expiration Time and WORM Expiration Time.

      WORM
      Whether a copy has been set as a WORM copy.
      • : The copy is not set as a WORM copy, that is, the copy is a common copy.
      • : The copy has been set as a WORM copy. The WORM copy cannot be deleted and its retention period cannot be shortened.
      • : The copy is being set as a WORM copy. A copy in this state cannot be deleted and the retention period cannot be modified.
      • : The copy fails to be set as a WORM copy. A copy that fails to be set as a WORM copy cannot be deleted and the retention period cannot be shortened.
      • : The copy has expired. This state is available only in 1.6.0 and later versions. It indicates that the WORM copy has expired. Copies in this state can be deleted.
      WORM Expiration Time
      NOTE:
      • This parameter is available only in 1.6.0 and later versions.
      If a copy is a WORM copy, the WORM expiration time is displayed. Otherwise, -- is displayed.
      NOTE:

      If the WORM expiration time is earlier than the WORM file system compliance clock time, the WORM validity period has expired. You can click the copy generation time to view the WORM file system compliance clock time. If the WORM file system compliance clock time is inconsistent with the system time, check the WORM status after the WORM file system compliance clock reaches the set expiration time.

      Copy Verification Status
      Verification status of the copy.
      • --: indicates that the current copy is not verified.
      • Available: indicates that the current copy has been verified and the data is correct.
      • Invalid: indicates that the current copy verification job fails or the copy data is inconsistent.

      Copies
      File-level Mount Status
      NOTE:

      This parameter is available only in 1.6.0 and later versions.
      File-level mounting status of a copy.
      • Mounted: A file-level mount point has been created.
      • Unmounted: No file-level mount point is created.
      • Mounting: A file-level mount point is being created.
      • Mounting failed: A file-level mount point fails to be created.
      • Unmounting: A file-level mount point is being deleted.
      • Unmounting failed: A file-level mount point fails to be deleted.

      Resources

      Name

      Cloud server name.

      Location

      Cloud server location.

      Status

      Whether a cloud server is registered with the current management page.

      • Existing: The cloud server has been registered on the current management page.
      • Not existing: The ECS is not registered on the current management page.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
