Configuring the Backup Network Plane (Applicable to FusionCompute 8.7.0 and Later Versions)

This section describes how to configure an independent backup network plane for CNA hosts to implement LAN-based backup and restoration. If the independent backup network plane is not configured, the management plane is still used for backup and restoration.

Procedure

  1. Log in to FusionCompute.
  2. Click in the navigation tree on the left to go to the Resource Pool page.
  3. Click the Host tab and select the CNA host for which you want to configure an independent backup network plane.
  4. Choose Configuration > Network > Logical Interface, click Add Service Management Port to go to the Add Service Management Port page.
  5. On the Port Binding page, select the network port of the host NIC that connects to the service plane and click Next.

  6. On the Connection Settings page, complete the configuration by referring to Table 1 and click Next.

    Table 1 Parameter description

    Parameter

    Description

    Name

    User-defined network plane name.

    VLAN ID

    VLAN ID planned for the service management plane. The value 0 indicates that no VLAN tag will be carried.

    IPv4 Address

    IP address of the service management port of the host.

    Subnet Mask

    Subnet mask planned for the service management plane.

    Routing Info

    This configuration is required only when storage links need to communicate with each other over Layer 3.

    Available Services

    Select CBT Traffic Management and deselect other options.

  7. On the Confirm Info page, confirm the current configuration, and click OK.

  8. Log in to the agent host as user root.
  9. Run the following command to open the configuration file:

    vi /opt/DataBackup/ProtectClient/Plugins/FusionComputePlugin/conf/hcpconf.ini

    In the preceding command, /opt indicates the ProtectAgent installation directory. Replace it with the actual installation directory.

  10. In the [FusionCompute] section of the configuration file, change the value of IP4LanbaseToken to one of backup agent host's IP addresses that is reachable to the IP address configured in 6, and save the configuration file.

    For example, if the backup network plane IP address planned on FusionCompute is 192.168.198.22 and IP address 192.168.198.61 of the backup agent host is reachable to 192.168.198.22 of FusionCompute, change the value of IP4LanbaseToken to 192.168.198.61.

    IP4LanbaseToken=192.168.198.61

    The backup agent host on which this operation is performed can only be used to back up and restore VMs on the CNA host configured with an independent backup network plane.
