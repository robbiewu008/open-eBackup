Viewing the List of Failed Files When an Object Storage Backup or Restoration Job Is Partially Successful

An object storage backup or restoration job is partially successful. You can perform the operations in this section to view the list of failed files.

Procedure

  1. Export logs.

    • A built-in client is used by the job.
      1. Choose System > Log Management.
    • An external client is used by the job.
      1. Choose Protection > Clients.
      2. Select the external client used by the job and choose More > Export Log.

  2. Download the exported logs and decompress them to any local directory.
  3. Go to the log directory, for example, logs\node-0\protectengine-e\agent\slog\Plugins\ObsPlugin.
  4. Search for the ID of the partially successful job in the log directory and go to the directory corresponding to the job ID.
  5. Open the .csv file in the directory and query all objects that fail to be backed up or restored.
  6. If there are multiple hosts where the client is installed, perform 3 to 5 on each host to query all objects that fail to be backed up or restored.
