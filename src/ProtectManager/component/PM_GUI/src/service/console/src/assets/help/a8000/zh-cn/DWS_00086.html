恢复单个或多个表

当需要恢复已备份的GaussDB(DWS)集群、Schema集或表集中的单个或多个表时，可以参考本节恢复单个或多个表至原位置或新位置。

背景信息

支持使用备份副本、复制副本、反向复制副本、级联复制副本、归档副本进行恢复。支持恢复至原位置（复制副本及其归档副本、级联复制副本及其归档副本不支持恢复至原位置）或新位置。

如果备份副本与归档副本同时存在，请优先恢复备份副本。

前提条件

  • 恢复前请确保恢复目标位置的数据目录剩余空间大于恢复副本缩减前大小，否则恢复将失败。
  • 表级恢复到新位置，恢复到新Schema前，请确保已在GaussDB(DWS)对应的数据库中创建该Schema，否则会导致恢复失败。

操作步骤

  1. 选择“数据利用 > 副本数据 > 大数据 > GaussDB(DWS)”。
  2. 您可以以GaussDB(DWS)资源维度或以副本维度查找副本，本节以资源维度为例进行说明。

    在“资源”页签，根据资源名称查找到需要恢复的资源，并单击资源名称。

  3. 依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 指定副本进行恢复。

    • 通过GaussDB(DWS)集群副本、恢复GaussDB(DWS)集群副本中的单个或多个表。
      1. 单击GaussDB(DWS)集群副本所在行的“更多 > 表级恢复”。
      2. 恢复GaussDB(DWS)集群副本中的表至原位置或新位置。

        对于1.5.0版本，请单击“请选择”，选择需要恢复的目标位置。

        相关参数说明如表1。
        表1 目标位置说明

        参数

        说明

        恢复至

        选择恢复至“原位置”或“新位置”。

        位置

        当选择恢复至“原位置”时，显示原位置。

        目标集群
        当选择恢复至“新位置”时，选择需要恢复的目标集群。您需要确认已将目标集群的安装配置文件（mppdb-install-config.xml，如果安装配置文件不为该名称，请重命名为该名称）复制到目标集群每个节点的/opt/huawei/目录下，并确保数据库用户有可读权限，否则会恢复失败。
        说明：

        如果数据库用户对配置文件无可读权限，可使用root用户登录目标集群每个节点，执行以下命令：

        sudo g+r /opt/huawei/mppdb-install-config.xml
      3. 选择要恢复的表所在数据库。
      4. 选择要恢复的单个或多个表。

        对于1.6.0及后续版本，您可以通过在搜索框输入关键字快速查找要恢复的单个或多个表。

      5. 当选择恢复至“新位置”时，需要输入表恢复后所属Schema的名称和表的名称。

        对于1.6.0及后续版本，您可以同时选中多个要设置的表，单击“批量设置恢复位置”，批量设置表恢复后所属Schema的名称。

      6. 单击“确定”，并根据界面提示完成确认。
        • 请确认恢复目标位置正确，且恢复目标位置剩余空间应大于恢复副本缩减前大小。
        • 对于1.6.0及后续版本，支持表级恢复到新位置。
        • 表级恢复只支持恢复表集的表结构和数据，若还需要额外恢复表集的view、function等信息，请根据GaussDB(DWS)对应版本的《产品文档》进行手工创建view、function等信息。
    • 通过Schema集/表集副本、恢复Schema集/表集副本中的单个或多个表。
      • 普通恢复（恢复表集副本中的所有表）

        Schema集副本不支持普通恢复。

        1. 单击表集副本所在行的“更多 > 恢复”。
        2. 恢复副本中的表至原位置或新位置。

          对于1.5.0版本，请单击“请选择”，选择需要恢复的目标位置。

          相关参数说明如表2。
          表2 目标位置说明

          参数

          说明

          恢复至

          选择恢复至“原位置”或“新位置”。

          位置

          当选择恢复至“原位置”时，显示原位置。

          目标集群

          当选择恢复至“新位置”时，选择需要恢复的目标集群。

          数据库

          对表集副本恢复至“新位置”时，需要选择恢复的目标数据库。
        3. 单击“确定”，并根据界面提示完成确认。
          • 请确认恢复目标位置正确，且恢复目标位置剩余空间应大于恢复副本缩减前大小。
          • 表级GDS恢复时，恢复目标端启动的GDS数量必须大于或等于源端备份时启动的GDS数量。

          • 表级恢复只支持恢复表集的表结构和数据，若还需要额外恢复表集的view、function等信息，请根据GaussDB(DWS)对应版本的《产品文档》进行手工创建view、function等信息。
      • 表级恢复（恢复Schema集或表集副本中的单个或多个表）。
        1. 单击Schema集或表集副本所在行的“更多 > 表级恢复”。
        2. 恢复副本中的表至原位置或新位置。

          对于1.5.0版本，请单击“请选择”，选择需要恢复的目标位置。

        3. 选择要恢复的表所在数据库。
        4. 选择要恢复的单个或多个表。
