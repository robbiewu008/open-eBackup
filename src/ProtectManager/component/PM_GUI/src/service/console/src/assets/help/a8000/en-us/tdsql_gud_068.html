Managing Database Instances

The OceanProtect allows you to perform the following operations, including modifying, removing, activating, or disabling protection for TDSQL instances.

Log in to the management page, choose Protection > Databases > TDSQL, click the Non-distributed Instances or Distributed Instances tab, and find the desired instance.

Table 1 describes the related operations.

Table 1 Database instance management operations

Operation

Description

Navigation Path

Modifying protection

Scenario

This operation enables you to modify a protection plan to perform SLA association again for an instance or modify user-defined script information. After the modification, the next backup will be executed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will apply to the next protection period.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the current SLA on the Protection > Protection Policies > SLAs page, rather than associate the resource with another SLA through protection modification.

    After you associate the resource with another SLA by modifying protection, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

In the row of the target instance, choose More > Modify Protection.

Removing protection

Scenario

You can remove the protection plan if the database no longer needs to be protected. After the removal, the system cannot execute protection jobs for the database.

Note

  • When a protection job of the database is running, the protection cannot be removed.
  • If the protection policy for a resource in the associated SLA changes, you are advised to modify the associated SLA on the Protection > Protection Policies > SLAs page, rather than perform SLA re-association after protection removal.

    After you remove protection and perform SLA re-association, if the copy of the original SLA still exists, the license capacity may be deducted repeatedly.

Choose More > Remove Protection.

Activating protection

Scenario

This operation enables you to re-activate the disabled protection plan of a database. After the protection plan is activated, the system backs up the database based on the protection plan.

Note

You can perform this operation only on a database that has been associated with an SLA and is in the Unprotected state.

Choose More > Activate Protection.

Disabling protection

Scenario

This operation enables you to disable a protection plan when periodic protection on the database is not required. After the protection plan is disabled, the system does not execute the automatic backup job of the database.

Note

Disabling protection does not affect ongoing backup jobs.

Choose More > Disable Protection.

Restoration

Scenario

This operation enables you to restore a database instance using a copy.

In the row of the target instance, choose More > Restore.

Performing a manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup.

Testing connectivity

Scenario

Manual connectivity test of the registered TDSQL instance is used to check the connectivity between the cluster and the agent host and between the cluster and the production environment.

In the row of the target instance, choose More > Test Connectivity.

Modification

Scenario

This operation allows you to modify the name of a TDSQL instance and backup agent hosts.

In the row of the target instance, choose More > Modify.

Deletion

Scenario

This operation allows you to delete a TDSQL instance if you no longer need to protect it.

Note

A TDSQL instance associated with an SLA cannot be deleted.

In the row of the target instance, choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
