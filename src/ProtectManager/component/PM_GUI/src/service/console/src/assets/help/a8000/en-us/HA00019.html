Adding Member Nodes

This section describes how to add a member node. After a member node is added, backup copies can be stored on the backup storage corresponding to the member node. For 1.6.0 and later versions, Internal Communication Network Plane is renamed as Internal Communication Network. This section uses 1.5.0 as an example.

Prerequisites

  • The internal communication network plane has been added for the active node, and at least two IP addresses have been reserved for the member node to be added.
  • (Applicable to 1.6.0 and later versions) The internal communication network plane has been added to the backup storage device to be upgraded.
  • The version of the member node to be added is the same as that of the active node.
  • The member node to be added is not added to other replication clusters or multi-domain clusters.
  • (Applicable to 1.5.0) The local storage corresponding to the member node to be added is not added as a backup storage unit.
  • (Applicable to 1.6.0 and later versions) The local storage corresponding to the member node to be added is not added as a backup storage device.
  • The IP address of the internal communication network plane of the member node to be added must be able to communicate with the IP address of the internal communication network plane of the active node.
  • No job is running on the member node to be added.
  • You have downloaded the member node's CA certificate of the Server Certificate type and imported it to the active node. The type is set to Backup/Multi-domain/Replication Cluster. For details about how to import a CA certificate, see "Downloading a Certificate" and "Importing a Certificate" in the OceanProtect DataBackup 1.5.0-1.6.0 Security Configuration Guide.
  • Ensure that the environment of the member node to be added is newly deployed.
    • The environment is deployed for the first time and only initialization has been performed.
    • For an environment where data protection services are performed, you need to reinstall the backup software and reinitialize the environment and ensure that no other operations have been performed. Verify that the following operations are not performed:
      • Add an external cluster (replication cluster, management cluster, or managed cluster).
      • Add an archive storage.
      • Add a storage device.
      • Create a backup storage unit group.
      • Set the SFTP service.
      • Register ProtectAgent.
      • Create an SLA.
      • Register the protection resources of related features.
      • A job has been created or is being executed, such as backup, replication, archiving, restoration, or certificate import.
      • A member node is added as a backup storage device, replication cluster, management cluster, or managed cluster and so on by other clusters.

Precautions

Service configuration data will be cleared after the OceanProtect, where there are jobs to be executed or data is stored, is added as a member node. Exercise caution when performing this operation.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  2. Click Backup Cluster.
  3. Click Create. Enter the information about the backup node to be added. Table 1 describes the related parameters.

    Table 1 Backup node parameters

    Parameter

    Description

    Name

    Name of the backup node to be added.

    IP Address

    Management IP address of the backup node.

    Port

    Management port of the backup node.

    The default port ID is 25081.

    Username

    Username of the system administrator of the backup node, for example, sysadmin.

    Password

    Password of the system administrator of the backup node.

    Internal Communication Network Plane Name

    NOTE:

    This parameter applies only to 1.5.0.

    Name of the internal communication network plane of the backup node. This network plane can reuse the backup network plane.

  4. Click OK.
  5. In the Danger dialog box that is displayed, confirm the information, select I have read and understand the consequences associated with performing this operation., and click OK. After a member node is added, you cannot log in to the management page of the member node.
