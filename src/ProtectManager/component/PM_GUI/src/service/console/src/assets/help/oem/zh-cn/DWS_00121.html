对于GaussDB(DWS) 8.1.3版本，执行备份任务时，因GaussDB(DWS)软件处理异常，备份任务失败

故障现象

GaussDB(DWS) 8.1.3版本使用客户端部署到生产主机和独立物理服务器场景下执行备份任务，任务失败，任务详情中显示“Roach命令执行失败”。

可能原因

在数据备份完成后，GaussDB(DWS)备份进程还未完全退出，上传进程就被拉起来了，导致业务处理冲突。

处理建议

  1. 登录下发备份命令的节点。
  2. 执行以下命令进入日志文件目录。

    cd /opt/DataBackup/ProtectClient/ProtectClient-E/slog/Plugins/GeneralDBPlugin/

  3. 执行grep {任务ID} dws.log | grep "Backup failed"命令，查看错误日志文件中是否包含以下错误信息：

    Failed to do gs roach. Roach operation backup failed

    示例如下：

    • 如果包含错误信息，请重新执行一次备份任务。
    • 如果不包含错误信息，请执行4。

  4. 登录已部署客户端的GaussDB(DWS)的其他CN节点，执行2~3。如果所有节点都不包含该错误信息，请收集日志并联系华为技术支持工程师协助处理。
