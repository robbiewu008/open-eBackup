Viewing OpenStack Information

After OpenStack is registered with the product, you can view OpenStack project and cloud server information on the management page.

Procedure

  1. Choose Protection > Cloud Platforms > OpenStack.
  2. On the Project tab page, view project information.

    Table 1 describes project details.
    Table 1 Project information

    Parameter

    Description

    Name

    Project name.

    Click the project name to view the project overview and jobs.

    • Overview

      Displays the name, SLA information, and cloud server information.

    • Jobs

      Displays all jobs, including backup and restoration jobs.

    Location

    Project location. The format is Domain/Project.

    SLAs

    If a project is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a project is protected.

    • Unprotected: The project is not associated with any SLA, or the project is associated with an SLA but protection is disabled.
    • Protected: The project is associated with an SLA and protection is activated.
    • Creating: The project is associated with an SLA and protection is being created.

    Owner

    Data protection administrator of the project.

    If a data protection administrator is authorized for the project, the data protection administrator name is displayed. If no data protection administrator is authorized, -- is displayed.

  3. On the Cloud Server tab page, view cloud server information.

    Table 2 describes cloud server details.
    Table 2 Cloud server information

    Parameter

    Description

    Name

    Cloud server name.

    Click the name of a cloud server to view the cloud server overview, copy data, and jobs.

    • Overview

      The name, SLA information, job execution time, total number of copies, and disk information are displayed.

    • Copy Data

      You can search for copies by year, month, or day.

      If is displayed below the time, a copy at that time exists. Click a date to go to the copy data page.

    • Jobs

      Displays all jobs, including backup and restoration jobs.

    Status

    Cloud server status. The value can be Online, Offline, or Suspended.

    Location

    Cloud server location. The format is Domain/Project/Cloud server.

    SLAs

    SLA associated with a cloud server.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a cloud server is protected.

    • Unprotected: The cloud server is not associated with any SLA, or the cloud server is associated with an SLA but protection is disabled.
    • Protected: The cloud server is associated with an SLA and protection is activated.
    • Creating: The cloud server is associated with an SLA and protection is being created.

  4. On the Cloud Server Group page, view the cloud server group information.

    Table 3 describes cloud server group details.
    Table 3 Cloud server group

    Parameter

    Description

    Name

    Name of a cloud server group.

    Cloud Server Quantity

    Number of VMs in a cloud server group.

    SLAs

    If the entire cluster is associated with an SLA, the SLA name is displayed. Otherwise, -- is displayed.

    SLA Compliance
    Whether a backup job is successfully executed based on the backup frequency or backup window set in the SLA.
    • If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
    • If the backup time does not meet the backup time window or backup frequency requirements, for example, if the backup frequency set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup frequency or backup window.

    Protection Status

    Whether a cloud server group is protected.

    • Unprotected: The cloud server group is not associated with any SLA, or the cloud server group is associated with an SLA but protection is disabled.
    • Protected: The cloud server group is associated with an SLA and protection is activated.
    • Creating: The cloud server group is associated with an SLA and protection is being created.

    Click the name of a cloud server group to view its basic information as well as the names and locations of cloud servers in the cloud server group. You can also view the current and historical jobs of the cloud server group on the Jobs tab page.
