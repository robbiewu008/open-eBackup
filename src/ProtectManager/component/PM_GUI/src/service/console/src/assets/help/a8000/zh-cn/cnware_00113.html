资源扫描或备份恢复任务失败，无明显错误提示

故障现象

资源扫描或备份恢复时仅提示任务失败，无其他明显错误提示。

可能原因

当CNware管理节点/计算节点负载偏高或资源数量较多时（具体规格取决于计算节点硬件能力），CNware接口响应速度变慢，导致备份软件调用接口超时失败。

背景信息

以下操作均以客户端的默认安装目录为“/opt”为例进行说明，实际操作时请根据客户端的实际安装目录替换。

操作步骤

  1. 通过PuTTY或SSH工具登录CNware虚拟机客户端所在主机。
  2. 执行以下命令客户端所在主机和生产环境连通性，其中xxx.xxx.xx.xx为CNware管理节点的IP地址。

    ping xxx.xxx.xx.xx

  3. 执行cat /opt/DataBackup/ProtectClient/ProtectClient-E/log/Plugins/VirtualizationPlugin/VirtualPlugin.log | grep "Http send request failed. Error is28"命令查询失败日志。

    如果有回显则执行4，否则请联系技术支持工程师。

  4. 执行vi /opt/DataBackup/ProtectClient/Plugins/VirtualizationPlugin/conf/hcpconf.ini命令打开配置文件。
  5. 找到[General]下代表接口建立连接超时时间的ConnectTimeOut和接口总超时时间TotalTimeOut，增大TotalTimeOut的值。

    一般只需要增大TotalTimeOut的值即可，当增大TotalTimeOut的值执行资源扫描或备份恢复任务失败后尝试增大ConnectTimeOut的值重试。

  6. 执行命令sh /opt/DataBackup/ProtectClient/Plugins/VirtualizationPlugin/stop.sh重启插件。
  7. 重新执行资源扫描或备份恢复任务。
