Manually Mounting a NAS File System or NAS Share

After a live mount is complete, you need to manually mount the generated share to the corresponding client.

Prerequisites

The NAS file system or NAS share has been live mounted. For details, see Live Mounting a NAS File System or Live Mounting a NAS Share.

Procedure

  • For an NFS share, perform the following operations:
    1. Log in to the Linux host to which the NFS share is to be mounted.
    2. Run the following command to mount the NFS share:

      mount -t nfs -o vers=3 Share IP address:/ Share path User-defined mount point

      Replace the Share IP address, Share path, and User-defined mount point in the command based on the site requirements. To obtain the Share IP address and Share path, go to the Live Mount > NAS File Systems or Live Mount > NAS Shares page, and choose More > View in the row that contains the desired live mount copy.

      Example:

      mount -t nfs -o vers=3 192.168.101.67:/ mount_1702111928963/ /mnt/mount_12345
  • For a CIFS share, perform the following operations:
    1. Log in to the Windows host to which the CIFS share is to be mounted.
    2. Run the following command to mount the CIFS share:

      net use Drive letter: \\Share IP address\Share name Password of the CIFS share user /user:CIFS share username

      Replace the Drive letter, Share IP address, Share name, Password of the CIFS share user, and CIFS share username in the command based on the site requirements.
      • Drive letter is a drive letter that has not been used by the Windows host.
      • To obtain the Share IP address, Share name, and CIFS share username, go to the Live Mount > NAS File Systems or Live Mount > NAS Shares page, and choose More > View in the row that contains the desired live mount copy.
      • Password of the CIFS share user is the password entered when the share user is created on DeviceManager.

      Example:

      net use C: \\192.168.101.68\mount_1702382814125 Admin@123 /user:dcf
