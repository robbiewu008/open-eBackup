恢复MySQL/MariaDB/GreatSQL数据库

当需要对已备份的数据库进行恢复时，可以参考本节使用副本恢复数据库至原位置或新位置。

前提条件

  • 执行恢复任务前，请执行以下命令关闭目标位置数据库。
    • MySQL/GreatSQL：systemctl stop mysqld
    • MariaDB：systemctl stop MariaDB
  • 恢复前请确保恢复目标位置的数据目录剩余空间大于恢复副本缩减前大小，否则恢复将失败。
  • 执行恢复任务前，需确保mysqlbinlog、mysqldump、mysqld、mysqladmin和mysql软连接配置正确。请参见步骤2：手动配置软连接。

操作步骤

  1. 选择“数据利用 > 副本数据 > 数据库 > MySQL/MariaDB/GreatSQL”。
  2. 您可以以MySQL/MariaDB/GreatSQL资源维度或以副本维度查找副本，本节以资源维度为例进行说明。

    在“资源”页签，根据资源名称查找到需要恢复的资源，并单击资源名称。

  3. 依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在行，单击“更多 > 恢复”。

    相关参数说明如表1所示。

    参数

    说明

    恢复至

    选择恢复至“原位置”或“新位置”。

    位置

    标签
    说明：

    仅1.6.0及后续版本支持此功能。

    当选择恢复至“新位置”时展示该参数。

    目标实例

    当选择恢复至“新位置”时，请选择恢复到的目标实例。

    说明：

    当恢复的资源类型为“单实例”和“集群实例”时展示该参数。

    目标数据库

    当选择恢复至“新位置”时，请选择恢复到的目标数据库。

    说明：

    当恢复的资源类型为“数据库”时展示该参数。

    数据库重命名

    支持对恢复的数据库进行重命名。

    说明：
    • 当恢复的资源类型为“数据库”时展示该参数。
    • 创建即时挂载的副本必须是“正常”状态。

    强制恢复

    默认不开启，开启后可以在数据库页损坏的情况下，将数据强制恢复。

    恢复前执行脚本

    您可以根据实际需求，在恢复任务执行前、执行成功后、执行失败后，执行自定义脚本。

    请输入脚本名称，脚本名称以.sh结尾。确保脚本已存放在客户端安装目录下的“DataBackup/ProtectClient/ProtectClient-E/sbin/thirdparty”目录，且登录数据库主机的用户（默认为root）对该脚本有可执行权限。

    说明：

    当配置了“恢复成功执行脚本”时，即使该脚本执行失败，本产品上也会显示恢复任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。
