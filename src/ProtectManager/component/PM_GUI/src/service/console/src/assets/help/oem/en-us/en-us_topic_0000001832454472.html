Function Overview

  • Full backup: backs up all data of resources at the current time point.
  • Log backup: backs up TiDB cluster logs.
  • Copy verification: verifies the integrity of a copy after the backup is complete or before the copy is used for restoration.
  • Source deduplication: reduces the amount of data to be transmitted, but occupies more client resources.
  • Transmission and storage encryption: adopts the AES256 encryption algorithm to encrypt backup data.

Replication

  • Backup copy replication: replicates backup copies from the source end to the target end. If a disaster occurs in the local data center, the replication copies can be used to restore service data.

Restoration

  • Copy-based restoration to the original or a new location: supports data restoration using backup copies and replication copies (data cannot be restored to the original location).
