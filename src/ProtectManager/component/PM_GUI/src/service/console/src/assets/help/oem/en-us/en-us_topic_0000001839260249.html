Managing Namespace and Dataset Protection

The system allows users to modify, remove, activate, or disable protection plans of namespaces and datasets. This section uses a namespace as an example to describe related operations. Operations on datasets are the same.

Choose Protection > Containers > Kubernetes CSI. On the Namespace tab page, locate the target namespace.

Table 1 describes the related operations.

Table 1 Operations related to a namespace protection plan

Operation

Description

Navigation Path

Modify Protection

Scenario

This operation enables you to modify a protection plan to re-associate a namespace with an SLA, modify an agent host, or configure an SLA application policy. After the modification, the next backup will be performed based on the modified protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.
  • If a backup job has been executed, reconfigure the SLA policy (change to the SLA with copy verification enabled) when modifying the protection plan. After the modification, the initial backup job will be converted to a full backup job if it is a non-full backup job.
  • If the SLA parameter associated with the resource changes (from copy verification disabled to copy verification enabled) after a backup job has been performed, when the protection plan for the resource is implemented after the change, the initial backup job will be converted to a full backup job if it is a non-full backup job.

In the row where the namespace resides, choose More > Modify Protection.

Remove Protection

Scenario

This operation enables you to remove the protection for a namespace when it does not need to be protected. After the removal, the system cannot execute protection jobs for the namespace. To protect the namespace again, associate an SLA with it again.

Note

When the protection job of the namespace is running, the protection cannot be removed.

In the row where the namespace resides, choose More > Remove Protection.

Manual Backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row where the namespace resides, choose More > Manual Backup.

Activate Protection

Scenario

After the protection plan for namespaces is disabled, you can activate it again. After the protection plan is activated, the system backs up, archives, and replicates the namespace based on the protection plan.

Note

You can perform this operation only on a namespace that has been associated with an SLA and is in the Unprotected state.

In the row where the namespace resides, choose More > Activate Protection.

Disable Protection

Scenario

This operation enables you to disable a protection plan if the namespace no longer needs to be periodically protected. After the protection plan is disabled, the system does not perform automatic backup, archive, and replication jobs for the namespace.

Note

Disabling a protection plan does not affect protection jobs that are running.

In the row where the namespace resides, choose More > Disable Protection.

Note: The system supports restoration for only namespaces, datasets, or PVCs. For details, see Restoration.
