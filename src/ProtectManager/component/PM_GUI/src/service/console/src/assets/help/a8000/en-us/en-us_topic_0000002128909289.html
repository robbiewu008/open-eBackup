How Do I Change the Time Zone and Time of the Agent Host?

If the time of the agent host is inconsistent with that of the OceanProtect, you can change the time zone and time of the agent host by referring to this section.

Linux OS

The following uses SUSE Linux Enterprise Server 12 SP4 as an example:

  1. Log in to the agent host as user root.
  2. Change the time zone of the agent host.

    • To change the time format to Coordinated Universal Time (UTC), run the following command:
      ln -sf /user/share/zoneinfo/UTC /etc/localtime
    • To change the time format to China Standard Time (CST), run the following command:
      ln -sf /user/share/zoneinfo/Asia/Shanghai /etc/localtime

      Replace Asia/Shanghai in the command with the target time zone.

  3. Run the following command to change the system time of the agent host:

    date -s "20241125 20:22:35"

  4. Run the following command to synchronize the hardware time with the system time:

    hwclock --systohc

  1. Log in to the agent host as user root.
  2. Run the following command to access System Management Interface Tool (SMIT):

    smit

  Change Time Zone Using User Inputted Values

Windows OS

The following uses Windows 10 as an example:

  1. Open Control Panel and click Clock and Region.
  2. Click Set the time and date.
  3. On the Date and Time page, click Change time zone..., select a time zone, and click OK.
  4. On the Date and Time page, click Change date and time..., set the time, and click OK.
  5. Click OK.
