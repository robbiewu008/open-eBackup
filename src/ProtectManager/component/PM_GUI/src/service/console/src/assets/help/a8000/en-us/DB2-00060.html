Managing DB2 Clusters or Tablespace Sets

This section describes how to modify or delete DB2 clusters registered with the OceanProtect.

Related Operations

Log in to the management page, choose Protection > Databases > DB2, click the Cluster or Tablespace Set tab, and locate the cluster or tablespace set on which you want to perform related operations.

Table 1 describes the related operations.

Table 1 Operations related to DB2 clusters or tablespace sets

Operation

Description

Navigation Path

Modifying cluster information

Context

This operation enables you to change the authentication mode and other information about a DB2 cluster.

In the row of the target cluster, choose More > Modify.

Deleting a cluster

Context

This operation enables you to delete a DB2 cluster if you no longer need to protect databases or tables in this cluster or restore data to it.

Note

  • A cluster cannot be deleted if a backup or restoration job is being performed.
  • A cluster cannot be deleted if a database or tablespace set in it has been associated with an SLA.
  • Deletion of a cluster will delete all its databases and tables.

In the row of the target cluster, choose More > Delete.
Authorizing resources
NOTE:

Only version 1.5.0 supports this function.

Context

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize this administrator as the system administrator to protect resources.

Note

  • The protection permissions on a resource that has been associated with an SLA cannot be granted. Before permission granting, remove the protection.
  • The protection permissions on a single resource can be granted only to a single data protection administrator.

In the row of the target cluster, choose More > Authorize Resource.
Reclaiming resources
NOTE:

Only version 1.5.0 supports this function.

Context

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

The protection permissions on a resource that has been associated with an SLA cannot be reclaimed. Before reclamation, remove the protection.

In the row of the target cluster, choose More > Reclaim Resource.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
