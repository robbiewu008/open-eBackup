Restoring Nutanix VMs

This section describes how to restore a backed-up VM to the original location or a new location.

Context

The OceanProtect allows you to restore VMs to the original or a new location using backup copies, replication copies (restoration to the original location is not supported), reverse replication copies (restoration to the original location is not supported), cascaded replication copies (restoration to the original location is not supported), archive copies (restoration to the original location is not supported after a replication copy is archived), or imported copies.
  • Restoration to the original location

    If you restore a VM to the original location, the system creates a VM in the path of the original VM (that is, the cluster or host to which the original VM belongs) and fully restores the backup data to the new VM.

  • Restoration to a new location

    For restoration to a new location, the system performs full restoration and restores the backup data to a specified new path. That is, the user specifies the host or cluster where the new VM is located and the corresponding storage pool.

Procedure

  1. Choose Explore > Copy Data > Virtualization > Nutanix.
  2. You can search for a copy by VM or copy. This section uses a VM as an example.

    Click the Resources tab, search for the VM to be restored by name, and click the VM name.

  3. On the Copy Data tab page, select the year, month, and day in sequence to locate the copy.

    If is displayed below a month or day, copies exist in the month or on the day.

  4. In the row of the target copy, choose More > Restore.
  5. Restore the VM to the original or a new location.

    • Restore the VM to the original location.

      Table 1 describes the related parameters.

      Table 1 Parameters for restoring a VM to the original location

      Parameter

      Description

      New VM Name

      Set the name of the VM to which the data is to be restored.

      Network Location

      Select the name of the port group to which the VM NIC is to be restored.

      Enable Network Adapters

      This option is used to set the connection status of the VM NIC after restoration. This function is disabled by default, that is, all VM NICs are disabled. After the VM is restored, you need to log in to the Nutanix management page to set the VM NIC connection. If this function is enabled, the network of the new VM may conflict with that of the original VM. In this case, check the network of the new VM after the VM is restored.

      Clients

      Specify the client for executing the restoration job. If this parameter is not specified, the system automatically allocates a client.

      Power on Automatically After Restoration

      If this function is enabled, the VM automatically starts after the restoration is complete.

      Copy Verification Before Restoration

      Verify the copy integrity before the restoration. This function cannot be enabled when the copy verification status is Invalid or --.
    • Restore the VM to a new location.

      Table 2 describes the related parameters.

      Table 2 Parameters for restoring a VM to a new location

      Parameter

      Description

      New VM Name

      Name of a VM to which the data is restored.

      Target Cluster

      Select the cluster where the target VM is located.

      Target Location
      Select the host where the target VM is located.
      NOTE:
      • If multiple hosts are selected, the Nutanix platform automatically schedules any selected host to start the target VM.
      • If no host is selected, the Nutanix platform automatically schedules any host to start the target VM.

      VM Disk Files

      Select the target datastore for storing VM disk files.

      Network Location

      Select the name of the port group to which the VM NIC is to be restored.

      Enable Network Adapters

      This option is used to set the connection status of the VM NIC after restoration. This function is disabled by default, that is, all VM NICs are disabled. After the VM is restored, you need to log in to the Nutanix management page to set the VM NIC connection. If this function is enabled, the network of the new VM may conflict with that of the original VM. In this case, check the network of the new VM after the VM is restored.

      Clients

      Specify the client for executing the restoration job. If this parameter is not specified, the system automatically allocates a client.

      Power on Automatically After Restoration

      If this function is enabled, the VM automatically starts after the restoration is complete.

      Copy Verification Before Restoration

      Verify the copy integrity before the restoration. This function cannot be enabled when the copy verification status is Invalid or --.

  6. Click OK.
