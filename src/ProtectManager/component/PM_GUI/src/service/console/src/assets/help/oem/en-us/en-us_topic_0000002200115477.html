Installing the Client (Automatic Push Mode, Applicable to 1.6.0 and Later Versions)

The client functions as a data connection component between the product and production environment and performs process control and data transmission during backup and restoration. Before backing up and restoring data, you need to install the client on the host. This section describes how to install a client for a host by allowing the system to automatically push the installation package to the host.

Precautions

  • If the OS of the host where the client is to be installed is Windows Server 2012 or earlier, the client cannot be installed in automatic push mode. In this case, install the client manually. For details, see Installing the Client (Manual Mode, Applicable to the Windows OS).
  • If the Windows host where the client is to be installed is a domain controller and the client needs to be installed on multiple domain controllers in the domain, manually install the client. For details, see Installing the Client (Manual Mode, Applicable to the Windows OS).

    You can run the wmic computersystem get domainrole /value command on the host to check whether the host is a domain controller. If DomainRole=4 or DomainRole=5 is displayed in the command output, the host is a domain controller.

  • If the protected resource is GaussDB (DWS) and the firewall is not required, run the systemctl stop SuSEfirewall2.service command to disable the firewall.
  • The GUI varies depending on the version.

Prerequisites

  • Ensure that the client is not installed on the host.
    • For a Linux, AIX, or Solaris host, run the vi /etc/profile command on the host. If the file does not contain the DATA_BACKUP_AGENT_HOME field, the client is not installed.
    • For a Windows host, run the sc query rdagent command on the host. If the command output indicates that the specified service is not installed, the client is not installed.
  • For a Linux, AIX, or Solaris host, ensure that the /mnt directory on the host is not mounted. You can run the mount | grep mnt command on the host for query.
  • For a Windows host, ensure that the C:\mnt directory on the host is not mounted. You can run the sc query rdagent command on the host for query. If the command output indicates that the specified service is not installed, the client is not installed.
  • You have obtained the IP address of the host where the client is to be installed. The IP address must be able to communicate with the backup network of the product.
  • You have obtained the username and password for logging in to the host.
  • If the host where the client is to be installed runs the Windows OS:
    • Ensure that the Windows remote management service has been enabled. For details, see Installation and configuration for Windows Remote Management.
    • If you install the client as a non-system administrator, ensure that the user has been added to the Administrators user group. For details, see How Can I Add a User to the Administrators User Group?.
  • During deployment of the product, a preset client software package of the corresponding version has been uploaded to the product WebUI. If the preset software package version does not meet your requirements, upload the software package of another version before installing the client. For details, see Uploading a Client Software Package.
  • When the host where the client is to be installed runs the Linux or AIX OS, if the value of ListenAddress in the /etc/ssh/sshd_config configuration file is modified, ensure that the value contains the backup network IP address of the host. Otherwise, connectivity check and push for installation will fail. That is, the value of ListenAddress is either the default value or the user-defined IP address plus the backup network IP address of the host.
  • If you have set aliases for rm, cp, mv, and other commands that need to be input in interactive mode on the Linux host where the client is to be installed, cancel the alias setting first. Otherwise, the installation will fail.

    For example, after you run the alias command on the host to view the configured command aliases, an alias is configured for the mv command if command output similar to alias mv='mv -i' is displayed. In this case, run the unalias mv command to cancel the alias configuration of the mv command.

Procedure

  1. Check whether the resource to be protected is an HDFS, HBase, Hive, Elasticsearch, Redis, or ClickHouse cluster and whether Kerberos authentication is configured in the cluster.

    • If yes, ensure that the time of the host where the client is to be installed, that of product, and that of Kerberos server are consistent before the installation. If the time is inconsistent, use the same NTP server. Otherwise, the client installation may fail.
    • If no, ensure that the time of the host where the client is to be installed is consistent with that of the product before the installation.

  2. Choose Protection > Clients > Clients.
  3. Click Register and set related parameters.

    Table 1 describes the parameters.

    Table 1 Parameters for installing the client

    Parameter

    Description

    Software Package Type
    Type of the client software package.
    • Client: Applies to all resources to be protected.
    • SAN Client: Select this type only if the following conditions are met: the protected resource is DB2; a Fibre Channel network is used in the AIX production environment; the SAN Client is required to implement backup.

    OS
    OS type of the host where the client is to be installed.
    NOTE:
    If OS is set to Windows:
    • Ensure that the Windows Remote Management (WS-Management) service on the host is in the Running state. For details, see How Do I Start the Windows Remote Management (WS-Management) Service?.
    • If the OS of the host where the client is to be installed is Windows Server 2012 or earlier, the client cannot be installed in automatic push mode. In this case, install the client manually. For details, see Installing the Client (Manual Mode, Applicable to the Windows OS).

    Application Type

    If Software Package Type is set to Client, select the application type that can be protected by the client.

    Message Signature Algorithm

    If OS is set to Linux, AIX, or Solaris, select a message signature algorithm.

    The message signature algorithm is used to verify the connectivity between the device and the host where the client is to be installed. hmac_sha2_256 is more secure than hmac_sha1 (which is insecure), and is selected by default.
    NOTE:

    After all parameters are set, click Test. If the system displays a message indicating that the client does not support the specified secure cipher suite, select another message signature algorithm. If both algorithms are not supported, upgrade the OpenSSH and SSL of the host.

    Sharing

    After this function is enabled, the client is applied to all data protection administrators.

    This parameter can be configured only when Software Package Type is set to Client, OS is set to Linux, and Application Type is set to HDFS, HBase, Hive, Huawei Cloud Stack, Huawei Cloud Stack GaussDB, Object Storage, or GaussDB (DWS).

    IP Type

    IP address type of the host where the client is to be installed. IPv4 and IPv6 are supported.

    Source Deduplication

    Source deduplication deletes redundant data from the source data before transmission. This reduces the amount of data to be transmitted but occupies more client resources (at least 4 GB of the client host memory and 30% of the maximum CPU usage will be occupied). You are advised not to enable source deduplication. If the NIC bandwidth is insufficient, that is, the amount of data to be backed up divided by the backup window is greater than the NIC bandwidth, you can enable source deduplication. For details about source deduplication, see the SourceDedupe User Guide of the desired version.

    To enable this function, ensure that the free memory of the host where the client is to be installed is greater than 2 GB.
    NOTE:
    • If LAN-free configuration needs to be performed for the client, enable this function.
    • After source deduplication is enabled, the system automatically installs OceanStor DataTurbo on the host where the client is installed and creates user dataturbo and user directory. Do not delete or modify the user and user directory.
      • For the Linux and AIX OSs, the user directory is /home/dataturbo.
      • For the Solaris OS, the user directory is /export/home/dataturbo.
      • For the Windows OS, the user directory is C:\Users\dataturbo.
    • If the host runs the Linux OS: After source deduplication is enabled, add the source deduplication user to sudoers before the first backup. For details, see How Can I Add a Source Deduplication User to sudoers? (Applicable to the Linux OS).
    • If source deduplication is not enabled during client installation, you can manually install OceanStor DataTurbo after the client is installed. For details, see Installing OceanStor DataTurbo.

    Network Plane Type
    If OS is set to Linux, select the network plane type of the client to be installed. The value can be Management network or Backup network.
    NOTE:
    • If Management network is selected, the product pushes the client software package to the host through the management network and automatically installs the client for the host.
      • If the IP address of the backup network is entered, the product automatically installs the client for the host through the backup network.
      • If no backup network IP address is entered, the product automatically selects the network connected to the product to install the client for the host.
    • If Backup network is selected, the product pushes the client software package to the host through the backup network and automatically installs the client for the host through the backup network.
    • If you need to specify the network used for backing up resources, you are advised to use the backup network IP address of the host for registration.

    Management Network IP Address

    When Network Plane Type is set to Management network, enter the management network IP address of the host where the client is to be installed.

    Backup Network IP Address
    Backup network IP address of the host where the client is to be installed. After the IP address is entered, it is used for registration. If this parameter is not specified, the system will check the connectivity of service port IP addresses and automatically select a reachable service port IP address for registration.
    NOTE:

    This parameter is displayed only when Network Plane Type is set to Management network.

    Backup network IP Address (Address Segment)
    Enter the backup network IP address or address segment of the host where the client is to be installed.
    NOTE:
    • For the Linux OS, this parameter is displayed only when Network Plane Type is set to Backup network.
    • Only IPv4 address segment is supported. The format is Start IP address-End IP address. Example: 10.10.10.1-10.10.10.10.

    Port
    SSH port or SFTP port of the host where the client is to be installed.
    • If a Windows OS is used, use the listening port of the Windows remote management service. The default value is 5985.
    • If another OS is used, use the listening port of the SSH service. The default value is 22.
    NOTE:
    • The listening port number of the SSH service must be the same as that of the SFTP service. Otherwise, the registration fails.
    • On the host where the client is to be installed, you can run the winrm enumerate winrm/config/listener command to query the listening port of the Windows remote management service. For details about how to install the Windows remote management service, see Installation and configuration for Windows Remote Management.

    Service IP Address
    When OS is set to Windows, you can determine whether to use the IP address of the host where the client is to be installed as the service IP address.
    • If you select Yes, the IP address is used for registration. The default value is Yes.
    • If you select No, the system will check the connectivity of service port IP addresses and automatically select a reachable service port IP address for registration.

    User Type
    If OS is set to Linux or AIX, select the user type of the host where the client is to be installed.
    • Administrator: a user who has the permission to run sudo commands. When registering a host, the administrator only needs to enter the username and password for logging in to the host.
    • Common User: a user who does not have the permission to run sudo commands. When registering a host, the common user needs to enter the sudo password and the username and password for logging in to the host.
    NOTE:

    If User Type is set to Common User, you need to add the common user to sudoers. Otherwise, the client installation will fail. For details, see How Can I Add a Common User to sudoers? (Applicable to the Linux OS).

    sudo Password-Free

    You can determine whether to use this function when User Type is set to Common User. This function controls whether a common user needs to enter the password of the current user when using the sudo permission.

    sudo Password

    If User Type is set to Common User, you need to enter the sudo password.

    Username
    Username and password of the host where the client is to be installed.
    NOTE:
    • If OS is set to Windows, the entered username cannot contain the domain name.

      For example, if the domain account is CHINA\Administrator, enter Administrator.

    • If OS is set to AIX and User Type is set to Common User, ensure that the entered user has the corresponding user directory. To query the user directory, perform the following steps:
      1. Use PuTTY to log in to the host where the client is to be installed.
      2. Run the following command to check whether the user directory exists:
        cd /home/Username

        If "No such file or directory" is displayed in the command output, no corresponding user directory exists. In this case, perform 3 to create one. Otherwise, the corresponding user directory exists.

      3. Run the following command to create a user directory:
        mkdir /home/Username

    Password

    Installation Directory

    Installation directory of the client.

    • If the OS is Linux, AIX, or Solaris, the client is installed in /opt by default and cannot be installed in /tmp, /dev, /bin, /usr, or the root directory. Special characters [|;&$><`\!] are not allowed in the name of the installation directory.
    • If the OS is Windows, the client is installed in the C:\ directory by default.
    NOTE:
    • The system automatically verifies the validity of the entered installation directory. The permission on the selected installation directory must be 755, and the user and owner group must be root.
    • The installation directory is stored in the /etc/profile configuration file. The environment variable is DATA_BACKUP_AGENT_HOME. Do not change this environment variable unless necessary.

    Auto-Sync Host Name

    During registration, the client name is the name reported by the host.

    After this function is enabled, the modified host names are automatically synchronized to the product WebUI. After this function is disabled, the registered client names are not automatically synchronized with the host names, but can be changed.

  4. Click Test.

    The system verifies whether the IP address, username, and password are correct. A maximum of 300 pieces of client information can be tested at a time.
    • In the Client Information area, you can click Modify or Delete corresponding to a client information record to modify or delete the information.
    • In the Client Information area, select multiple pieces of client information and click Batch Modify Username and Password to change the usernames and passwords in batches.

  5. After the test succeeds, click OK.

    The system automatically installs the client. The client can be installed on a maximum of 300 hosts at a time. If the host where the client is installed is displayed on the Clients page, the host has been successfully registered with the management page.
    • After the host is registered successfully, wait 5 minutes and then use the client.
    • After the client is installed, the system automatically creates user rdadmin and user directory on the host where the client is installed. Do not delete or modify the user and user directory.
      • For the Linux and AIX OSs, the user directory is /home/rdadmin.
      • For the Solaris OS, the user directory is /export/home/rdadmin.
      • For the Windows OS, the user directory is C:\Users\rdadmin.
