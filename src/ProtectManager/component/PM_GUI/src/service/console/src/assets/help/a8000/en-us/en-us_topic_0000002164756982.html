Managing Database Clusters

The OceanProtect allows you to perform operations, such as deletion and resource authorization or reclaiming, on the cluster where the database resides.

Related Operations

You need to log in to the management page, choose Protection > Databases > Oracle, click the Cluster tab, and find the host or cluster on which you want to perform related operations.

Table 1 describes the related operations.

Table 1 Operations related to an Oracle cluster

Operation

Description

Navigation Path

Authorizing Oracle resources

NOTE:

This parameter is available only in 1.5.0.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

In the row of the target cluster, choose More > Authorize Resource and select the data protection administrator for resource authorization.

Reclaiming Oracle resources

NOTE:

This parameter is available only in 1.5.0.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Note

The protection permissions on a resource that has been associated with an SLA cannot be reclaimed. Before reclamation, remove the protection.

In the row of the target cluster, choose More > Reclaim Resource.

Modifying a cluster

Scenario

This operation enables you to modify the information such as cluster name and nodes in the cluster.

In the row of the target cluster, choose More > Modify.

Deleting a cluster

Scenario

If you do not need to protect a database on an Oracle cluster or you want to use the database as a recovery target, you can delete the Oracle cluster. After deletion, related information will disappear from the management page.

Note

Before deleting a cluster, delete the databases that have been registered with the OceanProtect in the cluster.

In the row of the target cluster, choose More > Delete.
Adding a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to quickly filter and manage resources.

Note
  • The system administrator can add a maximum of 10,000 tags to a resource.
  • A non-system administrator must have the permission of Production resource management to add tags to resources.
  • A non-system administrator can add a maximum of five tags to a resource.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

This function is available only in 1.6.0 and later versions.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.
