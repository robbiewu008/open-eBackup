检查并配置Oracle数据库的Open状态

执行备份时，Oracle数据库所在主机必须处于开机状态，并且Oracle数据库为Open状态。以下步骤说明如何检查并配置Oracle数据库的Open状态。

操作步骤

  1. 登录Oracle数据库主机。

    • 如果是Linux操作系统，请使用PuTTY登录Oracle数据库主机，并执行su - oracle命令，切换到oracle账户。
    • 如果是Windows操作系统，请使用系统管理员登录Oracle数据库主机。

      ps -ef|grep ora

      1. 在键盘上按“Win+R”，打开运行窗口。
      2. 输入“services.msc”，进入服务界面查看服务状态。

        当以“Oracle”开头的服务状态为“正在运行”时，表示数据库已启动。

  3. 执行以下命令，登录Oracle数据库实例。

    • OS认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus / as sysdba

      • 如果是Windows操作系统，请执行以下操作：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus / as sysdba

    • 数据库认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus

        根据提示输入具备sysdba权限的用户：

        username as sysdba

        根据提示输入密码。

      • Windows OS执行以下命令：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus

          根据提示输入具备sysdba权限的用户：

          username as sysdba

          根据提示输入密码。

  4. 如果2检查到未启动数据库进程，执行以下命令启动数据库进程。否则请跳过。

    startup;

  5. 执行以下命令检查数据库状态。

    select instance_name,status from v$instance;

    • 当回显中“STATUS”为“open”时，表示数据库为Open状态。
    • 当回显中“STATUS”为非“open”时，请联系管理员将数据库开启至Open状态后再进行备份。
    SQL> select instance_name,status from v$instance; 
 INSTANCE_NAME    STATUS 
 ---------------- ------------ 
 orcl            open
