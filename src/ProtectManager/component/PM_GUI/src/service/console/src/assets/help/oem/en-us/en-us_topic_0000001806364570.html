Performing Manual Backup

You can manually execute a backup job to back up data immediately.

Prerequisites

Before the backup, ensure that the Exchange Server service and related dependent services are running properly. For details about the corresponding services, see Overview of Exchange services on Exchange servers.

Context

  • During manual backup, the parameters defined in the SLA (except Automatic Retry), such as Rate Limiting Policies, are applied to the manual backup task.
  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication or archive policy has been defined in the SLA, the will perform replication or archive once based on the SLA when you perform manual backup.
  • For mailbox backup, a registered user must have the ApplicationImpersonation role permissions. When the mailbox is backed up for the first time, the agent host automatically creates a user group named Databackup, which has the ApplicationImpersonation role permissions, and adds the registered user to the user group.

Procedure

  1. Choose Protection > Applications > Exchange.
  2. On the Email tab page, locate the row that contains the mailbox that you want to protect, and choose More > Manual Backup.

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

  3. Select a protection policy, which can be Full backup or Incremental backup.
  4. Set the name of the copy generated during manual backup.

    If this parameter is left empty, the system sets the name to Resource name_Timestamp by default.

  5. Click OK.
