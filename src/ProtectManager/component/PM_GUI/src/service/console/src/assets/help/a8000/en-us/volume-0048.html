About Live Mount

The OceanProtect supports live mount, in which a backup copy is mounted to the host that uses the data so that you can immediately restore data or access the backup copy data. Different from instant recovery, live mount does not automatically migrate data to the target database after the database is started using a copy and a mount copy is generated after the mounting. If you need to migrate data to the target platform, you can select the instant recovery mode or manually migrate data using the OceanProtect WebUI after live mount.

Application Scenarios

  • Data verification in the development and testing environment

    Quickly start the database using the backup copy, and run the test script and verify applications in the recovered database.

  • Data restoration within minutes

    In case the production host breaks down, you can start the database using the backup copy and temporarily use the database to serve as the production host until the production host is restarted, reducing the service interruption time.
