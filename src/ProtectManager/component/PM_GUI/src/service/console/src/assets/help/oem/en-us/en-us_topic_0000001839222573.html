Step 5: Registering a MySQL/MariaDB Database

After ProtectAgent is installed on the host where the MySQL/MariaDB database resides, the product automatically discovers the MySQL/MariaDB database host. You need to register the database of the host on the management page to protect the database or use the database as the restoration target.

Context

The product supports the backup of the MySQL/MariaDB database and other user-defined databases, but does not support the backup of the system databases information_schema, performance_schema, sys, and mysql. Therefore, the registration of the system database is not supported.

Prerequisites

Before registering a MySQL or MariaDB cluster instance, ensure that all cluster nodes of the database must be registered. Data cannot be backed up on unregistered nodes, and as a result, user data at the restoration time point will be inconsistent.

Data on all nodes in the cluster must be synchronized.

Registering a Single MySQL/MariaDB Instance

  1. Choose Protection > Databases > MySQL/MariaDB.
  2. Click the Instance tab.
  3. Click Register to register MySQL/MariaDB.

    Instances in the MySQL cluster environment cannot be used for single-instance registration.

  4. Select Type as Single instance.
  5. Configure the database instance and its authentication information.

    Table 1 describes the related parameters.

    Table 1 MySQL/MariaDB single-instance registration information

    Parameter

    Description

    Name

    Name of an instance, which can be customized.

    Hosts

    Database host to be registered.

    Add IP address for accessing MySQL

    The IP address for accessing MySQL instance.

    Username

    Enter the username created in Step 1: Enabling MySQL/MariaDB Database Permission.

    Password

    Password of the username.

    Port

    In the Linux OS, the port number is displayed. The default port number is 3306.

    Character Set

    NOTE:

    This parameter is displayed only in 1.6.0 and later versions.

    Database character set.

    MySQL Configuration File Path

    NOTE:

    This parameter is displayed only in 1.6.0 and later versions.

    This parameter is required if the configuration file is not stored in the default path such as /etc/my.cnf.

  6. Click OK.

Registering a MySQL/MariaDB Cluster Instance

  1. Create a cluster.

    1. On the MySQL/MariaDB page, click the Cluster tab.
    2. Click Register.
    3. On the Register page, customize the cluster name and select the type and cluster node.
      Table 2 describes the related parameters.
      Table 2 Parameters for registering a cluster

      Parameter

      Description

      Name

      User-defined cluster name.

      Type

      Cluster type, which can be Active/Standby Replication or PXC(Percona XtraDB Cluster).

      • If you select Active/Standby Replication, the node where the active MySQL/MariaDB instance resides must be included.
      • If you select PXC(Percona XtraDB Cluster), the nodes where the MySQL/MariaDB cluster instances are located must be included.

      Cluster Node

      Host where the MySQL/MariaDB instance is located.
      • The selected hosts must be from the same cluster.
      • If the database cluster is an active/standby cluster and the relay-log configuration item exists in the /etc/my.cnf file, set relay_log_recovery to on to avoid restoration failure.
    4. Click OK.

  2. Register the database of a cluster instance.

    1. On the MySQL/MariaDB page, click the Instance tab.
    2. Click Register.
    3. Select Type as Cluster instance.
    4. Enter an instance name and select a cluster.
    5. Click Add. In the displayed Add dialog box, add all nodes in the cluster instance.

      Table 3 describes the related parameters.

      Table 3 Node information of a cluster instance

      Parameter

      Description

      Hosts

      Select a host node in the cluster.

      eAppMySQL Address

      IP address for accessing the eAppMySQL service.

      Port

      In the Linux OS, the port number is displayed. The default port number is 3306.

      Username

      Username of the database.

      Password

      Password of the database.

      Character Set

      NOTE:

      This parameter is displayed only in 1.6.0 and later versions.

      Database character set.

      MySQL Configuration File Path

      NOTE:

      This parameter is displayed only in 1.6.0 and later versions.

      This parameter is required if the configuration file is not stored in the default path such as /etc/my.cnf.
    6. Click OK.
