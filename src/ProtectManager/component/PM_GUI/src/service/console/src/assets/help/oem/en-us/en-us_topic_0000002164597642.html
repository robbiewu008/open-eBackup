Checking the Network Connectivity When GaussDB (DWS) Backup or Restoration Fails

Fault Symptom

If a GaussDB (DWS) backup or restoration job fails, the job details contain information indicating that the Roach command fails to be executed.

Possible Cause

  1. The network between the GaussDB (DWS) node and the product is disconnected.
  2. The firewall is enabled on the GaussDB (DWS) node, but the ports required for GaussDB (DWS) backup and restoration are not enabled.

Suggestion

Check the network connectivity by referring to Checking Network Connectivity.

  • For cause 1, rectify the network fault.
  • For cause 2, open the ports required by the GaussDB (DWS) backup and restoration jobs. For details, see the Communication Matrix.
