执行Oracle 19C备份任务失败，错误详情包含“ORA-65250”错误码

现象描述

执行Oracle备份任务失败，错误详情如下：

RMAN-00571: ===========================================================
RMAN-00569: =============== ERROR MESSAGE STACK FOLLOWS ===============
RMAN-00571: ===========================================================
RMAN-03009: failure of backup command  on eBackup03 channel at xx/xx/2023 11:17:16
ORA-65250: invalid path specified for file - /mnt/databackup/oracle_backup_pathxxxx

可能原因

Oracle 19C开启了CDB模式，且备份的CDB里的一个PDB设置了db_create_file_dest参数。

根据官方文档，这是Oracle的一个已知BUG（BUG 30419450）。

处理步骤

  1. 登录Oracle数据库主机。

    • 如果是Linux操作系统，请使用PuTTY登录Oracle数据库主机，并执行su - oracle命令，切换到oracle账户。
    • 如果是Windows操作系统，请使用系统管理员登录Oracle数据库主机。

  2. 执行以下命令，登录Oracle数据库实例。

    • OS认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus / as sysdba

      • 如果是Windows操作系统，请执行以下操作：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus / as sysdba

    • 数据库认证：
      • 如果是Linux操作系统，请执行以下命令：

        export ORACLE_SID=数据库实例名称

        sqlplus

        根据提示输入具备sysdba权限的用户：

        username as sysdba

        根据提示输入密码。

      • Windows OS执行以下命令：
        1. 在键盘上按“Win+R”，打开运行窗口。
        2. 输入“cmd”，执行以下命令。

          set ORACLE_SID=数据库实例名称

          sqlplus

          根据提示输入具备sysdba权限的用户：

          username as sysdba

          根据提示输入密码。

  3. 执行以下命令，找到设置了db_create_file_dest参数的这个PDB，将其设置为空。

    alter session set container = <PDB NAME>;

    alter system set db_create_file_dest='' scope=both;

    命令中的<PDB NAME>请根据实际备份的PDB名称替换。

  4. 执行以下命令，确保db_create_db_file_dest参数为空，然后重新执行备份任务。

    show parameter db_create_file_dest;
