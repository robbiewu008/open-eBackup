步骤5：执行备份

执行备份前将需要保护的资源关联到指定SLA，系统将对这些资源按照SLA进行保护，并根据SLA周期性执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

注意事项

如果客户端所在主机为Windows OS，OceanProtect会默认将备份存储的文件系统挂载至“C:\mnt\databackup\”路径。如需修改挂载路径，可在“客户端安装目录\DataBackup\ProtectClient\ProtectClient-E\conf”目录下打开agent_cfg.xml配置文件，修改win_mount_public_path value配置项的值。

目录中的客户端安装目录请根据实际替换。

背景信息

  • 手动备份时，SLA中定义的参数（“失败后自动重试”除外），如“限速策略”会应用到手动备份任务中。
  • 手动备份产生的副本保留时长按照SLA中的定义进行保留。
  • 如果SLA中定义了复制策略或归档策略，则执行手动备份时，OceanProtect会按照SLA的定义执行一次复制或归档。

前提条件

备份前，请确保Active Directory服务正常运行。

操作步骤

  1. 选择“保护 > 应用 > Active Directory”。
  2. 选择需要保护的Active Directory域控制器，单击“保护”。

    您也可以勾选多个Active Directory域控制器进行批量保护。

  3. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

  4. 配置高级参数。

    相关参数说明如表1所示。
    表1 资源保护高级参数说明

    参数

    说明

    对象级备份

    默认开启。开启后生成的备份副本支持对象级恢复。

  5. 单击“确定”。

    如果当前系统时间已晚于SLA首次备份的开始时间，可以在弹出的对话框中选择立即执行手动备份或者根据SLA中设置的备份策略周期性自动执行备份。

  6. 可选：执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。

    1. 在需要备份的Active Directory域控制器所在行单击“更多 > 手动备份”。

      支持选择多个Active Directory域控制器批量进行手动备份。勾选多个Active Directory域控制器，在页面左上角选择“更多 > 手动备份”。

    2. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    3. 选择保护策略：全量备份。

      如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    4. 单击“确定”。
