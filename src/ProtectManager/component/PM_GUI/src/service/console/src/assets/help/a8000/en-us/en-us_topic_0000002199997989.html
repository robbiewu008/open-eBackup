Failed to Execute an Oracle 19C Backup Job and the Error Details Contain Error Code ORA-65250

Symptom

An Oracle backup job fails to be executed. The error details are as follows:

RMAN-00571: ===========================================================
RMAN-00569: =============== ERROR MESSAGE STACK FOLLOWS ===============
RMAN-00571: ===========================================================
RMAN-03009: failure of backup command  on eBackup03 channel at xx/xx/2023 11:17:16
ORA-65250: invalid path specified for file - /mnt/databackup/oracle_backup_pathxxxx

Possible Causes

The container database (CDB) mode is enabled for Oracle 19C, and the db_create_file_dest parameter is set for a pluggable database (PDB) in the backed-up CDB.

According to Oracle's official document, this is a known bug (BUG 30419450).

Troubleshooting

  1. Log in to the Oracle database host.

    • If the Linux OS is used, use PuTTY to log in to the Oracle database host and run the su - oracle command to switch to user oracle.
    • If the Windows OS is used, log in to the Oracle database host as the system administrator.

  2. Run the following commands to log in to the Oracle database instance.

    • OS authentication:
      • If the Linux OS is used, run the following commands:

        export ORACLE_SID=Database instance name

        sqlplus / as sysdba

      • If the Windows OS is used, perform the following operations:
        1. Press Win+R to open the Run window.
        2. Enter cmd and run the following commands:

          set ORACLE_SID=Database instance name

          sqlplus / as sysdba

    • Database authentication:
      • If the Linux OS is used, run the following commands:

        export ORACLE_SID=Database instance name

        sqlplus

        Enter the name of a user with the sysdba permissions as prompted:

        username as sysdba

        Enter the password as prompted.

      • If the Windows OS is used, perform the following operations:
        1. Press Win+R to open the Run window.
        2. Enter cmd and run the following commands:

          set ORACLE_SID=Database instance name

          sqlplus

          Enter the name of a user with the sysdba permissions as prompted:

          username as sysdba

          Enter the password as prompted.

  3. Run the following commands to find the PDB for which the db_create_file_dest parameter is set and leave it empty:

    alter session set container = <PDB NAME>;

    alter system set db_create_file_dest='' scope=both;

    Replace <PDB NAME> in the command with the actual name of the PDB that is backed up.

  4. Run the following command to ensure that the db_create_db_file_dest parameter is empty and perform the backup job again:

    show parameter db_create_file_dest;
