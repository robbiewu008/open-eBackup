查看TiDB副本信息

操作步骤

  1. 选择“数据利用 > 副本数据 > 数据库 > TiDB”。
  2. 选择副本浏览方式浏览副本信息。

    复制副本仅支持在复制到的目标集群查看。

    • 选择“资源”页签：以TiDB备份资源维度查看备份资源副本信息。当TiDB集群中表损坏或丢失，您需要恢复TiDB表时，建议从该入口进入查找相应时间点副本。

      参数说明如表1所示。

      表1 副本参数说明

      参数

      说明

      资源

      名称

      备份资源名称。

      类型

      备份资源类型。

      • TiDB集群。
      • TiDB数据库。
      • TiDB表集。

      位置

      受保护的TiDB集群节点名称。

      状态

      备份资源是否仍存在于当前集群。

      • 是，则状态为存在。
      • 否，则状态为不存在。

      副本

      数量

      基于该备份资源产生的所有副本数量。其中，复制副本数量仅在目标集群显示。

      复制副本SLA

      复制副本关联的归档SLA。

      如果已为该资源产生的复制副本配置归档SLA，则此处显示SLA名称。

      复制副本保护状态
      复制副本当前是否被保护。
      • 未保护：复制副本未关联SLA，或者复制副本已关联SLA但已禁用保护。
      • 已保护：复制副本已关联SLA，且已激活保护。
      • 创建中：复制副本已关联SLA，正在创建保护。仅1.5.0及之前版本存在该状态。
    • 选择“副本”页签：以TiDB备份资源的副本维度查看所有副本信息。当需要基于某个指定的副本进行恢复时，建议从该入口进入查找相应副本。

      参数说明如表2所示。

      表2 副本参数说明

      参数

      说明

      副本

      备份时间

      备份副本的生成时间。

      副本时间

      副本生成的时间。

      单击“副本时间”，可以查看副本过期时间等信息，过期时间显示“--”表示副本永久保留。

      状态
      副本的状态。
      • 正常：副本可用。
      • 无效：副本无效。
      • 删除中：副本处于正在删除过程中。
      • 恢复中：副本处于恢复过程中。
      • 删除失败：副本删除失败。

      位置

      副本所在的位置。

      • 当副本的“生成方式”为“备份”、“复制”、“级联复制”、“反向复制”时，副本存储在对应的备份存储单元，此处显示为对应备份存储单元的名称。
      • 当副本的“生成方式”为“对象存储归档”或“导入”时，副本存储在对象存储的桶上，此处显示为桶名称。
      • 当副本的“生成方式”为“磁带归档”时，副本存储在磁带库，此处显示介质集名称。

      生成方式
      副本的生成方式。
      • 备份：由备份任务产生的副本。
      • 复制：由复制任务产生的副本。

      副本类型

      生成副本的备份类型。

      包括：全量、日志。

      WORM
      副本是否已设置为WORM副本。
      • ：未设置。副本未设置为WORM副本，即副本为普通副本。
      • ：已设置。副本已设置为WORM副本，WORM副本不支持删除和缩短保留时间。
      • ：设置中。副本正在设置为WORM副本。处于该状态的副本不支持删除和修改保留时间。
      • ：设置失败。副本设置为WORM副本失败。设置失败的副本不支持删除和缩短保留时间。

      资源

      名称

      备份资源的名称。

      类型

      副本所在的备份资源类型。

      • TiDB集群
      • TiDB数据库
      • TiDB表集

      位置

      受保护的TiDB集群节点名称。

      状态

      备份资源是否仍存在于当前集群。

      • 是，则状态为存在。
      • 否，则状态为不存在。
