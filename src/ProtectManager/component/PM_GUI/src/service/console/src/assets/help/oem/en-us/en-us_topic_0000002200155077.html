Step 1: Registering the SAP HANA Database (Backint Backup Mode)

In Backint backup mode, backup data is written through third-party backup tools. Each SAP HANA service starts the Backint for SAP HANA agent and sends the backup data to third-party backup tools.

Procedure

  1. Choose Protection > Applications > SAP HANA.
  2. Register an SAP HANA instance.

    On the Instance tab page, click Register. Table 1 describes the parameters for registering an SAP HANA instance.
    Table 1 SAP HANA instance registration information

    Parameter

    Description

    Name

    User-defined instance name.

    Instance ID

    ID set during database installation. For details about how to query the instance ID, see Preparing for Backup.

    Hosts

    Host where the database resides.

    If the database is deployed in a cluster, select all hosts where the database resides.

    Authentication Mode

    Select Database authentication or Hdbuserstore mode.

    NOTE:
    • When the Database authentication mode is selected, you need to enter the username, password, and port number of the system database during registration.

    Hdbuserstore KEY
    If the Hdbuserstore mode is selected when the database is deployed in a cluster, you need to set the Hdbuserstore KEY parameter on all hosts where the instance resides.
    1. Use PuTTY to log in to the host where the SAP HANA database is located as user root.
    2. Run the following command to switch to the SAP system user <sid>adm to log in to the SAP HANA database terminal system. The username s00adm is used as an example.
      su - s00adm
    3. Run the following command to set the Hdbuserstore KEY parameter:
      hdbuserstore set S00SAPDB "hana1:30013" SYSTEM 1234

      In the preceding command, S00SAPDB is the KEY parameter set by the user, hana1 is the host name, 30013 is the default port number, and 1234 is the database password. Replace them with the actual values.

    4. Run the following command to check the Hdbuserstore KEY parameter:
      hdbuserstore list

      In the following command output, the Hdbuserstore KEY parameter is displayed in the line that contains KEY.

    System Database Username

    When Authentication Mode is set to Database authentication, enter the username and password of the system database.

    System Database Password

    System database port

    Port number set during database installation. The default port number is 30013.

    Log Backup

    Logs can be backed up only after this function is enabled.

    NOTE:

    Incremental or differential backup can be performed only after the log backup function is enabled.

    Regular Log Backup Interval

    Set the interval for backing up logs.

    NOTE:

    It is recommended that the interval be set to 50% to 100% of the log backup interval set in Table 1. For example, if the log backup interval set in the backup policy is 60 minutes, you are advised to set the interval to a value ranging from 30 minutes to 60 minutes.

    Log Backup Path

    Set the path for storing data log backup files.

    NOTE:
    • Ensure that the available space in the log backup path is greater than the space to be occupied by all log backup copies, and ensure that the SAP HANA log backup path permission is at least 755.
    • If the log backup path has been set in the production environment, it will be changed after this parameter is set.
    • The log backup path takes effect only after a backup job is executed.
    • The log backup directory must be a shared directory. To query the shared directory, perform the following steps:
      1. Use PuTTY to log in to the host where the SAP HANA database is located as user root.
      2. Run the following command to switch to the SAP system user:
        su - <sid>adm

        In the preceding command, sid indicates the SAP system ID. For example:

        hana1:~ # su - s00adm
s00adm@hana1:/usr/sap/S00/HDB00>

        In the command output, /usr/sap/S00/HDB00 is the soft link of the SAP system user directory.

      3. Run the following command to check the actual direction of the SAP system user directory and obtain the shared directory of SAP HANA:
        ll usr_Directory

        In the preceding command, usr_Directory indicates the soft link path obtained in 2. For example:

        s00adm@hana1:/usr/sap/S00/HDB00> ll /usr/sap/S00
total 4
lrwxrwxrwx 1 s00adm sapsys  27 Dec 11 09:41 HDB00 -> /home/hana/shared/K00/HDB00
drwxr-x--- 1 s00adm sapsys  38 Dec 11 09:48 hdblcm_uploads
drwxr-xr-x 1 s00adm sapsys 338 Dec 11 09:44 home
drwxr-x--- 1 s00adm sapsys  32 Dec 11 09:39 SYS

        In the command output, the upper level of the shared directory is /home/hana, which the shared directory of SAP HANA.

  3. Click OK.
  4. Register an SAP HANA database.

    On the Databases tab page, click Register. Table 2 describes the parameters for registering an SAP HANA database.
    Table 2 SAP HANA database registration information

    Parameter

    Description

    Type

    Select a database type.

    • System Database
    • Tenant Database
    NOTE:

    When registering a system database, tenant database, or cluster, you must have the backup administrator permission. You are advised to use SYSTEM as the system database username.

    Instance

    Select the corresponding registered instance.

    Name
    Name of the database to be protected, which must be the same as that of the database in the production environment. For details about how to query the name, see Preparing for Backup.
    NOTE:
    • If Type is set to System Database, select the database name that is set when the database is registered and installed.
    • If Type is set to Tenant Database, select the tenant database name set during tenant database registration.

    Hosts

    Host where the database resides.

    If the database is deployed in a cluster, select all hosts where the database is distributed.

    Authentication Mode

    Select Database authentication or Hdbuserstore mode.

    NOTE:
    • When the Database authentication mode is selected, you need to enter the username, password, and port number of the system database during registration.

    Hdbuserstore KEY
    If the Hdbuserstore mode is selected when the database is deployed in a cluster, you need to set the Hdbuserstore KEY parameter on all hosts where the database resides.
    1. Use PuTTY to log in to the host where the SAP HANA database is located as user root.
    2. Run the following command to switch to the SAP system user <sid>adm to log in to the SAP HANA database terminal system. The username s00adm is used as an example.
      su - s00adm
    3. Run the following command to set the Hdbuserstore KEY parameter:
      hdbuserstore set S00SAPDB "hana1:30013" SYSTEM 1234

      In the preceding command, S00SAPDB is the KEY parameter set by the user, hana1 is the host name, 30013 is the default port number, and 1234 is the database password. Replace them with the actual values.

    4. Run the following command to check the Hdbuserstore KEY parameter:
      hdbuserstore list

      In the following command output, the Hdbuserstore KEY parameter is displayed in the line that contains KEY.

    System Database Username

    When Authentication Mode is set to Database authentication, enter the username and password of the system database.

    System Database Password

    System database port

    Port number set during database installation. The default port number is 30013.

  5. Click OK.
