Restoring a File System

This section describes how to restore a backup file system to the original location or a new location.

Prerequisites

A file system backup copy has been generated.

Precautions

  • Restoration to a file system in WORM, CLONE, or non-UNIX security mode is not supported.
  • Restoration to a file system whose name starts with pvc_ is not supported.
  • Restoration to the secondary file system of asynchronous remote replication is not supported.
  • After HyperMetro brain-split occurs, the end that fails arbitration does not support restoration.
  • After an entire copy is used for restoration to the original location, new files added to the original file system after the copy is generated are not deleted. If you want to restore the file system to a certain point in time, you need to create a new file system on DeviceManager and specify the new file system for restoration.
  • Ensure that no other user is reading or writing the file system during the restoration. Otherwise, data may be inconsistent or even damaged.
  • The restoration operation is irreversible. If the restoration fails, data is not rolled back and some data may be damaged.

Procedure

  1. Choose Explore > Copy Data > Local File System.
  2. You can search for copies by resource or copy. This section uses the resource dimension as an example.

    On the Resources tab page, locate the file system to be restored by file system name and click the name.

  3. Click Copy Data and select the year, month, and day in sequence to find the copy.

    If is displayed below a month or day, copies exist in the month or on the day.

  4. In the row where the copy resides, choose More > Restore.
  5. Restore the file system to the original location or a new location.

    • Restore the file system to the original location.
      1. Select Original location for restoring the file system.
      2. Select an overwriting rule for files at the original location. Table 1 describes the parameters.
        Table 1 Parameters for restoring a file system

        Parameter

        Description

        Skip existing files

        The files that already exist in the file system are not processed during the restoration.

        Replace existing files

        The files that already exist in the file system are replaced during the restoration.

        Only replace the files older than the restoration file

        Among the files that already exist in the file system, only the outdated files are replaced.
    • Restore the file system to a new location.
      1. Select New location for restoring the file system.
      2. Select a file system.
      3. Select a share name.
      4. Select an overwriting rule for files at a new location. Table 1 describes the parameters.

  6. Click OK.
  7. Read the warning message carefully, select I have read and understand the consequences associated with performing this operation., and then click OK.
