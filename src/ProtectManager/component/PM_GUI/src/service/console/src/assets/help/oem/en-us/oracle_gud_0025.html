Step 6: (Optional) Enabling Backup Link Encryption

You need to enable backup link encryption if required.

Prerequisites

Ensure that the NFS Kerberos service has been configured for the storage system, and the rdadmin user and the OS user used for database installation have been added to the AD domain. (You can contact the application administrator to obtain the name and ID of the OS user.) Otherwise, the backup job may fail after backup link encryption is enabled. For details about how to configure the NFS Kerberos service, see "(Optional) Configuring the NFS Kerberos Service" in the Installation Guide specific to your product model.

Precautions

After backup link encryption is enabled, the ticket between the agent host and the KDC domain server has a validity period. If the ticket expires, backup jobs may fail. You are advised to prolong the validity period of the ticket in the Kerberos policy on the KDC domain server or configure the ticket to never expire. If the ticket is not configured to never expire, you must run the kinit or net command on the agent host to renew the ticket for each user added to the AD domain before it expires. For details, see "Configuring the Client" in the Installation Guide specific to your product model.

Creating a Local UNIX Authentication User Group

Before enabling backup link encryption, you need to add the user group to a local authentication user group if the current environment is not added to any domain environment.

  1. Log in to DeviceManager.

    • For some models of this product, perform the following operations:

  2. Choose Services > File Service > Authentication Users.
  3. On the UNIX Users tab page, select Local Authentication User Groups.
  4. Click Create.
  5. Configure local UNIX authentication user group parameters.

    • Name: Group to which the user belongs, which is set during database installation.
    • ID: Run the cat /etc/group command to view the ID of the group to which the user belongs.

Enabling Backup Link Encryption

  1. Choose System > Security > Data Security.
  2. Click Modify on the right of the Encryption Settings area and enable Backup Link Encryption.
  3. Click Save.
