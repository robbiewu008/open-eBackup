上传Kubernetes镜像压缩包至Kubernetes集群

本节以Kubernetes 1.23版本为例说明如何上传Kubernetes镜像压缩包至Kubernetes集群，不同版本操作可能略有差异。如果您有镜像仓库，也可上传镜像至镜像仓库。

前提条件

上传Kubernetes镜像压缩包前，请用户在OceanProtect管理界面上查询版本，再根据查询到的版本下载“OceanProtect_DataProtect_X.X.X_K8s_BackupImage.tgz”安装包，解压安装包后，获得对应的“k8s_backup_image_x86.tar”或“k8s_backup_image_arm64.tar”镜像压缩包。安装包通过以下方式获取。

  • 企业用户：点此前往
  • 运营商用户：点此前往
可通过以下方式查询版本。
  1. 单击右上角图标，选择“关于”。

    在弹框中“版本”所在行显示出版本。

操作步骤

  1. 登录Kubernetes集群任一节点，并上传安装包至该集群节点。
  2. 进入安装包路径后，执行以下命令解压安装包，获得对应的“k8s_backup_image_x86.tar”或“k8s_backup_image_arm64.tar”镜像压缩包。

    tar -xvzf OceanProtect_DataProtect_X.X.X_K8s_BackupImage.tgz

  3. 进入镜像压缩包路径后，执行以下命令上传镜像压缩包至Kubernetes集群节点，此处以k8s_backup_image_x86.tar镜像压缩包为例，ARM架构请上传相应的镜像压缩包。

    ctr -n k8s.io i import k8s_backup_image_x86.tar

  4. 重复执行1~3的操作，将Kubernetes镜像压缩包上传至其他集群节点。
