Step 8: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA. The system protects the resources and periodically executes backup jobs based on the SLA. You can perform a backup job immediately through manual backup.

Precautions

  • Copies generated by manual backup are retained for the period defined in the SLA.
  • Before the backup, do not add, modify, or delete data in the database on the management node.
  • Log backup will occupy a local storage space.

Procedure

  1. Choose Protection > Databases > GoldenDB.
  2. On the Instance tab, select the instance to be protected and click Protect.

    You can also select multiple instances for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resources to be protected, select an SLA without a WORM policy to avoid WORM policy conflicts.

  4. Set advanced backup parameters.

    Table 1 describes the related parameters.
    Table 1 Advanced backup parameters

    Parameter

    Description

    Script to Run Before Backup

    You can configure Script to Run Before Backup, Script to Run upon Backup Success, and Script to Run upon Backup Failure as required. When a message indicating that the prerequisite task starts to be executed is displayed during a backup job, the configured script will be executed in the corresponding scenario.

    Enter the absolute path and name of the script and ensure that the user who logs in to the database host has the permission to run the script.
    • Windows OSs support .bat scripts, for example, C:\script.bat.
    • Non-Windows OSs support .sh scripts, for example, /opt/script.sh.
      NOTE:

      If Script to Run upon Backup Success is configured, the backup job status is displayed as Successful on the product even if the script fails to be executed. Check whether the job details contain information indicating that a post-processing script fails to be run. If yes, correct the script in a timely manner.

  5. Click OK.

    If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  6. Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.
    1. Choose More > Manual Backup in the row of the instance to be backed up.
    2. Set the name of the copy generated during manual backup.
    3. If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.
    4. Select a protection policy, which can be full backup, incremental backup, or log backup.

      If the selected protection policy is different from that configured in the associated SLA for 1.6.0 and later versions, the WORM configuration does not take effect.

    5. Click OK.
