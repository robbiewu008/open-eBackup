Performing a Detection Manually

To execute a detection job immediately, you can perform manual detection.

Precautions

In HyperMetro domain scenarios, the secondary file system does not support intelligent detection.

Procedure

  1. Choose Data Security > Intelligent Detection.
  2. Click File Systems.
  3. In the row that contains the desired file system, choose More > Manual Detect.

    You can also select multiple file systems and choose More > Manual Detect.

  4. Enter the snapshot name. If you do not set this parameter, the default snapshot name snapshot_Timestamp is used.
  5. Configure the snapshot retention policy. Once the snapshot retention period ends, the system automatically deletes expired snapshots.
  6. Set Backup Copy In-Depth Detection.

    • After Backup Copy In-Depth Detection is enabled, you can adjust the sensitivity of the backup copy detection algorithm. If Sensitivity is set to High, an alarm may be triggered when a small amount of data is encrypted or similar operations are performed, increasing the risk of misreports. Medium sensitivity is recommended for service scenarios without special requirements.
    • The sensitivity adjustment of the backup copy detection algorithm takes effect only for VM, database, or Veeam-based host file backup copies.

  7. Set Uninfected Snapshot Lock. The Uninfected Snapshot Lock parameter means that snapshots for which no ransomware file is detected will be locked. After this function is enabled, uninfected snapshots will change to secure snapshots, and the snapshot retention period will be prolonged. Modification or deletion is not allowed before the snapshots expire.
  8. Click OK.
