Manually Executing Backup

If you want to perform a backup immediately, you can manually perform the backup.

Prerequisites

  • Before the backup, you need to ensure that the pexpect package has been installed on the agent host and install pexpect 3.2. For details, see "Installing pexpect" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.
  • Before the backup, you need to ensure that GBase 8a databases are running properly and the network connection of each node in the cluster is normal.
  • During the backup, the passwords of the GBase 8a agent host and database users cannot be changed.

Context

  • During the execution of a manual backup job, parameters (except Automatic Retry) defined in the SLA, such as Rate Limiting Policies, Job Failure Alarm, and Source Deduplication, take effect.
  • If a replication or archive policy has been defined in the SLA, the system will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Databases > General Databases.
  2. In the row that contains the database to be backed up, choose More > Manual Backup.

    You can select multiple resources to perform manual backup in batches. Select multiple resources and choose More > Manual Backup.

  3. Set the name of the copy generated through manual backup.

    If this parameter is left empty, the system sets the name to backup_Timestamp by default.

  4. Select a protection policy, which can be Full backup or Incremental backup.
  5. Click OK.
