Restoration Is Successful but the VM Fails to Be Started When the Original VM Is Overwritten for Restoration

Symptom

When restoring a VM to the original location, select Rewrite Original VM and Power on Automatically After Restoration in the restoration job. The restoration job is successfully executed, but the VM fails to be started. In addition, the restoration job uses the Hot-Add transmission mode, which can be found in the restoration job details.

Possible Causes

When a VM is restored in Hot-Add transmission mode, the VM disk is unmounted from the host where the client is installed after the data restoration is complete. However, the disk fails to be unmounted due to an internal error of the VDDK library and then gets locked. Consequently, the VM cannot be started.

Troubleshooting

  1. Use a browser to log in to the vCenter Server.

    Login address: https://vCenter Server IP address

  2. Manually unmount the VM disk from the host where the client is installed.

    1. Locate the host where the client is installed.
    2. Choose ACTIONS > Edit Settings.
    3. Locate the disk to be unmounted based on the restoration job ID.

      The restoration job ID can be obtained from the job details on OceanProtect, for example, b6f83bfa-c42d-476f-854a-f4f9f6b2e427.

    4. Click on the right of the disk to unmount the disk. When unmounting the disk, deselect Delete files from datastore.
    5. Click OK.

  3. Find the restored VM on the vCenter Server.
  4. Choose ACTIONS > Snapshots > Consolidate.
  5. After the consolidation is complete, choose ACTIONS > Power > Power On to start the VM.
