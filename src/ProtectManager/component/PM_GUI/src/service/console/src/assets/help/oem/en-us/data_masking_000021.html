Managing Anonymization Rules

You can view, modify, delete, and clone anonymization rules.

Precautions

Preset anonymization rules cannot be modified or deleted.

If a custom anonymization rule has been associated with an identification rule, you need to disassociate the anonymization rule from the identification rule before modifying or deleting the anonymization rule.

Management Operations

Table 1 describes the operations for managing anonymization rules.

Table 1 Operations for managing anonymization rules

Task

Description

Navigation Path

Viewing anonymization rules

You can view the quantity and details of anonymization rules.
In the navigation pane, choose Data Security > Data Anonymization > Anonymization Policies. Click the Anonymization Rules tab and view the names, rule sources, types, rule descriptions, rule examples, and number of associated identification rules of anonymization rules.
NOTE:

For 1.5.0, choose Explore > Data Utilization Policies in the navigation pane and click Anonymization Policies.

You can click the number of associated identification rules to view the related identification rules.

Modifying anonymization rules

You can modify the name and type of a custom anonymization rule.

In the Operation column on the right of the row containing the anonymization rule, choose More > Modify.

Deleting anonymization rules

You can delete a custom anonymization rule if it is no longer required.

In the Operation column on the right of the row containing the anonymization rule, choose More > Delete. You can delete multiple custom anonymization rules in batches.

Cloning anonymization rules

If an anonymization rule to be created is similar to an existing rule, you can clone and alter the existing rule to create one.

In the Operation column on the right of the row containing the anonymization rule, choose More > Clone.

If the anonymization rule to be cloned is a preset rule, click Clone.
