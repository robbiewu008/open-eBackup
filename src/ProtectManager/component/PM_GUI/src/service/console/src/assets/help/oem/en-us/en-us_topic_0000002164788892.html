Modifying a User

The product allows you to modify the information of existing users.

Precautions

The information about built-in accounts cannot be modified.

Procedure

  1. Choose System > Security > RBAC > Users.
  2. (Optional) Click next to Name to search for a user.
  3. In the row of the target user, click Modify on the right.
  4. Modify the user information. Table 1 describes the parameters.

    Table 1 Basic user parameters

    Parameter

    Description

    Login Authentication Mode

    Optional values are Login password and Login password+Email One-time password. This parameter is displayed only when the value of User Type is Local user.

    NOTE:
    • The Login password+Email One-time password mode provides higher security. To use this mode, ensure that you have configured the email address for receiving one-time passwords by referring to Managing the Email Service.
    • Built-in accounts support only the Login password mode.

    Recipient Email Address

    Email address for receiving one-time passwords. This parameter is displayed only when the user type is Local user and the login authentication mode is Login password+Email One-time password.

    Role

    Click Add. On the displayed page, select a built-in role provided by the system, or create a role and set user permissions. For details about how to create a role, see Creating a Role.

    Default Role

    Default role selected when the current user is created.

    Description

    Description of the new user.

    [Value range]

    The description can be left blank or contain up to 255 characters.

    User

    Max. Number of User Connections

    Maximum number of concurrent logins to the system using the new user. If this parameter is not set, the number is not limited.

    • : disabled

    The default status is .

    [Value range]

    The value ranges from 1 to 8.

    Default value: 5

    NOTE:

    For the Remote Device Administrator role, this parameter is not displayed.

  5. Click Next.
  6. (Optional) Associate the role with resource sets. This allows you to assign the manageable resource scope and bind different resource sets to different role permissions.

    1. Select the role to be associated with resource sets.
    2. Click Create Resource Set, customize the resource set name and description, and select the resources to be associated.
    3. click OK.

  7. Preview the modified user information and click Finish.
