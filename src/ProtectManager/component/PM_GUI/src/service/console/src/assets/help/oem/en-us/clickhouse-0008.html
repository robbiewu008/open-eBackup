Step 1: Registering a ClickHouse Cluster

Before backing up and restoring a ClickHouse database or table set, register the ClickHouse cluster with the product.

Prerequisites

ClickHouse hosts have been added.

Procedure

  1. Choose Protection > Big Data > ClickHouse.
  2. On the Cluster page, click Register to register a ClickHouse cluster.

    1. Enter the name of the ClickHouse cluster.
    2. Adds nodes. Table 1 describes related parameters.
      Table 1 Add ClickHouse nodes.

      Parameter

      Description

      Hosts

      Select the registered ClickHouse hosts.

      Client installation path

      Set the client installation path of the MRS cluster.

      Service IP Address

      Set the service IP address of a ClickHouse.

      Port

      Port for the client to connect to the ClickHouse server through TCP SSL. The default value is 21427. The value ranges from 21420 to 21469.

      Authentication Method

      Configure the authentication method for communication between the ClickHouse cluster and product. Set this parameter as follows:

      • Database authentication: Select this option when Kerberos authentication is disabled for the ClickHouse cluster. This mode authenticates usernames, which is less secure.
      • Kerberos Authentication: Select this option when Kerberos authentication is enabled for the ClickHouse cluster. The Kerberos protocol is used for identity authentication between the ClickHouse cluster and product.

      Database Username

      This parameter is mandatory only when Authentication Method is set to Database authentication.

      Enter the name of a user having the read and write permissions on the protected resources.

      Password

      This parameter is mandatory only when Authentication Method is set to Database authentication.

      Enter the password of the database user.

      Kerberos

      This parameter is mandatory only when Authentication Method is set to Kerberos Authentication.

      Select the created Kerberos authentication. When you register a ClickHouse cluster for the first time, click Create and configure Kerberos authentication parameters. Table 2 describes related parameters.

      Set Kerberos authentication parameters as described in Table 2.

      Table 2 Kerberos authentication parameters

      Parameter

      Description

      Name

      User-defined Kerberos authentication name.

      Principal Name

      Principal name of Kerberos authentication.

      NOTE:

      Set this parameter to the principal name configured on the Kerberos server, and ensure that the user must have read and write permissions on the resources to be accessed.

      Configuration Mode
      Kerberos authentication mode, which must be the same as that configured on the Kerberos server. Two authentication methods are available:
      • Password: Use the principal name and password for identity authentication.
      • Keytab File: Use the principal name and keytab file for identity authentication.

      Config File

      .conf configuration file (for example, krb5.conf) used during Kerberos authentication. Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the product.

      If renew_lifetime exists in the configuration file, comment it out. Otherwise, the registration may fail. Example: 
      #renew_lifetime = 7d

      Password

      This parameter is mandatory only when Configuration Mode is set to Password.

      Set this parameter to the password of Principal Name.

      Confirm Password

      Confirm password of Principal Name.

      Keytab File

      This parameter is mandatory only when Configuration Mode is set to Keytab File. .keytab configuration file (for example, krb5.keytab) used during Kerberos authentication.

      Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the product.

  3. Click OK. After the ClickHouse cluster is registered, the system automatically discovers the ClickHouse database. You can view the newly discovered database on the Databases tab page.
