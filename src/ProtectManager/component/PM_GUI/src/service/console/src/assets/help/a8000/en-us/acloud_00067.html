Managing Availability Zones, Resource Sets, or Cloud Servers

The OceanProtect allows you to modify, remove, activate, or disable protection for AZs, resource sets, or cloud servers. This section uses an AZ as an example to describe related operations. For resource sets or cloud servers, the operations are the same.

You need to log in to the management page and choose Protection > Cloud Platforms > Alibaba Cloud. On the AZs tab page, locate the target AZ.

Table 1 describes the related operations.

Table 1 Project protection plan operations

Operation

Description

Navigation Path

Modifying protection

Scenario

You can modify a protection plan to re-associate a project with an SLA or modify agent host information. After the modification, the next backup will be performed based on the new protection plan.

Note

  • The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.
  • If a backup job has been performed, reconfigure the SLA policy (change to the SLA with copy verification enabled) when modifying the protection plan. After the modification, the initial backup job will be converted to a full backup job if it is a non-full backup job.
  • If the SLA parameter associated with the resource changes (from copy verification disabled to copy verification enabled) after a backup job has been performed, when the protection plan for the resource is implemented after the change, the initial backup job will be converted to a full backup job if it is a non-full backup job.

Choose More > Modify Protection in the row that contains the target AZ.

Removing protection

Scenario

You can remove protection for an AZ that no longer needs to be protected. After the removal, the system cannot execute protection jobs for the AZ. To protect the AZ again, you need to associate an SLA with the AZ again.

Note

If a protection job of the AZ is in progress, protection cannot be removed.

Choose More > Remove Protection in the row that contains the target AZ.

Performing a manual backup

Scenario

This operation enables you to perform a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

Choose More > Manual Backup in the row that contains the target AZ.

Activating protection

Scenario

This operation enables you to activate a disabled protection plan for an AZ. After the protection plan is activated, the system executes backup, archiving, or replication for the AZ based on the plan.

Note

This operation can be performed only for an AZ that has been associated with an SLA and is in the Unprotected status.

Choose More > Activate Protection in the row that contains the target AZ.

Disabling protection

Scenario

This operation enables you to disable the protection plan of an AZ for which periodic protection is not required. After the protection plan is disabled, the system does not execute automatic backup, archiving, or replication for the AZ.

Note

Disabling a protection plan does not affect ongoing protection jobs.

Choose More > Disable Protection in the row that contains the target AZ.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Scanning Alibaba Cloud AZs

Scenario

The OceanProtect automatically scans AZs every hour. If any projects or cloud servers on the registered Alibaba Cloud change, the change will be automatically updated to the OceanProtect after the scan. This operation enables you to immediately update the changed information to the OceanProtect.

Note

If you do not select to apply an SLA to the newly created cloud server when creating a periodic backup job, the system will not associate the newly discovered cloud server with an SLA after the scan.

Choose More > Resource Scan.

Authorizing Alibaba Cloud resources

NOTE:

Only version 1.5.0 supports this function.

Scenario

By default, only the system administrator has the permission to protect resources. If you want to assign this permission to a data protection administrator, authorize the data protection administrator as the system administrator to protect resources.

Choose More > Authorize Resource and select a data protection administrator for authorization.

Reclaiming Alibaba Cloud resources

NOTE:

Only version 1.5.0 supports this function.

Scenario

This operation enables you to reclaim resources authorized to the data protection administrator. The data protection administration is not permitted to protect the resources that have been reclaimed.

Choose More > Reclaim Resource.
