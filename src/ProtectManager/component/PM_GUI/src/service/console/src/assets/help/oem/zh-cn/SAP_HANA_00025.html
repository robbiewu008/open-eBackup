步骤5：执行备份

执行备份前将需要保护的资源关联到指定SLA策略，系统将对这些资源按照SLA策略进行保护，并根据SLA策略周期性自动执行备份任务。如需立即执行一次备份任务时，可以通过手动备份实现。

背景信息

  • 手动备份时，SLA中定义的参数（“失败后自动重试”除外），如“限速策略”、“任务失败告警”、“源端重删”会应用到手动备份任务中。
  • 手动备份产生的副本保留时长按照SLA中定义的保留时长进行保留。

    如果手动备份时选择的保护策略未在SLA中定义，则产生的备份副本保留时长为永久保留。如SLA中未定义永久增量（合成全量）备份，而手动备份时选择的是永久增量（合成全量）备份，则本次手动永久增量（合成全量）备份产生的副本保留时长为永久保留。

  • 如果SLA定义了复制策略或归档策略，则执行手动备份时，系统会按照SLA的定义执行一次复制或归档。

操作步骤

  1. 选择“保护 > 应用 > SAP HANA”。
  2. 在“数据库”页签，勾选要保护的数据库，单击“保护”。
  3. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

    如果待保护的资源已设置WORM策略，为避免WORM策略冲突，请选择不包含WORM策略的SLA创建保护。

  4. 单击“确定”。

    如果当前系统时间已晚于SLA首次备份的开始时间，可以在弹出的对话框中选择立即执行手动备份或者根据SLA中设置的备份策略周期性自动执行备份。

  5. 可选：执行手动备份。

    当需要立即执行一次备份任务时，可以执行以下操作进行手动备份。否则，可跳过本步骤。

    1. 在需要备份的备份集所在行单击“更多 > 手动备份”。

      支持选择多个资源批量进行手动备份。勾选多个资源，在页面左上角选择“更多 > 手动备份”。

    2. 设置手动备份产生的副本名称。

      如果您不设置，系统默认设置副本名称为：backup_时间戳。

    3. 选择保护策略：全量备份或日志备份。

      对于1.6.0及后续版本，如果选择的保护策略与关联的SLA中配置的保护策略不一致，则WORM配置不生效。

    4. 单击“确定”。
