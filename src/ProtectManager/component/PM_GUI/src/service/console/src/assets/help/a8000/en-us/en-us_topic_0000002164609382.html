Restoring an SAP on Oracle Database

This section describes how to restore a backed-up database to the original or a new location.

Procedure

  1. Shut down the target database.

    Press Win+R, enter cmd, and press Enter to open Command Prompt on the Windows host where the target database resides. Perform the following steps to shut down the target database:

    1. Go to the run directory created in Manually Creating a run Directory and Copying Files to the Directory. Example:
      C:\>cd usr\sap\EH7\SYS\exe\run
    2. Run the following command to shut down the target database:
      brconnect -c force -l E -u userName/passWord -f dbshut -f
      userName and passWord indicate the name and password of the database user created in Starting the Database and Creating a Database User. Example:
      brspace -c force -l E -u user1/Huawei123 -f dbshow -c dbstate

  2. Choose Explore > Copy Data > Applications > SAP on Oracle.
  3. Search for copies by database or copy. This section describes how to search for copies by database.

    On the Resources tab page, locate the database to be restored by database name and then click the name.

  4. On the Copy Data tab page, select the year, month, and day in sequence to search for copies.

    If is displayed below a month or date, copies exist in the month or on the day.

  5. Restore data to a specified point in time or using a specific copy.

    1. Specify a copy or any point in time between two copies for restoration.
      • Specifying a copy for restoration

      • Specifying a point in time for restoration

        Data can be restored to a specific point in time in the blue part on the timeline. indicates that a copy exists at this point in time. You cannot specify a point in time for restoration if log backup is not performed at the point in time.

    1. Restore an SAP on Oracle database to the original or a new location.
      • Restore an SAP on Oracle database to the original location.
        1. Set Restore To to Original location.
        2. Enable or disable Channels.

          If this function is disabled, the default number of channels is 1. If this function is enabled, you can manually set the number of channels.

          The number of channels refers to the number of threads that can copy files at the same time for a restoration job. Each CPU can allocate a thread to execute a restoration job. The number of channels must be less than or equal to the number of CPUs on the host. Set this parameter based on the host CPU usage and number of CPUs.

        3. Click OK.
      • Restore an SAP on Oracle database to a new location.
        1. Set Restore To to New location.
        2. Set basic parameters for the restoration.

          Table 1 describes the related parameters.

          Table 1 Parameters of a restoration job

          Parameter

          Description

          Tag

          Name of a tag associated with the resource. You can filter databases by tag.

          Resources

          Name of the resource for restoring a database to a new location.
        3. Enable or disable Channels.

          If this function is disabled, the default number of channels is 1. If this function is enabled, you can manually set the number of channels.

          The number of channels refers to the number of threads that can copy files at the same time for a restoration job. Each CPU can allocate a thread to execute a restoration job. The number of channels must be less than or equal to the number of CPUs on the host. Set this parameter based on the host CPU usage and number of CPUs.

        4. Click OK.

  6. Click OK.
