Periodically Executing Backup

After resources to be protected are associated with an SLA, the system protects the resources based on the SLA.

Prerequisites

  • Before log backup, enable the TiDB log backup function in the production environment. For details, see Enabling or Disabling the Log Backup Function in the Production Environment.
  • A full backup must be performed before the first log backup or after the TiDB log backup function is enabled again in the production environment.

Procedure

  1. Choose Protection > Databases > TiDB.
  2. Click the Cluster tab (cluster, database, or table set backup is supported. Here, cluster backup is used as an example).
  3. Select the cluster to be protected and click Protect.

    You can select multiple clusters for batch protection.

  4. Select an SLA.

    You can also click Create to create an SLA.

  5. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
