Live Mounting a NAS Share

This section describes how to mount a NAS share copy to a target client for quick use.

Context

  • Mount Level indicates the mount level of the current backup copy. The initial value of Mount Level for a copy is 1.
  • If you select a copy whose Mount Level is 2 for mounting, you need to manually mount it to the target client based on the share path and share IP address. For details, see Manually Mounting a NAS File System or NAS Share.
  • Backup copies and replication copies can be used for live mount.

Procedure

  1. Choose Explore > Live Mount > Application Type > NAS Shares.
  2. Click Create.
  3. On the Select Resource tab page, select the source NAS share corresponding to a copy to be mounted, and click Next.
  4. On the Select Copy tab page, select the copy to be mounted and click Next.

    The copy to be live mounted must be in the Normal status.

  5. On the Mount Options tab page, configure mount parameters, and click Next.

    1. Configure basic and advanced parameters for live mount.
      Table 1 and Table 2 describe the related parameters.
      • You can set either or both of Bandwidth (MB/s) and Standard IOPS (8 KB). You are advised to set only Bandwidth (MB/s) for bandwidth-intensive services and only Standard IOPS (8 KB) for IOPS-intensive services. If you cannot determine the service type or the service type changes frequently, you can set both indicators and the system will control the traffic using the indicator with a smaller value.
      • You can query the conversion between the standard IOPS (8 KB) model and real I/O model in the conversion table on the GUI.
      • The maximum traffic of mounted copies is limited to ensure the performance of backup and restoration jobs. In addition, the minimum traffic of mounted copies is set to ensure the minimum performance of mounted copies.
      Table 1 Basic parameters for live mount

      Parameter

      Description

      File System Name

      Name of a mounted file system.

      The default name prefix is mount_.

      Share Protocol

      NFS or CIFS.

      You can configure NFS and CIFS shares at the same time.

      NFS

      Share Path

      Share path for accessing the file system. The path name is the same as the file system name and cannot be changed.

      Ensure that the backup resource supports the NFS share protocol. Otherwise, the NFS file permission information will not be contained during mounting.

      Share Client
      Information about the client that accesses a file system.
      • You can enter share client IP addresses, share client IP address segments, or an asterisk (*) to represent IP addresses of all share clients.
      • The IP addresses can be IPv4 addresses, IPv6 addresses, or a combination of IPv4 and IPv6 addresses.
      • You can enter multiple IP addresses or IP address segments and separate them by semicolons (;), spaces, or Enter. Example: 192.168.0.10;192.168.0.0/24;*

      UNIX Permission

      Permission level for a UNIX client to access the NFS share. Possible values are as follows:

      • Read-only: The client can only read files in the share.
      • Read-write: The client can read and write files in the share.
      • None: No operation is allowed on the share.

      root Permission Constraint

      Controls the root permission of a client. root_squash: does not allow the client to access the share as user root. Otherwise, the client will be mapped as an anonymous user. no_root_squash: allows the client to access the storage system as user root.

      CIFS

      Share Name

      Share name for accessing a file system.

      The default share name is mount_Timestamp. You can change it as needed.

      Ensure that the backup resource supports the CIFS share protocol. Otherwise, the CIFS file permission information will not be contained during mounting.

      Type

      Type of the user who accesses a CIFS share.

      • Everyone
      • Windows Local Authentication User
      • Windows Local Authentication User Group

      Permission

      Permission level for the added user or user group to access the CIFS share.

      The permission level from high to low is Forbidden > Full control > Read-write > Read-only. The highest permission takes effect. If a user is granted with a higher permission than its original one, the new permission takes effect immediately without re-authentication. For example, if the access permission of a user is Read-only and then the user is added to a user group with the Full control permission, the access permission of the user is upgraded to Full control and the user can access the CIFS share without re-authentication.

      Users

      Username for accessing a CIFS share.

      This parameter is mandatory when Type is set to Windows Local Authentication User or Windows Local Authentication User Group.

      If the required user or user group does not exist, create one on DeviceManager. For details about how to log in to DeviceManager, see Logging In to DeviceManager.

      NOTE:

      For the OceanProtect X series backup appliances or OceanProtect E1000 (with the OceanProtect used as backup storage), perform the following operations:

      Choose Services > File Service > Authentication Users > Windows Users from the navigation pane. On the Local Authentication Users or Local Authentication User Groups tab page, create a user or user group.

      Table 2 describes the advanced parameters for live mount.

      Table 2 Advanced parameters for live mount

      Parameter

      Description

      Bandwidth (MB/s)

      Min

      Minimum bandwidth for copy live mount.

      Max

      Maximum bandwidth for copy live mount.

      Burst

      Maximum burst bandwidth for copy live mount. The value of Burst must be greater than that of Max.

      The value of Burst can be set only after that of Max is set.

      Standard IOPS (8 KB)

      Min

      Average minimum number of standard I/O requests per second for copy live mount.

      Max

      Average maximum number of standard I/O requests per second for copy live mount.

      Burst

      Maximum burst IOPS for copy live mount. The value of Burst must be greater than that of Max.

      The value of Burst can be set only after that of Max is set.

      Max. Burst Duration (s)

      This parameter needs to be set only when the Burst parameter of the bandwidth or standard IOPS is set. It indicates the maximum duration of burst bandwidth or burst IOPS.

      Normalized Latency (8 KB)

      Max

      Maximum latency of the local storage system.
    2. Click Next.

  6. Preview the mount parameters. After confirming that the parameters are set correctly, click Finish.

    If a copy is used for live mount immediately after it is generated, the live mount job may fail because the copy is still in the initialization state. In this case, try again in 3 minutes.

  7. If you want to access a live mount copy, on the Live mount > NAS Shares page, choose More > View in the row that contains the live mount copy to be accessed. Then, obtain such information as the share path, share client, share IP address, share name, and username of the copy.
