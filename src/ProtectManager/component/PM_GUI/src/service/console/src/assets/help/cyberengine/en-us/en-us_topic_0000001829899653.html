Associating an Air Gap Policy

After creating an Air Gap policy, you need to associate the storage device with the Air Gap policy for the policy to take effect.

Prerequisites

Only storage devices in the Online state can be associated with Air Gap policies.

Procedure

  1. On the top menu bar, choose Data Security > Air Gap.
  2. Click the Storage Device tab.
  3. In the row that contains the desired storage device, choose More > Associated Policies.
  4. In the window that is displayed on the right, set related information. Table 1 describes the parameters.

    Table 1 Parameters for associating a policy

    Parameter

    Description

    Air Gap Policies

    Select a created Air Gap policy.

    Logical Port

    This parameter indicates the ports that support replication within the time window set in the Air Gap policy.

    NOTE:

    The selected logical port is the same as the replication port of the storage device.

    Linked Detection

    Intelligent detection and real-time detection are associated with the Air Gap policy. After this function is enabled, the system forcibly disconnects the replication link if the system detects that the file system of the current device is infected. In this case, the original Air Gap policy becomes invalid. After confirming that the file system is secure, you can choose More > Enable Policy on the storage device page to apply the policy.

  5. Click OK.
