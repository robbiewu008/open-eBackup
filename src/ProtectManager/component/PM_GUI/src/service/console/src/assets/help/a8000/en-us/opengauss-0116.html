Step 6: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA. The system protects the resources and periodically executes backup jobs based on the SLA. You can perform a backup job immediately through manual backup.

Precautions

  • During manual backup, the parameters (except Automatic Retry) defined in the SLA, such as Channels, Rate Limiting Policies, and Transmission and Data Encryption, are applied to the manual backup job.
  • Copies generated by manual backup are retained for the duration defined in the SLA.
  • If a replication or archive policy has been defined in the SLA, the OceanProtect Appliance will perform replication or archiving once based on the SLA when you perform manual backup.

Procedure

  1. Choose Protection > Databases > openGauss.
  2. Click the Instance and Databases tabs to protect instances and databases, respectively. The following uses database protection as an example.

    On the Databases tab page, select the database to be protected and click Protect.

    You can also select multiple databases for batch protection.

  3. Select an SLA.

    You can also click Create to create an SLA.

    If a WORM policy has been configured for the resources to be protected, select an SLA without a WORM policy to avoid WORM policy conflicts.

  4. Click OK.

    If the current system time is later than the start time of the first backup specified in the SLA, you can select Execute manual backup now in the dialog box that is displayed or choose to perform automatic backup periodically based on the backup policy set in the SLA.

  5. Optional: Execute manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.

    1. In the row of the target database, choose More > Manual Backup.

      You can select multiple resources to perform manual backup. Select multiple resources and choose More > Manual Backup in the upper left corner of the page.

    2. Set the name of the copy generated by manual backup.

      If this parameter is left unspecified, the system sets the name to backup_Timestamp by default.

    3. Select a protection policy, which can be Full Backup or Incremental Backup.

      For 1.6.0 and later versions, if the selected protection policy is different from that configured in the associated SLA, the WORM configuration does not take effect.

    4. Click OK.
