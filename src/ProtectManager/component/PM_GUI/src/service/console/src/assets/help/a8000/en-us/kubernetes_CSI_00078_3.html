Uploading the Kubernetes Installation Package and Obtaining the Image Name and Tag Information

Prerequisites

Before uploading the Kubernetes installation package, download the OceanProtect_DataProtect_X.X.X_K8s_BackupImage.tgz installation package based on the version and patch number queried on the OceanProtect WebUI. Decompress the installation package to obtain the compressed package k8s_backup_image_x86.tar or k8s_backup_image_arm64.tar of the corresponding product architecture. You can obtain the Kubernetes installation package by using the following methods:

  • For enterprise users: Click here.
  • For carrier users: Click here.
    You can query the version and patch number in the following way:
    1. Click in the upper right corner and choose About.
    2. In the dialog box that is displayed, the version and patch number are displayed in the line where Version is located.

Procedure

  1. Log in to the Kubernetes cluster in the background.
  2. Run the oc get csiNode command to check the nodes where the CSIDriver is installed. In the command output, the nodes whose DRIVERS value is not 0 are the worker nodes where the CSIDriver is installed.

  3. Upload the obtained image package to all worker nodes where the CSIDriver is installed.
  4. Log in to each worker node where the CSIDriver is installed. In the directory where the image package is stored, run the sudo podman load -i Image package name command to upload the image.
  5. After the upload is successful, run the sudo podman images command to view the image name and tag, which correspond to the values of the REPOSITORY and TAG parameters in the command output, respectively.
