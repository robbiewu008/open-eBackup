Step 5: (Optional) Configuring Log Backup

If log backup is configured in Step 4: Creating a Backup SLA, enable log backup for the SAP HANA database by following the instructions provided in this section for log backup to take effect. Skip this section in other scenarios.

Context

The SAP HANA database log backup function provided by the OceanProtect Appliance depends on the log backup capability of SAP HANA. After SAP HANA logs are backed up, they are backed up to the backup storage by the OceanProtect Appliance.

Procedure

  1. Log in to the system database.

    1. Use PuTTY to log in to the SAP HANA database host.
    2. Run the following command to switch to the system user:
      su - systemidadm

      Example:

      su - k00adm
    3. Run the following command to log in to the system database:
      hdbsql -i Instance ID -n Host name of the master node of the system database:SQL Port of the system database -d SYSTEMDB -u Username of the system database -p Password of the system database

      Example:

      hdbsql -i 00 -n localhost:30013 -d SYSTEMDB -u SYSTEM -p xxxxxxx

  1. Enable and configure the log backup function.

    1. Run the following command to enable SAP HANA database log backup:
      alter system alter configuration ('global.ini', 'SYSTEM') set ('persistence', 'enable_auto_log_backup') = 'yes' with reconfigure;
    2. Run the following command to set the directory for storing log backup copies:
      Ensure that the usable space of the log backup copy directory is greater than the space to be occupied by all log backup copies.
      alter system alter configuration ('global.ini', 'SYSTEM') set ('persistence', 'basepath_logbackup') = 'Directory for storing log backup copies'  with reconfigure;
    3. Run the following command to set the log backup frequency, that is, the interval for backing up logs:
      alter system alter configuration ('global.ini', 'SYSTEM') set ('persistence', 'log_backup_timeout_s') = 'Log backup interval(s)' with reconfigure;

      It is recommended that the interval be 50% to 100% of the log backup interval set in Table 1. If the log backup interval is set to 60 minutes in the backup policy, you are advised to set the interval to a value ranging from 30 to 60 minutes.
