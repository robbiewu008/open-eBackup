Checking the openGauss/CMDB Database Environment Information

After an openGauss/CMDB database is registered with the OceanProtect, you can view details about the database and the host or cluster where the database resides on the management page.

Procedure

  1. Choose Protection > Databases > CMDB/openGauss.
  2. View related information on the Databases tab page.

    1. View database information. Table 1 describes the related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 1 Database information

      Parameter

      Description

      Name

      Database name.

      Owning Cluster

      Name of the host or cluster where the database resides.

      Owning Instance

      Database instance name.

      SLAs

      If an SLA is associated with the database, the SLA name is displayed. Otherwise, -- is displayed.

      SLA Compliance
      Whether a backup job is successfully executed based on the backup interval or backup window set in the SLA.
      • Met: If the backup time of a backup job (except for the first full backup) meets the backup time window or backup interval requirements and the backup job is executed successfully, SLA compliance is met.
      • Missed: If the backup time does not meet the backup time window or backup interval requirements, for example, if the backup interval set in the SLA for a backup job is once every half an hour but the backup time exceeds half an hour, SLA compliance is missed. In this case, you are advised to adjust the backup interval or backup window.

      Protection Status

      Whether a database is protected.

      • Unprotected: The database is not associated with an SLA, or the database is associated with an SLA but protection is disabled.
      • Protected: The database is associated with an SLA and protection is activated.
      • Creating: The database is associated with an SLA and protection is being created.
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
    2. Click the name of a resource and check the overview, copy information, and related jobs.
      • Overview

        The unique cluster ID, SLA information, job execution time, and total number of copies are displayed.

      • Copy Data

        You can search for copies by year, month, or day.

        above the time indicates that a copy exists at the time.

      • Jobs

        Displays all jobs related to resources, including restoration and archive jobs.

  3. On the Cluster tab page, query the openGauss/CMDB cluster information.

    1. This page displays information about openGauss clusters registered with the OceanProtect.

      Table 2 describes the related parameters.

      Some parameters are not displayed by default. You can click next to Operation and select a parameter to display it on the current page.

      Table 2 openGauss/CMDB cluster information

      Parameter

      Description

      Name

      openGauss/CMDB cluster name.

      Status

      openGauss/CMDB cluster status.

      • Normal: The openGauss/CMDB cluster is available and the data has been backed up. All processes are running and the primary-standby relationship is normal.
      • Unavailable: The openGauss/CMDB cluster is unavailable.
      • Degraded: The openGauss/CMDB cluster is available but the data has not been backed up.

      Version

      openGauss/CMDB version number.

      Type

      Cluster type.

      • Single Server
      • Active/Standby Replication
      • Distributed CMDB Cluster
      Tag
      NOTE:

      This parameter is available only in 1.6.0 and later versions.

      If a tag has been added for the resource, the tag name is displayed. Otherwise, -- is displayed.
      Owner
      NOTE:

      This parameter is available only in 1.5.0.

      Data protection administrator of a host or cluster.

      If a host or cluster has been authorized to a data protection administrator, the name of the data protection administrator is displayed. If no data protection administrator is authorized, -- is displayed.
    2. Click the name of a cluster to view its basic information and instances.
      • Basic Info

        Displays the cluster name, openGauss/CMDB version number, cluster type, and nodes in the cluster.

      • Instance

        Displays the instances in the cluster.

  4. View related information on the Instance tab.

    1. This page displays details about openGauss/CMDB instances.

      • Asynchronous

      SLAs
