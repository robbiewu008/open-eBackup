Creating a File Name Extension Filtering Rule

Known ransomware file name extensions to be intercepted are preset in the system. You can refer to this section to add file name extensions.

Procedure

  1. On the navigation pane, choose Explore > Ransomware Protection > Ransomware Detection.
  2. On the File Name Extension Filtering Rule tab page, click Create.

    The Create File Name Extension Filtering Rule dialog box is displayed.

  3. Set parameters of the file name extension filtering rule.

    Table 1 describes the related parameters.

    Table 1 Parameters of the file name extension filtering rule

    Parameter

    Description

    File Name Extensions to Be Intercepted

    File name extensions to be intercepted. You can enter one or more file name extensions.

    [Value range]

    • The value contains 1 to 127 characters.
    • A file name extension can contain only digits, letters, spaces, and special characters (!"#$%&'()*+,-./:;<=>?@[\]^_`{|}~). Wildcards ? and * are supported, and wildcard * must be the last character. For example, T?X or Tx*.
    • Separate multiple file name extensions with commas (,).

    Associate with vStore

    Select whether to associate a vStore.

    • Off: The file name extension filtering rule is created but is not associated with any vStore.
    • On: The file name extension filtering rule is created and vStores to be associated are selected from the vStore list.
    NOTE:

    By default, the preset file name extension filtering rule is associated with all vStores in the system.

  4. Click OK.
