集群副本恢复到新集群，因下发恢复命令的节点解析结果出错，恢复任务失败

故障现象

集群副本恢复到新集群，恢复任务失败。

可能原因

所有GaussDB(DWS)节点数据恢复已成功并将结果返回给下发恢复命令的节点，但下发恢复命令的节点解析结果出错，导致恢复任务失败。

处理建议

  1. 登录下发恢复命令的GaussDB(DWS)节点。
  2. 依次执行以下命令切换到集群用户并加载环境变量。

    su - 集群用户名
    source 环境变量文件路径

  3. 执行cd $GAUSSLOG/roach/controller/命令进入日志文件目录，查看以“roach_controller”开头的日志文件中是否包含以下错误信息:

    Error: The return code for failed subprocess on xxx.xxx.xxx.xxx is 1

    示例如下：

    • 如果包含错误信息，请重新执行一次恢复任务。
    • 如果不包含错误信息，请收集日志并联系技术支持工程师协助解决。
