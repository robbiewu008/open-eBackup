Adding Object Storage (When the Archive Storage Is Object Storage)

This section describes how to add the object storage to the product when the archive storage is object storage. If the archive storage is not the object storage, skip this section.

Prerequisites

  • An object storage platform for storing archive copies is ready.
  • The archive network of the product has been configured and is properly connected to the object storage network.

    If the archive network is not configured, configure it by referring to "Configuring the Archive Network" in the Administrator Guide.

  • You have obtained the access address and AK/SK of the object storage platform.
  • If the object storage product is OceanStor Pacific, OceanStor 100D (FusionStorage OBS), Huawei Cloud OBS, or AWS S3, ensure that you have created a data bucket (for storing archive data) and an index bucket (for storing index data) on the object storage platform.

    To ensure data security, create a private bucket that can be used only by the product. Otherwise, archived data may be illegally intercepted or tampered with.

Precautions

When OceanStor Pacific 8.1.0, 8.1.1, or 8.1.2, is used as the archive storage or when OceanStor Pacific 8.1.3 or later is used as the archive storage and copies are archived to a protocol-interworking bucket of OceanStor Pacific, the following restrictions exist:

  • If the /index directory exists in a copy, the copy cannot be archived.
  • If a copy contains directories and files with the same name, the copy cannot be archived.

The preceding restrictions do not apply when the archive storage is OceanStor Pacific 8.1.3 or later and data is archived to native buckets of OceanStor Pacific.

Procedure

  1. Choose System > Infrastructure > Archive Storage Library.
  2. Click Add Archive Storage.
  3. Set archive storage parameters.

    Table 1 describes the related parameters.

    Table 1 Archive storage parameters

    Parameter

    Description

    Name

    Name of an archive storage, which is customized.

    Type

    Type of the archive storage.

    NOTE:

    For 1.5.0 and earlier versions, only S3-compliant object storage is supported.

    Object Storage Product

    Object storage product connected to the product.

    OceanStor Pacific, OceanStor 100D (FusionStorage OBS), Huawei Cloud OBS, Microsoft Azure Blob Storage (only supported in 1.6.0 and later versions), and AWS S3 are supported.

    NOTE:

    If you select OceanStor Pacific for Object Storage Product, you need to log in to DeviceManager of OceanStor Pacific by referring to Logging In to DeviceManager and perform the following operations to enable V2 authentication: choose Settings > Object Service Settings > Security Settings, and on the displayed Security Access Policy tab page, click to the right of Enable V2 Authentication.

    Connection Mode

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    Mode for connecting the product to Microsoft Azure Blob Storage. This parameter is displayed only when Object Storage Product is set to Microsoft Azure Blob Storage.

    Connection String

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    This parameter is displayed only when Object Storage Product is set to Microsoft Azure Blob Storage and Connection Mode is set to Connection string.

    To obtain the connection string of Microsoft Azure Blob Storage, perform the following steps:
    1. Log in to the Microsoft Azure official website.
    2. On the home page, choose Storage accounts and select the corresponding storage account name.
    3. On the Security + networking > Access keys page, obtain the value of Connection string.
    Blob Port
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    This parameter is displayed only when Object Storage Product is set to Microsoft Azure Blob Storage.

    Default port of the protocol for data transmission between the product and Microsoft Azure Blob Storage.
    • If Protocol is set to HTTP, port number 80 is used.
    • If Protocol is set to HTTPS, port number 443 is used.

    Blob Container

    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    This parameter is displayed only when Object Storage Product is set to Microsoft Azure Blob Storage and Connection Mode is set to Standard mode.

    Blob container for storing archive data. To obtain the container, perform the following steps:
    1. Log in to the Microsoft Azure official website.
    2. On the home page, choose Storage accounts and select the corresponding storage account name.
    3. On the Data storage > Containers page, obtain the container name.

    Endpoint

    Access address of an archive storage.

    Address format: domain name or service plane IP address of an archive storage.

    To obtain the endpoint of Microsoft Azure Blob Storage, perform the following steps:
    1. Log in to the Microsoft Azure official website.
    2. On the home page, choose Storage accounts and select the corresponding storage account name.
    3. On the Settings > Endpoints page, obtain the value of Blob service for the primary or secondary point.
    NOTE:
    • If the access address of archive storage is in domain name format, configure the DNS service by referring to "(Optional) Configuring DNS" in the Installation Guide to support domain name resolution.
    • If the access address of archive storage is in IP address format, the system uses port 80 to access the archive storage by default when the protocol is HTTP and uses port 443 when the protocol is HTTPS. If you do not use the default port to connect to the archive storage, enter both the IP address and the port used to connect to the archive storage, for example, 10.10.1.1:60.

    Protocol

    Protocol used for data transmission between the product and archive storage.

    • When HTTPS is used, import a valid certificate so that the product can verify the archive storage information. Obtain the certificate from the archive storage administrator in advance. For details about how to import a certificate, see "Adding an External Certificate" in the Security Configuration Guide.

      If the CA certificate is not a root CA certificate, prepare a complete certificate chain file before importing the certificate. For details, see "Creating a Certificate Chain File" in the Security Configuration Guide.

    • If HTTP is selected, the product cannot verify the archive storage environment information, which poses security risks. You are advised to select HTTPS for security purposes.
      NOTE:

      If you select OceanStor Pacific for Object Storage Product and HTTP for Protocol, you need to log in to DeviceManager of OceanStor Pacific by referring to Logging In to DeviceManager and perform the following operations to enable HTTP: choose Settings > Object Service Settings > Security Settings, and on the displayed Security Access Policy tab page, click to the right of Enable HTTP.

    Certificate

    This parameter is displayed only when HTTPS is used. Select a certificate whose Type is Object Storage to verify the archive storage environment information.

    NOTE:

    You can view the certificate Type in System > Security > Certificate.

    AK

    Access certificate for accessing the object storage.

    For Microsoft Azure Blob Storage, the AK value refers to the storage account name. To obtain the storage account name, perform the following steps:
    1. Log in to the Microsoft Azure official website.
    2. On the home page, choose Storage accounts and select the corresponding storage account name.
    3. On the Security + networking > Access keys page, obtain the value of Connection string.

    SK

    Security certificate for accessing the object storage.

    To obtain the SK of Microsoft Azure Blob Storage, perform the following steps:
    1. Log in to the Microsoft Azure official website.
    2. On the home page, choose Storage accounts and select the corresponding storage account name.
    3. On the Security + networking > Access keys page, obtain the value of Storage account name.

    Data bucket

    Name of the bucket that stores archived data. This parameter is unavailable when Object Storage Product is set to Microsoft Azure Blob Storage.

    Index bucket

    Name of the bucket that stores index data. This parameter is unavailable when Object Storage Product is set to Microsoft Azure Blob Storage.

    Use Proxy Server

    When the product uses an agent server to access the archive storage, configure the address, username, and password of the agent server. This parameter is unavailable when Object Storage Product is set to Microsoft Azure Blob Storage.

    NOTE:
    • If an agent server is required, ensure that the maximum size of the request body allowed by the agent server is greater than 100 MB.
    • The agent server supports only the HTTP protocol. The address format is http://Agent server IP address:Port number.

    Capacity Alarm Threshold

    Upper threshold of the used capacity of a bucket. When the used capacity reaches the threshold, an alarm is reported. This parameter is unavailable when Object Storage Product is set to AWS S3 or Microsoft Azure Blob Storage.

  4. Click OK and confirm as prompted.
