An Internal Error Occurs During the Subjob of a Backup Job

Symptom

An incremental, differential, or log backup job fails for GaussDB T, and a message indicating an internal error is displayed in the job details area on OceanProtect GUI.

Possible Cause

The system time of the host where the client is installed has been changed for multiple times. As a result, the copy time is disordered, and the system fails to query the latest copy.

Troubleshooting

  1. Log in to the database host where the backup fails and use Roach to perform a full backup.
  2. After the full backup is successful, use Roach to perform an incremental backup.

    • If the full backup fails and a message indicating that the latest full copy cannot be found and full backup needs to be performed again is displayed, perform 3.
    • If the backup fails and other messages are displayed, contact Huawei technical support engineers.

  3. Check whether the time of all GaussDB T hosts is consistent.

    • If no, change the host time to ensure that the time of all GaussDB T hosts is consistent.
    • If yes, perform 4.

  4. Find all previously generated copies and use Roach to delete all of them.
  5. On the OceanProtect WebUI, manually perform a full backup again and then perform an incremental backup again.

    • If the incremental backup succeeds, no further action is required.
    • If the incremental backup fails, contact Huawei technical support.
