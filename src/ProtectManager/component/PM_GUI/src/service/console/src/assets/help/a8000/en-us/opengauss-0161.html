Managing openGauss/CMDB

The OceanProtect allows you to modify, remove, activate, or disable protection for openGauss/CMDB databases.

Related Operations

You need to log in to the management page, choose Protection > Databases > openGauss, click the Databases or Instance tab, and find the target database or instance.

Table 1 describes related operations.

Table 1 Operations related to openGauss/CMDB database or instance protection

Operation

Description

Navigation Path

Modify Protection

Scenario

If you need to reselect an SLA associated with a resource, you can modify the protection plan. After the modification, the new protection policy will be used for the next replication or archiving.

Note

The modification does not affect the protection job that is being executed. The modified protection plan will take effect in the next protection period.

Choose More > Modify Protection in the row where the resource is located.

Remove Protection

Scenario

If you do not need to protect a resource, you can remove the protection. After the removal, the system cannot execute protection jobs for the resource. To protect the resource again, you need to re-associate it with an SLA.

Note

When a protection job of the resource is running, the protection cannot be removed.

Choose More > Remove Protection in the row where the resource is located.

Disable Protection

Scenario

This operation enables you to disable a protection plan when periodic protection for the resource is not required. After the protection plan is disabled, the system does not execute automatic replication or archive jobs of the resource.

Note

Disabling protection does not affect ongoing backup jobs.

Choose More > Disable Protection in the row where the resource is located.

Activate Protection

Scenario

After a protection plan for a resource is disabled, you can activate the protection plan to make it take effect again. After the protection plan is activated, the system backs up the resource based on the protection plan.

Note

This operation can be performed only on instances that have been associated with an SLA and are in the Unprotected state.

Choose More > Activate Protection in the row where the resource is located.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Performing restoration

Scenario

This operation enables you to use a copy to restore database data.

In the row of the target resource, choose More > Restoration.

Performing a manual backup

Scenario

This operation enables you to execute a backup immediately.

Note

Copies generated by manual backup are retained for the duration defined in the SLA.

In the row of the target resource, choose More > Manual Backup.
