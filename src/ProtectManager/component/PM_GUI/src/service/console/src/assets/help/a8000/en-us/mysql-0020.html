Step 7: (Optional) Enabling Backup Link Encryption

You need to enable backup link encryption if required.

Prerequisites

Ensure that the NFS Kerberos service has been configured for the storage system, and the username used for resource registration has been added to the AD domain. Otherwise, the backup job may fail after backup link encryption is enabled. For details about how to configure the NFS Kerberos service, see "(Optional) Configuring the NFS Kerberos Service" in the Installation Guide specific to your product model.

Precautions

After backup link encryption is enabled, the ticket between the agent host and the KDC domain server has a validity period. If the ticket expires, backup jobs may fail. You are advised to prolong the validity period of the ticket in the Kerberos policy on the KDC domain server or configure the ticket to never expire. If the ticket is not configured to never expire, you must run the kinit or net command on the agent host to renew the ticket for each user added to the AD domain before it expires. For details, see "Configuring the Client" in the Installation Guide specific to your product model.

Creating a Local UNIX Authentication User Group

Before enabling backup link encryption, if the current environment is not added to any domain environment, you need to add the user group to the local authentication user group.

  1. Log in to DeviceManager.

    • Perform the following operations for OceanProtect X series backup appliances:
      1. Choose System > Infrastructure > Cluster Management.
      2. On the Backup Clusters tab page, click a node name under the Local Cluster Nodes area.
      3. On the displayed Node Details page, click Open the device management platform to go to DeviceManager.
    • For OceanProtect E1000, log in to DeviceManager by referring to Logging In to DeviceManager.

  2. Choose Services > File Service > Authentication Users.
  3. On the UNIX Users tab page, select Local Authentication User Group.
  4. Click Create.
  5. Configure local UNIX authentication user group parameters.

    • Name: Group to which the user belongs, which is set during database installation.
    • ID: Run the cat /etc/group command to view the ID of the group to which the user belongs.

Enabling Backup Link Encryption

  1. Choose System > Security > Data Security.
  2. In the Encryption Settings area, click Modify on the right of the page and enable Backup Link Encryption.
  3. Click Save.
