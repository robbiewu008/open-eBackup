Step 1: Registering a Redis Cluster

Before backing up or restoring a Redis cluster, register the Redis cluster with the product.

Prerequisites

A Redis host has been added.

Procedure

  1. Choose Protection > Big Data > Redis.
  2. On the Cluster page, click Register to register a Redis cluster.

    1. Enter the Redis cluster name.
    2. Click Add to add all nodes in the Redis cluster. Table 1 describes the related parameters.
      Table 1 Parameters for adding Redis nodes

      Parameter

      Description

      Hosts

      Select the registered Redis host.

      Client installation path

      Installation path of the Redis application in the production environment.

      Service IP Address

      Redis service IP address.

      Port

      Port number of the Redis instance.

      Authentication Method

      Configure the authentication mode for communication between the Redis cluster and the product. Set this parameter based on the authentication mode enabled for the Redis cluster.

      • OS Authentication: Select this authentication mode when Kerberos authentication is disabled for the Redis cluster.
      • Kerberos Authentication: Select this authentication mode when Kerberos authentication is enabled for the Redis cluster. The Kerberos protocol is used for identity authentication between the Redis cluster and the product.

      Kerberos

      This parameter is required only when Authentication Method is set to Kerberos Authentication.

      Select the created Kerberos authentication. When you register a Redis cluster for the first time, click Create and configure Kerberos authentication parameters. Table 2 describes the related parameters.

      SSL Channel Encryption

      You are advised to enable channel encryption when important or sensitive information exists in Redis. Channel encryption encrypts data during data transmission, which deteriorates performance.

      Set Kerberos authentication parameters as described in Table 2.

      Table 2 Kerberos authentication parameters

      Parameter

      Description

      Name

      User-defined Kerberos authentication name.

      Principal Name

      Principal name of Kerberos authentication.

      NOTE:

      Set this parameter to the human-machine username of the big data platform. The user must have the read and management permissions on the resources to be accessed.

      Configuration Mode
      Kerberos authentication mode, which must be the same as that configured on the Kerberos server. Two authentication modes are available for Kerberos:
      • Password: Use the principal name and password for identity authentication.
      • Keytab File: Use the principal name and keytab file for identity authentication.

      Configuration File

      The .conf configuration file (for example, krb5.conf) used during Kerberos authentication. Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the product.

      Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.

      Due to problems in Kerberos, if renew_lifetime exists in the configuration file, comment it out. Otherwise, the registration may fail. Example: 
      #renew_lifetime = 7d

      Password

      This parameter is required only when Configuration Mode is set to Password.

      Set this parameter to the password of Principal Name.

      Confirm Password

      Confirm the password of Principal Name.

      Keytab File

      The .keytab configuration file (for example, krb5.keytab) used during Kerberos authentication. This parameter is required only when Configuration Mode is set to Keytab File.

      Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.

      Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the product.

  3. Click OK.
