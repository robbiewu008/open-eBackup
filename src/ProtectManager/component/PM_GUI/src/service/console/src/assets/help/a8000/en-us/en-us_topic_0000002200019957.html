Step 1: Registering a Cluster

Before backing up and restoring a Kubernetes cluster, register the Kubernetes cluster with the OceanProtect. One set of OceanProtect supports a maximum of four Kubernetes clusters.

Prerequisites

  • Kubernetes configuration file has been obtained and saved to a local path on the maintenance terminal.

    The configuration file name is kubeconfig. For details about how to obtain the file, see Obtaining the kubeconfig Configuration File.

  • The management IP address or domain name of the Kubernetes cluster storage backend has been obtained.
  • A storage device user whose role is Administrator and login mode is RESTful has been created to connect Kubernetes storage backend resources to the OceanProtect. For details about how to create a user, see the Administrator Guide of the corresponding storage device version.

Procedure

  1. Choose Protection > Containers > Kubernetes FlexVolume.
  2. Click Register to register the Kubernetes FlexVolume cluster information.

    Table 1 lists the Kubernetes FlexVolume cluster registration information.
    Table 1 Kubernetes FlexVolume cluster registration information

    Parameter

    Description

    Name

    User-defined cluster name.

    Configuration File

    Select the obtained Kubernetes configuration file kubeconfig.

    Clients

    Select a client that can be used to back up Kubernetes.

  3. In the Storage Resources area, click Add.

    Only production storage resources such as OceanStor V3, OceanStor V5, or OceanStor Dorado storage resources can be added.

    Table 2 describes the storage resource parameters.

    Table 2 Storage resource information

    Parameter

    Description

    IP Address/Domain Name

    Management IP address or domain name of a storage resource.

    Port

    Port for accessing storage resources. The default value is 8088.

    Username

    Username and password of a system administrator for accessing storage resources.

    Password

  4. Click OK.

    After the registration is successful, information about the registered Kubernetes cluster is displayed on the Cluster tab page. The system automatically discovers namespaces and StatefulSets in the Kubernetes cluster and displays them on the Namespace and StatefulSet tab pages, respectively.
