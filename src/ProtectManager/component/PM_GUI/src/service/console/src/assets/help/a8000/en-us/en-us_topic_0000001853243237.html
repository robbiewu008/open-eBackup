ProtectAgent Installation

Before application backup and restoration, you must install ProtectAgent.

An agent host where ProtectAgent is installed is called a ProtectAgent host. The ProtectAgent host is used for process control and data transmission during backup and restoration. Before resource protection, prepare a host for installing ProtectAgent and install the software.

Table 1 shows the process of installing ProtectAgent on an agent host.

Table 1 ProtectAgent installation process

Task

Description/Operation Guide

Agent host preparation
  • The agent host is the Exchange host.

    ProtectAgent should be deployed on all hosts where the protected Exchange database resides.

  • For details about the resource requirements of the agent host, see "Check Before ProtectAgent Installation (ProtectAgent Deployed in the Production Environment)" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.
  • For details about network planning for the agent host, see "Network Planning for Agent Hosts" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.

Configuration before ProtectAgent installation

Before installing the agent, ensure that the firewall is disabled on the host where the agent is installed.

ProtectAgent installation
  • For details, see "Installing ProtectAgent" in the OceanProtect Appliance 1.5.0-1.6.0 ProtectAgent Installation Guide.

Configuration after ProtectAgent installation

For the Exchange Server, you need to install the latest Microsoft Visual C++ Redistributable for Visual Studio 2012 runtime library component on the agent host. For details about how to obtain the component, visit the download link.
