步骤2：（可选）生成最小权限Token

后续注册Kubernetes集群，当选择了Token认证时，才需要执行本节操作生成最小权限Token。如无特殊需求，可使用超级管理员权限的Token。

操作步骤

  1. 查询CCE集群工作节点已绑定的EIP。

    1. 登录ManageOne运营面。
    2. 依次选择“服务列表 > 云容器引擎CCE > CCE集群”，选择需要查看的Kubernetes集群名进入集群信息页面。
    3. 依次选择“集群 > 节点管理”，单击“节点”页签，查看CCE集群所有工作节点已绑定的EIP。

  2. 通过1查到的EIP地址，登录CCE集群任意一个节点。
  3. 创建集群角色资源。在Kubernetes生产集群创建文件“pm-k8s-sa-role.yaml”，并写入以下内容。

    apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: pm-k8s-sa-role
rules:
  - apiGroups: [ "*" ]
    resources: [ "*" ]
    verbs: [ "list", "create", "get" ]
  - apiGroups: [ "*" ]
    resources: [ "pods", "persistentvolumeclaims", "volumesnapshots" ]
    verbs: [ "delete" ]
  - apiGroups: [ "*" ]
    resources: [ "statefulsets", "replicasets", "deployments", "daemonsets", "jobs", "cronjobs" ]
    verbs: [ "patch" ]

  4. 在Kubernetes集群的后台界面执行kubectl apply -f pm-k8s-sa-role.yaml命令，创建ClusterRole。
  5. 创建服务账户。在Kubernetes生产集群创建文件“pm-k8s-sa.yaml”，并写入以下内容。

    apiVersion: v1
kind: ServiceAccount
metadata:
  name: pm-k8s-sa
  namespace: kube-system

  6. 在Kubernetes集群的后台界面执行kubectl apply -f pm-k8s-sa.yaml命令，创建ServiceAccount。
  7. 创建ClusterRoleBinding。在Kubernetes生产集群上创建文件“pm-k8s-cluster-role-binding.yaml”，并写入以下内容。

    apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: pm-k8s-cluster-role-binding
subjects:
- kind: ServiceAccount
  name: pm-k8s-sa      
  namespace: kube-system
roleRef:
  kind: ClusterRole
  name: pm-k8s-sa-role
  apiGroup: rbac.authorization.k8s.io

  8. 在Kubernetes集群的后台界面执行kubectl apply -f pm-k8s-cluster-role-binding.yaml命令，创建ClusterRoleBinding。
  9. 创建Secret保存Token信息。在Kubernetes生产集群上创建文件“pm-k8s-secret.yaml”，并写入以下内容。

    apiVersion: v1
kind: Secret
type: kubernetes.io/service-account-token
metadata:
  namespace: kube-system
  name: pm-k8s-secret
  annotations:
    kubernetes.io/service-account.name: pm-k8s-sa

  10. 在Kubernetes集群的后台界面执行kubectl apply -f pm-k8s-secret.yaml命令，创建Secret。
  11. 获取最小权限Token。在生产集群上执行kubectl describe secret/pm-k8s-secret -n kube-system命令，回显中“token”字段的值即为所需的最小权限Token。
