Performing Periodic Backup

After the clusters, hosts, or VMs to be protected are associated with a specified SLA, the system periodically backs up the clusters, hosts, or VMs based on the SLA.

Context

If a VM whose SLA is inconsistent with that of the cluster or host, the VM will not initiate a backup job.

Procedure

  1. Choose Protection > Virtualization > CNware.
  1. Select the clusters, VMs, or hosts to be protected.

    1. In the left pane, select the registered CNware virtualization platform.
    2. In the right pane, click the Clusters, Hosts, or VMs tab, and select one or more clusters, hosts, or VMs to be protected.
      • If you select a host, the system will protect all VMs on the host.
      • If you select a cluster, the system will protect all VMs in the cluster.

        A maximum of 1,000 CNware VMs can be protected in a batch.

  2. Click Protect.
  3. If you select a VM, you can determine whether to enable automatic protection for disks. If automatic protection for disks is enabled, all disks (including newly detected disks) of the VM will be protected. If automatic protection for disks is disabled, manually select disks to be protected.
  4. Choose a preset or user-defined SLA.

    You can also click Create to create an SLA.

  5. Configure advanced settings based on the protected object.

    • If you select one or more VMs to protect, select an agent host. The system automatically selects an agent host by default.
    • If you select one or multiple hosts or clusters to protect, you can view or configure the following parameters:
      1. Choose whether to overwrite the existing VM SLAs.
        • Overwrite existing VM SLAs: If this option is selected, VMs in a cluster or host use the same SLA as the cluster or host.
        • Do not overwrite existing VM SLAs: This option is selected by default. If this option is selected, the existing SLAs of VMs in a cluster or host will not be overwritten.

        You can configure an SLA overwriting policy only when you create protection for a cluster or host for the first time. The policy cannot be modified later.

      2. You can choose one or more options from the following to apply the SLA:
        • Applies the SLA policy to all VMs without SLA associated on the selected host.
        • Applies the SLA policy to newly created VMs.

          After the virtualization environment is scanned, the newly discovered VMs are associated with the same SLA as the host.

      3. You can specify the VMs to be excluded or included in the protection plan.

        You can specify the VMs to be excluded or included based on VM names.

        The option supports exact match and fuzzy match of VM names. Fuzzy match rules include prefix match (for example, abc*), suffix match (for example, *abc), and middle match (for example, *abc*).

  6. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.
