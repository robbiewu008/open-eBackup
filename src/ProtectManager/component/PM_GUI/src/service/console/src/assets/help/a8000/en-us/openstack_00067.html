Managing the OpenStack Cloud Platform

After the OpenStack cloud platform is registered with the OceanProtect, you can perform OpenStack cloud platform operations on the management page.

You need to log in to the management page and choose Protection > Cloud Platforms > OpenStack to go to the OpenStack page. On the left of the OpenStack page, select the target cloud platform and perform related operations.

Table 1 describes the related operations.

Table 1 OpenStack-related operations

Operation

Description

Navigation Path

Viewing OpenStack registration information

Scenario

This operation enables you to view OpenStack registration information, such as the domain name, Keystone V3 address, and agent host.

Choose More > Environment Info.

Modifying OpenStack registration information

Scenario

This operation enables you to change the name of OpenStack registration information. If the Keystone V3 address, username, password, or certificate of the registered OpenStack is changed, perform this operation to change the corresponding information.

Note

Modify OpenStack registration information when no backup or restoration job is running. Otherwise, the job will fail.

Choose More > Modify Registration.

Table 1 describes the parameters of OpenStack registration information.

Canceling OpenStack registration

Scenario

If a cloud server in a cloud platform no longer needs to be protected or the cloud platform will not be used as the restoration target, perform this operation to delete the cloud platform.

Note

If a backup job is being performed, the registration cannot be canceled. You can perform deregistration after the job is complete.

Choose More > Unregister.

Scanning the OpenStack cloud platform

Scenario

The OceanProtect automatically scans the OpenStack environment every 24 hours. If any project or cloud server on the registered OpenStack cloud platform change, the change will be automatically updated to the OceanProtect after the scan. This operation enables you to immediately update the changed project or cloud server to the OceanProtect.

Note

If you do not select to apply the SLA policy to newly created cloud servers upon periodic backup creation, the system will not associate newly discovered cloud servers with any SLA after the scan.

Choose More > Resource Scan.

Testing OpenStack connectivity

Scenario

Manual connectivity test of the registered OpenStack environment is used to check the connectivity between the OpenStack environment and the agent host, and between the OpenStack environment and the production environment.

Choose More > Test Connectivity.
