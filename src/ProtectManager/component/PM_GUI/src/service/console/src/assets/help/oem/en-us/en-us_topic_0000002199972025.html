Step 3: Registering a Hive Cluster

Before Hive backup and restoration, register the Hive cluster with the product.

Prerequisites

  • Before registering a Hive cluster, ensure that you have obtained the following files. This section uses FusionInsight Manager 8.2.0 as an example. Operations may vary on different platforms. For details, see the product documentation of the corresponding big data platform.
    1. Log in to FusionInsight Manager using a browser.
    2. Choose Cluster > Name of the target cluster > Dashboard > More > Download Client. On the displayed page, select Configuration Files Only to download the Hive and HDFS client configuration file packages.
    3. Decompress the downloaded Hive client configuration file package to obtain the following files:
      • ../Hive/config/hiveclient.properties file
      • ../Hive/config/hive-site.xml file
      • ../Hive/config/core-site.xml file
    4. Decompress the downloaded HDFS client configuration file package to obtain the ../HDFS/config/hdfs-site.xml file.

    The following describes how to obtain the jaas.conf and hive-site.xml configuration files when the big data platform is Transwarp Data Hub (TDH). This part uses TDH 8.0.2 as an example. Operations may vary with versions. For details, see the Transwarp Data Hub product documentation.

    1. Use a browser to log in to TDH.
    2. Choose Dashboard > Cluster. On the Service List tab page, click the Hive cluster name configured in TDH.
    3. In the upper right corner of the Hive cluster page, click the icon and click Download Service Configurations to obtain jaas.conf and hive-site.xml from the downloaded package.
    4. Choose Dashboard > Cluster. On the Service List page, click the HDFS cluster name configured in TDH.
    5. In the upper right corner of the HDFS cluster page, click the icon and click Download Service Configurations to obtain core-site.xml and hdfs-site.xml from the downloaded package.
    6. Delete the following information from the core-site.xml file:
      <property>
    <name>hadoop.security.group.mapping.ldap.bind.password.file</name>
    <value>/etc/hdfs1/conf/ldap-conn-pass.txt</value>
</property>

Procedure

  1. Choose Protection > Big Data > Hive.
  2. On the Clusters page, click Register to register a Hive cluster.

    Table 1 describes the Hive cluster registration information.

    Description

    Name

    Customize a Hive cluster name.

    HiveServer2 Link

    Open the hiveclient.properties file obtained in Prerequisites. The value of zk.quorum is the configured value.

    NOTE:
    • If the big data platform is TDH, obtain the attribute value transwarp.docker.inceptor from the hive-site.xml file and replace the domain name in the value with the IP address of the corresponding production node. Input example: 127.0.0.0:8000

    Hive Version

    Hive version number.

    The following uses FusionInsight Manager 8.2.0 as an example to describe how to check the Hive version:

    1. Log in to FusionInsight Manager using a browser.
    2. Choose Cluster > Hive to view the Hive version.
    NOTE:

    If the big data platform is TDH and the big data platform version is 8.X or 6.X, the Hive version is 1.2.0.

    Authentication Mode

    Configure the authentication mode for communication between the Hive cluster and the product. Set this parameter based on the authentication mode configured for the Hive cluster.

    • Simple Authentication: Select this authentication mode when Kerberos authentication is disabled for the Hive cluster. In this authentication mode, usernames are used for authentication, which is less secure.
    • Kerberos Authentication: Select this authentication mode when Kerberos authentication is enabled for the Hive cluster. The Kerberos protocol is used for identity authentication between the Hive cluster and the product.

    Username

    This parameter is required only when Authentication Mode is set to Simple Authentication.

    NOTE:
    • Set this parameter to the human-machine username of the big data platform.
    • The user must have the SHOW_DATABASE permission on the database to be backed up, SELECT permission on the table to be backed up, CREATE and SHOW_DATABASE permissions on the database to be restored, and CREATE, DROP, ALTER, and SELECT permissions on the table to be restored.
    • The user must have the following permissions: read, write, and execution permissions on the temporary metadata storage path specified during creation of a backup set; read and execution permissions on the directory where the data of database tables to be backed up is located as well as the lower-layer files and directories; read, write, and execution permissions on the temporary metadata storage path specified during restoration; read, write, and execution permissions on the directory where the data to be restored to the target database table is located; read, write, and execution permissions on the directories where the table metadata and table data are located.

    Kerberos

    This parameter is required only when Authentication Mode is set to Kerberos Authentication.

    Select the created Kerberos authentication. When you register a Hive cluster for the first time, click Create and configure Kerberos authentication parameters. For details about the parameters, see Table 2.

    HiveServer2 Principal

    This parameter is required only when Authentication Mode is set to Kerberos Authentication.

    Open the hiveclient.properties file obtained in Prerequisites. The value of principal is the configured value.

    NOTE:

    If the big data platform is TDH, the value of this parameter is the value of principal in the Client module in the jaas.conf file.

    Clients

    Select the host where the client is installed for data protection.

    NOTE:

    Do not allocate a host where the client is installed to multiple big data clusters with different Kerberos authentication configurations or authentication modes. Otherwise, the backup or restoration job will fail.

    Certificate Verification

    This parameter is required only when Authentication Mode is set to Kerberos Authentication. The Certificate Verification function is enabled by default.

    If you want to encrypt data during communication between the product and the big data platform for secure communication, import the certificate and enter the certificate password.

    Import the JKS certificate generated in Step 2: (Optional) Generating and Obtaining a Certificate. The certificate password is the keystore password set in Step 2: (Optional) Generating and Obtaining a Certificate.

    NOTE:
    • This configuration item can be enabled only when the big data platform is Cloudera CDH.
    • After the Certificate Verification function is disabled, security risks may exist in the communication between the system and the big data platform. Exercise caution when performing this operation.

    Security Protocol

    This parameter is required only when Authentication Mode is set to Kerberos Authentication. The Security Protocol function is enabled by default.

    Whether to enable the TLS protocol.

    • If the TLS protocol is disabled, TLS 1.2 is used by default.
    • If the TLS protocol is enabled, TLS 1.2 or later is supported.
      NOTE:
      • This configuration item can be enabled only when the big data platform is Cloudera CDH.
      • After the Security Protocol function is disabled, security risks may exist in the communication between the system and the big data platform. Exercise caution when performing this operation.

    Set Kerberos authentication parameters. For details about related parameters, see Table 2.

    Table 2 Kerberos authentication parameters

    Parameter

    Description

    Name

    User-defined Kerberos authentication name.

    Principal Name

    Principal name of Kerberos authentication.

    NOTE:
    • Set this parameter to the human-machine username of the big data platform.
    • The principal must have the SHOW_DATABASE permission on the database to be backed up, SELECT permission on the table to be backed up, CREATE and SHOW_DATABASE permissions on the database to be restored, and CREATE, DROP, ALTER, and SELECT permissions on the table to be restored.
    • The principal must have the following permissions: read, write, and execution permissions on the temporary metadata storage path specified during creation of a backup set; read and execution permissions on the directory where the data of database tables to be backed up is located as well as the lower-layer files and directories; read, write, and execution permissions on the temporary metadata storage path specified during restoration; read, write, and execution permissions on the directory where the data to be restored to the target database table is located; read, write, and execution permissions on the directories where the table metadata and table data are located.

    Configuration Mode
    Kerberos authentication mode, which must be the same as that configured on the Kerberos server. Two authentication modes are available for Kerberos:
    • Password: Use the principal name and password for identity authentication.
    • Keytab File: Use the principal name and keytab file for identity authentication.
      NOTE:

      If the big data platform is MapReduce Service (MRS), Hive cluster registration may fail when password authentication is used due to problems of some MRS versions. Therefore, the keytab file authentication mode is recommended.

    Configuration File

    The .conf configuration file (for example, krb5.conf) used during Kerberos authentication. Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the management page.

    Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.

    If renew_lifetime exists in the configuration file, comment it out. Otherwise, the registration may fail. For example: 
    #renew_lifetime = 7d

    Password

    This parameter is required only when Configuration Mode is set to Password.

    Set this parameter to the password of Principal Name.

    Confirm Password

    Confirm password of Principal Name.

    Keytab File

    The .keytab configuration file (for example, krb5.keytab) used during Kerberos authentication. This parameter is required only when Configuration Mode is set to Keytab File.

    Take MRS as an example. For details about how to obtain the configuration file, see "Exporting an Authentication Credential File" in the MapReduce Service Product Documentation of the corresponding version.

    Contact the Kerberos server administrator to obtain the file from the Kerberos server and click to upload it to the management page.

  3. Click OK.
