恢复FusionCompute虚拟机磁盘

当需要对已备份的虚拟机指定磁盘恢复时，可以参考本节恢复虚拟机的磁盘至原虚拟机或新虚拟机。

背景信息

本产品支持使用备份副本、复制副本（不支持恢复至原位置），支持恢复至原位置或新位置。

操作步骤

  1. 选择“数据利用 > 副本数据 > 虚拟化 > FusionCompute”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在的行，单击“更多 > 磁盘恢复”。
  5. 勾选要恢复的磁盘，单击“选择”。
  6. 恢复虚拟机磁盘到原位置或新位置。

    • 恢复虚拟机磁盘到原位置。
      1. 选择恢复至“原位置”。
      2. “目标计算位置”默认为虚拟机原位置。
      3. 选择副本恢复时使用的代理主机。

        当用户未选择时，系统会从注册资源时配置的代理主机中选择1到多个代理主机执行恢复。

      4. 选择恢复后是否自动开机。
      5. 选择恢复前是否执行副本校验。

        开启后，副本恢复前将执行校验，校验副本的完整性。该操作将影响恢复性能。默认关闭。当未生成副本校验文件时，不支持开启此功能。

      6. 单击“确定”。
    • 恢复虚拟机磁盘到新位置。
      1. 选择恢复至“新位置”。
      2. 设置恢复虚拟机到新位置的位置信息。
        相关参数说明如表1所示。
        表1 新虚拟机信息

        参数

        说明

        目标计算位置

        选择备份磁盘恢复到的虚拟化环境和虚拟机。

        说明：

        目标虚拟机与源虚拟机的操作系统类型必须相同，如：同是Windows OS或同是Linux OS。

        存储位置

        为备份磁盘选择恢复到的数据存储。

        • 相同数据存储：所有磁盘恢复到相同的数据存储。
        • 不同数据存储：分别为各磁盘选择恢复到的数据存储。
      3. 选择副本恢复时使用的代理主机。

        当用户未选择时，系统会从注册资源时配置的代理主机中选择1到多个代理主机执行恢复。

      4. 选择恢复后是否自动开机。
      5. 选择恢复前是否执行副本校验。

        开启后，副本恢复前将执行校验，校验副本的完整性。该操作将影响恢复性能。默认关闭。当未生成副本校验文件时，不支持开启此功能。

  7. 单击“确定”。
