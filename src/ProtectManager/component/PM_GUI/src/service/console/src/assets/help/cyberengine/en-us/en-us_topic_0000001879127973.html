Restoring the Current System Management Data

This section describes how to restore management data to the system.

Precautions

  • If you use a backup copy at a certain point in time to restore the management data, the system management data will be overwritten. If an ongoing detection job exists in the system, the detection job will be interrupted. Therefore, exercise caution when performing this operation.
  • If the backup at a certain point in time is used to restore management data, ransomware detection snapshots generated after the point in time are not displayed on the page. Consequently, these ransomware detection snapshots cannot be used for restoration.
  • From the beginning of the management data restoration, you need to wait for about 5 minutes until the services become normal.
  • After the system is upgraded, the management data of the earlier version cannot be restored.
  • If the type of management data backup is Upgrade Backup (management data backup automatically performed by the system after the upgrade), you do not need to enter the data encryption password because the upgrade backup only backs up data locally and uses the key management system of the OceanCyber Appliance to encrypt the backup data.

Procedure

  1. Choose System > Settings > Management Data Backup.
  2. In the Operation column on the right of the management data backup to be restored, click More > Restore.
  3. Enter the data encryption password set during backup policy configuration and click OK.
  4. Read the warning carefully, select I have read and understand the consequences associated with performing this operation and click OK.
