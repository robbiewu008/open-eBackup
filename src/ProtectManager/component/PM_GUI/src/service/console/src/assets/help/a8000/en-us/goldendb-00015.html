Step 6: Performing Backup

Before performing backup, you need to associate the resources to be protected with a specified SLA policy. The system protects the resources based on the SLA policy and periodically performs backup jobs based on the SLA policy. You can perform a backup job immediately through manual backup.

Precautions

  • Copies generated by manual backup are retained for the period defined in the SLA.
  • Before the backup, do not add, modify, or delete data in the database of the management node.
  • If no log backup is defined in the SLA, log backup cannot be manually executed.

Procedure

  1. Choose Protection > Databases > GoldenDB.
  2. On the Instance tab, select the instance to be protected and click Protect.

    You can select multiple databases.

  3. Select an SLA.

    You can also click Create to create an SLA.

  4. Set advanced backup parameters.

    Table 1 describes the related parameters.
    Table 1 Advanced backup parameters

    Parameter

    Description

    Script to Run Before Backup

    You can configure Script to Run Before Backup, Script to Run upon Backup Success, and Script to Run upon Backup Failure based on site requirements. When the message "ProtectAgent started executing the prerequisite task." is displayed during a backup job, the configured script is executed in the corresponding scenario.

    Enter the absolute path and name of the script and ensure that the user who logs in to the database host has the permission to run the script.
    • Windows OSs support .bat scripts, for example, C:\script.bat.
    • Non-Windows OSs support .sh scripts, for example, /opt/script.sh.
      NOTE:

      If Script to Run upon Backup Success is configured, the backup job status is displayed as Successful on the OceanProtect Appliance even if the script fails to be run. Check whether the job details contain information indicating that a post-processing script fails to be run. If yes, correct the script in a timely manner.

  5. Click OK.

    If the current system time is later than the first backup start time of the SLA, you can select to perform backup immediately after the backup SLA is associated or wait for the next scheduling based on the backup policy set in the SLA.

  6. Perform manual backup.

    If you want to execute a backup job immediately, perform manual backup through the following operations. Otherwise, skip this step.
    1. Choose More > Manual Backup in the row of the instance to be backed up.
    2. Set the name of the copy generated during manual backup.
    3. If this parameter is left unspecified, the system sets the copy name to backup_Timestamp by default.
    4. Select a protection policy: full backup, incremental backup, or log backup.
    5. Click OK.
