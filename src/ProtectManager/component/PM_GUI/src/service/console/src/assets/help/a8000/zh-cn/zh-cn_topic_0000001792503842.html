创建周期性备份（废弃）

将需要保护的资源关联到指定SLA策略，系统将对这些资源按照SLA策略进行保护。

注意事项

全量备份失败后，如果下一次备份是日志备份，由于日志备份依赖上一次成功的全量备份副本，该日志备份任务将失败。此时，请手动执行一次全量备份。

操作步骤

  1. 选择“保护 > 数据库 > DB2”。
  2. 在“数据库”页签，选择需要保护的数据库，单击“保护”。

    您也可以勾选多个数据库进行批量保护。

  3. 选择SLA。

    您也可以单击“创建”，创建新的SLA。

  4. 配置数据库备份高级参数。

    相关参数说明如表1所示。
    表1 数据库备份高级参数说明

    备份前执行脚本

    您可以根据实际需求，配置“备份前执行脚本”、“备份成功执行脚本”和“备份失败执行脚本”，当备份任务事件中出现“开始执行前置任务”提示后，配置的脚本会在对应场景下执行。不同操作系统对脚本的格式要求不同，具体请参考不同操作系统脚本格式要求。

    此处请输入脚本的绝对路径和名称，并确保登录数据库主机的用户对该脚本有可执行权限。
    • Windows OS支持的脚本类型为“.bat”，如输入：C:\script.bat。
    • 非Windows OS支持的脚本类型为“.sh”，如输入：/opt/script.sh。
      说明：

      当配置了“备份成功执行脚本”时，即使该脚本执行失败，管理界面上也会显示备份任务的状态为“成功”。请您留意任务详情中是否有后置脚本执行失败的相关提示，如有请及时修正脚本。

  5. 单击“确定”。

    如果当前系统时间已晚于该SLA首次备份的开始时间，则关联备份SLA后可以选择在当前立即执行备份或者根据SLA中设置的备份策略等待下次调度。
