Step 3: Creating a Replication Network Logical Port

When a storage device is OceanStor Dorado 6.1.3 or later, or OceanStor 6.1.3 or later, NAS file system backup can be implemented using the replication function between the production storage and product. In this case, a logical port for the replication network needs to be created separately on the storage device and the product for remote replication data transmission. NAS share backup is implemented if other types of storage devices are used, and a logical port is not required for the replication network. In this case, skip this section.

Prerequisites

  • The replication network IP addresses of the primary and secondary storage systems have been planned and are in the same network segment as the backup network IP addresses.
  • A logical port whose Role is Management + service and Data Protocol is NFS + CIFS has been created on DeviceManager of the product. For details, see "Creating a Logical Port" in the Installation Guide.

Procedure

The following describes how to create a logical port for the replication network on the product. This also applies to the creation on a storage device.

  1. Log in to DeviceManager.
  2. Choose Services > Network > Logical Ports.
  3. Click Create to create a logical port.

    Table 1 describes the related parameters.

    Table 1 Parameters related to a logical port

    Parameter

    Description

    Name

    Name of a replication network logical port, which can be customized.

    Role

    Select Replication.

    IP Address Type

    IP address type of a replication network logical port, which can be IPv4 or IPv6.

    IP Address

    IP address of a replication network logical port

    Subnet Mask

    Configure a subnet mask if the IP address type is IPv4.

    Prefix

    Configure a prefix if the IP address type is IPv6.

    Gateway

    Gateway used for the IP address of a replication network logical port

    Port Type

    Users can create a logical port for the replication network by selecting Ethernet port, bond port, or VLAN.

    By default, the port used for replication has been bonded during the initialization of the product. Therefore, the bond port needs to be selected if there is no other operation (such as deleting the bond port).

    Home Port

    Ethernet port, bond port, or VLAN to which the created logical port belongs.

    Select any one of the ports after the system has filtered all ports used for the replication network.

  4. Click OK.
  5. Repeat the preceding steps to configure a logical port for each home port.
