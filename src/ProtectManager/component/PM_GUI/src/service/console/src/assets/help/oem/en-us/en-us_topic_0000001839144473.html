Authorizing a Local Cluster Data Protection Administrator

After a managed cluster is added to a local cluster, the system administrator of the local cluster has the system administrator permissions of the managed cluster. This section describes how to grant the data protection administrator permission of a managed cluster to the data protection administrator of a local cluster.

Precautions

  • Permissions cannot be granted to multiple local cluster data protection administrators.
  • A data protection administrator of a local cluster can be granted data protection administrator permissions of multiple managed clusters. For each managed cluster, the permissions of only one data protection administrator can be granted to the data protection administrator of the local cluster.

Procedure

  1. Choose System > Infrastructure > Cluster Management.
  1. In the multi-domain cluster area, choose More > Authorize in the row that contains the desired managed cluster.
  2. In the Data Protection Administrator area, select the local cluster data protection administrator to be authorized.
  3. In the Authorization Information area, enter the username and password of the data protection administrator of the managed cluster.
  4. Click OK.
  5. In the displayed dialog box, confirm the information and click OK.

    After authorization is complete, you can click the authorized user of the managed cluster in the cluster list to view the authorization information.
