Downloading a Client Software Package

If you want to manually install the client on the host, first download the client software package by referring to this section.

Precautions

The GUI varies depending on the version.

Procedure

  1. Choose Protection > Clients > Clients.
  2. Click Software Package Management and set related parameters.

    Table 1 describes the parameters.

    Table 1 Downloading a client software package

    Parameter

    Description

    Mode

    Choose Download.
    Agent Type
    NOTE:

    This parameter is available only in 1.5.0.
    Select an agent type based on the type of the protected resource.
    • VMware backup agent: Select this type when the protected resource is a VMware VM.
    • General backup agent: Select this type for protected resources other than VMware.
    • SAN Client agent: Select this type if the protected resource is DB2 and the AIX or Solaris host uses a Fibre Channel network.
    Software Package Type
    NOTE:

    This parameter is available only in 1.6.0 and later versions.
    Type of the client software package.
    • Client: Applies to all resources to be protected.
    • SAN Client: Select this type only if the following conditions are met: the protected resource is DB2; a Fibre Channel network is used in the AIX production environment; the SAN Client is required to implement backup.

    OS

    OS type of the host where the client is installed.
    Download Package Type
    NOTE:

    This parameter is available only in 1.6.0 and later versions.

    When OS is set to Linux, select the type of the package to be downloaded. The supported types are zip and tar.

    Application Type

    If Agent Type is set to General backup agent or Software Package Type is set to Client, select the application type that can be protected by the client.

    Sharing

    After this function is enabled, the client is applied to all data protection administrators.

    This parameter can be configured only when Software Package Type is set to Client, OS is set to Linux, and Application Type is set to HDFS, HBase, Hive, Huawei Cloud Stack, Huawei Cloud Stack GaussDB, Object Storage, or GaussDB (DWS).

    Private Key Password

    User-defined private key password. Enter a password according to the complexity requirements prompted.

    The private key password is used to encrypt the private key file in the client software package. The private key file is used for authentication between the host where the client is installed and the product. Keep the private key password secure for future use during installation.

    NOTE:
    • If OS is set to AIX,or Solaris, special characters ^`{_|}~ cannot be contained in the entered private key password. Otherwise, the installation will fail when the software package is used to install the client.
    • After you enter the private key password, the product uses the password to encrypt the private key file in the client software package.
    • If you have imported the client certificate by referring to the Security Configuration Guide before downloading the client software package, you can enter the private key password that is the same as that of the client certificate or customize the private key password.

    Confirm Private Key Password

    Enter the private key password again.

  3. Click OK.

    Obtain the downloaded client software package.

    If you fail to download the client software package using Internet Explorer 11 on the Windows Server, refer to What Should I Do If I Fail to Download the Client Software Package Using Internet Explorer 11 on the Windows Server?.
