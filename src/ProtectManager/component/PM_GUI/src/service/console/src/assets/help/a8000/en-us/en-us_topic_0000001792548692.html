(Optional) Step 4: Enabling Backup Link Encryption

You need to enable backup link encryption if required.

Prerequisites

Ensure that the NFS Kerberos service has been configured for the storage system. Otherwise, backup jobs may fail after backup link encryption is enabled. For details about how to configure the NFS Kerberos service, see "(Optional) Configuring the NFS Kerberos Service" in the OceanProtect Appliance 1.5.0-1.6.0 Installation Guide. When adding users to the AD domain, add two users whose usernames are rdadmin and the username used for resource registration in Step 3: Registering the GaussDB T Database. If the user already exists in the AD domain, you do not need to add it.

Precautions

After backup link encryption is enabled, the ticket between the agent host and the KDC domain server has a validity period. If the ticket expires, backup jobs may fail. You are advised to prolong the validity period of the ticket in the Kerberos policy on the KDC domain server or configure the ticket to never expire. If the permanent validity period is not configured, you must run the kinit command on the agent host to re-obtain tickets for users added to the AD domain before the tickets are about to expire. For details, see "Configuring the Client" in the OceanProtect Appliance 1.5.0-1.6.0 Installation Guide.

Procedure

  1. Choose System > Security > Data Security.
  2. In the Encryption Settings area, click Modify on the right of the page and enable Backup Link Encryption.
  3. Click Save.
