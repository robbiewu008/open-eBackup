GBase Restoration Job Fails Due to Inconsistent Database Topology Structures

Symptom

Failed to restore the GBase incremental copy to a new location. The error code is 1577212932.

Possible Cause

The database topology of the backup copy is inconsistent with that of the target location for restoration.

Procedure

  1. Log in to the backup copy and the database in the target location. Run the following command to switch to the database view:

    su - Database username
    gccli

  2. Run the following command in the two databases to view the node information:

    show nodes;

    View the values of primary part and duplicate part in the command output and check whether the parameter values of the two databases are consistent.

    • If they are inconsistent, the two database topologies are different. You need to ensure that the topologies of the two databases are the same.
    • Otherwise, contact Huawei technical support.
