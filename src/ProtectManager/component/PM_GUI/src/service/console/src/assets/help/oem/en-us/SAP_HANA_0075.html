Managing Instances

The product supports authorization, reclamation, modification, and deletion for resources of instances.

You need to log in to the management page, choose Protection > Applications > SAP HANA, and find the target instance on the Instance tab page.

Table 1 describes the related operations.

Table 1 Operations related to database protection plans

Operation

Description

Navigation Path

Testing connectivity

Scenario

This operation enables you to test the connectivity between the product and a database.

In the row of the target instance, choose More > Test Connectivity.

Modification

Scenario

This operation allows you to modify the database registration information after the instance name, host, authentication mode, or log backup configuration is changed.

In the row of the target instance, choose More > Modify.

Deletion

Scenario

This operation enables you to delete an instance if you no longer need to protect the database in the instance.

Note

The instance to which a database associated with an SLA belongs cannot be deleted.

In the row of the target instance, choose More > Delete.
Adding a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to quickly filter and manage resources.
Choose More > Add Tag.
NOTE:
  • On the displayed Add Tag dialog box, you can select existing tags or click Create to create a tag.
  • To modify or delete a tag, click Go to Tag Management on the Add Tag page to go to the Tag Management page.
Removing a tag
NOTE:

Only 1.6.0 and later versions support this function.

Scenario

This operation enables you to remove the tag of a resource when it is no longer needed.

Choose More > Remove Tag.

Note: The procedure for restoring the SAP HANA database is the same as that in Restoration.
