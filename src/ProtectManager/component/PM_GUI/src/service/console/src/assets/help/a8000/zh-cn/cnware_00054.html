即时恢复CNware虚拟机

当需要对已备份的虚拟机进行即时恢复时，可以参考本节即时恢复虚拟机至原位置或新位置。

背景信息

支持使用备份副本、复制副本、即时挂载产生的副本（副本需要已从虚拟机卸载）进行即时恢复，支持恢复至原位置或新位置。

  • 恢复至原位置
    • 恢复到原位置时，系统将在原虚拟机所在路径下（即原虚拟机所属的集群或主机）创建一个新的虚拟机，将备份数据全量恢复到新虚拟机。
    • 恢复至原位置时，不支持使用复制副本、即时挂载产生的副本进行恢复。
  • 恢复至新位置

    恢复至新位置时，系统进行全量恢复，将备份数据恢复到指定的新路径下，即由用户指定的新虚拟机所在的主机/集群和对应的存储池。

操作步骤

  1. 选择“数据利用 > 副本数据 > 虚拟化 > CNware”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在的行，单击“更多 > 即时恢复”。
  5. 恢复虚拟机至原位置或新位置。

    • 恢复虚拟机至原位置。
      1. 选择恢复至“原位置”。
      2. 设置新虚拟机的名称。

        新虚拟机不能与所在集群或主机下的其他虚拟机同名。

      3. 设置高级配置信息。

        相关参数说明如表1所示。

        表1 原位置高级配置信息

        参数

        说明

        代理主机

        选择副本恢复使用的代理主机。

        恢复后自动开机

        开启后，恢复后，云服务器将自动开机。默认关闭。

        删除原虚拟机

        开启选项后，会在恢复成功后删除原虚拟机。

        恢复前执行副本校验

        开启后，副本恢复前将执行校验，校验副本的完整性。该操作将影响恢复性能。默认关闭。当未生成副本校验文件时，不支持开启。

        网络位置

        选择虚拟机网卡要恢复到的端口组名称。

        启用网络适配器

        该选项用于设置恢复后的虚拟机网卡的连接状态。默认不启用，所有虚拟机网卡都处于未启用状态，虚拟机恢复后，需要登录CNware管理界面设置虚拟机网卡连接；启用后，可能导致新虚拟机网络与原虚拟机网络冲突，请恢复虚拟机后，检查新虚拟机网络。
      4. 单击“确定”。

        即时恢复后的虚拟机将自动开机。

    • 恢复虚拟机至新位置。
      1. 选择恢复至“新位置”。
      2. 设置恢复虚拟机到新位置的名称和位置信息。
        相关参数说明如表2所示。
        表2 新位置信息

        参数

        说明

        新虚拟机名称

        自定义新虚拟机的名称。

        目标位置

        新虚拟机恢复的目标主机。
      3. 选择恢复到目标存储池的策略。
        • 把所有虚拟磁盘迁移到同一个存储池：需要选择存储池并设置目标置备类型。
        • 把虚拟磁盘迁移到指定的存储池：需要为每块虚拟磁盘选择迁移的目标存储池并设置目标置备类型。
      4. 设置高级配置信息。

        相关参数说明如表3所示。

        表3 新位置高级配置信息

        参数

        说明

        代理主机

        选择副本恢复使用的代理主机。

        恢复后自动开机

        开启后，恢复后，云服务器将自动开机。默认关闭。

        恢复前执行副本校验

        开启后，副本恢复前将执行校验，校验副本的完整性。该操作将影响恢复性能。默认关闭。当未生成副本校验文件时，不支持开启。

        网络位置

        启用网络适配器

        该选项用于设置恢复后的虚拟机网卡的连接状态。默认不启用，所有虚拟机网卡都处于未启用状态，虚拟机恢复后，需要登录CNware管理界面设置虚拟机网卡连接；启用后，可能导致新虚拟机网络与原虚拟机网络冲突，请恢复虚拟机后，检查新虚拟机网络。
      5. 单击“确定”。

        即时恢复后的虚拟机将自动开机。
