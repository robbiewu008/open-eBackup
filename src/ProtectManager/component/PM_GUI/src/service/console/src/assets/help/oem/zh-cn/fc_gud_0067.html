整机恢复到新位置失败，错误详情包含"ErrorCode: 10300005"

现象描述

FusionCompute虚拟机恢复到新位置失败，错误详情中显示：{"ErrorCode": "10300005", "ErrorDesc":"The entered parameter VM CPU limit is invalid. Please enter a valid value."}。

可能原因

备份虚拟机设置了CPU限制频率，且备份虚拟机所在主机和恢复目标位置主机的CPU频率不同，导致CPU限制频率的下限值不同。

方法一

重新设置FusionCompute虚拟机待恢复至的“目标计算位置”，请选择为与备份虚拟机所在主机相同CPU频率的目标主机。

处理步骤：

  1. 选择“数据利用 > 副本数据 > 虚拟化 > FusionCompute”。
  2. 支持以虚拟机维度或以副本维度查找副本，本节以虚拟机维度为例进行说明。

    在“资源”页签，根据虚拟机名称查找到需要恢复的虚拟机，并单击虚拟机名称。

  3. 在“副本数据”页签，依次选择年、月、天找到副本。

    当时间上显示，即表示该月、该天存在副本。

  4. 在副本所在的行，单击“更多 > 恢复”。
  5. 恢复虚拟机至相同CPU频率的目标主机。

    1. 选择恢复至“新位置”。
    2. 设置恢复虚拟机到新位置的位置信息，其中，“目标计算位置”选择和备份虚拟机所在主机相同CPU频率的目标主机。
    3. 按需设置剩余参数。

  6. 单击“确定”。

方法二

在FusionCompute管理系统中选择目标位置主机创建虚拟机，后使用磁盘恢复功能恢复磁盘。

处理步骤：

  1. 使用浏览器，登录FusionCompute。

    登录地址：https://FusionCompute的VRM节点的浮动IP地址:8443

  2. 在FusionCompute管理系统上创建虚拟机。该虚拟机用于保存待恢复FusionCompute虚拟机的磁盘数据。

    1. 选择“资源池 ”。
    2. 单击“创建虚拟机”。其中，创建虚拟机的计算资源信息和数据存储信息选择为需要恢复FusionCompute虚拟机的目标位置。
    3. 单击“确定”。

  3. 使用FusionCompute虚拟机磁盘恢复功能，将磁盘信息恢复至2创建的虚拟机中。磁盘恢复的具体操作步骤参见恢复FusionCompute虚拟机磁盘。
